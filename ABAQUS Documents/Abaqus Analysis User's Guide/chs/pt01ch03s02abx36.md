# 3.2.36 将 Abaqus 数据转换为 MSC.ADAMS 模态中性文件





**产品：**Abaqus/Standard

##### **参考**

- [『Abaqus 执行程序：概述』，第 3.1.1 节](pt01ch03s01abo02.md)
- [『将 Abaqus 数据转换为用于 MSC.ADAMS 分析的模态中性文件格式』，Abaqus 例题指南第 15.1.7 节](../exa/exa-link.md#exa-pst-adams)

### 概述

MSC.Software Corporation 的 ADAMS/Flex 产品可用于在 MSC.ADAMS 中执行动态分析时考虑组件的柔性。ADAMS/Flex 依赖于诸如 Abaqus 之类的有限元分析代码，以 MSC.ADAMS 可用的形式提供组件的柔性信息。**abaqus adams** 转换器可用于创建 MSC.ADAMS 组件的 Abaqus 模型，并将 Abaqus 结果转换为 MSC.ADAMS 模态中性（`.mnf`）文件，即 ADAMS/Flex 所需的格式。

### 使用转换器

以下过程总结了 **abaqus adams** 转换器的典型用法：

1. 为 MSC.ADAMS 模型的每个柔性组件创建 Abaqus 模型。每个组件建模为 Abaqus 子结构。
2. 运行 Abaqus 分析。有关更多信息，请参阅『准备子结构 SIM 数据库文件』（第 3.2.36 节）。
3. 运行 **abaqus adams** 转换器，读取分析生成的子结构 SIM 数据库，并为 MSC.ADAMS 创建模态中性（`.mnf`）文件。
4. 将模态中性文件读入 MSC.ADAMS。必须为 MSC.ADAMS 中的每个柔性零件创建单独的模态中性文件。

### 准备子结构 SIM 数据库文件

本节描述将生成 ADAMS/Flex 所需结果量的子结构 SIM 数据库的准备工作。

#### Abaqus 子结构模型

在 MSC.ADAMS 中考虑组件柔性的第一步是将该组件建模为 Abaqus 子结构。此过程涉及创建组件的 Abaqus 有限元模型。使用子结构构建 Abaqus 模型的一般指南在『使用子结构』，第 10.1.1 节（第 10.1.1 节）中描述。

#### 设置 Abaqus 模型以创建模态中性文件

当您创建要转换到 MSC.ADAMS 的子结构时，子结构生成步骤必须包含以下选项：

```
[*SUBSTRUCTURE GENERATE](../key/key-link.md#usb-kws-ssubgenerate), MASS MATRIX=YES, RECOVERY MATRIX=YES
[*FLEXIBLE BODY](../key/key-link.md#usb-kws-hflexiblebody), TYPE=ADAMS
```
此外，您可以添加以下数据以将应力和/或应变转换到 MSC.ADAMS：
```
*ELEMENT RECOVERY MATRIX, POSITION=AVERAGED AT NODES
S,
E,
```

### 单位

MSC.ADAMS 程序要求您定义组件模型中使用的单位，而 Abaqus 则不需要。因此，在创建模态中性文件时，您必须明确声明模型中使用的单位。在 **abaqus adams** 执行过程中执行此操作的方法与 ADAMS/View **Units Settings** 对话框中的方法非常相似。可以使用 **abaqus adams** 执行程序上的 **units** 选项指定预定义的单位系统。或者，可以使用 **abaqus adams** 执行程序上的 **length**、**mass**、**force** 和 **time** 选项指定单独的长度、质量、力和时间单位。指定的任何单独单位都会覆盖单位系统中相应的单位。默认单位系统是 `mks`。**units** 选项的有效单位系统列于表 3.2.36-1（第 3.2.36-1 表）中。

**表 3.2.36–1** 有效的单位系统。
| 单位系统 | 长度单位 | 质量单位 | 力单位 | 时间单位 |
| --- | --- | --- | --- | --- |
| `mks` | 米 | 千克 | 牛顿 | 秒 |
| `mmks` | 毫米 | 千克 | 牛顿 | 秒 |
| `cgs` | 厘米 | 克 | 达因 | 秒 |
| `ips` | 英寸 | 磅质量 | 磅力 | 秒 |

**length**、**mass**、**force** 和 **time** 选项各自的有效选项如下：

**长度单位**

长度单位的有效选项是
- `meters`
- `millimeters`, `mm`
- `centimeters`, `cm`
- `kilometers`, `km`
- `inches`, `inch`, `in`
- `feet`, `foot`, `ft`
- `mile`

**质量单位**

质量单位的有效选项是
- `kilograms`, `kg`
- `megagram`, `tonne`
- `gram`, `g`
- `pound_mass`, `lbm`, `pound`
- `slug`
- `kpound_mass`
- `ounce_mass`

**力单位**

力单位的有效选项是
- `newtons`, `N`
- `knewton`, `kN`
- `kilogram_force`, `kgf`
- `dyne`
- `ounce_force`
- `pound_force`, `lbf`, `pound`
- `kpound_force`

**时间单位**

时间单位的有效选项是
- `seconds`, `sec`
- `milliseconds`, `ms`
- `minutes`, `min`
- `hours`

可以在 Abaqus 环境文件（`abaqus_v6.env`）中定义单位选项的默认值。可以通过 **adams_units_family** 参数定义 **units** 选项的默认值。可以分别通过 **adams_length_units**、**adams_mass_units**、**adams_time_units** 和 **adams_force_units** 参数定义 **length**、**mass**、**time** 和 **force** 选项的默认值。

### 转换具有负特征值的模态

通常，对于三维中无预应力的无约束子结构，您希望找到六个具有相关零特征值的刚体模态。预应力子结构的情况通常不同，它们可能具有少于六个具有零特征值的模态。预应力可能会将一些预期的零点改变为显著正值或负值，具体取决于预应力的符号。

默认情况下，转换器会删除具有负特征值的模态，并重新正交化简化基。如果您想保留具有负特征值的模态，请定义环境变量 `MDI_MNFWRITE_OPTIONS`。
- 在 UNIX 平台上，键入以下命令： ``` setenv MDI_MNFWRITE_OPTIONS negative_roots_OK ```
- 在 Windows 平台上，键入以下命令： ``` set MDI_MNFWRITE_OPTIONS=negative_roots_OK ```

在这种情况下，转换器将以与所有其他模态相同的方式处理具有负特征值的模态。

要确定模型在转换器转换时是否会有负特征值，您可以在输入文件中添加一个没有边界条件的特征频率提取步骤。

### 命令行选项

**job**

此选项指定结果转换期间使用的输入和输出文件名。*job-name* 值用于构造默认子结构 SIM 数据库文件名 `*job-name*.sim`。输出模态中性文件被命名为 `*job-name*.mnf`。

如果此选项从命令行中省略，系统将提示您输入此值。

**substructure_sim**

此选项指定子结构 SIM 数据库（`.sim`）文件的名称（如果它与 `*job-name*.sim` 不同）。该文件通常命名为 `*job-name*_Z*nn*.sim`。

**units**

此选项指定模型的单位系统。可能的值是 `mmks`、`mks`、`cgs` 或 `ips`，它们对应于具有相同名称的 ADAMS/View 选项。默认值为 `mks`。

此选项可以在 Abaqus 环境文件中定义如下：

```
adams_unit_family=*unit-family*
```

**length**

此选项指定模型的长度单位。如果指定了此选项，它将覆盖指定单位系统的长度单位。

此选项可以在 Abaqus 环境文件中定义如下：

```
adams_length_units=*length-unit*
```

**mass**

此选项指定模型的质量单位。如果指定了此选项，它将覆盖指定单位系统的质量单位。

此选项可以在 Abaqus 环境文件中定义如下：

```
adams_mass_units=*mass-unit*
```

**time**

此选项指定模型的时间单位。如果指定了此选项，它将覆盖指定单位系统的时间单位。

此选项可以在 Abaqus 环境文件中定义如下：

```
adams_time_units=*time-unit*
```

**force**

此选项指定模型的力单位。如果指定了此选项，它将覆盖指定单位系统的力单位。

此选项可以在 Abaqus 环境文件中定义如下：

```
adams_force_units=*force-unit*
```

**mnf_elset**

此选项定义一组单元，其面将导出到模态中性文件，因此可在 MSC.ADAMS 中查看。此选项不影响求解的力学行为。
