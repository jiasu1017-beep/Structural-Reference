### Overview

This section provides a reference to the axisymmetric solid elements available in Abaqus/Standard. These elements are intended for analysis of hollow bodies, such as pipes and pressure vessels. They can also be used to model solid bodies, but spurious stresses may occur at zero radius, particularly if transverse shear loads are applied.

### Conventions

Coordinate 1 is *r*, coordinate 2 is *z*. Referring to the figures shown in ["Choosing the element's dimensionality," Section 27.1.2](pt06ch27s01aus111.md), the *r*-direction corresponds to the global *X*-direction in the ![](../graphics/usb_eqn04481.gif) plane and the negative global *Z*-direction in the ![](../graphics/usb_eqn04482.gif) plane, and the *z*-direction corresponds to the global *Y*-direction. Coordinate 1 must be greater than or equal to zero.

Degree of freedom 1 is ![](../graphics/usb_eqn00266.gif), degree of freedom 2 is ![](../graphics/usb_eqn00267.gif). The ![](../graphics/usb_eqn04483.gif) degree of freedom is an internal variable: you cannot control it.

### Element-based loading

Even though the symmetry in the *r*–*z* plane at ![](../graphics/usb_eqn04428.gif) allows the modeling of half of the initially axisymmetric structure, the loading must be specified as the total load on the full axisymmetric body. ### Node ordering and face numbering on elements

The node ordering in the first *r*–*z* plane of each element, at ![](../graphics/usb_eqn00141.gif), is shown below. Each element must have *N* more planes of nodes defined, where *N* is the number of Fourier modes. The node ordering is the same in each plane. You can specify the nodes in each plane. Alternatively, you can specify the node ordering in the first *r*–*z* plane of an element, and Abaqus/Standard will generate all other nodes for the element by adding successively a constant offset to each node for each of the *N* planes of the element. By default, Abaqus/Standard uses an offset of 100000 (see ["Element definition," Section 2.2.1](pt01ch02s02aus11.md)).
