# 1.5 技术支持

Abaqus 通过遍布全球的支持办事处网络提供**技术支持**（用于解决建模或分析问题）和**系统支持**（用于解决安装、许可及硬件相关问题）。各地区联系信息可在 **Locations** 页面查询，网址为 [www.3ds.com/simulia](http://www.3ds.com/simulia)。您也可以通过 [www.3ds.com/simulia](http://www.3ds.com/simulia) 网站的 **Support** 页面获取帮助。在联系支持办事处时，请说明您需要的是技术支持（Abaqus 分析过程中遇到问题）还是系统支持（Abaqus 无法正确安装、许可无法正常工作或出现其他硬件相关问题）。

我们欢迎您对 Abaqus 软件、支持计划或文档提出任何改进建议。我们将确保考虑您的所有增强请求以供未来版本参考。如果您对 SIMULIA 提供的服务或产品有建议，请访问 [www.3ds.com/simulia](http://www.3ds.com/simulia)。如需投诉，请联系您的支持办事处或访问 [www.3ds.com/simulia](http://www.3ds.com/simulia) 上的 **Quality Assurance** 页面。

## 1.5.1 技术支持

SIMULIA 技术支持工程师可以帮助您澄清 Abaqus 功能并通过提供使用 Abaqus 的一般信息及其在特定分析中的应用信息来检查错误。如果您对某个分析有疑虑，我们建议您尽早联系我们，因为在项目开始阶段解决问题通常比在分析结束时试图纠正问题更容易。

请在拨打技术支持热线之前准备好以下信息，并在任何书面联系中包含：

- 您正在使用的 Abaqus 版本。
  - Abaqus/Standard、Abaqus/Explicit 和 Abaqus/CFD 的版本号显示在数据（`.dat`）文件的顶部。
- 您运行 Abaqus 的计算机类型。
- 任何问题的症状，包括确切的错误消息（如果有）。
- 您已经尝试过的解决方案或测试。

对于特定问题的支持，任何可用的 Abaqus 输出文件都可能有助于回答支持工程师可能提出的问题。

支持工程师将尝试根据您提供的模型描述和遇到的困难描述来诊断您的问题。通常，支持工程师需要模型草图，这些草图可以通过电子邮件、传真或邮寄方式发送。最终结果的绘图或分析终止处附近的结果也可能需要用于理解可能导致问题的原因。

如果支持工程师无法根据这些信息诊断您的问题，可能会要求您提供输入数据。数据可以附加到在线系统中的支持请求中。也可以通过电子邮件、ftp、光盘或 DVD 发送。

所有支持请求都会被跟踪。此跟踪使您（以及支持工程师）能够监控特定问题的进度，并检查我们是否有效地解决支持问题。您必须注册该系统才能查询支持问题。如果您通过系统外的渠道联系我们讨论现有的支持问题，并且您知道支持请求编号，请提供该编号，以便我们查阅数据库查看最新的处理情况。

## 1.5.2 系统支持

Abaqus 系统支持工程师可以帮助您解决与 Abaqus 的安装和运行相关的问题，包括许可困难，这些不在技术支持范围内。

您应该仔细按照 [Abaqus Installation and Licensing Guide](http://www.3ds.com/support/knowledge-base) 中的说明安装 Abaqus。如果您遇到安装或许可问题，首先请查看 [Abaqus Installation and Licensing Guide](http://www.3ds.com/support/knowledge-base) 中的说明以确保已正确遵循。如果此方法无法解决问题，请在 Dassault Systèmes 知识库中搜索已知的安装问题，网址为 [www.3ds.com/support/knowledge-base](http://www.3ds.com/support/knowledge-base)。如果此方法仍无法解决您的情况，请在在线系统中创建支持请求。发送任何可用于定义问题的信息：中止分析的错误消息或所遇到问题的详细说明。在可能的情况下，请发送 `abaqus info=support` 命令的输出。

## 1.5.3 对学术机构的支持

根据学术许可协议的条款，我们不为学术机构用户提供支持，除非该机构也购买了技术支持。请联系我们了解更多详情。
