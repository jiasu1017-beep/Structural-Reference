# 3.3.1 使用Abaqus环境设置





**产品：** Abaqus/Standard  Abaqus/Explicit  

##### **参考**

- ["Abaqus执行程序：概述，" 第3.1.1节](pt01ch03s01abo02.md)

### 概述

Abaqus环境设置允许您控制Abaqus作业执行的各种方面。例如，您可以：
- 通过更改与内存相关的参数来"调整"Abaqus以提高其性能。
- 控制临时文件（scratch files）的写入位置和方式。
- 为作业参数提供默认值，否则这些参数将必须在命令行上给出。

作业执行的许多其他方面可以通过环境设置进行配置。本节讨论其中一些方面；其他主要面向Abaqus站点管理员的方面在[Abaqus安装和许可指南](../sgb/sgb-link.md#sgb)中详细讨论。

### 环境设置层次结构

Abaqus环境设置按以下顺序处理：

1. 主机级环境设置。这些设置应用于在指定计算机上运行的所有Abaqus作业。
2. 用户级环境设置。这些设置应用于在您的账户中运行的所有Abaqus作业。对于Abaqus在Windows平台上在您的主目录中定位环境文件，必须使用`HOME`环境变量或`HOMEDRIVE`和`HOMEPATH`环境变量的组合来指定您主目录的完整路径。
3. 作业级环境设置。这些设置仅应用于指定的Abaqus作业。

环境设置可以多次指定。如果在多个级别定义或在同一级别给出两次，则最后处理的值将作为该设置使用的值。

Abaqus环境设置使用特定目录中的特殊文件设置。主机级设置在`abaqus`账户目录的`site`目录中设置。您可以通过在主目录和/或当前目录中创建环境文件`abaqus_v6.env`来更改这些设置。主目录文件中设置的将应用于您运行的所有作业。当前目录文件中的设置将仅应用于从当前目录运行的作业。

### 语法

环境文件中给出的条目必须使用Python语言语法。条目的形式为：

```
parameter=value
```

以下是Python语法规则的简要概述：
- 参数必须始终有一个值。该值可以是任何有效的Python常量或表达式。
- 字符串值必须用一对双引号或单引号括起来。
- 注释前有井号（#）。井号后同一行上的所有字符都将被忽略。引号字符串内的井号是字符串的一部分，而不是注释的开始。
- 空行将被忽略。
- 如果双引号字符串中包含嵌入式单引号，则不需要特殊处理。例如，`"my value's"`被翻译为`my value's`。对于嵌入单引号字符串中的双引号也是如此。如果用反斜杠（\）字符作为前缀，则可以嵌入与外层引号相同类型的引号。
- 三引号（"""）字符串可以跨越多行，字符串内的引号不需要特殊处理。条目的形式为：
  ``` parameter=""" multi-line value """ ```
- 列表必须用括号（（ ））或方括号（[ ]）括起来。列表中的各个项目用逗号分隔。如果列表用括号括起来且只包含一个值，则必须在值后面加逗号。字符串列表项必须用引号括起来。条目的形式为：
  ``` parameter=(value1, value2, value3) ```

### 故障排除

由错误的环境设置引起的问题可以通过使用以下命令进行诊断：

```
`abaqus` `information`=`environment`
```

此命令打印所有当前环境设置。

### 命令行默认参数

以下参数为各种设置提供默认值，否则必须在命令行上指定这些设置（请参阅["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行，" 第3.2.2节](pt01ch03s02abx02.md)以及["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD协同仿真执行，" 第3.2.4节](pt01ch03s02abx04.md)）。命令行上给出的值将覆盖环境文件中指定的值。

****cpus****

如果有并行处理可用，则使用的处理器数量。协同仿真执行程序的默认值是`2`；否则，默认值为`1`。

****domains****

Abaqus/Explicit中的并行域数量。如果该值大于1，则无论**parallel**和**cpus**参数的值如何，都将执行域分解。但是，如果**parallel**=`domain`，则**cpus**的值必须能被**domains**的值整除。如果未设置此参数，则如果在分析运行期间**parallel**=`domain`，则域数默认值为使用的处理器数；如果**parallel**=`loop`，则默认值为1。

****double_precision****

如果您未在**abaqus**命令行上指定精度版本，则运行的Abaqus/Explicit的默认精度版本。可能的值有`EXPLICIT`（仅以双精度运行Abaqus/Explicit分析）、`BOTH`（Abaqus/Explicit打包程序和分析均以双精度运行）、`CONSTRAINT`（Abaqus/Explicit中的约束打包程序和约束求解器以双精度运行，而Abaqus/Explicit打包程序和分析继续以单精度运行）或`OFF`（Abaqus/Explicit打包程序和分析均以单精度运行）。默认值为`OFF`。

****parallel****

如果您未在**abaqus**命令行上指定并行方法，则Abaqus/Explicit中的默认并行方法。可能的值有`DOMAIN`或`LOOP`；默认值为`DOMAIN`。

****run_mode****

如果您未在**abaqus**命令行上指定运行模式，则默认运行模式（交互式、后台或批处理）。对于**abaqus** **analysis**，默认值为`"background"`；对于**abaqus** **viewer**，默认值为`"interactive"`。

****scratch****

用于临时文件的目录。此目录必须存在（即Abaqus不会创建它），并且必须具有写权限。在UNIX平台上，默认值是`$TMPDIR`环境变量的值；如果未定义`$TMPDIR`，则为`/tmp`。在Windows平台上，默认值是`%TEMP%`环境变量的值；如果此变量未定义，则为`\TEMP`。在分析期间，将在此目录下创建一个子目录来保存分析临时文件。子目录的名称由您的用户名、作业ID和作业的进程标识符构成。分析完成后，子目录及其内容将被删除。

****standard_parallel****

如果您未在**abaqus**命令行上指定并行模式，则Abaqus/Standard中的默认并行执行模式。如果此参数设置为`ALL`，则元素操作和求解器都将并行运行。如果此参数设置为`SOLVER`，则只有求解器将并行运行。默认并行执行模式为`ALL`。

****gpus****

如果您未在**abaqus**命令行上指定GPGPU求解器加速选项，则Abaqus/Standard中的GPGPU直接求解器加速设置。默认情况下，GPGPU求解器加速未激活。此参数的值是Abaqus/Standard分析中使用的GPGPU数量。在基于MPI的分析中，这是每台主机上使用的GPGPU数量。

****unconnected_regions****

如果此变量设置为`ON`，则Abaqus/Standard将在**datacheck**分析期间为模型中未连接的区域在输出数据库中创建元素集和节点集。使用此选项创建的元素集和节点集名为`MESH COMPONENT *N*`，其中*N*是组件编号。默认值为`OFF`。

****order_parallel****

如果您未在**abaqus**命令行上指定排序模式，则Abaqus/Standard中直接稀疏求解器的排序模式。如果此参数设置为`OFF`，则排序过程将不会并行运行。如果此参数设置为`ON`，则排序过程将并行运行。默认排序模式为`ON`。

### 系统资源参数

以下环境文件变量可以在代码安装后设置，以更改Abaqus使用的资源，从而提高系统性能。默认情况下，Abaqus会检测机器上的物理内存（或集群中每个计算节点上的物理内存），并根据机器平台分配可用内存的百分比（详细信息请参阅Dassault Systmes知识库：[www.3ds.com/support/knowledge-base](http://www.3ds.com/support/knowledge-base)）。您可以通过指定数字后跟百分比符号来覆盖默认百分比。该变量也可以定义为兆字节数或千兆字节数。有关更改Abaqus使用的系统资源的更多详细信息，请参阅["管理Abaqus中的内存和磁盘使用，" 第3.4.1节](pt01ch03s04aus31.md)。

****memory****

在输入文件预处理期间和Abaqus/Standard分析阶段可以分配的最大内存量或最大物理内存百分比。对于计算机集群上的并行执行，此内存限制指定每个进程可以分配的最大内存量。

### 系统定制参数

以下是一些常用环境文件参数的讨论。参数的完整列表可以在[Abaqus安装和许可指南](../sgb/sgb-link.md#sgb)中找到。

****ask_delete****

如果此参数设置为`OFF`，系统不会询问您是否应删除具有相同文件名的旧作业文件；文件将自动被删除。默认值为`ON`。

****auto_calculate****

如果此参数设置为`ON`，则在分析结束且执行程序检测到需要输出数据库文件转换时，后处理计算器将自动启动。如果此参数设置为`OFF`，即使执行程序检测到需要，后处理计算器也不会在分析结束时运行。默认值为`ON`。

****auto_convert****

如果此参数设置为`ON`且Abaqus/Explicit分析使用**parallel**=`domain`并行运行，则**convert**=`select`、`convert**=`state`和**convert**=`odb`选项将在分析结束时自动运行。默认值为`ON`。

****average_by_section****

此参数仅用于Abaqus/Standard分析。如果此参数设置为`OFF`，则写入数据（`.dat`）文件和结果（`.fil`）文件的平均区域基于元素结构。如果此参数设置为`ON`，平均区域还会考虑元素属性和材料常量的基础值。在具有许多截面和/或材料定义的问题中，默认值`OFF`通常比非默认值`ON`提供更好的性能。有关平均方案的更多详细信息，请参阅["输出到数据和结果文件，" 第4.1.2节](pt02ch04s01aus39.md)。

****mp_host_list****

要用于基于MPI的并行Abaqus分析的主机名列表，包括每台机器上使用的处理器数量；例如：

```
mp_host_list=[['maple',1],['pine',1],['oak',2]]
```

表示如果为分析指定的**cpus**为4，则分析将在名为`maple`的机器上使用一个处理器，在名为`pine`的机器上使用一个处理器，在名为`oak`的机器上使用两个处理器。主机列表中定义的处理器总数必须大于或等于为分析指定的**cpus**数量。如果未定义主机列表，Abaqus将在本地系统上运行。使用支持的排队系统时，不需要定义此参数。如果已定义，它将被排队环境覆盖。

****mp_mode****

将此变量设置为`MPI`以指示系统上提供了MPI组件。将**mp_mode**=`THREADS`设置为使用基于线程的并行化方法。默认值是适用的`MPI`。

****odb_output_by_default****

如果此参数设置为`ON`，将自动生成输出数据库输出。如果此参数设置为`OFF`，则必须在输入文件中放置输出数据库请求关键字才能获取输出数据库输出。默认值为`ON`。

****onCaeStartup****

在Abaqus/CAE开始之前要执行的可选函数。请参阅[Abaqus安装和许可指南](../sgb/sgb-link.md#sgb-env-exa-custom-startup)第4.3.3节中的"自定义Abaqus/CAE启动"中的此函数示例。

### 协同仿真参数

以下环境文件变量为使用直接耦合接口的求解器之间的协同仿真提供默认设置。这包括Abaqus/Standard到Abaqus/Explicit的协同仿真，以及Abaqus与某些第三方分析程序之间的协同仿真。

****cosimulation_port****

将**cosimulation_port**设置为连接使用的端口号。默认值为`48000`。

****cosimulation_timeout****

将**cosimulation_timeout**设置为超时时间（以秒为单位）。如果Abaqus在指定时间内未收到来自耦合分析程序的任何通信，则Abaqus将终止。默认值为`3600`秒。

以下环境文件变量提供设置，允许您为使用协同仿真执行程序提交的协同仿真作业分配CPU。这包括Abaqus/Standard到Abaqus/Explicit、Abaqus/Standard到Abaqus/CFD以及Abaqus/CFD到Abaqus/Explicit的协同仿真（请参阅["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD协同仿真执行，" 第3.2.4节](pt01ch03s02abx04.md)）。

****cpus_weight_std****

此选项控制向Abaqus/Standard分析分配CPU的方式。Abaqus/Standard分析的实际CPU分配按照此值以及**cpus_weight_xpl**、**cpus_weight_cfd**和**cpus**的设置按比例进行。

****cpus_weight_xpl****

此选项控制向Abaqus/Explicit分析分配CPU的方式。Abaqus/Explicit分析的实际CPU分配按照此值以及**cpus_weight_std**、**cpus_weight_cfd**和**cpus**的设置按比例进行。

****cpus_weight_cfd****

此选项控制向Abaqus/CFD分析分配CPU的方式。Abaqus/CFD分析的实际CPU分配按照此值以及**cpus_weight_std**、**cpus_weight_xpl**和**cpus**的设置按比例进行。

****portpool****

将此变量设置为一对用冒号分隔的TCP/UDP端口号，表示协同仿真执行程序在建立子进程之间的连接时使用的端口号范围的起始值和结束值。

### 环境文件示例

使用先前讨论的参数的示例环境文件如下所示。名为`abaqusinc.env`的示例环境文件包含在发布的`site`子目录中，以显示SIMULIA使用的选项。

#### UNIX环境文件：

```
ask_delete=OFF
# 以下参数导致临时文件写入
# /tmp。
scratch="/tmp"
```

#### Windows环境文件：

```
ask_delete=OFF
# 以下参数导致临时文件写入
# c:上的tmp目录。
scratch="c:/tmp"
```
