# 6.3 执行命令







所有命令最终都在 kernel 进程中执行，但有几种方式可以实现：
- 您可以通过在命令行上使用 --start 或 --replay 选项来从文件执行 kernel 命令。
- 您可以通过使用****文件**![](../graphics/images/arrow.gif)**运行脚本****来从文件执行 kernel 命令。
- 您可以在 Abaqus/CAE CLI 中输入 kernel 命令。
- GUI 模式基础设施可以将命令字符串从 GUI 发送到 kernel 进程执行（详见"命令处理"，第 7.2.4 节）。
- 您可以使用 `sendCommand` 函数直接从 GUI 发出 kernel 命令。

`sendCommand` 函数接受三个参数：
- 一个必需的字符串参数，指定要在 kernel 中执行的命令。
- 两个可选的布尔参数，*writeToReplay* 和 *writeToJournal*。

可选的布尔参数控制 `sendCommand` 函数是否将命令写入重放文件或日志文件。默认情况下，`sendCommand` 函数将命令写入重放文件，但不写入日志文件。如果命令以任何方式修改了模型，您应该将命令记录到重放文件和日志文件中。但是，如果命令仅修改会话数据（例如视口的视图），您应该将命令记录到重放文件中，但不应记录到日志文件中。按照惯例，用户应该能够通过重放其重放文件来完全重新创建交互会话的结果。只有写入日志文件的命令才会在应用程序中止时可用于数据恢复。

Abaqus Scripting Interface 命令会自动记录到日志中。因此，如果您使用 `sendCommand` 函数发出 Abaqus Scripting Interface 命令，不应设置 *writeToJournal*=True。否则，命令将在日志文件中被记录两次。如需更多信息，请参阅 [Abaqus/CAE User's Guide](../usi/usi-link.md#usi-dbs-files-command) 的第 9.5 节"Abaqus/CAE 命令文件"。

如果您编写自己的 kernel 脚本模块和函数，应该知道您可以使用 `journalMethodCall` 函数将命令记录到日志文件中。与在 `sendCommand` 函数中使用 *writeToJournal* 参数相比，此选项更可取。如果命令使用内置的 Abaqus Scripting Interface 命令更改了 Mdb 对象，则您的命令不应调用 `journalMethodCall`，因为这些命令默认会被记录到日志。更改 Mdb 的 customData 的命令应该调用 `journalMethodCall`。有关说明 `journalMethodCall` 函数常见用法的示例，请参阅 [Abaqus Scripting Reference Guide](../ker/ker-link.md#ker-utl-customization-journalmethodcall-pyc) 的第 53.11.1 节"journalMethodCall"。

一般来说，您应该将 `sendCommand` 函数包装在 try 块中，以捕获 kernel 命令可能抛出的任何异常。为了能够捕获异常，它们应该是基于类的异常，而不是简单的字符串。例如：

```
from abaqusGui import sendCommand
try:
    sendCommand("mdb.customData.myCommand('Cmd-1', 50, 200)"
except ValueError, x:
    print 'an exception was raised: ValueError: %s' % (x)
except:
    exc_type, exc_value = sys.exc_info()[:2]
    print 'error. %s.%s'%(exc_type.__name__, exc_value)

```
