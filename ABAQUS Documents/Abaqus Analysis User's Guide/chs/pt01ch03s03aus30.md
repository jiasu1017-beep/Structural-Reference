# 3.3.1 使用Abaqus环境设置





**产品：**Abaqus/Standard  Abaqus/Explicit  

##### **参考**

- ["Abaqus执行程序：概述，" 3.1.1节](pt01ch03s01abo02.md)

### 概述

Abaqus环境设置允许您控制Abaqus作业执行的各个方面。例如，您可以：
- 通过更改与内存相关的参数来"调整"Abaqus以提高其性能。
- 控制临时文件的写入位置和方式。
- 为作业参数提供默认值，否则这些参数将必须在命令行上给出。

可以通过环境设置配置作业执行的许多其他方面。其中一些在本节中讨论；其他主要供Abaqus站点管理员感兴趣的详细内容在[Abaqus安装和许可指南](../sgb/sgb-link.md#sgb)中讨论。

### 环境设置层次结构

Abaqus环境设置按以下顺序处理：

1. 主机级环境设置。这些设置应用于在指定计算机上运行的所有Abaqus作业。
2. 用户级环境设置。这些设置应用于在您的帐户中运行的所有Abaqus作业。要使Abaqus在Windows平台上找到主目录中的环境文件，必须使用`HOME`环境变量或`HOMEDRIVE`和`HOMEPATH`环境变量的组合指定主目录的完整路径。
3. 作业级环境设置。这些设置仅应用于指定的Abaqus作业。

环境设置可以多次指定。如果在多个级别定义或在同级别两次给出，则最后处理的值将是使用的值。

Abaqus环境设置使用特定目录中的特殊文件设置。主机级设置在`abaqus`帐户目录的`site`目录中设置。您可以通过在主目录和/或当前目录中创建环境文件`abaqus_v6.env`来更改这些设置。主目录文件中的设置将应用于您运行的所有作业。当前目录文件中的设置将仅应用于从当前目录运行的作业。

### 语法

环境文件中的条目必须使用Python语言语法给出。条目采用以下形式：

```
parameter=value
```
以下是Python语法规则的简要概述：- 参数必须始终有值。该值可以是任何有效的Python常量或表达式。
- 字符串值必须包含在一对双引号或单引号中。
- 注释前有井号（#）。行中井号后的所有字符都会被忽略。引号内的井号是字符串的一部分，不是注释的开头。
- 空行被忽略。
- 如果嵌入的单引号位于双引号字符串内，则不需要特殊处理。例如，`"my value's"`被翻译为`my value's`。嵌入在单引号字符串内的双引号也是如此。可以通过反斜杠（\）字符前缀来嵌入与封闭引号相同类型的引号。三引号（"""）字符串可以跨越多行，字符串内的引号不需要特殊处理。条目采用以下形式：``` parameter=""" multi-line value """ ```
- 列表必须用括号（（ ））或方括号（[]）括起来。列表中的各个项用逗号分隔。如果列表用括号括起来且只包含一个值，则必须在值后面跟一个逗号。字符串列表项必须用引号括起来。条目采用以下形式：``` parameter=(value1, value2, value3) ```

### 故障排除

可以通过使用以下命令诊断由错误的环境设置引起的问题：

```
`abaqus` `information`=`environment`
```
此命令打印所有当前环境设置。

### 命令行默认参数

以下参数为否则必须在命令行上指定的各种设置提供默认值（请参阅["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行，" 3.2.2节](pt01ch03s02abx02.md)和["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD协同仿真执行，" 3.2.4节](pt01ch03s02abx04.md)）。命令行上给出的值会覆盖环境文件中指定的值。

****cpus****

如果有可用并行处理，则使用的处理器数。对于协同仿真执行程序默认为`2`；否则，默认为`1`。

****domains****

Abaqus/Explicit中的并行域数。如果该值大于1，无论**parallel**和**cpus**参数的值如何，都将执行域分解。但是，如果**parallel**=`domain`，则**cpus**的值必须能被**domains**的值整除。如果未设置此参数，则如果**parallel**=`domain`，则域数默认为分析运行期间使用的处理器数；如果**parallel**=`loop`，则为1。

****double_precision****

如果您未在**abaqus**命令行上指定精度版本，则运行的Abaqus/Explicit的默认精度版本。可能的值为`EXPLICIT`（仅以双精度运行Abaqus/Explicit分析）、`BOTH`（以双精度运行Abaqus/Explicit打包程序和分析）、`CONSTRAINT`（Abaqus/Explicit中的约束打包程序和约束求解器以双精度运行，而Abaqus/Explicit打包程序和分析继续以单精度运行）或`OFF`（以单精度运行Abaqus/Explicit打包程序和分析）。默认为`OFF`。

****parallel****

如果您未在**abaqus**命令行上指定并行方法，则Abaqus/Explicit中的默认并行方法。可能的值为`DOMAIN`或`LOOP`；默认值为`DOMAIN`。

****run_mode****

如果您未在**abaqus**命令行上指定运行模式（交互、后台或批处理），则为默认运行模式。**abaqus** **analysis**的默认值为`"background"`，而**abaqus** **viewer**的默认值为`"interactive"`。

****scratch****

用于临时文件的目录。此目录必须存在（即Abaqus不会创建它），并且必须分配写权限。在UNIX平台上，默认值是`$TMPDIR`环境变量的值；如果未定义$TMPDIR，则为`/tmp`。在Windows平台上，默认值是`%TEMP%`环境变量的值；如果未定义此变量，则为`\TEMP`。在分析期间，将在此目录下创建一个子目录来保存分析临时文件。子目录的名称由您的用户名、作业ID和作业的进程标识符构建。分析完成后，删除子目录及其内容。

****standard_parallel****

如果您未在**abaqus**命令行上指定并行模式，则Abaqus/Standard中的默认并行执行模式。如果此参数设置为`ALL`，则元素操作和求解器都将并行运行。如果此参数设置为`SOLVER`，则只有求解器将并行运行。默认并行执行模式为`ALL`。

****gpus****

如果您未在**abaqus**命令行上指定GPGPU求解器加速选项，则Abaqus/Standard中的GPGPU直接求解器加速设置。默认情况下，不激活GPGPU求解器加速。此参数的值是在Abaqus/Standard分析中使用的GPGPU数量。在基于MPI的分析中，这是每台主机上使用的GPGPU数量。

****unconnected_regions****

如果此变量设置为`ON`，Abaqus/Standard将在**datacheck**分析期间为模型中未连接的区域创建元素集和节点集。使用此选项创建的元素集和节点集名为`MESH COMPONENT *N*`，其中*N*是组件号。默认值为`OFF`。

****order_parallel****

如果您未在**abaqus**命令行上指定排序模式，则Abaqus/Standard中直接稀疏求解器的排序模式。如果此参数设置为`OFF`，则排序过程不会并行运行。如果此参数设置为`ON`，则排序过程将并行运行。默认排序模式为`ON`。

### 系统资源参数

以下环境文件变量可以在代码安装后设置，以更改Abaqus使用的资源，从而提高系统性能。默认情况下，Abaqus会检测机器上的物理内存（或集群中每个计算节点上的物理内存），并根据机器平台分配可用内存的百分比（详细信息请参阅[Abaqus知识库](http://www.3ds.com/support/knowledge-base)）。您可以通过指定跟百分号的数字来覆盖默认百分比。该变量也可以定义为兆字节或千兆字节的数值。更改Abaqus使用的系统资源的详细信息在["管理Abaqus中的内存和磁盘使用，" 3.4.1节](pt01ch03s04aus31.md)中给出。

****memory****

在输入文件预处理期间和Abaqus/Standard分析阶段可以分配的最大内存量或物理内存的最大百分比。对于计算机集群上的并行执行，此内存限制指定每个进程可以分配的最大内存量。

### 系统定制参数

以下是对常用的一些其他环境文件参数的讨论。参数的完整列表可在[Abaqus安装和许可指南](../sgb/sgb-link.md#sgb)中找到。

****ask_delete****

如果此参数设置为`OFF`，系统不会询问是否应删除相同文件名的旧作业文件；文件将被自动删除。默认值为`ON`。

****auto_calculate****

如果此参数设置为`ON`，则在分析结束时，如果执行程序检测到需要输出数据库文件转换，则会自动启动后处理计算器。如果此参数设置为`OFF`，则即使执行程序检测到有必要，后处理计算器也不会在分析结束时运行。默认值为`ON`。

****auto_convert****

如果此参数设置为`ON`且Abaqus/Explicit分析与**parallel**=`domain`并行运行，则**convert**=`select`、`convert**=`state`和`convert**=`odb`选项将在分析结束时自动运行。默认值为`ON`。

****average_by_section****

此参数仅用于Abaqus/Standard分析。如果此参数设置为`OFF`，则写入数据（`.dat`）文件和结果（`.fil`）文件的平均区域基于元素结构。如果此参数设置为`ON`，则平均区域还考虑元素属性和材料常量的基础值。在具有许多截面和/或材料定义的问题中，非默认值`ON`通常会比默认值`OFF`给出更好的性能。有关平均方案的更多详细信息，请参阅["输出到数据和结果文件，" 4.1.2节](pt02ch04s01aus39.md)。

****mp_host_list****

用于基于MPI的并行Abaqus分析的主机名列表，包括每台机器上使用的处理器数；例如，

```
mp_host_list=[['maple',1],['pine',1],['oak',2]]
```
表示如果为分析指定的**cpus**数为4，则分析将在名为`maple`的机器上使用一个处理器，在名为`pine`的机器上使用一个处理器，在名为`oak`的机器上使用两个处理器。主机列表中定义的处理器总数必须大于或等于为分析指定的**cpus**数。如果未定义主机列表，Abaqus将在本地系统上运行。使用支持的排队系统时，不需要定义此参数。如果已定义，它将被排队环境覆盖。

****mp_mode****

将此变量设置为`MPI`以指示系统上提供了MPI组件。将**mp_mode**=`THREADS`设置为使用基于线程的并行化方法。在适用的情况下，默认值为`MPI`。

****odb_output_by_default****

如果此参数设置为`ON`，将自动生成输出数据库输出。如果此参数设置为`OFF`，则必须在输入文件中放置输出数据库请求关键字以获取输出数据库输出。默认值为`ON`。

****onCaeStartup****

在Abaqus/CAE开始之前执行的可选函数。请参阅["定制Abaqus/CAE启动，" Abaqus安装和许可指南4.3.3节](../sgb/sgb-link.md#sgb-env-exa-custom-startup)，了解此函数的示例。

### 协同仿真参数

以下环境文件变量为使用直接耦合接口的求解器之间的协同仿真提供默认设置。这包括Abaqus/Standard到Abaqus/Explicit协同仿真，以及Abaqus与某些第三方分析程序之间的协同仿真。

****cosimulation_port****

将**cosimulation_port**设置为用于连接的端口号。默认值为`48000`。

****cosimulation_timeout****

将**cosimulation_timeout**设置为超时时间（以秒为单位）。如果在指定时间内未收到耦合分析程序的任何通信，Abaqus将终止。默认值为`3600`秒。

以下环境文件变量提供允许您为使用协同仿真执行程序的协同仿真作业分配CPU的设置。这包括Abaqus/Standard到Abaqus/Explicit、Abaqus/Standard到Abaqus/CFD和Abaqus/CFD到Abaqus/Explicit协同仿真（请参阅["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD协同仿真执行，" 3.2.4节](pt01ch03s02abx04.md)）。

****cpus_weight_std****

此选项控制分配给Abaqus/Standard分析的CPU。实际CPU分配是按此值以及**cpus_weight_xpl**、**cpus_weight_cfd**和**cpus**的设置成比例进行的。

****cpus_weight_xpl****

此选项控制分配给Abaqus/Explicit分析的CPU。实际CPU分配是按此值以及**cpus_weight_std**、**cpus_weight_cfd**和**cpus**的设置成比例进行的。

****cpus_weight_cfd****

此选项控制分配给Abaqus/CFD分析的CPU。实际CPU分配是按此值以及**cpus_weight_std**、**cpus_weight_xpl**和**cpus**的设置成比例进行的。

****portpool****

将此变量设置为一对用冒号分隔的TCP/UDP端口号，表示协同仿真执行程序在建立子进程之间的连接时使用的端口号范围的起始值和结束值。

### 环境文件示例

下面显示了一些使用前面讨论的参数的环境文件示例。名为`abaqusinc.env`的示例环境文件包含在版本的`site`子目录中，用于显示SIMULIA使用的选项。

#### UNIX环境文件：

```
ask_delete=OFF
# The following parameter causes the scratch files to 
# be written to /tmp.
scratch="/tmp"
```

#### Windows环境文件：

```
ask_delete=OFF
# The following parameter causes the scratch files to 
# be written to the tmp directory on c:.
scratch="c:/tmp"
```
