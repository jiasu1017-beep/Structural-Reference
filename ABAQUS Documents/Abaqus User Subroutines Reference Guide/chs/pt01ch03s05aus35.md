### 概述

Abaqus/CFD中的并行执行：
- 减少需要大量增量的分析运行时间；
- 减少包含大量节点和单元的分析运行时间；
- 产生与分析使用的处理器数量无关的分析结果；和
- 可用于使用基于MPI的域分解并行实现以及共享内存计算机和计算机集群。

### 调用并行处理

Abaqus/CFD对共享内存和分布式内存计算机都使用显式消息传递实现的基于域的并行性。所有Abaqus/CFD提供的过程及其相关功能都是完全并行的（["并行执行：概述，" 第3.5.1节](pt01ch03s05aus32.md)）。通过指定要使用的处理器数量来调用并行执行。

| **输入文件用法：** | 在命令行上输入以下内容： |
| --- | --- |
|  | **abaqus** **job**=*job-name* **cpus**=*n* 例如，以下输入将在两个处理器上运行名为"manifold"的作业：``` abaqus job=manifold cpus=2 ``` |

| **Abaqus/CAE用法：** | 作业模块：作业编辑器：**并行化**：切换开启**使用多个处理器**，并指定处理器数量*n* |
| --- | --- |

### 基于域的并行性

Abaqus/CFD使用域分解消息传递范式实现其并行化。使用基于单元的分解策略，最大限度地减少子域之间所需的通信数量，同时在处理器之间提供几乎均匀的计算工作分配。域的数量精确映射到给定计算的用户指定处理器数量。负载平衡过程也是并行实现的，因此您可以避免在计算开始时耗时的串行负载平衡过程。已尽一切努力确保Abaqus/CFD为广泛的应用提供可扩展的并行解决方案。Abaqus/CFD中的所有过程和功能都提供完全并行的实现。所有输出都为用户自动序列化，因此并行域和原始用户输入之间不需要转换。此外，这允许Abaqus/CFD在任意数量的处理器上无缝重启，不管原始计算使用了多少处理器。

#### 协同仿真

协同仿真技术（["协同仿真：概述，" 第17.1.1节](pt04ch17s01abo17.md)）用于Abaqus/CFD与Abaqus/Standard或Abaqus/Explicit的运行时耦合，可以与串行或并行运行的Abaqus/CFD一起使用。

#### 重启

在重启分析的步骤中定义的功能没有限制。用于重启分析的处理器数量不需要与原始分析中使用的处理器数量相同。

### 输出

没有输出限制。
