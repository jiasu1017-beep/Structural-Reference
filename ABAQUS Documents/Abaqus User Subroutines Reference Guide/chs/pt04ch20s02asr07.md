# 20.2.7 aStudy.output()





### 20.2.7 ***aStudy*.output()**：指定参数化研究结果的来源。

**产品：** Abaqus/Standard  Abaqus/Explicit  

此命令应在任何结果收集命令之前使用。它用于指定从何处收集参数化研究的结果。

##### **参考**

- ["参数化研究脚本，" 第20.1.1节](pt04ch20s01aus108.md)

### **命令：**

***aStudy*.output** (**file**=, **instance**=, **overlay**=, **request**=, **step**= , **frameValue**= | **inc**= | **mode**= )

### **可选数据：**

**file**

将**file**设置为FIL或ODB，以指定必须从结果（`.fil`）文件或输出数据库（`.odb`）文件读取结果。

如果从此命令中省略**file**，将从结果（`.fil`）文件读取结果。

**instance**

此选项仅当从包含多个实例的输出（`.odb`）数据库收集结果时适用。

将**instance**设置为要收集结果的部件实例名称。此名称必须用匹配引号括起来。

如果为在装配上定义的集合收集结果，则不得在此命令和**gather**命令中都指定**instance**。

如果从此命令中省略**instance**且需要实例名称来收集结果，则必须在**gather**命令中指定。

**overlay**

此选项仅当从输出（`.odb`）数据库收集结果时适用。

此选项用于控制内存使用和执行速度之间的权衡。将**overlay**设置为OFF（默认值）或ON，以指定是否应覆盖用于输出数据库的内存。如果内存使用有问题，将**overlay**设置为ON，以指定在执行特定输出数据库的**gather**后必须覆盖用于输出数据库的内存。如果执行速度比内存使用更重要，将**overlay**设置为OFF，以指定必须为每个输出数据库分配内存。

**overlay**选项会影响您单独操作输出数据库（`.odb`）文件的能力。当您将**overlay**设置为OFF时，文件在从**gather**命令访问结果后将保持打开状态，这可能会影响您单独访问或操作文件的能力（例如删除文件）。将**overlay**设置为ON，以便在访问基于**gather**的结果后单独访问`.odb`文件。

**request**

此选项仅当从输出（`.odb`）数据库收集结果时适用。

将**request**设置为FIELD或HISTORY，以指定必须从输出数据库中的场数据还是历史数据收集结果。

如果从此命令中省略**request**，将从场数据收集结果。

**step**

将**step**设置为要收集结果的源分析步骤编号。

如果从此命令中省略**step**，则必须在**gather**命令中指定。

##### **可选且互斥的数据：**

**frameValue**

此选项仅当从输出（`.odb`）数据库收集结果时适用。

将**frameValue**设置为要收集结果的指定分析步骤中分析增量的步骤时间或频率值。**frameValue**也可以设置为符号常量LAST，以指定从步骤的最后一个增量收集结果。如果在指定的**frameValue**没有可用结果，将发出警告并从最接近的增量收集结果。

如果从此命令中省略**frameValue**，则从步骤的最后一个增量收集结果，或从**gather**命令中指定的增量收集结果。

**inc**

将**inc**设置为要收集结果的非频率分析步骤的分析增量编号。**inc**也可以设置为符号常量LAST，以指定从指定步骤的最后一个增量收集结果。

如果从此命令中省略**inc**，则从步骤的最后一个增量收集结果，或从**gather**命令中指定的增量收集结果。

此选项不适用于从输出（`.odb`）数据库收集历史结果。

**mode**

将**mode**设置为要跨参数化研究变体收集结果的指定频率分析步骤的模态编号。

如果省略**mode**，则从**gather**命令中指定的模态收集结果，或从步骤的第一个模态收集结果。
