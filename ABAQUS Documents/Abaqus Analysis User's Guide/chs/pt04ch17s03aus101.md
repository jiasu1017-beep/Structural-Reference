### 概述

本节讨论特定于使用 Abaqus/Standard 程序的电磁到结构和电磁到热协同仿真的分析设置和执行详细信息。支持时谐或瞬态电磁分析与静态、瞬态隐式动态、耦合温度-位移或瞬态热传递分析之间的协同仿真。

电磁到热传递协同仿真分析适用于感应加热等应用，涉及双向耦合：来自电磁分析的焦耳热产生驱动热传递分析并确定温度分布，而温度分布又通过温度依赖性材料属性（如电导率和磁导率）影响电磁场。对于电磁-热耦合，支持时谐或瞬态电磁分析与瞬态热传递分析之间的协同仿真。

电磁到瞬态隐式动态协同仿真适用于电磁成型等应用，其中来自电磁分析的洛伦兹体力驱动瞬态动态分析。支持瞬态电磁分析与静态或瞬态隐式动态分析之间的协同仿真。然而，耦合是单向的；即，域（在本例中为金属工件）变形对电磁场的影响未被考虑。因此，仅当变形对电磁场的影响相对较小时才应使用此类分析。

### 识别协同仿真接口区域

电磁和结构模型之间的相互作用通过公共体积接口区域发生。

您必须使用元素集在 Abaqus/Standard 分析之间指定体积接口区域。您必须在两个 Abaqus/Standard 模拟中的区域定义保持一致；换句话说，您必须在两个分析中定义相同的接口区域。

您可以在两个模型定义中共享的区域中使用不同的网格。

| **输入文件用法：** | 使用以下选项在 Abaqus/Standard 模型中定义基于元素的协同仿真区域： |
| --- | --- |
| | ``` [*CO-SIMULATION REGION](../key/key-link.md#usb-kws-hcosimulationregion), TYPE=VOLUME *elset_A* ``` 每个 Abaqus/Standard 分析中只能定义一个 [*CO-SIMULATION REGION](../key/key-link.md#usb-kws-hcosimulationregion) 选项。此外，只能定义一个元素集。 |

### 定义会合方案

SIMULIA 协同仿真引擎配置文件用于定义时间积分过程和两个 Abaqus/Standard 分析之间交换的频率。

常用耦合方案有预定义模板可用。创建配置文件时，您可以参考这些模板。本节描述会合方案设置和预定义配置文件模板。

#### 时间积分方案

您可以强制两个瞬态 Abaqus/Standard 分析使用相同的增量大小，或者您可以允许增量大小不同（subcycling）。您为耦合选择的时间积分方案会影响解决方案的计算成本和准确性，但不会影响解决方案稳定性。当使用 subcycling 方法时，此数据交换不表示对 Abaqus/Standard 增量的约束；Abaqus/Standard 分析使用其正常时间增量逻辑推进时间，但根据需要在耦合步骤大小间隔执行数据交换。

时谐电磁过程在频域中定义，与瞬态分析不同，没有与之关联的解决方案时间尺度。引入与协同仿真分析中涉及的时谐电磁程序相关联的伪解决方案时间尺度是方便的，从而便于在后一分析的某些解决方案时间间隔处与瞬态分析耦合。时谐电磁分析的伪时间尺度遵循瞬态热传递分析中的解决方案时间尺度，并在每个耦合步骤中按以下描述的方式重置。

#### 定义耦合方案

顺序显式耦合方案（也称为高斯-塞德尔耦合算法）和迭代耦合方案可用于电磁到结构和电磁到热协同仿真。电磁分析必须始终主导协同仿真，而热传递或应力分析始终滞后于协同仿真。所有预定义模板都使用上述主导-滞后序列设置。

#### 耦合步骤大小

耦合步骤大小是两次连续协同仿真数据交换之间的时间段。对于瞬态电磁到瞬态热传递或瞬态隐式动态协同仿真，耦合步骤大小可指定为等于各分析自动时间增量方案确定的时间步长大小的最小值，或等于恒定的用户定义值。

当主导电磁分析为时谐时，耦合步骤大小可指定为等于滞后瞬态热传递或隐式动态分析的时间步长大小，或等于恒定的用户定义值。在后一种情况下，时谐电磁分析将求解每个连续恒定用户定义耦合步骤大小结束时的场，而滞后热传递或应力分析通常将进行 subcycling，直到达到目标耦合步骤时间。

对于迭代耦合，两个分析必须在每个增量结束时耦合，不应使用 subcycling。如果在此情况下使用 subcycling，则在迭代期间交换的更新解决方案将仅用于最后一个增量，而之前增量（上次耦合和当前耦合之间）更新的累积效应将丢失。

```
abaqus fetch job=exa_em_std_export
```
示例文件 `exa_em_std_export.xml` 如下所示。
```
<?xml version="1.0" encoding="utf-8"?>
<CoupledMultiphysicsSimulation>
   <template_em_std_export>
      <EM_Job>*em_job_name*</EM_Job>
      <HeatTransfer_Job>*ht_job_name*</HeatTransfer_Job>

</CoupledMultiphysicsSimulation>
```
