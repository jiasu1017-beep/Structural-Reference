**产品：**Abaqus/Standard  Abaqus/CAE

### 概述

多载荷工况分析：
- 用于研究结构在步骤内定义的 distinct 载荷集和边界条件集（每个集称为一个载荷工况）作用下的线性响应；
- 可能比等效的多扰动步骤分析更高效；
- 允许在载荷工况之间改变机械载荷和边界条件；
- 包括基础状态的影响；并且
- 可以与静态扰动、直接求解稳态动态和基于SIM的稳态动态分析一起执行。

### 载荷工况

载荷工况是指包含特定加载条件的载荷集、边界条件和基础运动的集合。例如，在简化的飞机模型中，飞机操作环境可分为五个载荷工况：（1）起飞，（2）爬升，（3）巡航，（4）下降，和（5）着陆。通常，载荷工况以单位载荷或指定边界条件来定义，多载荷工况分析指的是同时求解载荷工况集合中每个载荷工况的响应。这些响应可以在后处理过程中进行缩放和线性组合，以表示实际加载环境。载荷工况的其他后处理操作也很常见，例如找出所有载荷工况中的最大Mises应力。可以在Abaqus/CAE的Visualization模块中请求此类载荷工况操作（参见[Abaqus/CAE User's Guide](../usi/usi-link.md#usi)）。

### 使用多载荷工况

多载荷工况分析在概念上等价于将载荷工况定义映射到连续扰动步骤的多步骤分析。然而，多载荷工况分析通常比等效的多步骤分析更高效。例外情况是存在大量不适用于所有载荷工况的边界条件时（即某些自由度在一个载荷工况中被约束但在另一个中未约束）。很难定义"大量"是多少，因为它取决于模型。两种分析方法的相对性能可以通过对多载荷工况分析和等效多步骤分析执行数据检查分析来评估。数据检查分析将每个步骤的资源信息写入数据文件，包括最大波前、浮点运算次数和所需的最小内存。如果多载荷工况步骤的这些数字相对于等效多步骤分析的所有步骤明显更大（在比较之前应将浮点运算次数相加），则多步骤分析将更高效。

虽然通常更高效，但多载荷工况分析可能比等效的多步骤分析消耗更多内存和磁盘空间。因此，对于大型问题或具有很多载荷工况的问题，强烈建议如上所述比较多载荷工况分析与等效多步骤分析之间的资源使用情况。如果多载荷工况分析的资源需求被认为太大，可以考虑将载荷工况分配到几个步骤中。结果分析（多载荷工况和多步骤的混合）将需要更少的资源，同时保留相对于纯多步骤分析的效率优势。

### 定义载荷工况

您可以在静态扰动、直接求解稳态动态和基于SIM的稳态动态分析中定义载荷工况。载荷工况定义不会传播到后续步骤。只有以下类型的指定条件可以在载荷工况定义中指定：
- 边界条件
- 集中载荷
- 分布载荷
- 分布表面载荷
- 基于惯性的载荷
- 基础运动

以下章节描述了管理这些指定条件的附加规则。在包含载荷工况定义的步骤中，不能出现其他类型的指定条件。所有其他有效的分析组件（如输出请求）必须在载荷工况定义之外指定。

每个载荷工况定义都被分配一个名称用于后处理。

| **输入文件用法：** | 使用第一个选项开始一个载荷工况，使用第二个选项结束一个载荷工况： |
| --- | --- |
|  | ``` [*LOAD CASE](../key/key-link.md#usb-kws-hloadcase), NAME=*name* [*END LOAD CASE](../key/key-link.md#usb-kws-hendloadcase) ``` 在载荷工况定义中指定的指定条件仅适用于该载荷工况。在静态扰动和直接求解稳态动态分析中，可以在载荷工况定义之外指定指定条件（在这种情况下，它们适用于步骤中的所有载荷工况）。 |

| **Abaqus/CAE用法：** | Load模块：**Create Load Case**：**Name**：*name* |
| --- | --- |
|  | 在Abaqus/CAE中，如果步骤包含载荷工况，则步骤中的所有指定条件必须包含在一个或多个载荷工况中。 |

### 分析过程

载荷工况只能在具有以下过程的扰动步骤中定义：
- Static
- Direct-solution, steady-state dynamic
- SIM-based, steady-state dynamic

与其他扰动步骤一样，多载荷工况分析将包括先前常规步骤（基础状态）的非线性效应。以下分析技术不支持在载荷工况步骤上下文中使用：
- 从特定载荷工况重启
- 使用除全局分析中第一个载荷工况之外的其他载荷工况的结果进行子模型分析
- 导入和传输结果
- 循环对称分析
- 等值积分
- 设计灵敏度分析

### 边界条件

在同一个步骤中，边界条件可以在载荷工况定义之外和之内指定。在步骤中的载荷工况定义之外指定边界条件等效于将其包含在步骤中的所有载荷工况定义中（即，边界条件将应用于所有载荷工况）。除非任何边界条件在扰动步骤中被移除，否则在基础状态中活动的边界条件将传播到扰动步骤中的所有载荷工况。如果在具有载荷工况的步骤中移除了任何边界条件（无论在载荷工况定义之外还是之内），基础状态边界条件将不会传播到步骤中的任何载荷工况。更多信息请参见["Abaqus/Standard和Abaqus/Explicit中的边界条件，" 34.3.1节](pt07ch34s03aus118.md)。

**注意：**在Abaqus/CAE中，如果步骤包含载荷工况，则步骤中的所有边界条件必须包含在一个或多个载荷工况中。边界条件只能与静态扰动和直接求解稳态动态分析中的载荷工况一起使用。

### 载荷

在静态扰动和直接求解稳态动态分析中，集中载荷、分布载荷和分布表面载荷可以在同一个步骤中的载荷工况定义之外和之内指定。惯性relief载荷可以在载荷工况定义之外或之内指定，但不能同时两者都指定。在步骤中的载荷工况定义之外指定这些载荷类型之一等效于将其包含在步骤中的所有载荷工况定义中（即，加载将应用于所有载荷工况）。

在基于SIM的稳态动态分析中，集中载荷、分布载荷、分布表面载荷和基础运动只能在同一个步骤中的载荷工况定义之内指定。不支持惯性relief载荷。

载荷工况不能用于包含aqua载荷的模型（参见["Abaqus/Aqua分析，" 6.11.1节](pt03ch06s11at30.md))。

与任何扰动步骤一样，扰动载荷必须完全在扰动步骤内定义（参见["应用载荷：概述，" 34.4.1节](pt07ch34s04aus120.md)）。

**注意：**在Abaqus/CAE中，如果步骤包含载荷工况，则步骤中的所有载荷必须包含在一个或多个载荷工况中。

### 预定义场

场变量不能在具有载荷工况的步骤中指定。

### 单元

载荷工况不能用于包含压电元件的模型（参见["压电分析，" 6.7.2节](pt03ch06s07at21.md)）。

### 输出

在包含一个或多个载荷工况的步骤中，支持对输出数据库的场和历史输出请求以及对数据文件的输出请求。不支持对结果文件的输出请求。输出请求只能在载荷工况定义之外指定，它们适用于步骤中的所有载荷工况。输出请求的步骤传播规则与其他扰动步骤相同（参见["输出，" 4.1.1节](pt02ch04s01aus38.md)）。

在特定分析过程中通常可用的大多数场和历史输出变量在多载荷工况分析期间也可用（参见["Abaqus/Standard输出变量标识符，" 4.2.1节](pt02ch04s02abv01.md)）。对于基于SIM的稳态动态分析适用额外的限制；更多信息请参见["动态分析过程概述中的使用SIM架构进行模态叠加动态分析，" 6.3.1节](pt03ch06s03abo07.md#usb-anl-alineardynamics)。

对应于每个载荷工况的场输出存储在输出数据库的单独框架中，载荷工况名称作为框架属性包含在内。为了区分历史输出变量的载荷工况，载荷工况名称被附加到历史变量名称之后。可以使用Abaqus/CAE的Visualization模块和Abaqus Scripting Interface（参见[Abaqus Scripting User's Guide第9章，"使用Abaqus Scripting Interface访问输出数据库"](../cmd/cmd-link.md#cmd-odb-api-pyc)）来访问和操作载荷工况输出。Abaqus/Standard不对载荷工况操作的物理有效性执行一致性检查。例如，即使组合结果可能没有物理意义，也允许具有不同边界条件的两个载荷工况的线性叠加。
