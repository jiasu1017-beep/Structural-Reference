### 概述

接触属性：
- 定义控制表面接触时行为的机械表面相互作用模型；和
- 可选择性地应用于一般接触域内的特定区域。

### 分配接触属性

Abaqus/Explicit中的默认接触属性模型假定法向"硬"接触、无摩擦、无热相互作用等。您可以为一般接触域的指定区域分配非默认接触属性定义（表面相互作用）。

接触属性分配在一般接触相互作用活动的所有分析步骤中传播。

用于指定应分配非默认接触属性的区域的表面名称不必与用于指定一般接触域的表面名称对应。在许多情况下，接触相互作用将为大域定义，而非默认接触属性将分配给该域的子集。任何落在一般接触域之外的区域的接触属性分配将被忽略。如果指定区域重叠，最后分配优先。

| **输入文件用法：** | ``` [*CONTACT PROPERTY ASSIGNMENT*](../key/key-link.md#usb-kws-hcontpropassign) *surface_1*, *surface_2*, *interaction_property_name* ``` |
| --- | --- |
| | 此选项必须与[*CONTACT*](../key/key-link.md#usb-kws-hcontact)选项结合使用。每个步骤最多应出现一次；数据行可以根据需要重复，以将接触属性分配给不同区域。如果省略第一个表面名称，则假定为包含整个一般接触域的默认表面。如果省略第二个表面名称或与第一个表面名称相同，则假定为第一个表面与其自身的接触。请记住，表面可以定义为跨越多个未连接的实体，因此自接触不限于单个实体与自身的接触。如果省略相互作用属性名称，则假定为Abaqus/Explicit中默认接触属性的未命名集。如果指定了相互作用属性名称，它也必须作为输入文件模型部分中[*SURFACE INTERACTION*](../key/key-link.md#usb-kws-hsurfaceinteraction)选项上NAME参数的值出现。 |

| **Abaqus/CAE用法：** | 相互作用模块：**创建相互作用**：**一般接触（Explicit）**：**接触属性**：**单个属性分配**：**编辑**：在左侧列中选择表面和接触属性，点击中间的箭头将它们转移到接触属性分配列表，或**全局属性分配**：*interaction_property_name* |
| --- | --- |
| | 在Abaqus/CAE中，您必须为每个一般接触相互作用分配接触属性定义；Abaqus/CAE不假定默认接触相互作用属性。 |

#### 示例

以下接触属性分配在一般接触分析的第一步中指定：
- 将`contProp1`全局分配给整个一般接触域；
- 将`contProp2`局部分配给`surf1`的自接触；
- 将默认Abaqus接触属性局部分配给`surf2`和`surf3`之间的接触；和
- 将`contProp3`局部分配给整个接触域与`surf4`之间的接触。

```
[*SURFACE INTERACTION*](../key/key-link.md#usb-kws-hsurfaceinteraction), NAME=contProp1
[*FRICTION*](../key/key-link.md#usb-kws-hfriction)
0.1
[*SURFACE INTERACTION*](../key/key-link.md#usb-kws-hsurfaceinteraction), NAME=contProp2
[*FRICTION*](../key/key-link.md#usb-kws-hfriction)
0.15
[*SURFACE INTERACTION*](../key/key-link.md#usb-kws-hsurfaceinteraction), NAME=contProp3
[*FRICTION*](../key/key-link.md#usb-kws-hfriction)
0.20
[*STEP*](../key/key-link.md#usb-kws-hstep)
Step1
[*DYNAMIC*](../key/key-link.md#usb-kws-hdynamic), EXPLICIT
…
[*CONTACT*](../key/key-link.md#usb-kws-hcontact)
[*CONTACT INCLUSIONS*](../key/key-link.md#usb-kws-hcontactinclusions), ALL EXTERIOR
[*CONTACT PROPERTY ASSIGNMENT*](../key/key-link.md#usb-kws-hcontpropassign)
 , , contProp1 
surf1, surf1, contProp2 
surf2, surf3,
 , surf4, contProp3 
```

### 更改接触属性

一般接触相互作用的接触属性模型独立于使用它们的步骤，不能跨步修改。要更改给定步骤中使用的接触属性，您必须指定引用不同接触属性模型的新接触属性分配。

#### 示例

例如，以下输入可用于更改在上一个示例开始的分析第二步中用于整个一般接触域与`surf4`之间接触的摩擦系数：

```
[*STEP*](../key/key-link.md#usb-kws-hstep)
Step2
[*DYNAMIC*](../key/key-link.md#usb-kws-hdynamic), EXPLICIT
…
[*CONTACT*](../key/key-link.md#usb-kws-hcontact)
[*CONTACT PROPERTY ASSIGNMENT*](../key/key-link.md#usb-kws-hcontpropassign)
 , surf4, contProp2
```
