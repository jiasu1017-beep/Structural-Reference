# 82.6.4 计算模态贡献因子





第一个插件计算所有请求的量（MCF），然后将它们作为历史输出添加到输出数据库。该插件只能从 Visualization 模块访问。进入 Visualization 模块并通过从主菜单栏选择 ****Plug-ins**![](../graphics/images/arrow.gif)**NVH**![](../graphics/images/arrow.gif)**Compute modal Contribution Factors**** 来启动插件。

插件显示 **Open ODB** 对话框，您输入由 ["准备结构和/或声学数据，" 第82.6.2节](pt08ch82s06s02.md) 中描述的 Abaqus/Standard 分析生成的输出数据库的名称。插件将计算出的模态贡献因子值添加回同一输出数据库；您必须对输出数据库具有写访问权限。

输入输出数据库名称后，插件显示 **Compute MCF** 对话框，如 [图82-1](pt08ch82s06s04.md#usi-plg-computemcf) 所示。

**图82-1** **Compute MCF** 对话框。

![](../graphics/usi-plg-computemcf.png)

您必须从 **Compute MCF** 对话框中选择一个有效的频率步长。有效的频率步长是后面跟着模态稳态动态步长的步长（不是稳态动态、直接或稳态动态、子空间）。此外，有效的频率步长包括至少一个模态的 GU 和 GPU 输出。您选择的每个频率步长的有效模态稳态动态步长显示在 **Valid SSD Steps** 列表中。

**Compute MCF** 对话框还要求提供 **Response Points**。您可以选择以下格式的任意组合：
- 零件实例名称后跟节点标签列表
- 零件实例名称后跟实例级节点集列表
- 装配级节点集

最后，您必须选择场输出变量，可以是以下任一选项：声压（POR）、位移（U）、速度（V）或加速度（A）。您只能从频率步长请求 U 或 POR。对于除 POR 之外的所有输出变量，您还必须请求相应的分量标签。例如，如果您请求 U，则还必须请求 U1、U2 或 U3 之一。

点击 **OK** 关闭 **Compute MCF** 对话框后，插件会处理所选频率步长的所有有效模态稳态动态步长，以及所有指定的响应点和频率步长中的所有模态。
