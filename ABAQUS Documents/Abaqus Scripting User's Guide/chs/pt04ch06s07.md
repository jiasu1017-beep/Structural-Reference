# 6.7 从GUI访问kernel数据

您可以使用`abaqusGui`模块或`kernelAccess`模块从Abaqus/CAE的GUI访问kernel mdb和session对象。您以相同的方式从每个模块访问对象：

```
from abaqusGui import mdb, session
```

或

```
from kernelAccess import mdb, session
```

在每种情况下，导入的对象都是kernel中实际对象的代理。

您可以查询`abaqusGui`模块的mdb和session代理对象的属性，但不能将它们用于任意方法调用（允许使用repository方法，如`keys()`、`values()`和`items()`）。`abaqusGui`代理对象定期从kernel更新，访问它们是进程内函数调用（快速）。但是，在某些情况下，代理对象可能会过时。例如，当脚本正在运行时，代理对象不会更新，直到脚本结束。

您可以使用`kernelAccess`模块的mdb和session代理对象执行任何Abaqus Scripting Interface kernel命令。除了查询kernel对象的属性外，您还可以调用它们的方法并获取任何返回值，就像您在kernel中执行代码一样。`kernelAccess`代理对象始终是最新的，因为访问它们会同步调用kernel对象，创建进程间通信（IPC）流量。当您使用`kernelAccess`代理对象而不是`abaqusGui`模块代理对象时，这种与kernel的即时交互会产生性能劣势。例如，调用Part对象的`getVolume`方法：

```
from kernelAccess import mdb, session
partNames = mdb.models['Model-1'].parts.keys()
v = mdb.models['Model-1'].parts['Part-1'].getVolume()
```

此过程通过IPC机制进行GUI-kernel通信，因此不建议在性能敏感的地方使用。换句话说，您应该仅将此过程用于访问数据或调用不需要"长时间"执行的方法。如果性能确实成为问题，您可以从`abaqusGui`模块而不是`kernelAccess`模块访问mdb和session对象。

尽管您可以在应用程序启动脚本完成之前执行的脚本中导入`kernelAccess`模块，但您无法在应用程序启动脚本完成之前查询mdb和session对象。换句话说，您可以在GUI初始构建期间执行的代码中在脚本中导入`kernelAccess`模块；但是，您不应该尝试访问mdb或session对象，直到由于GUI中的某些用户交互而需要它为止。有关更多信息，请参阅["启动脚本，" 第11.2节](pt06ch11s02.md)。
