# 2.4 隐式与显式程序的比较

Abaqus/Standard 和 Abaqus/Explicit 能够求解多种问题。隐式和显式方法的特性决定了哪种方法适用于特定问题。对于两种方法都可以求解的问题，求解效率决定了应使用哪个产品。理解隐式和显式方法的特性将帮助您回答这个问题。[表 2–2] 列出了分析产品之间的主要差异，这些差异将在本指南的相关章节中详细讨论。

**表 2–2** Abaqus/Standard 与 Abaqus/Explicit 之间的主要差异。

| 量 | Abaqus/Standard | Abaqus/Explicit |
|---|---|---|
| 单元库 | 提供广泛的单元库。 | 提供非常适合显式分析的广泛单元库。可用的单元是 Abaqus/Standard 中可用单元的子集。 |
| 分析过程 | 提供通用和线性扰动过程。 | 提供通用过程。 |
| 材料模型 | 提供广泛的材料模型。 | 与 Abaqus/Standard 中可用的材料模型类似；一个显著差异是允许使用失效材料模型。 |
| 接触公式 | 具有求解接触问题的强大能力。 | 具有强大的接触功能，可轻松求解即使最复杂的接触模拟。 |
| 求解技术 | 使用基于刚度的求解技术，该技术无条件稳定。 | 使用显式积分求解技术，该技术有条件稳定。 |
| 磁盘空间和内存 | 由于增量中可能的迭代次数较多，磁盘空间和内存使用量可能很大。 | 磁盘空间和内存使用量通常比 Abaqus/Standard 小得多。 |

## 2.4.1 在隐式分析和显式分析之间选择

对于许多分析，很明显应该使用 Abaqus/Standard 还是 Abaqus/Explicit。例如，正如 [第 8 章，"非线性"] 中所示，Abaqus/Standard 更高效地求解光滑非线性问题；另一方面，Abaqus/Explicit 是波传播分析的明确选择。然而，某些静态或准静态问题可以用任一程序很好地模拟。通常，这些问题通常使用 Abaqus/Standard 求解，但由于接触或材料复杂性可能导致收敛困难，从而产生大量迭代。这些分析在 Abaqus/Standard 中成本很高，因为每次迭代需要求解大量线性方程组。

Abaqus/Standard 必须通过迭代来确定非线性问题的解，而 Abaqus/Explicit 通过**显式**地从上一个增量的结束时刻推进运动状态来确定解，而无需迭代。即使给定的分析使用显式方法可能需要大量时间增量，如果同一分析在 Abaqus/Standard 中需要多次迭代，则 Abaqus/Explicit 分析可能更高效。

Abaqus/Explicit 的另一个优势是它对于相同模拟比 Abaqus/Standard 需要更少的磁盘空间和内存。对于两个程序的计算成本可能相当的问题，Abaqus/Explicit 节省的大量磁盘空间和内存使其具有吸引力。

## 2.4.2 隐式和显式分析中网格细化的成本

使用显式方法，计算成本与单元数量成正比，与最小单元尺寸大致成反比。因此，网格细化通过增加单元数量和减小最小单元尺寸来增加计算成本。例如，考虑一个具有均匀方形单元的三维模型。如果网格在所有三个方向上细化两倍，计算成本增加两倍，这是由于单元数量增加所致，而最小单元尺寸减小又增加两倍。网格细化后，分析的总计算成本增加 2^4 倍，即 16 倍。磁盘空间和内存需求与单元数量成正比，与单元尺寸无关；因此，这些需求增加 8 倍。

虽然显式方法的网格细化成本增加预测相当简单，但使用隐式方法时成本更难预测。困难来自于单元连接性与求解成本之间的问题依赖关系，这种关系在显式方法中不存在。使用隐式方法，经验表明对于许多问题，计算成本大致与自由度数量的平方成正比。考虑相同的具有均匀方形单元的三维模型示例。在所有三个方向上细化网格两倍会使自由度数量增加约 2^3 倍，导致计算成本增加约 (2^3)^2 倍，即 64 倍。磁盘空间和内存需求以相同方式增加，尽管实际增加难以预测。
