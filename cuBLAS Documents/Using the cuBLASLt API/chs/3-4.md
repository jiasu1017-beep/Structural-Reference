## 3.4. cuBLASLt API 参考文档

### 3.4.1. cublasLtCreate()

```c++

cublasStatus_t
      cublasLtCreate(cublasLtHandle_t *lighthandle)

```

此函数用于初始化 cuBLASLt 库，并创建一个指向持有 cuBLASLt 库上下文的不透明结构的句柄。该函数会在主机和设备上分配轻量级硬件资源，且必须在调用任何其他 cuBLASLt 库函数之前执行。

cuBLASLt 库上下文与当前 CUDA 设备绑定。要在多个设备上使用该库，需要为每个设备创建一个 cuBLASLt 句柄。此外，在使用与特定设备绑定的句柄调用 cuBLASLt 函数之前，必须将该设备设置为当前设备。

另请参见：[cuBLAS Context](#cublas-context)。

**参数：**

| 参数 | 内存 | 输入/输出 | 描述 |
| --- | --- | --- | --- |
| lightHandle |  | 输出 | 指向为创建的 cuBLASLt 上下文所分配的 cuBLASLt 句柄的指针。 |

**返回：**

| 返回值 | 描述 |
| --- | --- |
| CUBLAS_STATUS_SUCCESS | 分配成功完成。 |
| CUBLAS_STATUS_NOT_INITIALIZED | cuBLASLt 库未初始化。通常发生在以下情况：<br>未首先调用 cublasLtCreate()<br>cuBLASLt 例程调用的 CUDA Runtime API 发生错误，或<br>硬件设置错误。 |
| CUBLAS_STATUS_ALLOC_FAILED | cuBLASLt 库内部的资源分配失败。这通常由 cudaMalloc() 失败引起。<br>解决方法：在函数调用之前，尽可能释放先前分配的内存。 |
| CUBLAS_STATUS_INVALID_VALUE | lighthandle 为 NULL |

有关有效返回码的完整列表，请参阅 [cublasStatus_t](#cublasstatus-t)。

### 3.4.2. cublasLtDestroy()

```c++

cublasStatus_t
      cublasLtDestroy(cublasLtHandle_t lightHandle)

```

此函数用于释放 cuBLASLt 库使用的硬件资源。此函数通常是对特定句柄调用 cuBLASLt 库的最后一个调用。由于 cublasLtCreate() 会分配一些内部资源，而调用 cublasLtDestroy() 释放这些资源时将隐式调用 `cudaDeviceSynchronize()`，因此建议尽量减少调用这些函数的次数。

**参数：**

| 参数 | 内存 | 输入/输出 | 描述 |
| --- | --- | --- | --- |
| lightHandle |  | 输入 | 指向要销毁的 cuBLASLt 句柄的指针。 |

**返回：**

| 返回值 | 含义 |
| --- | --- |
| CUBLAS_STATUS_SUCCESS | cuBLASLt 上下文已成功销毁。 |
| CUBLAS_STATUS_NOT_INITIALIZED | cuBLASLt 库未初始化。 |
| CUBLAS_STATUS_INVALID_VALUE | lightHandle 为 NULL |

有关有效返回码的完整列表，请参阅 [cublasStatus_t](#cublasstatus-t)。

支持的标志：

**返回：** mask 的前一个值。

### 3.4.17. cublasLtMatmul()

```c++

cublasStatus_t cublasLtMatmul(
      cublasLtHandle_t               lightHandle,
      cublasLtMatmulDesc_t           computeDesc,
      const void                    *alpha,
      const void                    *A,
      cublasLtMatrixLayout_t         Adesc,
      const void                    *B,
      cublasLtMatrixLayout_t         Bdesc,
      const void                    *beta,
      const void                    *C,
      cublasLtMatrixLayout_t         Cdesc,
      void                          *D,
      cublasLtMatrixLayout_t         Ddesc,
      const cublasLtMatmulAlgo_t    *algo,
      void                          *workspace,
      size_t                         workspaceSizeInBytes,
      cudaStream_t                   stream);

```

此函数根据以下操作计算矩阵 A 和 B 的矩阵乘法以产生输出矩阵 D：

`D = alpha*(A*B) + beta*(C),`

其中 `A`、`B` 和 `C` 是输入矩阵，`alpha` 和 `beta` 是输入标量。

> **注意**

注意
此函数同时支持原地矩阵乘法（C == D 且 Cdesc == Ddesc）和非原地矩阵乘法（C != D，两个矩阵必须具有相同的数据类型、行数、列数、批次大小和内存顺序）。在非原地情况下，C 的主维度可以不同于 D 的主维度。具体来说，C 的主维度可以为 0 以实现行或列广播。如果省略 Cdesc，则此函数假定它等于 Ddesc。

`workspace` 指针必须至少按 256 字节的倍数对齐。
关于 `workspaceSizeInBytes` 的建议与 [cublasSetWorkspace()](#cublassetworkspace) 部分中提到的相同。

**支持的数据类型：**

[cublasLtMatmul()](#cublasltmatmul) 支持以下 computeType、scaleType、Atype/Btype 和 Ctype。脚注可在本节末尾找到。

| computeType | scaleType | Atype/Btype | Ctype | Bias Type 6 |
| --- | --- | --- | --- | --- |
| CUBLAS_COMPUTE_16F 或<br>CUBLAS_COMPUTE_16F_PEDANTIC | CUDA_R_16F | CUDA_R_16F | CUDA_R_16F | CUDA_R_16F 6 |
| CUBLAS_COMPUTE_32I 或<br>CUBLAS_COMPUTE_32I_PEDANTIC | CUDA_R_32I | CUDA_R_8I | CUDA_R_32I | 不支持后处理。 |
| CUDA_R_32F | CUDA_R_8I | CUDA_R_8I | 不支持后处理。 |
| CUBLAS_COMPUTE_32F 或<br>CUBLAS_COMPUTE_32F_PEDANTIC | CUDA_R_32F | CUDA_R_16BF | CUDA_R_16BF | CUDA_R_16BF 6 |
| CUDA_R_16F | CUDA_R_16F | CUDA_R_16F 6 |
| CUDA_R_8I | CUDA_R_32F | 不支持后处理。 |
| CUDA_R_16BF | CUDA_R_32F | CUDA_R_32F 6 |
| CUDA_R_16F | CUDA_R_32F | CUDA_R_32F 6 |
| CUDA_R_32F | CUDA_R_32F | CUDA_R_32F 6 |
| CUDA_C_32F 7 | CUDA_C_8I 7 | CUDA_C_32F 7 | 不支持后处理。 |
| CUDA_C_32F 7 | CUDA_C_32F 7 |
| CUBLAS_COMPUTE_32F_FAST_16F 或<br>CUBLAS_COMPUTE_32F_FAST_16BF 或<br>CUBLAS_COMPUTE_32F_FAST_TF32 或<br>CUBLAS_COMPUTE_32F_EMULATED_16BFX9 | CUDA_R_32F | CUDA_R_32F | CUDA_R_32F | CUDA_R_32F 6 |
| CUDA_C_32F 7 | CUDA_C_32F 7 | CUDA_C_32F 7 | 不支持后处理。 |
| CUBLAS_COMPUTE_64F 或<br>CUBLAS_COMPUTE_64F_PEDANTIC 或<br>CUBLAS_COMPUTE_64F_EMULATED_FIXEDPOINT | CUDA_R_64F | CUDA_R_64F | CUDA_R_64F | CUDA_R_64F 6 |

要使用 IMMA 内核，必须满足以下要求集之一（首选第一个）：

1. 使用常规数据排序：

所有矩阵指针必须 4 字节对齐。为获得更好的性能，此条件应以 16 代替 4。
矩阵 A、B、C 的主维度必须是 4 的倍数。
仅支持"TN"格式 - A 必须转置，B 不转置。
指针模式可以是 CUBLASLT_POINTER_MODE_HOST、CUBLASLT_POINTER_MODE_DEVICE 或 CUBLASLT_POINTER_MODE_ALPHA_DEVICE_VECTOR_BETA_HOST。使用后一种模式时，内核支持 CUBLASLT_MATMUL_DESC_ALPHA_VECTOR_BATCH_STRIDE 属性。
维度 m 和 k 必须是 4 的倍数。
2. 在 Ampere（计算能力 8.0）或 Turing（计算能力 7.5）（但不适用于 Hopper，计算能力 9.0 或更高版本）架构上使用 IMMA 特定数据排序 - A、C、D 矩阵使用 CUBLASLT_ORDER_COL32，B 矩阵使用 CUBLASLT_ORDER_COL4_4R2_8C（在 Turing 或 Ampere 架构上）或 CUBLASLT_ORDER_COL32_2R_4R4（在 Ampere 架构上）：

矩阵 A、B、C 的主维度必须满足与内存排序相关的特定条件（请参阅 cublasLtOrder_t）。
矩阵乘法描述符必须在 B 矩阵上指定 CUBLAS_OP_T，在 A 和 C 矩阵上指定 CUBLAS_OP_N（默认）。
如果使用 scaleType CUDA_R_32I，则 alpha 和 beta 唯一支持的值是 0 或 1。
指针模式可以是 CUBLASLT_POINTER_MODE_HOST、CUBLASLT_POINTER_MODE_DEVICE、CUBLASLT_POINTER_MODE_DEVICE_VECTOR 或 CUBLASLT_POINTER_MODE_ALPHA_DEVICE_VECTOR_BETA_ZERO。这些内核不支持 CUBLASLT_MATMUL_DESC_ALPHA_VECTOR_BATCH_STRIDE。
仅支持"NT"格式 - A 不转置，B 转置。

### 3.4.42. cublasLtMatrixTransform()



```c++

cublasStatus_t cublasLtMatrixTransform(
      cublasLtHandle_t lightHandle,
      cublasLtMatrixTransformDesc_t transformDesc,
      const void *alpha,
      const void *A,
      cublasLtMatrixLayout_t Adesc,
      const void *beta,
      const void *B,
      cublasLtMatrixLayout_t Bdesc,
      void *C,
      cublasLtMatrixLayout_t Cdesc,
      cudaStream_t stream);


```


此函数根据以下操作对输入矩阵 A 和 B 执行矩阵变换操作以产生输出矩阵 C：


`C = alpha*transformation(A) + beta*transformation(B),`


其中 A、B 是输入矩阵，alpha 和 beta 是输入标量。变换操作由 transformDesc 指针定义。此函数可用于更改数据的内存顺序或对值进行缩放和偏移。


**参数**：


| 参数 | 内存 | 输入/输出 | 描述 |

### 3.4.1. cublasLtCreate()



```c++

cublasStatus_t
      cublasLtCreate(cublasLtHandle_t *lighthandle)


```


此函数初始化 cuBLASLt 库并创建一个包含 cuBLASLt 库上下文的不透明结构句柄。它在主机和设备上分配轻量级硬件资源，必须在任何其他 cuBLASLt 库调用之前调用。


cuBLASLt 库上下文与当前 CUDA 设备绑定。要在多个设备上使用库，必须为每个设备创建一个 cuBLASLt 句柄。此外，在使用与特定设备绑定的句柄调用 cuBLASLt 函数之前，必须将该设备设置为当前设备。


另请参阅：cuBLAS Context。


**参数：**


| 参数 | 内存 | 输入/输出 | 描述 |
| --- | --- | --- | --- |
| lightHandle |  | 输出 | 指向为创建的 cuBLASLt 上下文分配的 cuBLASLt 句柄的指针。 |


**返回：**


| 返回值 | 描述 |
| --- | --- |
| CUBLAS_STATUS_SUCCESS | 分配成功完成。 |
| CUBLAS_STATUS_NOT_INITIALIZED | cuBLASLt 库未初始化。这通常发生在以下情况：cublasLtCreate() 未首先调用、cuBLASLt 例程调用的 CUDA Runtime API 发生错误，或硬件设置错误。 |
| CUBLAS_STATUS_ALLOC_FAILED | cuBLASLt 库内部的资源分配失败。这通常由 cudaMalloc() 失败引起。纠正方法：在函数调用之前，尽可能多地释放先前分配的内存。 |
| CUBLAS_STATUS_INVALID_VALUE | lighthandle 为 NULL |


有关有效的返回码完整列表，请参阅 cublasStatus_t。

```c++

cublasStatus_t cublasLtMatmul(
      cublasLtHandle_t               lightHandle,
      cublasLtMatmulDesc_t           computeDesc,
      const void                    *alpha,
      const void                    *A,
      cublasLtMatrixLayout_t         Adesc,
      const void                    *B,
      cublasLtMatrixLayout_t         Bdesc,
      const void                    *beta,
      const void                    *C,
      cublasLtMatrixLayout_t         Cdesc,
      void                          *D,
      cublasLtMatrixLayout_t         Ddesc,
      const cublasLtMatmulAlgo_t    *algo,
      void                          *workspace,
      size_t                         workspaceSizeInBytes,
      cudaStream_t                   stream);


```


此函数根据以下操作计算矩阵 A 和 B 的矩阵乘法以产生输出矩阵 D：


`D = alpha*(A*B) + beta*(C),`


其中 A、B 和 C 是输入矩阵，alpha 和 beta 是输入标量。


> **注意**

注意
此函数支持原位矩阵乘法（C == D 且 Cdesc == Ddesc）和异位矩阵乘法（C != D，两种矩阵必须具有相同的数据类型、行数、列数、批量大小和内存顺序）。在异位情况下，C 的主维度可以与 D 的主维度不同。具体来说，C 的主维度可以为 0 以实现行或列广播。如果省略 Cdesc，则此函数假定它等于 Ddesc。


`workspace` 指针必须至少对齐到 256 字节的倍数。
关于 `workspaceSizeInBytes` 的建议与 cublasSetWorkspace() 部分中提到的相同。
