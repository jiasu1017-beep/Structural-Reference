# 4.1.25 UTEMP、UFIELD、UMASFL和UPRESS





### 4.1.25 [`UTEMP`](../sub/sub-link.md#sub-xsl-utemp)、[`UFIELD`](../sub/sub-link.md#sub-xsl-ufield)、[`UMASFL`](../sub/sub-link.md#sub-xsl-umasfl)和[`UPRESS`](../sub/sub-link.md#sub-xsl-upress)

**产品：**Abaqus/Standard  

### 测试功能

用于定义温度、场变量、质量流率和等效压力应力的用户子程序。

### I. 使用用户子程序设置温度和场数据

### 测试单元

T3D2

### 问题描述

这组测试验证当使用用户子程序设置值时，温度和场变量值被正确传递到结构。这些测试是["定义温度、场变量和压力应力值，" 第5.1.26节](ch05s01abv342.md)中描述的测试的修改。在那些测试中使用结果文件的地方，在此处用用户子程序替换。正在分析的结构是由10个T3D2单元组成的悬臂桁架。

测试如下：

**utmpfvs1.inp**

此文件测试使用用户子程序设置温度和多个场变量。温度和所有三个场变量的变化随时间线性如下：

|  | 初始值 | 最终值 |
| --- | --- | --- |
| 温度 | 100 | 200 |
| 场变量1 | 100 | 200 |
| 场变量2 | 200 | 250 |
| 场变量3 | 100 | 200 |

**utmpfvs2.inp**

此文件测试在问题中不存在温度的情况下使用用户子程序设置场变量。这是一个重要的测试，因为温度和场变量在内部存储的方式。场变量随时间线性变化，如下：

|  | 初始值 | 最终值 |
| --- | --- | --- |
| 场变量 | 100 | 200 |

（与["定义温度、场变量和压力应力值，" 第5.1.26节](ch05s01abv342.md)中测试[xtfvtrs3.inp](../eif/xtfvtrs3.inp)类似的测试被省略，因为此分析不会测试本节其他测试尚未覆盖的任何功能。）

**utmpfvs4.inp**

这是一个涉及温度和一个场变量的三步问题。在第一步中，使用幅值曲线将温度设置为200，将场变量设置为250。在第二步中，温度和场变量被设置两次：首先，从结果文件读取值，然后用户子程序将所有值乘以2。这导致在步骤中温度斜升到400，场变量斜升到500。使用的结果文件如下：

xtfvtrt1.fil ![](../graphics/ver_eqn01447.gif) 温度

xtfvtrt2.fil ![](../graphics/ver_eqn01447.gif) 场变量1

（这两个热传递问题在["定义温度、场变量和压力应力值，" 第5.1.26节](ch05s01abv342.md)中进一步描述。）在第三步中，温度和场变量都重置为其初始条件。

以下必须由此测试确认：
- 用户子程序必须与其他步骤中用于设置温度和场变量的方法平滑衔接。
- 用户子程序必须能够访问从结果文件设置的值，并能够修改这些值。
- 如果在同一步骤中通过数据行输入设置温度或场变量，然后由用户子程序修改，则必须忽略数据行上的值。
- 变量`KSTEP`必须在两个用户子程序中都可用。

**utmpfvsr.inp**

此分析从第三步重新启动[utmpfvs4.inp](../eif/utmpfvs4.inp)。温度和场变量都使用用户子程序设置如下：

|  | 初始值 | 最终值 |
| --- | --- | --- |
| 温度 | 0 | 100 |
| 场变量 | 0 | 50 |

**utmpfvsn.inp**

此文件测试在用户子程序[`UFIELD`](../sub/sub-link.md#sub-xsl-ufield)中同时设置所有场变量。最终结果与[utmpfvs1.inp](../eif/utmpfvs1.inp)中获得的结果相同。

### II. 复合壳温度载荷来自用户子程序

### 测试单元

S4R5、S8R5

### 问题描述

这组测试验证用户子程序[`UTEMP`](../sub/sub-link.md#sub-xsl-utemp)和[`UFIELD`](../sub/sub-link.md#sub-xsl-ufield)与复合结构壳的结合使用。这些测试是["定义温度、场变量和压力应力值，" 第5.1.26节](ch05s01abv342.md)中描述的测试的修改。在那些问题中从结果文件获得的值在此处用用户子程序设置。分析了具有通过横截面规定温度或场变量分布的三层复合壳。每层使用三个温度点和五个截面积分点。温度和场变量通过用户子程序每层可用的三个值的线性插值分配给这五个点。这些分析的结果验证了此插值的正确性。

用户子程序针对4节点壳和8节点壳进行测试。

### 结果与讨论

温度和场变量分布选择与热传递问题[xtmpcst4.inp](../eif/xtmpcst4.inp)和[xtmpcst8.inp](../eif/xtmpcst8.inp)中获得的结果相同，因此应力分析的结果可以直接与[xtmpcss4.inp](../eif/xtmpcss4.inp)、[xtmpcss8.inp](../eif/xtmpcss8.inp)、[xfvcss4x.inp](../eif/xfvcss4x.inp)和[xfvcss8x.inp](../eif/xfvcss8x.inp)的结果进行比较。（关于热传递问题的描述，请参见["定义温度、场变量和压力应力值，" 第5.1.26节](ch05s01abv342.md)。）温度/场变量分布如下：
- 第1层底部的温度/场变量为425.0。
- 第1层顶部和第2层底部的温度/场变量为373.2。
- 第2层顶部和第3层底部的温度/场变量为336.8。
- 第3层顶部的温度/场变量为287.5。

每层的顶部和底部之间存在线性变化。

可以看出，温度和场变量值被正确传递到结构复合壳。

### III. 间隙传导问题，其中场变量和质量流率使用用户子程序[`UFIELD`](../sub/sub-link.md#sub-xsl-ufield)和[`UMASFL`](../sub/sub-link.md#sub-xsl-umasfl)设置

### 测试单元

C3D8T、DC3D8、DCC3D8、DINTER4、INTER4T

### 问题描述

这些测试验证场变量和质量流率在热传递和耦合温度-位移分析期间被正确传递到结构。这些测试是["热特性，" 第2.3.1节](ch02s03abv172.md)和["`GAPCON`，" 第4.1.6节](ch04s01abv281.md)中描述的测试的修改。测试是使用三维单元的均匀一维热通量的情况。因此，位于热流方向特定平面上的所有节点的温度结果相同。在所有情况下，以多个增量执行稳态热传递分析。预定义场变量或质量流率的值在求解过程中变化，这反过来影响界面处的传导，从而影响解。

### 结果与讨论

结果与精确解相符。

umasflghs.inp中使用的用户子程序[`UMASFL`](../sub/sub-link.md#sub-xsl-umasfl)。

[ufieldgcs.inp](../eif/ufieldgcs.inp)
