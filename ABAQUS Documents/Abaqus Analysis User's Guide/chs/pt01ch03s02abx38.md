### 概述

您可以使用 **abaqus encrypt** 实用程序来防止未经授权使用 Abaqus 输入数据。该实用程序将数据文件转换为加密的、密码保护的格式，只有授权的 Abaqus 输入方才能访问。该实用程序旨在加密您通过引用包含在输入（`.inp`）文件或其他数据文件中的数据。例如，您可以加密一个包含模型所有专有材料数据的文件，然后通过引用将加密的数据文件包含在一个未加密的 Abaqus 输入文件中。请参阅『在 Abaqus 中定义模型』，第 1.3.1 节（第 1.3.1 节）中的『包含加密数据文件』，了解如何在 Abaqus 输入文件中包含加密数据文件。

您可以加密任何输入文件。但是，Abaqus 无法直接运行加密的 Abaqus 输入文件；加密文件必须包含在未加密的文件中。

### 指定其他访问级别和控制

您可以自定义加密，以便只有具有特定 Abaqus 功能许可证或来自特定站点的用户才能包含或解密文件。例如，您可以指定只有 Abaqus/Standard 用户可以访问该文件。您还可以防止任何用户解密加密文件，无论其许可证和站点如何；最终用户仍然可以通过在未加密的 Abaqus 输入文件中通过引用包含加密数据来在分析中使用加密数据，前提是用户知道加密文件的密码。

### 安全和支持注意事项

Abaqus 加密实现的主要目的是防止未经授权使用加密输入数据，而不是防止向授权用户披露加密数据。使用加密数据运行 Abaqus 分析输入可能会产生未加密的输出文件。加密输入文件中包含的材料和连接器行为信息在输出中不可见。只有加密输入文件中包含的材料和连接器行为信息在输出中不可见。这意味着满足访问标准的加密数据接收者（如密码、许可证功能或 SiteID）将能够以未加密形式重建某些输入。加密数据提供商应考虑签订合同协议来保护专有数据。加密数据的使用者必须对加密输入产生的文件的安全性负责，并应考虑限制对结果分析文件的访问。

Abaqus 技术支持无法检索加密数据文件的丢失密码。接收加密数据的使用者如有任何技术支持问题，应联系数据提供商。

### 向加密文件的开头添加注释

加密文件时，Abaqus 会在文件开头添加以下未加密的注释行：

```
** encrypted input
```
请勿修改或删除此头注释。但是，您可以在此头注释和加密数据的第一行之间插入其他注释行。这些加密后注释行可以描述加密文件的内容、提供发布编号，或显示有关加密数据的版权和法律信息。有关注释行语法的更多信息，请参阅『输入语法规则』，第 1.2.1 节（第 1.2.1 节）。

但是，您不应在加密数据的行内添加加密后注释行。如果要编辑或修改数据内部的注释行，必须首先解密数据。

### 命令行选项

**input**

此选项指定您要加密或解密的数据文件的名称。

如果从命令行中省略此选项，Abaqus 将提示您输入其值。

**output**

此选项指定加密或解密后的数据文件的名称。

如果从命令行中省略此选项，Abaqus 将提示您输入其值。

**password**

此选项指定此加密或解密的密码。密码区分大小写。

如果在加密数据时从命令行中省略此选项，Abaqus 将提示您输入其值。如果在解密数据时密码输入不正确或从命令行中省略，Abaqus 会报告输入文件已损坏或密码不正确。

**license**

此选项仅适用于文件加密。

此选项指定最终用户必须获得许可的 Abaqus 功能或功能列表，以便他们包含或解密此加密数据文件。您可以使用逗号分隔的列表来允许访问一系列 Abaqus 功能的任何一个被许可人。

Abaqus 许可证文件中出现的任何功能名称都有效。这些可能包括以下功能：`foundation`、`standard`、`explicit`、`design`、`aqua`、`ams`、`cae`、`viewer`、`cae_nogui`、`cmold`、`moldflow`、`safe`、`cadporter_catia`、`cadporter_catiav5`、`cadporter_ideas`、`cadporter_parasolid`、`cadporter_proe`、`afcv5_structural` 和 `afcv5_thermal`。

**siteid**

此选项仅适用于文件加密。

此选项指定可以包含或解密此加密数据文件的 Abaqus Site ID 或站点 ID。您可以使用逗号分隔的列表来允许多个站点访问该文件。此选项仅在同时使用 **license** 选项时才能使用。

要确定您的 Abaqus Site ID，请从命令提示符处运行 `abaqus whereami`。

**include_only**

此选项仅适用于文件加密。

此选项指定无法使用 **abaqus decrypt** 执行程序解密加密输入数据；此类数据只能包含在 Abaqus 输入文件中。

如果您尝试解密使用 **include_only** 选项加密的文件，Abaqus 会发出一条错误消息，指出输入文件可以包含在分析中，但没有资格解密。

**expiration**

此选项仅适用于文件加密。

此选项指定最终用户不再可以解密或包含加密数据文件的日期。日期必须以 YYYY-MM-DD 形式提供。

### 示例

以下示例说明了使用 **encrypt** 执行程序可能的不同加密方法。

#### 创建加密文件

最简单的加密场景是，Abaqus 用户在将加密版本发送给授权最终用户之前，创建名为 `material_data.inp` 的文件的加密副本，该文件包含模型的所有材料数据。加密可防止未经授权的用户在传输过程中访问加密文件。要创建名为 `material_data_enc.inp` 的 `material_data.inp` 的加密副本，请发出以下命令：

```
abaqus encrypt input=material_data.inp
   output=material_data_enc.inp password=e1No9c2z
```
收到文件后，最终用户可以运行 **abaqus decrypt** 执行程序来创建原始未加密材料数据文件的副本。由于此示例中选择的加密选项，最终用户只需要加密文件的密码即可解密。要解密加密数据文件 `material_data_enc.inp`，生成未加密文件 `material_data.inp`，请发出以下命令：
```
abaqus decrypt input=material_data_enc.inp
   output=material_data.inp password=e1No9c2z
```
或者，最终用户可以跳过解密步骤，运行一个通过引用包含加密数据的分析。要通过引用将加密文件包含在 Abaqus 输入文件中，请将以下语句添加到输入文件：
```
*INCLUDE, INPUT=material_data_enc.inp, PASSWORD=e1No9c2z
```

#### 按许可证功能或站点 ID 限制对解密文件的访问

您可以指定最终用户只有在具有特定 Abaqus 功能的有效许可证、在特定站点运行 Abaqus 或同时满足这两个条件时才能访问文件。要加密只能由拥有 Abaqus/Explicit 许可证且在 `09YYY` 站点运行软件的用户访问的数据文件，请发出以下命令：

```
abaqus encrypt input=material_data.inp
   output=material_data_enc.inp password=e1No9c2z
   license=explicit siteid=09YYY
```
最终用户可以尝试使用与前一示例中指定的相同解密或包含语法来访问文件 `material_data_enc.inp`。对于此加密文件，Abaqus 会在向用户提供文件访问权限之前验证最终用户拥有 Abaqus/Explicit 许可证并且正在 `09YYY` 站点运行 Abaqus。如果最终用户的许可证或站点设置与加密期间指定的设置不匹配，Abaqus 会发出一条错误消息，列出访问该文件所需的许可证或站点。

#### 创建必须包含才能被 Abaqus 使用的加密文件

您可以使用 **include_only** 选项来防止最终用户直接使用 **abaqus decrypt` 解密文件。授权用户可以通过在 Abaqus 输入文件中通过引用包含使用 **include_only** 选项加密的文件来访问文件。加密输入文件中的材料和连接器行为定义不会写入输出数据库。此外，如果加密文件用作模型任何部分的输入，则输出到数据文件的所有材料和连接器行为定义都会被抑制。要创建只能通过引用包含在其他输入文件中的加密文件，请发出以下命令：

```
abaqus encrypt input=material_data.inp
   output=material_data_enc.inp password=e1No9c2z include_only
```
生成的加密文件可以使用与前一示例相同的语法通过引用包含在 Abaqus 输入文件中。如果您尝试解密使用 **include_only` 选项加密的文件，Abaqus 会返回一条错误消息。
