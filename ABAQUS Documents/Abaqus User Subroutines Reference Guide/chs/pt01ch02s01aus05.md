### 概述

本节描述在Abaqus输入文件中定义节点的方法。在Abaqus/CAE等预处理程序中，您定义模型几何而不是节点和单元；当您对几何进行网格划分时，预处理程序会自动创建分析所需的节点和单元。虽然本节中讨论的概念通常适用于Abaqus/CAE创建的输入文件中的节点定义，但此处描述的方法和技术仅在您手动创建输入文件时适用。

节点定义包括：
- 为节点分配节点编号；
- （可选）指定用于定义节点的局部坐标系；
- 通过指定坐标来定义各个节点；
- 将节点分组到节点集；
- 通过增量生成、复制现有节点或在区域的边界之间填充节点来创建节点；和
- 将一组节点从一个坐标系映射到另一个坐标系。

如果任何一个节点被多次指定，则使用最后给出的指定。

Abaqus将在继续分析之前消除所有不必要的节点。此功能很有用，因为它允许将点定义为仅用于网格生成目的的节点。

### 为节点分配节点编号

每个单独的节点必须有一个称为节点编号的数值标签，在定义节点时分配。节点编号必须是正整数，允许的最大节点编号为999999999（有关整数输入的信息，请参见["输入语法规则，" 第1.2.1节"](pt01ch01s02aus01.md)）。节点不需要连续编号。

Abaqus模型可以根据部件实例的装配来定义（请参见["定义装配，" 第2.10.1节"](pt01ch02s10aus28.md)）。在这种模型中，所有节点必须属于部件、部件实例，或者在参考节点的情况下属于装配。节点编号在部件、部件实例或装配中必须唯一；但它们可以在不同的部件或部件实例中重复。

### 指定用于定义节点的局部坐标系

有时在局部坐标系中定义节点坐标然后将其转换到全局坐标系会很方便。您可以定义节点坐标系；Abaqus会将局部（![](../graphics/usb_eqn00108.gif)）坐标值转换并旋转到全局坐标系。转换在输入后立即进行，并将应用于节点坐标系定义后输入或生成的所有节点坐标。

转换仅影响节点定义中的节点坐标输入。节点坐标系定义不能用于
- 施加载荷和边界条件——请参见["变换坐标系，" 第2.1.5节"](pt01ch02s01aus09.md)，相反；或
- 应力、应变和单元截面力的分量输出——请参见["取向，" 第2.2.5节"](pt01ch02s02aus15.md)，相反。

除了定义节点坐标系外，您还可以用局部矩形、圆柱形或球形系统定义各个节点或节点集（请参见["为节点坐标指定局部坐标系](pt01ch02s01aus05.md#usb-int-inode-define-csys)"）。如果节点坐标系生效，并且您为特定节点或节点集定义指定了局部坐标系，则输入坐标首先根据节点定义中指定的局部系统进行转换，然后根据节点坐标系进行转换。

您通过指定局部系统中三个点的全局坐标来设置坐标系规范：局部系统的原点（[图2.1.1-1](pt01ch02s01aus05.md#ksystem)中的点*a*）、局部![](../graphics/usb_eqn00109.gif)轴上的点（[图2.1.1-1](pt01ch02s01aus05.md#ksystem)中的点*b*），以及局部系统![](../graphics/usb_eqn00110.gif)平面上（或附近）局部![](../graphics/usb_eqn00111.gif)轴上的点（[图2.1.1-1](pt01ch02s01aus05.md#ksystem)中的点*c*）。

**图2.1.1-1** 节点坐标系。

![](../graphics/ksystem-nls.png)

#### 在部件定义内定义节点坐标系

当您在部件（或部件实例）定义内定义节点坐标系时，它仅在该部件（或部件实例）定义内生效。在其他部件中定义的节点不受影响。

您相对于部件坐标系指定局部（![](../graphics/usb_eqn00108.gif)）坐标值，随后可根据为实例给出的定位数据进行平移和/或旋转（请参见["定义装配，" 第2.10.1节"](pt01ch02s10aus28.md)）。

### 通过指定坐标来定义各个节点

您可以通过指定定义节点的节点编号和坐标来定义各个节点。Abaqus对所有节点使用右手直角笛卡尔坐标系，除了轴对称模型，此时必须将节点坐标给定为径向和轴向位置。有关方向定义的更多信息，请参见["约定，" 第1.2.2节"](pt01ch01s02aus02.md)。

在根据部件实例的装配定义的模型中，请在部件（或部件实例）的局部坐标系中给出节点坐标。请参见["定义装配，" 第2.10.1节"](pt01ch02s10aus28.md)。

| **输入文件用法：** | ``` [*NODE](../key/key-link.md#usb-kws-mnode) ``` |

#### 从文件中读取节点定义

节点定义可以从备用文件读入Abaqus。此类文件名的语法在["输入语法规则，" 第1.2.1节"](pt01ch01s02aus01.md)中描述。

| **输入文件用法：** | ``` [*NODE](../key/key-link.md#usb-kws-mnode), INPUT=*file_name* ``` |

#### 为节点坐标指定局部坐标系

您可以指定使用局部矩形笛卡尔、圆柱形或球形坐标系来定义特定节点坐标。这些坐标系如[图2.1.1-2](pt01ch02s01aus05.md#kngen)所示。

**图2.1.1-2** 坐标系。

![](../graphics/kngen-nls.png)

此坐标系规范完全局部于节点定义。当读取节点数据时，坐标会立即转换为矩形笛卡尔坐标。如果节点坐标系也生效（请参见["指定用于定义节点的局部坐标系](pt01ch02s01aus05.md#usb-int-inode-system-option)"），则这些是节点坐标系定义的局部矩形笛卡尔坐标，随后将转换为全局笛卡尔坐标。

### 将节点分组到节点集

节点集用作定义载荷、约束、属性等时的便捷交叉引用。节点集是模型的基本引用，应使用它们来辅助输入定义。节点集的成员可以是各个节点或其他节点集。一个单独的节点可以属于多个节点集。

节点可以在创建时或定义后将节点分组到节点集。在任一情况下，每个节点集都被分配一个名称。节点集名称最多可以包含80个字符。

相同的名称可以用于节点集和单元集。

默认情况下，节点集中的节点将按升序排列，并删除重复节点。这样的集合称为排序节点集。您可以选择创建非排序节点集，如后所述，这对于匹配两个或多个节点集的特征很有用。例如，如果在两个节点集之间定义多点约束（["广义多点约束，" 第35.2.2节"](pt08ch35s02aus130.md)），将在集合1中的第一个节点和集合2中的第一个节点之间创建约束，然后在集合1中的第二个节点和集合2中的第二个节点之间创建约束，等等。确保以所需方式组合节点非常重要。因此，有时最好指定节点集以未排序顺序存储。

一旦节点被分配到节点集，可以将其他节点添加到同一节点集；但是，不能从节点集中删除节点。

#### 创建非排序节点集

您可以选择按给出节点的顺序将节点分配到新节点集（或添加到现有节点集）。节点编号不会重新排列，也不会删除重复项。

此非排序节点集将影响节点复制、节点填充、线性约束方程、多点约束以及与保留自由度关联的子结构节点。非排序节点集只能通过直接定义（如本文所述）或通过复制非排序节点集来创建。使用其他方式对节点集的任何添加或修改都将导致排序节点集。

| **输入文件用法：** | ``` [*NSET](../key/key-link.md#usb-kws-mnset), NSET=*name*, UNSORTED ``` |

#### 在创建节点时将节点分配到节点集

有几种方法可以在创建节点时将节点分配到节点集。

| **输入文件用法：** | 使用以下任一选项： |
| --- | --- |
|  | ``` [*NODE](../key/key-link.md#usb-kws-mnode), NSET=*name* [*NCOPY](../key/key-link.md#usb-kws-mncopy), NEW SET=*name* [*NFILL](../key/key-link.md#usb-kws-mnfill), NSET=*name* [*NGEN](../key/key-link.md#usb-kws-mngen), NSET=*name* [*NMAP](../key/key-link.md#usb-kws-mnmap), NSET=*name* ``` |

#### 将先前定义的节点分配到节点集

您可以通过直接列出形成集合的节点、通过生成节点集或通过从单元集生成节点集，将先前定义的节点（通过指定坐标、通过在两个边界之间填充节点或通过增量生成）分配到节点集。

##### 直接列出定义集合的节点

您可以直接列出形成节点集的节点。先前定义的节点集以及各个节点都可以分配到节点集。

| **输入文件用法：** | ``` [*NSET](../key/key-link.md#usb-kws-mnset), NSET=*name* ``` |
| --- | --- |
|  | 例如，以下行将节点1、3、10、11和集合A11中的所有节点添加到集合A12： ``` [*NSET](../key/key-link.md#usb-kws-mnset), NSET=A12 1, 3 10, 11, A11 ``` 只有在A11的定义出现在A12之前时，才能将节点集A11分配到节点集A12。节点集A12中的所有节点将按升序排序。如果在[*NSET](../key/key-link.md#usb-kws-mnset)选项上包含UNSORTED参数，则节点集A12将按数据行上指定的顺序包含节点。 |

##### 生成节点集

要生成节点集，您必须指定第一个节点 ![](../graphics/usb_eqn00124.gif)、最后一个节点 ![](../graphics/usb_eqn00125.gif) 以及这些节点之间节点编号的增量*i*。从 ![](../graphics/usb_eqn00124.gif) 到 ![](../graphics/usb_eqn00125.gif) 以*i*为增量的所有节点将被添加到集合。因此，*i*必须是使 ![](../graphics/usb_eqn00126.gif) 为整数的整数（不是分数）。默认值是 ![](../graphics/usb_eqn00127.gif)。

| **输入文件用法：** | ``` [*NSET](../key/key-link.md#usb-kws-mnset), NSET=*name*, GENERATE ``` |
| --- | --- |
|  | 例如，以下行将节点100到120（增量为10）的所有节点添加到集合A13： ``` [*NSET](../key/key-link.md#usb-kws-mnset), NSET=A13, GENERATE 100, 120, 10 ``` |

##### 从单元集生成节点集

您可以指定先前定义的单元集的名称（["单元定义，" 第2.2.1节"](pt01ch02s02aus11.md)），在这种情况下，定义此单元集中包含的单元的节点将被分配到指定的节点集。此方法只能用于定义排序节点集。

| **输入文件用法：** | ``` [*NSET](../key/key-link.md#usb-kws-mnset), NSET=*name*, ELSET=*name* ``` |
| --- | --- |
|  | 例如，以下行将定义单元50和100的节点（节点1、2、3和4）添加到节点集A14： ``` [*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=B21 50, 1, 2 100, 3, 4 [*ELSET](../key/key-link.md#usb-kws-melset), ELSET=B1 50, 100 [*NSET](../key/key-link.md#usb-kws-mnset), NSET=A14, ELSET=B1 ``` 单元集B1可以被分配到节点集A14，因为B1的定义出现在A14之前。 |

##### 更新用于定义其他节点集的节点集的限制

如果节点集是从先前定义的节点集构建的，则不会考虑这些集合的后续更新。

| **输入文件用法：** | ``` [*NSET](../key/key-link.md#usb-kws-mnset), NSET=*name* ``` |
| --- | --- |
|  | 例如，以下行添加节点1和2，但不添加节点3，到集合SET-AB，同时将节点1和3添加到集合SET-A： ``` [*NSET](../key/key-link.md#usb-kws-mnset), NSET=SET-A 1, [*NSET](../key/key-link.md#usb-kws-mnset), NSET=SET-B 2, [*NSET](../key/key-link.md#usb-kws-mnset), NSET=SET-AB SET-A, SET-B [*NSET](../key/key-link.md#usb-kws-mnset), NSET=SET-A 3, ``` |

#### 定义部件和装配集

在根据部件实例的装配定义的模型中，所有节点集必须在部件、部件实例或装配定义中定义。如果在部件（或部件实例）定义中定义节点集，则可以直接引用节点编号。要定义装配级节点集，必须通过在每个节点编号前加部件实例名称和一个"."来标识要添加到集合的节点（如["定义装配，" 第2.10.1节"](pt01ch02s10aus28.md)中所解释）。装配级节点集可以与部件级节点集具有相同的名称。

##### 示例

以下输入定义了一个节点集`set1`，它属于部件`PartA`，并将由`PartA`的每个实例继承：

```
*PART, NAME=PartA
   ...
   *NSET, NSET=set1
    1,3,26,500
*END PART
```

具有相同名称的节点集在装配级别定义如下：
```
*ASSEMBLY, NAME=Assembly-1
   *INSTANCE, NAME=PartA-1, PART=PartA
    ...
   *END INSTANCE
   *INSTANCE, NAME=PartA-2, PART=PartA
    ...
   *END INSTANCE
   *NSET, NSET=set1
    PartA-1.1, PartA-1.3, PartA-1.26, PartA-1.500
    PartA-2.1, PartA-2.3, PartA-2.26, PartA-2.500
*END ASSEMBLY
```

装配级节点集`set1`包含属于部件实例`PartA-1`和`PartA-2`的节点集`set1`中的所有节点。因此，节点被分配到两个独立的节点集：一个在部件实例级别，一个在装配级别。可以创建与属于部件集的节点完全不同的装配级节点集`set1`；部件级和装配级节点集是独立的。但是，由于在本例中相同的节点被分配到部件级和装配级节点集`set1`，因此装配级集合也可以定义为
```
*ASSEMBLY, NAME=Assembly-1
   *INSTANCE, NAME=PartA-1, PART=PartA
    ...
   *END INSTANCE
   *INSTANCE, NAME=PartA-2, PART=PartA
    ...
   *END INSTANCE
   *NSET, NSET=set1
    PartA-1.set1, PartA-2.set1
*END ASSEMBLY
```

此节点集定义等效于前一个示例，其中节点被单独列出。

##### 定义装配级节点集的替代方法

有时通过引用部件级节点集来定义装配级节点集并不方便。在这种情况下，包含许多节点的集合定义可能会变得相当冗长。因此，提供了一种替代方法。

| **输入文件用法：** | ``` [*NSET](../key/key-link.md#usb-kws-mnset), NSET=*NsetName*, INSTANCE=*InstanceName* ``` |
| --- | --- |
|  | 以下示例展示了两种等效方式定义装配级节点集：一种是通过在每个节点编号前加部件实例名称（如上所示），另一种是使用更紧凑的INSTANCE表示法： ``` *ASSEMBLY, NAME=Assembly-1 *INSTANCE, NAME=PartA-1, PART=PartA ... *END INSTANCE *INSTANCE, NAME=PartA-2, PART=PartA ... *END INSTANCE *NSET, NSET=set2 PartA-1.11, PartA-1.12, PartA-1.13, PartA-1.14, PartA-2.21, PartA-2.22, PartA-2.23, PartA-2.24 *NSET, NSET=set3, INSTANCE=PartA-1 11, 12, 13, 14 *NSET, NSET=set3, INSTANCE=PartA-2 21, 22, 23, 24 *END ASSEMBLY ``` 当多次使用相同名称的[*NSET](../key/key-link.md#usb-kws-mnset)选项时（如`set3`的情况），第二次使用[*NSET](../key/key-link.md#usb-kws-mnset)的节点将追加到第一次使用[*NSET](../key/key-link.md#usb-kws-mnset)创建的集合。 |

#### Abaqus/CAE创建的内部节点集

在Abaqus/CAE中，许多建模操作是通过用鼠标拾取几何来执行的。例如，可以通过拾取几何部件实例上的点来施加集中载荷。由于[*CLOAD](../key/key-link.md#usb-kws-hcload)选项引用节点集，因此必须将此"拾取的"几何转换到输入文件中的节点集。Abaqus/CAE为这些集合分配名称并将其标记为内部。您可以使用Abaqus/CAE可视化模块中的显示组来查看这些内部集（请参见[Abaqus/CAE用户指南第78章，"使用显示组显示模型子集](../usi/usi-link.md#uss-dgp)"）。

| **输入文件用法：** | ``` [*NSET](../key/key-link.md#usb-kws-mnset), NSET=*NsetName*, INTERNAL ``` |

### 传输节点集

### 通过在两个边界之间填充节点来创建节点

您可以通过在两个边界之间填充节点来创建节点。在这种情况下，您指定其成员形成边界的两个节点集、沿边界节点之间每条线的间隔数以及从第一个边界集端点的节点编号开始的节点编号增量。

设*l*等于要在两个边界节点集之间创建的节点线数；沿每条边界节点之间线的间隔数随后由 ![](../graphics/usb_eqn00128.gif)给出。

设*n*等于从第一个边界集端点的节点编号开始的节点编号增量；对于第一个边界节点集中的每个节点（![](../graphics/usb_eqn00129.gif)），另一个边界节点集中的相应节点（![](../graphics/usb_eqn00130.gif)）的编号必须使得 ![](../graphics/usb_eqn00131.gif) 为整数。

定义区域边界的节点集按照节点填充定义出现在输入文件中的方式使用：仅使用在节点填充定义之前添加到集合中的那些节点。排序和非排序节点集都可以使用。尚未给出坐标的节点被假定位于原点（0.,0.,0.)。

通过此方法创建的节点位于两个集合中相应节点之间的直线上。如果集合具有不同数量的节点，则较长集合中的额外节点将被忽略。默认情况下，沿线的节点间距是均匀的。

映射功能不能用于根据部件实例的装配定义的模型。

提供以下不同的映射：简单缩放；简单平移和/或旋转；斜笛卡尔；圆柱形；球形；环形；以及仅在Abaqus/Standard中的混合二次。前五种映射如[图2.1.1-13](pt01ch02s01aus05.md#knmap-coordsys)所示。

**图2.1.1-13** 坐标系；角度以度为单位。

![](../graphics/knmap-coordsys-nls.png)

混合二次映射如[图2.1.1-14](pt01ch02s01aus05.md#inode-nmap-blended)所示。

**图2.1.1-14** 使用混合二次映射将实体网格展开到曲线块。

#### 映射前缩放局部坐标

对于除混合二次映射之外的所有映射，您可以指定要应用于映射前局部坐标的缩放因子。

此功能对于"拉伸"某些给定的坐标很有用。例如，在局部系统使用某些角度坐标和某些距离坐标的情况下（圆柱形、球形等），可能更倾向于在角度方向使用距离度量生成网格，然后缩放到映射的角度坐标系。

提供了两种不同的缩放方法。

##### 直接指定缩放因子

缩放节点的第一种方法是相对于局部系统原点直接指定缩放因子。在这种情况下，缩放在与从一个坐标系到另一个坐标系的映射同时进行。

| **输入文件用法：** | ``` [*NMAP](../key/key-link.md#usb-kws-mnmap), NSET=*name* *first data line* *second data line* *scale factor for first local coord, scale factor for second local coord, scale factor for third local coord* ``` |

##### 相对于参考点指定缩放

或者，您可以相对于原点以外的点进行缩放。用于缩放的参考点可以通过使用其坐标或用户节点编号来定义。

#### 通过混合二次进行映射

要在Abaqus/Standard中通过混合二次进行映射，您可以定义多达20个"控制节点"的新的（映射的）坐标：这些是被映射节点块的角点和边中节点。在这种情况下，映射类似于20节点砖块等参单元的映射。可以省略任何边中节点，从而允许沿该块边缘的线性插值。Abaqus/Standard不检查集合中的节点是否位于由角点和边中节点定义的块的物理空间内：这些控制节点简单地定义映射函数，然后应用于集合中的所有节点。

控制节点应定义一个"良好"形状的块；例如，边中节点应接近边缘的中点。否则，映射可能会非常扭曲。例如，带有四分之一节点处边中节点的裂纹尖端20节点单元的节点将无法正确映射，因此不应作为控制节点。

混合映射仅适用于三维分析。

| **输入文件用法：** | ``` [*NMAP](../key/key-link.md#usb-kws-mnmap), NSET=*name*, TYPE=BLENDED ``` |
