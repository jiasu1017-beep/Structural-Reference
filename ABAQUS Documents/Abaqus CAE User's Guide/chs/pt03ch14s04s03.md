# 14.4.3 输出请求的传播





当你在分析中创建第一个步骤时，Abaqus/CAE会根据你为该步骤选择的分析过程生成默认场和历史输出请求。这些默认输出请求会被传播到后续步骤。**场输出请求管理器**和**历史输出请求管理器**是步骤相关管理器，它们显示输出请求在步骤之间的传播和状态。

常规步骤中请求的输出与线性扰动步骤中请求的输出无关。此外，输出请求的传播行为在常规步骤和线性扰动步骤之间有所不同。

**常规步骤**

Abaqus/CAE为模型中的第一个常规步骤创建默认场输出请求，该默认输出请求会被传播到所有后续常规步骤。类似地，如果你创建新的输出请求或修改默认输出请求，新请求或修改后的请求会被传播到后续常规步骤。

如果你在步骤序列中插入新的常规步骤，则先前常规步骤的输出请求会被传播到新步骤。

**线性扰动步骤**

Abaqus/CAE为模型中的第一个线性扰动步骤创建默认场输出请求，该默认输出请求会被传播到所有使用相同分析过程的后续线性扰动步骤；例如，所有频率分析。类似地，如果你创建新的输出请求或修改默认输出请求，新请求或修改后的请求会被传播到所有使用相同分析过程的后续步骤。

如果你在步骤序列中插入新的线性扰动步骤，则使用相同分析过程的先前线性扰动步骤的输出请求会被传播到新步骤。如果你创建使用不同分析过程的线性扰动步骤，Abaqus/CAE会创建新的默认输出请求。新的默认输出请求会被传播到所有使用相同分析过程的后续线性扰动步骤。

你应该注意以下行为：
- 如果你在现有常规步骤序列的开头插入新的常规步骤，Abaqus/CAE不会为新步骤创建默认输出请求。类似地，如果你在相同过程类型的现有线性扰动步骤序列的开头插入新的线性扰动步骤，Abaqus/CAE不会为新步骤创建默认输出请求。在这两种情况下，你都必须为新步骤创建新的输出请求。或者，你可以使用输出请求管理器将后续步骤的输出请求移动到新步骤。
- 如果你删除包含新输出请求的步骤（常规或线性扰动），Abaqus/CAE会从请求已传播到的所有后续步骤中删除该输出请求。
- 如果某个步骤不包含输出请求，Abaqus/CAE在生成输入文件时会在作业模块中显示警告。
