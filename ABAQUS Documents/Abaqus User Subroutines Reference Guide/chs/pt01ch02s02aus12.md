### 概述

本节描述在Abaqus输入文件中定义单元的方法。在Abaqus/CAE等前处理器中，您定义模型几何而不是节点和单元；当您对几何进行网格划分时，前处理器会自动创建分析所需的节点和单元。尽管本节中讨论的概念通常适用于Abaqus/CAE创建的输入文件中的单元定义，但此处描述的方法和技术仅在您手动创建输入文件时适用。

单元定义包括：
- 为单元分配单元编号；
- 通过指定单元的节点来定义单个单元；
- 将单元分组为单元集；以及
- 通过增量生成或复制现有单元从现有单元创建单元。

如果任何单元被指定多次，则使用最后给出的规格。

### 为单元分配单元编号

每个单独单元必须有一个称为单元编号的数字标签，该标签在定义单元时分配。单元编号必须为正整数，允许的最大单元编号为999999999（有关整数输入的信息，请参阅["输入语法规则，" 第1.2.1节](pt01ch01s02aus01.md)）。单元编号不必是连续的。

Abaqus模型可以以部件实例装配的形式定义（参见["定义装配，" 第2.10.1节](pt01ch02s10aus28.md)）。在这种模型中，几乎所有单元都必须属于某个部件或部件实例。唯一的例外是质量、转动惯量、电容、连接器、弹簧和阻尼单元，它们可以属于部件或装配。单元编号在部件、部件实例或装配中必须是唯一的；但可以在不同的部件或部件实例中重复。

### 通过指定节点来定义单个单元

您可以通过指定单元编号和定义单元的节点来定义单个单元。此外，您必须指定单元类型。单元必须从[第六部分，"单元"](pt06.md)中指定的单元类型中选择；或者，在Abaqus/Standard中，它可以是用户定义的单元（["用户定义的单元，" 第32.15.1节](pt06ch32s15alm60.md)）或子结构（["使用子结构，" 第10.1.1节](pt04ch10s01aus58.md)）。

| **输入文件用法：** | ``` [*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=*name* ``` |
| --- | --- |
|  | 例如，以下行通过定义其节点（2, 3, 9, 7, 5, 8, 12, 16）创建类型为C3D8R的单元编号11： ``` [*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=C3D8R 11, 2, 3, 9, 7, 5, 8, 12, 16 ``` |

#### 将大节点编号用于使用多个节点的单元

定义单元时适用以下规则：
- 每个单元的连通性被视为一条逻辑记录，可以使用任意数量的输入行来指定它。Abaqus将读取单元的第一行，如果行以逗号结尾且单元定义未完成，则将下一行视为连续行。
- 可以使用任意数量的连续行。
- 对于具有可变节点数的单元（如C3D27）（请参阅["实体（连续体）单元，" 第28.1.1节](pt06ch28s01alm01.md)），最后一行不应以逗号结尾，否则Abaqus会将下一个单元定义解释为当前单元的延续。

 例如，
```
[*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=C3D20
100001, 100001, 100002, 100003, 100004, 100005, 100006, 100007,
100008, 100009, 100010, 100011, 100012, 100013, 100014, 100015,
100016, 100017, 100018, 100019, 100020
```

#### 定义垫片单元

有几种方法可以定义垫片单元。（请参阅["垫片单元：概述，" 第32.6.1节](pt06ch32s06abo30.md)；["在模型中包含垫片单元，" 第32.6.3节](pt06ch32s06alm48.md)；以及["定义垫片单元的初始几何，" 第32.6.4节](pt06ch32s06alm49.md)，了解有关垫片单元的更多信息；它们仅在Abaqus/Standard中可用。）

在第一种方法中，您通过指定单元编号和定义单元的节点来定义单个单元。

在第二种方法中，您仅指定垫片单元底面上的节点，以及一个正偏移编号，用于定义顶面上相应的节点。对于18节点垫片单元，您给出前八个节点，然后是中间表面节点；即完整单元节点连通性中的节点17。

如果两个单元面都是接触面的一部分，Abaqus/Standard可以自动生成18节点垫片单元的面中节点。要调用此功能，您可以在上述任何输入方法中输入空白而不是实际节点编号。然后Abaqus/Standard将自动生成面中节点的节点编号和坐标。

| **输入文件用法：** | 使用以下选项指定单元编号和定义单元的节点： |
| --- | --- |
|  | ``` [*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=*name* ``` 使用以下选项指定单元底面上的节点和顶面的正偏移编号： ``` [*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=*name*, OFFSET=*offset number* ``` |

##### 使用实体单元连通性定义垫片单元

垫片单元的节点编号方案与连续体单元的节点编号方案不对应，如果使用的网格生成器不直接支持垫片单元，或者在热应力分析中使用连续体单元对垫片进行热传导建模，这将很不方便。对于这种情况，您可以指定使用实体单元连通性来定义垫片单元。默认情况下，假设实体的第一个（S1）面与垫片单元的第一个（SNEG）面对应。如果等效实体单元的方向不同，请指定实体单元上对应于垫片单元第一个面的面号。实体单元必须在每个面上具有与相应垫片单元相同数量的节点；面之间的任何节点都将被忽略。18节点垫片单元是一个例外。如果两个单元面都是接触面的一部分，则可以使用20节点砖单元的连通性，Abaqus/Standard将自动生成面中节点的节点编号和坐标。

Abaqus/Standard在读取数据后立即将实体单元连通性转换为正常垫片单元连通性。因此，对数据（.dat）、结果（.fil）和输出数据库（.odb）文件的所有输出都将使用正常的垫片单元连通性。

| **输入文件用法：** | 使用以下选项为垫片单元指定实体单元编号，其中实体单元的第一个面对应于垫片单元的第一个面： |
| --- | --- |
|  | ``` [*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=*name*, SOLID ELEMENT NUMBERING ``` 使用以下选项为垫片单元指定实体单元连通性，并指定与垫片单元第一个面对应的实体单元的面： ``` [*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=*name*, SOLID ELEMENT NUMBERING=*face number* ``` |

##### 示例

以下行创建GK3D12M单元编号11，具有节点编号1, 2, 3, 4, 5, 6, 1001, 1002, 1003, 1004, 1005和1006：

```
[*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=GK3D12M
11, 1, 2, 3, 4, 5, 6, 1001, 1002, 1003, 1004, 1005, 1006
```

以下行也可以创建相同的单元连通性：
```
[*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=GK3D12M, OFFSET=1000
11, 1, 2, 3, 4, 5, 6
```

等效实体单元为C3D15，输入如下：
```
[*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=GK3D12M, SOLID ELEMENT NUMBERING
11, 1, 2, 3, 1001, 1002, 1003, 4, 5, 6, 1004, 1005, 1006,  
501, 502, 503
```

其中节点501、502和503将不被使用。

#### 定义内聚单元

有三种方法可以定义内聚单元。（请参阅["内聚单元：概述，" 第32.5.1节](pt06ch32s05abo29.md)；["使用内聚单元建模，" 第32.5.3节](pt06ch32s05alm42.md)；以及["定义内聚单元的初始几何，" 第32.5.4节](pt06ch32s05alm43.md)，了解有关内聚单元的更多信息。）
- 在第一种方法中，您指定定义单元的单元编号和所有节点。
- 在第二种方法中，您仅指定内聚单元底面上的节点，Abaqus将根据您指定的偏移编号创建其余节点并为其编号。
- 在第三种方法中（仅适用于孔隙压力内聚单元），您指定底面和顶面上的节点。Abaqus将根据您指定的偏移编号创建其余的面中节点。

##### 通过指定所有节点来定义内聚单元

使用此方法，您可以指定定义内聚单元的所有节点。请参阅["二维内聚单元库，" 第32.5.8节](pt06ch32s05ael30.md)；["三维内聚单元库，" 第32.5.9节](pt06ch32s05ael31.md)；以及["轴对称内聚单元库，" 第32.5.10节](pt06ch32s05ael32.md)，了解单元节点编号定义。

| **输入文件用法：** | 使用以下选项指定单元编号和定义单元的节点： |
| --- | --- |
|  | ``` [*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=*name* ``` 例如，以下行创建具有节点编号1, 2, 3, 4, 1001, 1002, 1003和1004的COH3D8单元编号11： ``` [*ELEMENT](../key/link/key-link.md#usb-kws-melement), TYPE=COH3D8 11, 1, 2, 3, 4, 1001, 1002, 1003, 1004 ``` |

##### 通过仅指定底面节点来定义内聚单元

使用此方法，您仅指定内聚单元底面上的节点和一个正偏移编号。对于位移内聚单元，偏移编号被添加到底面节点编号以创建顶面上的相应节点。对于孔隙压力内聚单元，偏移编号首先被添加到顶面节点编号以创建中面上的相应节点。

| **输入文件用法：** | 使用以下选项指定单元底面上的节点和剩余面或多个面的节点的正偏移编号： |
| --- | --- |
|  | ``` [*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=*name*, OFFSET=*offset number* ``` 例如，以下行创建具有节点编号1, 2, 3, 4, 1001, 1002, 1003和1004的COH3D8单元编号11： ``` [*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=COH3D8, OFFSET=1000 11, 1, 2, 3, 4 ``` 以下行创建孔隙压力内聚单元COH3D8P单元编号11，具有节点编号1, 2, 3, 4, 1001, 1002, 1003, 1004, 2001, 2002, 2003和2004（节点1、2、3和4定义底面；节点1001、1002、1003和1004定义顶面；节点2001、2002、2003和2004定义中间面）： ``` [*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=COH3D8P, OFFSET=1000 11, 1, 2, 3, 4 ``` |

##### 通过仅指定底面和顶面节点来定义孔隙压力内聚单元

使用此方法，您仅指定孔隙压力内聚单元的底面和顶面上的节点以及一个正偏移编号。偏移编号被添加到中间面的底面节点编号以创建相应的节点。

| **输入文件用法：** | 使用以下选项指定孔隙压力内聚单元的底面和顶面上的节点以及剩余中间面节点的正偏移编号： |
| --- | --- |
|  | ``` [*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=*name*, OFFSET=*offset number* ``` 例如，以下行创建孔隙压力内聚单元COH3D8P单元编号11，具有节点编号1, 2, 3, 4, 1001, 1002, 1003, 1004, 2001, 2002, 2003和2004（节点1、2、3和4定义底面；节点1001、1002、1003和1004定义顶面；节点2001、2002、2003和2004定义中间面）： ``` [*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=COH3D8P, OFFSET=2000 11, 1, 2, 3, 4, 1001, 1002, 1003, 1004 ``` |

### 将单元分组为单元集

单元集用作定义荷载、属性等时的便捷交叉引用。单元集是模型的基本引用，应用于辅助输入定义。单元集的成员可以是单个单元或其他单元集。单个单元可以属于多个单元集。

单元可以在创建时或创建后分组为单元集。在任一情况下，每个单元集都被分配一个名称。单元集名称最多可以包含80个字符。

节点集和单元集可以使用相同的名称。

单元集内的所有单元将按其单元编号升序排列，并删除重复项。

一旦单元被分配到单元集，可以将其他单元添加到同一单元集；但是，不能从单元集中移除单元。

#### 在创建单元时将其分配到单元集

有几种方法可以在创建单元时将单元分配到单元集。

| **输入文件用法：** | 使用以下任一选项： |
| --- | --- |
|  | ``` [*ELEMENT](../key/key-link.md#usb-kws-melement), ELSET=*name* [*ELGEN](../key/key-link.md#usb-kws-melgen), ELSET=*name* [*ELCOPY](../key/key-link.md#usb-kws-melcopy), NEW SET=*name* ``` |

#### 将先前定义的单元分配到单元集

您可以通过直接列出形成集合的单元或通过生成单元集，将先前定义的单元（通过指定节点、增量生成或复制现有单元）分配到单元集。

##### 直接列出形成集合的单元

您可以直接列出形成单元集的单元。先前定义的单元集以及单个单元都可以被分配到单元集。

| **输入文件用法：** | ``` [*ELSET](../key/key-link.md#usb-kws-melset), ELSET=*name* ``` |
| --- | --- |
|  | 例如，以下行将单元3、13和20添加到集合`LEFT`： ``` [*ELSET](../key/key-link.md#usb-kws-melset), ELSET=LEFT 20 3, 13 ``` 以下行将单元5和16添加到现有集合`LEFT`： ``` [*ELSET](../key/key-link.md#usb-kws-melset), ELSET=LEFT 5, 16 ** 上述数据行等同于指定5, 16, LEFT ``` 以下行将单元22、14和集合`LEFT`中的所有单元添加到集合`B`： ``` [*ELSET](../key/key-link.md#usb-kws-melset), ELSET=B 22, 14, LEFT ``` 因此，单元集`B`包含以下单元：3、5、13、14、16、20和22。由于`LEFT`的定义出现在`B`的定义之前，因此单元集`LEFT`可以被分配到单元集`B`。 |

#### 定义部件和装配集

在以部件实例装配形式定义的模型中，所有单元集必须在部件、部件实例或装配定义中定义。如果在部件（或部件实例）内定义单元集，则可以直接引用单元编号。要定义装配级单元集，必须通过在每个单元编号前加上部件实例名称和"."来标识要添加到集合的单元（如["定义装配，" 第2.10.1节](pt01ch02s10aus28.md)中所述）。装配级单元集可以与部件级单元集具有相同的名称。

##### 示例

以下输入定义了一个属于部件`PartA`的单元集`set1`，并将由`PartA`的每个实例继承：

```
*PART, NAME=PartA
   ...
   *ELSET, ELSET=set1
    1,3,26,500
*END PART
```

具有相同名称的单元集在装配级别定义如下：
```
*ASSEMBLY, NAME=Assembly-1
   *INSTANCE, NAME=PartA-1, PART=PartA
    ...
   *END INSTANCE
   *INSTANCE, NAME=PartA-2, PART=PartA
    ...
   *END INSTANCE
   *ELSET, ELSET=set1
    PartA-1.1, PartA-1.3, PartA-1.26, PartA-1.500
    PartA-2.1, PartA-2.3, PartA-2.26, PartA-2.500
*END ASSEMBLY
```

装配级单元集`set1`包含属于部件实例`PartA-1`和`PartA-2`的部件实例`set1`中的所有单元。因此，单元被分配到两个独立的单元集：一个在部件实例级别，一个在装配级别。可以创建一个与属于部件集的单元完全不同的装配级单元集`set1`；部件级和装配级单元集是独立的。但是，由于在此示例中相同的单元被分配到部件级和装配级单元集`set1`，因此可以通过以下方式定义装配级集合
```
*ASSEMBLY, NAME=Assembly-1
   *INSTANCE, NAME=PartA-1, PART=PartA
    ...
   *END INSTANCE
   *INSTANCE, NAME=PartA-2, PART=PartA
    ...
   *END INSTANCE
   *ELSET, ELSET=set1
    PartA-1.set1, PartA-2.set1
*END ASSEMBLY
```

此单元集定义等效于前面的示例，其中单元是单独列出的。

##### 定义装配级单元集的替代方法

有时通过引用部件级单元集来定义装配级单元集并不方便。在这种情况下，包含许多单元的集合定义可能会非常冗长。因此，提供了一种替代方法。

| **输入文件用法：** | ``` [*ELSET](../key/key-link.md#usb-kws-melset), ELSET=*ElsetName*, INSTANCE=*InstanceName* ``` |
| --- | --- |

#### Abaqus/CAE创建的内部单元集

在Abaqus/CAE中，许多建模操作是通过用鼠标选择几何来执行的。例如，可以通过在几何部件实例的面上来创建曲面。由于[*SURFACE](../key/key-link.md#usb-kws-msurface)选项引用单元集，因此这个"选择"的几何必须被转换到输入文件中的单元集。这些集合由Abaqus/CAE分配名称并标记为内部。您可以使用Abaqus/CAE可视化模块中的显示组来查看这些内部集合（请参阅《Abaqus/CAE用户指南》第78章，"使用显示组显示模型的子集"(../usi/usi-link.md#uss-dgp)）。

| **输入文件用法：** | ``` [*ELSET](../key/key-link.md#usb-kws-melset), ELSET=*ElsetName*, INTERNAL ``` |

### 通过增量生成从现有单元创建单元

您可以从现有单元增量生成单元。新创建的单元始终与主单元具有相同的单元类型。

Abaqus首先通过使用节点和单元编号中规定的增量复制给定单元的节点模式来生成一排单元。然后可以重复此排以形成一层，一层也可以重复形成块。

要生成一排单元，您必须指定以下信息：
- 主单元编号。主单元必须在指定生成时存在，尽管它可以是在此同一单元生成中刚刚定义的单元。
- 要在生成的第一排中定义的单元数量，包括主单元。
- 排中从单元到单元相应节点的节点编号增量。默认为1。所有单元节点编号（除特殊用途节点外，稍后讨论）将以相同的值增加。
- 排中单元编号的增量。默认为1。

要复制此新创建的主排以创建一层单元，您必须指定以下附加信息：
- 要定义的行数，包括主行。
- 从行到行相应节点的节点编号增量。
- 从行到行相应单元的单元编号增量。

要复制此新创建的主层以创建单元块，您必须指定以下附加信息：
- 要定义的层数，包括主层。
- 从层到层相应节点的节点编号增量。
- 从层到层相应单元的单元编号增量。

| **输入文件用法：** | ``` [*ELGEN](../key/key-link.md#usb-kws-melgen) ``` |
| --- | --- |
|  | 例如，形成如图2.2.1-1所示的四分之一圆柱体的单元可以通过以下行生成： ``` [*ELGEN](../key/key-link.md#usb-kws-melgen) 1, 3, 1, 1, 5, 10, 10, 6, 100, 100 ``` |

#### 特殊用途节点的增量

默认情况下，以下节点不会递增：
- IRS型拖链单元的刚体参考节点；和
- 用于定义空间中梁或框架的第一个横截面轴方向的节点。

您可以指定所有节点都应该递增。您可以按照上述描述定义节点编号之间的增量。通常，只有对于用于定义空间中梁的第一个横截面轴方向的节点，才需要所有节点的递增。

| **输入文件用法：** | ``` [*ELGEN](../key/key-link.md#usb-kws-melgen), ALL NODES ``` |
| --- | --- |

### 通过复制现有单元创建单元

您可以通过复制现有单元来创建新单元。您必须标识要复制的现有单元集，并指定一个整数常量，将其添加到现有单元的节点编号中以定义新单元的节点编号。同样，您必须指定一个整数常量，将其添加到现有单元的单元编号中以定义正在创建的单元的单元编号。

**图2.2.1-1** 单元生成示例。

![](../graphics/ielement-elgen-exa-nls.png)

您可以将新创建的单元分配到单元集。如果不为新创建的单元指定单元集名称，则它们不会被分配到单元集。

| **输入文件用法：** | ``` [*ELCOPY](../key/key-link.md#usb-kws-melcopy), OLD SET=*name*, NEW SET=*new_name*, SHIFT NODES=*number*, ELEMENT SHIFT=*number* ``` |
| --- | --- |
|  | 例如，以下数据行将生成集合`B`中的新单元，这些单元是集合`A`中所有单元在处理此选项时的副本，每个单元编号和每个节点编号增加1000。在处理该行时，集合`A`的成员是所有单元生成和单元集定义行中出现在此[*ELCOPY](../key/key-link.md#usb-kws-melcopy)选项之前的输入文件中定义为集合`A`成员的单元。 ``` [*ELCOPY](../key/key-link.md#usb-kws-melcopy), OLD SET=A, NEW SET=B, ELEMENT SHIFT=1000, SHIFT NODES=1000 ``` |

#### 连续体单元的特别注意事项

复制现有单元时，您可以选择修改正在创建的单元的节点编号顺序，以避免创建违反Abaqus逆时针单元编号约定的连续体单元。当节点是通过复制现有节点生成的时，通常需要这种修改（["通过复制现有节点创建节点" in "节点定义，" 第2.1.1节](pt01ch02s01aus05.md#usb-int-inode-copy)）。

| **输入文件用法：** | ``` [*ELCOPY](../key/key-link.md#usb-kws-melcopy), REFLECT ``` |
| --- | --- |
|  | 例如，假设单元1在单元集`A`中，由节点1、2、3、4定义。以下数据行将生成单元编号11，也在集合`A`中，节点为11、14、13和12： ``` [*ELCOPY](../key/key-link.md#usb-kws-melcopy), OLD SET=A, NEW SET=A, ELEMENT SHIFT=10, SHIFT NODES=10, REFLECT ``` 如果未使用REFLECT参数，则新单元将由节点序列11、12、13、14定义，这将违反连续体单元使用的逆时针单元编号约定（请参见图2.2.1-2）。 |
