# 13.2 标题栏





默认情况下，标题栏中显示的字符串由传递给`AFXApp`构造函数的参数构建，如下所示：

```
AFXApp(appName, vendorName, productName, 
    majorNumber, minorNumber, updateNumber, prerelease)
```

其中`majorNumber`是版本号，`minorNumber`是发布号，`updateNumber`是更新号，`prerelease`是预发布号。

标题使用以下格式生成：

```
productName + majorNumber + '.' + minorNumber + 
    '-' + updateNumber
```
例如，以下语句：
```
AFXApp(productName="Abaqus/CAE", 
    majorNumber=6, minorNumber=12, updateNumber=1)
```
在标题栏中生成以下字符串：`Abaqus/CAE 6.12-1`

如果你没有在应用程序构造函数中指定major、minor和update号，它们默认为当前Abaqus/CAE发布号。类似地，如果你指定了发布号但没有指定产品名称，发布号默认为当前Abaqus/CAE发布号。如果你在`AFXApp`构造函数中将*prerelease*参数设置为True，则更新号前会加上`PRE`。例如，`Abaqus/CAE 6.12-PRE1`。

此外，如果用户打开了模型数据库，标题栏字符串包含当前模型数据库的名称；例如，`Abaqus/CAE 6.12–1 MDB: C:\projects\cars\engines\turbo-1.cae`。

如果当前模型数据库的名称（包括路径）超过50个字符，则通过仅显示由"…"分隔的前25个和最后25个字符来缩写名称。

如果你不想要默认的标题处理，你可以通过在`AFXMainWindow`构造函数中指定标题来覆盖它。如果你在`AFXMainWindow`构造函数中指定了标题，Abaqus GUI Toolkit会忽略应用程序构造函数中的参数，而使用指定的标题。即使你覆盖了默认的标题处理，当前模型数据库的名称和当前视口（最大化时）的名称将继续显示在标题栏中。

你可以使用以下语句访问窗口标题栏中显示的字符串：

```
title = getAFXApp().getAFXMainWindow().getTitle()
```
