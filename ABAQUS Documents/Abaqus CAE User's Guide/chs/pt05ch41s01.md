# 41.1 作业诊断概述





Abaqus在分析您的模型时会将诊断信息写入输出数据库，以及您请求的任何其他输出。输出数据库中的诊断信息是写入消息和状态文件的诊断信息的子集。您可以使用**作业诊断**对话框访问在Abaqus/Standard或Abaqus/Explicit分析作业期间写入输出数据库的诊断信息。

对于Abaqus/Standard分析，诊断信息可用于作业以及每个步骤、增量、尝试和迭代。[图41-1](pt05ch41s01.md#usv-out-jobdiag)显示Abaqus/Standard分析迭代的接触诊断。您可以选择要查看的接触信息类型，并可以选择一列对表格数据进行排序。

![](../graphics/usv-out-jobdiag-nls.png)

对于Abaqus/Explicit分析，诊断信息可用于作业以及每个步骤。由于典型Abaqus/Explicit分析中增量数量很大，诊断信息记录用于摘要（心跳）增量以及生成诊断信息的任何其他增量。摘要增量之间的间隔取决于CPU时间和分析中指定的输出量。诊断数据写入状态文件和输出数据库。

您可以使用**作业诊断**对话框来确定分析何时结束以及是否发出了任何警告。如果在作业期间发出了警告，您可以查看警告并评估您的结果是否可能受到影响。**作业诊断**对话框还提供更详细的信息来解释大多数警告和错误的含义或可能原因，如果警告与节点或单元关联，还可以帮助您在视口中定位它们。

您可以在分析期间或结束后查看诊断信息。但是，**作业诊断**对话框不会自动更新。如果您在分析运行时查看诊断信息，必须关闭然后重新打开输出数据库以显示任何新的诊断信息。
