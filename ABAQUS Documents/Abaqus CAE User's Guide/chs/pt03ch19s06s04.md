# 19.6.4 模型与重启分析之间的关系





当您首次提交基于模型的分析作业时，Abaqus/CAE 根据模型的定义生成输入文件；然后，该输入文件被提交进行分析。输入文件包含由网格模块生成的单元和节点定义，以及您使用 Abaqus/CAE 指定的材料、步骤、输出请求、载荷、相互作用等。

要运行与模型关联的作业的重启分析，建议您将模型复制到新模型。新模型包含原始模型的所有单元和节点定义，以及所有材料、步骤、输出请求、载荷、相互作用等。要请求重启分析，您可以编辑新模型的属性，并指定分析从原始分析的指定步骤继续。当您创建引用新模型的新重启作业时，Abaqus/CAE 将作业类型设置为**重启**。

当您提交新作业进行分析时，Abaqus/CAE 会根据重启信息生成输入文件。不会向重启输入文件写入任何零件、装配或属性数据。仅写入以下数据：
- 重新启动后出现的步骤。
- 与重新启动步骤之后出现的步骤关联的步骤相关对象的数据；例如，载荷、边界条件、场和输出请求。
- 由与重新启动步骤之后出现的步骤关联的步骤相关对象使用的新区域和幅值。

Abaqus 从原始分析生成的重启文件中读取零件、装配和属性数据，以及在重启步骤之前出现的步骤的数据。因此，尽管模型包含所有这些信息，但这些信息不会写入提交给 Abaqus/Standard 或 Abaqus/Explicit 的输入文件中。这是一个重要的考虑因素。您对重启分析中模型所做的某些更改（例如材料属性）不会出现在被分析的输入文件中。
