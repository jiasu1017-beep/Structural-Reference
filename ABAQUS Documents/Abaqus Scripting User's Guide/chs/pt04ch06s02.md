# 6.2 Kernel 和 GUI 进程







Abaqus/CAE 在两个进程中执行：kernel 进程和 GUI 进程。

**Kernel 进程**

Kernel 进程保存 Abaqus/CAE 用于执行建模操作的所有数据和方法；例如，创建零件和对装配进行网格划分。Kernel 进程可以独立于 GUI 进程运行。

**GUI 进程**

GUI 是用户指定 Abaqus/CAE 输入的一种便捷方式。内核命令字符串通过进程间通信（IPC）协议从 GUI 进程发送到 kernel 进程。Kernel 进程解释并执行内核命令字符串。如果内核命令抛出异常，异常会被传播回 GUI 进程，在那里应该被正确捕获和处理，通常通过显示错误对话框。

Abaqus/CAE 使用 IPC 协议来实现 kernel 和 GUI 进程之间的通信。例如，GUI 通常需要查询 kernel 以获取现有零件名称列表，或获取即将从对话框编辑的特定载荷的值。同样，当某些 kernel 值更改时，GUI 可能需要被通知，以便 GUI 可以更新自身；例如，在**作业监视器**对话框中发布新作业消息。

Abaqus/CAE 使用目标和消息以及内置于 Abaqus GUI Toolkit 中的 GUI 更新过程来实现 GUI 进程内的通信。例如，当当前视口更改时，选项对话框可能需要更新，或者当用户点击特定按钮时，对话框中的某些组件可能需要变灰。
