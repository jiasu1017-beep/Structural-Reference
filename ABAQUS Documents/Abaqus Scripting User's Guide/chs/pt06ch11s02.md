# 11.2 启动脚本





每个应用程序都从一个简短的启动脚本开始。启动脚本执行以下任务：
- 初始化应用程序对象。应用程序对象负责高级功能，例如管理消息队列和计时器、更新GUI以及控制主窗口。它不是一个可见对象。
- 实例化主窗口。主窗口是用户首次启动应用程序时看到的内容，并提供对所有应用程序功能的访问。
- 创建并运行应用程序。一旦应用程序运行，它将进入一个事件循环，在其中等待对用户输入（如鼠标点击）的响应。

以下是一个典型的启动脚本示例。

```
from abaqusGui import *
import sys
from caeMainWindow import CaeMainWindow

#Define a custom callback, myStartupCB(), that will be invoked 
#once the application has finished its startup processing
#
def myStartupCB():
    from myStartupDB import MyStartupDB
    db = MyStartupDB()
    db.create()
    db.show()

# Initialize the application object
#
app = AFXApp('Abaqus/CAE', 'SIMULIA')
app.init(sys.argv)
# Construct the main window
#
CaeMainWindow(app)

# Create the application
#
app.create()

# Register the custom startup callback
#
# NOTE: This call must be made after app.create()
setStartupCB( myStartupCB )

#Run the application
#
app.run()
```
脚本中的第一条语句从`abaqusGui`模块导入所有构造函数，包括`AFXApp`构造函数，该模块包含对整个Abaqus GUI Toolkit的访问。还导入了`sys`模块，因为需要将参数传递给应用程序的`init`方法。导入`sys`模块后，脚本导入主窗口构造函数。

此启动脚本已自定义为包含一个启动回调函数，该函数将在应用程序启动后显示一个自定义对话框**MyStartupDB**。回调在import语句之后、应用程序初始化之前定义。接下来的语句实例化并初始化应用程序对象。应用程序对象在["应用程序对象，"第12.1节](pt06ch12s01.md)中有更详细的讨论。然后脚本实例化主窗口。主窗口是用户启动应用程序时看到的内容。主窗口在[第13章，"主窗口"](pt06ch13.md)中有更详细的讨论。

应用程序构造函数创建该对象所需的所有数据结构，`app.create()`语句创建应用程序对象所需的所有GUI窗口。接下来注册自定义回调启动函数。

`app.run()`语句显示应用程序（包括自定义启动对话框），然后进入事件循环。事件循环等待用户交互。

启动自定义应用程序时，你可能希望在Abaqus/CAE执行过程中使用*–noStartup*选项来阻止Abaqus/CAE显示其自身的启动对话框。有关更多信息，请参见[Abaqus Analysis User's Guide](../usb/usb-link.md#usb-int-dcaeproc)第3.2.6节["Abaqus/CAE执行，"](../usb/usb-link.md#usb-int-dcaeproc)。
