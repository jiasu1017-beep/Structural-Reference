# D.3 在 Linux 上远程运行 Abaqus





可以在 NFS 挂载的文件系统上运行 Abaqus，也可以从远程计算机导出显示；但是，用户可能会在使用这些替代方案时遇到性能和可靠性问题。

### D.3.1 NFS 挂载的文件系统

在 NFS 挂载的文件系统上运行 Abaqus 有多种场景；使用 NFS 挂载的文件系统的影响因 NFS 使用方式而异。在 NFS 挂载的文件系统上运行 Abaqus 最常见的场景如下：
- Abaqus 安装在远程文件系统上。CPU、保存目录（请参阅["作业变量，"第 4.1.10 节](ch04s01.md#sgb-chp-env-job)）和临时目录是本地的（在与将运行 Abaqus 的计算机上）。在这种情况下，Abaqus 可执行文件和共享库在需要时通过网络从安装 Abaqus 的文件系统加载到本地内存中。处理和输出在本地进行。如果有足够的本地内存来防止代码频繁分页，网络流量将相对较轻。当本地内存不足以防止分页时，性能将受到影响，可靠性可能会受到影响。
- Abaqus 安装在远程文件系统上。保存目录和/或临时目录不在执行 Abaqus 的 CPU 本地。在这种情况下，程序文件和在执行期间使用的数据通过网络写入。可能会有大量数据跨网络传输，并且可能会对性能产生不利影响。Abaqus 性能可能非常慢，并且网络的其他用户可能会受到影响，因为网络可能会变得饱和。此外，Abaqus 不会捕获由 NFS 引起的文件错误，因此访问 NFS 挂载文件的失败（即使是暂时的）将导致 Abaqus 作业失败。应尽可能避免在这种配置中使用 Abaqus。如果 Abaqus/Standard 作业需要此配置，用户应在移动临时目录之前将保存目录移动到 NFS 挂载的文件系统。

### D.3.2 导出显示

此配置仅与 Abaqus/CAE 和 Abaqus/Viewer 相关。在这种配置中，Abaqus 安装在远程文件系统上，CPU、保存目录和临时目录位于同一远程计算机上，通过远程登录访问。所有处理都在远程计算机上进行，只有输出消息或图形被导出到本地计算机并显示。众所周知，这种方法会导致性能问题。OpenGL 和 GLX 库之间的轻微不兼容可能会导致严重的图形问题，在某些情况下，可能会阻止 Abaqus/CAE 或 Abaqus/Viewer 运行。不支持在这种配置中使用 Abaqus，应尽可能避免。
