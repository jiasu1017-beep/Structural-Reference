### 概述

Abaqus可以在分析期间创建以下输出文件：
- 包含由分析输入文件处理器生成的模型和历史定义打印输出的数据文件，以及在Abaqus/Standard中分析运行期间写入的结果打印输出；
- 包含用于使用Abaqus/CAE可视化模块（Abaqus/Viewer）后处理的结果以及在Abaqus/Standard中的诊断信息的ODB输出数据库文件；
- 包含用于在3DEXPERIENCE平台上使用Physics Results Explorer应用进行高性能后处理的结果的SIM数据库文件；
- Abaqus/Explicit中的选定结果文件；
- 在Abaqus/Standard和Abaqus/Explicit中包含用于外部软件后处理的结果的结果文件（在Abaqus/Explicit中，此文件通过转换选定结果文件生成）；
- 在Abaqus/Standard和Abaqus/Explicit中包含关于求解进度的诊断消息的消息文件；
- 包含分析状态信息以及在Abaqus/Explicit中关于稳定时间增量的诊断消息和信息的状态文件；和
- 在Abaqus/CFD中使用备用文件格式的输出文件。

Abaqus可以创建用于重启分析的文件——请参阅["重启分析，" 第9.1.1节](pt04ch09s01aus53.md)。在Abaqus/Standard中，这些文件也可用于提取分析期间未请求的结果输出。

### 数据文件

数据文件（`*job-name*.dat`）是一个文本文件，包含关于模型定义（由分析输入文件处理器生成）的信息，以及在Abaqus/Standard中的结果表格输出。分析输入文件处理器信息包括模型定义、历史定义以及在处理输入数据时检测到的任何错误和警告消息的标识。

#### 控制写入数据文件的分析输入文件处理器信息量

您可以控制由Abaqus/Standard和Abaqus/Explicit中的分析输入文件处理器写入数据文件的信息量。

#### 请求打印结果

在Abaqus/Standard中，输出变量的值可以以表格格式在整个分析过程中打印到数据文件。您可以在分析运行期间控制以下类型的打印输出：单元输出、节点输出、接触表面输出、能量输出、紧固件相互作用输出、模态输出、截面输出和辐射输出——请参阅["输出到数据和结果文件，" 第4.1.2节](pt02ch04s01aus39.md)和["空腔辐射，" 第41.1.1节](pt09ch41s01aus187.md)。您在每个输出表中指定要打印的变量，对于单元变量，指定要打印它们的位置（在积分点、在单元质心、在节点或平均在节点）。具有转换的节点上的节点变量可以以全局或局部坐标系写入（请参阅["变换坐标系，" 第2.1.5节](pt01ch02s01aus09.md)）。可用变量列表在["Abaqus/Standard输出变量标识符，" 第4.2.1节](pt02ch04s02abv01.md)中给出。结果输出到数据文件的请求作为步骤定义的一部分提出。

#### 在数据文件中查看零件和装配信息

Abaqus模型可以根据零件实例的装配来定义（请参阅["定义装配，" 第2.10.1节](pt01ch02s10aus28.md)）。在这种模型中，节点和单元编号可以在不同零件的定义中重复。这些本地编号由Abaqus在内部转换为唯一的全局编号，写入数据文件的输出以这些内部编号给出。用户定义的编号和内部编号之间的映射在数据文件中打印（如果在数据文件中请求了包含节点和单元编号的任何输出）。

出现在数据文件中的集合和表面名称以装配和零件实例名称为前缀，用下划线分隔（例如`Assembly_Part1–1_setname`）。

在零件或零件实例内定义的局部坐标系根据零件实例定义中给出的定位数据进行了平移和旋转。

### 输出数据库

输出数据库是一个中性二进制文件。与重启或二进制结果文件不同，它可以直接从一个计算平台复制到另一个平台，而无需转换。

#### 输出数据库的格式

Abaqus输出数据库有两种格式：ODB和SIM。默认情况下，结果输出以ODB格式创建。对于Abaqus/Standard或Abaqus/Explicit分析，您可以选择在同一作业期间以两种格式写入结果。只有SIM格式的结果可以导入到3DEXPERIENCE平台进行高性能后处理。更多信息，请参阅["以SIM格式写入结果的限制](pt02ch04s01aus38.md#usb-out-ooutput-sim-limitations)"下文。
- ODB输出数据库（`*job-name*.odb`）用于以零件实例装配的形式存储模型信息和分析结果。Abaqus/CAE的可视化模块（Abaqus/Viewer）使用此输出数据库进行后处理分析结果和查看诊断信息。
- SIM数据库文件（`*job-name*.sim`）包含模型和结果信息。3DEXPERIENCE平台上的Physics Results Explorer应用使用此数据库进行高性能后处理。

| **输入文件用法：** | 使用以下命令行选项以SIM格式写入Abaqus/Standard或Abaqus/Explicit分析的结果： |
| --- | --- |
|  | **abaqus** **job**=*job-name* **resultsformat**=`sim` 使用以下命令行选项以ODB格式和SIM格式写入Abaqus/Standard或Abaqus/Explicit分析的结果：**abaqus** **job**=*job-name* **resultsformat**=`both` 使用以下命令行选项以SIM格式写入Abaqus/CFD分析的场输出结果：**abaqus** **job**=*job-name* **field**=`sim` 使用以下命令行选项以SIM格式写入Abaqus/CFD分析的历史输出结果：**abaqus** **job**=*job-name* **history**=`sim` |

| **Abaqus/CAE用法：** | 使用以下输入以SIM格式写入Abaqus分析的结果： |
| --- | --- |
|  | 作业模块：作业编辑器：**常规**：**结果格式**：**SIM** 使用以下输入以ODB格式和SIM格式写入Abaqus/Standard或Abaqus/Explicit分析的结果：作业模块：作业编辑器：**常规**：**结果格式**：**两者** |

##### 处理浮点数据

默认情况下，浮点数据以单精度写入ODB输出数据库文件。您可以选择以双精度将浮点节点场输出数据写入ODB输出数据库文件；请参阅["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行，" 第3.2.2节](pt01ch03s02abx02.md)，了解更多详情。

对于Abaqus/Standard和Abaqus/Explicit分析，浮点数据以单精度写入SIM数据库，但节点坐标以双精度写入。对于Abaqus/CFD分析，浮点数据以双精度写入SIM数据库。

##### 在Abaqus/CAE中打开输出数据库

您可以在Abaqus/CAE中打开旧版本Abaqus的输出数据库文件，但Abaqus 5.8输出数据库文件无法在版本6中打开。来自先前版本Abaqus的输出数据库文件必须在打开时转换到当前版本。如果您使用的是旧版本的Abaqus/CAE，则无法打开由新版本Abaqus创建的输出数据库文件。

##### 选择输出格式

您选择输出格式取决于您的高性能可视化体验水平、Physics Results Explorer应用以及后处理需求。
- 如果您仍在学习使用高性能可视化，并希望将结果与Abaqus/Viewer进行比较，请以两种格式写入结果。
- 如果模型很大，您需要Physics Results Explorer应用的改进性能以及Abaqus/Viewer的功能，请以两种格式写入结果。
- 如果您确信Physics Results Explorer应用中的高性能可视化功能提供您需要的所有功能，请以SIM格式写入结果。

#### 请求输出到输出数据库

您可以从["Abaqus/Standard输出变量标识符，" 第4.2.1节](pt02ch04s02abv01.md)、["Abaqus/Explicit输出变量标识符，" 第4.2.2节](pt02ch04s02xbv01.md)和["Abaqus/CFD输出变量标识符，" 第4.2.3节](pt02ch04s02cbv01.md)中选择要写入输出数据库的变量。以下类型的输出可用：单元输出、节点输出、接触表面输出、能量输出、积分输出、时间增量输出、紧固件相互作用输出、模态输出和辐射输出。此外，Abaqus/Standard和Abaqus/Explicit中写入消息文件（请参阅["Abaqus/Standard和Abaqus/Explicit中的消息文件](pt02ch04s01aus38.md#usb-out-ooutput-message)"）的诊断信息子集以及Abaqus/Explicit状态文件（请参阅["状态文件](pt02ch04s01aus38.md#usb-out-ooutput-status)"）也包含在输出数据库中。请参阅["输出到输出数据库，" 第4.1.3节](pt02ch04s01aus40.md)，了解如何生成输出数据库请求的详细说明。

输出数据库中存储了三类信息："场"输出、"历史"输出和诊断信息。场输出旨在用于模型大部分的相对不频繁输出。Abaqus/CAE使用场输出在可视化模块中生成等高线图、变形形状图、符号图和*X–Y*图。历史输出旨在以相当高的频率为模型的小部分输出。Abaqus/CAE使用历史输出在可视化模块中生成*X–Y*图。请参阅["输出到输出数据库，" 第4.1.3节](pt02ch04s01aus40.md)，了解场和历史输出的详细描述。诊断信息旨在提供用于Abaqus/CAE的收敛信息；更多信息，请参阅[Abaqus/CAE用户指南第41章"查看诊断输出"](../usi/usi-link.md#usv-output)。

### 选定结果文件

Abaqus/Explicit选定结果文件（`*job-name*.sel`）存储用户选择的结果，这些结果将转换为结果文件（`*job-name*.fil`）以供其他商业后处理包进行后处理。

可以请求单元输出、节点输出和能量输出（请参阅["输出到数据和结果文件，" 第4.1.2节](pt02ch04s01aus39.md)，了解更多详情）；可用于输出的变量列在["Abaqus/Explicit输出变量标识符，" 第4.2.2节](pt02ch04s02xbv01.md)中。您可以为给定的节点集或单元集写入用户选择的结果子集，频率高于重启间隔。您可以在步骤定义中指定输出请求，这允许您选择写入选定结果文件的数据量以避免使用过多的磁盘存储。例如，当处理非常大的模型时，您可能只选择在整个模型的步骤中写入当前位移和等效塑性应变20次，并在某个节点上写入加速度历史200次。

### 结果文件

Abaqus/Standard和Abaqus/Explicit中的Abaqus结果文件（`*job-name*.fil`）可以由外部后处理器读取，以生成*X–Y*图或打印的表格输出。大多数商业有限元结果展示包提供使用Abaqus结果文件作为输入的转换器。结果文件也可以作为将分析结果导入您自己的后处理程序的方便媒介。[^1]

[^1]: ["访问结果文件信息，" 第5.1.3节](pt02ch05s01aus43.md)，提供了如何读取此文件的详细信息。

从热传递、热电或热电-结构分析的结果文件输出的温度可用作相同网格的应力分析的输入（请参阅["顺序耦合热应力分析，" 第16.1.2节](pt04ch16s01at39.md)）。

#### 在Abaqus/Standard中获取结果文件输出

在Abaqus/Standard中，您可以从["Abaqus/Standard输出变量标识符，" 第4.2.1节](pt02ch04s02abv01.md)中选择要写入结果文件的变量，方式与输出到数据文件的类似。您必须专门请求将值写入结果文件，否则不会提供。单元输出、节点输出、接触表面输出、能量输出、模态输出和辐射输出可用——请参阅["输出到数据和结果文件，" 第4.1.2节](pt02ch04s01aus39.md)和["空腔辐射，" 第41.1.1节](pt09ch41s01aus187.md)，了解更多详情。

##### 在步骤开始时获取结果

您可以请求将步骤开始时的求解状态（零增量）写入Abaqus/Standard结果文件。零增量文件输出仅适用于时间控制增量方案步骤的过程，因此以下过程被排除：
- 线性静态扰动分析（["静态应力分析，" 第6.2.2节](pt03ch06s02at01.md)）
- ["特征值屈曲预测，" 第6.2.3节](pt03ch06s02at02.md)
- ["自然频率提取，" 第6.3.5节](pt03ch06s03at10.md)
- ["基于模式的稳态动态分析，" 第6.3.8节](pt03ch06s03at13.md)
- ["响应谱分析，" 第6.3.10节](pt03ch06s03at15.md)
- ["随机响应分析，" 第6.3.11节](pt03ch06s03at16.md)

如果您请求零增量结果文件输出，它将为给定分析中的所有有效过程生成。

如果您使用结果文件驱动子模型，则必须请求零增量结果文件输出以在数据检查分析中生成零增量结果文件（请参阅["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行，" 第3.2.2节](pt01ch03s02abx02.md)）；请参阅["基于节点的子模型，" 第10.2.2节](pt04ch10s02aus61.md)，了解更多讨论。

#### 结果文件格式

Abaqus/Standard或Abaqus/Explicit中的Abaqus结果文件组织为顺序文件，可以是二进制或ASCII格式。如果文件将在与写入文件的计算机不同的计算机系统上读取，则需要ASCII格式。ASCII格式允许在不同计算机系统之间传输结果文件，而无需转换二进制数据。如果文件将始终在同一系统上使用，或在使用相同二进制格式的系统上使用，则不需要ASCII格式。如果结果文件始终位于同一台计算机上，默认的二进制格式通常是存储文件的最有效方式。对于大型问题，ASCII格式的文件将比相同文件的二进制格式大得多。

##### 控制Abaqus/Explicit中结果文件的格式

Abaqus/Explicit始终在文件转换期间以二进制格式写入结果文件输出，但二进制Abaqus/Explicit结果文件可以使用**ascfil**工具转换为ASCII格式（["ASCII翻译结果（`.fil`）文件，" 第3.2.13节](pt01ch03s02abx13.md)）。

##### ASCII格式

["结果文件输出格式，" 第5.1.2节](pt02ch05s01afi01.md)定义了写入结果文件的记录内容；这些描述也适用于以ASCII格式写入的结果文件。所有这些文件中的数据项都是整数、浮点数或字符串。当请求ASCII格式时，每个数据项在写入文件之前被翻译成等效的字符串。这些字符串以80字符的逻辑记录形式写入，记录顺序如记录定义中所述。

每个80字符的逻辑记录在开始下一个之前完全填满，因此任何数据项都可以拆分，部分字符在一个逻辑记录中，其余在下一个。通常，每个数据项紧跟在其前一个数据项之后。例外情况是，对于结果文件记录键2001，Abaqus将用空白字符填充逻辑记录，以便可以立即将记录写入物理存储介质。然后Abaqus插入一个由80个空白组成的逻辑记录，这允许正确处理文件结束。

每个"记录"的开始用星号（*）表示。每个浮点数以字符D开头，后跟E22.15或D22.15格式的数字，具体取决于写入结果文件的Abaqus版本是使用单精度还是双精度。每个字符串以字符A开头，后跟八个字符（如果字符串少于八个字符，则字符串的右侧为空白；超过八个字符的字符串每次写入八个字符）。每个整数以字符I开头，后跟一个两位整数，给出整数中的十进制位数，然后是整数本身（写为十进制数字）。

例如，对于单元编号为5且节点为195、198、205和204的S4R元素，记录键1900将写为

```
*I 18I 41900I 15AS4R     I 3195I 3198I 3205I 3204
```

而对于节点135和6个自由度的记录键101将写为
```
*I 19I 3101I 3135D1.280271914214298E-10D1.500000000000036E+00
D-1.074629835784448E-46D 6.983222716550941E-12
D-4.084928798492785E-13D-1.072688441364597E-10
```

##### 结果文件中浮点数据的精度

写入结果文件的浮点数据的精度取决于生成数据的可执行文件的精度。Abaqus/Standard始终使用双精度；因此，浮点数据始终以双精度写入Abaqus/Standard结果文件。Abaqus/Explicit可以单精度或双精度运行；请参阅["定义分析，" 第6.1.2节](pt03ch06s01abo05.md)，了解更多关于Abaqus/Explicit可执行文件的精度级别的信息。如果使用Abaqus/Explicit的双精度可执行文件，浮点数据以双精度写入Abaqus/Explicit结果文件；同样，如果使用Abaqus/Explicit的单精度可执行文件，浮点数据以单精度写入Abaqus/Explicit结果文件。

#### 最大化结果文件的效率

在Abaqus/Standard中，每个单元输出请求（一行上输入的标识键的集合）前面都有一个"单元头"记录（请参阅["结果文件输出格式，" 第5.1.2节](pt02ch05s01afi01.md)）。因此，通过在同一行上输入相同"类型"（单元积分点变量、单元截面变量、整个单元变量等）的所有单元输出变量，可以最小化结果文件的大小。（请参阅["输出到数据和结果文件，" 第4.1.2节](pt02ch04s01aus39.md)，了解输出变量类型的说明。）鼓励合并输出变量条目，因为这将减小结果文件的大小。

##### 示例

例如，以下输出请求可用于在应力/位移分析中请求结果文件中单元变量的输出：

```
[*EL FILE](../key/key-link.md#usb-kws-helfile)

[*EL FILE](../key/key-link.md#usb-kws-helfile), REBAR

```

（钢筋数量的输出请求不必与底层单元输出请求相同。）

### 状态文件

状态文件（`*job-name*.sta`）是一个文本文件，包含关于分析进度信息。

#### Abaqus/Standard或Abaqus/CFD状态文件

Abaqus/Standard或Abaqus/CFD状态文件为分析的每个增量包含一个80字符记录，并在分析每个增量完成时更新。此记录直接写入辅助存储，因此在分析作业执行时可以检查状态文件，从而提供分析进度监视器。除了在Abaqus/Standard中在状态文件中指定要监视的自由度变量（如下所述）之外，写入Abaqus/Standard或Abaqus/CFD状态文件的信息无法控制。

#### Abaqus/Explicit状态文件

在Abaqus/Explicit中，状态文件（`*job-name*.sta`）默认包含模型的质量和惯性属性、初始稳定时间增量信息、包括总累积CPU使用率和当前时间增量大小的分析进度概要，以及处理每个步骤所需的内存估计。您可以控制其他输出，包括总动能、能量平衡、具有最小稳定时间增量的单元标识符以及由于质量缩放导致的模型总质量变化百分比。

写入Abaqus/Explicit状态文件的摘要增量频率取决于CPU分钟的分析持续时间和分析中指定的输出量。以下列表提供了关于何时将摘要增量写入状态文件的一般指南。

通常在以下情况下写入摘要信息：
- 每次写入重启信息、场输出到输出数据库或结果文件输出时。
- 如果问题需要少于20个增量，则每个增量一次。
- 对于短分析（少于40 CPU分钟），步骤中20次。
- 对于超过40 CPU分钟的分析，每2 CPU分钟一次。

当分析运行时，可以在状态文件中监视自由度变量。您还可以将其他诊断信息写入状态文件（请参阅["显式动态分析，" 第6.3.3节](pt03ch06s03at08.md)和["Abaqus/Explicit分析中的接触诊断，" 第39.2.1节](pt09ch39s02aus185.md)，了解更多详情）。包括每个摘要增量的信息的诊断信息的一部分也写入输出数据库以供Abaqus/CAE使用（更多信息，请参阅["在Abaqus/Standard和Abaqus/Explicit中请求诊断信息"中的"输出到输出数据库，" 第4.1.3节](pt02ch04s01aus40.md#usb-out-odboutput-diagnostic)）。

只能在与数据打包或分析期间检测到的错误也会写入状态文件。

### Abaqus/CFD中的备用输出格式

默认情况下，当您在Abaqus/CFD中请求输出时，输出将发送到输出数据库文件。但是，您可以选择为场和历史输出选择备用文件格式。场输出可以发送到EXODUS-II格式的文件；历史输出可以发送到逗号分隔值（CSV）格式的文件。

您以与["请求输出到输出数据库](pt02ch04s01aus38.md#usb-out-ooutput-requesting)"中所述相同的方式请求场和历史输出。要选择备用输出格式，您在运行Abaqus/CFD分析时在命令行上设置**field**和**history**选项。更多信息，请参阅["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行，" 第3.2.2节](pt01ch03s02abx02.md)。

#### EXODUS-II格式的场输出

EXODUS-II格式被第三方后处理器广泛支持，用于计算固体力学和计算流体动力学。此格式是二进制的、机器无关的，非常适合非结构化网格上的瞬态模拟结果。

EXODUS-II格式及相关的EXODUS-II/NEMESIS编程API由Sandia国家实验室开发。此开源软件可根据BSD许可证获得。源代码和文档可在[http://sourceforge.net/projects/exodusii](http://sourceforge.net/projects/exodusii)找到。

EXODUS-II格式本身无法表示所有Abaqus/CFD输出功能。[表4.1.1-1](pt02ch04s01aus38.md#table-alt-output-mods)中列出的功能无法直接表示，将被省略或修改。

**表4.1.1-1** EXODUS-II格式中的Abaqus/CFD输出功能表示。
| 功能 | 注释 |
| --- | --- |
| 零件和装配 | 节点和单元编号不包括零件实例名称，而是按顺序编号（请参阅["命名约定"中的"定义装配，" 第2.10.1节](pt01ch02s10aus28.md#usb-int-ipartassy-naming)） |
| 单元集 | 不支持通用单元集，将被省略 |
| 振幅 | 不支持 |

EXODUS-II格式使用文件扩展名`exo`。对于分析运行的并行处理，EXODUS-II输出定向到多个文件（每个处理器创建一个文件），这对某些第三方后处理器很有用。文件名为`*job*.exo.*rank*`，其中*rank*是从0到CPU数量减1的数字。相比之下，您可以将并行执行的场输出写入单个文件（`*job*.exo`）；该文件使用NEMESIS库以EXODUS-II格式写入。

|  | **abaqus** **job**=*job-name* **field**=`exodus` 在Abaqus/CFD中使用以下命令行选项将场输出以EXODUS-II格式写入单个文件以进行并行执行：**abaqus** **job**=*job-name* **field**=`nemesis` |

| **Abaqus/CAE用法：** | 您无法在Abaqus/CAE中选择场输出的备用格式。 |

#### CSV格式的历史输出

逗号分隔值（CSV）格式是一种基于文本的输出格式。CSV文本文件的格式由一行或多行注释组成，后跟每行逗号分隔的数据。CSV文件中的注释以字符`#`开头。CSV文件中的每列都有一个注释，描述网格位置、零件实例和输出请求标签。网格位置的可能值为节点、单元或表面。向量输出请求还包括分量；即1、2或3。

此格式使用文件扩展名`csv`。CSV格式的历史输出为每个步骤的每个输出请求标签创建一个文件。如果作业在并行运行，并且与历史输出请求关联的集合由于域分解而在处理器之间拆分，则会创建其他文件。在这种情况下，每个处理器上将有一个文件，其中存在该集合。文件名为`*job*_*output-request*_*rank*_*step-number*.csv`，其中*rank*是从0到CPU数量减1的数字。

| **输入文件用法：** | 在Abaqus/CFD中使用以下命令行选项将历史输出写入备用文件格式： |
| --- | --- |
|  | **abaqus** **job**=*job-name* **history**=`csv` |

| **Abaqus/CAE用法：** | 您无法在Abaqus/CAE中选择历史输出的备用格式。 |

### 在多个步骤中请求输出

通常，输出请求适用于定义它们的步骤以及所有后续步骤，直到它们被重新指定。但是，线性扰动步骤（仅在Abaqus/Standard中可用；见下文和["一般和线性扰动过程，" 第6.1.3节](pt03ch06s01aus44.md)）的输出规范独立于一般分析步骤的输出请求，仅适用于连续的线性扰动步骤序列。

数据库输出、打印输出和结果文件输出是Abaqus中独立的输出模式；因此，更改一种形式的输出的规范不会影响其他形式。

#### 一般分析步骤

除非您重新定义它们，否则在分析的第一个一般分析步骤中使用默认输出请求。对于后续一般分析步骤，除非您重新定义它们，否则保留先前一般步骤中每种输出形式的定义。

#### 线性扰动步骤

除非在步骤中重新定义，否则在连续线性扰动步骤序列的第一个中使用默认输出请求。如果定义了后续线性扰动步骤而没有中间一般分析步骤，则除非在步骤中重新定义，否则保留先前扰动步骤中每种输出模式的定义。如果定义了中间一般步骤，则默认输出请求再次在线性扰动步骤中使用，除非在步骤中重新定义。

### Abaqus/Standard中的单元矩阵输出

在Abaqus/Standard中，您可以为每个步骤将单元刚度矩阵以及质量矩阵（如果有）写入文件。对于热传递元素，如果请求刚度矩阵输出，则写入算子矩阵。

单元矩阵输出仅适用于没有内部节点的单元（除非这些节点没有活动自由度）且没有声学或内部自由度的单元。禁止单元矩阵输出的单元示例包括声学、管道、弯头、框架、间隙和接口单元，以及具有傅里叶模式的轴对称单元。单元矩阵输出不适用于具有耦合场的单元，例如耦合温度-位移单元和孔隙压力单元。对于不兼容模式和混合单元，禁止刚度矩阵输出，而质量矩阵输出可用。子结构矩阵输出请求用于将子结构的缩减刚度矩阵、质量矩阵和荷载情况向量写入文件（请参阅["定义子结构，" 第10.1.2节](pt04ch10s01aus59.md)）。

单元矩阵输出无法在基于模式的动态分析（响应谱、稳态动态、模态动态或随机响应）中请求。但是，它可以在特征频率提取分析中请求，以输出质量和刚度矩阵。

单元矩阵的写入不考虑节点条件的影响；因此，边界条件、集中载荷和多点约束的影响不包括在此输出中。自由度始终在全局方向，即使在关联元素的节点上定义了局部坐标系（["变换坐标系，" 第2.1.5节](pt01ch02s01aus09.md)）。

您必须选择请求输出的单元集。对于根据零件实例的装配定义的模型（["定义装配，" 第2.10.1节](pt01ch02s10aus28.md)），与单元矩阵输出一起写入的单元编号是由Abaqus/Standard生成的内部编号。内部编号与原始单元编号和零件实例名称之间的映射在数据文件中提供。

### 在Abaqus/Standard中从重启数据恢复其他结果输出

Abaqus/Standard中重启所需的数据包含在多个文件中，这些文件在您请求为分析写入重启数据时生成：重启（`.res`）、分析数据库（`.mdl`和`.stt`）、零件（`.prt`）和输出数据库（`.odb`）文件。[^1]

[^1]: ["重启分析，" 第9.1.1节](pt04ch09s01aus53.md)更详细地描述了重启数据的写入。

在Abaqus/Standard中，您可以从重启数据中提取输出并将其写入新的数据（`.dat`）、结果（`.fil`）和输出数据库（`.odb`）文件，方法是使用后处理分析过程。如果原始分析包含用户子程序，则后处理分析过程需要指定用户子程序。数据、结果和输出数据库文件输出请求的定义与["输出到数据和结果文件，" 第4.1.2节](pt02ch04s01aus39.md)和["输出到输出数据库，" 第4.1.3节](pt02ch04s01aus40.md)中的描述相同。输出请求的定义应与分析中的定义完全相同，只是：

1. 输出频率规范没有意义，因此被忽略（除非您正在从先前的直接循环或低循环疲劳分析恢复其他输出）。相反，您在后处理过程定义中指定要生成输出的每个增量。
2. 不提供输出数据库的默认输出。此外，模型信息（如边界条件）不会写入输出数据库。
3. 无法从重启文件中恢复单元集能量信息，因为它不会写入重启文件。
4. 对于不支持重启的过程，输出不可用；例如，线性扰动过程。

为分析定义的单元集和节点集可用于在后处理过程定义中定义输出集。也可以为后处理过程定义其他集合。您指定从重启文件中需要输出的步骤号。您无法在步骤开始时获得结果（见下文）。

| **输入文件用法：** | ``` [*POST OUTPUT](../key/key-link.md#usb-kws-hpostoutput), STEP=*step_number* ``` |
| --- | --- |
|  | 当使用[*POST OUTPUT](../key/key-link.md#usb-kws-hpostoutput)选项时，它必须作为输入文件中的第一个选项出现。不需要来自分析输入文件的任何数据行。此选项可以根据需要重复多次以获得进一步输出。由于[*POST OUTPUT](../key/key-link.md#usb-kws-hpostoutput)是一个纯后处理过程，因此分析选项不得出现在输入文件中。 |

| **Abaqus/CAE用法：** | Abaqus/CAE不支持重启数据的后处理。 |
| --- | --- |

#### 示例

可以使用以下输入文件提交作业。生成重启数据的分析在提交作业时必须指定（使用Abaqus执行程序的**oldjob**参数）。此示例创建一个新的数据（`.dat`）文件，其中包含表格数据。前两个表将包含步骤1的增量为5和10的数据，并将给出集合`CLAMP`中节点的约束反力，该集合在分析运行时被定义。下一个表将包含步骤2中增量为3的数据，并将给出在此后处理中定义的新节点集`TIP`的位移。

```
[*HEADING](../key/key-link.md#usb-kws-mheading)
[*POST OUTPUT](../key/key-link.md#usb-kws-hpostoutput), STEP=1
 5, 10
[*NODE PRINT](../key/key-link.md#usb-kws-hnodeprint), NSET=CLAMP
 RF,
[*POST OUTPUT](../key/key-link.md#usb-kws-hpostoutput), STEP=2
 3,
[*NSET](../key/key-link.md#usb-kws-mnset), NSET=TIP
 1200, 1203, 1205
[*NODE PRINT](../key/key-link.md#usb-kws-hnodeprint), NSET=TIP
```
