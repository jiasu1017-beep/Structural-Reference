# 1.2.1 输入语法规则





**产品：** Abaqus/Standard  Abaqus/Explicit  

##### **参考**

- ["在 Abaqus 中定义模型，" 第 1.3.1 节"](pt01ch01s03aus03.md)

### 概述

本节描述控制 Abaqus 输入文件的语法规则。

Abaqus 中的所有数据定义都通过选项块完成——描述问题定义一部分的数据集。您选择与特定应用相关的选项。选项通过输入文件中的行定义。Abaqus 输入文件中使用三种类型的输入行：*关键字*行、*数据*行和*注释*行。仅支持 7 位 ASCII 字符，输入文件中每行的末尾必须带有回车符。
- 关键字行引入选项，通常带有*参数*，参数以逗号分隔的单词或短语形式出现在关键字行上。参数用于定义选项的行为。参数可以独立使用，也可以有值，可以是必需的或可选的。
- 数据行用于提供数值或字母数字条目，跟在大多数关键字行后面。
- 以星号（**）开头的行（位于第 1 和第 2 列）是注释行。这些行可以放在文件中的任何位置。它们被 Abaqus 忽略，因此只会在文件的初始列表中打印。文件中这些行的出现次数和位置没有限制。

相关参数和数据行（包括每行条目数）在 [Abaqus 关键字参考指南](../key/key-link.md#key) 描述每个选项的章节中描述。本节描述适用于所有关键字和数据行的一般规则。

### 关键字行

输入关键字行时适用以下规则：
- 每个关键字行的第一个非空白字符必须是星号（*）。
- 如果给出任何参数，关键字后面必须跟逗号（,）。
- 参数必须用逗号分隔。
- 关键字行上的空白被忽略。
- 一行最多可包含 256 个字符，包括空白。
- 关键字和参数不区分大小写。
- 参数值通常不区分大小写。此规则唯一的例外是由 Abaqus 外部强加的，例如区分大小写的操作系统上的文件名。
- 关键字、参数，在大多数情况下，参数值不必完整拼写，但必须给出足够的字符以将它们与以相同方式开头的其他关键字、参数和参数值区分开来。Abaqus 首先在每个关联文本字符串中搜索精确匹配。如果未找到精确匹配，Abaqus 然后根据每个关键字、参数或参数值中的最少唯一字符数进行搜索。关键字行中的任何项目都可以省略嵌入的空白。如果参数值用于提供数字或文件名，应提供完整值。
- 如果参数有值，则使用等号（=）。该值可以是整数、浮点数或字符串，具体取决于上下文。例如，``` [*ELASTIC](../key/key-link.md#usb-kws-melastic), TYPE=ISOTROPIC, DEPENDENCIES=1 ```
- 当参数值是表示项目名称的字符串时，除非值被引号括起来，否则不应使用大小写作为区分值的方法。例如，Abaqus 不区分以下定义：``` [*MATERIAL](../key/key-link.md#usb-kws-mmaterial), NAME=STEEL [*MATERIAL](../key/key-link.md#usb-kws-mmaterial), NAME=Steel ```
- 同一参数不应在单个关键字行上出现多次。如果参数在单个关键字行上有多个设置，Abaqus 会忽略除其中一个之外的所有设置。
- 有时需要延续关键字行；例如，由于参数很多。如果关键字行最后一个字符是逗号，则下一行被解释为该行的延续。例如，上面的 [*ELASTIC](../key/key-link.md#usb-kws-melastic) 关键字行也可以给出为 ``` [*ELASTIC](../key/key-link.md#usb-kws-melastic), TYPE=ISOTROPIC, DEPENDENCIES=1 ```
- 某些关键字必须与其他关键字结合使用；例如，[*ELASTIC](../key/key-link.md#usb-kws-melastic) 和 [*DENSITY](../key/key-link.md#usb-kws-mdensity) 关键字必须与 [*MATERIAL](../key/key-link.md#usb-kws-mmaterial) 关键字结合使用。这些相关关键字必须在输入文件中分组在一个块中；不相关的关键字不能在此块内指定。
- 某些选项允许将 INPUT 或 FILE 参数设置为备用文件的名称。此类文件名可以包括完整路径名或相对路径名。相对路径名必须相对于提交作业的目录。如果未指定路径，则假定文件位于提交作业的目录中。子结构库必须位于提交作业的同一目录中；不能使用完整路径名来指定子结构库名称。对于由 INPUT 参数引用的文件，文件名必须包含任何扩展名（例如，`elem.inp`）。对于由 FILE 参数引用的文件，在大多数情况下必须给出不带扩展名的名称，因为 Abaqus 假定要读取的文件具有与选项相关的文件类型的正确扩展名：重新启动文件的 `.res`（["重新开始分析，" 第 9.1.1 节"](pt04ch09s01aus53.md)）和结果文件的 `.fil`（["输出，" 第 4.1.1 节"](pt02ch04s01aus38.md)）。但是，当结果文件（`.fil`）或输出数据库文件（`.odb`）与选项相关时，可能适用特殊规则（详情请参阅 ["Abaqus/Standard 和 Abaqus/Explicit 中的初始条件，" 第 34.2.1 节"](pt07ch34s02aus116.md) 和 ["顺序耦合热应力分析，" 第 16.1.2 节"](pt04ch16s01at39.md)）。在区分大小写的计算机上，文件或子结构库名称必须具有正确的大小写。无论用户仅指定文件名、相对路径名还是完整路径名，包括路径在内的完整名称最多可包含 256 个字符。除非名称被引号括起来，否则文件名、相对路径名或完整路径名内的所有空格都被忽略，在这种情况下，名称内的所有空格都被保留。

### 数据行

数据行用于提供比作为选项参数更容易以列表形式给出的数据。大多数选项需要一行或多行数据行；如果需要，数据行必须紧跟在引入选项的关键字行后面。输入数据行时适用以下规则：
- 数据行最多可包含 256 个字符，包括空白。尾随空白被忽略。
- 所有数据项必须用逗号（,）分隔。空数据字段通过省略逗号之间的数据来指定。Abaqus 将对任何省略的必需数值数据使用零值，除非指定了默认值。
- 一行只能包含指定数量的项目。
- 行末尾的空数据字段可以忽略。
- 浮点数最多可占用 20 个空格，包括符号、小数点和任何指数表示法。浮点数可以带或不带指数输入。任何指数（如果输入）必须以 E 或 D 开头，后跟可选的（）或（+）。以下行显示输入相同浮点数的四种可接受方式：``` -12.345 -1234.5E-2 -1234.5D-2 -1.2345E1 ```
- 整型数据项最多可占用 9 位数字。
- 字符串最多可包含 80 个字符，不区分大小写。
- 在特定情况下允许延续行（请参阅 ["单元定义，" 第 2.2.1 节"](pt01ch02s02aus11.md)）。如果允许，在前一行末尾用逗号表示此类行。单个数据项不能跨多行输入。

在许多情况下，与选项一起使用的参数选择决定了所需的数据行类型。例如，有五种不同的方法来定义线性弹性材料（["弹性行为：概述，" 第 22.1.1 节"](pt05ch22s01abo19.md)）。您指定的数据行必须与 [*ELASTIC](../key/key-link.md#usb-kws-melastic) 选项上给出的 TYPE 参数值一致。

#### 集合

Abaqus 数据定义方法最有用的功能之一是*集合*的可用性。集合可以是节点集合或单元集合。您为每个集合提供一个名称（1-80 个字符，第一个字符必须是字母）。该名称提供了一种引用集合所有成员的方法。例如，假设对于所示的结构，我们希望对集合 `MIDDLE` 中的所有节点施加对称边界条件，并且边缘 `SUPPORT` 是固定的。

**图 1.2.1–1** 集合使用示例。

![](../graphics/iusing-set-exa.png)

我们将相关节点组装成集合，并通过以下方式指定边界条件
```
[*BOUNDARY](../key/key-link.md#usb-kws-hboundary)
 MIDDLE, ZSYMM
 SUPPORT, PINNED
```
集合是整个 Abaqus 中的基本引用方式，建议使用集合。选择有意义的集合名称可以轻松识别哪些数据属于模型的哪一部分。有关集合的进一步讨论，请参阅 ["节点定义，" 第 2.1.1 节"](pt01ch02s01aus05.md) 和 ["单元定义，" 第 2.2.1 节"](pt01ch02s02aus11.md)。

#### 标签

标签（如集合名称、表面名称和钢筋名称）不区分大小写，除非用引号括起来（除非从用户子程序访问；请参阅 ["用户子程序：概述，" 第 18.1.1 节"](pt04ch18s01aus104.md)）。标签最多可包含 80 个字符。除非用引号括起来，否则标签内的所有空格都被忽略，在这种情况下，标签内的所有空格都被保留。用引号括起来的标签不必以字母开头，可能不包含句点（.），也不应包含逗号和等号等字符。这些限制不适用于用引号括起来的标签，除非标签是材料名称。材料名称必须始终以字母开头，即使名称用引号括起来。

标签不能以双下划线开头和结尾（例如，__STEEL__）。此标签格式保留供 Abaqus 内部使用。

以下是带引号和不带引号输入的标签示例：

```
[*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=SPRINGA, ELSET="One element"
1,1,2
[*SPRING](../key/key-link.md#usb-kws-mspring), ELSET="One element"
1.0E-5,
[*NSET](../key/key-link.md#usb-kws-mnset), ELSET="One element", NSET=NODESET
[*BOUNDARY](../key/key-link.md#usb-kws-hboundary)
nodeset,1,2
```

#### 重复数据行

某些选项只列出一行数据行。在只允许一行数据行的情况下，数据行标题"第一行（也是唯一）"表示这一点。例如 [*DYNAMIC](../key/key-link.md#usb-kws-hdynamic) 选项。在许多情况下，可以重复显示的单个数据行来定义一个变量作为另一个变量的函数；此选择在一行数据后注明。例如，可以给出双轴测试数据表来定义超弹性材料：

许多选项需要多行数据行；这些由数据行标题"第一行："、"第二行："等表示。例如，必须使用恰好两行数据来定义壳单元的局部方向（[*ORIENTATION](../key/key-link.md#usb-kws-morientation)），至少需要三行数据来定义各向异性弹性（[*ELASTIC](../key/key-link.md#usb-kws-melastic)）。

在许多情况下，数据行可以重复，这在一行数据后注明。与重复单个数据行一样，必须按正确顺序给出数据集，以便 Abaqus 能够正确插值数据。

##### 排序数据行

每当一个变量被定义为另一个变量的函数时，必须按适当顺序给出数据，以便 Abaqus 能够正确地为中间值插值。假设在给定的独立变量范围外，所定义的变量保持恒定，但涉及损伤的非线性弹性垫片厚度行为除外，在该行为中，数据基于用户指定的数据计算的最后斜率进行外推。

如果定义的属性只是一个变量的函数（如上面所示的 [*BIAXIAL TEST DATA](../key/key-link.md#usb-kws-mbitestdata)），则应按独立变量递增值的顺序给出数据。

如果定义的属性是多个独立变量的函数，则必须在与第二个变量的固定值（按升序）、然后是第三个变量等的条件下，给出属性随第一个变量变化的关系。数据行必须始终按独立变量递增值的顺序排序。此过程确保属性在属性所依赖的任何独立变量值处被完整且唯一地定义。
