### 概述

Abaqus/CFD中的并行执行：
- 减少需要大量增量的分析的计算时间；
- 减少包含大量节点和元素的分析的计算时间；
- 产生的分析结果与用于分析的处理器数无关；并且
- 可用于使用基于MPI的域分解并行实现的共享内存计算机和计算机集群。

### 调用并行处理

Abaqus/CFD对共享内存和分布式内存计算机使用基于域的显式消息传递并行化。 Abaqus/CFD提供的所有过程及其相关功能完全并行（["并行执行：概述，" 3.5.1节](pt01ch03s05aus32.md)）。通过指定要使用的处理器数来调用并行执行。

| **输入文件用法：** | 在命令行上输入以下内容： |
| --- | --- |
|  | **abaqus** **job**=*job-name* **cpus**=*n* 例如，以下输入将在两个处理器上运行名为"manifold"的作业：``` abaqus job=manifold cpus=2 ``` |

| **Abaqus/CAE用法：** | 作业模块：作业编辑器：**并行化**：切换到**使用多个处理器**，并指定处理器数*n* |
| --- | --- |

### 基于域的并行化

Abaqus/CFD使用域分解消息传递范式实现其并行化。使用基于元素的分解策略，将子域之间通信所需的数量最小化，同时在处理器之间提供几乎均匀的计算工作分配。域数恰好映射到给定计算的指定处理器数。负载平衡程序也是并行实现的，因此您可以避免在计算开始时耗时的串行负载平衡程序。尽一切努力确保Abaqus/CFD为广泛的应用提供可扩展的并行解。所有过程和功能在Abaqus/CFD中以完全并行实现提供。所有输出自动为用户序列化，因此并行域和原始用户输入之间没有转换。此外，这允许Abaqus/CFD无缝重启，无论原始计算使用了多少处理器。

#### 协同仿真

协同仿真技术（["协同仿真：概述，" 17.1.1节](pt04ch17s01abo17.md)）用于在运行时耦合Abaqus/CFD与Abaqus/Standard或Abaqus/Explicit，可与串行或并行运行的Abaqus/CFD一起使用。

#### 重启

可以在重启分析的步骤中包含的功能没有限制。重启分析使用的处理器数不需要与原始分析使用的处理器数相同。

### 输出

输出没有限制。
