**产品：**Abaqus/Standard  Abaqus/Explicit

### 概述

输出数据库报告工具将信息从Abaqus输出数据库（`.odb`）文件打印到格式化报告。默认情况下，报告以纯文本格式打印；但是，您也可以创建HTML和CSV（逗号分隔值）格式的报告。

### 输出数据库结构

每个输出数据库由两个主要部分组成：模型数据和结果数据。数据库进一步分解为容器的层次结构，如[图3.2.20-1](pt01ch03s02abx20.md#odb-reportstructure)所示。

**图3.2.20-1** 输出数据库的结构

![](../graphics/odb-reportstructure-nls.png)

报告中可能出现的数据位于每个分支最右侧的容器中。这些容器可用于对输出数据库的四个主要分支进行分类：- **mesh**分支终止于保存模型节点坐标和元素连接信息的容器。
- **sets**分支终止于保存模型中集合和曲面名称及节点或元素标签的容器。
- **fieldOutputs**分支终止于保存分析现场输出变量值的容器。这些值进一步分解为其向量或张量属性：**invariants**、**components**和**orientation**。
- **historyOutputs**分支终止于保存分析历史输出变量值的容器。

模型数据部分树中的容器是单一容器：每个模型有一个用于网格信息的容器和一个用于集合信息的容器。然而，结果部分树中的容器表示多个容器的聚合。对于多步骤分析，输出数据库将为分析的每个步骤提供一个单独的**step**容器。在每个**step**容器内有多个**frames**和**historyRegions**容器。在每个单独的**frames**容器内有多个**fieldOutputs**容器，依此类推。输出数据库为这些单独容器分配名称或值以帮助区分和识别它们。

有关输出数据库结构的更详细讨论，请参阅["输出数据库的对象模型，" Abaqus脚本用户指南10.5节](../cmd/cmd-link.md#cmd-odb-intro-structure-cpp)。

### 生成摘要报告

如果仅使用必需的文件格式命令行选项生成报告，则报告将是输出数据库的简要摘要。此摘要包含以下信息列表：
- 部件实例名称
- 模型中的节点和元素数量
- 集合和曲面名称
- 步骤和载荷工况名称
- 步骤中的帧数
- 现场和历史输出变量名称

此摘要中的信息可帮助您确定输出数据库中容器的名称和值。

### 向报告添加信息

您可以使用额外的命令行选项创建更全面的报告。大多数这些选项对应于[图3.2.20-1](pt01ch03s02abx20.md#odb-reportstructure)中概述的输出数据库结构中的容器。使用这些选项指定容器的名称或值可指示工具提取该容器中的数据并将其添加到生成的报告中。容器名称和值并不总是唯一的，可能在输出数据库中出现多次。例如，对应于第1帧的容器可能会出现在每个多步骤分析步骤容器中；类似地，保存特定现场输出变量的容器通常出现在步骤的每一帧中。工具会将所有这些容器实例添加到报告中。

要细化容器选择，您可以组合选项。当命令行上从同一分支指示多个容器时，工具仅报告两个容器共有的数据。例如，如果两个选项指定第1步的容器和第3帧的容器，工具将仅把第一步骤第三帧的结果数据添加到报告中。如果指定来自不同分支的容器，则每个容器的数据都会被添加到报告中。例如，如果两个选项指定集合容器和历史区域容器，则集合数据和历史输出数据都会被添加到报告中。

您通过将相关选项设置为该容器的名称或值来识别特定容器。要包含多个相同类型的容器，请将选项设置为逗号分隔的列表。名称区分大小写。如果名称包含空格，必须将整个值放在双引号中（`*container name*`）。

### 附加选项

### 命令行选项

#### 必需选项

执行`abaqus odbreport`时必须至少包含以下选项之一。它们告诉工具在哪里找到输出数据库以及在哪里打印报告。将两个选项一起使用可使报告的文件名与输出数据库名称不同。

**job**

此选项用于指定生成报告的文件名。如果省略此选项，工具会将报告打印到标准输出设备。

**odb**

此选项用于指定从中生成报告的输出数据库（`.odb`）文件。如果省略此选项，工具会在当前目录中查找名为`*job-name*.odb`的输出数据库。

#### 文件格式选项

**mode**

此选项指定生成报告的文件格式。如果省略此选项，报告为纯文本格式，文件扩展名为`.rep`。如果**mode**=`HTML`，报告为HTML格式，文件扩展名为`.htm`。如果**mode**=`CSV`，报告为逗号分隔值格式，文件扩展名为`.csv`。

#### 生成完整输出数据库报告的选项

**all**

此选项用于报告分析中每个步骤的所有可用模型信息和结果信息；每一步的基准状态（零帧）的数据不包含在报告中。对于大型输出数据库，报告会非常长。

#### 报告模型数据的选项

以下选项从输出数据库的模型数据部分提取信息。

**mesh**

此选项用于报告与模型网格关联的节点坐标和元素连接。

**sets**

此选项用于报告与模型关联的所有集合和曲面的名称和内容。

#### 报告结果数据的选项

以下选项从输出数据库的结果数据部分提取信息。

**results**

此选项用于报告输出数据库中的所有现场和历史输出变量值。如果包含与特定结果容器对应的其他选项，则忽略此选项。

**step**

此选项用于报告指定步骤的现场和历史输出变量值。调用此选项时，必须将其设置为至少一个步骤名称。如果**step**=`_LAST_`，报告仅包含分析最后一步的结果。

**steps**容器是输出数据库的**fieldOutputs**和**historyOutputs**分支共用的。如果将**step**选项与现场输出变量选项结合使用，则报告中仅显示现场输出变量数据。类似地，如果将**step**选项与历史输出变量选项结合使用，则报告中仅显示历史输出变量数据。如果将**step**选项与现场和历史输出变量选项结合使用，则两种类型的数据都会显示在报告中。

#### 报告现场输出变量的选项

以下选项从输出数据库**fieldOutputs**分支中的容器提取信息。

**frame**

此选项用于报告指定帧的现场输出变量值。调用此选项时，必须将其设置为至少一个帧号、载荷工况名称或帧描述。初始（或"零增量"）帧只能通过设置**frame**=`0`来标识。如果**frame**=`_LAST_`，报告仅包含每个包含步骤最后帧的结果。

**framevalue**

此选项用于报告指定帧值的现场输出变量值。每个帧可以通过与帧号唯一的帧值来标识。帧值是与帧关联的时间、特征模态号或频率点。

此选项可用作**frame**选项的替代或补充。调用此选项时，必须将其设置为至少一个帧值。您提供的值不需要精确；工具将找到具有最接近帧值的帧。

**field**

此选项用于报告指定的现场输出变量值。如果调用此选项时未设置为任何变量名称，则报告中包含所有现场变量容器。

#### 报告不同现场变量属性的选项

如果未调用以下任何选项，工具会自动为每个现场变量报告分量（以及如果适用，方向）。否则，工具仅报告这些选项指定的属性。不适用于所有现场变量的不变量和方向。

**components**

此选项用于报告所有现场输出变量的分量。

**invariants**

此选项用于报告所有现场输出变量的不变量值。

**orientation**

此选项用于报告每个现场输出变量的局部坐标系。

#### 报告历史输出变量的选项

以下选项从输出数据库**historyOutputs**分支中的容器提取信息。

**histregion**

此选项用于报告指定历史区域的历史输出变量值。调用此选项时，必须将其设置为至少一个历史区域名称。

**history**

此选项用于报告指定的历史输出变量值。如果调用此选项时未设置为任何变量名称，则报告中包含所有历史变量容器。

#### 附加选项

以下选项为报告增加了额外的控制级别和详细信息。它们不直接与输出数据库结构关联，也不会向报告添加任何数据库信息。它们必须与前面描述的选项结合使用。

**instance**

此选项用于将报告的模型和结果数据限制为模型中的特定部件或装配实例。它不直接与任何输出数据库容器关联，也不会向报告添加任何数据。

调用此选项时，必须将其设置为至少一个实例名称。如果**instance**=`_NONE_`，报告包含整个装配和模型的数据。

**blocked**

此选项用于根据部件实例、元素类型和截面点将现场输出变量表细分为块。如果您有兴趣分离大型模型不同区域的输出，这将非常有用。默认情况下，表按变量名称和帧组织。

此选项指示报告工具使用现场批量数据API访问输出数据库。有关字段批量数据API如何操作的详细信息，请参阅["使用批量数据访问输出数据库，" Abaqus脚本用户指南10.10.7节](../cmd/cmd-link.md#cmd-odb-intro-bulkdata-cpp)。此选项的额外好处是在处理大量现场变量时增强了工具的性能，从而加快了报告生成速度。如果报告中没有现场输出变量，或者还指定了**invariants**选项，则此选项无效。

**extrema**

此选项用于在每个节点坐标和现场输出变量表末尾报告最大值和最小值。默认情况下，这些极值不会出现在报告中。如果报告中没有节点坐标或现场输出变量，则此选项无效。

### 示例

以下示例说明了**odbreport**执行程序的功能和不同选项组合的效果。

#### 文件命名和格式化

以下命令生成输出数据库`beam.odb`的简要摘要，格式为名为`beam.rep`的纯文本文件：

```
abaqus odbreport job=beam
```
要创建相同格式的报告并命名为`beamreport.htm`，请执行以下命令：
```
abaqus odbreport job=beamreport odb=beam mode=html
```

#### 向报告添加信息

使用额外的命令行选项将指定容器的数据添加到报告中。以下命令创建一个报告，列出模型中节点坐标和元素连接以及与名为**Apply weight**的步骤关联的所有输出变量值：

```
abaqus odbreport job=beam mesh step="Apply weight"
```
您可以使用选项组合来细化列出的结果数据。在以下示例中，工具仅报告在**Apply weight**步骤中从名为**Node350**的历史区域输出的历史输出变量值：
```
abaqus odbreport job=beam step="Apply weight" 
   histregion=Node350
```
如果容器由不唯一的名称或值标识，生成的报告将包含该容器的所有出现。以下命令创建一个报告，列出在每个单独步骤的第三帧中输出的现场变量RF的值：
```
abaqus odbreport job=beam frame=3 field=RF
```
要报告RF的大小而不是其分量，请使用**invariants**选项：
```
abaqus odbreport job=beam frame=3 field=RF invariants
```
要将多个相同类型的容器添加到报告，您可以设置逗号分隔列表的选项。以下命令报告在**Apply weight**和**Side load**步骤期间输出的现场输出变量U和S的所有值：
```
abaqus odbreport job=beam step="Apply weight","Side load"
   field=U,S
```

#### 附加选项

使用**instance**选项将报告的信息限制在模型的特定部分。以下命令报告`motor.odb`数据库中每个步骤最后帧的集合名称和节点，以及S的值。但是，只有与部件实例**pistonA**相关的信息会出现在报告中：

```
abaqus odbreport job=motor sets frame=_LAST_ field=S 
   instance=pistonA
```

#### 选择帧

**frame**和**framevalue**选项可以接受多种值类型，使它们成为强大的报告构建选项。由于这种多样性，有时需要同时调用两个选项来指定特定帧。例如，考虑输出数据库`plate.odb`，它是稳态动态分析的结果。分析研究了板在20个不同频率下对三种不同载荷工况的响应。因此，输出数据库在每个频率下包含三种不同载荷工况的结果。您对45 Hz下名为**lc2**的载荷工况的响应感兴趣。设置**frame**=`lc2`将报告每个频率（共20帧）的载荷工况**lc2**的现场变量。设置**framevalue**=`45`将报告与45 Hz频率关联的每个载荷工况的现场变量（共3帧）。要将报告限制为感兴趣的单个帧，必须同时调用两个选项：

```
abaqus odbreport job=plate frame=lc2 framevalue=45
```
