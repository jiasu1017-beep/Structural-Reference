# 19.8.8 控制作业并行执行







要找到作业编辑器，请从主菜单栏中选择****作业**![](../graphics/images/arrow.gif)**编辑**![](../graphics/images/arrow.gif)***作业名称*****。使用**并行化**选项卡页面控制 Abaqus 分析作业的并行执行，如下所示：
- 如果并行处理可用，请选择要用于分析的处理器数量。
- 对于 Abaqus/Standard 分析，您可以选择使用 GPGPU 加速直接稀疏求解器并指定 GPGPU 数量。有关更多信息，请参见["Abaqus/Standard 中的并行执行，"《Abaqus 分析用户手册》第 3.5.2 节](../usb/usb-link.md#usb-int-astdparallel)。
- 对于 Abaqus/Explicit 分析，您可以选择以下内容：- 域数量。如果您选择域级并行化，则域数量必须等于处理器数量或为处理器数量的倍数。当域数量是处理器数量的倍数时，您可以根据需要切换**激活动态负载平衡**。此选项可以提高分析作业的性能。例如，如果您有 4 个处理器并选择 8、12、16 等域，则动态负载平衡功能变为可用。动态负载平衡默认关闭。此选项等同于 Abaqus/Explicit 执行过程中的 `dynamic_load_balancing` 选项。有关更多信息，请参见["Abaqus/Standard、Abaqus/Explicit 和 Abaqus/CFD 执行，"《Abaqus 分析用户手册》第 3.2.2 节](../usb/usb-link.md#usb-int-danalysisproc)。动态负载平衡最有可能在具有强时间依赖性和/或空间变化的计算负载的应用中提高计算速度和效率。大多数不平衡问题在域数量为处理器数量的 2 到 4 倍时会看到最佳性能提升。有关更多详细信息，请参见["Abaqus/Explicit 中的并行执行"中的"域级并行化"，《Abaqus 分析用户手册》第 3.5.3 节](../usb/usb-link.md#usb-int-aparallelexecution-domain)。- 并行化方法是**域**（默认）还是**循环**。域级方法将模型拆分为主干拓扑域，并将其均匀分配给可用处理器。循环级方法并行化代码中负责大部分计算成本的低级循环。如果您为并行化方法选择**循环**，则必须为多处理模式选择**默认**。有关更多信息，请参见["Abaqus/Explicit 中的并行执行，"第 3.5.3 节](../usb/usb-link.md#usb-int-aparallelexecution)。
- 对于 Abaqus/Standard 和 Abaqus/Explicit 分析，您都可以选择多处理模式是**默认**、**线程**还是 **MPI**（消息传递接口）。默认多处理模式取决于分析产品执行所在的平台。**注意：**您使用"步骤"模块在 Abaqus/Standard 迭代和直接稀疏求解器之间进行选择。有关更多信息，请参见["使用步骤编辑器，"第 14.10 节](pt03ch14s10.md)。
