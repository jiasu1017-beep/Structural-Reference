### 概述

Abaqus/Standard 中的干扰配合：
- 默认情况下，当接触 formulation 计算初始配置中表面之间的过盈量时发生；
- 默认情况下在步骤的第一个增量中消除；
- 可以在多个增量中逐渐消除；
- 在消除过盈量时会在模型中产生应力和应变；
- 可以为基于表面的接触对和接触单元指定；以及
- 不能为自接触指定。

Abaqus/Standard 提供了替代方法，通过应变自由调整来消除初始过盈量，并对与初始配置计算不同的特定过盈量或间隙进行建模。这些方法在 ["在 Abaqus/Standard 接触对中调整初始表面位置和指定初始间隙，" 第 36.3.5 节](pt09ch36s03aus149.md) 中讨论。

### 解决过度的初始过盈量

如果模型的初始配置中存在大的过盈量，Abaqus/Standard 可能无法在单个增量中消除干扰配合。Abaqus/Standard 提供了允许在多个增量中逐渐消除过盈量的替代方法。

如果模型中的穿透为 ![](../graphics/usb_eqn06186.gif)，您可以声明 ![](../graphics/usb_eqn06187.gif) 或请求自动热配合。在任一情况下，Abaqus/Standard 将认为两个物体在模拟开始时刚好接触。随着容许干扰 ![](../graphics/usb_eqn06183.gif) 在步骤中降低到零，Abaqus/Standard 将表面推开，直到不再有容许穿透。

有三种不同的方式来指定容许干扰 ![](../graphics/usb_eqn06183.gif)。默认情况下，在所有情况下，指定容许干扰的值在步骤开始时立即应用，然后在线性地在步骤中斜降到零，除非您指定定义特定容许干扰-时间变化的振幅参考。建议您在单独于分析其余部分的步骤中指定容许干扰；额外的载荷可能会对干扰配合的消除和具有部分消除干扰的载荷响应产生不利影响，可能导致非物理结果。一旦消除了过盈量，您可以在新步骤中继续分析。

当指定接触干扰时，输出变量 COPEN 在步骤期间不反映实际过盈量值；它仅在步骤结束时反映实际值。

您必须指定应适用容许干扰的接触对或接触单元。

#### 使用非默认振幅曲线用于容许干扰

您可以通过创建振幅曲线（详情见 ["振幅曲线，" 第 34.1.2 节](pt07ch34s01aus115.md)）然后从此接触干扰定义中引用此曲线来定义随时间变化的容许接触干扰。但是，如果使用 Riks 方法（见 ["不稳定性坍塌和后屈曲分析，" 第 6.2.4 节](pt03ch06s02at03.md)），振幅将被忽略。

| **输入文件用法：** | ``` [*CONTACT INTERFERENCE](../key/key-link.md#usb-kws-hcontactinterfer), AMPLITUDE=*amplitude_curve_name* ``` |

| **Abaqus/CAE 用法：** | Interaction 模块： interaction editor: **Interference Fit**: **Gradually remove slave node overclosure during the step**, **Uniform allowable interference**, **Amplitude:** *amplitude_curve_name* |
| --- | --- |

#### 移除或修改容许接触干扰

默认情况下，只有特定接触干扰定义定义或重新定义的容许接触干扰会被修改。或者，您可以指定所有先前定义的容许接触干扰都应从模型中移除，并且只有此定义中的那些才会保留。

| **输入文件用法：** | 使用以下选项添加或修改容许接触干扰定义： |
| --- | --- |
|  | ``` [*CONTACT INTERFERENCE](../key/key-link.md#usb-kws-hcontactinterfer), OP=MOD ``` 使用以下选项移除所有先前定义的容许接触干扰： ``` [*CONTACT INTERFERENCE](../key/key-link.md#usb-kws-hcontactinterfer), OP=NEW ``` |

| **Abaqus/CAE 用法：** | Abaqus/CAE 中的接触干扰与为之定义的相互作用一起传播。您不能在 Abaqus/CAE 中一次移除所有先前定义的接触干扰。 |
| --- | --- |

#### 为整个表面指定相同的容许接触干扰

可以为从表面上的每个节点或接触单元指定集的每个从节点指定单个容许干扰 ![](../graphics/usb_eqn06183.gif)。各种接触单元族的从节点概念在其各自章节中讨论。指定的容许接触干扰包含在请求详细接触打印时在消息文件中报告的从节点当前穿透中。因此，侵入主表面的距离小于容许干扰的任何从节点将报告为开放。

#### 使用自动"热配合"方法

此方法仅在分析的第一步期间适用，不需要干扰值。使用此方法，Abaqus/Standard 为每个从节点分配一个不同的 ![](../graphics/usb_eqn06183.gif)，等于该节点的初始穿透（或如果点初始开放则为零），除了对于有限滑动、面-面 formulation，在这种情况下，相同的 ![](../graphics/usb_eqn06183.gif) 值（对应于接触对的最大穿透）被分配给所有初始闭合的约束。这些自动计算的容许接触干扰不包含在请求详细接触打印时在消息文件中报告的当前穿透中。

当使用自动"热配合"方法时，只能使用默认振幅曲线（线性斜降到零量级）。

| **输入文件用法：** | ``` [*CONTACT INTERFERENCE](../key/key-link.md#usb-kws-hcontactinterfer), SHRINK ``` |

| **Abaqus/CAE 用法：** | Interaction 模块： interaction editor: **Interference Fit**: **Gradually remove slave node overclosure during the step**, **Automatic shrink fit** |
| --- | --- |

#### 使用偏移向量施加容许接触干扰

在此方法中，您指定统一的容许干扰 ![](../graphics/usb_eqn06183.gif) 和方向 ![](../graphics/usb_eqn00842.gif)。容许干扰值 ![](../graphics/usb_eqn06183.gif) 定义偏移向量的大小。在 Abaqus/Standard 确定接触条件之前，相对偏移 ![](../graphics/usb_eqn06188.gif) 被应用于从节点。在某些应用中，例如螺纹连接器的接触模拟，沿指定方向偏移表面比简单地允许干扰更有效。

[图 36.3.4-2](pt09ch36s03aus148.md#acontact-dir-def) 说明了使用带偏移向量的容许接触干扰与使用统一容许接触干扰可能产生的潜在差异。在情况（a）中，除了容许干扰 ![](../graphics/usb_eqn06183.gif) 外，还定义了偏移方向 ![](../graphics/usb_eqn00842.gif)，而在情况（b）中，使用标准方法，容许干扰 ![](../graphics/usb_eqn06183.gif)。

**图 36.3.4-2** 方向定义对干扰容纳的影响：a) 有方向，b) 无方向。

![](../graphics/acontact-dir-def.png)

在两种情况中 ![](../graphics/usb_eqn06183.gif) 的大小相同，但在情况（a）中它小于穿透，在情况（b）中它大于穿透。在情况（a）中，从节点 A 立即检测到接触，穿透沿段 ![](../graphics/usb_eqn06189.gif) 消除，因为节点 A 在 Abaqus/Standard 检查接触之前沿方向 ![](../graphics/usb_eqn00842.gif) 偏移。偏移后，Abaqus/Standard 确定节点 A 最接近段 ![](../graphics/usb_eqn06189.gif)，并将节点移动到该段上。在情况（b）中，从节点 A 检测到与段 ![](../graphics/usb_eqn06190.gif) 的接触，因为当节点 A 保持在其初始位置时该段是最接近的段。因此，如果未提供偏移方向，节点 A 将沿段 ![](../graphics/usb_eqn06190.gif) 滑动。

### 面-面离散化的干扰配合

因为接触条件在每个约束位置周围区域以平均值 enforced，对于面-面接触，当面-面约束处于零穿透状态时，在从节点处可能会观察到穿透或间隙。

大的干扰可能难以使用有限滑动、面-面 formulation 消除。使用此 formulation，过盈量倾向于沿从 facet 法线方向消除；而使用节点-面接触，过盈量倾向于沿主表面法线方向消除。[图 36.3.4-3](pt09ch36s03aus148.md#rnb-chpinter-interffitcompare) 说明了在不同法线方向导致干扰配合期间不希望的切向运动的情况。在某些情况下，可能更好使用节点-面离散化来消除大的初始过盈量。

### 摩擦和接触干扰

通常，实际装配过程被建模为干扰配合问题。如果需要摩擦界面属性，它们通常应在初始干扰消除后引入。初始干扰问题应在无摩擦条件下建模，因为物理装配过程通常不是精确建模的。摩擦可以在后续步骤中引入（见 ["在 Abaqus/Standard 分析期间改变摩擦属性" 在 "摩擦行为，" 第 37.1.5 节](pt09ch37s01aus169.md#usb-cni-afriction-change-std)。
