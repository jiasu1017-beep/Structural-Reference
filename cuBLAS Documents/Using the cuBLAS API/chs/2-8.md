## 2.8. 类BLAS扩展函数

本节描述执行矩阵-矩阵运算的BLAS扩展函数。

### 2.8.1. cublas<t>geam()

[见上方代码块]

此函数支持64位整数接口。

此函数执行矩阵-矩阵加法/转置运算：

$C = \alpha\text{op}(A) + \beta\text{op}(B)$

其中 $\alpha$ 和 $\beta$ 是标量，$A$、$B$ 和 $C$ 是以列优先格式存储的矩阵，维度分别为 $\text{op}(A)$ $m \times n$、$\text{op}(B)$ $m \times n$ 和 $C$ $m \times n$。对于矩阵 $A$：

$\text{op}(A) = \left\{ \begin{matrix}
A & {\text{if }\textsf{transa == CUBLAS\_OP\_N}} \\
A^{T} & {\text{if }\textsf{transa == CUBLAS\_OP\_T}} \\
A^{H} & {\text{if }\textsf{transa == CUBLAS\_OP\_C}} \\
\end{matrix} \right.$

$\text{op}(B)$ 对于矩阵 $B$ 的定义类似。

如果 C 不与 A 或 B 重叠，则该操作是原地（out-of-place）操作。

原地模式支持以下两种操作：

$C = \alpha\text{*}C + \beta\text{op}(B)$

$C = \alpha\text{op}(A) + \beta\text{*}C$

对于原地模式，如果 `C == A`，则 `ldc == lda` 且 `transa == CUBLAS_OP_N`。如果 `C === B`，则 `ldc == ldb` 且 `transb == CUBLAS_OP_N`。如果用户不满足上述要求，则返回 `CUBLAS_STATUS_INVALID_VALUE`。

该操作包括以下特殊情况：

用户可以通过设置 `*alpha = beta = 0` 来将矩阵 C 重置为零。

用户可以通过设置 `*alpha = 1 and *beta = 0` 来转置矩阵 A。

| --- | --- | --- | --- |
| handle |  | input | cuBLAS 库上下文的句柄。 |
| transa |  | input | op(A) 操作，为非转置或（共轭）转置。 |
| transb |  | input | op(B) 操作，为非转置或（共轭）转置。 |
| m |  | input | 矩阵 op(A) 和 C 的行数。 |
| n |  | input | 矩阵 op(B) 和 C 的列数。 |
| alpha | 主机或设备 | input | 用于乘法的 <type> 标量。如果 *alpha == 0，则 A 不必是有效输入。 |
| A | 设备 | input | <type> 数组，维度为 lda x n（如果 transa == CUBLAS_OP_N，则 lda >= max(1, m)，否则为 lda x m，lda >= max(1, n)）。 |
| lda |  | input | 用于存储矩阵 A 的二维数组的前导维度。 |
| B | 设备 | input | <type> 数组，维度为 ldb x n（如果 transb == CUBLAS_OP_N，则 ldb >= max(1, m)，否则为 ldb x m，ldb >= max(1,n)）。 |

| ldc |  | input | 用于存储矩阵 C 的二维数组的前导维度。 |

### 2.8.2. cublas<t>dgmm()

[见上方代码块]

此函数支持64位整数接口。

此函数执行矩阵-矩阵乘法：

$C = \left\{ \begin{matrix}
{A \times diag(X)} & {\text{if }\textsf{mode == CUBLAS\_SIDE\_RIGHT}} \\
{diag(X) \times A} & {\text{if }\textsf{mode == CUBLAS\_SIDE\_LEFT}} \\
\end{matrix} \right.$

其中 $A$ 和 $C$ 是以列优先格式存储的矩阵，维度为 $m \times n$。$X$ 是大小为 $n$ 的向量（如果 `mode == CUBLAS_SIDE_RIGHT`）或大小为 $m$ 的向量（如果 `mode == CUBLAS_SIDE_LEFT`）。$X$ 是从一维数组 x 以步长 `incx` 收集得到的。`incx` 的绝对值是步长，`incx` 的符号是步长的方向。如果 `incx` 为正，则从前向后遍历 x。否则，从后向前遍历 x。X 的公式为：

$X\lbrack j\rbrack = \left\{ \begin{matrix}
{x\lbrack j \times incx\rbrack} & {\text{if }incx \geq 0} \\
{x\lbrack(\chi - 1) \times |incx| - j \times |incx|\rbrack} & {\text{if }incx < 0} \\
\end{matrix} \right.$

其中，如果 `mode == CUBLAS_SIDE_LEFT` 则 $\chi = m$，如果 `mode == CUBLAS_SIDE_RIGHT` 则 $\chi = n$。

示例 1：如果用户想执行 $diag(diag(B)) \times A$，则 $incx = ldb + 1$，其中 $ldb$ 是矩阵 `B` 的前导维度，可以是行主序或列主序。

示例 2：如果用户想执行 $\alpha \times A$，有两种选择：使用 `*beta == 0` 和 `transa == CUBLAS_OP_N` 的 cublas<t>geam()，或使用 `incx == 0` 和 `x[0] == alpha` 的 cublas<t>dgmm()。

该操作是原地（out-of-place）操作。原地操作仅在 `lda == ldc` 时有效。

| --- | --- | --- | --- |
| handle |  | input | cuBLAS 库上下文的句柄。 |
| mode |  | input | 如果 mode == CUBLAS_SIDE_LEFT 则左乘，如果 mode == CUBLAS_SIDE_RIGHT 则右乘 |
| m |  | input | 矩阵 A 和 C 的行数。 |
| n |  | input | 矩阵 A 和 C 的列数。 |
| A | 设备 | input | <type> 数组，维度为 lda x n，lda >= max(1, m) |
| lda |  | input | 用于存储矩阵 A 的二维数组的前导维度。 |
| x | 设备 | input | 一维 <type> 数组，如果 mode == CUBLAS_SIDE_LEFT 则大小为 abs(incx) x m，如果 mode == CUBLAS_SIDE_RIGHT 则大小为 abs(incx) x n |
| incx |  | input | 一维数组 x 的步长。 |
| C | 设备 | in/out | <type> 数组，维度为 ldc x n，ldc >= max(1, m)。 |
| ldc |  | input | 用于存储矩阵 C 的二维数组的前导维度。 |

此函数可能返回的错误值及其含义如下表所示。

| 错误值 | 含义 |
| --- | --- |
| CUBLAS_STATUS_SUCCESS | 操作成功完成 |
| CUBLAS_STATUS_NOT_INITIALIZED | 库未初始化 |
| CUBLAS_STATUS_INVALID_VALUE | 如果 m < 0 或 n < 0，或
如果 mode 不是 CUBLAS_SIDE_LEFT 和 CUBLAS_SIDE_RIGHT 之一，或
如果 lda < max(1, m)，或
如果 ldc < max(1, m) |
| CUBLAS_STATUS_EXECUTION_FAILED | 函数未能在 GPU 上启动 |

### 2.8.4. cublas<t>getrsBatched()

### 2.8.7. cublas<t>geqrfBatched()

### 2.8.11. cublas<t>gemmEx()

### 2.8.13. cublasGemmBatchedEx()

[见上方代码块]

此函数支持64位整数接口。

此函数是 cublas<t>gemmBatched() 的扩展，执行一批矩阵的矩阵乘法，并允许用户分别为 A、B 和 C 矩阵数组指定数据类型、计算精度和要运行的 GEMM 算法。与 cublas<t>gemmBatched() 一样，批次被认为是"统一"的，即所有实例对于各自的 A、B 和 C 矩阵具有相同的维度（m、n、k）、前导维度（lda、ldb、ldc）和转置（transa、transb）。批次中每个实例的输入矩阵和输出矩阵的地址从调用者传递给函数的指针数组中读取。支持参数组合列在本节的后面部分。

> **注意**
>
> 注意
> 提供 cublasGemmBatchedEx() 函数的第二个变体是为了与 C++ 应用程序代码向后兼容，其中 computeType 参数的类型为 cudaDataType 而不是 cublasComputeType_t。C 应用程序仍可使用更新后的函数签名进行编译。

$C\lbrack i\rbrack = \alpha\text{op}(A\lbrack i\rbrack)\text{op}(B\lbrack i\rbrack) + \beta C\lbrack i\rbrack,\text{ for i } \in \lbrack 0,batchCount - 1\rbrack$

其中 $\alpha$ 和 $\beta$ 是标量，$A$、$B$ 和 $C$ 是指向以列优先格式存储的矩阵的指针数组，维度分别为 $\text{op}(A\lbrack i\rbrack)$ $m \times k$、$\text{op}(B\lbrack i\rbrack)$ $k \times n$ 和 $C\lbrack i\rbrack$ $m \times n$。对于矩阵 $A$：

$\text{op}(A) = \left\{ \begin{matrix}
A & {\text{if }\textsf{transa == CUBLAS\_OP\_N}} \\
A^{T} & {\text{if }\textsf{transa == CUBLAS\_OP\_T}} \\
A^{H} & {\text{if }\textsf{transa == CUBLAS\_OP\_C}} \\
\end{matrix} \right.$

$\text{op}(B\lbrack i\rbrack)$ 对于矩阵 $B\lbrack i\rbrack$ 的定义类似。

> **注意**
>
> 注意
> \(C\lbrack i\rbrack\) 矩阵不得重叠，即各个 gemm 操作必须能够独立计算；否则，行为未定义。

在某些问题规模上，在不同的 CUDA 流中多次调用 cublas<t>gemm() 可能比使用此 API 更有优势。

### 2.8.14. cublasGemmStridedBatchedEx()

[见上方代码块]

此函数支持64位整数接口。

此函数是 cublas<t>gemmStridedBatched() 的扩展，执行一批矩阵的矩阵乘法，并允许用户分别为 A、B 和 C 矩阵指定数据类型、计算精度和要运行的 GEMM 算法。与 cublas<t>gemmStridedBatched() 一样，批次被认为是"统一"的，即所有实例对于各自的 A、B 和 C 矩阵具有相同的维度（m、n、k）、前导维度（lda、ldb、ldc）和转置（transa、transb）。批次中每个实例的输入矩阵 A、B 和输出矩阵 C 位于相对于前一实例中位置的固定元素偏移处。用户传递指向第一个实例的 A、B 和 C 矩阵的指针以及元素偏移量（以元素数计）——strideA、strideB 和 strideC，这些偏移量决定后续实例中输入和输出矩阵的位置。

> **注意**
>
> 注意

$C + i*{strideC} = \alpha\text{op}(A + i*{strideA})\text{op}(B + i*{strideB}) + \beta(C + i*{strideC}),\text{ for i } \in \lbrack 0,batchCount - 1\rbrack$

$\text{op}(A) = \left\{ \begin{matrix}
A & {\text{if }\textsf{transa == CUBLAS\_OP\_N}} \\
A^{T} & {\text{if }\textsf{transa == CUBLAS\_OP\_T}} \\
A^{H} & {\text{if }\textsf{transa == CUBLAS\_OP\_C}} \\
\end{matrix} \right.$

$\text{op}(B\lbrack i\rbrack)$ 对于矩阵 $B\lbrack i\rbrack$ 的定义类似。

A[\text{idx}] & {\text{if }\textsf{$\mathrm{transa\_array}\lbrack i\rbrack$ == CUBLAS\_OP\_N}} \\
A[\text{idx}]^{T} & {\text{if }\textsf{$\mathrm{transa\_array}\lbrack i\rbrack$ == CUBLAS\_OP\_T}} \\
A[\text{idx}]^{H} & {\text{if }\textsf{$\mathrm{transa\_array}\lbrack i\rbrack$ == CUBLAS\_OP\_C}} \\
\end{matrix} \right.$

对于组 $i$ 中的矩阵 $B[\text{idx}]$，$\text{op}(B[\text{idx}])$ 的定义类似。

> **注意**
>
> 注意
> \(C\lbrack\text{idx}\rbrack\) 矩阵不得重叠，即各个 gemm 操作必须能够独立计算；否则，将出现未定义行为。

在某些问题规模上，在不同的 CUDA 流中多次调用 cublasGemmBatchedEx() 可能比使用此 API 更有优势。
