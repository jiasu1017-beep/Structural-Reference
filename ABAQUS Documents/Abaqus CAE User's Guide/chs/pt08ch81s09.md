# 81.9 GUI 插件是如何执行的？





Abaqus/CAE 通过向为该插件在注册命令中指定的 GUI 对象发送消息来执行 GUI 插件。与内核插件一样，第一次调用 GUI 插件时，Abaqus/CAE 会用该插件的目录更新内核的 *sys.path* 列表。此外，Abaqus/CAE 还会将插件的 *kernelInitString* 发送到内核，并更新 GUI 的 *sys.path* 列表以包含该插件的目录。您可以使用 *kernelInitString* 来初始化内核，以执行插件 GUI 将发送的命令。下次调用该插件时，只会将消息发送到 GUI 对象。

如果需要将不在插件目录中的文件导入 GUI，必须按前面示例所示扩充 *sys.path* 列表。此外，如果需要扩充内核的 *sys.path* 列表，应在插件的 *kernelInitString* 中提供类似以下代码（在此示例中，包含插件的文件名为 `myForm_plugin.py`）：

```
from abaqusGui import AFXForm, getAFXApp

class MyForm(AFXForm):

    *form code goes here*

import os

# Full path (with name) to this file
absPath = os.path.abspath(__file__)  

# Full directory specification
absDir = os.path.dirname(absPath)    

# Full subdirectory specification
subDir = os.path.join(absDir, 'mySubDir') 
initString  = "sys.path.append('%s')\n" % subDir

# myModule is located in subDir
initString += 'import myModule'  
toolset = getAFXApp().getAFXMainWindow().getPluginToolset()
toolset.registerGuiMenuButton(buttonText='My Utility', 
    object=MyForm(toolset), kernelInitString=initString )
```
