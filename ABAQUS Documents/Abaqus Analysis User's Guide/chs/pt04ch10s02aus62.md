# 12.4.3 Surface-based 子模型

**产品：** Abaqus/Standard 和 Abaqus/Explicit

##### **参考资料**

- ["子模型技术概述，" 第 12.4.1 节](pt04ch10s02aus60.md)
- ["节点-based 子模型，" 第 12.4.2 节](pt04ch10s02aus61.md)
- [*SUBMODEL](../key/key-link.md#usb-kws-hsubmodel)

### 概述

Surface-based 子模型允许您从全局模型传递面到面的位移约束到子模型。这种方法特别适用于：
- 接触问题
- 复杂几何过渡区域
- 需要保持表面协调性的情况

### 定义 Surface-based 子模型

| **输入文件用法：** | ``` [*SUBMODEL](../key/key-link.md#usb-kws-hsubmodel), STEP=*step*, INC=*increment*, SURFACE ``` |

### 界面定义

子模型需要定义与全局模型对应的表面：

| **输入文件用法：** | ``` [*SURFACE](../key/key-link.md#usb-kws-msurface), NAME=*name* *surface_definition* ``` |

### 连接到全局模型

子模型表面通过以下方式连接到全局模型：

| **输入文件用法：** | ``` [*SUBMODEL](../key/key-link.md#usb-kws-hsubmodel), STEP=*step*, INC=*increment*, SURFACE, GLOBAL SURFACE=*global_surface* ``` |

### 约束方法

Surface-based 子模型使用以下约束方法：

| **方法** | **描述** |
| --- | --- |
| Tiie | 绑定连接 |
| MPC | 多点约束 |
| Kinematic | 运动约束 |

| **输入文件用法：** | ``` [*SUBMODEL](../key/key-link.md#usb-kws-hsubmodel), SURFACE, CONSTRAINT=TIED ``` |

### 驱动区域

驱动区域定义从全局模型获取位移的子模型表面区域：

| **输入文件用法：** | ``` [*SUBMODEL](../key/key-link.md#usb-kws-hsubmodel), DRIVING REGION, SURFACE=*surface* ``` |

### 数据传递

从全局模型传递到子模型的数据：

| **数据类型** | **描述** |
| --- | --- |
| 表面位移 | 整个表面的位移场 |
| 应力状态 | 表面应力分布 |
| 接触状态 | 接触压力和滑移 |

### 应用场景

#### 接触分析

当子模型区域包含与全局模型其他部分的接触时：

1. 定义接触对
2. 从全局模型获取接触面的位移
3. 应用到子模型边界

#### 过渡区域

在粗细网格交界处使用子模型：

1. 全局模型使用较粗网格
2. 子模型在过渡区域使用细网格
3. 通过表面连接保持协调

### 输出

Surface-based 子模型输出包括：

| **输出类型** | **描述** |
| --- | --- |
| 表面位移 | 整个表面的位移分布 |
| 应力 | 表面和内部的应力 |
| 接触变量 | 接触压力和滑移 |
| 反力 | 界面约束反力 |

### 验证

验证表面-based 子模型：

| **检查** | **描述** |
| --- | --- |
| 界面位移 | 检查与全局模型的连续性 |
| 应力过渡 | 确认应力平滑传递 |
| 约束力 | 验证界面力平衡 |

### 局限性

| **局限性** | **描述** |
| --- | --- |
| 表面匹配 | 全局和子模型表面必须对应 |
| 单元兼容性 | 单元类型必须兼容 |
| 约束类型 | 只能使用绑定或运动约束 |
| 非线性 | 非线性响应可能需要迭代 |

### 输入文件模板

```
*-heading
Global model
**
*STEP
*STATIC
*BOUNDARY
*LOAD
*OUTPUT
*SURFACE FILE
*END STEP
**
*-heading
Surface-based submodel
**
*SUBMODEL, STEP=1, INC=50, SURFACE,
GLOBAL STEP=1, GLOBAL INC=50
*SURFACE, NAME=SUBSURF
*SUBMODEL, DRIVING REGION, SURFACE=SUBSURF,
GLOBAL SURFACE=GLOBSURF, CONSTRAINT=TIED
*STEP
*STATIC
*SURFACE INTERACTION
*CONTACT PAIR
SUBSURF, GLOBSURF
*BOUNDARY
SUBSURF, ENCASTRE
*LOAD
*OUTPUT
*ELEMENT OUTPUT
*END STEP
```
