# 6.4 示例：货运吊车





[图 6-11](ch06s04.md#gss-cargo-static) 显示了一台轻载货运吊车。您需要确定吊车在承载 10 kN 载荷时的静态挠度。您还应识别结构中的关键构件和节点：即应力和载荷最高的部位。因为这是静力分析，所以您将使用 Abaqus/Standard 分析货运吊车。

**图 6-11** 轻载货运吊车草图。

![](../graphics/gss-cargo.png)

吊车由两个通过交叉支撑连接在一起的桁架结构组成。每个桁架结构中的两个主要构件是钢箱形梁（箱形横截面）。每个桁架结构通过内部支撑进行加强，内部支撑焊接在主要构件上。连接两个桁架结构的交叉支撑用螺栓固定在桁架结构上。这些连接几乎不能传递力矩，因此被视为铰接节点。内部支撑和交叉支撑都使用钢箱形梁，其横截面比桁架结构的主要构件小。两个桁架结构在其端部（在 E 点）以允许在 3 方向上独立移动并所有旋转的方式连接，同时约束 1 和 2 方向的位移相同。吊车在 A、B、C 和 D 点牢固地焊接在大型结构上。吊车的尺寸如图 6-12 所示。在以下图中，桁架 A 是由构件 AE、BE 及其内部支撑组成的结构；桁架 B 由构件 CE、DE 及其内部支撑组成。

**图 6-12** 货运吊车的尺寸（单位：m）。

![](../graphics/gss-cargo-crane-dim-nls.png)

吊车主要构件中典型横截面尺寸与全局轴向长度的比值远小于 1/15。用于内部支撑的最短构件中的比值约为 1/15。因此，使用梁单元对吊车进行建模是有效的。

### 6.4.1 坐标系

您应使用图 6-11 和图 6-12 中所示的默认全局直角笛卡尔坐标系。将坐标原点放置在 A 点和 D 点的中间。如果您使用不同的原点或坐标系方向构建模型，请确保模型中的输入数据反映您的坐标系，而不是此处所示的坐标系。

### 6.4.2 网格设计

货运吊车将用三维细长三次梁单元（B33）进行建模。这些单元中的三次插值允许我们对每个构件使用单个单元，并在施加弯曲载荷时获得准确结果。您在模拟中使用的网格如图 6-13 所示。

**图 6-13** 货运吊车的网格。

![](../graphics/gss-cargo-crane-v.png)

吊车中的焊接接头提供了从一个单元到下一个单元的平移和旋转的完全连续性。因此，您只需要在模型中每个焊接接头处设置一个节点。连接交叉支撑到桁架结构的螺栓接头以及桁架结构尖端的连接是不同的。由于这些接头不能为所有节点自由度提供完全连续性，因此需要在连接处为每个单元设置独立的节点。然后，必须通过使用 [*MPC](../key/key-link.md#usb-kws-mmpc)、[*BOUNDARY](../key/key-link.md#usb-kws-hboundary) 或 [*EQUATION](../key/key-link.md#usb-kws-mequation) 选项在这些独立节点之间给出适当的约束。[*MPC](../key/key-link.md#usb-kws-mmpc) 和 [*EQUATION](../key/key-link.md#usb-kws-mequation) 选项在后面有更详细的讨论。

### 6.4.3 前处理——创建模型

此示例的完整输入文件是 `crane.inp`，可在"货运吊车"第 A.4 节中找到。用于创建前几页所示的节点和单元的 Abaqus 输入选项可在"货运吊车"第 A.4 节中找到。如果您希望使用 Abaqus/CAE 创建整个模型，请参阅《Abaqus 入门：交互版》第 6.4 节的"示例：货运吊车"。

### 6.4.4 检查输入文件——模型数据

本节描述此示例的输入文件中如何描述模型数据。这些数据包括输入文件标题、节点和单元、梁截面和方向、约束以及边界条件的描述。

**标题**

此示例中使用的标题提供了模型和所用单位的简短描述：

```
*HEADING
3-D model of light-service cargo crane
S.I. Units (m, kg, N, sec)
```

**节点坐标和单元连接**

在 [*NODE](../key/key-link.md#usb-kws-mnode) 选项块中定义节点坐标。如果您决定使用编辑器执行此操作，您可能需要使用"货运吊车"第 A.4 节中的网格生成命令。在此示例中，创建了一个名为 `ATTACH` 的节点集；此节点集包含 A、B、C 和 D 点处的节点，即吊车连接到母结构的点。

```
*NSET, NSET=ATTACH
100, 107, 200, 207
```

在模型中创建与图 6-15 中所示单元对应的单元，但请记住您的编号可能不同。（但是，拥有相同的编号会使某些建模功能的定义更容易。）在此模拟中您需要几个单元集。当定义单元时，它们被分组到以下单元集中：

```
OUTA	    100, 101, 102, 103, 104, 105, 106
BRACEA	  110, 111, 112, 113, 114
OUTB	    200, 201, 202, 203, 204, 205, 206
BRACEB	  210, 211, 212, 213, 214
CROSSEL	 300, 301, 302, 303, 304, 305, 306, 307

其中这些单元编号指的是图 6-15 中所示的单元。单元集 `OUTA` 和 `OUTB` 包含两个桁架结构的主要外构件。单元集 `BRACEA` 和 `BRACEB` 包含建模每个桁架结构内部支撑的单元。单元集 `CROSSEL` 包含连接两个桁架结构的交叉支撑。

使用参数 SECTION=BOX 指定箱形截面。第一行数据包含截面尺寸，即图 6-16 中所示箱形截面的尺寸 *a*、*b*、![](../graphics/gsk_eqn00068.gif)、![](../graphics/gsk_eqn00069.gif)、![](../graphics/gsk_eqn00070.gif) 和 ![](../graphics/gsk_eqn00071.gif)。图 6-16 中所示的尺寸是吊车中两个桁架主要构件的尺寸。

**图 6-16** 主要构件的横截面几何尺寸（单位：m）。

![](../graphics/gss-localbeam-nls.png)

主要构件的梁截面轴应这样定向：使梁的 1 轴垂直于俯视图中桁架结构所在的平面（参见图 6-12），而梁的 2 轴垂直于该平面中的单元。通过在 [*BEAM GENERAL SECTION](../key/key-link.md#usb-kws-mbeamgensect) 选项的第二行数据上给出梁 1 轴的大致方向（![](../graphics/gsk_eqn00060.gif) 向量）来指定此方向。为了获得正确的法向，![](../graphics/gsk_eqn00062.gif)，在这种情况下，您需要提供一个非常准确的 ![](../graphics/gsk_eqn00060.gif)。提供一个近似方向会更容易一些，这将是负 3 方向。然而，鉴于 Abaqus 在给定 ![](../graphics/gsk_eqn00061.gif) 和 ![](../graphics/gsk_eqn00060.gif) 时用于确定 ![](../graphics/gsk_eqn00062.gif) 的逻辑，如果我们使用这个近似的 ![](../graphics/gsk_eqn00060.gif)，法向 ![](../graphics/gsk_eqn00062.gif) 会相对于其正确的方向稍微旋转。您可以为两个桁架结构中的所有单元指定相同的 ![](../graphics/gsk_eqn00060.gif) 方向。第三行数据包含弹性模量和剪切模量，假定为中等强度钢，![](../graphics/gsk_eqn00072.gif) = 200.0 GPa，![](../graphics/gsk_eqn00073.gif) = 0.25，且 ![](../graphics/gsk_eqn00074.gif) = 80.0 GPa。这些建模数据包含在输入文件的以下选项块中：

-0.1118, 0.0, -0.9936
200.E9,80.E9
*BEAM GENERAL SECTION, SECTION=BOX, ELSET=OUTB
0.10,0.05,0.005,0.005,0.005,0.005
-0.1118, 0.0, 0.9936
200.E9,80.E9
```

支撑构件的梁截面尺寸如图 6-17 所示。交叉支撑和每个桁架结构内部的支撑具有相同的梁截面几何形状，但它们的梁截面轴方向不同。因此，必须使用独立的 [*BEAM GENERAL SECTION](../key/key-link.md#usb-kws-mbeamgensect) 选项。支撑由与主要构件相同的钢制成。

![](../graphics/gss-crosssect-geom-nls.png)

```

我们做一些假设，使交叉支撑的方向更容易指定。所有梁的法向（![](../graphics/gsk_eqn00062.gif) 向量）应大致位于货运吊车平面图的平面内（参见图 6-12）。此平面与全局 1-3 平面略有偏斜。同样，定义这种方向的一种简单方法是在单元属性选项上提供一个近似 ![](../graphics/gsk_eqn00060.gif) 向量，该向量垂直于此平面。该向量应几乎平行于全局 2 方向。由于从一个单元到下一个单元的法向之间的角度始终大于 20 度，因此不在节点处对法向进行平均。

根据交叉支撑构件的确切方向，我们可能需要为每个交叉支撑单元单独定义法向。这样的练习与您已经通过为两个桁架结构定义法向所做的非常相似。由于方形交叉支撑构件主要承受轴向载荷，其变形对横截面方向不敏感。因此，我们接受 Abaqus 计算的默认法向为正确的。交叉支撑的大致 ![](../graphics/gsk_eqn00060.gif) 向量与 y 轴对齐。以下选项块指定交叉支撑：

```
*BEAM GENERAL SECTION, SECTION=BOX, ELSET=CROSSEL
0.03,0.03,0.003,0.003,0.003,0.003
0.0,1.0,0.0
200.E9,80.E9
```

**梁截面方向**

在此模型中，如果您仅提供定义近似 ![](../graphics/gsk_eqn00060.gif) 向量方向的数据，则会出现建模错误。节点处梁法向的平均值（参见"梁单元曲率"第 6.1.3 节）导致 Abaqus 对货运吊车模型使用不正确的几何形状。要查看此问题，您可以使用 Abaqus/Viewer 显示梁截面轴和梁切向量（参见"后处理"第 6.4.7 节）。然而，吊车模型中的法向在 Abaqus/Viewer 中看起来是正确的；但实际上，它们略有错误。您还可以通过检查数据（`.dat`）文件中打印的平均节点法向来发现此类建模错误。货运吊车错误模型中的一些法向如下面的输出所示：

![](../graphics/gss-c6-node-def-input-nls.png)

问题是节点 102 的法向向量，它与定义桁架 A 下部主要构件的其他节点处的法向不匹配（参见图 6-14）。四个单元（101、102、112 和 113）包含节点 102。当节点处梁法向的平均值产生多个独立法向时，节点处的额外法向也会打印在数据文件中（详见"梁单元曲率"第 6.1.3 节）。吊车模型的正确几何形状需要在节点 102 处有三个独立的梁法向：一个用于支撑单元 112 和 113，另一个用于单元 101 和 102 的单一法向。上面 NODE DEFINITIONS 表中显示的节点 102 的法向不是单元 101 和 102 所需的法向。如果是这样，它将与节点 100、101 或 103 处显示的法向相匹配。也不是其他两个单元中任何一个的正确法向，它们的法向如下所示打印在数据文件中。

这个错误的平均法向意味着单元 101、102 和 113 具有沿梁轴从单元一端到另一端呈现曲率的截面几何形状，这不是预期的几何形状。您必须使用 [*NORMAL](../key/key-link.md#usb-kws-mnormal) 选项显式定义节点 102 处单元 113 的法向。显式指定法向方向可防止 Abaqus 应用其平均算法。对于吊车相对侧的对应单元，您还必须对桁架 B 的单元 213、节点 202 使用 [*NORMAL](../key/key-link.md#usb-kws-mnormal)。

节点 104 和 204 处每个桁架结构尖端处的法向也存在同样的问题，同样是因为单元 103 和 104 之间的角度小于 20 度。由于我们正在建模直梁单元，因此每个单元在两个节点处的法向都是恒定的。因此，您应放在输入文件中的 [*NORMAL](../key/key-link.md#usb-kws-mnormal) 选项块有六行数据。如果您使用图 6-14 和图 6-15 中所示的编号方案，则应将以下选项块添加到您的输入文件中：

![](../graphics/gss-c6-node-normal-input-nls.png)

**多点约束**

与内部桁架支撑不同，交叉支撑用螺栓固定在桁架构件上。您可以假定这些螺栓连接无法传递旋转或扭矩。在这些位置定义的重复节点需要定义此约束。在 Abaqus 中，可以使用多点约束（MPC）或约束方程来定义这些节点之间的约束。

多点约束允许在模型的不同自由度之间施加约束。Abaqus 中提供了大量的 MPC 库。（有关完整列表和每个 MPC 的描述，请参阅《Abaqus 分析用户指南》第 35.2.1 节"线性约束方程"。）[*MPC](../key/key-link.md#usb-kws-mmpc) 选项的格式为

```
[*MPC](../key/key-link.md#usb-kws-mmpc)
*<MPC 类型>*,*<节点 1 或节点集 1>*,*<节点 2 或节点集 2>*, ......
```

通过使用节点集，您可以用单行数据定义多个相同类型的约束。模拟螺栓连接所需的 MPC 类型是 PIN。此 MPC 创建的铰接节点约束两个节点处的位移相等，但如果节点处存在旋转，则它们保持独立。

吊车模型中有许多螺栓连接。以下是"货运吊车"第 A.4 节中模型的完整 [*MPC](../key/key-link.md#usb-kws-mmpc) 选项块：

```
*MPC
PIN,101,301
PIN,102,302
PIN,103,303
PIN,105,305
PIN,106,306
PIN,201,401
PIN,202,402
PIN,203,403
PIN,205,405
PIN,206,406
```

将类似的选项块添加到您的模型中，更改节点编号以对应您的模型中的编号。如果两个桁架结构上的所有节点都已分组到一个名为 `TRUSNODE` 的节点集中，而交叉支撑上的所有节点都已分组到一个名为 `CROSNODE` 的节点集中，则选项块可以缩短为以下内容：

```
*NSET, NSET=TRUSNODE, UNSORTED
101,102,103,105,106,201,202,203,205,206
*NSET, NSET=CROSNODE, UNSORTED
301,302,303,305,306,401,402,403,405,406
*MPC
PIN, TRUSNODE, CROSNODE
```

如果节点集作为 MPC 类型后的第一项提供，则第二项可以是另一个节点集或单个节点。当 [*MPC](../key/key-link.md#usb-kws-mmpc) 选项的数据行包含节点集然后是单个节点时，如下面所示，Abaqus 会在该集中每个节点与指定的单个节点之间创建 MPC 约束。例如，以下选项块将在节点 301 和节点集 `TRUSNODE` 中的每个节点之间创建铰接连接。

```
[*MPC](../key/key-link.md#usb-kws-mmpc)
PIN, TRUSNODE, 301
```

```
*BOUNDARY
ATTACH, ENCASTRE

### 6.4.5 检查输入文件——历史数据

以下选项指定静态线性扰动模拟：

```
*STEP, PERTURBATION
Static tip load on crane
*STATIC
```

**载荷**

在负 y 方向上向节点 104 施加 10 kN 的集中载荷。由于存在连接节点 104 和 204 的 y 位移的约束方程，载荷由两个节点平均承担。以下 [*CLOAD](../key/key-link.md#usb-kws-hcload) 选项块为两个节点提供相等的载荷：

```
*CLOAD
104,2,-1.0E4
```

**输出请求**

将节点的位移（U）和反作用力及力矩（RF）以及单元中的截面力和力矩（SF）写入输出数据库，以供 Abaqus/Viewer 后处理，如以下选项块所示：

```
*OUTPUT, FIELD
*NODE OUTPUT
U, RF
*ELEMENT OUTPUT
SF,
*END STEP
```

### 6.4.6 运行分析

将输入存储在名为 `crane.inp` 的文件中。使用以下命令运行分析

```
abaqus job=crane
```
