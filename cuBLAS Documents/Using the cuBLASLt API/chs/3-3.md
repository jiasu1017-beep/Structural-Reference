### 3.3.3. cublasLtHandle_t


[cublasLtHandle_t](#cublaslthandle-t) 类型是一个指向不透明结构的指针类型，该结构保存 cuBLASLt 库上下文。使用 [cublasLtCreate()](#cublasltcreate) 初始化 cuBLASLt 库上下文并返回指向保存 cuBLASLt 库上下文的不透明结构的句柄，使用 [cublasLtDestroy()](#cublasltdestroy) 销毁先前创建的 cuBLASLt 库上下文描述符并释放资源。


> **注意**

注意
cuBLAS 句柄（cublasHandle_t）封装了一个 cuBLASLt 句柄。任何有效的 cublasHandle_t 都可以通过简单类型转换用作 cublasLtHandle_t。但是，与 cuBLAS 句柄不同，cuBLASLt 句柄不绑定到任何特定的 CUDA 上下文（从 CUDA 12.8 开始的绑定到图形上下文的 CUDA 上下文除外）。如果在当前 CUDA 上下文绑定到图形上下文时创建了 cuBLASLt 句柄，则 cuBLASLt 会检测相应的共享内存限制并将其记录在句柄中。

### 3.3.23. cublasLtOrder_t

[cublasLtOrder_t](#cublasltorder-t) 是一个枚举类型，用于指示矩阵的数据顺序。

| 值 | 描述 |
| --- | --- |
| CUBLASLT_ORDER_COL | 数据以列主序格式排列。前导维度是内存中到下一列开头的步长（以元素为单位）。 |
| CUBLASLT_ORDER_ROW | 数据以行主序格式排列。前导维度是内存中到下一行开头的步长（以元素为单位）。 |
| CUBLASLT_ORDER_COL32 | 数据以 32 列的列主序瓦片排列。前导维度是到下一个 32 列组开头的步长（以元素为单位）。
例如，如果矩阵有 33 列和 2 行，则前导维度必须至少为 32 * 2 = 64。 |
| CUBLASLT_ORDER_COL4_4R2_8C | 数据以复合瓦片的列主序瓦片排列，总共 32 列和 8 行。瓦片由以交替模式交织的 4 列内的 4 个偶或奇行的内部瓦片组成。
前导维度是到下一个 32 列宽组的第一个 32 列 × 8 行瓦片开头的步长（以元素为单位）。例如，如果矩阵有 33 列和 1 行，则前导维度必须至少为 (32 * 8) * 1 = 256。 |
| CUBLASLT_ORDER_COL32_2R_4R4 | 数据以复合瓦片的列主序瓦片排列，总共 32 列和 32 行。瓦片内的元素偏移计算为 (((row % 8) / 2 * 4 + row / 8) * 2 + row % 2) * 32 + col。
前导维度是到下一个 32 列宽组的第一个 32 列 × 32 行瓦片开头的步长（以元素为单位）。例如，如果矩阵有 33 列和 1 行，则其前导维度必须至少为 (32 * 32) * 1 = 1024。 |
