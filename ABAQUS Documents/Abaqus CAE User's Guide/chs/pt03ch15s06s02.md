# 15.6.2 接触检测算法





表面必须满足两个要求才能被自动接触检测工具识别：
- 表面之间的距离必须小于或等于指定的分离容差。
- 表面必须直观上相对，如下所定义。

Abaqus/CAE将两个表面之间的分离定义为表面上最近点之间的距离。此距离报告在接触对候选表的「Separation」列中。分离容差是接触检测搜索期间的主要输入。您应该指定一个分离容差，使其包含模型中所有潜在接触表面之间的分离距离。更多信息，请参见下文「选择分离容差和扩展角」。

**注意：**「Separation」列中报告的值可能与Abaqus/Standard在分析期间使用的分离不完全对应。在分析期间应用的某些自动表面增强（以提高接触稳健性，如主表面平滑和表面扩展）可能导致Abaqus/CAE预处理器中计算的分离与Abaqus/Standard分析之间存在轻微差异。有关自动表面增强和接触公式的更多详细信息，请参阅「在Abaqus/Standard中定义接触对」，Abaqus Analysis User's Guide第36.3节。

如果两个在最近点处构建的表面法线彼此之间在135到225度之间（请参见[图15-4](pt03ch15s06s02.md#itn-detection-angles)），则两个表面被视为直观上相对。换句话说，表面在最近点处必须彼此偏离小于45度。不可能调整或忽略表面方向要求。

**图15-5** 参与潜在接触的两个物体。为简单起见，物体以二维形式呈现。

![](../graphics/itn-detection-algorithm-example-nls.png)

虚线表示根据表面X计算的分离容差。平行于表面X的表面B被识别为接触对的一部分，因为它既在分离容差范围内又与表面X直观上相对。类似地，表面C满足这两个条件。表面D虽然与表面X直观上相对，但在任何点都不在分离容差范围内；表面D不被考虑包含在接触对中。表面A虽然在分离容差范围内，但与表面X不是直观上相对；因此，表面A也被排除在任何接触对定义之外。连接的表面（A、B、C和D）彼此之间不形成接触对。默认情况下，Abaqus/CAE仅搜索不同部件实例上的表面。但是，即使您启用在同一实例内搜索（请参见下文「定义同一实例内和自接触的接触」），这些表面也不会满足方向要求。

### 定义接触对的附加标准

在使用分离和方向检查编译潜在接触对列表后，接触检测工具可以执行一系列附加检查，以调整表面定义使其更有用和更真实。所有这三个附加检查都是可选的，但默认情况下它们是启用的。

**扩展表面**

默认情况下，接触检测工具识别的任何表面都会被扩展以包含相邻模型面，即使相邻面不满足分离和方向要求。20度角是根据检测表面与公共边上相邻面之间的法线偏移来测量的。您可以使用「Extend each surface found by angle」选项修改扩展角。当面被添加到表面定义时，Abaqus/CAE也会检查与新添加面相邻的任何面。如果扩展表面包含来自单独定义的接触对的面，Abaqus/CAE会消除任何冗余定义。例如，考虑为[图15-5](pt03ch15s06s02.md#itn-detection-algorithm-example)中的模型在20度内扩展表面。Abaqus/CAE创建一个接触对：一个表面由面X组成，另一个表面由面B、C和D组成。面D在面C的20度内，面C在面B的20度内；包含面C和面X的冗余接触对被消除，因为它被更大的接触对所包含。

**在指定角度内合并接触对**

您可以使用「Merge pairs when surfaces are within angle」选项将多个接触对组合成单个定义。接触对涉及的面必须相邻且必须在指定角度内（如上所述）。合并选项不扩展面；它仅合并正式识别的接触对。默认情况下，表面在20度内的接触对由接触检测工具合并。合并选项通常用作表面扩展的替代方案，以自动合并接触对候选而不将表面定义扩展到分离容差之外。例如，对于[图15-5](pt03ch15s06s02.md#itn-detection-algorithm-example)中的模型，在不扩展表面的情况下合并20度内的接触对会产生一个接触对：一个表面由面X组成，另一个表面由面B和C组成。

**检查表面重叠**

默认情况下，接触检测工具消除任何其表面不"重叠"的接触对；如果从一个表面的任何点发出的法线不通过相对表面，则两个表面不重叠。例如，[图15-6](pt03ch15s06s02.md#itn-detection-algorithm-overlap)中的表面不重叠，即使它们可能通过分离和方向检查。

**图15-6** 非重叠表面。为简单起见，物体以二维形式呈现。

![](../graphics/itn-detection-overlap.png)

您可以使用「Include opposing surfaces that do not overlap」选项来抑制表面重叠检查，并允许为非重叠表面创建接触对。

### 几何的接触检测

Abaqus/CAE通过将模型划分为单独的面开始搜索由几何组成的模型。面由连接的几何边或分区所包围的区域组成。一旦识别出所有面，Abaqus/CAE会比较这些面以确定它们是否满足分离和方向要求，然后通过应用扩展、合并和重叠检查来从面定义表面（请参见上文「定义接触对的附加标准」）。满足所有要求的任何两个表面都被标记为接触对候选。

Abaqus/CAE自动为检测到的接触对中的表面分配主从角色。解析刚性或离散刚性表面始终被分配主角色；如果接触对涉及两个刚性表面，则主从角色的分配是任意的。对于涉及两个可变形表面的接触对，Abaqus/CAE首先确定表面几何是否已被网格划分，并将主角色分配给具有较粗网格的表面。如果网格信息不可用，则面积较大的表面成为主表面。分配主从角色的算法不考虑底层刚度或单元分配的差异；如果这些因素在您的接触相互作用中起重要作用，您应该在创建相互作用之前审查主从角色分配。有关主从角色分配的进一步讨论，请参见「在Abaqus/Standard中定义接触对」，Abaqus Analysis User's Guide第36.3.1节中的「选择接触对中使用的表面」。

### 网格模型的接触检测

接触检测也适用于网格模型。网格模型的搜索算法与几何模型的工作方式大致相同，但它使用单元面而不是几何面。默认情况下，Abaqus/CAE仅搜索不同部件实例之间的接触对。导入Abaqus/CAE的网格模型通常仅由单个部件实例组成；因此，在这些模型上使用接触检测之前，您应该启用在同一实例内搜索（请参见下文「定义同一实例内和自接触的接触」的更多详细信息）。

**警告：**与基于几何的搜索不同，基于网格的表面的报告分离不一定是最近点的精确距离，而是一个近似值。如果指定的搜索容差相对于特征单元尺寸非常大，则此近似值的精度会大大降低。

在单元面上定义表面之前，Abaqus/CAE应用与几何面相同的扩展、合并和重叠检查（请参见上文「定义接触对的附加标准」）。由于单元面通常比几何面小得多，因此您应该始终允许表面有一些扩展以从表面定义中获得足够的覆盖；[图15-7](pt03ch15s06s02.md#itn-detection-mesh-surf)比较了不允许表面扩展时几何和网格几何创建的表面。

**图15-7** 不允许表面扩展时几何（左）和网格几何（右）创建的表面之间的差异。

![](../graphics/itn-detection-mesh-surf-nls.png)

如果您对模型进行重新网格划分，则在单元面上定义的任何表面可能会变得无效。因此，基于这些面的相互作用和约束也变得无效。

在为网格表面分配主从角色时，刚性表面始终成为主表面；如果接触对涉及两个刚性表面，则主从角色的分配是任意的。对于涉及两个可变形表面的接触对，Abaqus/CAE考虑每个表面上的网格密度；具有较粗网格的表面成为主表面。如果两个表面上的网格密度相同，则主从角色的分配是任意的。分配主从角色的算法不考虑底层刚度或单元类型的差异；如果这些因素在您的接触相互作用中起重要作用，您应该在创建相互作用之前审查主从角色分配。有关主从角色分配的进一步讨论，请参见「在Abaqus/Standard中定义接触对」，Abaqus Analysis User's Guide第36.3.1节中的「选择接触对中使用的表面」。

接触检测工具不检测几何与孤儿单元或解析表面与孤儿单元之间的接触。如果您的模型包含已从几何进行网格划分的部件实例，您可以使用接触检测对话框的「Advanced」选项卡页面上的选项来指示这些实例在搜索期间应被视为几何（默认）还是单元网格。如果您的模型同时包含几何和孤儿网格单元的实例，您应该首先对所有几何进行网格划分，然后执行基于网格的搜索以捕获所有可能的接触对。

在大多数情况下，几何比网格几何更能忠实地表示被建模的对象。此外，基于几何的相互作用和约束不受重新网格划分的影响。然而，网格是分析中使用的几何。网格离散化可能导致两种表示之间分离距离的轻微差异，这在精确分析中可能变得重要。执行搜索后，您可以使用「Recalculate Separation」选项检查各个接触对的本机几何和网格几何之间的差异。

### 检测过度闭合表面

如果装配中的两个面在任何点相交，接触检测工具会将这些面报告为过度闭合接触对。出现在接触对候选表中的过度闭合接触对仍必须满足表面方向要求。「Separation」列中的红色零表示接触对中的两个表面正在相交。

**注意：**「Separation」列中的黑色零意味着两个表面在其最近点正好接触。在这种情况下没有过度闭合或相交。

如果您扩展或合并过度闭合表面以包含未过度闭合的面，Abaqus/CAE会将整个接触对报告为过度闭合。

在创建接触相互作用之前，您应该目视检查所有过度闭合表面。具有严重过度闭合的模型应该进行调整以消除过度闭合（或至少减轻其严重程度）。轻微的过度闭合可以通过使用接触调整选项（可在接触对候选表中获得）或干扰配合选项（可在接触相互作用编辑器中获得）来解决。

面必须相交才能被报告为过度闭合。如果一个面完全封闭在另一个部件实例内，则自动接触检测工具不会将该面报告为过度闭合。这样的面可能仍然满足与封闭实例上外部面之间的分离和方向要求。默认情况下，Abaqus/CAE从接触对候选表中消除封闭面，因为表面不"重叠"（请参见「定义接触对的附加标准」）。如果您禁用重叠检查，Abaqus/CAE会为封闭面报告接触对候选，但接触对候选表不提供任何表面过度闭合或穿透的指示。因为接触检测工具不认为这些面是过度闭合的，所以默认情况下应用于过度闭合表面的调整选项（请参见「默认相互作用和约束参数」，第15.6.3节）不会应用于此接触对。如果封闭面嵌入在任何外部面的分离容差深度之外，则自动接触检测工具不会将那些面识别为接触对候选。

例如，考虑[图15-8](pt03ch15s06s02.md#itn-detection-enclosed-face)中的模型。

**图15-8** 在此模型中，圆柱形部件实例的一端完全封闭在另一个部件实例内。

![](../graphics/itn-detection-enclosed-face-nls.png)

为此搜索指定的分离容差为0.1。部件实例B末端的圆形面在分离容差范围内，并且与部件实例A上的矩形面直观上相对，但没有相交。接触对候选表列出一个由圆形面和矩形面组成的正常接触对，分离距离为0.06。部件实例B的圆柱侧面被列为过度闭合，因为它与部件实例A的矩形面相交。

虽然接触检测工具不认为完全封闭的表面是过度闭合的，但此类表面在分析期间仍被视为过度闭合。严重的过度闭合通常会导致收敛困难。在审查接触对候选表中的过度闭合接触对时，检查相邻表面是否有完全封闭的面。

### 定义同一实例内和自接触的接触

您可以使用接触检测工具来定义同一部件实例或模型实例的不同区域之间的接触。此功能对于以单个部件实例导入Abaqus/CAE的复杂模型特别有用。如果您启用「Include pairs with surfaces on the same instance」选项，Abaqus/CAE会检查同一部件实例或模型实例上的不同几何或单元面，以确定它们是否满足分离和方向要求。在满足要求的任何面上定义表面和接触对。

在某些情况下，表面扩展选项会导致主表面和从表面重叠。如果主表面和从表面由相同的面组成，Abaqus/CAE会自动调整接触对以创建自接触相互作用，在变形期间表面与自身接触。在这种情况下创建单个表面。

### 壳的注意事项

如果为壳分配了截面定义，则接触检测工具会在分离计算中考虑壳的厚度。报告的分离已根据壳截面分配中指定的厚度和偏移进行了调整。您可以使用「Account for shell thickness and offset」选项在接触检测搜索期间忽略壳截面属性。变化的厚度分布从不在分离计算中考虑。

接触检测工具自动选择创建表面的壳侧面（更多信息，请参见「指定区域的特定侧面或端部」，第73.2.5节）。选择侧面使得最近点处的表面法线直观上相对。

在使用包含孤儿壳单元的模型时，请确保元素法线方向在元素之间一致；也就是说，正元素面（SPOS）都应该位于壳结构的同一侧（更多信息，请参见「壳单元概述」，Abaqus Analysis User's Guide第29.6.1节）。如果元素法线方向不一致，Abaqus/CAE会误解元素面之间的角度，表面扩展和合并操作也无法正常工作。

对于某些基于样条的壳或面，相同表面可能与壳的两侧相互作用（请参见例如[图15-9](pt03ch15s06s02.md#itn-detection-spline)）。

**图15-9** 基于样条的壳。

![](../graphics/itn-detection-spline.png)

通常，您应该为壳的每一侧定义一个单独的接触对。但是，接触检测工具无法创建涉及相同两个面的多个接触对；它将定义一个接触对，并根据最近点处的方向选择壳侧面。您必须手动为壳的另一侧定义另一个接触对。

接触检测工具不创建任何双侧表面。如果合适，您可以在模型树中编辑创建的表面定义以使其成为双侧（请参见「编辑集合和表面」，第73.3.5节）。
