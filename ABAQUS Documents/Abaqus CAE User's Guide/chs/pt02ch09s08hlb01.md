# 9.9.1 将会话对象和会话选项保存到文件





默认情况下，Abaqus/CAE中的许多对象和选项仅在当前会话期间保留。要将这些会话对象或会话选项保留供将来的Abaqus/CAE会话使用，请将它们保存到模型数据库、输出数据库或XML格式的设置文件中。如果将设置保存到模型数据库或输出数据库，Abaqus/CAE会在您打开该文件时使用这些设置作为新的默认值；如果保存到设置文件，则必须将这些设置从该文件加载到会话中才能使用它们。

当您保存会话对象或选项时，您可以包括在该会话中指定的所有会话特定设置，也可以从各个类别中选择设置。例如，您可以将会话中的所有显示组和路径定义保存到文件，同时排除所有视图切割定义。当您选择某个类别时，**保存会话对象和选项（Save Session Objects & Options）**对话框会保存该类别中的所有定义；您不能将单个显示组保存到文件同时排除其他显示组。如果您希望仅一部分显示组可供后续会话使用，则必须在将会话对象保存到文件之前从会话中删除其他显示组。

在将会话对象和选项保存到文件时，您必须注意对象依赖关系。例如，自由体切割可能引用先前定义的显示组，因此如果您想保留自由体切割，保存显示组和自由体切割都有意义。同样，如果您想将活动视图切割和自由体切割的列表保存到文件，还应该保存视图切割和自由体切割本身。

会话对象通常是您在会话中定义的项目，如显示组或视图切割；而会话选项通常是对话框中的设置，如**通用绘图选项（Common Plot Options）**对话框。您可以保存以下会话对象：
- 显示组
- 路径
- X-Y数据对象
- 自由体定义
- 视图切割（仅在可视化模块中）
- 自由体切割和视图切割的活动状态
- 当前选定的视图和**视图**工具栏上指定的11个视图

您可以保存以下会话选项：
- **ODB显示选项（ODB Display Options）**
- **结果选项（Result Options）**
- **通用绘图选项（Common Plot Options）**
- **等值线图选项（Contour Plot Options）**
- **叠加绘图选项（Superimpose Plot Options）**
- **材料方向绘图选项（Material Orientation Plot Options）**
- **层压板堆叠绘图选项（Ply Stack Plot Options）**
- **符号绘图选项（Symbol Plot Options）**
- **自由体绘图选项（Free Body Plot Options）**
- **视图切割选项（View Cut Options）**（仅来自**自由体（Free Body）**和**切片（Slicing）**选项卡页面）
- **颜色映射（Color Mapping）**

仅当输出数据库以写权限打开时，您才能保存会话对象和选项到输出数据库。有关更多信息，请参阅["打开模型数据库或输出数据库，"第9.7.2节"](pt02ch09s06hlb02.md)。

**要将会话对象或会话选项保存到文件：**

1. 从主菜单栏选择****文件（File）**![](../graphics/images/arrow.gif)**保存会话对象（Save Session Objects）****。**提示：**您也可以从**文件**工具栏单击![](../graphics/icoR_persistenceSave.png)。出现**保存会话对象和选项（Save Session Objects & Options）**对话框。
2. 从**目标（Destination）**选项中，选择您要保存会话对象和选项的文件类型，并选择文件（如果适用）：- 选择**文件（File）**保存为XML格式的设置文件，并指定文件名。- 选择**MDB（*.cae）**保存到当前模型数据库。- 选择**ODB**保存到输出数据库，并指定当前会话中打开的输出数据库之一。
3. 指定您要保存的会话对象或会话选项的类别：- 如果您想保存所有会话对象或所有会话选项，请分别切换**对象（Objects）**或**选项（Options）**。- 如果您想单独选择会话对象或会话选项的类别，请展开**对象（Objects）**或**可视化选项（Visualization Options）**容器，然后切换您要保存的类别。
4. 单击**确定（OK）**。如果您选择**MDB（*.cae）**作为目标文件，您必须选择****文件（File）**![](../graphics/images/arrow.gif)**保存（Save）****或****文件（File）**![](../graphics/images/arrow.gif)**另存为（Save As）****来保存模型数据库。
