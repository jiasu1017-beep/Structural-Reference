# 81.7 内核插件是如何执行的？





Abaqus 通过向内核发出形式为 *moduleName.functionName* 的命令来执行内核插件。模块名称和函数名称是您为该插件在注册命令中提供的名称。

当 Abaqus/CAE 启动并导入插件时，插件所在的目录由 Abaqus/CAE 存储。第一次调用插件时，Abaqus/CAE 会用该插件的目录更新内核的 *sys.path* 列表。下次调用该插件时，Abaqus/CAE 会发出插件内部的命令，但不更新 *sys.path*。

例如，考虑前面部分描述的 `myUtils_plugin.py` 和 `myUtils.py` 文件。假设您将这两个文件存储在主目录中 `abaqus_plugins` 目录下的一个名为 `myPlugins` 的子目录中。第一次点击 **Plug-ins** 菜单中的 **Print Current Viewport** 时，Abaqus/CAE 会向内核发送以下命令：

```
import sys
sys.path.append('*path to your home dir*/abaqus_plugins/myPlugins')
import myUtils
myUtils.printCurrentViewport()
```

下次您点击 **Plug-ins** 菜单中的 **Print Current Viewport** 时，Abaqus/CAE 只向内核发送以下命令：

```
myUtils.printCurrentViewport()
```

由于 Abaqus/CAE 更新了 *sys.path* 列表，您的插件代码不需要执行此任务来导入它需要的模块。假设这些模块位于与插件相同的目录中。如果您必须导入不在同一目录中的模块，则必须扩充 *sys.path* 列表。如果需要扩充 *sys.path* 列表，应使用自动确定文件位置的函数，而不是在文件中硬编码插件的路径。这样可以轻松地将插件移动到不同的位置，而无需修改其代码。以下 `myUtils.py` 的示例说明了扩充 *sys.path* 列表的方法：

```
import sys, os

# Full path (with name) to this file
absPath = os.path.abspath(__file__)  

# Full directory specification
absDir = os.path.dirname(absPath)    

# Full subdirectory specification
subDir = os.path.join(absDir, 'mySubDir') 
sys.path.append(subDir)

# myModule is located in subDir
import myModule  

*rest of module code*
```
