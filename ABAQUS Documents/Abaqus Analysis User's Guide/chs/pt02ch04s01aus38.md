### 概述

Abaqus在分析过程中可以创建以下输出文件：
- 一个数据文件，包含由分析输入文件处理器生成的模型和历史定义的打印输出，以及在Abaqus/Standard中，分析运行期间写入的结果的打印输出；
- 一个ODB输出数据库文件，包含用于Abaqus/CAE的Visualization模块（Abaqus/Viewer）进行后处理的结果，以及在Abaqus/Standard中的诊断信息；
- 一个SIM数据库文件，包含用于3DEXPERIENCE平台上的Physics Results Explorer应用进行高性能后处理的结果；
- Abaqus/Explicit中的选定结果文件；
- 一个结果文件，包含用于外部软件进行后处理的结果，适用于Abaqus/Standard和Abaqus/Explicit（在Abaqus/Explicit中，该文件通过转换选定结果文件生成）；
- 一个消息文件，包含关于求解过程的诊断消息，适用于Abaqus/Standard和Abaqus/Explicit；
- 一个状态文件，包含关于分析状态的信息，以及在Abaqus/Explicit中的诊断消息和关于稳定时间增量的信息；以及
- Abaqus/CFD中使用备用文件格式的输出文件。

Abaqus可以创建用于重启分析的文件——请参见["重启分析，" 9.1.1节](pt04ch09s01aus53.md)。在Abaqus/Standard中，这些文件也可用于提取分析期间未请求的输出结果。

### 数据文件

数据文件（`*job-name*.dat`）是一个文本文件，包含模型定义信息（由分析输入文件处理器生成）和在Abaqus/Standard中的结果表格输出。分析输入文件处理器信息包括模型定义、历史定义以及在处理输入数据时检测到的任何错误和警告消息。

#### 控制写入数据文件的分析输入文件处理器信息量

您可以控制由分析输入文件处理器写入数据文件的信息量，适用于Abaqus/Standard和Abaqus/Explicit。

| **输入文件用法：** | 在输入文件的模型定义部分使用以下选项： |
| --- | --- |
|  | `[*PREPRINT](../key/key-link.md#usb-kws-mpreprint)` |

| --- | --- |

#### 请求打印结果

在Abaqus/Standard中，可以在整个分析过程中以表格格式将输出变量的值打印到数据文件。您可以控制分析运行期间以下类型的打印输出：元素输出、节点输出、接触表面输出、能量输出、紧固件相互作用输出、模态输出、截面输出和辐射输出——请参见["输出到数据和结果文件，" 4.1.2节](pt02ch04s01aus39.md)和["腔体辐射，" 9.1节](pt09ch41s01aus187.md)。您可以指定要在每个输出表中打印的变量，对于元素变量，还可以指定打印位置（在积分点、元素质心、节点处或在节点处平均）。具有变换的节点处的节点变量可以以全局或局部坐标系写入（请参见["变换坐标系，" 2.1.5节](pt01ch02s01aus09.md)）。可用变量列表在["Abaqus/Standard输出变量标识符，" 4.2.1节](pt02ch04s02abv01.md)中给出。对数据文件的结果输出作为步骤定义的一部分请求。

#### 在数据文件中查看部件和装配信息

Abaqus模型可以用部件实例的装配体来定义（请参见["定义装配体，" 2.10.1节](pt01ch02s10aus28.md)）。在这种模型中，节点和元素编号可以在不同部件的定义中重复。这些局部编号在Abaqus内部转换为唯一的全局编号，写入数据文件的输出以这些内部编号给出。如果在数据文件中请求了包含节点和元素编号的任何输出，则用户定义的编号与内部编号之间的映射将被打印到数据文件中（在步骤数据之后）。

出现在数据文件中的集合和表面名称以装配体和部件实例名称为前缀，以下划线分隔（例如，`Assembly_Part1-1_setname`）。

在部件或部件实例内定义的局部坐标系根据部件实例定义中给出的定位数据进行平移和旋转。

### 输出数据库

输出数据库是一个中性二进制文件。与重启或二进制结果文件不同，它可以直接从一种计算平台复制到另一种平台，无需转换。

#### 输出数据库的格式

Abaqus输出数据库有两种格式可用：ODB和SIM。默认情况下，结果输出以ODB格式创建。对于Abaqus/Standard或Abaqus/Explicit分析，您可以选择在同一作业期间以两种格式写入结果。只有SIM格式的结果可以导入3DEXPERIENCE平台进行高性能后处理。有关更多信息，请参见下面的["以SIM格式写入结果的限制"](pt02ch04s01aus38.md#usb-out-ooutput-sim-limitations)。
- ODB输出数据库（`*job-name*.odb`）用于以部件实例装配体的形式存储模型信息和分析结果。Visualization模块的Abaqus/CAE（Abaqus/Viewer）使用此输出数据库进行后处理分析结果和查看诊断信息。
- SIM数据库文件（`*job-name*.sim`）包含模型和结果信息。3DEXPERIENCE平台上的Physics Results Explorer应用使用此数据库进行高性能后处理。

| **输入文件用法：** | 使用以下命令行选项以SIM格式在Abaqus/Standard或Abaqus/Explicit分析中写入结果： |
| --- | --- |
|  | **abaqus** **job**=*job-name* **resultsformat**=`sim` 使用以下命令行选项以ODB格式和SIM格式在Abaqus/Standard或Abaqus/Explicit分析中写入结果：**abaqus** **job**=*job-name* **resultsformat**=`both` 使用以下命令行选项以SIM格式在Abaqus/CFD分析中写入场输出结果：**abaqus** **job**=*job-name* **field**=`sim` 使用以下命令行选项以SIM格式在Abaqus/CFD分析中写入历史输出结果：**abaqus** **job**=*job-name* **history**=`sim` |

| **Abaqus/CAE用法：** | 使用以下输入以SIM格式在Abaqus分析中写入结果： |
| --- | --- |
|  | Job模块：job编辑器：**General**：**Results Format**：**SIM** 使用以下输入以ODB格式和SIM格式在Abaqus/Standard或Abaqus/Explicit分析中写入结果：Job模块：job编辑器：**General**：**Results Format**：**Both** |

##### 浮点数据的处理

默认情况下，浮点数据以单精度写入ODB输出数据库文件。您可以选择以双精度将浮点节点场输出数据写入ODB输出数据库文件；有关详细信息，请参见["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行，" 3.2.2节](pt01ch03s02abx02.md)。

对于Abaqus/Standard和Abaqus/Explicit分析，浮点数据以单精度写入SIM数据库，节点坐标除外（以双精度写入）。对于Abaqus/CFD分析，浮点数据以双精度写入SIM数据库。

##### 在Abaqus/CAE中打开输出数据库

您可以在Abaqus/CAE中打开旧版本Abaqus的输出数据库文件，但Abaqus 5.8输出数据库文件无法在版本6中打开的情况除外。来自先前版本Abaqus的输出数据库文件必须在打开时转换到当前版本。如果您使用的是旧版本的Abaqus/CAE，则无法打开从较新版本Abaqus创建的输出数据库文件。

##### 选择输出格式

输出格式的选择取决于您对高性能可视化、Physics Results Explorer应用和后处理需求的经验水平。
- 如果您仍在学习使用高性能可视化，并希望将结果与Abaqus/Viewer进行比较，请以两种格式写入结果。
- 如果模型很大，您需要Physics Results Explorer应用的改进性能以及Abaqus/Viewer的功能，请以两种格式写入结果。
- 如果您确信Physics Results Explorer应用中的高性能可视化功能提供了您需要的所有功能，请以SIM格式写入结果。

#### 请求输出到输出数据库

您可以从["Abaqus/Standard输出变量标识符，" 4.2.1节](pt02ch04s02abv01.md)、["Abaqus/Explicit输出变量标识符，" 4.2.2节](pt02ch04s02xbv01.md)和["Abaqus/CFD输出变量标识符，" 4.2.3节](pt02ch04s02cbv01.md)中选择要写入输出数据库的变量。以下类型的输出可用：元素输出、节点输出、接触表面输出、能量输出、积分输出、时间增量输出、紧固件相互作用输出、模态输出和辐射输出。此外，Abaqus/Standard和Abaqus/Explicit中写入消息文件的一部分诊断信息（请参见["Abaqus/Standard和Abaqus/Explicit中的消息文件"](pt02ch04s01aus38.md#usb-out-ooutput-message)）和Abaqus/Explicit状态文件（请参见["状态文件"](pt02ch04s01aus38.md#usb-out-ooutput-status)）也包含在输出数据库中。请参见["输出到输出数据库，" 4.1.3节](pt02ch04s01aus40.md)，了解如何生成输出数据库请求的详细说明。

输出数据库中存储了三类信息："场"输出、"历史"输出和诊断信息。场输出旨在对模型的较大部分进行相对不频繁的输出。Abaqus/CAE使用场输出在Visualization模块中生成等值线图、变形形状图、符号图和*X–Y*图。历史输出旨在以相当高的频率对模型的较小部分进行输出。Abaqus/CAE使用历史输出在Visualization模块中生成*X–Y*图。请参见["输出到输出数据库，" 4.1.3节](pt02ch04s01aus40.md)，了解场输出和历史输出的详细描述。诊断信息旨在提供用于Abaqus/CAE的收敛信息；有关更多信息，请参见[Abaqus/CAE User's Guide的第41章，"查看诊断输出"](../usi/usi-link.md#usv-output)。

### 选定结果文件

Abaqus/Explicit选定结果文件（`*job-name*.sel`）存储用户选择的结果，这些结果被转换为结果文件（`*job-name*.fil`），供其他商业后处理包进行后处理。

可以请求元素输出、节点输出和能量输出（请参见["输出到数据和结果文件，" 4.1.2节](pt02ch04s01aus39.md)，了解详细信息）；可用于输出的变量列在["Abaqus/Explicit输出变量标识符，" 4.2.2节](pt02ch04s02xbv01.md)中。您可以以比重启间隔更频繁的间隔为给定的节点集或元素集写入结果的用户选择子集。您可以在步骤定义中指定输出请求，这使您可以有选择地控制写入选定结果文件的数据量，以避免使用过多的磁盘存储。例如，在处理非常大的模型时，您可能选择仅在整个模型中写入当前位移和等效塑性应变20次，并在该步骤中在一个节点处写入加速度历史200次。

### 结果文件

Abaqus/Standard和Abaqus/Explicit中的Abaqus结果文件（`*job-name*.fil`）可以被外部后处理器读取，以生成*X–Y*图或打印的表格输出。大多数商业有限元结果展示包提供翻译器，使用Abaqus结果文件作为输入。结果文件也可以作为一种方便的媒介，用于将分析结果导入您自己的后处理程序。["访问结果文件信息，" 5.1.3节](pt02ch05s01aus43.md)提供了有关如何读取此文件的详细信息。

来自热传递、热-电或热-电-结构分析的温度结果文件输出可用作相同网格的应力分析的输入（请参见["顺序耦合热-应力分析，" 16.1.2节](pt04ch16s01at39.md)）。

#### 在Abaqus/Standard中获得结果文件输出

在Abaqus/Standard中，您可以从["Abaqus/Standard输出变量标识符，" 4.2.1节](pt02ch04s02abv01.md)中选择要写入结果文件的变量，方式类似于为数据文件打印输出选择变量。您必须明确请求将值写入结果文件，否则不会提供任何内容。元素输出、节点输出、接触表面输出、能量输出、模态输出和辐射输出可用——请参见["输出到数据和结果文件，" 4.1.2节](pt02ch04s01aus39.md)和["腔体辐射，" 9.1节](pt09ch41s01aus187.md)，了解详细信息。

##### 在步骤开始时获取结果

您可以请求将步骤开始时的求解状态（零增量）写入Abaqus/Standard结果文件。零增量文件输出仅适用于时间概念控制所选程序的增量方案的步骤，因此以下程序被排除：
- 线性静态扰动分析（["静态应力分析，" 6.2.2节](pt03ch06s02at01.md)）
- ["特征值屈曲预测，" 6.2.3节](pt03ch06s02at02.md)
- ["固有频率提取，" 6.3.5节](pt03ch06s03at10.md)
- ["基于模式的稳态动态分析，" 6.3.8节](pt03ch06s03at13.md)
- ["响应谱分析，" 6.3.10节](pt03ch06s03at15.md)
- ["随机响应分析，" 6.3.11节](pt03ch06s03at16.md)

如果您请求零增量结果文件输出，它将为给定分析中的所有有效程序生成。

如果您使用结果文件来驱动子模型，则必须请求零增量结果文件输出（请参见["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行，" 3.2.2节](pt01ch03s02abx02.md)）；请参见["基于节点的子建模，" 10.2.2节](pt04ch10s02aus61.md)，进行进一步讨论。

|  | `[*FILE FORMAT](../key/key-link.md#usb-kws-hfileformat)`选项可以作为模型数据或历史数据给出，但在输入文件中只能出现一次。 |

#### 在Abaqus/Explicit中获得结果文件输出

Abaqus/Explicit结果文件是由选定结果文件生成的顺序访问文件（请参见["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行，" 3.2.2节](pt01ch03s02abx02.md)）。结果文件以["结果文件输出格式，" 5.1.2节](pt02ch05s01afi01.md)中所述的格式包含请求的结果。

#### 部件和装配信息

Abaqus模型可以用部件实例的装配体来定义（请参见["定义装配体，" 2.10.1节](pt01ch02s10aus28.md)）。但是，结果文件不包含部件和装配记录。

在用部件实例的装配体定义的模型中，节点和元素编号可以在不同部件的定义中重复。这些局部编号在Abaqus内部转换为唯一的全局编号，写入结果文件的输出以全局（内部）编号给出。如果请求了包含节点和元素编号的任何结果文件输出，则用户定义的编号与内部编号之间的映射将被打印到数据文件中。

出现在结果文件中的集合和表面名称以装配体和部件实例名称为前缀，以下划线分隔（例如，`Assembly_Part1-1_setname`）。

在部件或部件实例内定义的局部坐标系根据部件实例定义中给出的定位数据进行平移和旋转。

#### 结果文件的格式

Abaqus/Standard或Abaqus/Explicit中的Abaqus结果文件组织为顺序文件，可以是二进制或ASCII格式。如果文件将在与写入文件不同的计算机系统上读取，则需要ASCII格式。ASCII格式允许结果文件在不同计算机系统之间传输，而无需转换二进制数据。如果文件将始终在同一系统上使用或使用相同二进制格式的系统上使用，则不需要ASCII格式。如果结果文件输出始终位于同一台计算机上，则默认的二进制格式通常是存储文件最有效的方式。对于大型问题，ASCII格式的文件将比相同文件的二进制格式大得多。

##### 控制Abaqus/Standard中结果文件的格式

Abaqus/Standard可以以二进制或ASCII格式写入结果文件。默认格式是二进制。

结果文件输出必须以相同格式为整个分析写入。重新启动问题时无法更改格式。

Abaqus/Standard结果文件的格式也可以在Abaqus/Standard环境文件中控制（请参见["使用Abaqus环境设置，" 3.3.1节](pt01ch03s03aus30.md)）。分析中指定的格式优先于环境文件中定义的值。

此外，Abaqus执行程序中的**ascfil**工具（["结果（`.fil`）文件的ASCII转换，" 3.2.13节](pt01ch03s02abx13.md)）可用于在分析完成后将二进制Abaqus/Standard结果文件（`*job-name*.fil`）转换为ASCII格式（`*job-name*.fin`）。

| --- | --- |

##### 控制Abaqus/Explicit中结果文件的格式

Abaqus/Explicit在文件转换期间始终以二进制格式写入结果文件，但可以使用**ascfil**工具（["结果（`.fil`）文件的ASCII转换，" 3.2.13节](pt01ch03s02abx13.md)）将二进制Abaqus/Explicit结果文件转换为ASCII格式。

##### ASCII格式

["结果文件输出格式，" 5.1.2节](pt02ch05s01afi01.md)定义了写入结果文件的记录内容；这些描述也适用于以ASCII格式写入的结果文件。这些文件中的所有数据项都是整数、浮点数或字符字符串。当请求ASCII格式时，每个数据项在写入文件之前会被转换为等效的字符串。这些字符串以记录定义中描述的顺序写入80字符的逻辑记录中。

每个80字符的逻辑记录在开始下一个记录之前完全填满，因此任何数据项都可以拆分，部分字符在一个逻辑记录中，其余在下一个记录中。每个数据项通常紧跟在其前任之后。例外情况是，对于结果文件记录键2001，Abaqus将用空白字符填满逻辑记录，以便可以立即将记录写入物理存储介质。然后Abaqus插入一个由80个空白组成的逻辑记录，这样可以正确处理文件结束。

每个"记录"的开始用星号（*）表示。每个浮点数以字符D开头，后跟以E22.15或D22.15格式的数字，这取决于写入结果文件的Abaqus版本使用的是单精度还是双精度。每个字符字符串以字符A开头，后跟八个字符（如果字符字符串少于八个字符，则字符串的右侧为空白；超过八个字符的字符字符串每次写入八个字符）。每个整数以字符I开头，后跟一个两位整数，表示整数中的十进制位数，然后是整数本身（以十进制数字写入）。

例如，对于元素编号为5、节点为195、198、205和204的S4R元素，记录键1900将写成

```
*I 18I 41900I 15AS4R     I 3195I 3198I 3205I 3204
```
对于节点135和6个自由度，记录键101将写成
```
*I 19I 3101I 3135D1.280271914214298E-10D1.500000000000036E+00
D-1.074629835784448E-46D 6.983222716550941E-12
D-4.084928798492785E-13D-1.072688441364597E-10
```

##### 结果文件中浮点数据的精度

写入结果文件的浮点数据的精度取决于生成数据的可执行文件的精度。Abaqus/Standard始终使用双精度；因此，浮点数据始终以双精度写入Abaqus/Standard结果文件。Abaqus/Explicit可以在大多数机器上以单精度或双精度运行；请参见["定义分析，" 6.1.2节](pt03ch06s01abo05.md)，了解Abaqus/Explicit可执行文件精度级别的详细信息。如果使用Abaqus/Explicit的双精度可执行文件，则浮点数据以双精度写入Abaqus/Explicit结果文件；同样，如果使用Abaqus/Explicit的单精度可执行文件，则浮点数据以单精度写入Abaqus/Explicit结果文件。

#### 最大化结果文件的效率

在Abaqus/Standard中，每个元素输出请求（在一行上输入的识别键集合）前面有一个"元素头"记录（请参见["结果文件输出格式，" 5.1.2节](pt02ch05s01afi01.md)）。因此，可以通过在同一行上输入所有相同"类型"的元素输出变量（元素积分点变量、元素截面变量、整体元素变量等）来最小化结果文件的大小。（请参见["输出到数据和结果文件，" 4.1.2节](pt02ch04s01aus39.md)，了解输出变量类型的说明。）鼓励合并输出变量条目，因为这将减小结果文件的大小。

例如，可以使用以下输出请求来请求在应力/位移分析的结果文件中输出元素变量：

```
[*EL FILE](../key/key-link.md#usb-kws-helfile), REBAR

```
（钢筋量的输出请求不必与底层元素输出请求相同。）

### 状态文件

状态文件（`*job-name*.sta`）是一个文本文件，包含有关分析进展的信息。

#### Abaqus/Standard或Abaqus/CFD状态文件

Abaqus/Standard或Abaqus/CFD状态文件为每个增量包含一个80字符的记录，并在分析每个增量完成后更新。此记录在增量完成后直接写入辅助存储。因此，状态文件可以在分析作业执行时进行检查，从而提供分析进展的监控。除了在Abaqus/Standard状态文件中指定要监控的自由度变量（如下所述）外，写入Abaqus/Standard或Abaqus/CFD状态文件的信息无法控制。

#### Abaqus/Explicit状态文件

在Abaqus/Explicit中，状态文件（`*job-name*.sta`）默认包含模型的质心和惯性属性、初始稳定时间增量信息、分析进展摘要（包括总累计CPU使用率和当前时间增量大小）以及处理每个步骤所需内存的估计。您可以控制其他输出，包括总动能、能量平衡、具有最小稳定时间增量的元素标识符以及由于质量缩放导致的模型总质量变化百分比。

写入Abaqus/Explicit状态文件摘要增量的频率取决于分析以CPU分钟为单位的持续时间和分析中指定的输出量。以下列表提供了有关何时将摘要增量写入状态文件的一般指导。

摘要信息通常会在以下情况下写入：
- 每次写入重启信息、输出数据库的场输出或结果文件输出时。
- 如果问题需要的增量少于20个，则每个增量一次。
- 对于短期分析（少于40个CPU分钟），在步骤中写入20次。
- 对于超过40个CPU分钟的分析，每2个CPU分钟写入一次。

自由度变量可以在分析运行时在状态文件中被监控。您还可以将其他诊断信息写入状态文件（请参见["显式动态分析，" 6.3.3节](pt03ch06s03at08.md)和["Abaqus/Explicit分析中的接触诊断，" 39.2.1节](pt09ch39s02aus185.md)，了解详细信息）。状态文件中的部分诊断信息（包括每个摘要增量的信息）也写入输出数据库，供Abaqus/CAE使用（有关更多信息，请参见["输出到输出数据库中的请求Abaqus/Standard和Abaqus/Explicit中的诊断信息，" 4.1.3节](pt02ch04s01aus40.md#usb-out-odboutput-diagnostic)）。

只能在打包Abaqus/Explicit数据或分析期间检测到的错误也会写入状态文件。

### Abaqus/CFD中的备用输出格式

默认情况下，当您在Abaqus/CFD中请求输出时，输出会发送到输出数据库文件。但是，您可以选择为场输出和历史输出选择备用文件格式。场输出可以发送到EXODUS-II格式的文件；历史输出可以发送到逗号分隔值（CSV）格式的文件。

您以与["请求输出到输出数据库"](pt02ch04s01aus38.md#usb-out-ooutput-requesting)中所述相同的方式请求场和历史输出。要选择备用输出格式，请在运行Abaqus/CFD分析时在命令行上设置**field**和**history**选项。有关更多信息，请参见["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行，" 3.2.2节](pt01ch03s02abx02.md)。

#### EXODUS-II格式的场输出

EXODUS-II格式被第三方后处理器广泛支持，适用于计算固体力学和计算流体动力学。此格式是二进制的、机器无关的，非常适合非结构化网格上的瞬态模拟结果。

EXODUS-II格式及其相关的EXODUS-II/NEMESIS编程API用于读取和写入，由Sandia National Laboratories开发。此开源软件可根据BSD License使用。源代码和文档可在[http://sourceforge.net/projects/exodusii](http://sourceforge.net/projects/exodusii)找到。

EXODUS-II格式本机无法表示所有Abaqus/CFD输出功能。表中列出的功能无法直接表示，会被省略或修改。

**表4.1.1-1** EXODUS-II格式中Abaqus/CFD输出功能的表示。
| 功能 | 注释 |
| --- | --- |
| 部件和装配体 | 节点和元素编号不包括部件实例名称，而是按顺序编号（请参见["定义装配体中的命名约定，" 2.10.1节](pt01ch02s10aus28.md#usb-int-ipartassy-naming)） |
| 元素集 | 不支持通用元素集，会被省略 |
| 振幅 | 不支持 |

EXODUS-II格式使用文件扩展名`exo`。对于分析运行的并行处理，EXODUS-II输出被定向到多个文件（每个处理器创建一个文件），这对某些第三方后处理器很有用。文件名为`*job*.exo.*rank*`，其中*rank*是从0到CPU数量减1的数字。相比之下，您可以将并行执行的场输出写入单个文件（`*job*.exo`）；该文件使用NEMESIS库以EXODUS-II格式写入。

#### CSV格式的历史输出

逗号分隔值（CSV）格式是一种基于文本的输出格式。CSV文本文件的格式由一行或多行注释组成，然后是每帧历史输出一行逗号分隔的数据。CSV文件中的注释以字符`#`开头。CSV文件中的每列都有一个注释，描述网格位置、部件实例和输出请求标签。网格位置的可能值为node、element或surface。向量输出请求还包括分量；即1、2或3。

此格式使用文件扩展名`csv`。CSV格式的历史输出为每个步骤的每个输出请求标签创建一个文件。如果作业以并行方式运行，并且与历史输出请求关联的集合由于域分解而在处理器之间拆分，则会创建其他文件。在这种情况下，每个存在该集合的处理器将有一个文件。文件名为`*job*_*output-request*_*rank*_*step-number*.csv`，其中*rank*是从0到CPU数量减1的数字。

| **输入文件用法：** | 使用以下命令行选项在Abaqus/CFD中写入历史输出到备用文件格式： |
| --- | --- |
|  | **abaqus** **job**=*job-name* **history**=`csv` |

| **Abaqus/CAE用法：** | 您无法在Abaqus/CAE中为历史输出选择备用格式。 |

### 多步骤中的请求输出

通常，输出请求适用于定义它们的步骤及其后的所有后续步骤，直到它们被重新指定。但是，线性扰动步骤（仅在Abaqus/Standard中可用；请参见下文和["一般和线性扰动程序，" 6.1.3节](pt03ch06s01aus44.md)）的输出规范与一般分析步骤的输出请求独立处理，仅适用于连续的线性扰动步骤序列。

数据库输出、打印输出和结果文件输出是Abaqus中独立的输出模式；因此，更改一种形式输出的规范不会影响其他形式。

#### 一般分析步骤

除非您重新定义它们，否则在分析的第一个一般分析步骤中使用默认输出请求。对于后续的一般分析步骤，除非您重新定义它们，否则保持从先前一般步骤的每种形式输出的定义。

#### 线性扰动步骤

除非在步骤中重新定义它们，否则在任意线性扰动步骤序列中的第一个步骤中使用默认输出请求。如果定义了后续线性扰动步骤而没有中间一般分析步骤，则保持从先前扰动步骤的每种模式输出的定义，除非您重新定义它们。如果定义了中间一般步骤，则再次在线性扰动步骤中使用默认输出请求，除非您在步骤中重新定义它们。

### Abaqus/Standard中的元素矩阵输出

在Abaqus/Standard中，您可以将元素刚度矩阵以及可用的质量矩阵写入每个步骤的文件。对于热传递元素，如果请求刚度矩阵输出，则写入算子矩阵。

元素矩阵输出仅适用于没有内部节点的元素（除非这些节点没有活动自由度）且没有声学或内部自由度。禁止元素矩阵输出的元素示例包括声学、管道、弯头、框架、间隙和界面元素，以及具有傅里叶模式的轴对称元素。元素矩阵输出不适用于具有耦合场的元素，如耦合温度-位移元素和孔隙压力元素。对于不兼容模式和混合元素，禁止刚度矩阵输出，而质量矩阵输出可用。子结构矩阵输出请求用于将子结构的缩减刚度矩阵、质量矩阵和载荷向量写入文件（请参见["定义子结构，" 10.1.2节](pt04ch10s01aus59.md)）。

元素矩阵输出无法在基于模式的动态分析（响应谱、稳态动态、模态动态或随机响应）中请求。但是，可以在特征值提取分析中请求该输出，以输出质量矩阵和刚度矩阵。

元素矩阵的写入不考虑节点条件的影响；因此，边界条件、集中载荷和多点约束的影响不包含在此输出中。自由度始终在全局方向上，即使在元素关联的节点处定义了局部坐标系（["变换坐标系，" 2.1.5节](pt01ch02s01aus09.md)）。

您必须选择要请求输出的元素集。对于用部件实例装配体定义的模型（["定义装配体，" 2.10.1节](pt01ch02s10aus28.md)），随元素矩阵输出写入的元素编号是由Abaqus/Standard生成的内部编号。内部编号与原始元素编号和部件实例名称之间的映射在数据文件中提供。

### 从Abaqus/Standard中的重启数据恢复其他结果输出

Abaqus/Standard中重启所需的数据包含在多个文件中，这些文件是在您请求为分析写入重启数据时生成的：重启（`.res`）、分析数据库（`.mdl`和`.stt`）、部件（`.prt`）和输出数据库（`.odb`）文件。["重启分析，" 9.1.1节](pt04ch09s01aus53.md)更详细地描述了重启数据的写入。

在Abaqus/Standard中，您可以从重启数据中提取输出，并使用后处理分析程序将其写入新的数据（`.dat`）、结果（`.fil`）和输出数据库（`.odb`）文件。如果原始分析包含用户子程序，则后处理分析程序需要指定用户子程序。数据、结果和输出数据库文件输出请求的定义方式与["输出到数据和结果文件，" 4.1.2节](pt02ch04s01aus39.md)和["输出到输出数据库，" 4.1.3节](pt02ch04s01aus40.md)中的描述相同。输出请求的定义应与分析中的定义完全相同，但以下情况除外：

1. 输出频率规范没有意义，因此被忽略（除非您从先前的直接循环或低周疲劳分析中恢复其他输出）。相反，您可以在后处理过程定义中指定要生成输出的每个增量。
2. 不提供输出数据库的默认输出。此外，模型信息（如边界条件）不会写入输出数据库。
3. 无法恢复元素集能量信息，因为它不会写入重启文件。
4. 对于不支持重启的程序，输出不可用；例如，线性扰动程序。

为分析定义的元素集和节点集可用于定义后处理过程中的输出集。也可以为后处理过程定义其他集合。您需要指定重启文件中需要输出的步骤编号。您无法在步骤开始时获取结果（见下文）。

| **输入文件用法：** | `[*POST OUTPUT](../key/key-link.md#usb-kws-hpostoutput), STEP=*step_number*` |

|  | 使用`[*POST OUTPUT](../key/key-link.md#usb-kws-hpostoutput)`选项时，它必须作为输入文件中的第一个选项出现。分析输入文件不需要数据行。可以根据需要多次重复此选项以获得更多输出。由于`[*POST OUTPUT](../key/key-link.md#usb-kws-hpostoutput)是一个纯粹的后处理过程，因此输入文件中不能出现分析选项。 |
