# 14.3.1 什么是步骤？





Abaqus/CAE模型使用以下两种类型的步骤：

**初始步骤**

Abaqus/CAE在模型步骤序列的开头创建一个特殊的初始步骤，并将其命名为`Initial`。Abaqus/CAE仅为你的模型创建一个初始步骤，并且不能重命名、编辑、替换、复制或删除。

初始步骤允许你定义适用于分析最开始时的边界条件、预定义场和交互。例如，如果边界条件或交互在整个分析过程中都适用，通常在这些条件中应用它会很方便。同样，当第一个分析步骤是线性扰动步骤时，在初始步骤中施加的条件构成扰动基础状态的一部分。

**分析步骤**

初始步骤之后是一个或多个分析步骤。每个分析步骤与一个特定过程相关联，该过程定义了在步骤期间要执行的的分析类型，如静态应力分析或瞬态热传递分析。你可以以任何有意义的方式更改步骤间的分析过程，因此在执行分析时具有很大的灵活性。由于模型的状态（应力、应变、温度等）在所有常规分析步骤中都会更新，因此先前历史的影响始终包含在每个新分析步骤的响应中。

你可以定义的分析步骤数量没有限制，但步骤序列存在限制。（更多信息，请参见["步骤序列限制，"第14.3.3节](pt03ch14s03s03.md)。）

你使用**Step**菜单中的项目来创建步骤，选择和定义步骤期间使用的分析过程，以及管理现有步骤。或者，你可以从主菜单栏选择****Step**![](../graphics/images/arrow.gif)**Manager****来显示**步骤管理器**。

例如，考虑以下管道系统截面的分析：

**初始步骤：**

施加边界条件以固定管道的左端，仅允许右端轴向移动。

**步骤1：压缩**

对管道右端施加压缩力。此步骤是常规分析步骤。

**步骤2：特征模态**

计算管道在其压缩状态下的振动频率和模态。此步骤是线性扰动步骤。

[图14-1](pt03ch14s03s01.md#sim-steplist)显示了你创建这些步骤后的**步骤管理器**。

**图14-1****步骤管理器**。

![](../graphics/sim-steplist-nls.png)

管理器列出了分析中的所有步骤以及关于每个步骤的一些突出细节。步骤2 `Eigenmodes`被缩进以显示它是基于步骤1 `Compress`结束时模型状态的线性扰动步骤。
