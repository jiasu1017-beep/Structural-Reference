# 19.6.5 在向模型添加更多分析步骤后重启





重启功能最常见的用途是分析模型，然后向模型添加一个或多个步骤并继续分析。本节描述此用法的一个示例。

假设您已完成以下操作：
- 创建了名为 `Model-A` 的模型，该模型有两个分析步骤，`Step-1` 和 `Step-2`。
- 使用步骤模块中的**编辑重启请求**对话框在每个步骤结束时输出重启信息。
- 创建了使用 `Model-A` 的作业 `Job-A`。
- 分析了模型。

在研究分析结果后，您决定向模型添加另一个步骤 `Step-3`。重启功能允许您计算 `Step-3` 的结果，而无需重复 `Step-1` 和 `Step-2` 的计算。以下步骤描述了建议遵循的过程：

1. 将 `Model-A` 复制到新模型（例如 `Model-A-restart`），并使 `Model-A-restart` 成为当前模型。
2. 向 `Model-A-restart` 添加新的 `Step-3`。
3. 在 `Step-3` 中添加新的规定条件（载荷、边界条件、相互作用、场或输出请求），或修改从 `Step-2` 传播的规定条件。
4. 从主菜单栏，选择 ****模型**![](../graphics/images/arrow.gif)**编辑属性**![](../graphics/images/arrow.gif)**Model-A-restart****。在出现的**编辑模型属性**对话框中，执行以下操作：- 输入 `Job-A` 作为将从中读取重启数据的作业。- 设置重启位置：- 输入 `Step-2` 以指示将从中读取数据的步骤。- 选择**从步骤结束时重启**。`Step-3` 将在 `Step-2` 结束时继续分析。
5. 将 `Job-A` 复制到使用 `Model-A-restart` 的新作业（例如 `Job-A-restart`）。
6. 从主菜单栏，选择 ****作业**![](../graphics/images/arrow.gif)**编辑**![](../graphics/images/arrow.gif)**Job-A-restart****。在出现的**编辑作业**对话框中，选择**重启**作为作业类型。
7. 提交 `Job-A-restart` 进行分析。
