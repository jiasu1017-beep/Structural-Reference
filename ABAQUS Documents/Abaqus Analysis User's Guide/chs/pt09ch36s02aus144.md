### 概述

Abaqus/Standard 中与通用接触算法关联的数值控制：
- 对于大多数问题，不应从其默认设置进行修改；
- 可用于默认设置未提供经济有效的解决方案的问题；
- 可用于控制主-从角色和滑动 formulation；以及
- 在某些情况下，可以选择性地应用于通用接触域内的特定区域。

### 接触 formulation

通用接触算法使用有限滑动、面-面接触 formulation，这在 ["Abaqus/Standard 中的接触 formulations，" 第 38.1.1 节](pt09ch38s01aus177.md) 中讨论。其他接触 formulations 不适用于 Abaqus/Standard 中的通用接触。

### 约束 enforcement 方法

通用接触算法默认使用惩罚方法 enforcement 主动接触约束。其他约束 enforcement 方法可以作为表面相互作用（即接触属性）定义的一部分进行指定，如 ["Abaqus/Standard 中的接触约束 enforcement 方法，" 第 38.1.2 节](pt09ch38s01aus178.md) 中所述。将接触属性分配给通用接触相互作用的内容在 ["Abaqus/Standard 中通用接触的接触属性，" 第 36.2.3 节](pt09ch36s02aus141.md) 中讨论。

### 摩擦的数值控制

与摩擦关联的数值控制在 ["摩擦行为，" 第 37.1.5 节](pt09ch37s01aus169.md) 中讨论。

### 梁-梁接触

梁-梁接触的激活在 ["在 Abaqus/Standard 中定义通用接触相互作用，" 第 36.2.1 节](pt09ch36s02aus139.md) 中讨论。

### 主从角色

通用接触使用的面-面接触 formulation 使用主-从方法生成单个接触约束，如 ["Abaqus/Standard 中的接触 formulations，" 第 38.1.1 节](pt09ch38s01aus177.md) 中所述。Abaqus/Standard 为通用接触域中分离体上的接触分配默认纯主-从角色。内部表面使用命名约定 `General_Contact_Faces_*k*` 自动生成，其中 `*k*` 对应于自动分配的组件号。默认情况下，编号较低的组件表面将作为编号较高组件表面的主表面。您可以通过在 Abaqus/CAE 的 Visualization 模块中查看自动生成的内部表面来确定默认纯主-从角色（见 [Abaqus/CAE 用户指南第 78 章，"使用显示组显示模型的子集"](../usi/usi-link.md#uss-dgp)）。体内自接触默认使用平衡主-从接触进行处理，每个表面节点在某些约束中作为主节点，在其他约束中作为从节点。

例如，如果通用接触域跨越三个分离体，则自动创建以下三个用于通用接触的内部"组件表面"：
- `General_Contact_Faces_1`
- `General_Contact_Faces_2`
- `General_Contact_Faces_3`

默认情况下，列出的第一个表面作为其他两个的主表面，而 `General_Contact_Faces_2` 作为 `General_Contact_Faces_3` 的主表面。这三个表面中每个表面的自接触默认使用平衡主-从接触建模。

#### 指定非默认主-从角色

您可以通过指定纯主-从角色或通过指定应使用平衡主-从接触来覆盖默认主-从角色。默认主-从处理在大多数情况下效果很好。在修改主-从分配时，除了本节讨论的其他因素外，请记住以下几点：
- 在分配替代主-从角色时，不要使用内部生成的组件表面（而是使用您定义的表面名称）。
- 主-从角色分配是模型定义的一部分，不能从步骤到步骤修改。
- ["在 Abaqus/Standard 中定义接触对，" 第 36.3.1 节](pt09ch36s03aus145.md#usb-cni-acontactpair-twosurf) 中讨论的为接触对分配纯主-从角色的指南也适用于您为通用接触重新分配纯主-从角色的情况。
- ["在 Abaqus/Standard 中定义接触对，" 第 36.3.1 节](pt09ch36s03aus145.md#usb-cni-acontactpair-symm) 中讨论的平衡（对称）主-从接触对的限制也适用于您为通用接触重新分配平衡主-从接触的情况。平衡主-从接触可能由于约束数量增加和过度约束的可能性而导致稳健性降低。

|  | ``` [*CONTACT FORMULATION](../key/key-link.md#usb-kws-hcontformulation), TYPE=MASTER SLAVE ROLES *surf_1*, *surf_2*, SLAVE ``` 使用以下选项指示第一个表面应被视为主表面： ``` [*CONTACT FORMULATION](../key/key-link.md#usb-kws-hcontformulation), TYPE=MASTER SLAVE ROLES *surf_1*, *surf_2*, MASTER ``` 如果省略第一个表面名称，则假定一个包含整个通用接触域的默认表面。必须指定第二个表面名称。使用以下选项指定两个表面之间应使用平衡主-从接触： ``` [*CONTACT FORMULATION](../key/key-link.md#usb-kws-hcontformulation), TYPE=MASTER SLAVE ROLES *surf_1*, *surf_2*, BALANCED ``` 如果省略第一个表面名称，则假定一个包含整个通用接触域的默认表面。如果省略第二个表面名称，则假定第一个表面与自身之间的接触。 |

##### 自动生成的接触排除

Abaqus/Standard 自动为与指定纯主-从角色相反的主-从角色生成接触排除；因此，对于两个表面重叠的区域，自接触被排除。例如，指定 `surf_A` 和 `surf_B` 之间的通用接触相互作用应使用纯主-从接触，并认为 `surf_A` 是从表面，这将在内部生成排除：`surf_A` 的主面接触 `surf_B` 的从面；`surf_A` 和 `surf_B` 重叠区域的自接触将被排除。如果省略第二个表面名称或与第一个表面名称相同，则会发出错误消息，因为此输入将导致表面的自接触被排除。

### 滑动时接触力重新分配的平滑度

您可以控制滑动时节点接触力重新分配的平滑度。默认设置（通常适当）会导致接触力重新分配的平滑度与从表面底层单元的阶次相同；即线性单元的线性重新分配平滑度，以及二阶单元的二次重新分配平滑度。二次重新分配平滑度通常倾向于改善收敛行为，并改善快速变化的接触应力区域内的接触应力分辨率。然而，二次重新分配平滑度倾向于增加每个约束涉及的节点数量，这可能增加方程求解器的计算成本。线性重新分配平滑度倾向于更好地解析主动接触区域边缘附近的接触应力，因此偶尔会产生更好的收敛行为。

| **输入文件用法：** | 使用以下选项指示滑动时接触力重新分配的平滑度应与从表面底层单元的阶次相同： |
| --- | --- |
|  | ``` [*CONTACT FORMULATION](../key/key-link.md#usb-kws-hcontformulation), TYPE=SLIDING TRANSITION *surf_1*, *surf_2*, ELEMENT ORDER SMOOTHING ``` 如果省略第一个表面名称，则假定一个包含整个通用接触域的默认表面。如果省略第二个表面名称，则假定第一个表面与自身之间的接触。使用以下选项指示滑动时接触力重新分配的线性平滑度： ``` [*CONTACT FORMULATION](../key/key-link.md#usb-kws-hcontformulation), TYPE=SLIDING TRANSITION *surf_1*, *surf_2*, LINEAR SMOOTHING ``` 如果省略第一个表面名称，则假定一个包含整个通用接触域的默认表面。如果省略第二个表面名称，则假定第一个表面与自身之间的接触。使用以下选项指示滑动时接触力重新分配的二次平滑度： ``` [*CONTACT FORMULATION](../key/key-link.md#usb-kws-hcontformulation), TYPE=SLIDING TRANSITION *surf_1*, *surf_2*, QUADRATIC SMOOTHING ``` 如果省略第一个表面名称，则假定一个包含整个通用接触域的默认表面。如果省略第二个表面名称，则假定第一个表面与自身之间的接触。 |

### 通用接触的其他全局数值控制

可以为通用接触从步骤到步骤全局修改一些额外的数值接触控制；您不能为通用接触域内的各个表面配对指定接触控制。您可以应用接触稳定化来解决模型中建立接触之前发生的刚体模式，并且可以调整 Abaqus/Standard 用于确定接触穿透和分离的容差；这两种技术都在 ["在 Abaqus/Standard 中调整接触控制，" 第 36.3.6 节](pt09ch36s03aus150.md) 中讨论。
