# 3.2.2 Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行





**产品：** Abaqus/Standard  Abaqus/Explicit  Abaqus/CFD

##### **参考文献**

- ["Abaqus执行过程：概述，" 第3.1.1节](pt01ch03s01abo02.md)

### 概述

Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD通过运行Abaqus执行过程执行。几个参数可以在命令行或环境文件中设置（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)）。或者，您可以使用方便的Abaqus/CAE用户界面从输入文件提交Abaqus分析并设置分析参数；请参阅["理解分析作业，" Abaqus/CAE用户指南第19.2节](../usi/usi-link.md#usi-ana-concepts)。

Abaqus对文件名强制执行字符限制。对于任何对文件的命令行引用，文件名（包括路径描述）的总长度不能超过256个字符。

### 命令行选项

#### 必需选项

**job**

此选项的值指定运行期间生成的所有文件的名称，以及在**continue**、**convert**和**recover**阶段读取的文件的名称。

如果命令行中省略了此选项，系统将提示您输入其值（仅使用["获取信息，" 第3.2.1节](pt01ch03s02abx01.md)中描述的信息选项时除外）。如果未提供**input**选项，过程将在当前目录中查找名为`*job-name*.inp`的输入文件。

#### 确定分析执行哪些阶段的互斥选项

所有选项都是顺序无关的。如果不存在这些选项，则假定为**analysis**选项。**convert**选项是互斥规则的例外：**convert**可以与除**datacheck**、**parametercheck**、**syntaxcheck**和**information**之外的任何选项一起出现。**convert**和**parametercheck**选项不适用于Abaqus/CFD。

**analysis**

此选项指示要执行完整的Abaqus分析（或重启Abaqus分析）。

**datacheck**

此选项指示运行仅用于数据检查。将不执行分析。如果使用此选项，将保存继续分析所需的所有文件。

**parametercheck**

此选项指示运行仅用于输入参数检查（必须已使用参数定义；请参阅["参数化输入，" 第1.4.1节](pt01ch01s04aus04.md)）。将不执行分析或数据检查。此选项不适用于Abaqus/CFD。

**continue**

此选项指示运行从先前数据检查运行结束的点开始。

**convert**

此参数的值指示哪些文件将被后处理。此选项不适用于Abaqus/CFD。

结果可以按以下方式转换：分析运行后立即转换，作为分析运行后的单独运行，或在分析运行时转换：

1. 要运行分析并随后转换结果，请将**convert**选项与**job**和**analysis**选项结合使用。
2. 要转换先前运行的分析的结果，请将**convert**选项与**job**选项结合使用。
3. 要转换当前正在运行的作业的结果，请将**convert**选项与**oldjob**选项（命名正在运行的作业）和**job**选项（为**convert**选项生成的文件提供新名称）结合使用。

如果**convert**=`select`，Abaqus/Explicit选择的结果文件（`*job-name*.sel`）将被转换为标准Abaqus结果文件（`*job-name*.fil`）。如果分析使用**parallel**=`domain`并行运行，单独的选择结果文件（`*job-name*.sel.*n*`）将在转换为标准Abaqus结果文件之前先转换为单个选择结果文件（`*job-name*.sel`）。

如果**convert**=`odb`，输出数据库（`*job-name*.odb`）将使用后处理计算器进行转换（请参阅["后处理计算器，" 第4.3.1节](pt02ch04s03aus42.md)）。仅当请求["后处理计算器，" 第4.3.1节](pt02ch04s03aus42.md)中列出的输出类型时，才需要此转换。

如果**convert**=`state`，如果分析使用**parallel**=`domain`并行运行，则单独的Abaqus/Explicit状态文件（`*job-name*.abq.*n*`）将被转换为单个Abaqus/Explicit状态文件（`*job-name*.abq`）。

如果**convert**=`all`，将执行所有适用的转换选项。

**recover**

此选项仅适用于Abaqus/Explicit。它指示将在状态文件中最后一个可用的步骤和增量处重新启动分析。此功能可用于在灾难性故障后重启，例如超出CPU限制或磁盘配额（请参阅["重启分析，" 第9.1.1节](pt04ch09s01aus53.md)）。如果原始分析使用**parallel**=`domain`并行运行，则必须使用**parallel**=`domain`和相同数量的处理器重新启动。

**syntaxcheck**

此选项指示运行仅用于检查输入文件的语法。此选项不使用任何许可证令牌。将不执行分析，且不能使用**continue**选项继续进行分析。仅生成数据（`.dat`）和输出数据库（`.odb`）文件以供查看。在Abaqus/Explicit分析中，输出数据库中的模型数据可能不完整。

**information**

此选项将当前生效的安装和环境信息写入屏幕或文件`*job-name*.log`。有关此选项每个值的输出信息，请参阅["获取信息，" 第3.2.1节](pt01ch03s02abx01.md)。如果将**information**选项与**analysis**选项结合使用，则作业必须在后台运行以将信息文本写入日志文件。

#### 分析模块的其他可用选项

**input**

此选项用于指定输入文件名，可以带或不带`.inp`扩展名（如果未提供扩展名，Abaqus将自动附加）。如果未提供此选项，过程将在当前目录中查找名为`*job-name*.inp`的输入文件。如果找不到`*job-name*.inp`，过程将提示输入输入文件名。

**user**

此选项指定包含要在分析中使用的用户子程序的源文件或目标文件的名称。用户例程的名称可以包含路径名，可以带或不带文件扩展名。Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD接受以C、C++或FORTRAN编写的用户子程序。

如果给定了扩展名，程序将根据文件类型采取适当的操作。如果文件名没有扩展名，程序将搜索C、C++或FORTRAN源文件。如果源文件不存在，则改为搜索目标文件。执行过程使用包含用户子程序的目标文件创建共享库，分析在执行期间使用该库。

如果需要经常使用相同的用户子程序，请考虑设置**usub_lib_dir**环境文件参数，并使用**abaqus make**执行过程创建包含用户子程序的共享库。这将避免每次需要时重新编译和/或重新链接用户子程序。如果分析调用的用户子程序包含在用户库中，则不需要**user**选项。如果指定了**user**选项，则不会使用**usub_lib_dir**环境文件参数指定的目录中的用户库。

当使用**double**选项运行Abaqus/Explicit分析时，**user**选项不能用于指定目标文件，因为Abaqus/Explicit双精度运行需要单精度和双精度对象。在这种情况下，您必须设置**usub_lib_dir**环境文件参数，并将单精度和双精度目标文件放在指定目录中；或者，您可以提供用户子程序源。

**oldjob**

此选项指定先前运行的文件名称，从该文件开始重新启动或后处理（Abaqus/Standard仅；请参阅["从Abaqus/Standard重启数据中恢复其他结果输出"中的"输出，" 第4.1.1节](pt02ch04s01aus38.md#usb-out-ooutput-postoutput)），或从中导入结果。不允许使用路径或文件扩展名。当重新启动、后处理、对称模型生成或导入分析从重启或结果文件读取数据时，需要此选项。*oldjob-name*必须与当前*job-name*不同。

**fil**

此选项指定是否将重新启动运行中指定的就结果文件的数据包含在新结果文件的开始处（默认）。如果使用**fil**=`new`，新结果文件将仅包含从重新启动发生的分析点到来的数据。此功能用于Abaqus/Standard运行，以将重新启动的分析输出连接成单个连续结果文件。非重启作业不能使用此功能将结果文件输出追加到旧结果文件；必须使用**abaqus append**执行过程来实现此目的。不允许为Abaqus/Explicit运行设置**fil**=`new`。此选项不适用于Abaqus/CFD。

**globalmodel**

此选项指定全局模型的结果文件或输出数据库文件的名称，从中获取结果以驱动子模型分析。每当子模型分析或子模型边界条件从全局模型的结果读取数据时，都需要此选项。文件扩展名是可选的。如果全局模型同时存在结果文件和输出数据库文件，且未给出扩展名，则将使用结果文件。此选项不适用于Abaqus/CFD。

**cpus**

此选项指定分析运行期间要使用的处理器数量（如果并行处理可用）。此参数的默认值是1，可以在环境文件中更改（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)）。

**parallel**

此选项指定用于Abaqus/Explicit中基于线程的并行处理的方法。可能的值是`domain`和`loop`。如果**parallel**=`domain`，则使用域级方法将模型分解为几何域。如果**parallel**=`loop`，则使用循环级方法并行化低级循环。有关这些方法的更多信息，请参阅["Abaqus/Explicit中的并行执行，" 第3.5.3节](pt01ch03s05aus34.md)。默认值是`domain`，可以在环境文件中更改（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)）。

**domains**

此选项指定Abaqus/Explicit中的并行域数量。如果值大于1，将执行域分解，无论**parallel**和**cpus**选项的值如何。但是，如果**parallel**=`domain`，则**cpus**的值必须能被**domains**的值整除。默认值设置为分析运行期间使用的处理器数量（如果**parallel**=`domain`），如果**parallel**=`loop`则为1。默认值可以在环境文件中更改（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)）。重新启动分析使用与原始分析相同数量的并行域，此选项指定的值将被忽略。

**dynamic_load_balancing**

对于Abaqus/Explicit中的域并行执行（**parallel**=`domain`），其中域数量大于CPU数量，此选项激活动态负载平衡方案。Abaqus/Explicit将尝试通过周期性重新分配域到处理器以最小化负载不平衡来提高计算效率（请参阅["Abaqus/Explicit中的并行执行，" 第3.5.3节](pt01ch03s05aus34.md)）。

**mp_mode**

如果此选项设置为`mpi`，则在适用时将使用基于MPI的并行化方法。设置**mp_mode**=`threads`以使用基于线程的并行化方法。在Windows平台上，如果安装了MPI组件，则默认值是`mpi`；否则，基于线程的并行执行是默认行为。在所有其他平台上，默认值是`mpi`。默认设置可以在环境文件中更改（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)）。对于Abaqus/CFD，只能使用**mp_mode**=`mpi`。

**standard_parallel**

此选项指定Abaqus/Standard中的并行执行模式。可能的值是`all`和`solver`。如果**standard_parallel**=`all`，则元素操作和求解器都将并行运行。如果**standard_parallel**=`solver`，则只有求解器将并行运行。默认值是**standard_parallel**=`all`，在支持基于MPI的并行化的平台上。

并行执行模式也可以在环境文件中设置（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)）。

**gpus**

此选项指定Abaqus/Standard直接求解器的加速。此选项仅在配备适当GPGPU硬件的计算机上才有意义。默认情况下，GPGPU求解器加速未激活。此参数的值是在Abaqus/Standard分析中使用的GPGPU数量。在基于MPI的分析中，这是每台主机上使用的GPGPU数量。

基于GPGPU的求解器加速也可以在环境文件中设置（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)）。

**memory**

输入文件预处理期间和Abaqus/Standard分析阶段可以分配的最大内存量或物理内存的最大百分比（请参阅["在Abaqus中管理内存和磁盘使用，" 第3.4.1节](pt01ch03s04aus31.md)）。默认值可以在环境文件中更改（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)）。此选项不适用于Abaqus/CFD。

**interactive**

此选项将导致作业交互运行。对于Abaqus/Standard和Abaqus/CFD，日志文件将输出到屏幕；对于Abaqus/Explicit，状态文件和日志文件将输出到屏幕。默认**run_mode**可以在环境文件中设置（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)）。

**background**

此选项将提交作业在后台运行，这是默认设置。日志文件输出将保存在当前目录中的文件`*job-name*.log`中。提交作业的默认方法可以通过使用**run_mode**参数在环境文件中设置（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)）。

**queue**

此选项将作业提交到批处理队列。如果选项出现时没有值，作业将提交到系统默认队列。允许使用带引号的字符串。可用队列是特定于站点的。请联系您的站点管理员以了解更多关于本地排队功能的信息。使用**information**=`local`查看已安装的本地排队功能。提交作业的默认方法可以通过使用**run_mode**参数在环境文件中设置（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)）。

**after**

此选项与**queue**选项结合使用，以指定作业将在所选批处理队列中开始的时间。通过Abaqus环境文件支持每个站点的此功能。（详细信息请参阅[Abaqus安装和许可指南](../sgb/sgb-link.md#sgb)。）

**double**

此选项用于指定要为Abaqus/Explicit使用双精度可执行文件。可能的值是`both`、`constraint`、`explicit`和`off`。此功能也通过Abaqus环境文件支持，环境变量为**double_precision**（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)）。

如果**double**=`both`，Abaqus/Explicit打包程序和分析都将以双精度运行。

如果**double**=`constraint`，Abaqus/Explicit约束打包和约束求解器将以双精度运行，而Abaqus/Explicit打包程序和Abaqus/Explicit分析继续以单精度运行。

如果**double**=`explicit`，Abaqus/Explicit分析将以双精度运行，而打包程序仍将以单精度运行。默认值是`explicit`。

如果**double**=`off`，则在必要时覆盖环境文件设置，以单精度调用Abaqus/Explicit打包程序和Abaqus/Explicit分析。有关何时使用双精度可执行文件的讨论，请参阅["定义分析，" 第6.1.2节](pt03ch06s01abo05.md)。

**scratch**

此选项用于指定用于临时文件的目录名称。在UNIX平台上，默认值是`$TMPDIR`环境变量的值，如果未定义`$TMPDIR`，则为`/tmp`。在Windows平台上，默认值是`%TEMP%`环境变量的值，如果未定义此变量，则为`\TEMP`。在分析期间，将在此目录下创建一个子目录来存放分析临时文件。此参数的默认值可以在环境文件中设置（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)）。

**output_precision**

此选项指定写入输出数据库文件（`*job-name*.odb`）的节点场输出精度。使用**output_precision**=`full`会导致Abaqus/Standard分析的双精度场输出。要获取Abaqus/Explicit分析的双精度场输出，除了使用**output_precision**=`full`外，还要使用**double**选项。节点历史输出仅以单精度提供。此选项不能与**recover**选项一起使用。

**resultsformat**

此选项指定结果的输出格式。如果**resultsformat**=`odb`，则仅以ODB格式写入输出。如果**resultsformat**=`sim`，则仅以SIM格式写入输出。如果**resultsformat**=`both`，则以ODB和SIM两种格式写入输出。默认值是`odb`。更多信息，请参阅["输出"中的"输出数据库"第4.1.1节](pt02ch04s01aus38.md#usb-out-ooutput-formats)。此选项不适用于Abaqus/CFD。

**field**

此选项指定Abaqus/CFD的场输出格式。如果**field**=`odb`，则以ODB格式写入场输出。如果**field**=`sim`，则以SIM格式写入场输出。如果**field**=`exodus`，则将场输出写入EXODUS-II格式的文件，每个处理器一个文件。要获取并行执行的单个文件，请使用**field**=`nemesis`；文件使用NEMESIS库以EXODUS-II格式写入。默认值是`odb`。更多信息，请参阅["输出"中的"Abaqus/CFD中的替代输出格式"第4.1.1节](pt02ch04s01aus38.md#usb-out-ooutput-alternate)。

**history**

此选项指定Abaqus/CFD的历史输出格式。如果**history**=`odb`，则以ODB格式写入历史输出。如果**history**=`sim`，则以SIM格式写入历史输出。如果**history**=`csv`，则以逗号分隔值格式写入历史输出文件。

默认值取决于**field**选项的设置。当**field**=`odb`时，默认值为**history**=`odb`。当**field**=`sim`时，默认值为**history**=`sim`。当**field**=`exodus`或`nemesis`时，默认值为**history**=`csv`。更多信息，请参阅["输出"中的"Abaqus/CFD中的替代输出格式"第4.1.1节](pt02ch04s01aus38.md#usb-out-ooutput-alternate)。

**port**

此选项用于指定使用直接耦合接口进行求解器之间协同仿真的TCP/UDP端口号，其中包括Abaqus与某些第三方分析程序之间的协同仿真。将**port**设置为用于连接的端口号。默认值是`48000`。Abaqus用于启动通信的默认端口号可以使用环境文件中的**cosimulation_port**参数设置（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)）。此选项与**host**选项结合使用。更多信息，请参阅["Abaqus执行过程：概述"中的"选择TCP/UDP端口号"第3.1.1节](pt01ch03s01abo02.md#usb-int-doverview-port)。

**host**

此选项用于指定使用直接耦合接口进行求解器之间协同仿真的主机名，其中包括Abaqus与某些第三方分析程序之间的协同仿真。此选项指定托管连接的主机名称。请参阅第三方程序文档以确定是否需要**host**选项。此选项与**port**选项结合使用。

**csedirector**

此选项用于指定使用SIMULIA协同仿真引擎进行协同仿真时的连接（例如，host:port）。**csedirector**条目标识SIMULIA协同仿真引擎director进程的监听端口的主机名和TCP/UDP端口号。

**timeout**

此选项用于指定使用直接耦合接口或SIMULIA协同仿真引擎建立协同仿真连接的超时值（以秒为单位）。如果Abaqus在指定时间内未收到来自耦合分析程序的任何通信，则终止。默认值是3600秒。使用直接耦合接口时，Abaqus使用的默认超时值可以使用环境文件中的**cosimulation_timeout**参数设置（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)）。

#### datacheck模块的其他可用选项

**unconnected_regions**

此选项用于请求Abaqus/Standard在分析输出数据库中为未连接区域创建单元集和节点集。设置**unconnected_regions**=`yes`以创建名为MESH COMPONENT *N*的单元集和节点集，其中*N*是组件编号。

### 示例

以下示例说明了**abaqus**执行过程的不同功能和能力。

#### 在Abaqus/Standard中运行分析

使用以下命令在后台运行名为"c8"的热传导分析：

```
abaqus analysis job=c8 background
```

以下命令将在后台运行作业c8，并将当前环境设置输出到日志文件：
```
abaqus analysis job=c8 information=environment background
```

对热传导分析c8的后续分析是"c10"，这是一个静态分析，使用c8的温度数据作为输入。温度数据从c8结果文件作为预定义场读取。执行过程扫描Abaqus/Standard输入文件以查找此类文件依赖关系。在本例中，过程将在当前目录中查找带有扩展名`.fil`的c8结果文件。结果文件标识符可以包含路径名（请参阅["输入语法规则，" 第1.2.1节](pt01ch01s02aus01.md)），执行过程将在指定目录中查找。在任何一种情况下，如果文件不存在，将发出错误消息。使用以下命令在"long"队列中运行作业c10：
```
abaqus analysis job=c10 queue=long
```

此作业作为"c11"重新启动，使用c10的最终结果作为蠕变分析的起点。使用以下命令在默认队列中运行此作业：
```
abaqus analysis job=c11 oldjob=c10 queue=
```

使用以下命令运行名为"draw_imp"的Abaqus/Standard分析，该分析从先前运行的名为"draw_exp"的Abaqus/Explicit分析导入结果：
```
abaqus analysis job=draw_imp oldjob=draw_exp
```

#### 在Abaqus/Explicit中运行分析

使用以下命令将名为"beam"的Abaqus/Explicit分析提交到默认队列：

```
abaqus analysis job=beam convert=all queue=
```

从以下命令系列可以获得等效的结果：
```
abaqus datacheck job=beam interactive
abaqus continue job=beam queue=
abaqus convert=all job=beam interactive
```

请注意，CPU密集型分析选项以批处理模式运行，而其他选项以交互方式运行。

#### 在Abaqus/CFD中运行分析

使用以下命令使用128个核心并行提交名为"cylinder"的Abaqus/CFD分析：

```
abaqus analysis job=cylinder cpus=128
```

#### 运行分析的不同阶段

使用以下命令对名为"parmodel"的输入文件执行参数检查运行：

```
abaqus job=parmodel parametercheck
```

使用以下命令对名为"parmodel"的输入文件执行数据检查运行（如果在此作业之后运行，则再次执行参数检查）：
```
abaqus job=parmodel datacheck
```

以下命令将继续先前的**datacheck**作业以执行分析：
```
abaqus job=parmodel continue
```
