### 概述

过约束意味着施加多个一致的或不一致的运动约束。许多模型的节点自由度存在过约束。这种过约束可能导致不准确的解或不相容。可能导致过约束的常见情况包括（但不限于）：
- 涉及边界条件或多点约束的接触从节点；
- 参与基于曲面的绑定约束的曲面边缘被包含在接触从曲面中或具有对称边界条件；以及
- 边界条件施加到已经参与耦合或刚体约束的节点。

Abaqus/Standard执行的过约束检查：
- 检查由以下组合引起的过约束：基础运动、边界条件、接触对、耦合约束、线性约束方程、独立于网格的点焊、多点约束、刚体约束和基于曲面的绑定约束；
- 检查由通过连接器单元、耦合单元、专用接触单元和具有不可压缩材料行为的单元引入的运动约束导致的过约束；
- 通过详细消息识别导致过约束的约束；
- 自动解析在模型预处理期间和Abaqus/Standard分析期间检测到的有限的一致过约束集；
- 使用方程求解器检测无法自动解析的过约束；以及
- 可以修改默认行为。

### 过约束：一般说明

一般来说，过约束一词指的是多个约束作用在同一自由度上。过约束分为*一致的*（如果所有约束彼此兼容）或*不一致的*（如果约束彼此不兼容）。一致的过约束也称为*冗余*约束，不一致的过约束也称为*冲突*约束。

在Abaqus/Standard中，以下类型的约束组合可能导致过约束：
- 边界条件或基础运动，
- 接触对，
- 耦合约束，
- 独立于网格的点焊，
- 多点约束或线性约束方程，
- 基于曲面的绑定约束，以及
- 刚体约束。

除这些约束外，以下单元施加运动约束，当彼此组合使用或与上述约束组合使用时，可能导致过约束：
- 连接器单元，
- 专用接触单元，以及
- 用于不可压缩材料响应的混合单元。

[图35.6.1-1](pt08ch35s06aus138.md#aoverconstr-gen-example)展示了一致过约束的说明。

**图35.6.1-1** 具有冗余约束的模型。

![](../graphics/aoverconstr-gen-example-nls.png)

上块由三个单独网格化的区域构建，它们使用基于曲面的绑定约束连接在一起。此块与下刚性块接触，该块通过指定刚体约束而变为刚性。刚性块的参考节点是固定的。上块左边缘使用对称边界条件，上块和下块之间定义了粗糙摩擦的表面相互作用。可以识别以下冗余约束：
- 相交绑定约束：在(A)处，三个节点共享相同位置，它们的相对运动由两个基于曲面的绑定约束约束（一个垂直，一个水平）。只需要两个约束（两个从属节点和一个独立节点）就可以完全约束三个节点的运动，但内部生成三个约束（水平绑定约束一个，垂直绑定约束两个）。因此，存在一个冗余约束。
- 绑定约束和对称边界条件：在(B)处，节点141和151的运动在水平方向上由对称边界条件约束，但它们的相对运动也由基于曲面的绑定约束约束。因此，存在一个冗余约束。
- 粗糙摩擦和对称边界条件：在(C)处，节点101在水平方向上由对称边界条件约束。粗糙摩擦接触在与边界条件相同的方向上作用。因此，存在一个冗余约束。
- 绑定约束和接触相互作用：在(D)处，节点801和301参与基于曲面的绑定约束，但两个接触约束（每个节点一个）在垂直方向上作用。因此，存在一个冗余约束。

即使在这个简单的模型中，冗余约束的数量也多得惊人。如果不正确处理，冗余约束可能导致收敛困难，甚至不收敛。此外，在即使存在收敛困难也能获得解的情况下，报告的反力和接触压力可能不准确。

Abaqus/Standard检查本节列出的多数约束和单元类型的组合约束使用是否适当。根据所涉及约束的复杂性，Abaqus/Standard识别三类一致和不一致的过约束。

**在模型预处理器中检测到的过约束**

许多相对简单的过约束可以通过检查在节点处定义的约束来识别。如果检测到一致的过约束，则自动消除不必要的约束并生成警告消息。如果过约束不一致，则停止分析并生成错误消息。

**在Abaqus/Standard分析中检测和解析的过约束**

某些涉及接触相互作用的过约束可能仅由于接触状态的变化而在分析过程中变得过约束。某些这种情况可以被Abaqus/Standard检测到并自动消除。发出适当的消息。

**由方程求解器检测到的过约束**

许多过约束涉及各种约束定义和单元类型之间的复杂相互作用。这些情况的自动解析可能不可能。在这种情况下，方程求解器将检测到过约束，并将发出列出问题潜在原因的详细消息。

### 在模型预处理器中检测到的过约束

在本节中，我们考虑涉及以下两个或多个的过约束：
- 基于曲面的绑定约束，
- 刚体约束，
- 边界条件，以及
- 连接器单元。

虽然在模型预处理器中自动处理的案例数量有限，但许多经常遇到的情况会得到纠正。在预处理器中自动解析的过约束列表根据所涉及的约束类型组织。每个案例都用示例说明。

#### 相交绑定约束

[图35.6.1-2](pt08ch35s06aus138.md#aoverconstr-intersecting-ties)显示相交绑定约束定义的示例。

**图35.6.1-2** 由相交绑定约束引起的一致过约束。

![](../graphics/aoverconstr-intersecting-ties-nls.png)

在这两种情况下，至少有一个节点（如果不正确处理）将被冗余约束。在左侧的情况下，属于三个曲面的三个边缘重叠（为清晰起见，此处显示为爆炸视图）。每端的三个端节点占据相同位置。因此，存在一个冗余绑定约束。在右侧的情况下，四个相邻网格使用四个绑定约束"粘合"在一起。只需要三个约束来"粘合"中心节点，但生成了四个（每个绑定约束一个）。因此，在两种情况下都不需要一个约束，并且移除一个约束。

**图35.6.1-3** 由绑定和刚体约束组合引起的一致过约束。

![](../graphics/aoverconstr-tie-rigbody-nls.png)

#### 两个刚体之间的绑定约束

[图35.6.1-3](pt08ch35s06aus138.md#aoverconstr-tie-rigbody)(b)显示两个刚体之间绑定约束的示例。如果两个曲面在两个或三个点以上（二维或三维分析）由绑定约束连接，绑定约束定义是冗余的。在两个参考节点之间放置连接器类型BEAM，并移除绑定约束。

#### 可变形体和刚体之间的绑定约束

[图35.6.1-3](pt08ch35s06aus138.md#aoverconstr-tie-rigbody)(c)显示通过基于曲面的绑定约束将可变形体连接到刚体的示例。如果绑定约束定义中的从属曲面属于刚体，则绑定和刚体约束对于从属节点是冗余的。如果可能，Abaqus/Standard将在绑定约束定义中切换主从曲面。如果由于其他建模限制无法切换主从曲面，则发出错误消息并停止分析。

#### 相交刚体

[图35.6.1-4](pt08ch35s06aus138.md#aoverconstr-rigbod-in-rigbod)(a)说明两个刚体部分重叠的情况，因此两个体的并集作为一个刚体行为。然而，该区域中节点的运动由两个刚体参考节点的运动控制；因此，模型是过约束的。在[图35.6.1-4](pt08ch35s06aus138.md#aoverconstr-rigbod-in-rigbod)(b)中，几个刚体包含在一个更大的刚体定义中。属于包含体的节点将过约束。

**图35.6.1-4** 包含其他刚体的刚体。

![](../graphics/aoverconstr-rigbod-in-rigbod-nls.png)

在这两种情况下，刚体约束将仅对属于多个刚体的节点执行一次。为了强制执行集合的刚体行为，在刚体参考节点之间生成类型BEAM的连接器单元，以确保相交刚体定义之间的刚性连接。

#### 绑定约束和边界条件

当基于曲面的绑定约束和边界条件一起使用时，有许多过约束情况，如图35.6.1-5[所示](pt08ch35s06aus138.md#aoverconstr-tie-and-boundary)。

**图35.6.1-5** 涉及绑定约束和边界条件的过约束。

![](../graphics/aoverconstr-tie-and-boundary-nls.png)

在第一种情况下，节点A和B由绑定约束约束一起移动。垂直对称边界条件将约束两个节点在水平方向的运动，产生一个冗余约束。在第二种情况下，两个指定的边界条件冲突，从而产生冲突约束。

对于每个具有边界条件的绑定从属节点，Abaqus/Standard首先确定绑定约束涉及哪些独立节点（参见["网格绑定约束，" 第35.3.1节](pt08ch35s03aus132.md)）。如果只涉及一个独立节点，Abaqus/Standard将把边界条件从从属节点转移到独立节点。如果在转移过程中在独立节点处检测到冲突的边界条件，则停止分析并发出错误消息。如果涉及多个独立节点，Abaqus/Standard检查约束中涉及的所有节点处指定的边界条件是否相同。如果没有识别到冲突，则独立节点处的边界条件是冗余的，因此被忽略。否则，发出错误消息并停止分析。

#### 刚体约束和边界条件

刚体约束和边界条件的组合可能导致过约束模型，当边界条件在除参考节点以外的节点处指定时（[图35.6.1-6](pt08ch35s06aus138.md#aoverconstr-rigbody-and-boundary)）。在[图35.6.1-6](pt08ch35s06aus138.md#aoverconstr-rigbody-and-boundary)(a)中，边界条件在属于刚体的多个节点处指定。在[图35.6.1-6](pt08ch35s06aus138.md#aoverconstr-rigbody-and-boundary)(b)中，对称边界条件在刚体的平坦表面上指定，物体围绕垂直于对称平面的轴在参考节点处旋转。

**图35.6.1-6** 由施加在刚体节点上的边界条件引起的过约束。

![](../graphics/aoverconstr-rigbody-and-boundary-nls.png)

在情况(a)中，如果指定的边界条件与刚性约束不一致，模型将不一致地过约束。在情况(b)中，如果参考节点具有对称边界条件，则不需要在平坦表面节点处有对称边界条件。Abaqus/Standard将尝试移除在从属节点处指定的所有边界条件，并在参考节点处重新定义它们。为此，检查在从属节点处指定的边界条件的一致性。如果边界条件不相同，则发出错误消息并停止分析（因为否则需要非线性方程组的求解来评估边界条件是否一致）。否则，Abaqus/Standard将尝试将通过属节点处的边界条件与参考节点处的边界条件合并，方法为：
- 检查重叠边界条件的一致性；
- 将从属节点处指定的但未在参考节点处指定的边界条件移到参考节点处；以及
- 在参考节点处应用额外的零旋转边界条件，以补偿从从属节点移除的位移约束。

举例说明，参见[图35.6.1-6](pt08ch35s06aus138.md#aoverconstr-rigbody-and-boundary)(b)：由于在从属节点处指定的对称边界条件彼此一致，因此从从属节点移除并应用到参考节点（2方向的边界条件）。此外，对称约束排除了关于1和3方向的旋转；因此，在这些轴上对参考节点施加零旋转边界条件。

#### 连接器单元和刚体

在大多数情况下，涉及连接器单元的冗余约束的检测和自动解析不能通过简单检查所涉及的约束来完成。但是，[图35.6.1-7](pt08ch35s06aus138.md#aoverconstr-connect-rigbody)中所示的示例足够简单，可以自动解析。假设连接器单元连接到其旋转自由度依赖于参考节点旋转的刚体上的节点。在[图35.6.1-7](pt08ch35s06aus138.md#aoverconstr-connect-rigbody)(a)中，假设连接器单元强制执行某些运动约束。它们是冗余的，因为刚体定义将所有节点的运动约束到刚体参考节点的运动。Abaqus/Standard自动从模型中移除连接器单元。

**图35.6.1-7** 涉及刚体和连接器单元的冗余约束。

![](../graphics/aoverconstr-connect-rigbody-nls.png)

当连接器单元放置在两个刚体之间时（如图[35.6.1-7](pt08ch35s06aus138.md#aoverconstr-connect-rigbody)(b)所示），模型可能冗余地约束。如[图35.6.1-7](pt08ch35s06aus138.md#aoverconstr-connect-rigbody)(b)所示，如果在两个刚体之间放置类型BEAM（或WELD）的连接器单元，则连接是刚性的，两个刚体之间的任何其他连接器单元都是冗余的。Abaqus/Standard将自动移除这些冗余连接器单元。

当放置在两个刚体之间的连接器单元集合强制执行超过两个刚体之间必要的平移和旋转约束，但连接器都不是类型BEAM（或WELD）时，仅发出警告消息以发出过约束情况信号。在这些情况下，由于两个刚体之间的连接可能会变得欠约束，因此无法自动消除任何连接器单元。举例说明，假设在[图35.6.1-7](pt08ch35s06aus138.md#aoverconstr-connect-rigbody)(b)中，两个连接器都是类型SLOT和TRANSLATOR。因此，在两个刚体之间强制执行四个平移约束（在三维中），渲染系统过约束，因为只需要三个平移约束就可以完全约束两个体之间的相对平移。但是，如果从模型中消除SLOT，模型将变得欠约束且与原始模型不同。仅在此情况下发出警告消息。

#### 耦合约束和刚体

当运动学耦合约束中涉及的所有或部分节点属于同一刚体时，耦合约束变得冗余。情况如图[35.6.1-8](pt08ch35s06aus138.md#aoverconstr-coupling-rigbody)所示。节点101是涉及节点1001-1005的耦合约束的参考节点。同时，节点1001-1003包含在以参考节点102的刚体定义中。

**图35.6.1-8** 涉及耦合约束和刚体的冗余约束。

![](../graphics/aoverconstr-coupling-rigbody-nls.png)

如果耦合约束被定义为运动学的，它将不会在节点1001-1003处强制执行，以避免模型过约束。移除的过约束可能不一致，例如当两个参考节点处规定了不相容的边界条件时。但是，约束将在节点1004和1005处强制执行，因为这些节点不属于刚体。

如果使用分布耦合约束，则模型不会过约束。但是，如果将节点101添加到刚体定义中而节点1004和1005未包含在耦合约束中，则模型将过约束。实际上，耦合约束中涉及的所有节点已经被刚体定义约束，使耦合约束冗余。为了避免过约束，Abaqus/Standard在这种情况下将不强制执行耦合约束。

#### 耦合约束和边界条件

当在分布耦合约束中涉及的所有节点上指定边界条件时，模型可能变得过约束。Abaqus/Standard将发出警告消息，概述潜在过约束的原因。

#### 点焊和刚体

["独立于网格的紧固件，" 第35.3.4节](pt08ch35s03aus135.md)中讨论了刚体参与独立于网格的点焊定义时可能出现的潜在过约束。

### 在分析期间检测和解析的过约束

有许多情况是接触相互作用与其他约束类型组合可能导致过约束。由于接触状态通常在分析期间发生变化，因此无法在模型预处理器中检测到与接触相关的冗余约束。相反，这些检查在分析期间执行。由于与接触相互作用相关的复杂性，只能自动解析有限数量的冗余约束情况。

#### 接触相互作用和绑定约束

当从属节点用于基于曲面的绑定约束（["网格绑定约束，" 第35.3.1节](pt08ch35s03aus132.md)）也是接触中的从属节点时，冗余约束是常见的。如[图35.6.1-9](pt08ch35s06aus138.md#aoverconstr-contact-and-tie)所示。在[图35.6.1-9](pt08ch35s06aus138.md#aoverconstr-contact-and-tie)(a)中，节点5和9由绑定约束连接，两者都与主曲面接触。由于两个节点绑定在一起，其中一个接触约束是冗余的。[图35.6.1-9](pt08ch35s06aus138.md#aoverconstr-contact-and-tie)(b)给出了类似的情况：两个不匹配的实体网格由绑定约束连接，并与平面刚性曲面定义接触。节点S是绑定约束中的从属节点，因此其运动由节点B和C的运动决定。因此，施加在节点S处的任何接触约束都是冗余的。此外，节点G和H处的接触约束是冗余的，因为这些节点的运动分别由节点B和C决定。

**图35.6.1-9** 由接触相互作用和绑定约束引起的冗余约束。

![](../graphics/aoverconstr-contact-and-tie-nls.png)

为了在绑定约束中涉及的所有节点处于接触时消除这些冗余，Abaqus/Standard将自动应用绑定型约束在绑定约束关联的拉格朗日乘数之间。冗余接触约束被消除。从相关绑定独立节点处的压力和摩擦力恢复从属节点处的接触压力和摩擦力。

##### 删除接触单元以移除过约束

您可以应用约束控制删除与绑定从属节点关联的接触单元，而不是让Abaqus通过绑定拉格朗日乘数来移除过约束。如果您使用此技术，则无法获得绑定从属节点的接触相关输出。

| **输入文件用法：** | ``` [*CONSTRAINT CONTROLS](../key/key-link.md#usb-kws-hconstraintcontrols), DELETE SLAVE ``` |

#### 接触相互作用和规定边界条件

如果调用具有默认"硬接触"公式（["接触压力-闭合关系，" 第37.1.2节](pt09ch37s01aus166.md)）的法向接触或具有拉格朗日乘数公式的摩擦接触（参见["摩擦行为，" 第37.1.5节](pt09ch37s01aus169.md)），接触相互作用和规定边界条件可能导致冗余约束。Abaqus/Standard尝试解析涉及刚性曲面的接触对的这些类型的冗余约束。

##### 与法向接触相互作用相关的检查

在[图35.6.1-10](pt08ch35s06aus138.md#aoverconstr-contact-normal-bc)中，固定的解析刚性主曲面与在垂直于接触表面的方向上具有固定边界条件的从属节点接触。如果在分析的特定增量期间节点处于接触状态，则接触约束是冗余的，在该增量期间不会被强制执行。如果从属节点处的边界条件与刚性曲面参考节点处的边界条件冲突，则发出错误消息并停止分析。

**图35.6.1-10** 涉及法向接触相互作用和边界条件的过约束。

![](../graphics/aoverconstr-contact-normal-bc-nls.png)

只有当主曲面定义为解析刚性曲面时，才会自动移除与过约束相关的接触和边界条件。在所有其他情况下，如果在分析期间发生过约束，则方程求解器（见下文）发出零枢轴消息，并清楚概述导致过约束的约束链。

##### 与拉格朗日摩擦相关的检查

[图35.6.1-11](pt08ch35s06aus138.md#aoverconstr-lagfric-boundary)描述了一个常见的冗余约束情况。对称边界条件与拉格朗日摩擦组合是冗余的。

**图35.6.1-11** 拉格朗日摩擦和边界条件。

![](../graphics/aoverconstr-lagfric-boundary-nls.png)

从属节点处于接触状态，曲面的切线方向与从属节点处规定的边界条件方向大致相同。为避免冗余，在该节点处，Abaqus/Standard将从拉格朗日摩擦公式切换到默认的惩罚公式（["摩擦行为，" 第37.1.5节](pt09ch37s01aus169.md)）（如果主节点在切线方向上的运动被规定）。

### 在方程求解器中检测到的过约束

所有无法在预处理或分析期间识别和解析的过约束都需要由方程求解器检测。示例包括以下模型：从属节点被规定边界条件驱动进入部分固定的刚性曲面的接触相互作用；与多个主曲面的接触；刚体由连接器单元连接的闭环和多环机构；以及更多。默认情况下，方程求解器过约束检查在分析期间持续执行。

Abaqus/Standard不会解析由方程求解器检测到的过约束。相反，将发出包含涉及过约束的运动约束信息的详细消息。该消息首先通过求解器中高斯消去的零枢轴信息识别涉及一致或不一致过约束的节点（["直接线性方程求解器，" 第6.1.5节](pt03ch06s01aus46.md)）。然后发出包含约束信息的详细消息。

[图35.6.1-12](pt08ch35s06aus138.md#aoverconstr-hard-to-detect)中所示的4杆机构说明了此策略。

**图35.6.1-12** 难以检测的冗余约束。

![](../graphics/aoverconstr-hard-to-detect-nls.png)

四个三维刚体定义如下：具有参考节点10001的刚体包含节点2和101；具有参考节点10002的刚体包含节点3和102；具有参考节点10003的刚体包含节点4和103；具有参考节点10004的刚体包含节点1和104。四个刚体与四个JOIN和REVOLUTE组合连接器单元连接，定义如下：单元20001在节点1和101之间；单元20002在节点2和102之间；单元20003在节点3和103之间；单元20004在节点4和104之间。每个连接器单元强制执行三个平移和两个旋转约束（["连接器：概述，" 第31.1.1节](pt06ch31s01abo28.md)），所有四个铰链轴方向平行。底部刚体（参考节点10004）是固定的。底部左侧REVOLUTE连接器（单元20001）的运动被规定为使机构旋转。

当Abaqus/Standard尝试为此模型求解时，在分析的第一个增量中识别出三个零枢轴，表明模型中有三个约束过多。最终，必须移除三个约束才能使模型正确约束。在这个简单的示例中，自由度和约束计数确认了过约束的数量，如下所示。模型中有四个刚体，共24个自由度。参考节点10004通过边界条件完全固定，约束六个自由度；规定的连接器运动强制执行一个旋转约束，约束一个自由度。因此，剩余17个自由度。四个连接器单元中的每一个都强制执行五个约束，共20个约束。因此，模型中有三个约束过多，与方程求解器识别的零枢轴数量相匹配。为了帮助您识别应移除的约束，在消息（`.msg`）文件中生成以下消息，概述生成过约束的约束链：

```
 ***WARNING: SOLVER PROBLEM. ZERO PIVOT WHEN PROCESSING ELEMENT 20004
             INTERNAL NODE 1 D.O.F. 4     

OVERCONSTRAINT CHECKS:  An overconstraint was detected at one of the 
Lagrange multipliers associated with element 20004. There are 
multiple constraints applied directly or chained constraints that 
are applied indirectly to this element. The following is a list of 
nodes and chained constraints between these nodes that most likely 
lead to the detected overconstraint.

LAGRANGE MULTIPLIER: 4 <-> 104: connector element 20004 type 
                     JOIN REVOLUTE constraining 3 translations  
                     and  2 rotations   
..4 -> 10003: *RIGID BODY (or *COUPLING-KINEMATIC)  
....10003 -> 103: *RIGID BODY (or *COUPLING-KINEMATIC)  
......103 -> 3: connector element 20003 type JOIN REVOLUTE 
                constraining 3 translations  and  2 rotations   
........3 -> 10002: *RIGID BODY (or *COUPLING-KINEMATIC)  
..........10002 -> 102: *RIGID BODY (or *COUPLING-KINEMATIC)  
............102 -> 2: connector element 20002 type JOIN REVOLUTE 
                      constraining 3 translations and 2 rotations   
..............2 -> 10001: *RIGID BODY (or *COUPLING-KINEMATIC)  
................10001 -> 101: *RIGID BODY (or *COUPLING-KINEMATIC)  
..................101 -> 1: connector element 20001 type 
                            JOIN REVOLUTE constraining  3 
                            translations  and 2 rotations
....................1 -> 10004: *RIGID BODY (or *COUPLING-KINEMATIC)
......................10004 -> *BOUNDARY in degrees of freedom  
                                1  2  3  4  5  6   
......................10004 -> 104: *RIGID BODY 
                                    (or *COUPLING-KINEMATIC) 
....................1 -> 101: connector element 20001 with 
                              *CONNECTOR MOTION in components  4 

 Please analyze these constraint loops and remove unnecessary 
 constraints.
```

首先，消息识别出用户定义的或在这种情况下内部定义的（拉格朗日乘数）节点，在该节点处识别出零枢轴。例如，此输出中典型的一行发布与一个约束相关的信息。例如，输出中的第一行
```
LAGRANGE MULTIPLIER: 4 <-> 104: connector element 20004 type 
                     JOIN REVOLUTE constraining 3 translations 
                     and  2 rotations   
```

通知您零枢轴发生的拉格朗日乘数强制执行与用户定义节点4和104之间的连接器单元20004相关的五个约束（JOIN和REVOLUTE）之一。后续每一行传达与约束链中一个约束相关的信息，这些约束链起源于零枢轴节点或与它们相邻的链。例如，行
```
....10003 -> 103: *RIGID BODY (or *COUPLING - KINEMATIC)
```

通知您在节点10003和103之间存在刚体约束，而行
```
.....................10004 -> *BOUNDARY in degrees of freedom  
                               1  2  3  4  5  6   
```

指出在节点10004处存在边界条件约束，固定自由度1到6。

缩进级别（节点编号前的点）识别约束链中的链接。每次找到约束链接到特定链中的另一个节点时，缩进增加两个点，并打印约束信息。例如，从消息顶部开始，拉格朗日乘数连接到节点4，节点4连接到节点10003，节点10003连接到节点103，依此类推。当某行的缩进小于或等于上一行的缩进时，约束链在上一行结束。例如，一个链在行
```
.....................10004 -> *BOUNDARY in degrees of freedom  
                               1  2  3  4  5  6   
```
结束，因为下一行具有相同的缩进。

可以识别三个约束链（与找到的三个零枢轴对应），它们最有可能在上述模型中产生过约束。从顶部开始，首先可以识别出一个终止于边界条件（接地）的约束链：

```
Lagrange multiplier: 4 –> 10003 –> 103 –> 3 –> 10002 –> 2 –> 
10001 –> 101 –> 1 –> 10004 –> *BOUNDARY
```

由于以节点10004开头的两行具有相同的缩进，因此应该预期另一个约束链包含这两行中的约束信息。实际上，可以识别出一个闭环约束：
```
Lagrange multiplier : 4–> 10003 –> 103 –> 3 –> 10002 –> 2 –> 
10001 –> 101 –> 1 –> 10004 –> 104 <-> 4 
```

最后，由于以节点1开头的两行具有相同的缩进，因此预期一个单独的约束链将包含输出中的最后一行。识别出第三个（闭环）
```
101 –> 1 –> 101
```

。

如果约束链终止于空闲端（不是约束结束），则该链对产生过约束没有任何贡献。在此示例中没有此类链。

#### 纠正过约束模型

自动生成包含与特定零枢轴关联的约束链中所有节点的节点集，可以在Abaqus/CAE的Visualization模块中显示。

没有独特的方法来移除此模型中的过约束。例如，如果将一个JOIN和REVOLUTE（五约束）组合替换为SLOT连接器单元（仅强制执行平面内两个平移约束），则不存在冗余。或者，您可以移除一个连接器单元中的REVOLUTE，并在另一个连接器单元中使用SLOT连接代替JOIN。

另一种选择是放宽一些约束。在此处概述的示例中，可以用弹性体替换一个或多个刚体。您还可以使用具有适当弹性刚度的CARTESIAN和CARDAN连接类型（参见["连接器行为，" 第31.2.1节](pt06ch31s02alm27.md)）来放宽基于拉格朗日乘数的约束（如JOIN或REVOLUTE）。

在分析约束链后，您必须决定必须移除哪些约束以使模型正确约束，并且最适合建模目标。对于此示例，不能随机移除三个约束。例如，移除六个边界条件中任意三个的组合会使问题变得更糟：模型仍然过约束，并且向模型添加了三个刚体模式。此外，您应该移除不影响模型运动学的约束。例如，您不能完全移除任何连接器单元的JOIN连接，因为模型将与最初设计的不同。

### 控制过约束检查

默认情况下，Abaqus/Standard将尝试移除尽可能多的冗余约束，如上节所述。当无法移除冗余约束或检测到不一致的过约束时，会发出详细消息，识别导致过约束的约束。您可以通过为模型或步骤规定约束控制来修改此默认行为。

过约束可能产生有害的和不可预测的行为。因此，强烈建议在模型预处理和分析期间都使用过约束检查，至少在首次运行模型时。此外，建议更改原始模型以纠正Abaqus/Standard识别的任何过约束。只有在建立模型没有过约束的信心后，才应关闭约束检查。关闭约束检查的唯一优点是分析速度稍有提高。

#### 绕过过约束检查

可以在输入文件预处理期间和分析期间执行的过约束检查被绕过。不建议绕过这些检查，因为它可能允许具有过约束的模型进入分析代码。绕过过约束检查不是步骤相关的；即，该设置定义为模型数据并影响整个分析。

| **输入文件用法：** | ``` [*CONSTRAINT CONTROLS](../key/key-link.md#usb-kws-hconstraintcontrols), NO CHECKS ``` |
| --- | --- |

#### 防止自动冗余约束解析

可以阻止模型预处理器中的自动模型修改。在这种情况下，Abaqus/Standard仍将执行过约束检查，但不会执行自动冗余约束解析；仅会发出适当的错误消息。防止约束解析不是步骤相关的；即，该设置定义为模型数据并影响整个分析。

| **输入文件用法：** | ``` [*CONSTRAINT CONTROLS](../key/key-link.md#usb-kws-hconstraintcontrols), NO CHANGES ``` |
| --- | --- |

#### 更改过约束检查的频率

默认情况下，过约束检查在分析的每个增量执行。您可以为分析中的每个步骤修改这些检查的频率（以增量为单位）。如果频率设置为零，则在该分析步骤期间不执行过约束检查。频率规格在后续步骤中保持，直到重置。

| **输入文件用法：** | ``` [*CONSTRAINT CONTROLS](../key/key-link.md#usb-kws-hconstraintcontrols), CHECK FREQUENCY=*n* ``` |
| --- | --- |

#### 检测到过约束时停止分析

默认情况下，即使检测到过约束，分析也会继续。可以更改此行为，但这是基于步骤的。可以设置为在步骤中首次检测到过约束时停止分析，或者仅在尽管存在过约束但获得收敛解时停止。此设置在后续步骤中保持，直到重置。

| **输入文件用法：** | 使用以下选项之一： |
| --- | --- |
|  | ``` [*CONSTRAINT CONTROLS](../key/key-link.md#usb-kws-hconstraintcontrols), TERMINATE ANALYSIS=FIRST OCCURRENCE [*CONSTRAINT CONTROLS](../key/key-link.md#usb-kws-hconstraintcontrols), TERMINATE ANALYSIS=CONVERGED ``` |
