# 18.1.1 用户子程序：概述





##### **参考文献**

- ["Abaqus/Standard、Abaqus/Explicit 和 Abaqus/CFD 执行," 第 3.2.2 节](pt01ch03s02abx02.md)
- [Abaqus 用户子程序参考指南](../sub/sub-link.md#sub)

### 概述

用户子程序：
- 用于增加几个 Abaqus 功能的功能，单独使用通常的数据输入方法可能过于受限；
- 为分析提供了极其强大和灵活的工具；
- 编写为 C、C++ 或 FORTRAN 代码，在执行分析时必须包含在模型中，如下所述；
- 必须包含在重启运行中，如果需要可以修改，因为它们不保存到重启文件（请参阅 ["重启分析," 第 9.1.1 节"](pt04ch09s01aus53.md)）；
- 不能相互调用；以及
- 在某些情况下可以调用 Abaqus 中也可用的实用程序（请参阅 ["可用的实用程序," 第 18.1.3 节"](pt04ch18s01aus106.md)）。

### 在模型中包含用户子程序

您可以通过指定包含子程序的 C、C++ 或 FORTRAN 源文件或预编译目标文件的名称来在模型中包含一个或多个用户子程序。详细信息在 ["Abaqus/Standard、Abaqus/Explicit 和 Abaqus/CFD 执行," 第 3.2.2 节"](pt01ch03s02abx02.md) 中提供。

| **输入文件用法：** | 在命令行上输入以下内容： |
| --- | --- |
| | **abaqus** **job**=*job-name* **user**={*source-file* | *object-file*} |

| **Abaqus/CAE 用法：** | 作业模块：作业编辑器：**General**：**User subroutine file** |
| --- | --- |

### 在 Abaqus 中管理外部数据库并与其他软件交换信息

在 Abaqus 中，有时需要设置运行时环境并管理与用户子程序结合使用的外部数据文件或并行进程的交互。例如，可能存在需要外部计算的与历史相关的量，每增量一次，用于分析期间；或者，在收敛增量结束时，需要将用户子程序中跨多个单元在 COMMON 块变量中累积的输出量写入外部文件以进行后处理。可以通过 Abaqus/Standard 中的用户子程序 [`UEXTERNALDB`](../sub/sub-link.md#sub-xsl-uexternaldb) 和 Abaqus/Explicit 中的 [`VEXTERNALDB`](../sub/sub-link.md#sub-xsl-vexternaldb) 执行此类操作。此用户界面可用于与另一个代码交换数据，允许 Abaqus 和另一个代码之间的"交错"。

### 编写用户子程序

用户子程序应非常仔细地编写。为确保其成功实施，应遵循以下规则和指南。有关各个子程序的详细讨论，包括编码接口和要求，请参阅 [Abaqus 用户子程序参考指南](../sub/sub-link.md#sub)。

#### 必需的 `INCLUDEs`

用 FORTRAN 编写的每个用户子程序必须在参数列表之后的第一个语句包含以下语句之一：
- Abaqus/Standard：

```
include 'aba_param.inc'
```
- Abaqus/Explicit：

```
include 'vaba_param.inc'
```
如果在主用户子程序和后续子程序之间交换变量，您应在所有子程序中指定上述 include 语句以保持精度。

每个 C 和 C++ 用户子程序必须包含语句

```
#include <aba_for_c.h>
```
此文件包含 FORTRAN 到 C 接口互操作性的宏。

文件 `aba_param.inc`、`vaba_param.inc` 和 `aba_for_c.h` 由 Abaqus 安装程序安装在系统上，包含重要的安装参数。这些语句告诉 Abaqus 执行程序编译并将用户子程序与 Abaqus 的其余部分链接，自动包含 `aba_param.inc` 或 `vaba_param.inc` 文件。无需找到文件并将其复制到任何特定目录；Abaqus 会知道在哪里找到它。

#### 命名约定

如果用户子程序调用其他子程序或使用 COMMON 块传递信息，这些子程序或 COMMON 块应以字母 K 开头，因为这个字母从不用于启动 Abaqus 中任何子程序或 COMMON 块的名称。

用 C 或 C++ 编写的用户子程序将从 FORTRAN 调用；因此，它们必须符合 FORTRAN 调用约定：C 或 C++ 子程序的名称必须包装在 `FOR_NAME` 宏中；例如，

```
 extern "C" void FOR_NAME(film)(...) { ... }
```
并且参数必须通过引用传递。

#### 重新定义变量

用户子程序必须执行其预期功能，而不覆盖 Abaqus 的其他部分。特别是，您应该只重新定义本章中标识为"要定义的变量"的那些变量。重新定义"传入信息的变量"将产生不可预测的影响。

#### 编译和链接问题

如果在子程序的编译或链接过程中遇到问题，请确保 Abaqus 环境文件（此文件的默认位置是 Abaqus 安装的 `site` 子目录）包含 [Abaqus 安装和许可指南](../sgb/sgb-link.md#sgb) 中指定的正确编译和链接命令。这些命令应由 Abaqus 站点管理员在安装期间设置。参数的数量和类型必须与文档中指定的内容相对应。参数类型或数量的不匹配可能导致平台相关的链接或运行时错误。

#### 内存分配注意事项

您的用户子程序将与 Abaqus 共享内存资源。当您需要使用大型数组或其他大型数据结构时，您应该动态分配它们的内存，以便从堆而不是栈分配内存。未能动态分配大型数组可能导致栈溢出错误并中止 Abaqus 分析。有关使用原生 FORTRAN ALLOCATABLE 数组进行动态分配的示例，请参阅 ["创建数据文件以便于肘单元结果的后处理：FELBOW," Abaqus 例题指南第 15.1.6 节](../exa/exa-link.md#exa-pst-felbow)。Abaqus 还为用户提供了另一种更方便的方式来分配自己的存储（请参阅 ["可分配数组," Abaqus 用户子程序参考指南第 2.1.23 节](../sub/sub-link.md#sub-utl-localarrays)。

#### 测试和调试

在开发用户子程序时，在将它们用于生产分析工作之前，在用户子程序是模型中唯一复杂方面的小示例上彻底测试它们。

如果需要，可以使用 FORTRAN 单元 7 将调试输出写入 Abaqus/Standard 消息（`.msg`）文件，或使用 FORTRAN 单元 6 写入 Abaqus/Standard 数据（`.dat`）文件或 Abaqus/Explicit 日志（`.log`）文件；这些单元不应由您的例程打开，因为它们已由 Abaqus 打开。

FORTRAN 单元 15 到 18 或大于 100 的单元可用于读取或写入其他用户指定的信息。使用其他 FORTRAN 单元可能会干扰 Abaqus 文件操作；请参阅 ["Abaqus 使用的 FORTRAN 单元号," 第 3.7.1 节"](pt01ch03s07aus37.md)。您必须打开这些 FORTRAN 单元；并且由于使用了暂存目录，OPEN 语句中必须使用文件的完整路径名。

可以将环境变量 `ABA_PARALLEL_DEBUG` 设置为打开详细模式，显示编译和链接命令，并启用用户子程序的调试。

#### 终止分析

在从用户子程序内部终止分析时，应使用实用程序 `XIT`（Abaqus/Standard）或 `XPLB_EXIT`（Abaqus/Explicit）而不是 `STOP`。这将确保正确关闭与分析关联的所有文件（["终止分析," Abaqus 用户子程序参考指南第 2.1.15 节"](sub/sub-link.md#sub-utl-uxit)）。

### 以部件实例装配定义的模型

Abaqus 模型可以定义为部件实例的装配（请参阅 ["定义装配," 第 2.10.1 节"](pt01ch02s10aus28.md)）。

#### 参考坐标系

虽然可以为每个部件实例定义局部坐标系，但所有变量（如当前坐标）都以全局坐标系传递给用户子程序，而不是以部件局部坐标系传递。此规则的唯一例外是当用户子程序接口专门指示变量在用户定义的局部坐标系中时（["方向," 第 2.2.5 节"](pt01ch02s02aus15.md)，或 ["变换坐标系," 第 2.1.5 节"](pt01ch02s01aus09.md)）。局部坐标系最初可能相对于部件坐标系定义，但已根据为部件实例给定的定位数据进行了变换。因此，创建了相对于装配（全局）坐标系的新局部坐标系。这个新坐标系定义是用户子程序中用于局部方向的定义。

#### 节点和单元编号

传递给用户子程序的节点和单元编号是 Abaqus 生成的内部编号。这些编号本质上是全局的；所有内部节点和单元编号都是唯一的。如果需要原始编号和部件实例名称，请从用户子程序内部调用实用子程序 `GETPARTINFO`（Abaqus/Standard）或 `VGETPARTINFO`（Abaqus/Explicit）（请参阅 ["获取部件信息," Abaqus 用户子程序参考指南第 2.1.5 节"](sub/sub-link.md#sub-utl-ugetpartinfo)）。调用这些例程的开销并非无关紧要，因此建议尽量少用。

另一个实用子程序 `GETINTERNAL`（Abaqus/Standard）或 `VGETINTERNAL`（Abaqus/Explicit）可用于检索与给定部件实例名称和局部编号对应的内部节点或单元编号。

#### 集和表面名称

传递给用户子程序的集和表面名称始终以装配和部件实例名称为前缀，用下划线分隔。例如，属于装配 `Assembly1` 中部件实例 `Part1-1` 的名为 `surf1` 的表面将作为

```
Assembly1_Part1-1_surf1
```
传递给用户子程序

#### 由依赖于解的状态变量控制的单元删除

如果在分析中定义了由状态变量控制的单元删除（请参阅 ["用户定义的机械材料行为," 第 26.7.1 节"](pt05ch26s07abm69.md)），则控制删除标志的状态变量的值可以由任何使用状态变量的用户子程序修改，只要该用户子程序在材料点处被调用。

#### 输出

用户定义的、依赖于解的状态变量可以写入数据（`.dat`）文件、输出数据库（`.odb`）文件和结果（`.fil`）文件；输出标识符 SDV 和 SDV*n* 可用作单元积分变量（请参阅 ["Abaqus/Standard 输出变量标识符," 第 4.2.1 节"](pt02ch04s02abv01.md) 和 ["Abaqus/Explicit 输出变量标识符," 第 4.2.2 节"](pt02ch04s02xbv01.md)）。对于用户子程序 [`VFRIC`](../sub/sub-link.md#sub-xsl-vfric)、[`VUINTER`](../sub/sub-link.md#sub-xsl-vuinter)、[`VFRICTION`](../sub/sub-link.md#sub-xsl-vfriction)、[`VUINTERACTION`](../sub/sub-link.md#sub-xsl-vuinteraction) 和 [`VWAVE`](../sub/sub-link.md#sub-xsl-vwave)，这些变量的输出不可用。

### 字母数字数据

字母数字数据（如表面或材料的标签（名称））始终以大写形式传递给用户子程序。因此，这些标签与相应小写字符的直接比较将失败。所有此类比较必须使用大写。此类比较的示例可在 ["UMAT," Abaqus 用户子程序参考指南第 1.1.41 节"](sub/sub-link.md#sub-rtn-uumat) 中找到。它说明了在需要定义多个用户定义材料模型时，用户子程序 [`UMAT`](../sub/sub-link.md#sub-xsl-umat) 内部的代码设置。变量 `CMNAME` 与 `MAT1` 和 `MAT2` 进行比较（即使在材料名称可能分别定义为 `mat1` 和 `mat2` 的情况下）。

### Abaqus/Explicit 中的精度

Abaqus/Explicit 安装了单精度和双精度可执行文件。要使用双精度可执行文件，您必须在运行分析时指定双精度（请参阅 ["Abaqus/Standard、Abaqus/Explicit 和 Abaqus/CFD 执行," 第 3.2.2 节"](pt01ch03s02abx02.md)）。用户子程序中以字母 a 到 h 和 o 到 z 开头的所有变量将自动以您运行的可执行文件的精度定义。可执行文件的精度在 `vaba_param.inc` 文件中定义，无需显式定义变量的精度。

### Abaqus/Explicit 中的向量化

Abaqus/Explicit 用户子程序使用向量接口编写，这意味着数据块被传递给用户子程序。例如，向量化用户材料例程（[`VFABRIC`](../sub/sub-link.md#sub-xsl-vfabric) 和 [`VUMAT`](../sub/sub-link.md#sub-xsl-vumat)）传递 `nblock` 个材料点的应力、应变、状态变量等。`vaba_param.inc` 定义的参数之一是 `maxblk`，即最大块大小。如果用户子程序需要临时数组的维数，可以按 `maxblk` 进行维数。

### 并行化

在并行运行作业时可以使用用户子程序。在线程并行模式下，需要防止对公共块、公共文件和其他共享资源的访问竞争条件。为此提供了特殊的实用程序例程。有关详细信息，请参阅 ["确保线程安全," Abaqus 用户子程序参考指南第 2.1.22 节"](sub/sub-link.md#sub-utl-ugetthreadid)。可以设置环境变量 `ABA_PARALLEL_DEBUG` 以增加详细程度，并帮助在并行运行中出现问题时进行故障排除。

### 用户子程序调用

Abaqus 中可用的大多数用户子程序在分析步骤的每个增量期间至少被调用一次。但是，如下所述，许多子程序的调用频率或多或少。

#### 定义材料、单元或界面行为的子程序

大多数用于定义材料、单元或界面行为的用户子程序在每个增量的第一次迭代中为每个材料点、单元或从表面节点调用两次，以便可以适当制定所选步骤程序的模型初始刚度矩阵。在增量内每个后续迭代中，每个材料点、单元或从表面节点仅调用一次这些子程序。

默认情况下，在瞬态隐式动态分析（["使用直接积分的隐式动态分析," 第 6.3.2 节"](pt03ch06s03at07.md)）中，Abaqus/Standard 在每个动态步骤开始时计算加速度。Abaqus/Standard 必须在零增量之前为每个材料点、单元或从表面节点额外调用两次用于定义材料、单元或界面行为的用户子程序。如果初始加速度计算被抑制，则不会进行对用户子程序的额外调用。如果在瞬态隐式动态步骤中检查半增量残差容差，Abaqus/Standard 必须在每个增量结束时为每个材料点、单元或从表面节点额外调用一次这些用户子程序（[`UVARM`](../sub/sub-link.md#sub-xsl-uvarm) 除外）。如果抑制了半增量残差的计算，则不会进行对用户子程序的额外调用。

[`UHARD`](../sub/sub-link.md#sub-xsl-uhard)、[`UHYPEL`](../sub/sub-link.md#sub-xsl-uhypel)、[`UHYPER`](../sub/sub-link.md#sub-xsl-uhyper) 和 [`UMULLINS`](../sub/sub-link.md#sub-xsl-umullins) 在平面应力分析中使用时会被更频繁地调用。

#### 定义初始条件或方向的子程序

用于定义初始条件或方向的用户子程序在分析的每个步骤的第一次迭代之前调用。

#### 定义预定义场的子程序

用于定义预定义场的用户子程序在为相关步骤的第一次增量的所有迭代调用（每当需要当前场变量时）。

#### 子程序调用的验证

如果对用户子程序被调用的频率有任何疑问，可以在小型示例上测试子程序来获取此信息，如前所述。当前步骤和增量编号通常传递给这些子程序，可以将它们打印为调试输出（也在前面讨论过）。调用子程序的迭代编号可能不会传递给用户子程序；但是，如果从子程序向消息（`.msg`）文件发送打印输出（["输出," 第 4.1.1 节"](pt02ch04s01aus38.md)），只要消息文件的输出在每个增量写入，输出在此文件中的位置将给出迭代编号。

### 实用程序例程

有多种实用程序例程可帮助编写用户子程序的代码。您将实用程序例程包含在用户子程序内部。调用时，实用程序例程将执行预定义的功能或操作，其输出或结果可以集成到用户子程序中。一些实用程序例程仅适用于特定的用户子程序。每个实用程序例程在 ["实用程序," Abaqus 用户子程序参考指南第 2.1 节"](sub/sub-link.md#sub-utl-utilities) 中详细讨论。

#### 实用程序例程中提供的变量

以下实用程序例程需要使用从调用它们的用户子程序传入的 Abaqus 提供的变量：
- `GETNODETOELEMCONN`
- `GETVRM`
- `GETVRMAVGATNODE`
- `GETVRN`
- `IGETSENSORID`
- `IVGETSENSORID`
- `MATERIAL_LIB_MECH`
- `MATERIAL_LIB_HT`

这些变量在传入用户子程序时会正确定义；您不能修改变量或为实用程序例程创建替代变量。

例如，`GETVRM` 实用程序例程需要变量 `JMAC`，它是从 Abaqus/Standard 传入用户子程序 [`UVARM`](../sub/sub-link.md#sub-xsl-uvarm) 和其他用户子程序的，而 `GETVRM` 是其支持的实用程序。变量 `JMAC` 代表一个 Abaqus 数据结构，不需要您进一步处理。如果您从用户子程序 [`UVARM`](../sub/sub-link.md#sub-xsl-uvarm) 中使用 `GETVRM` 实用程序例程，您将从 [`UVARM`](../sub/sub-link.md#sub-xsl-uvarm) 中将 `JMAC` 变量传入 `GETVRM`。
