# 13.4.1 模型实例数据保存在输入文件中







当Abaqus/CAE为包含模型实例的模型装配生成输入（`.inp`）文件时，会生成一个单一的扁平化装配。所有模型实例子装配都作为实例的扁平列表写在单一装配块下。

来自模型实例原始模型的大多数特征都被保存在输入文件中，但有一些例外：
- 模型实例子装配中的历史级特征都不会被保存在输入文件中。来自模型实例的模型级特征会被保存在输入文件中；例如，材料、截面分配、皮肤、筋条和方向。在模型实例中定义的材料和单元控件以模型名称为前缀添加到主装配中；例如，`*model-name*#*material-name*`。来自模型实例的其他模型级数据（如初始条件和幅值定义）不会被保存在输入文件中。
- 在模型实例中零件级定义的工程特征（如质量和惯性单元、弹簧和阻尼器）会被保存到输入文件中，但在原始模型中装配级定义的工程特征不会。
- 来自模型实例的集合和表面会被保存在输入文件中。这些集合和表面名称也以模型实例名称为前缀添加到主装配中；例如：``` *model-instance-name*#Set1 ```
- 来自模型实例的约束、参考点、连接点、连接线和线框会被保存在输入文件中。
- 对于约束，模型实例名称将被添加到约束名称前面；例如：``` *model-instance-name*#*constraint-name* ```
- 连接点、连接线和线框可通过在子装配中创建的集合访问。

存在以下限制：
- 不支持为包含模型实例的模型生成扁平输入文件。
- 不支持对包含模型实例的模型进行重启动分析。
- 不支持包含子结构实例的模型的实例。
- 不支持包含组装紧固件的模型的实例。
