# 58.3.1 用于映射的点云数据文件格式





点云数据文件必须是纯文本文件，格式为 **XYZ** 或 **Grid**。**XYZ** 格式的点云数据文件必须包含一组坐标处的所需场值。对于直角坐标系，点必须由 *X*、*Y* 和 *Z* 坐标给出。如果使用圆柱坐标系或球坐标系，则必须使用相应的坐标。**Grid** 格式包含三维网格中各点的场值。Abaqus/CAE 会进行插值以填充网格数据文件中任何缺失的场值。

### **XYZ** 格式

**XYZ** 格式的点云数据文件必须包含多行数据。对于直角坐标系，每行必须由 *X*、*Y* 和 *Z* 坐标组成，后跟该点的场值。每行中的值可以用空格、制表符或逗号的任意组合分隔；每个空格、制表符或逗号被视为单个字段分隔符。以下示例中的逗号分隔值（CSV）是一种常用格式：

```
X1,Y1,Z1,value
X2,Y2,Z2,value
X3,Y3,Z3,value
etc.
```
如果使用圆柱坐标系或球坐标系，则必须在数据文件中给出相应的坐标；请参见["点云映射场的坐标系，" 第58.3.2节](pt06ch58s03s02.md)。

### **Grid** 格式

**Grid** 格式的点云数据文件定义三维网格中各点的场值。对于直角坐标系，这是一个平面或立方体网格。[图 58–2](pt06ch58s03s01.md#fld-planargrid-nls) 显示了已导入**编辑映射场**对话框的 **Grid** 格式点云数据的简单示例。

**图 58–2** **Grid** 格式的点云数据。

![](../graphics/fld-planargrid-nls.png)

网格数据由一组文件组成，每个文件包含一个平面。例如，您可以拥有一个用于 *Z*=3 平面的文件。此文件将包含 *X* 和 *Y* 坐标的网格以及每个点处的场值；例如：

```
a,  -2,    -1,    0,     1,     2
-2, 0.146, 0.141, 0.139, 0.137, 0.131
-1, 0.141, 0.121, 0.116, 0.111, 0.100
 0, 0.139, 0.116, 0.105, 0.101, 0.094
 1, 0.133, 0.129, 0.122, 0.114, 0.107
 2, 0.128, 0.120, 0.111, 0.102, 0.090
```

在此文件中，*X* 坐标出现在第一列（左侧），*Y* 坐标出现在第一行（顶部），从第二个位置开始。第一个位置包含一个虚拟值，当您将文件导入**创建映射场**对话框中的数据表时，Abaqus/CAE 会忽略该值。在上面的示例文件中，虚拟值是字符 `a`；您可以在此位置使用任何值，因为该值将被忽略。

网格数据文件每行中的值可以用空格、制表符或逗号的任意组合分隔；每个空格、制表符或逗号被视为单个字段分隔符。

完整的网格数据文件集必须为每个平面包含一个文件；例如，*Z*=0、*Z*=1、*Z*=2、*Z*=3 处的 *X*–*Y* 平面。也可以使用 *Y*–*Z*、*X*–*Z*、*Y*–*X*、*Z*–*Y* 和 *Z*–*X* 平面。您可以在**创建映射场**对话框中选择不同的网格平面。

如果您的网格数据文件有任何缺失的场值，Abaqus/CAE 会进行插值以填充缺失的点；您无需在**创建映射场**对话框中填充这些缺失值。

如果使用圆柱坐标系或球坐标系，则必须在网格数据文件中给出相应的坐标；请参见["点云映射场的坐标系，" 第58.3.2节](pt06ch58s03s02.md)。
