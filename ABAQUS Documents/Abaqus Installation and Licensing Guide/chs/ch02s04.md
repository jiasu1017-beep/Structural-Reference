# 2.4 Abaqus 产品安装详情



建议您在安装 Abaqus 产品之前安装 Abaqus 文档；如果不这样做，则必须执行额外步骤才能从 Abaqus 访问 HTML 文档。

要安装 Abaqus 产品，必须安装并运行与 Abaqus 6.14 兼容的 Abaqus 网络许可证服务器；要执行产品安装，您必须至少知道一个活动 Abaqus 许可证服务器的名称。

所有 Abaqus 产品和示例问题都作为 Abaqus 产品安装的一部分进行安装。某些 Abaqus 环境设置会根据 Abaqus 许可证中定义的参数进行适当定义。在安装过程中会自动验证许可产品的子集。

用于安装 Abaqus 产品的命令和选项列在["Abaqus 产品安装，"第 2.1.4 节](ch02s01.md#sgb-chp-install-glance-prod)中。

您必须在 Abaqus 父目录中具有写权限才能安装 Abaqus 产品。在 Windows 平台上，Abaqus 快捷方式和所需的环境变量（包括 **PATH**）被添加到**所有用户**配置文件。

### 2.4.1 Visual C++ 和 MPI 库

当您在 Windows 平台上从启动屏幕选择**安装 Abaqus 产品和许可证**时，Microsoft Visual C++ 2010、2008 和 2005 运行时库会随 Abaqus 一起自动安装。这些库是运行 Abaqus 所必需的。以下库会被安装：
- 64 位 2010 库
- 64 位和 32 位 2008 SP1 库
- 64 位和 32 位 2005 SP1 库

请注意，在 Windows/x86-64 上，需要并安装 2005 和 2008 库的 32 位和 64 位版本。

如有必要，您可以通过从启动屏幕选择**安装 Microsoft Visual C++ 2005 SP1、2008 SP1 和 2010 SP1 运行时库**来独立于产品安装安装 Microsoft Visual C++ 运行时库。或者，您可以从 Abaqus Licensing & Products DVD 执行以下安装程序：

```
win86_64/2010_SP1_vcredist_x64.exe *(64 位库)*
win86_64/2008_SP1_vcredist_x64.exe *(64 位库)*
win86_64/2005_SP1_vcredist_x64.exe *(64 位库)*
win86_64/2008_SP1_vcredist_x86.exe *(32 位库)*
win86_64/2005_SP1_vcredist_x86.exe *(32 位库)*

```

必须安装消息传递接口 (MPI) 组件才能：1) 在 Abaqus/Standard 中使用基于 MPI 的并行作业执行，2) 在 Abaqus/Explicit 中使用域级并行化，或 3) 在 Abaqus/CFD 中运行任何作业（无论 CPU 数量如何）。如果您的 Abaqus 用户将运行这些类型的模拟，则必须预先安装所需的 MPI 组件或允许 Abaqus 安装程序为您安装它们：
- 在 Windows/x86-64 上，Abaqus 产品安装程序会在必要时自动安装 Microsoft MPI 3.0 库。如果需要，您可以，通过从启动屏幕选择**安装 Microsoft HPC MPI Redistributable Pack** 来单独安装这些库。如果这些库已安装在您的计算机上，则不会显示此选项。您还可以通过从命令行执行 Abaqus Licensing & Products DVD 上的以下文件来单独启动 Microsoft MPI 安装程序： ``` win86_64/msmpi_3.0_x64.msi ``` （但是，如果您有 64 位 Windows HPC Server 2008，则不需要此操作，因为 MPI 组件已包含在操作系统中。）
- 在 Linux 64 位 (lnx86-64) 操作系统上，Abaqus 产品安装程序会安装 Platform Computing Message Passing Interface 库 (Platform MPI 9.1.3)。

有关 MPI 和基于线程的并行处理模式的更多信息，请参阅["并行执行，"Abaqus Analysis User's Guide 第 3.5 节](../usb/usb-link.md#usbapar)。

任何将与 Abaqus 产品结合使用的其他第三方软件（编译器、操作系统补丁等）应在安装 Abaqus 产品之前安装。

### 2.4.2 产品安装期间要输入的信息

当安装程序运行时，系统会提示您输入以下各项。当请求文件位置或目录时，您可以单击**选择**来浏览路径。
- 网络许可证服务器主机名。安装过程将尝试自动确定此信息，但您可以选择更改默认选择。确保输入了适当的许可证服务器主机。如果要使用一组冗余服务器，请在对话框中提供的框中输入冗余服务器的主机名（有关安装冗余许可证服务器的详细信息，请参阅["冗余 FLEXnet 许可证服务器配置，"第 3.4 节](ch03s04.md)）。安装程序将根据在许可证服务器上的 Abaqus 许可证文件中检测到的信息在 Abaqus 环境文件中定义设置。如果许可证服务器包含学术教学许可证，Abaqus 将被配置为默认使用此许可证（请参阅["许可证管理参数，"第 4.1.6 节](ch04s01.md#sgb-chp-env-net)）。
- Abaqus HTML 文档的 URL 或基本安装目录 (`*simulia_dir*`) 的路径（如果仅安装了 PDF 或您更喜欢 PDF 而非 HTML）。在安装程序提示时输入此路径将配置 `abaqus doc` 命令打开 PDF 而非 HTML 文档。请参阅["Abaqus 文档，"Abaqus Analysis User's Guide 第 3.2.11 节](../usb/usb-link.md#usb-int-ddocproc)，了解更多信息。如果您，在产品之前未安装 Abaqus HTML 文档或未指定 URL，则必须执行额外步骤才能提供对 HTML 文档的访问（请参阅["安装后设置文档 URL，"第 C.2 节](ap03s02.md)）。
- 安装目录。指定 Abaqus 父目录（称为 `*abaqus_dir*`），即存储所有已安装 Abaqus 版本的位置。默认位置在 Linux 平台上是当前工作目录，在 Windows 平台上是 `C:\SIMULIA\Abaqus`。您不应将 Abaqus 安装在指示特定 Abaqus 版本号的目录中；版本特定的目录将由安装程序创建。例如，运行 Abaqus 6.14-1 所需的所有文件将安装在 *abaqus_dir*`/6.14--1/` 中。
- Abaqus 起始目录的位置（仅限 Windows 平台）。指定在 Abaqus 交互会话期间创建的用户文件的位置以及 Abaqus 命令窗口的主目录。默认位置是 `C:\temp`。（用户以后可以更改此设置；请参阅["Windows 平台上的常见自定义，"第 5.1.2 节](ch05s01.md#sgb-chp-view-cust-common)。）确保起始目录对所有用户具有写权限。
- 如果安装程序检测到广域网 (WAN) 许可证文件，系统会提示您输入有关计算机地理位置的信息。此信息用于自动设置 **computer_location** 环境文件参数，并启用报告生成的许可证使用日志记录。有关更多信息，请参阅["报告 FLEXnet 许可证使用数据，"第 3.8 节](ch03s08.md)。

安装完成后，安装记录和许可产品列表将写入 `*abaqus_dir*/*release*/installation_info/` 目录中名为 `Abaqus 6.14_*release*_InstallLog.log` 的文件。此文件有助于排除问题，不应删除。

### 2.4.3 在多台计算机上重复产品安装

当您在计算机上安装 Abaqus 产品时，安装会在 `*abaqus_dir*/*release*/installation_info` 目录中创建一个名为 `installer.properties` 的文件。此文件称为重放文件，包含产品安装期间使用的信息，例如安装目录、许可证服务器主机和文档 URL。`-replay` 命令选项可用于在相同平台上重复以前安装而无需为每次安装重新输入相同信息。您可以使用此选项执行无提示安装（不显示图形用户界面）。它使用重放文件中存储的信息执行与以前安装相同的安装（相同的安装目录、相同的许可证服务器主机等）。

为第一次安装输入的信息必须对使用 `-replay` 选项的后续安装有效。使用此选项时，会跳过所有安装错误检查。在执行安装之前，您应确保系统满足[附录 A，"系统要求说明"](ap01.md)中的要求，并且您在安装目录中具有写权限和足够的磁盘空间。要使用 `-replay` 选项，请执行以下操作：

1. 使用图形用户界面（不使用 `-replay` 选项）在计算机上安装 Abaqus 产品。
2. 将 `installer.properties` 文件（重放文件）复制到要安装 Abaqus 产品的其他计算机上（必须是相同平台）或复制到已挂载的网络驱动器上。
3. 使用 `-replay` 选项在另一台计算机上安装 Abaqus 产品，如下所示：**Linux 平台** ``` /*prod_dir*/setup -replay *full_path_to_replay_file* ``` **Windows 平台** ``` *prod_drive*:\setup.exe -replay *full_path_to_replay_file* ``` Abaqus 产品将被无提示安装，并且绕过安装前系统检查和安装后验证。
4. 手动运行验证过程以检查 Abaqus 产品的安装（请参阅[附录 E，"验证过程"](ap05.md)）。此步骤是可选的，但强烈建议执行。
5. 使用 `-replay` 选项在每台剩余计算机上重复产品安装。

### 2.4.4 使用 Abaqus 命令

安装过程为每个 Abaqus 版本创建一个唯一命令。此命令写在 Abaqus 父目录 (*abaqus_dir*) 下的 `Commands` 目录中。例如，运行 Abaqus 6.14-1 的命令是 `abq6141`。运行后续 Abaqus 6.14 维护版本 的命令是 `abq614*n*`，其中 *n* 是维护交付版本。

安装过程还会创建一个名为 `abaqus` 的链接，指向已安装最新版本的 Abaqus 的命令。要列出所有 Abaqus 命令行选项，请将 `help` 命令行选项与相应的 Abaqus 执行命令一起使用。例如，要查看 Abaqus 6.14-1 的命令行选项，请键入 `abq6141 help`。有关 Abaqus 命令行选项的详细说明，请参阅[Abaqus Analysis User's Guide 第 3 章，"作业执行"](../usb/usb-link.md#usbdchapter)。

在 Windows 平台上，`Commands` 目录路径会在安装期间自动添加到所有用户的路径中。在 Linux 平台上，应手动将 `Commands` 目录添加到每个用户的路径。或者，可以在系统级别或为每个用户定义指向 `Commands` 目录中相应命令的每个 Abaqus 版本的链接。
