# 2.4 隐式和显式过程的比较







Abaqus/Standard 和 Abaqus/Explicit 能够解决各种问题。隐式和显式过程的特性决定了哪种方法适合给定问题。对于可以使用任一方法解决的问题，可以用哪种方法更有效地解决决定了使用哪个产品。理解隐式和显式过程的特性将帮助您回答这个问题。[表 2-2](ch02s04.md#gsa-abasics-comparison) 列出了分析产品之间的主要差异，这些差异在本指南的相关章节中有详细讨论。

**表 2-2** Abaqus/Standard 和 Abaqus/Explicit 之间的主要差异。
| 物理量 | Abaqus/Standard | Abaqus/Explicit |
| --- | --- | --- |
| 单元库 | 提供广泛的单元库。 | 提供非常适合显式分析的广泛单元库。可用的单元是 Abaqus/Standard 中可用单元的子集。 |
| 分析过程 | 提供通用和线性扰动过程。 | 提供通用过程。 |
| 材料模型 | 提供多种材料模型。 | 与 Abaqus/Standard 中可用的类似；一个显著差异是允许失效材料模型。 |
| 接触公式 | 具有解决接触问题的鲁棒能力。 | 具有即使是最复杂的接触模拟也能轻松解决的鲁棒接触功能。 |
| 求解技术 | 使用基于刚度的无条件稳定求解技术。 | 使用有条件稳定的显式积分求解技术。 |
| 磁盘空间和内存 | 由于增量中可能的迭代次数很多，磁盘空间和内存使用可能很大。 | 磁盘空间和内存使用通常比 Abaqus/Standard 小得多。 |

### 2.4.1 在隐式和显式分析之间选择

对于许多分析，使用 Abaqus/Standard 还是 Abaqus/Explicit 是很清晰的。例如，正如[第 8 章，"非线性"](ch08.md) 中所演示的，Abaqus/Standard 在解决平滑非线性问题方面更高效；另一方面，Abaqus/Explicit 是波传播分析的明确选择。然而，某些静态或准静态问题可以用任一程序很好地模拟。通常，这些是通常使用 Abaqus/Standard 解决但可能由于接触或材料复杂性而难以收敛的问题，导致大量迭代。这样的分析在 Abaqus/Standard 中成本很高，因为每次迭代需要求解一个大型线性方程组。

Abaqus/Standard 必须迭代以确定非线性问题的解，而 Abaqus/Explicit 通过**显式**地从先前增量结束时的运动状态推进来确定解，而无需迭代。尽管给定的分析使用显式方法可能需要大量时间增量，但如果同一分析在 Abaqus/Standard 中需要多次迭代，则分析在 Abaqus/Explicit 中可能更高效。

Abaqus/Explicit 的另一个优势是它对于相同模拟需要的磁盘空间和内存比 Abaqus/Standard 少得多。对于两个程序的计算成本可能相当的问题，Abaqus/Explicit 的大量磁盘空间和内存节省使其具有吸引力。

### 2.4.2 隐式和显式分析中网格细化的成本

使用显式方法，计算成本与单元数量成正比，与最小单元尺寸大致成反比。因此，网格细化通过增加单元数量和减小最小单元尺寸来增加计算成本。例如，考虑一个具有均匀方形单元的三维模型。如果网格在所有三个方向上细化两倍，由于单元数量增加，计算成本增加 2×2×2 倍，由于最小单元尺寸减小，计算成本增加 2 倍。网格细化后，分析的总计算成本增加 24 倍，即 16 倍。磁盘空间和内存需求与单元数量成正比，与单元尺寸无关；因此，这些需求增加 8 倍。

虽然预测显式方法网格细化的成本增加相当直接，但使用隐式方法时成本更难预测。困难源于单元连接性与求解成本之间的问题依赖关系，这种关系在显式方法中不存在。使用隐式方法，经验表明对于许多问题，计算成本大致与自由度数量的平方成正比。考虑相同的具有均匀方形单元的三维模型示例。在所有三个方向上将网格细化两倍会将自由度数量增加约 2³，导致计算成本增加约 (2³)² 倍，即 64 倍。磁盘空间和内存需求以相同方式增加，尽管实际增加难以预测。
