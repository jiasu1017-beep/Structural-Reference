# 81.8 覆盖插件





启动时，Abaqus/CAE 首先在 Abaqus 父目录中搜索插件，然后在 Abaqus 主目录中搜索，最后在当前目录中搜索。插件名称区分大小写。例如，名为 `myPlugin` 的插件与名为 `MyPlugin` 的插件不同。如果 Abaqus/CAE 在多个目录中找到名称和大小写相同的插件，则搜索中后找到的插件将覆盖早期找到的插件。这种方法使您可以使用位于主目录或当前目录中的自定义版本插件；但是，在大多数情况下，您应该避免插件命名冲突。

Python 按照 *sys.path* 列表指定的顺序搜索模块，并在找到模块名称匹配后立即停止搜索。例如，如果您在一个名为 `myUtils.py` 的实用工具模块存储在两个不同的插件目录中，并且这些目录包含不同的代码，则两个插件将引用同一个模块——*sys.path* 列表中首先找到的那个模块。

您可以使用 Python 的包样式来存储和引用模块，以避免意外覆盖插件或访问错误的模块。首先创建一个详细命名的目录，该目录不太可能被其他人复制，并将您的 `_plugin.py` 文件存储在该目录中。在第一个目录内创建另一个同名的目录，并将您的其他插件文件放在新目录中，同时添加一个名为 `__init__.py` 的空文件，使目录结构如下：

```
torqueCalculator
   torqueCalculator_plugin.py
   torqueCalculator
      __init__.py
      torqueUtils.py
      torqueForm.py
      torqueDB.py         
```

在代码中导入文件时，使用包名来限定导入。例如，在 `torqueForm.py` 内部，您可以使用以下语句限定导入：

```
from torqueCalculator.torqueDB import TorqueDB
```
