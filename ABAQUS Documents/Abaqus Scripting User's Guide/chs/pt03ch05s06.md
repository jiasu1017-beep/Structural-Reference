# 5.6 数据对话框







数据对话框是用于从用户收集数据的对话框。相比之下，消息对话框仅显示消息，工具箱仅包含按钮。本节介绍如何创建数据对话框。以下主题将被涵盖：
- "数据对话框概述"，第 5.6.1 节
- "构造函数"，第 5.6.2 节
- "Bailout"，第 5.6.3 节
- "构造函数内容"，第 5.6.4 节
- "转换"，第 5.6.5 节

### 5.6.1 数据对话框概述

数据对话框是用于从用户收集数据的对话框。相比之下，消息对话框仅显示消息，工具箱仅包含按钮。`AFXDataDialog` 设计为与模式结合使用，以从用户收集数据。然后数据在命令中被处理。如果您需要发出命令，应该使用 `AFXDataDialog`。如果对话框属于模块或非持久工具集，也应该使用 `AFXDataDialog`，以便 GUI 基础设施可以在用户切换模块时正确管理对话框。

`AFXDataDialog` 类派生自 `AFXDialog`，并提供以下附加功能：
- Bailout 机制。
- 设计用于与表单结合使用的标准操作区域按钮行为。
- 关键字用法。
- 定义对话框中 GUI 状态更改的转换。

### 5.6.2 构造函数

`AFXDataDialog` 构造函数有两个原型。两种原型之间的区别是对话框的遮挡行为，如下例所示：
- 以下语句创建一个在与主窗口重叠时始终遮挡主窗口的对话框：``` AFXDataDialog(mode, title, actionButtonIds=0, opts=DIALOG_NORMAL, x = 0, y = 0, w = 0, h = 0 ) ```
- 以下语句创建一个在与组件重叠时始终遮挡其所有者组件（通常是对话框）的对话框。``` AFXDataDialog(mode, owner, title, actionButtonIds=0, opts=DIALOG_NORMAL, x = 0, y = 0, w = 0, h = 0 ) ```

构造对话框时，您将从 `AFXDataDialog` 类派生。构造函数体中您应该做的第一件事是调用基类构造函数以正确初始化对话框。然后，您将通过添加组件来构建对话框的内容。例如：

```
class MyDB(AFXDataDialog):

    # My constructor
    def __init__(self):

        # Call base class constructor
        AFXDataDialog.__init__(self, form,  'My Dialog',
            self.OK|self.CANCEL)

        # Add widgets next...
```

当对话框被取消显示时，它会从屏幕上移除。默认情况下，对话框在被取消显示时被删除。删除对话框会移除与对话框关联的 GUI 资源以及对话框的数据结构。相比之下，您可以选择在对话框被取消显示时销毁对话框。销毁对话框仅移除 GUI 资源，保留对话框的数据结构。

如果您想在对话框的显示之间保留某些对话框 GUI 状态，则应该指定对话框仅在被取消显示时被销毁。因此，当对话框再次显示时，它会保留其数据结构，旧状态仍然完整。例如，假设您的对话框包含一个表格，用户调整了表格某列的大小。如果您仅在对话框被取消显示时销毁对话框，则表格列大小将在下次显示对话框时被记住。要指定对话框应在被取消显示时被销毁，请将 DIALOG_UNPOST_DESTROY 标志添加到对话框构造函数的 *opts* 参数中。

### 5.6.3 Bailout

`AFXDataDialog` 通过在对话框构造函数中指定位标志来支持自动 bailout 处理。如果您请求 bailout 处理，并且用户更改了对话框中的某些值并按下**取消**，应用程序会显示标准警告对话框。以下语句请求 bailout 处理：

```
AFXDataDialog.__init__(self, form, 'Create Part',
    self.OK|self.CANCEL,
    DIALOG_ACTIONS_SEPARATOR|DATADIALOG_BAILOUT)
```

**图 5–6** Bailout 示例。

![](../graphics/dlg-dialogs-bailout-nls.png)

显示标准警告对话框后，行为如下：如果用户从标准警告对话框点击**是**，数据对话框将如同用户最初按下**确定**一样被处理。
- 如果用户从标准警告对话框点击**否**，数据对话框将被取消显示而不进行任何处理。
- 如果用户从标准警告对话框点击**取消**，数据对话框将保持显示状态，不采取任何操作。

### 5.6.4 构造函数内容

您使用对话框的构造函数来创建将出现在对话框中的组件。为了使 GUI 与应用程序状态保持同步，反之亦然，您使用关键字作为组件的目标。关键字定义为表单的成员，表单作为对话框构造函数参数传递给您。如需更多信息，请参阅"AFXKeywords"，第 6.5.8 节。以下脚本显示如何使用关键字构造对话框。[图 5-7](pt03ch05s06.md#cus-dlg-datadialogs) 显示了示例脚本生成的**图形选项**对话框。

**图 5–7** **图形选项**数据对话框。

![](../graphics/cus-dlg-datadialogs-nls.png)

```
class GraphicsOptionsDB(AFXDataDialog):

    #~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
    def __init__(self, form):

        AFXDataDialog.__init__(self, form, 'Graphics Options',
            self.OK|self.APPLY|self.DEFAULTS|self.CANCEL)

        # Hardware frame
        #
        gb = FXGroupBox(self, 'Hardware', 
            FRAME_GROOVE|LAYOUT_FILL_X)
        hardwareFrame = FXHorizontalFrame(gb, 
            0, 0,0,0,0, 0,0,0,0)
        FXLabel(hardwareFrame, 'Driver:')
        FXRadioButton(hardwareFrame, 'OpenGL',
            form.graphicsDriverKw, OPEN_GL.getId())
        FXRadioButton(hardwareFrame, 'X11',  
            form.graphicsDriverKw, X11.getId())
        FXCheckButton(gb, 'Use double buffering',
            form.doubleBufferingKw)
        displayListBtn = FXCheckButton(gb, 'Use display lists',
            form.displayListsKw)

        # View Manipulation frame
        #

            FRAME_GROOVE|LAYOUT_FILL_X)

        FXLabel(hf, 'Drag mode:')
        FXRadioButton(hf, 'Fast (wireframe)', form.dragModeKw,
            FAST.getId())
        FXRadioButton(hf, 'As is', form.dragModeKw, 
            AS_IS.getId())
        FXCheckButton(gb, 'Auto-fit after rotations', 
            form.autoFitKw)

```

### 5.6.5 转换

转换提供了一种在对话框中更改 GUI 状态的便捷方式。转换用于在其他控件被激活时来糊化组件或轮换区域。如果对话框中的行为可以用简单转换来描述，您可以使用 `addTransition` 方法来产生状态更改。

转换将关键字的值与指定值进行比较。如果运算符条件满足，则向指定目标对象发送消息。转换具有以下原型：

```
addTransition(keyword,
operator, value, tgt, sel, ptr)
```

例如，当用户在**零件显示选项**对话框中选择**线框**作为渲染样式时，Abaqus/CAE 执行以下操作：
- 来糊化**在隐藏渲染样式中显示虚线**按钮。
- 来糊化**在着色渲染样式中显示边**按钮。
- 选中**显示轮廓边**按钮。

这些转换可以描述如下：
- 如果渲染样式关键字的值等于 WIREFRAME，则向**显示虚线...**按钮发送 ID_DISABLE 消息。
- 如果渲染样式关键字的值等于 WIREFRAME，则向**显示边...**按钮发送 ID_DISABLE 消息。
- 如果渲染样式关键字的值等于 WIREFRAME，则向**显示轮廓边**按钮发送 ID_ENABLE 消息。

您可以按如下方式编写这些转换：

```
self.addTransition(form.renderStyleKw, AFXTransition.EQ,
    WIREFRAME.getId(), showDottedBtn,
    MKUINT(FXWindow.ID_DISABLE, SEL_COMMAND), None)

self.addTransition(form.renderStyleKw, AFXTransition.EQ,
    WIREFRAME.getId(), showEdgesBtn,
    MKUINT(FXWindow.ID_DISABLE, SEL_COMMAND), None)

self.addTransition(form.renderStyleKw, AFXTransition.EQ,
    WIREFRAME.getId(), showSilhouetteBtn,
    MKUINT(FXWindow.ID_ENABLE, SEL_COMMAND), None) 
```

您还可以使用 `addTransition` 方法的最后一个参数将额外的用户数据传递给对象。[图 5-8](pt03ch05s06.md#dlg-dialogs-transition) 显示了一个使用转换来控制应用程序如何来糊化组件的示例。

### 5.6.6 更新您的 GUI

如果对话框的 GUI 行为不能用简单转换来描述（例如，如果您需要根据其他两个按钮的设置来糊化一个按钮），您可以使用 `processUpdates` 方法来更新您的 GUI。`processUpdates` 方法在每个 GUI 更新周期中被调用，因此您不应在此方法中执行耗时的操作。一般来说，您应该执行启用和禁用，或显示和隐藏组件等任务。例如：

```
def processUpdates(self):

    if self.form.kw1.getValue() == 1 and \
       self.form.kw2.getValue() == 2:

           self.btn1.disable()
    else:
           self.btn1.enable()
```

如果您需要执行的任务很耗时，您应该编写自己的消息处理程序，仅在某些特定用户操作时被调用。例如，如果您需要扫描 ODB 以获取有效数据，您可以使得对话框的提交按钮向您的对话框发送一条消息。该消息将调用您的消息处理程序来执行扫描。这样，扫描仅在用户提交对话框时发生，而不是在每个 GUI 更新周期中发生。有关消息处理程序的更多信息，请参阅"目标和消息"，第 6.5.4 节。

### 5.6.7 操作区域

`AFXDataDialog` 类为所有可能出现在操作区域中的按钮提供标准处理。[表 5-2](pt03ch05s06.md#cus-dlg-dialgs-actionarea) 显示了当点击每个按钮时应用程序执行的操作。

**表 5–2** 操作区域按钮。
| 按钮 | 操作 |
| --- | --- |
| **确定** | 向表单发送 (ID_COMMIT, SEL_COMMAND) 消息及其按钮 ID。 |
| **应用** | 向表单发送 (ID_COMMIT, SEL_COMMAND) 消息及其按钮 ID。 |
| **继续** | 向表单发送 (ID_GET_NEXT, SEL_COMMAND) 消息。 |
| **默认** | 向表单发送 (ID_SET_DEFAULTS, SEL_COMMAND) 消息。 |
| **取消** | 检查 bailout，向表单发送 (ID_DEACTIVATE, SEL_COMMAND) 消息。 |
| 标题栏中的"**x**" | 执行**取消**按钮操作。 |

如果您的对话框有多个"应用"按钮，您可以通过将按钮的消息路由到表单中的应用消息处理程序来处理此问题。在表单中，您可以使用 `getPressedButtonId` 方法来确定按下了哪个按钮并采取适当的操作。例如，在对话框构造函数中：

```
self.appendActionButton('Plot', self, self.ID_PLOT)
FXMAPFUNC(self, SEL_COMMAND, self.ID_PLOT, 
    AFXDataDialog.onCmdApply
self.appendActionButton('Highlight', self, self.ID_HIGHLIGHT)
FXMAPFUNC(self, SEL_COMMAND, self.ID_HIGHLIGHT, 
    AFXDataDialog.onCmdApply)
```

并在您的表单代码中：

```
def doCustomChecks(self):

    if self.getPressedButtonId() == self.getCurrentDialog().ID_PLOT:
        # Enable plot commands, disable highlight commands
    else:
        # Enable highlight commands, disable plot commands
    return True
```
