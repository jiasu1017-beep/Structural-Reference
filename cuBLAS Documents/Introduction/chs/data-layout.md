## 1.1. 数据布局

为与现有 Fortran 环境保持最大兼容性，cuBLAS 库采用列优先存储和从 1 开始的索引。由于 C 和 C++ 使用行优先存储，使用这些语言编写的应用程序无法使用二维数组的本机数组语义。相反，应该定义宏或内联函数以在一维数组之上实现矩阵。对于以机械方式从 Fortran 移植到 C 的代码，可以选择保留从 1 开始的索引以避免转换循环的需要。在这种情况下，矩阵第 "i" 行第 "j" 列元素的数组索引可以通过以下宏计算：

```c++
// 将二维矩阵索引映射到一维数组索引（Fortran 风格，从1开始）
#define IDX2F(i,j,ld) ((((j)-1)*(ld))+((i)-1))
```

其中，ld 指矩阵的主维度，在列优先存储的情况下，它是已分配矩阵的行数（即使只使用其中的子矩阵）。对于以原生方式编写的 C 和 C++ 代码，最可能选择从 0 开始的索引，在这种情况下，矩阵第 "i" 行第 "j" 列元素的数组索引可以通过以下宏计算：

```c++
// 将二维矩阵索引映射到一维数组索引（C 风格，从0开始）
#define IDX2C(i,j,ld) (((j)*(ld))+(i))
```
