# 58.7.3 映射场的搜索控制





在**创建映射场**对话框的 **Mapper Controls** 选项卡页面上，您可以调整搜索容差参数。对于点云数据源和输出数据库网格源，搜索控制略有不同。

Abaqus/CAE 尝试将您的所有源数据点及其相关场值映射到目标模型上的点。根据每个源点离网格目标模型最近节点的距离，Abaqus/CAE 必须决定是使用还是丢弃每个源点。这些决定基于您在 **Mapper Controls** 选项卡页面上选择的搜索容差距离值。

### 点云映射场的搜索控制

正负法向容差值定义源数据点可以离目标模型外表面的距离。目标模型表面在未变形有限元模型中进行插值。默认情况下，每个源数据点必须位于目标模型平均单元特征尺寸乘以 0.05 计算的距离内。此距离沿目标表面的正法向向量测量（参见[图 58–4](pt06ch58s06s01.md#usi-fld-map-controls-normals-nls)）。

**图 58–4** 正法向搜索距离。

![](../graphics/usi-fld-map-controls-normals-nls.png)

您可以更改容差距离值，以控制哪些源数据点将被包含或忽略。您可以将容差值定义为网格模型中平均单元尺寸的相对分数，或定义为模型中使用的长度单位的绝对距离。

对于 Abaqus/CAE 无法映射源点的任何目标点，它将替换**创建映射场**对话框中输入的**默认未映射场值**。默认值为零。对于点云数据源和输出数据库网格源，都会应用默认场值。

对于点云数据源，**Mapper Controls** 选项卡页面上提供以下选项和搜索容差值：

**容差类型**

如果选择 **Relative**（默认），则容差值被解释为网格模型中平均单元特征尺寸的分数（不是百分比）。如果选择 **Absolute**，则容差值被视为模型中使用的长度单位的精确距离。

**正法向搜索距离容差**

在此距离内（沿目标表面基础几何体的正法向向量测量）的任何源点都会被映射并包含在分析中（参见[图 58–4](pt06ch58s06s01.md#usi-fld-map-controls-normals-nls)）。此容差仅适用于表面（而非体积）目标映射。默认值为网格模型中平均单元特征尺寸的 0.05。

**负法向搜索距离容差**

在此距离内（沿目标表面基础几何体的负法向向量测量）的任何源点都会被映射并包含在分析中。此容差仅适用于表面（而非体积）目标映射。默认值为网格模型中平均单元特征尺寸的 0.15。

**边界搜索距离容差**

此容差指定源数据点必须位于其中的平面内距离，超出网格目标模型单元的区域（参见[图 58–5](pt06ch58s06s01.md#usi-fld-map-controls-boundary-nls)）。将此容差从默认值 0.01（平均单元尺寸的分数）增加有效地允许您扩展每个目标单元面以便进行映射。增加此容差可能有助于源数据比目标模型网格更粗的情况。

映射到目标模型时，场值会被插值。边界搜索容差适用于表面映射和体积映射。

**图 58–5** 边界搜索距离。

![](../graphics/usi-fld-map-controls-boundary-nls.png)

**邻域搜索距离容差**

位于邻域搜索容差之外的任何源数据点都会被忽略，不会用于分析。此容差适用于表面和体积目标映射。

Abaqus/CAE 使用距离加权算法来插值网格目标模型上的场数据值。Abaqus/CAE 总是尝试在目标上插值源值。如果 Abaqus/CAE 发现任何无法进行插值的点，则使用距离加权算法，该算法将获取邻域搜索距离内的节点。距离加权不考虑任何其他现有容差（正/负法向搜索容差或边界搜索容差）。您可以通过使用非常小的邻域搜索距离来停用距离加权，在这种情况下，Abaqus/CAE 会在这些节点上应用**默认未映射场值**。

Abaqus/CAE 在尝试将源点映射到目标点时进行三次传递。Abaqus/CAE：

1. 尝试在目标上插值场值。第一次传递时忽略邻域搜索容差。
2. 使用距离加权算法获取邻域搜索距离内的源点。
3. 对于任何仍未被映射的目标点，Abaqus/CAE 应用**默认未映射场值**。

### 输出数据库场的搜索控制

法向容差值定义源数据点可以离目标模型外表面的距离。默认情况下，每个源数据点必须位于目标模型平均单元特征尺寸乘以 0.05 计算的距离内。此距离沿目标表面的法向向量测量。容差值被解释为网格模型中平均单元特征尺寸的分数（不是百分比）。

您可以更改默认容差值，以确定哪些源数据点将被包含或丢弃。

对于 Abaqus/CAE 无法映射源点的任何目标点，它将替换**创建映射场**对话框中输入的**默认未映射场值**。默认值为零。对于点云数据源和输出数据库网格源，都会应用默认场值。

对于输出数据库源，**Mapper Controls** 选项卡页面上提供以下选项和搜索容差值：

**法向搜索距离容差**

在此距离内（沿目标表面基础几何体的正法向向量测量）的任何源点都会被映射并包含在分析中。此容差适用于表面和体积目标映射。默认值为网格模型中平均单元特征尺寸的 0.05。

**边界搜索距离容差**

此容差指定源数据点必须位于其中的平面内距离，超出网格目标模型单元的区域（参见[图 58–5](pt06ch58s06s01.md#usi-fld-map-controls-boundary-nls)）。将此容差从默认值 0.01（平均单元尺寸的分数）增加有效地允许您扩展每个目标单元面以便进行映射。增加此容差可能有助于源数据比目标模型网格更粗的情况。

映射到目标模型时，场值会被插值。边界搜索容差适用于表面映射和体积映射。

### 体积到表面映射控制

**创建映射场**对话框的 **Mapper Controls** 选项卡页面包含**目标表面映射算法**选项，让您选择 **Surface** 或 **Volumetric** 映射。

您的目标区域必须是三种类型之一：体积、表面或网格中的一组节点。如果目标是体积，Abaqus/CAE 始终执行体积映射。但是，如果目标是表面或一组节点，您必须选择要 Abaqus/CAE 用于应用源数据的算法。选择 **Surface** 会导致 Abaqus/CAE 使用表面投影算法，将源数据投影到目标表面形心或节点。选择 **Volumetric** 会导致 Abaqus/CAE 使用体积插值算法，将源数据插值到目标单元形心或节点。两种算法的区别仅影响源体积的场值如何应用于目标表面。

对于映射算法，目标表面或体积在 Abaqus/CAE 的几何模型或网格上下文中的定义如下：
- 几何模型中的体积是三维单元
- 网格中的体积由单元组成
- 三维几何模型中的表面是几何面
- 网格中的"表面"由单元面组成

当您映射到节点且目标节点由节点集（基于网格）定义时，表面与体积算法的选择也适用。但是，如果目标节点从模型几何体定义，则 Abaqus/CAE 始终对几何面使用表面映射，对几何单元使用体积映射。

目标区域是您在其中应用载荷、相互作用、边界条件或预定义场的主模型。如果您在不同属性中应用相同的映射场源，它们的行为可能不同，因为目标区域类型可能不同。

[表 58–6](pt06ch58s06s01.md#usi-fld-usingmap-volsurf) 描述了 Abaqus/CAE 对输出数据库源执行体积或表面映射的情况，以及何时必须在这两者之间进行选择。

**表 58–6** 输出数据库源的体积与表面映射情况。
| 输出数据库源 | 目标区域类型 |
| --- | --- |
| 几何单元（3D 单元或节点） | 壳或表面/面1 | 节点集 |
| 节点数据 | 体积 | 体积或表面 | 体积或表面 |
| 基于单元的数据2 | 体积 | 体积或表面 | 体积或表面 |
| 基于表面的数据 | 不适用（错误） | 表面 | 表面 |
| 1 出于映射算法的目的，表面可以是三维单元的面。 |
| 2 Visualization 模块中的输出数据库平均控制会影响基于单元源数据的连续性。 |

如果目标区域是壳或表面，则体积算法将源体积（输出数据库中）的整个场值插值到目标表面区域。或者，表面算法仅将源体积表面的场值投影到目标表面区域。

当目标表面可以引用三维单元面时，表面算法将输出数据库源数据投影到目标表面形心或节点。体积算法将源数据插值到目标单元形心或节点。
