### 6.5.1 构建GUI命令

您可以使用`AFXGuiCommand`类来构建GUI命令。`AFXGuiCommand`类接受以下参数：

***mode***

模式通过GUI中的控件（通常是菜单按钮）激活。一旦模式被激活，它负责收集用户输入、处理输入、发送命令以及执行与该模式或其发送的命令相关的任何错误处理。有关模式的详细讨论，请参阅[第7章，"模式"](pt04ch07.md)。Abaqus GUI工具包提供两种模式：

**表单模式**

表单模式提供对话框界面。表单模式使用一个或多个对话框收集用户输入。

**过程模式**

过程模式提供一个接口，通过在应用程序的提示区域中提示输入，引导用户完成一系列步骤。

***method***

一个指定kernel命令方法的String。

***objectName***

一个指定kernel命令对象的String。

***registerQuery***

一个布尔值，指定是否在对象上注册查询。

例如，以下语句创建一个编辑图形选项的命令：

```
cmd = AFXGuiCommand(self, 'setValues',
   'session.graphicsOptions', True)
```

如果模式中有多个GUI命令，则命令按在模式中创建的相同顺序处理。有关创建GUI命令的更多示例，请参阅["表单示例，" 第7.3.1节](pt04ch07s03.md#cus-mod-modes-form-example)和["过程示例，" 第7.4.1节](pt04ch07s04.md#cus-mod-procedure-example)。

### 6.5.2 GUI命令和当前对象

Abaqus/CAE中的大多数命令对当前对象进行操作；例如，当前视口或当前部件。作为一种便利，模式在解释GUI命令中指定的对象时识别一种特殊语法。如果在某些 repository 后的方括号中放置`%s`，模式将用当前名称替换`%s`。您应该始终使用此`%s`语法，而不是硬编码名称，这样当前名称将始终用于命令中。

支持以下当前对象：

| 对象规范 | 模式解释 |
| --- | --- |
| `mdb.models[%s]` | 当前模型 |
| `mdb.models[%s].parts[%s]` | 当前部件 |
| `mdb.models[%s].sketches[%s]` | 当前草图 |
| `session.odbs[%s]` | 当前输出数据库 |
| `session.viewports[%s]` | 当前视口 |

### 6.5.3 保持GUI和命令最新

如果命令编辑对象，您应该通过在GUI命令构造函数中为*registerQuery*参数指定True来请求在该对象上注册查询。注册查询将导致当模式启动时以及当kernel值发生任何变化时，与AFXGuiCommand关联的关键字将使用kernel值进行更新。例如，

```
cmd = AFXGuiCommand(
    mode, 'PointSection', 'mdb.models[%s]', True)
```

此外，模式将`session.viewports[%s]`识别为特殊的repository。模式自动在session上注册查询，以便如果用户切换当前视口，命令将保持最新。以下示例说明了特殊语法：

```
cmd = AFXGuiCommand(
    mode,'setValues','session.viewports[%s]', True)

cmd = AFXGuiCommand(
    mode,'bringToFront','session.viewports[%s]', True)
```

### 6.5.4 目标和消息

Abaqus GUI工具包采用目标/消息系统来实现GUI进程内的通信。目标/消息系统与例如Motif的回调机制形成对比。所有窗口部件都可以从任何其他窗口部件发送和接收消息。消息由两个组件组成：

- 消息*类型*
- 消息*ID*

消息类型表示发生了什么类型的事件；例如，单击按钮。消息*ID*标识消息的发送者。

Abaqus GUI工具包中的大多数窗口部件都接受指定其目标和ID的参数。即使窗口部件不将目标和ID作为参数，您也可以使用`setTarget`和`setSelector`方法设置这些属性。例如，

```
FXButton(parent, 'Label', tgt=self, sel=self.ID_1)

groupBox = FXGroupBox(parent)
groupBox.setTarget(self)
groupBox.setSelector(self.ID_2)
```

窗口部件能够发送多种类型的消息。两种最常见的消息类型是SEL_COMMAND和SEL_UPDATE。SEL_COMMAND消息类型通常表示窗口部件被"提交"；例如，用户单击了按钮。当窗口部件请求其目标更新其状态时，会发送SEL_UPDATE消息；有关更多信息，请参阅["自动GUI更新，" 第6.5.5节](pt04ch06s05.md#cus-com-commands-automatic)。

消息使用目标类中定义的映射将消息路由到消息处理程序。您可以通过指定当收到具有特定类型和ID的消息时要调用的方法来在映射中添加条目。这些概念在[图6-2](pt04ch06s05.md#bas-basics-targets)中进行了说明。

**图6-2** 目标和消息。

![](../graphics/bas-basics-targets-nls.png)

消息映射使用`FXMAPFUNC`函数定义（请参阅下面的示例）。此宏接受四个参数：*self*、*消息类型*、*消息ID*和方法名。方法名必须用类名限定：*className.methodName*。当收到的消息的类型和ID与`FXMAPFUNC`条目中定义的内容匹配时，将调用相应的方法。如果您有大量要在消息映射中定义的ID，可以使用`FXMAPFUNCS`函数，它接受一个附加参数：*self*、*消息类型*、*起始消息ID*、*结束消息ID*和方法名。

对象使用消息处理程序对消息做出反应。所有消息处理程序具有相同的原型，其中包含：

- 消息的发送者
- 消息选择器
- 一些"用户数据"

您可以使用`SELTYPE`和`SELID`函数从选择器中提取消息的类型和ID。

以下代码显示了消息映射、消息ID和消息处理程序如何协同工作：

```
class MyClass(BaseClass):

    [
        ID_1,
        ID_2,
        ID_LAST
    ] = range(BaseClass.ID_LAST, BaseClass.ID_LAST+3)

    def __init__(self):

        BaseClass.__init__(self)
        FXMAPFUNC(self, SEL_COMMAND, self.ID_1,
            MyClass.onCmdPrintMsg)
        FXMAPFUNC(self, SEL_COMMAND, self.ID_2,
            MyClass.onCmdPrintMsg)

        FXButton(self, 'Button 1', None, self, self.ID_1)
        FXButton(self, 'Button 2', None, self, self.ID_2)

    def onCmdPrintMsg(self, sender, sel, ptr):

        if SELID(sel) == self.ID_1:
            print 'Button 1 was pressed.'
        elif SELID(sel) == self.ID_2:
            print 'Button 2 was pressed.'
        return 1
```

前面的示例首先生成一个派生类中使用的ID列表。由于窗口部件具有特定目标，窗口部件的ID不一定是全局唯一的；它只需要在目标类及其基类中是唯一的。为了自动处理此编号，惯例是在每个类中定义ID_LAST。派生类应使用其基类中定义的ID_LAST值开始其编号。此外，派生类应将其自己的ID_LAST定义为派生类中的最后一个ID。派生自该派生类的类将能够使用该ID开始其编号。ID_LAST不应被任何窗口部件使用。ID_LAST的唯一目的是在类之间提供自动编号方案。

示例继续通过使用`FXMAPFUNC`函数添加条目来构建消息映射。在此示例中，当收到类型为`SEL_COMMAND`且ID为`ID_1`或`ID_2`的消息时，脚本调用`onCmdPrintMsg`方法。

两个按钮窗口部件将其目标设置为self（`MyClass`）。但是，当每个窗口部件发送消息时，窗口部件发送不同的消息ID，消息处理程序检查ID以确定发送者。例如，如果用户单击第一个按钮，按钮向`MyClass`发送`(ID_1, SEL_COMMAND)`消息。类的消息映射将该消息路由到`onCmdPrintMsg`方法。`onCmdPrintMsg`方法检查传入消息的ID并打印`Button 1 was pressed`。

重要的是，您的消息处理程序必须返回正确的值以确保GUI保持最新。在消息处理程序中返回`1`告诉工具包消息已被处理。反之，如果消息被处理，工具包假定可能发生了需要更新的某些内容，工具包会启动GUI更新过程。在消息处理程序中返回`0`告诉工具包消息未被处理；因此，工具包不会启动GUI更新过程。

消息通常作为GUI中某些交互的结果由GUI基础设施发送。但是，您可以通过调用对象的`handle`方法直接向对象发送消息。`handle`方法接受三个参数：*sender*、*selector*和*userData*。发送者通常是发送消息的对象。选择器由消息ID和消息类型组成。您可以使用`MKUINT`函数创建选择器，例如`MKUINT(ID_1, SEL_COMMAND)`。用户数据必须为*None*，因为Abaqus GUI工具包不支持此功能。

### 6.5.5 自动GUI更新

当没有更多事件要处理时（通常是在GUI空闲并等待某些用户交互时），Abaqus GUI工具包会自动启动GUI更新。在自动GUI更新过程中，每个窗口部件向其目标发送SEL_UPDATE消息以请求更新。通过这种方式，GUI不断轮询应用程序状态以保持自身最新。

例如，在自动GUI更新期间，复选按钮向其目标发送更新消息。目标检查某些应用程序状态，并确定是否应该选中复选按钮。如果应该选中按钮，目标发送ID_CHECK消息；否则，它发送ID_UNCHECK消息。

工具包中的窗口部件是双向的；也就是说，它们可以处于*push*状态或*pull*状态。

**push状态**

在*push*状态下，窗口部件正在收集用户输入并将其发送到应用程序。当窗口部件处于*push*状态时，它不参与自动GUI更新过程。因为窗口部件不参与自动GUI更新过程，所以用户控制输入，而不是GUI尝试更新窗口部件。

**pull状态**

在*pull*状态下，窗口部件正在查询应用程序以保持最新。

### 6.5.6 数据目标

在典型的GUI应用程序中，您需要执行以下操作：

1. 初始化对话框中的值。
2. 显示对话框以允许用户进行更改。
3. 从对话框收集更改。

此外，如果某些应用程序状态在对话框显示时更新，您可能希望对话框更新其状态。数据目标旨在使GUI程序员更容易完成这些任务。本节描述数据目标如何工作。以下各节描述了Abaqus GUI工具包如何扩展此概念，以用于构建发送到kernel的命令的关键字。

数据目标充当某些应用程序状态与GUI窗口部件之间的双向中介。多个窗口部件可以连接到数据目标，但数据目标仅作用于一个应用程序状态。当用户使用GUI更改值时，数据目标监视的应用程序状态会自动更新。相反，当应用程序状态更新时，连接到数据目标的窗口部件会自动更新。

如["自动GUI更新，" 第6.5.5节](pt04ch06s05.md#cus-com-commands-automatic)中所述，窗口部件可以处于push状态或pull状态。

**Push状态**

在push状态下，窗口部件正在收集用户输入并将其发送到应用程序。[图6-3](pt04ch06s05.md#bas-basics-datatargets1)说明了数据目标如何与处于push状态的窗口部件配合工作。顺序如下：

1. 首先，用户在文本字段中输入值`7`，然后按**Enter**。
2. 这会触发文本字段窗口部件向其目标（数据目标）发送`(ID, SEL_COMMAND)`消息。
3. 数据目标通过向发送者（文本字段窗口部件）发送消息来响应，请求文本字段中的值。数据目标使用该值更新其数据的值。

**图6-3** 具有处于push状态的文本字段窗口部件的数据目标。

![](../graphics/bas-basics-datatargets1-nls.png)

**Pull状态**

在pull状态下，窗口部件正在查询应用程序以保持最新。[图6-4](pt04ch06s05.md#bas-basics-datatargets2)说明了数据目标如何与处于pull状态的窗口部件配合工作。顺序如下：

1. 当GUI空闲时，它启动GUI更新。
2. GUI更新触发每个窗口部件向其目标发送`(ID, SEL_UPDATE)`消息。
3. 在这种情况下，数据目标通过向发送者（文本字段窗口部件）发送消息来响应，告诉它将其值设置为数据目标数据的值。

**图6-4** 具有处于pull状态的文本字段窗口部件的数据目标。

### 6.5.7 选项与值模式

数据目标以两种模式之一工作：值模式或选项模式。上一节中的示例描述了值模式。当某些数据的实际值是您感兴趣的时，您使用值模式。相反，当您需要从多个项目中进行选择且值不是特别重要时，您使用选项模式。

对于在push状态下与窗口部件一起工作的选项模式下的数据目标，行为与上一节中描述的值模式类似。当用户单击按钮时，按钮向其目标发送`(ID, SEL_COMMAND)`消息。反过来，目标通过向发送者发送消息来响应，请求将数据目标的数据更新为发送者消息ID的值。

对于在pull状态下与窗口部件一起工作的选项模式下的数据目标，行为与上一节中描述的值模式略有不同。在GUI更新期间，数据目标根据发送者的ID是否与数据目标数据的值匹配，向发送者发送选中或取消选中消息。

例如，[图6-5](pt04ch06s05.md#bas-basics-options)说明了一个数据目标在选项模式下与三个处于pull状态的单选按钮配合工作。假设数据目标监视的数据值为13，单选按钮的消息ID分别为2、13和58。顺序如下：

1. 在GUI更新期间，第一个单选按钮向数据目标发送`(2, SEL_UPDATE)`消息。
2. 数据目标将消息ID（2）与其数据的值（13）进行比较，由于值不匹配，向单选按钮发送取消选中消息。
3. 然后第二个单选按钮向数据目标发送`(13, SEL_UPDATE)`消息。
4. 数据目标比较值，由于值匹配，向单选按钮发送选中消息。
5. 类似地，第三个按钮从数据目标接收取消选中消息，因为消息ID及其数据的值不匹配。

通过这种方式，Abaqus GUI工具包自动维护单选按钮行为（任何时候只有一个按钮会被选中）。

**图6-5** 在选项模式和pull状态下在三个单选按钮上运行的数据目标。

### 6.5.8 AFXKeywords

关键字生成GUI命令的参数。这些关键字属于命令，但也作为成员存储在模式中。因此，您可以轻松地将关键字连接到更新关键字值的对话框中的窗口部件。有关更多信息，请参阅[第5章，"对话框"](pt03ch05.md)。

`AFXKeyword`是工具包中关键字的基类。`AFXKeyword`类派生自数据目标，因此它自动保持GUI和应用程序数据相互同步。有关更多信息，请参阅["数据目标，" 第6.5.6节](pt04ch06s05.md#cus-com-commands-datatargets)。

`AFXKeyword`类通过保存额外值（如关键字的名称、默认值和先前值）来扩展`FXDataTarget`类的功能。关键字的GUI命令使用此信息来构建kernel命令字符串。

您可以将关键字指定为可选或必需。必需关键字始终由GUI命令发出。可选关键字，如果其值自上次提交命令以来没有更改，则不会由GUI命令发出。如果自上次提交以来没有任何关键字更改，当模式提交时不会发出任何GUI命令。

支持以下类型的关键字：

- `AFXIntKeyword(cmd, name, isRequired, defaultValue)`
- `AFXFloatKeyword(cmd, name, isRequired, defaultValue)`
- `AFXStringKeyword(cmd, name, isRequired, defaultValue)`
- `AFXBoolKeyword(cmd, name, booleanType, isRequired, defaultValue)`
- `AFXSymConstKeyword(cmd, name, isRequired, defaultValue)`
- `AFXTupleKeyword(cmd, name, isRequired, minLength, maxLength, opts)`
- `AFXTableKeyword(cmd, name, isRequired, minLength, maxLength, opts)`
- `AFXObjectKeyword(cmd, name, isRequired, defaultValue)`

每个关键字支持的数据类型由其构造函数的名称暗示，AFXObjectKeyword除外。对象关键字是支持将变量名指定为关键字值的关键字。

所有关键字的原型相似。关键字的前两个参数是：

- 一个GUI命令对象。
- 一个指定关键字名称的String。

所有关键字还支持一个决定关键字是必需还是可选的参数。如果关键字是必需的，它将始终与命令一起发送。如果关键字是可选的，则仅当其值发生更改时才发送。但是，如果关键字连接到的窗口部件被隐藏，则无论关键字是必需还是可选，都不会发送该关键字。

大多数关键字支持指定默认值。当您构造关键字时，其值被设置为默认值。如果您使用关键字的`setDefaultValue`方法更改默认值，除非您还调用关键字的`setValueToDefault`方法，否则不会影响关键字的值。相反，如果您只想更改关键字的值而不更改其默认值，您应该使用关键字的`setValue`方法。

当模式向kernel发出命令时，关键字将按照在模式中创建它们的相同顺序排序。

在模式类中存储关键字时，惯例是使用关键字标签加上`Kw`作为关键字对象的名称。例如，

```
self.rKw = AFXIntKeyword(self.cmd, 'r',  True)
self.tKw = AFXFloatKeyword(self.cmd, 't',  True)
self.nameKw = AFXStringKeyword(cmd, 'name', True, 'Part-1')
self.twistKw = AFXBoolKeyword(cmd, 'twist',
    AFXBoolKeyword.ON_OFF, 0)
self.typeKw = AFXSymConstKeyword(cmd, 'type', True,
    SHADED.getId())
self.imageSizeKw = AFXTupleKeyword(cmd, 'imageSize', False,
    1, 2, AFXTUPLE_TYPE_FLOAT)
```

### 6.5.9 表达式求值

`AFXFloatKeyword`和`AFXIntKeyword`都支持表达式求值。这意味着您可以在连接到`AFXFloatKeyword`或`AFXIntKeyword`的文本字段中输入数学表达式，该表达式将得到求值。例如，您可以在连接到`AFXFloatKeyword`的文本字段中输入以下任何表达式：

```
3 + (7 * 22)
2 * 3.1415 * 1.5
125/55.8

```

表达式将随命令一起发送，因此它会出现在重放和日志文件中，但一旦命令在kernel中处理，只存储结果值，表达式会丢失。

表达式求值始终可用于`AFXFloatKeyword`，但对于`AFXIntKeyword`是可选的（默认是执行表达式求值）。如果您将`AFXIntKeyword`连接到`AFXList`或`AFXComboBox`，并且列表或组合框中显示的选择不表示数值，则必须禁用表达式求值。例如：

```
表单代码片段：

    self.orderKw = AFXIntKeyword(cmd=cmd, name='order',
        isRequired=False, defaultValue=1, evalExpression=False)

对话框代码片段：

    combo = AFXComboBox(self, 8, 3, 'Order:', form.orderKw, 0)
    combo.appendItem('First', 1)
    combo.appendItem('Second', 2)
    combo.appendItem('Third', 3)

```

此代码的命令片段将如下所示：

```
someCommand(order=2, ...)
```

### 6.5.10 将关键字连接到窗口部件

关键字在GUI中的使用方式是将它们设置为窗口部件的目标。`AFXDataDialog`类将模式作为其构造函数参数之一。对话框使用构造函数中提供的模式来访问存储在模式中的关键字。此外，对话框使用关键字作为对话框中窗口部件的目标。

除了目标之外，窗口部件还有一个消息ID。重要的是，为使关键字在适当模式下（值或选项）操作，必须设置适当的ID。有关更多信息，请参阅["数据目标，" 第6.5.6节](pt04ch06s05.md#cus-com-commands-datatargets)。在大多数情况下，消息ID应使用零值；零值表示关键字应在值模式下操作。下表总结了关键字的消息ID用法，以下各节为每种类型的关键字提供了示例。

| 关键字 | ID | 描述 |
| --- | --- | --- |
| `AFXIntKeyword` | 0 | 关键字在值模式下操作。当关键字连接到文本字段、列表、组合框或微调器时使用此选项。 |
| | >0 | 关键字在选项模式下操作。当关键字连接到单选按钮时使用此选项。 |
| `AFXFloatKeyword` | 0 | 关键字在值模式下操作。 |
| `AFXStringKeyword` | 0 | 关键字在值模式下操作。 |
| `AFXBoolKeyword` | 0 | 关键字在值模式下操作。此关键字应仅与仅允许布尔值的窗口部件（如`FXCheckButton`）一起使用。 |
| `AFXSymConstKeyword` | 0 | 关键字在值模式下操作。当关键字连接到列表或组合框时使用此值。 |
| | >0 | 关键字在选项模式下操作。当关键字连接到单选按钮时使用符号常量的ID值。不要将此关键字与`FXCheckButton`一起使用。 |
| `AFXTupleKeyword` | 0 | 关键字在值模式下操作。当整个元组从单个窗口部件收集时使用此值。 |
| | 1, 2, 3, ... | 关键字仅对元组的第*n*个元素在值模式下操作，其中*n*=ID。当每个元素的输入从单独的窗口部件收集时使用此值。 |
| `AFXTableKeyword` | 0 | 关键字在值模式下操作。 |
| `AFXObjectKeyword` | 0 | 关键字在值模式下操作。 |

### 6.5.11 布尔、整数、浮点数和字符串关键字示例

以下语句说明了布尔、整数、浮点数和字符串关键字的使用：

```
# 带有复选框的布尔关键字

FXCheckButton(self, 'Show node labels', mode.nodeLabelsKw, 0)

self.tree = AFXOptionTreeList(parent, 6)
```

### 6.5.12 符号常量关键字示例

符号常量提供了一种为命令参数指定选择的方法，使命令更具可读性。例如，显示选项命令的*renderStyle*参数有三个选择。我们可以使用从1到3的整数值来对这些选择进行编号。但是，使用整数值会导致命令不太易读；例如，`renderStyle=2`。或者，如果我们为每个选择定义符号常量，命令会变得更易读；例如，`renderStyle=HIDDEN`。在内部，符号常量包含一个整数ID，可通过其`getId()`方法访问。符号常量可在GUI和kernel进程中使用。通常您应该创建一个定义符号常量的模块，然后将此模块导入到您的kernel和GUI脚本中。

您可以从symbolicConstants模块导入`SymbolicConstant`构造函数。构造函数接受一个字符串参数。按照惯例，字符串参数使用全大写字母，单词之间用下划线分隔，变量名与字符串参数相同。例如，

```
from symbolicConstants import SymbolicConstant
AS_IS = SymbolicConstant('AS_IS')
```

对于符号常量关键字，您可以使用零值或符号常量的ID值作为消息ID。符号常量具有唯一的整数ID，用于设置符号常量关键字的值，以及一个用于命令生成的字符串表示。要访问符号常量的整数ID，请使用其`getId`方法。

如果关键字连接到列表或组合框窗口部件，您应该在窗口部件构造函数中使用零作为ID。`AFXList`、`AFXComboBox`和`AFXListBox`窗口部件设计为将符号常量关键字作为目标处理。当向列表或组合框添加项目时，符号常量的ID作为用户数据传递。这些窗口部件通过将其值设置为用户数据与目标值匹配的项（而不是将其值设置为与目标索引匹配的项）来做出反应。以下示例说明了如何将组合框连接到符号常量关键字：

```
combo = AFXComboBox(hwGb, 18, 4, 'Highlight method:',
    mode.highlightMethodHintKw, 0)
combo.appendItem('Hardware Overlay', HARDWARE_OVERLAY.getId())
combo.appendItem('Software Overlay', SOFTWARE_OVERLAY.getId())
combo.appendItem('XOR', XOR.getId())
combo.appendItem('Blend', BLEND.getId())
```

如果关键字连接到单选按钮，您应该使用与该单选按钮对应的符号常量的ID作为消息ID。由于所有符号常量的ID都大于零，这告诉关键字在选项模式下操作。以下示例说明了如何将符号常量关键字与单选按钮一起使用：

```
from abaqusConstants import *
...

# 建模空间
#
gb = FXGroupBox(self, 'Modeling Space',
    FRAME_GROOVE|LAYOUT_FILL_X)
FXRadioButton(gb, '3D', mode.dimensionalityKw,
    THREE_D.getId(), LAYOUT_SIDE_LEFT)
FXRadioButton(gb, '2D Planar', mode.dimensionalityKw,
    TWO_D_PLANAR.getId(), LAYOUT_SIDE_LEFT)
FXRadioButton(gb, 'Axisymmetric',
    mode.dimensionalityKw, AXISYMMETRIC.getId(),
    LAYOUT_SIDE_LEFT)
```

### 6.5.13 元组关键字示例

对于元组关键字，消息ID为零值表示整个元组将被更新。例如，您可以使用单个文本字段收集用户的*X*、*Y*和*Z*输入。在这种情况下，用户输入的逗号分隔字符串用于设置整个元组关键字的值。例如，如果您定义元组关键字如下：

```
self.viewVectorKw = AFXTupleKeyword(cmd, 'viewVector',
    True, 3, 3)
```

您可以按如下方式将元组关键字连接到单个文本字段：

```
AFXTextField(self, 12, 'View Vector (X,Y,Z)',
    mode.viewVectorKw, 0)
```

或者，您可以使用三个单独的文本字段来收集*X*、*Y*和*Z*输入。每个文本字段窗口部件使用的消息ID等于它们对应的元组元素编号（从1开始）。例如，1对应元组的第一个元素；2对应元组的第二个元素，依此类推。在这种情况下，我们可以按如下方式将关键字连接到三个文本字段：

```
AFXTextField(self, 4, 'X:', mode.viewVectorKw, 1)
AFXTextField(self, 4, 'Y:', mode.viewVectorKw, 2)
AFXTextField(self, 4, 'Z:', mode.viewVectorKw, 3)
```

### 6.5.14 表关键字示例

`AFXTableKeyword`必须连接到表窗口部件。这种类型的关键字将导致命令参数成为一个元组的元组。表关键字中的值可以是Int、Float或String。

默认最小行数为0，默认最大行数为1，表示行数无限制。表的大小可以变化，因为用户可以添加或删除行；因此，您通常指定最小和最大行数的默认值。例如，要生成创建XY数据的命令，您可以在表单中定义以下关键字

```
self.cmd = AFXGuiCommand(self, 'XYData', 'session')
self.nameKw = AFXStringKeyword(self.cmd, 'name', True)
self.dataKw = AFXTableKeyword(
   self.cmd, 'data', True, 0, -1, AFXTABLE_TYPE_FLOAT)
```

在对话框中，您使用零作为选择器值将表关键字连接到表。

```
table = AFXTable(vf, 6, 3, 6, 3,
   form.dataKw, 0, AFXTABLE_NORMAL|AFXTABLE_EDITABLE)
```

如果您有一个只关心其中单列值的表，您可以利用`AFXColumnItems`对象来跟踪选择。例如，如果表包含"名称"和"描述"列，您可能只需要所选行中的名称用于您的命令。在这种情况下，您可以使用`AFXColumnItems`来保持元组关键字与表中选定行的名称同步，如以下代码所示：

```
ci = AFXColumnItems(referenceColumn=0, tgt=form.tupleKw, sel=0)
table = AFXTable(self, 4, 2, 4, 2, ci, 0,
   AFXTABLE_NORMAL|AFXTABLE_ROW_MODE|AFXTABLE_EXTENDED_SELECT

```

### 6.5.15 对象关键字示例

`AFXObjectKeyword`有一个变量名作为其值。在大多数情况下，您在前面带有某些设置命令的命令中使用`AFXObjectKeyword`。例如，

```
p = mdb.models['Model-1'].parts['Part-1']
session.viewports['Viewport: 1'].setValues(displayedObject=p)
```

在此示例中，在表单中您将"手动"发出第一个命令，并使用对象关键字作为`AFXGuiCommand`的一部分，以使用"p"作为变量名发出第二个命令。例如，

```
self.cmd = AFXGuiCommand(self, 'setValues',
    'session.viewports[%s]')
self.doKw = AFXObjectKeyword(self.cmd, 'displayedObject',
    True, 'p')
```

您还在需要选择的程序中使用`AFXObjectKeyword`。有关更多信息，请参阅["过程模式中的选择，" 第7.5节](pt04ch07s05.md)。

### 6.5.16 默认值对象

当用户按下对话框中的"默认值"按钮时，可以使用默认值对象将命令中的关键字值恢复为默认值。您可以按如下方式向命令注册默认值对象：

```
self.registerDefaultsObject(cmd,
    'session.defaultGraphicsOptions')
```

此外，`AFXGuiCommand`类有一个`setKeywordValuesToDefaults`方法，您可以使用它来初始化命令中所有关键字的状态。在大多数情况下，您在模式的`getFirstDialog`方法中使用`setKeywordValuesToDefaults`方法来初始化所有关键字的状态。因此，应用程序将在每次显示对话框时初始化命令中关键字的值。

如果没有指定默认值对象，当用户按下对话框中的"默认值"按钮时，命令使用关键字构造函数中指定的默认值。
