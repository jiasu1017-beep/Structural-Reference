# 3.4.12 更改步骤中对象的状态





步骤相关管理器包含在某些情况下可用于更改特定步骤中对象状态的按钮。这些按钮标记为**向左移动**、**向右移动**、**激活**和**停用**。

是否可以更改步骤中对象的状态取决于步骤的过程以及对象在该步骤中的状态。管理器仅允许您对对象的历史进行有效更改。如果某个按钮的操作会导致无效的状态更改，则该按钮会变得不可用。有关更多信息，请参阅["修改步骤相关对象的历史，" 第3.4.6节"](pt01ch03s04s06.md)。

以下列表描述了操作步骤相关对象状态的技术：

**选择您要更改的状态：**

单击位于感兴趣对象的行和感兴趣步骤的列交汇处的单元格。

该对象在该步骤中的状态会变为突出显示，在大多数情况下，对话框右侧的一些或所有按钮会变为可用。按钮的可用性取决于对象在当前步骤、前一步骤和后续步骤中的状态。

例如，下图中选择了`Step-3`中**Pressure**的`Created`状态：

![](../graphics/int-select-nls.png)

使用变得可用的按钮来操作您在所选步骤中选择的对象的状态，如下所述。

**将所选步骤中的状态移动到前一步骤：**

单击**向左移动**，将突出显示的状态从所选步骤移动到前一步骤。

例如，在上面显示的历史中，选择了`Step-3`中**Pressure**的`Created`状态。如果单击**向左移动**，历史会如下更改：

![](../graphics/int-moveleft-nls.png)

**Pressure**的`Created`状态会移动到`Step-2`，并在`Step-3`中被`Propagated`替换。

**将所选步骤中的状态移动到后续步骤：**

单击**向右移动**，将突出显示的状态从所选步骤移动到后续步骤。

例如，在下面显示的历史中，选择了`Step-5`中**Pressure**的`Modified`状态。

![](../graphics/int-moveright1-nls.png)

如果单击**向右移动**，历史会如下更改：

![](../graphics/int-moveright2-nls.png)

**Pressure**的`Modified`状态会移动到`Step-6`（表示对**Pressure**的修改在`Step-6`中生效），`Step-5`中的`Modified`被`Propagated`替换。

**在所选步骤中停用对象：**

单击**停用**，在所选步骤中停用对象。

例如，在下面显示的历史中，选择了`Step-4`中**Pressure**的`Propagated`状态。

![](../graphics/int-deactivate1-nls.png)

如果单击**停用**，历史会如下更改：

![](../graphics/int-deactivate2-nls.png)

**Pressure**在`Step-4`中的`Propagated`状态会变为`Inactive`，并且所有后续步骤中的状态都会变为`Inactive`。

**注意：**您不能使用**预定义场管理器**停用预定义场；您必须在预定义场编辑器中选择**重置为初始**（例如，请参阅["定义温度场，" 第16.11.9节"](pt03ch16s11hlb09.md)）。

**警告：**如果在对象状态为`Modified`的步骤中停用该对象，对对象的修改会丢失。如果稍后在该步骤中重新激活对象，该对象的原始未修改版本会在该步骤及所有后续步骤中变为活动状态。

**在所选步骤中重新激活对象：**

单击**激活**，在所选步骤中重新激活对象。

例如，在上面的历史中，选择了`Step-4`中**Pressure**的`Inactive`状态。如果单击**激活**，历史会如下更改：

![](../graphics/int-reactivate-nls.png)

**Pressure**在`Step-4`中的`Inactive`状态会在`Step-4`及任何后续步骤中变为`Propagated`。

**注意：****激活**按钮仅在对象首次被停用的步骤中可用。

您可以使用模型树查看步骤相关对象的状态、编辑对象以及停用和重新激活对象。但是，您必须使用步骤相关管理器通过在步骤序列中向左或向右移动来修改对象的历史。有关更多信息，请参阅["模型树概述，" 第3.5.1节"](pt01ch03s05s01.md)。
