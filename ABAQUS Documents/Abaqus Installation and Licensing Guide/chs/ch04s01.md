# 4.1 使用 Abaqus 环境文件



Abaqus 执行过程（请参阅[Abaqus Analysis User's Guide 第 3 章，"作业执行"](../usb/usb-link.md#usbdchapter)）读取环境文件以确定用于运行作业的各种参数。该过程按以下顺序在三个目录中查找环境文件 `abaqus_v6.env`：

1. Abaqus 安装的 `/SMA/site/` 子目录（请参阅[附录 B，"Abaqus 发布目录和文件"](ap02.md)）。环境文件必须存在于此目录中。当作业提交到远程队列时，此文件中的设置会被忽略。相反，使用远程计算机上的 `/SMA/site/` 子目录中的设置。
2. 用户的主目录。此环境文件是可选的，会影响从用户帐户提交的所有作业。在 Windows 平台上，要使 Abaqus 在用户的主目录中找到环境文件，必须使用 `HOME` 环境变量或 `HOMEDRIVE` 和 `HOMEPATH` 环境变量的组合指定用户主目录的完整路径。
3. 当前工作目录。此环境文件是可选的，会影响从当前工作目录提交的所有作业。

如果同一参数在多个环境文件中定义或在环境文件中定义多次，则使用遇到的最后一个定义。

Abaqus 6.14 的环境文件使用 Python 语法。请参阅[Abaqus Scripting User's Guide](../cmd/cmd-link.md#cmd)获取有关 Python 语法的更详细信息。Abaqus 6.14 的环境文件必须命名为 `abaqus_v6.env`。尝试使用 Version 5 格式的环境文件会导致错误。

环境文件条目的语法如下：

```
parameter=value
```

所有参数都必须有值。以下语法规则也适用：- 所有参数都区分大小写。
- 字符串值必须用一对双引号或单引号括起来。
- 注释前有井号 (#)。行上井号后的所有字符都会被忽略。引号内的井号是字符串的一部分，不是注释的开头。
- 空行会被忽略。
- 列表必须用方括号括起来 ([ ])。列表中的各个项目用逗号分隔。条目形式如下： ``` parameter=[value1, value2, value3] ```
- 元组必须用括号括起来 (( ))。元组中的各个项目用逗号分隔。如果元组用括号括起来且只包含一个值，则必须在值后面跟一个逗号。条目形式如下： ``` parameter=(value1, value2) ```
- 如果嵌入的单引号位于双引号字符串内，则不需要特殊处理。例如，`"my value's"` 被翻译为 `my value's`。嵌入双引号在单引号字符串中的情况相同。如果嵌入的引号与包围引号的类型相同，则可以通过在前面加反斜杠 (\) 字符来嵌入。列表或元组中的字符串必须用引号括起来。
- 三引号 (""") 字符串可以跨越多行，字符串内的引号不需要特殊处理。条目形式如下： ``` parameter=""" multi-line value """ ```

许多环境文件参数的示例位于 Abaqus 安装的 `/SMA/site/` 子目录中的 `abaqus_v6.env` 和 `abaqusinc.env` 环境文件中。将早期版本的自定义 `abaqus_v6.env` 设置合并到当前版本时要小心。早期版本的设置可能与新版本不兼容。以下各节介绍了可用参数。

### 4.1.1 内存和磁盘管理参数

有关 Abaqus/Standard 和 Abaqus/Explicit 的内存和磁盘资源管理的详细讨论，请参阅["管理 Abaqus 中的内存和磁盘使用，"Abaqus Analysis User's Guide 第 3.4.1 节](../usb/usb-link.md#usb-int-dmemory)。相关参数列在此处以及用于管理 Abaqus/CAE 和 Abaqus/Viewer kernel 中内存的单个参数 **abq_ker_memory** 旁边。

内存大小的可用单位是 mb（兆字节）和 gb（吉字节）。如果未指定单位，则大小以兆字节为单位。

****abq_ker_memory****

Abaqus/CAE 和 Abaqus/Viewer kernel 可分配的最大内存量，以 MB（兆字节）为单位。如果超出限制，Abaqus/CAE 会显示错误消息。

如果 kernel 内存大小达到 **abq_ker_memory** 值或机器的虚拟内存限制，则会显示以下消息：`Operation did not complete due to a memory allocation failure.`

为获得最佳性能，内存限制应设置为小于机器上的物理内存量。允许的最小设置是 256 MB。

****scratch****

用作临时文件的目录的完整路径名。在 Linux 上的默认值是 `$TMPDIR` 环境变量的值，如果未定义 `$TMPDIR`，则为 `/tmp`。在 Windows 上的默认值是 `%TEMP%` 环境变量的值，如果未定义 `%TEMP%`，则为 `\TEMP`。在分析期间，会在此目录下创建一个子目录来存放分析临时文件。子目录的名称由用户的用户名、作业 ID 和作业的进程号构成。分析完成后会删除子目录及其内容。

****memory****

在输入文件预处理期间和 Abaqus/Standard 分析阶段可分配的最大内存量或物理内存的最大百分比。此参数的默认值在不同平台上有所不同；详细信息请参阅 Dassault Systèmes 知识库，网址为 [www.3ds.com/support/knowledge-base](http://www.3ds.com/support/knowledge-base)。

### 4.1.2 并行化参数

Abaqus 中的并行化在以下各节中详细讨论：
- ["Abaqus/Standard 中的并行执行，"第 3.5.2 节](../usb/usb-link.md#usb-int-astdparallel)
- ["Abaqus/Explicit 中的并行执行，"第 3.5.3 节](../usb/usb-link.md#usb-int-aparallelexecution)
- ["Abaqus/CFD 中的并行执行，"第 3.5.4 节](../usb/usb-link.md#usb-int-acfdparallel)

相关参数列在此处。

****auto_convert****

如果此参数设置为 `ON` 并且 Abaqus/Explicit 分析使用 **parallel**=`domain` 并行运行，则 **convert**=`select`、`convert**=`state` 和 **convert**=`odb` 选项将在分析结束时根据需要自动运行。默认值是 `ON`。

****cpus****

如果在分析运行期间可用并行处理，则要使用的处理器数量。此参数的默认值是 1。

****domains****

Abaqus/Explicit 中的并行域数量。如果值大于 1，则无论 **parallel** 和 **cpus** 变量的值如何，都会执行域分解。但是，如果 **parallel**=`DOMAIN`，则 **cpus** 的值必须能平均分配到 **domains** 的值中。如果未设置此参数，则如果 **parallel**=`DOMAIN`，则域数量默认为分析运行期间使用的处理器数量；如果 **parallel**=`LOOP`，则为 1。

****gpus****

在 Abaqus/Standard 中使用 GPGPU 硬件激活直接求解器加速。此参数的值应该是分析要使用的 GPGPU 数量。在基于 MPI 的并行 Abaqus/Standard 分析中，这是每台主机上要使用的 GPGPU 数量。

****max_cpus****

如果并行处理可用，则允许的最大处理器数量。如果未设置此参数，则允许的处理器数量等于系统上可用处理器的数量。

****mp_file_system****

基于 MPI 的并行 Abaqus 分析可用的文件系统类型。必须将此参数设置为一个元组；例如，

```
mp_file_system=(SHARED,LOCAL)
```
元组中的第一项指的是提交作业的目录，而第二项指的是作业的临时目录。如果托管目录的文件系统是 `LOCAL`，则 Abaqus 会将所需的分析文件复制到远程主机，并在运行结束时将输出文件复制回来。在这种情况下，必须能够在 **mp_host_list** 中的所有主机上创建作业的目录结构。`SHARED` 文件系统意味着主机共享相同的文件系统，不需要文件传输。使用推荐的默认设置 (`DETECT`, `DETECT`)，Abaqus 将确定存在的文件系统类型。基于 MPI 的并行 Abaqus/Explicit 分析仅在使用用户子程序时才会使用临时目录，而 Abaqus/Standard 通常会在此目录中写入大型临时文件。在本地文件系统上运行通常会提高性能。

****mp_host_list****

基于 MPI 的并行 Abaqus 分析要使用的主机计算机名称列表，包括每台机器上要使用的处理器数量；例如， 

```
mp_host_list=[['maple',1],['pine',1],['oak',2]]
```
表示如果为分析指定的 **cpus** 数量为 4，则分析将在名为 `maple` 的机器上使用一个处理器，在名为 `pine` 的机器上使用一个处理器，在名为 `oak` 的机器上使用两个处理器。主机列表中定义的总处理器数量必须大于或等于为分析指定的 **cpus** 数量。如果未定义主机列表，Abaqus 将在本地系统上运行。使用支持的队列系统时，不需要定义此参数。如果定义了它，队列环境将覆盖它。

****mp_mode****

将此变量设置为 `MPI` 以指示系统上有 MPI 组件可用。将 **mp_mode**=`THREADS` 设置为使用基于线程的并行化方法。默认值是 `MPI`（如果适用）。

****mp_mpi_implementation****

此变量确定要使用的底层 MPI 实现。通常，不需要指定此变量。

****mp_mpirun_options****

传递给基于 MPI 的并行 Abaqus 分析的 MPI 启动程序的选项字符串。通常，不需要指定此变量。

****mp_mpirun_path****

一个字典，用于为给定的 MPI 实现定义 MPI 启动程序的完整路径。例如，在 Windows 上，

```
mp_mpirun_path={NATIVE: 'C:\\Program Files\\
   Microsoft HPC Pack 2008 R2\\bin\\mpiexec.exe'}
```

****mp_num_parallel_ftps****

当使用基于 MPI 的并行化执行并行文件暂存时，此参数控制同时进行的 MPI 文件传输的数量。第一项控制进出临时目录的文件传输。第二项控制进出分析工作目录的文件传输。将任一值设置为 1 会禁用并行文件暂存过程。文件暂存的使用取决于 **mp_file_system** 中指定的值。

****mp_rsh_command****

用于在 **mp_host_list` 中指定的机器上打开远程 shell 的首选命令。如果文件系统不是共享的，Abaqus 需要打开远程 shell 来创建和删除目录和文件。此选项的默认值是平台相关的；例如， 

```
mp_rsh_command='ssh -n -l %U %H %C'
```
使用以下占位符：

| **%U** | 用户名。 |
| --- | --- |
| **%H** | 打开远程 shell 的主机。 |
| **%C** | 要在主机上执行的命令。 |

如果此参数设置为使用安全 shell，Abaqus 自动使用安全复制 (`scp`) 来回复制文件。默认情况下，此参数被忽略，而有利于内置 MPI `rsh/scp` 命令。

****order_parallel****

如果您未在 **abaqus** 命令行上指定并行排序模式，则为 Abaqus/Standard 中的默认直接求解器排序模式。如果此参数设置为 `OFF`，则不会并行执行求解器排序。如果此参数设置为 `ON`，则求解器排序将并行运行。并行求解器排序的默认值是 `ON`。

****parallel****

如果您未在 **abaqus** 命令行上指定并行方法，则为 Abaqus/Explicit 中的默认并行方程求解方法。可能的值是 `DOMAIN` 或 `LOOP`；默认值是 `DOMAIN`。

****standard_parallel****

如果您未在 **abaqus** 命令行上指定并行模式，则为 Abaqus/Standard 中的默认并行执行模式。如果此参数设置为 `ALL`，则元素操作和求解器都将并行运行。如果此参数设置为 `SOLVER`，则只有求解器将并行运行。默认并行执行模式是 `ALL`。

### 4.1.3 作业自定义参数

****abq_cosimulation_lower_port****

此变量指定可用于两个 Abaqus 分析之间协同仿真的 TCP/UDP 端口号范围的最低端口号；仅对 Abaqus/CAE 有效。默认值是 48000。

****abq_cosimulation_upper_port****

此变量指定可用于两个 Abaqus 分析之间协同仿真的 TCP/UDP 端口号范围的最高端口号；仅对 Abaqus/CAE 有效。如果未指定此值，则设置为比 **abq_cosimulation_lower_port** 高 1000。

****auto_calculate****

如果此参数设置为 `ON`，则在分析结束时，如果执行过程检测到需要输出数据库后处理，则会自动启动后处理计算器。如果此参数设置为 `OFF`，则即使执行过程检测到需要处理输出数据库文件，后处理计算器也不会在分析结束时运行。默认值是 `ON`。

****average_by_section****

此参数仅用于 Abaqus/Standard 分析。如果此参数设置为 `OFF`，则写入数据 (`.dat`) 文件和结果 (`.fil`) 文件的输出的平均区域基于元素结构。如果此参数设置为 `ON`，则平均区域也会考虑元素属性和材料常量的底层值。在具有许多截面和/或材料定义的问题中，默认值 `OFF` 通常会比非默认值 `ON` 提供更好的性能。请参阅["输出到数据和结果文件，"Abaqus Analysis User's Guide 第 4.1.2 节](../usb/usb-link.md#usb-out-oprintfile)，了解有关平均方案的更多详细信息。

****cae_error_limit****

此变量定义将从分析作业发送到 Abaqus/CAE 的最大错误消息数量；仅对 Abaqus/CAE 有效。默认值是 50。

****cae_no_parts_input_file****

此变量定义由 Abaqus/CAE 生成的输入文件的格式；仅对 Abaqus/CAE 有效。如果此变量设置为 `ON`，则 Abaqus/CAE 将生成没有部件和装配的输入文件。默认值是 `OFF`。有关更多信息，请参阅["编写没有部件和装配的输入文件，"Abaqus/CAE User's Guide 第 9.10.4 节](../usi/usi-link.md#usi-dbs-flatteninput)。

****cae_warning_limit****

此变量定义将从分析作业发送到 Abaqus/CAE 的最大警告消息数量；仅对 Abaqus/CAE 有效。默认值是 200。

****double_precision****

如果您未在 **abaqus** 命令行上指定精度版本，则要运行的 Abaqus/Explicit 默认精度版本。可能的值是 `EXPLICIT`（仅 Abaqus/Explicit 分析以双精度运行）、`BOTH`（Abaqus/Explicit 打包程序和分析都以双精度运行）、`CONSTRAINT`（Abaqus/Explicit 中的约束打包程序和约束求解器以双精度运行，而 Abaqus/Explicit 打包程序和分析继续以单精度运行）或 `OFF`（Abaqus/Explicit 打包程序和分析都以单精度运行）。默认是 `OFF`。

****max_history_requests****

此参数指定 Abaqus 分析中允许的最大历史请求数量。默认值是 10,000。Abaqus 中的历史输出旨在用于对模型小部分进行相对频繁的输出请求，并显示在 Abaqus/CAE 可视化模块 (Abaqus/Viewer) 的 X-Y 图中。请求大量历史输出会导致 Abaqus 作业的分析和后处理性能出现问题。对于向量或张量值输出变量，每个分量被视为单个请求。对于元素变量，历史输出将在每个积分点生成。例如，为 C3D10M 元素的张量变量 S（应力）请求历史输出将生成 24 个历史输出请求：（6 个分量）×（4 个积分点）。在为向量和张量值变量请求历史输出时，建议在适用的情况下选择各个分量。在需要大量历史输出的情况下，建议将数据写入输出数据库 (`.odb`) 作为场输出，从中可以使用 Abaqus/CAE 的可视化模块提取历史数据。

****odb_output_by_default****

如果此参数设置为 `ON`，则会自动生成输出数据库输出。如果此参数设置为 `OFF`，则必须在输入文件中放置输出数据库请求关键字以获取输出数据库输出并允许分析重新启动。默认值是 `ON`。

****onCaeGraphicsStartup****

在 Abaqus/CAE 或 Abaqus/Viewer 开始之前要执行的可选函数。此函数允许用户更改图形选项。请参阅["调整图形卡，"第 5.3 节](ch05s03.md)，了解有关此函数的更多信息。此函数通常不应更改。

****onCaeStartup****

在 Abaqus/CAE 开始之前要执行的可选函数。请参阅["自定义 Abaqus/CAE 启动，"第 4.3.3 节](ch04s03.md#sgb-env-exa-custom-startup)，了解此函数的示例。

****onDesignStartup****

在 Abaqus/Design 开始之前要执行的可选函数。

****onJobCompletion****

在 Abaqus 作业完成后要执行的可选函数。首先执行当前目录中 Abaqus 环境文件中指定的函数，然后执行用户主目录中的函数，最后执行 Abaqus 安装环境文件中的函数。同一环境文件中的多个函数将导致仅使用最后一个定义。请参阅["作业变量，"第 4.1.10 节](ch04s01.md#sgb-chp-env-job)，获取此函数可用变量的列表。

****onJobStartup****

在 Abaqus 作业开始之前要执行的可选函数。请参阅["作业变量，"第 4.1.10 节](ch04s01.md#sgb-chp-env-job)，获取此函数可用变量的列表。

****onViewerStartup****

在 Abaqus/Viewer 开始之前要执行的可选函数。

****printed_output****

默认情况下，所有 [*PREPRINT](../key/key-link.md#usb-kws-mpreprint) 参数的值都是 NO，不会将结果打印到数据文件。设置 **printed_output** 参数等于 `ON` 以在数据文件中获取与在输入文件中包含

```
*PREPRINT, CONTACT=YES, ECHO=YES, HISTORY=YES, MODEL=YES
```

相同的预打印信息。将 **printed_output` 设置为 `ON` 也可能导致大量表格结果打印到数据文件（除非使用打印输出控制选项来限制输出）。如果输入文件是关于部件和装配的，将 **printed_output** 设置为 `ON` 将导致零件-装配映射打印在数据文件中，而不考虑 [*PREPRINT](../key/key-link.md#usb-kws-mpreprint) 选项上的设置；这允许用户将打印输出与输入文件中定义的部件-装配关联起来。此变量的默认值是 `OFF`。

****run_mode****

如果用户在运行分析产品时未在 **abaqus** 命令行上指定运行模式，则为默认运行模式 (`INTERACTIVE`、`BACKGROUND` 或 `BATCH`)。默认值是 `BACKGROUND`。除非定义了批处理队列，否则不应将此变量设置为 `BATCH`。

****split_dat****

如果此变量设置为 `ON`，则数据文件将拆分为两部分。用户输入处理的输出将放在带有 `.pre` 扩展名的文件中。分析输出文件仍将具有 `.dat` 文件扩展名。默认值是 `OFF`。

****unconnected_regions****

如果此变量设置为 `ON`，则 Abaqus/Standard 会在 **datacheck** 分析期间为模型中未连接区域创建元素和节点集。使用此选项创建的元素和节点集名为 `MESH COMPONENT *N*`，其中 *N* 是分量编号。

### 4.1.4 系统自定义参数

****admin****

此参数防止对环境文件参数进行未经授权的修改。将此参数设置为您希望在较低级别 `abaqus_v6.env` 文件中无法更改的环境文件参数列表。除非另有说明，所有系统和作业自定义参数都可以被锁定。安装目录中的命令具有最高优先级，其次是用户主目录中的命令，然后是当前工作目录中的命令。因此，Abaqus 用户无法更改 Abaqus 帐户管理员锁定的环境文件命令。

****ask_delete****

如果此参数设置为 `OFF`，则不会询问用户是否应删除相同名称的旧作业文件；文件将自动删除。默认值是 `ON`。

****compile_cpp****

C++ 编译命令。SIMULIA 使用的命令包含在 Abaqus 安装的 `/SMA/site/` 子目录中的 `abaqus_v6.env` 文件中。此命令通常不应更改。它可以是字符串或字符串元组。如果命令是字符串元组，则每个字符串必须代表一个命令行参数。占位符的值由 Abaqus 执行过程或命令行选项确定，用户无法修改。占位符的值会替换 **compile_cpp** 字符串中的占位符。以下占位符可用：

| %`I` | 搜索 include 文件的目录。 |
| --- | --- |

****compile_fortran****

FORTRAN 编译命令。SIMULIA 使用的命令包含在 Abaqus 安装的 `/SMA/site/` 子目录中的 `abaqus_v6.env` 文件中。此命令通常不应更改。它可以是字符串或字符串元组。如果命令是字符串元组，则每个字符串必须代表一个命令行参数。

### 4.1.6 许可证管理参数

许可证管理自定义参数根据当前网络条件和用户需求控制 Abaqus 许可证服务器的行为。Abaqus 许可证服务器使用默认参数安装，这些参数应适合大多数用户。以下参数可用于自定义：

****abaquslm_license_file****

此参数提供运行 Abaqus FLEXnet 许可证服务器的计算机的主机名，并在产品安装期间设置。此参数不适用于 DSLS 许可证服务器。如果使用单个服务器，此参数应设置为 *port*@*license_server_host*，其中 *port* 是端口号，*license_server_host* 是运行服务器的计算机的名称。如果端口号在 27000 到 27009 之间，则不需要包含它。如果使用冗余服务器，则应将参数设置为一个列表，其中包含冗余服务器主机及其端口的名称。例如，如果一组三个冗余服务器的主机名是 maple、pine 和 oak，则以下参数定义是适当的：

```
abaquslm_license_file="27000@maple:27000@pine:27000@oak"
```
在 Linux 平台上使用冒号分隔主机名，在 Windows 平台上使用分号分隔主机名。

****academic****

此参数指示学术 Abaqus 客户端应使用研究还是教学许可证令牌。将此参数设置为 `TEACHING` 将强制 Abaqus 客户端仅使用教学许可证令牌。将此参数设置为 `RESEARCH` 或删除参数将强制 Abaqus 仅使用研究许可证令牌。此参数在产品安装期间自动设置：如果许可证服务器包含 Abaqus 教学许可证文件，则安装将参数设置为 `TEACHING`；否则，参数设置为 `RESEARCH`。

****cae_timeout****

Abaqus/CAE 或 Abaqus/Viewer 会话在将令牌返回到许可证服务器之前因无用户活动而保持空闲的分钟数。默认值是 60 分钟。

****computer_location****

指示本地客户端计算机位置的字符串。此参数允许您按位置细分许可证使用报告。许可证使用报告实用程序根据 **computer_location** 名称编译和组织数据。默认值是空字符串。如果不更改此默认值，许可证使用报告将不会在报告中区分不同位置。

****dsls_license_config****

Dassault Systèmes 许可证服务器 (DSLS) 配置文件 (`DSLicSrv.txt`) 的路径。此文件确定要与 Abaqus 一起使用的 Dassault Systèmes 许可证服务器。例如， 

**Linux 平台**：

```
/opt/simulia/license/DSLicSrv.txt
```

**Windows 平台**： （必须使用双反斜杠）

```
C:\\SIMULIA\\License\\DSLicSrv.txt
```
有关 `DSLicSrv.txt` 文件的更多信息，请参阅*Dassault Systèmes 许可证服务器安装和配置指南*（`DSLS.pdf`）中的"配置客户端"。

****license_server_type****

Abaqus 客户端使用的许可证服务器软件的类型。可能的值是 `FLEXNET`（默认值）和 `DSLS`。请参阅["Dassault Systèmes 许可证服务器安装，"第 2.1.2 节](ch02s01.md#sgb-chp-install-glance-lic-ds)和["FLEXnet 许可证安装，"第 2.1.3 节](ch02s01.md#sgb-chp-install-glance-lic)，了解更多信息。

****lmhanglimit****

如果当前没有可用的许可证，Abaqus 客户端在许可证队列中等待的分钟数。默认值 0 强制作业无限期留在许可证队列中，除非被用户终止。

****lminteractivequeuing****

此参数指示交互式 Abaqus/CAE 或 Abaqus/Viewer 会话是否应在没有许可证可用时排队等候。要允许交互运行的 Abaqus/CAE 或 Abaqus/Viewer 会话排队等候许可证，请将此参数设置为 `ON`。默认值是 `OFF`。**lmlicensequeuing** 参数用于为没有图形用户界面的排队会话。

****lmlicensequeuing****

此参数指示 Abaqus 分析作业或使用 **noGUI** 选项的 Abaqus/CAE 或 Abaqus/Viewer 会话是否应在没有许可证可用时排队等候。默认值是 `ON`。如果此参数设置为 `OFF`，则如果没有许可证可用，作业和 Abaqus/CAE 或 Abaqus/Viewer 会话将立即终止。**lminteractivequeuing** 参数用于为交互运行的会话排队。

****lmlog****

此参数指示是否应将许可证使用数据写入 FLEXnet 调试日志文件。要使用 Abaqus 许可证使用报告实用程序（请参阅["报告 FLEXnet 许可证使用数据，"第 3.8 节](ch03s08.md)），此参数必须设置为 `ON`（默认值）。要禁止调试日志文件中的许可证使用数据，请将此参数设置为 `OFF`。

****lmproject****

此可选参数可用于记录有关公司内部项目名称或编号的信息。**lmproject** 参数可以设置为任何字符串值；例如，`lmproject=``"*turbomachinery-project-23*"``。此参数可以在每个用户主目录中的环境文件中设置，可以随时编辑以更改为不同的项目名称。

有关 Abaqus 许可证检出和相关项目名称的信息记录在许可证服务器上，可以通过使用 `project` 访问器获取历史报告来检索。请参阅["使用 `reporttool` 实用程序，"第 3.10 节](ch03s10.md)。

****lmqueuesleep****

Abaqus 客户端在检查许可证队列以查看是否有足够空闲令牌之前等待的秒数。默认值是 30 秒，这是允许的最小值。增加此值将减少发生许可证排队时的网络流量。

****lmsvrdownlimit****

如果许可证服务器当前不可用，Abaqus 客户端尝试连接许可证服务器的分钟数。默认值 0 强制作业无限期尝试连接，除非被用户终止。

### 4.1.7 面向对象的批处理队列参数

用于定义分析批处理队列的面向对象接口易于自定义和扩展。分析批处理队列可从 Abaqus 执行过程命令行使用，一旦被实例化并插入到分析队列字典中。请参阅["定义分析批处理队列，"第 4.2 节](ch04s02.md)，了解更多信息。

****queues****

队列名称和对象的字典。队列名称及其对应的对象实例作为键/值对插入字典。特殊队列名称 **default** 可用于指定默认队列。当 **run_mode** 设置为 `BATCH` 且未在命令行上指定 **queue** 选项时，使用默认队列。当在命令行上指定了 **queue** 选项但命名队列为空白或在 **queues** 字典中不存在时，也使用默认队列。

### 4.1.8 基于字符串的批处理队列参数

用于定义分析批处理队列的基于字符串的接口允许访问固定数量的选项。以下参数可用于构造命令字符串来控制 Abaqus 分析的执行。唯一的要求是字符串必须是执行它的计算机系统上的有效命令。请参阅["定义基于字符串的分析批处理队列，"第 4.3.7 节](ch04s03.md#sgb-env-exa-stringbatchqueue)，了解示例。

****after_prefix****

当在命令行上指定 **after** 选项时，作为提交命令的一部分输出的可选前缀。当在命令行上指定 **after** 选项时，**%A** 被 **after_prefix** 字符串替换。

****queue_cmd****

当 **run_mode** 设置为 `BATCH` 时用于提交批处理作业的默认命令。此参数一旦交换了其占位符，必须在执行它的计算机系统上是有效命令。占位符的值由 Abaqus 执行过程的命令行上或环境文件参数中指定的输入定义。占位符的值会替换 **queue_cmd** 字符串中的占位符。以下占位符可用：

| **%%** | 百分号 (%) 字符。 |
| --- | --- |
| **%A** | 当在命令行上指定 **after** 选项时，**after_prefix** 字符串被替换为 **%A**。 |
| **%L** | 日志文件名。此令牌将替换为 `*job-name*.log`，其中 *job-name* 是 **job** 命令行选项定义的值。 |
| **%P** | 当在命令行上指定 **queue** 选项时，**queue_prefix** 字符串（见下文）被替换为 **%P**。 |
| **%Q** | **queue** 命令行选项中的队列名称。 |
| **%S** | 命令脚本文件名。此令牌将替换为 `*job-name*.com`，其中 *job-name* 是 **job** 命令行选项定义的值。 |
| **%T** | **after** 命令行选项中的时间。 |
| **%J** | **job** 命令行选项中的作业名称。 |
| **%E** | 此令牌被 Python 可执行文件的完整路径替换。 |
| **%O** | 输出目录的完整路径。 |

****queue_name****

批处理命令名称列表，通常用于将作业提交到默认队列（由 **queue_cmd** 定义）以外的队列。然后，在环境文件的其他地方，这些命令别名必须出现在等号左侧，右侧是所需的命令字符串。此命令字符串与 **queue_cmd** 参数具有相同的格式。只要最终结果是有效的系统命令，它就可以在其构造中使用可替换的占位符。

****queue_prefix****

当指定队列名称时，作为提交命令的一部分输出的可选前缀。默认值是空字符串。

### 4.1.9 Adams 翻译单位参数

**abaqus adams** 翻译器引用本节中的环境文件参数来指定在其创建的模态中性 (`.mnf`) 文件中使用的单位的默认值。有关 ADAMS/View 中有效单位规范的更多信息，请参阅["将 Abaqus 数据翻译为 MSC.ADAMS 模态中性文件，"Abaqus Analysis User's Guide 第 3.2.36 节](../usb/usb-link.md#usb-int-dabaadmproc-units)中的"单位"。这些参数仅提供默认值；如果您使用 **abaqus adams** 执行过程中的参数之一指定了单位偏好，则您的选择会覆盖默认设置。

****adams_units_family****

模型的单位系统类型。可能的值是 `mmks`、`mks`、`cgs` 或 `ips`，对应于同名的 ADAMS/View 选项。

****adams_length_units****

模态中性 (`.mnf`) 文件中为模型指定的长度单位。

****adams_mass_units****

模态中性 (`.mnf`) 文件中模型的质量单位。

****adams_time_units****

模态中性 (`.mnf`) 文件中的时间单位。

****adams_force_units****

模态中性 (`.mnf`) 文件中的力单位。

### 4.1.10 作业变量

以下变量可在 **onJobStartup** 或 **onJobCompletion** 函数中使用：

****id****

作为 **job** 命令行选项的值指定的作业标识符。

****savedir****

提交作业的目录的路径名。

****scrdir****

临时目录的路径名。

****analysisType****

要执行的 analysis type。可能的值是 `EXPLICIT` 和 `STANDARD`。

此外，对于基于 MPI 的并行作业，以下变量在 **onJobStartup** 或 **onJobCompletion** 函数中可用：

****host_list****

用于分析的主机计算机名称列表，包括每台机器上使用的处理器数量。格式与 **mp_host_list** 环境变量相同（请参阅["并行化参数，"第 4.1.2 节](ch04s01.md#sgb-chp-env-parallel)）。

****local_host****

用于确定提交作业的主机名称的标识符列表（例如，主机名、IP 地址、别名等）。

****rsh_command****

用于在分析期间使用的机器上打开远程 shell 的命令。格式与 **mp_rsh_command** 环境变量相同（请参阅["并行化参数，"第 4.1.2 节](ch04s01.md#sgb-chp-env-parallel)）。

****file_system****

一个元组，显示基于 MPI 的并行作业使用的文件系统类型。元组中的第一项指的是提交作业的目录，而第二项指的是作业的临时目录。对于不使用用户子程序的基于 MPI 的并行 Abaqus/Explicit 分析，除非用户设置，否则临时目录将保持为 `DETECT`。

****cpus****

跨所有主机机器用于分析的处理器总数。

以下变量在 **onJobStartup** 和 **onJobCompletion** 函数之外可用：

****abaqus_version****

包含 Abaqus 版本的字符串。

****analysisType****

要执行的 analysis type。可能的值是 `EXPLICIT` 和 `STANDARD`。

****applicationName****

要执行的 Abaqus 执行过程的名称；例如，`analysis`、`cae` 或 `viewer`。
