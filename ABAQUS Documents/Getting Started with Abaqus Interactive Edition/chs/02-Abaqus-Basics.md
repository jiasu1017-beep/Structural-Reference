## 2.1 Abaqus分析模型的组成部分

Abaqus模型由几个不同的组件组成，这些组件共同描述了要分析的实际问题和要获得的结果。分析模型至少由以下信息组成：离散化几何、单元截面属性、材料数据、载荷和边界条件、分析类型和输出请求。本章的讨论集中在结构应用上。流体力学也有类似的概念。

### 离散化几何

有限元和节点定义了在Abaqus中建模的物理结构的基本几何。模型中的每个单元代表物理结构的一个离散部分，而物理结构又由许多相互连接的单元表示。单元通过共享节点相互连接。节点的坐标和单元的连接性（即哪些节点属于哪些单元）构成了模型几何。模型中所有单元和节点的集合称为网格。通常，网格只是结构实际几何的近似。

单元类型、形状和位置，以及网格中使用的单元总数，都会影响从模拟获得的结果。网格密度越大（即网格中单元数量越多），结果越精确。随着网格密度的增加，分析结果收敛于唯一解，而分析所需的计算机时间也增加。从数值模型获得解通常是正在模拟的物理问题的解的近似。模型几何、材料行为、边界条件和载荷中近似的程度决定了数值模拟与物理问题的匹配程度。

### 单元截面属性

Abaqus有多种单元，其中许多单元的几何不能完全由节点坐标定义。例如，复合壳的层或工字梁截面的尺寸不是由单元的节点定义的。这些额外的几何数据被定义为单元的物理属性，对于完全定义模型几何是必要的（参见第三章"有限元和刚体"）。

### 材料数据

必须为所有单元指定材料属性。虽然高质量的材料数据通常很难获得，特别是对于更复杂的材料模型，但Abaqus结果的有效性受材料数据准确性和完整性的限制。

### 载荷和边界条件

载荷会使物理结构变形，从而在其中产生应力。最常见的载荷形式包括：

- 点载荷
- 表面上的压力载荷
- 表面上的分布牵引力
- 壳边缘上的分布边缘载荷和弯矩
- 体力，如重力
- 热载荷

边界条件用于约束模型的某些部分保持固定（零位移）或按指定量移动（非零位移）。

在静态分析中，必须使用足够的边界条件来防止模型沿任何方向作为刚体移动；否则，不受约束的刚体运动会导致刚度矩阵奇异。求解器问题会在求解阶段发生，并可能导致模拟过早停止。Abaqus/Standard如果在模拟期间检测到求解器问题，会发出警告消息。您学会解释此类错误消息是很重要的。如果您在静态应力分析期间看到"数值奇异"或"零枢轴"警告消息，应检查模型的全部或部分是否缺少对刚体平移或旋转的约束。刚体运动可以包括分量的平移和旋转。潜在的刚体运动取决于模型的维数。

| 维数 | 可能的刚体运动 |
|------|---------------|
| 三维 | 1、2和3方向的平移 |
|      | 绕1、2和3轴的旋转 |
| 轴对称 | 2方向的平移 |
|        | 绕3轴的旋转（仅轴对称刚体） |
| 平面应力 | 1和2方向的平移 |
| 平面应变 | 绕3轴的旋转 |

默认情况下，1、2和3方向与全局笛卡尔坐标系的轴对齐。

在动态分析中，惯性力会阻止模型立即发生无限运动，只要模型中的所有独立部分都有一些质量；因此，动态分析中的求解器问题警告通常表示其他建模问题，如过度塑性。

### 分析类型

Abaqus可以执行许多不同类型的模拟，但本指南只介绍两种最常见的：静态和动态应力分析。

在静态分析中，获得结构对施加载荷的长期响应。在其他情况下，可能对结构对载荷的动态响应感兴趣：例如，对组件上突然载荷的影响（如冲击期间）或建筑物在地震中的响应。

### 输出请求

Abaqus模拟可以生成大量输出。为了避免过度使用磁盘空间，您可以将输出限制为解释结果所需的内容。

通常，使用诸如Abaqus/CAE之类的预处理程序来定义模型的必要组件。

## 2.2 Abaqus/CAE简介

Abaqus/CAE是完整的Abaqus环境，提供了一个简单、一致的界面，用于创建Abaqus模型、交互式提交和监控Abaqus作业，以及评估Abaqus模拟的结果。Abaqus/CAE分为模块，每个模块定义了建模过程的逻辑方面；例如，定义几何、定义材料属性和生成网格。当您从一个模块移动到另一个模块时，您构建模型。当模型完成后，Abaqus/CAE生成一个输入文件，您可以将该文件提交给Abaqus分析产品。Abaqus/Standard或Abaqus/Explicit读取Abaqus/CAE生成的输入文件，执行分析，向Abaqus/CAE发送信息以允许您监控作业的进度，并生成输出数据库。最后，您使用可视化模块读取输出数据库并查看分析结果。

### 2.2.1 启动Abaqus/CAE

要启动Abaqus/CAE，您需要在操作系统提示符下输入命令：

```
abaqus cae
```

其中`abaqus`是用于运行Abaqus的命令。在您的系统上此命令可能不同。

当Abaqus/CAE启动时，会出现"开始会话"对话框（如图2-1所示）。以下会话启动选项可用：

- **创建模型数据库：使用Standard/Explicit模型**：允许您开始新的Abaqus/Standard或Abaqus/Explicit分析。
- **创建模型数据库：使用CFD模型**：允许您开始新的Abaqus/CFD分析。
- **创建模型数据库：使用电磁模型**：允许您开始新的电磁分析。
- **打开数据库**：允许您打开以前保存的模型或输出数据库文件。
- **运行脚本**：允许您运行包含Abaqus/CAE命令的文件。
- **开始教程**：允许您从在线文档开始入门教程。
- **最近文件**：允许您打开在Abaqus/CAE中最近打开的五个模型数据库文件或输出数据库文件之一。

### 2.2.2 主窗口的组成部分

您通过主窗口与Abaqus/CAE进行交互。图2-2显示了主窗口中出现的组件。

![图2-2 主窗口的组成部分](../graphics/gst-main-window-nls.png)

**图2-2** 主窗口的组成部分

组成部分包括：

#### 标题栏

标题栏指示您正在运行的Abaqus/CAE版本和当前模型数据库的名称。

#### 菜单栏

菜单栏包含所有可用菜单；菜单提供对产品中所有功能的访问。根据您从上下文栏选择的模块，菜单栏中会出现不同的菜单。

#### 工具栏

工具栏提供对菜单中也可用项目的快速访问。

#### 上下文栏

Abaqus/CAE分为一组模块，每个模块允许您处理模型的一个方面；上下文栏中的"模块"列表允许您在这些模块之间移动。上下文栏中的其他项目是您正在工作的模块的功能；例如，上下文栏允许您在创建模型几何时检索现有零件。

#### 模型树

模型树为您提供模型及其包含的对象（如零件、材料、步骤、载荷和输出请求）的图形概览。此外，模型树提供了在模块之间移动和管理对象的方便、集中工具。如果您的模型数据库包含多个模型，您可以使用模型树在模型之间移动。当您熟悉模型树后，您会发现可以快速执行主菜单栏、模块工具箱和各种管理器中找到的大多数操作。

#### 结果树

结果树为您提供输出数据库和其他会话特定数据（如X-Y图）的图形概览。如果您在一个会话中打开了多个输出数据库，您可以使用结果树在输出数据库之间移动。当您熟悉结果树后，您会发现可以快速执行可视化模块中主菜单栏和工具箱中找到的大多数操作。

#### 工具箱区域

当您进入模块时，工具箱区域显示适合该模块的工具箱中的工具。工具箱允许快速访问菜单栏中也可用的大多数模块功能。

#### 画布和绘图区域

画布可以想象为一个无限屏幕或公告板，您可以在上面发布视口。绘图区域是画布的可见部分。

#### 视口

视口是画布上Abaqus/CAE显示模型的窗口。

#### 提示区域

提示区域显示在过程期间遵循的说明；例如，当您创建集合时，它会要求您选择几何。

#### 消息区域

Abaqus/CAE在消息区域中打印状态信息和警告。要调整消息区域的大小，请拖动顶部边缘；要查看已滚出消息区域的信息，请使用右侧的滚动条。默认情况下会显示消息区域，但它使用与命令行界面相同的空间。

> **注意：** 如果在命令行界面处于活动状态时添加了新消息，Abaqus/CAE会将消息区域图标周围的背景颜色更改为红色。显示消息区域时，背景会恢复为正常颜色。

#### 命令行界面

您可以使用命令行界面键入Python命令并使用内置在Abaqus/CAE中的Python解释器计算数学表达式。该界面包括主提示符（`>>>`）和辅助提示符（`...`），以指示何时必须缩进命令以符合Python语法。

默认情况下命令行界面是隐藏的，但它使用与消息区域相同的空间。单击主窗口左下角的"命令"标签可以从消息区域切换到命令行界面。单击"消息"标签返回消息区域。

### 2.2.3 什么是模块？

如前所述，Abaqus/CAE分为称为模块的功能单元。每个模块仅包含与建模任务特定部分相关的工具。例如，"网格"模块仅包含创建有限元网格所需的工具，而"作业"模块仅包含用于创建、编辑、提交和监控分析作业的工具。

您可以从上下文栏中的"模块"列表中选择模块（如图2-3所示）。

![图2-3 选择模块](../graphics/gst-module-list-nls.png)

**图2-3** 选择模块

菜单中模块的顺序对应于您可能遵循的创建模型的逻辑序列。在许多情况下，您必须遵循这个自然顺序来完成建模任务；例如，您必须在创建装配之前创建零件。尽管模块的顺序遵循逻辑序列，但Abaqus/CAE允许您随时选择任何模块，而不管模型的状态如何。但是，某些明显的限制适用；例如，您不能将截面属性（如工字梁的横截面尺寸）分配给尚未创建的几何。

完成的模型包含Abaqus开始分析所需的所有内容。Abaqus/CAE使用模型数据库来存储您的模型。当您启动Abaqus/CAE时，"开始会话"对话框允许您在内存中创建新的空模型数据库。启动Abaqus/CAE后，您可以通过从主菜单栏选择**File** → **Save**将模型数据库保存到磁盘；要从磁盘检索模型数据库，请选择**File** → **Open**。

以下列表简要介绍了Abaqus/CAE中可用的模块以及您可以在每个模块中执行的建模任务：

#### Part（零件）模块

零件模块允许您通过直接在Abaqus/CAE中绘制几何或从其他几何建模程序导入几何来创建单个零件。

#### Property（属性）模块

截面定义包含有关零件或零件区域属性的信息，如区域的关联材料定义和横截面几何。在属性模块中，您可以创建截面和材料定义，并将其分配给零件区域。

#### Assembly（装配）模块

当您创建零件时，它存在于自己的坐标系中，独立于模型中的其他零件。您使用装配模块来创建零件实例以及其他模型的实例，并相对于全局坐标系中的彼此定位实例，从而创建装配。Abaqus模型仅包含一个装配。

#### Step（步骤）模块

您使用步骤模块来创建和配置分析步骤以及相关的输出请求。步骤序列提供了一种方便的方式来捕获模型中的更改（如载荷和边界条件更改）；输出请求可以根据需要在步骤之间变化。

#### Interaction（相互作用）模块

在相互作用模块中，您可以指定模型区域之间或模型区域与其周围环境之间的机械和热相互作用。相互作用的示例包括两个表面之间的接触。其他可以定义的相互作用包括约束，如绑定、方程和刚体约束。Abaqus/CAE不识别零件实例或装配区域之间的机械接触，除非在相互作用模块中指定了这种接触。装配中两个表面的简单物理接近不足以指示表面之间的任何类型的相互作用。相互作用是步骤相关对象，这意味着您必须指定它们处于活动状态的分析步骤。

#### Load（载荷）模块

载荷模块允许您指定载荷、边界条件和预定义场。载荷和边界条件是步骤相关对象，这意味着您必须指定它们处于活动状态的分析步骤；一些预定义场是步骤相关的，而其他仅在分析开始时应用。

#### Mesh（网格）模块

网格模块包含允许您在Abaqus/CAE内创建的装配上生成有限元网格的工具。可提供各种级别的自动化和控制，以便您可以创建满足分析需求的网格。

#### Optimization（优化）模块

优化模块允许您创建优化任务，该任务可用于根据一组目标和约束来优化模型拓扑。

#### Job（作业）模块

完成定义模型的所有任务后，您可以使用作业模块来分析模型。作业模块允许您交互式提交作业进行分析并监控其进度。可以同时提交和监控多个模型和运行。

#### Visualization（可视化）模块

可视化模块提供有限元模型和结果的图形显示。它从输出数据库获取模型和结果信息；您可以通过在步骤模块中修改输出请求来控制写入输出数据库的信息。

#### Sketch（草图）模块

草图是用于帮助形成定义Abaqus/CAE本机零件几何的二维轮廓。您使用草图模块来创建定义平面零件、梁的草图，或创建分区，或创建可以拉伸、扫掠或旋转以形成三维零件的草图。

主窗口的内容会随着您在模块之间移动而变化。从上下文栏中的"模块"列表中选择模块会导致上下文栏、模块工具箱和菜单栏更改以反映当前模块的功能。

每个模块在本指南中提供的教程示例中会有更详细的讨论。

### 2.2.4 什么是模型树？

模型树提供模型中项目层次结构的可视化描述。它位于主窗口左侧的"模型"标签下方。图2-4显示了一个典型的模型树。

![图2-4 模型树](../graphics/uss-int-tree-nls.png)

**图2-4** 模型树

模型树中的项目由小图标表示；例如，"步骤"图标。此外，项目旁边的括号表示该项目是一个容器，括号中的数字表示容器中的项目数。您可以单击模型树中的"+"和"-"符号来展开和折叠容器。左右箭头键执行相同的操作。

模型树中容器和项目的排列反映了您创建模型的预期顺序。如前所述，模块菜单中模块的顺序遵循类似的逻辑——您先创建零件再创建装配，先创建步骤再创建载荷。这种排列是固定的——您不能移动模型树中的项目。

模型树提供主菜单栏和模块管理器的大部分功能。例如，如果您双击"零件"容器，您可以创建新零件（等同于从主菜单栏选择**Part** → **Create**）。

本指南中讨论的示例说明将侧重于使用模型树访问Abaqus/CAE的功能。只有在必要时才会考虑菜单栏操作（例如，在创建有限元网格或后处理结果时）。

结果树使用与模型树相同的空间。单击主窗口左侧的"结果"标签可以从模型树切换到结果树。结果树提供对会话特定功能的访问（即仅在可视化模块中可用的功能）。结果树将在本指南包含的后处理练习过程中介绍。

## 2.3 示例：创建起重葫芦模型

这个起重葫芦示例（如图2-5所示）将引导您完成Abaqus/CAE建模过程，使用模型树并向您展示创建和分析简单模型的基本步骤。葫芦是一个简单的销接头桁架模型，左端被约束，右端安装在滚轮上。构件可以在接头处自由旋转。框架被防止面外移动。首先在Abaqus/Standard中进行模拟，以确定结构在施加10 kN载荷（如图2-5所示）时的静态挠度和其构件中的峰值应力。第二次在Abaqus/Explicit中进行模拟，假设载荷突然施加，以研究框架的动态响应。

![图2-5 起重葫芦示意图](../graphics/gsx-hoist-schematic-nls.png)

**图2-5** 起重葫芦示意图

对于起重葫芦示例，您将执行以下任务：

- 绘制两维几何并创建表示框架的零件。
- 定义框架的材料属性和截面属性。
- 装配模型。
- 配置分析过程和输出请求。
- 将载荷和边界条件应用于框架。
- 为框架生成网格。
- 创建作业并提交进行分析。
- 查看分析结果。

Abaqus提供了复制此问题完整分析模型的脚本。如果您遇到按照以下说明操作的困难或希望检查您的工作，请运行这些脚本之一。

#### 单位

在开始定义此模型或任何模型之前，您需要决定将使用哪套单位。Abaqus没有内置的单位系统。在Abaqus中输入数据时不要包括单位名称或标签。所有输入数据必须以一致的单位指定。

| 物理量 | SI | SI (mm) | US Unit (ft) | US Unit (inch) |
|--------|-----|---------|--------------|----------------|
| 长度 | m | mm | ft | in |
| 力 | N | N | lbf | lbf |
| 质量 | kg | tonne (10³ kg) | slug | lbf s²/in |
| 时间 | s | s | s | s |
| 应力 | Pa (N/m²) | MPa (N/mm²) | lbf/ft² | psi (lbf/in²) |
| 能量 | J | mJ (10⁻³ J) | ft lbf | in lbf |

## 2.4 隐式和显式程序的比较

Abaqus/Standard和Abaqus/Explicit都能够解决各种问题。隐式和显式程序的特性决定了哪种方法适合给定问题。对于可以用任一种方法解决的问题，效率可以决定使用哪种产品。理解隐式和显式程序的特性将帮助您回答这个问题。表2-2列出了分析产品之间的关键差异。

**表2-2** Abaqus/Standard和Abaqus/Explicit之间的关键差异

| 量 | Abaqus/Standard | Abaqus/Explicit |
|----|-----------------|----------------|
| 单元库 | 提供广泛的单元库 | 提供适合显式分析的广泛单元库。可用的单元是Abaqus/Standard中可用单元的子集 |
| 分析过程 | 提供通用和线性扰动过程 | 提供通用过程 |
| 材料模型 | 提供广泛的材料模型 | 与Abaqus/Standard中可用的相似；一个显著区别是允许失效材料模型 |
| 接触公式 | 具有解决接触问题的稳健能力 | 具有稳健的接触功能，可轻松解决即使是最复杂的接触模拟 |
| 求解技术 | 使用基于刚度的求解技术，该技术是无条件稳定的 | 使用显式积分求解技术，该技术是条件稳定的 |
| 磁盘空间和内存 | 由于增量中可能的迭代次数较多，磁盘空间和内存使用可能很大 | 磁盘空间和内存使用通常比Abaqus/Standard小得多 |

### 2.4.1 在隐式和显式分析之间选择

对于许多分析，使用Abaqus/Standard还是Abaqus/Explicit是清楚的。例如，如第八章"非线性"所示，Abaqus/Standard在解决平滑非线性问题时更高效；另一方面，Abaqus/Explicit是波传播分析的明确选择。然而，某些静态或准静态问题可以用任一程序很好地模拟。通常，这些是通常使用Abaqus/Standard解决但可能因接触或材料复杂性而难以收敛的问题，导致大量迭代。在Abaqus/Standard中，这样的分析代价昂贵，因为每次迭代需要求解一个大型线性方程组。

虽然Abaqus/Standard必须迭代来确定非线性问题的解，但Abaqus/Explicit通过从前一个增量结束时的运动学状态显式推进来确定解，而无需迭代。即使给定的分析使用显式方法可能需要大量时间增量，如果同一分析在Abaqus/Standard中需要多次迭代，则在Abaqus/Explicit中可能更高效。

Abaqus/Explicit的另一个优势是它比Abaqus/Standard需要更少的磁盘空间和内存。当两个程序 computational cost 可比时，Abaqus/Explicit的显著磁盘空间和内存节省使其具有吸引力。

### 2.4.2 隐式和显式分析中网格细化的成本

使用显式方法，计算成本与单元数量成正比，大约与最小单元尺寸成反比。因此，网格细化通过增加单元数量和减小最小单元尺寸来增加计算成本。例如，考虑一个具有均匀方形单元的三维模型。如果在所有三个方向上网格细化因子为2，则由于单元数量的增加，计算成本增加2×2×2倍，由于最小单元尺寸的减小，再增加2倍。分析的总计算成本通过细化网格增加2⁴或16倍。磁盘空间和内存需求与单元数量成正比，与单元尺寸无关；因此，这些需求增加8倍。

虽然使用显式方法预测网格细化的成本增加相当直接，但使用隐式方法时更难预测。困难来自于单元连接性与求解成本之间的问题依赖关系，这种关系在显式方法中不存在。使用隐式方法，经验表明对于许多问题，计算成本大约与自由度数量的平方成正比。考虑相同的具有均匀方形单元的三维模型示例。在所有三个方向上网格细化因子为2会将自由度数量增加约2³，导致计算成本增加约(2³)²或64倍。磁盘空间和内存需求以相同的方式增加，尽管实际增加难以预测。

只要网格相对均匀，显式方法在模型规模增加时比隐式方法显示更大的成本节省。图2-20说明了使用显式和隐式方法时成本与模型规模的比较。对于这个问题，自由度数量与单元数量成比例。

## 2.5 小结

- Abaqus/CAE可用于创建完整的Abaqus分析模型。分析产品（Abaqus/Standard或Abaqus/Explicit）读取Abaqus/CAE生成的输入文件，执行分析，向Abaqus/CAE发送信息以允许您监控作业的进度，并生成输出数据库。您使用可视化模块读取输出数据库并查看分析结果。
- 创建模型后，您可以执行数据检查分析。错误和警告消息打印到"作业监视器"对话框。
- 使用Abaqus/CAE中的可视化模块图形验证模型几何和边界条件，使用数据检查阶段生成的输出数据库文件。
- 始终检查结果是否满足基本工程原理，如平衡。
- Abaqus/CAE的可视化模块允许您以各种方式图形化地可视化分析结果，还允许您编写表格数据报告。
- 在隐式或显式方法之间选择主要取决于问题的性质。
