# 3.5 FLEXnet 许可证服务器管理器 lmgrd



FLEXnet 许可证服务器管理器 `lmgrd` 处理与客户端应用程序的初始联系，将连接传递给适当的供应商守护程序。它还启动和重启供应商守护程序。

`lmgrd` 程序是 FLEXnet Licensing 的主守护程序。当您调用 `lmgrd` 时，程序会查找包含有关供应商和功能信息的许可证文件。在 Linux 平台上，建议您以非特权用户（不是 root）身份运行 `lmgrd`。

有关 FLEXnet Licensing 的更多信息，请参阅 FLEXnet Licensing End User Guide Version 11.6。此指南可从 **Support** 页面上的 **Licensing** 部分下载，网址为 [www.3ds.com/simulia](http://www.3ds.com/simulia)。

**语法和选项**

```
`lmgrd` [`-c *license_file_list*`] [`-l [+]*debug_log_path*`] [`-2 -p`] [`-local`] [`-x lmdown`] [`-x lmremove`] [`-z`] [`-v`] [`-help`]
```

**`-c *license_file_list*`**

指定要使用的许可证文件。

**`-l [+]*debug_log_path*`**

将调试信息写入 *debug_log_path* 文件。此选项使用字母 `l`，而不是数字 1。在 *debug_log_path* 前添加 + 字符会追加日志条目。

**`-2 -p`**

将 `lmdown`（请参阅["lmdown，"第 3.6.2 节](ch03s06.md#sgb-chp-net-lic-lmdown)）、`lmreread`（请参阅["lmreread，"第 3.6.6 节](ch03s06.md#sgb-chp-net-lic-lmreread)）和 `lmremove`（请参阅["lmremove，"第 3.6.5 节](ch03s06.md#sgb-chp-net-lic-lmremove)）的使用限制为 FLEXnet Licensing 管理员（默认为 root）。如果有一个名为"lmadmin"的 Linux 组，则使用受限属于该组的成员。如果 root 不是该组的成员，则 root 没有权限使用上面列出的任何三个工具。如果您在启动 `lmgrd` 时使用 `-2 -p`，则在 Windows 平台上没有任何用户可以使用 `lmdown` 关闭许可证服务器。

**`-local`**

限制 `lmdown` 和 `lmreread` 只能从运行 `lmgrd` 的同一台计算机上运行。

**`-x lmdown`**

禁用 `lmdown` 命令（没有任何用户可以运行 `lmdown`）。如果 `lmdown` 被禁用，您需要通过 `kill` *pid*（Linux 平台）或通过 Windows **任务管理器**或 Windows 服务停止 `lmgrd` 和供应商守护程序进程来停止 `lmgrd`。在 Linux 平台上，验证 `kill` 命令不包含 `-9` 选项。

**`-x lmremove`**

禁用 `lmremove` 命令（没有任何用户可以运行 `lmremove`）。

**`-z`**

在前台运行。默认行为是在后台运行。如果存在 `-l *debug_log_path*`，则不使用窗口。如果没有指定 `-l` 选项，则会为 `lmgrd` 和每个供应商守护程序使用单独的窗口。

**`-v`**

打印 `lmgrd` 版本号和版权信息并退出（不启动 `lmgrd`）。

**`-help`**

显示使用信息并退出。
