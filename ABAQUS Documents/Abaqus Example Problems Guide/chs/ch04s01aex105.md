# 4.1.1 多体机构模型中过约束的解决

**产品：** Abaqus/Standard   

当多个一致或不一致的运动约束被施加到同一个自由度时，就会发生过约束。过约束可能导致不准确的解或阻止收敛。大量过约束情况会在预处理阶段或 Abaqus/Standard 分析期间被自动检测并最终解决（参见"过约束检查，" Abaqus Analysis User's Guide 的第 35.6.1 节）。预处理阶段未解决的绝大多数过约束都会被方程求解器检测到。Abaqus/Standard 中以下症状可识别此类过约束模型：
- 在消息（.msg）文件中发出的零主元警告消息，表明方程组是秩亏的。
- 不合理的大的反作用力。
- 消息文件中非常大的时间平均力。
- 位移解违反施加的约束。

默认情况下，过约束检查由方程求解器在分析过程中持续执行。Abaqus/Standard 不会解决这些过约束。相反，会发出详细消息，描述生成过约束的建模特征，以帮助用户解决问题。消息首先使用求解器中高斯消除的零主元信息来识别涉及一致或不一致过约束的节点。

### 几何与模型

本示例涉及解决多体机构模型中的过约束。该模型由九个通过连接器元件互连的刚体组成。图中命名的刚体连接方式如下：
- LINK 通过两个 CVJOINT（JOIN + CONSTANT VELOCITY）连接器元件连接到 CRANK 和 DISH。每个刚体绕其自身轴旋转。
- TRANS CONNECT 使用 JOIN 连接器元件连接到 DISH。TRANS CONNECT 还通过 TRANSLATOR（SLOT + ALIGN）连接器元件被约束为沿 TRANS GUIDE 定义的方向平移。此外，TRANS CONNECT 通过 HINGE（JOIN + REVOLUTE）连接连接到 SLIDE BLOCK。
- SLIDE BLOCK 依次使用 TRANSLATOR（SLOT + ALIGN）连接器元件被约束沿 SLIDE GUIDE 滑动。
- TRANS GUIDE 和 ROD GUIDE 使用 CYLINDRICAL（SLOT + REVOLUTE）连接器元件连接。
- ROD 使用 TRANSLATOR（SLOT + ALIGN）连接器元件被允许在 ROD GUIDE 中滑动。

### 载荷与边界条件

刚体 ROD 的参考节点 10009 和刚体 SLIDE GUIDE 的参考节点 10006 被完全固定。此外，刚体 DISH 的参考节点 10001 和刚体 CRANK 的参考节点 10003 沿全局 X 和 Y 方向的平移和旋转被约束。机构使用边界条件来施加绕全局 Z 方向 360 度的旋转。

### 理解过约束消息

当 Abaqus/Standard 尝试为此模型寻找解时，在分析的第一增量中识别出两个零主元，表明模型中有两个过约束。必须识别并移除这些过约束才能使模型得到正确约束。

### 修正过约束模型

自动生成包含与特定零主元相关的约束链中所有节点的节点集。在大多数过约束模型中，有许多方法可以解决过约束。

### 结果与讨论

识别过约束的另一种方法是绘制受限自由度处的反作用力。在过约束模型中，即使位移解看起来可接受，这些力也非常大。一旦解决过约束并重新运行分析，消息文件中不会生成零主元，连接器反作用力变得非常小。

### 输入文件

- mbmech_overconst.inp - 过约束多体机构模型
- mbmech_overconst_crig.inp - 使用 CONNECTOR ELASTICITY, RIGID 选项的过约束多体机构模型
- mbmech_resolved.inp - 解决过约束的多体机构模型
- mbmech_resolved_crig.inp - 使用 CONNECTOR ELASTICITY, RIGID 选项解决过约束的多体机构模型
- mbmech_nodedefs.inp - 节点定义
- mbmech_elemdefs.inp - 单元定义
- mbmech_rigbdefs.inp - 刚体定义
- mbmech_conndefs_oc.inp - 过约束模型中使用的连接器元件定义
- mbmech_conndefs_res.inp - 已解决模型中使用的连接器元件定义
- mbmech_cardan_elas.inp - 使用 JOIN + CARDAN 连接的多体机构模型
- mbmech_cardan_elas_parts.inp - 部件定义
