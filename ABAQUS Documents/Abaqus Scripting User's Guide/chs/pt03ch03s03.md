# 3.3 列表和组合框





本节描述Abaqus GUI Toolkit中允许您从列表中选择一个或多个项目的widget。
- 当GUI中有足够空间且同时显示所有或大多数选项会有帮助时，您使用列表widget。
- 当需要节省GUI空间且仅显示当前选项时，您使用组合框。

### 3.3.1 列表

`AFXList`允许从其项目中选择一个或多个。

`AFXList`创建的列表支持以下选择策略：

**LIST_SINGLESELECT**

用户可以选择零个或一个项目。

**LIST_BROWSESELECT**

始终选择一个项目。

**LIST_MULTIPLESELECT**

用户可以选择零个或多个项目。

**LIST_EXTENDEDSELECT**

用户可以选择零个或多个项目；允许拖动、Shift和Ctrl选择。

`AFXDialog`基类有专门处理列表双击消息的代码。如果用户在列表中双击，对话框首先尝试调用**Apply**按钮消息处理程序。如果未找到**Apply**按钮消息处理程序，对话框尝试调用**Continue**按钮消息处理程序。如果未找到**Continue**按钮消息处理程序，对话框尝试调用**OK**按钮消息处理程序。因此，您不需要在脚本中执行任何操作来获得此行为。

但是，如果您有特殊的双击处理需求，可以通过将AFXLIST_NO_AUTOCOMMIT指定为列表的选项标志之一来关闭此双击行为。如果关闭双击行为，则必须在对话框中捕获列表的SEL_DOUBLECLICKED消息并适当处理。

**注意：**因为列表可能与其他类型的widget结合使用，列表不会在其周围绘制边框。因此，如果您想要列表周围的边框，必须通过将列表放在其他widget（如框架）内来提供边框。如果您不想要水平滚动条，请使用HSCROLLING_OFF标志；此标志强制列表将其宽度调整到其最宽项目的宽度。

以下是垂直框架内列表的示例：

```
vf = FXVerticalFrame(parent, FRAME_THICK|FRAME_SUNKEN,
    0, 0, 0, 0, 0, 0, 0, 0)
list = AFXList(vf, 3, tgt, sel, LIST_BROWSESELECT|HSCROLLING_OFF)
list.appendItem('Thin')
list.appendItem('Medium')
list.appendItem('Thick')
```

### 3.3.2 组合框

`AFXComboBox`从其项目中提供一种选择。`AFXComboBox`将只读文本字段与下拉列表相结合。

在父参数之后，`AFXComboBox`构造函数的接下来三个参数是文本字段的宽度、列表暴露时可见的项目数以及标签。如果您将宽度指定为零，组合框将自动调整为列表中最宽项目的宽度。例如：

```
comboBox = AFXComboBox(p, 0, 3, 'AFXComboBox:')
comboBox.appendItem('Item 1')
comboBox.appendItem('Item 2')
comboBox.appendItem('Item 3')
```

### 3.3.3 列表框

`AFXListBox`widget从其项目中提供一种选择。`AFXListBox`与`AFXComboBox`的不同之处在于`AFXListBox`显示的项目可以包含图标。例如：

```
listBox = AFXListBox(parent, 3, 'AFXListBox:', keyword)
listBox.appendItem('Item 1', thinIcon)
listBox.appendItem('Item 2', mediumIcon)
listBox.appendItem('Item 3', thickIcon)
```
