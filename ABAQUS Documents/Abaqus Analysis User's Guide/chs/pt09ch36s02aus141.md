### 概述

Abaqus/Standard 中通用接触的接触初始化控制：
- 可用于指定初始过盈量是否应在不产生应力应变的情况下消除，或作为干扰配合在一系列增量中逐渐消除；以及
- 可用于指定非默认搜索区域，以确定在应变自由调整或干扰配合情况下哪些节点会受到影响。

Abaqus/Standard 根据初始几何状态中的间隙或穿透状态初始化接触状态。默认情况下，小初始接触过盈量通过表面节点的应变自由位置调整来消除。您可以定义替代接触初始化方法，然后将其分配给接触相互作用。例如，您可以选择将某些相互作用的初始过盈量作为干扰配合处理。

### 默认接触初始化方法

默认情况下，通用接触算法在预处理期间调整表面节点的初始位置，以消除小的初始表面过盈量而不产生模型中的应变或应力，如图 [图 36.2.4-1](pt09ch36s02aus142.md#aadjustsurf-adj) 所示。这些调整旨在纠正与网格生成相关的微小不匹配。

**图 36.2.4-1** 应变自由调整以消除过盈量后的接触表面配置。

![](../graphics/aadjustsurf-zero.png)

通用接触自动为接触相互作用分配主从角色，如 ["Abaqus/Standard 中通用接触的数值控制，" 第 36.2.6 节](pt09ch36s02aus144.md) 中所述。Abaus/Standard 根据从表面上海支单元 facet 的大小计算过盈量容差。如果两个表面在初始状态下被过盈量小于计算容差的距离闭合，则从表面的节点将被重新定位到关联的主表面上（使用应变自由调整）。默认情况下，初始间隙不会通过默认调整进行更改。如果从表面的某部分初始过盈量大于计算容差，Abaqus/Standard 会自动为该表面部分及其关联的主表面生成接触排除。因此，通用接触不会在模型初始配置中严重过盈的表面（或表面部分）之间创建相互作用，这些表面可以在整个分析过程中自由穿透。

通用接触使用有限滑动、面-面接触 formulation，因此穿透/间隙计算是在有限区域上作为平均值计算的；因此，在调整后，个别表面节点上可能存在穿透和间隙。默认调整不会解决与壳或膜相关的两个参考表面的初始交叉情况，尽管 ["为壳表面分配接触初始化"](pt09ch36s02aus142.md#usb-cni-agenlcontinitializationstd-shells) 中讨论了解决此类情况的技术。

### 定义替代接触初始化方法

如果默认行为不符合要求，您可以定义替代接触初始化方法。例如，您可能希望增加深度穿透的容差，或指定某些开口应调整为"刚好接触"状态。此外，某些分析要求将初始过盈量作为干扰配合处理，而不是使用应变自由调整来消除。要修改接触初始化行为，您必须定义一个或多个替代接触初始化方法，然后识别哪些表面配对将使用哪些方法。

您为每个接触初始化方法分配一个名称。此名称用于将接触初始化方法分配给特定表面配对（见下文 ["分配接触初始化方法"](pt09ch36s02aus142.md#usb-cni-agenlcontinitializationstd-assign)）。

| **输入文件用法：** | ``` [*CONTACT INITIALIZATION DATA](../key/key-link.md#usb-kws-hcontactinitdata), NAME=*contact_initialization_method_name* ``` |
| --- | --- |

| **Abaqus/CAE 用法：** | Interaction 模块： ****Interaction**![](../graphics/images/arrow.gif)**Contact Initialization**![](../graphics/images/arrow.gif)**Create****: **Name:** *contact_initialization_method_name* |
| --- | --- |

#### 增加应变自由调整的搜索区域

如上所述，在 ["默认接触初始化方法"](pt09ch36s02aus142.md#usb-cni-agenlcontinitializationstd-default) 中，表面之间的初始间隙和大的初始过盈量不会通过默认接触初始化方法进行调整。您可以选择指定非默认搜索距离，分别在相互作用中表面的上方和下方；在这些搜索距离内的从表面将使用应变自由节点调整直接重新定位到其关联的主表面上。Abaqus/Standard 在计算这些搜索距离时会考虑壳厚度。

指定表面上方的搜索距离用于封闭表面之间的小初始间隙。指定表面下方的搜索距离用于增加 Abaqus/Standard 在执行应变自由调整时使用的默认过盈量容差；如果您指定的搜索距离小于默认过盈量容差，Abaqus/Standard 将使用默认容差。与默认初始化行为一样，对于大于指定搜索区域的初始过盈量，将创建接触排除。

增加应变自由调整的搜索区域范围可能会增加分析的计算成本。通常不建议您指定大的搜索区域，因为这可能导致节点在大距离上重新定位时网格失真。

| **输入文件用法：** | ``` [*CONTACT INITIALIZATION DATA](../key/key-link.md#usb-kws-hcontactinitdata), SEARCH ABOVE=*a*, SEARCH BELOW=*b* ``` |
| --- | --- |

| **Abaqus/CAE 用法：** | Interaction 模块： ****Interaction**![](../graphics/images/arrow.gif)**Contact Initialization**![](../graphics/images/arrow.gif)**Create****: **Resolve with strain-free adjustments**: **Ignore overclosures greater than:** *b*, **Ignore initial openings greater than:** *a* |
| --- | --- |

#### 指定初始间隙距离

默认情况下，上述应变自由调整会调整初始节点位置，使表面"刚好接触"（零穿透/分离）。或者，您可以指定一个初始间隙距离，Abaqus/Standard 会进行调整以达到您指定的初始间隙距离。调整仅适用于满足上述搜索区域容差的区域。如果需要大的应变自由调整才能达到指定的初始间隙距离，可能会发生网格失真。

| **输入文件用法：** | ``` [*CONTACT INITIALIZATION DATA](../key/key-link.md#usb-kws-hcontactinitdata), INITIAL CLEARANCE=*h* ``` |
| --- | --- |

| **Abaqus/CAE 用法：** | Interaction 模块： ****Interaction**![](../graphics/images/arrow.gif)**Contact Initialization**![](../graphics/images/arrow.gif)**Create****: **Specify clearance distance:** *h* |
| --- | --- |

#### 建立干扰配合模型

可选地，Abaqus/Standard 中的通用接触算法可以将初始过盈量作为干扰配合处理。通用接触算法使用热配合方法在分析的第一步（如果第一步使用多个载荷增量）逐渐消除干扰距离，如图 [图 36.2.4-2](pt09ch36s02aus142.md#acontact-interfer-grad) 所示，这样在特定增量完成时消除的干扰比例大约对应于完成的步长比例。干扰消除时会产生应力和应变。与总是在第一个增量中完全消除干扰（这是接触对的默认行为）相比，在多个增量中逐渐消除干扰可以提高"干扰配合加载"发生非线性响应情况下的稳健性。通常建议您在干扰配合消除期间不施加其他载荷。

**图 36.2.4-2** 接触干扰配合的逐步消除。

![](../graphics/acontact-interfer-fit-nls.png)

因为对于通用接触使用的面-面接触 formulation，接触条件是在每个约束位置周围区域以平均值 enforced，因此在从节点处可能会观察到穿透或间隙，而面-面约束处于零穿透状态。

| **输入文件用法：** | ``` [*CONTACT INITIALIZATION DATA](../key/key-link.md#usb-kws-hcontactinitdata), INTERFERENCE FIT ``` |
| --- | --- |

| **Abaqus/CAE 用法：** | Interaction 模块： ****Interaction**![](../graphics/images/arrow.gif)**Contact Initialization**![](../graphics/images/arrow.gif)**Create****: **Treat as interference fits** |
| --- | --- |

##### 增加干扰配合的容差

Abaqus/Standard 根据从表面上海支单元 facet 的大小计算过盈量容差（见上文 ["默认接触初始化方法"](pt09ch36s02aus142.md#usb-cni-agenlcontinitializationstd-default)）。两个表面之间的干扰配合仅影响那些过盈量小于计算容差的从表面；对于过盈量大于计算容差的表面，接触将被完全忽略。

可选地，您可以重新定义过盈量容差，以将更大的过盈量包含在干扰配合中。如果您指定的容差小于默认计算容差，Abaqus/Standard 将使用默认计算容差。

| **输入文件用法：** | ``` [*CONTACT INITIALIZATION DATA](../key/key-link.md#usb-kws-hcontactinitdata), INTERFERENCE FIT, SEARCH BELOW=*b* ``` |
| --- | --- |

| **Abaqus/CAE 用法：** | Interaction 模块： ****Interaction**![](../graphics/images/arrow.gif)**Contact Initialization**![](../graphics/images/arrow.gif)**Create****: **Treat as interference fits**: **Ignore overclosures greater than:** *b* |
| --- | --- |

##### 指定干扰距离

默认情况下，干扰距离由网格的初始过盈量暗示；或者，您可以指定干扰距离。在这种情况下，Abaqus/Standard 首先对节点位置进行应变自由调整，使调整后配置中的初始过盈量对应于指定的干扰距离，然后调用上面讨论的热配合方法，如图 [图 36.2.4-3](pt09ch36s02aus142.md#acontact-interfer-grad2) 所示。如果需要大的应变自由调整才能达到指定的干扰距离，可能会发生网格失真。

**图 36.2.4-3** 指定的干扰距离与原始网格暗示的干扰距离不同时的处理。

![](../graphics/acontact-interfer-fit2-nls.png)

应变自由调整和后续热配合消除的搜索区域至少与前面讨论的不指定干扰距离情况下的搜索区域一样大。搜索区域将包含至少与指定干扰配合一样大的过盈量和至少与可选指定的表面上搜索距离一样大的开口。

| **输入文件用法：** | ``` [*CONTACT INITIALIZATION DATA](../key/key-link.md#usb-kws-hcontactinitdata), INTERFERENCE FIT=*h*, SEARCH ABOVE=*a*, SEARCH BELOW=*b* ``` |

##### 在消除干扰配合时停用摩擦

摩擦模型的存在会降低消除干扰配合的稳健性。通常建议您在 Abaqus/Standard 消除干扰配合时暂时停用摩擦模型。您可以使用"改变摩擦"方法在第一步中停用摩擦模型，同时消除干扰配合，如 ["在 Abaqus/Standard 分析期间改变摩擦属性"](pt09ch37s01aus169.md#usb-cni-afriction-change-std) 中所述。

##### 在干扰配合消除中首选接触对的情况

大的干扰可能难以使用有限滑动、面-面 formulation 来消除。使用此 formulation，过盈量倾向于沿从 facet 法线方向消除；而使用节点-面 formulation（仅适用于接触对算法），过盈量倾向于沿主表面法线方向消除。[图 36.2.4-4](pt09ch36s02aus142.md#rnb-chp-interffitcompare) 说明了一个情况，其中不同的法线方向导致干扰配合期间不希望的切向运动。在某些情况下，可能更好使用节点-面离散化通过接触对算法消除大的初始过盈量（见 ["在 Abaqus/Standard 中建立接触干扰配合模型，" 第 36.3.4 节](pt09ch36s03aus148.md)）。

**图 36.2.4-4** 具有大干扰配合的示例中接触 formulation 的比较。

### 分配接触初始化方法

您可以将接触初始化方法分配给选定的表面配对。

接触初始化分配中使用的表面名称不必与用于指定通用接触域的表面名称相对应。在许多情况下，非默认接触初始化方法将分配给整个通用接触域的子集。任何在通用接触域之外的区域的接触初始化分配都会被忽略。如果指定的相互作用重叠，则最后分配的优先。

| **输入文件用法：** | 使用以下选项分配接触初始化方法： |
| --- | --- |
|  | ``` [*CONTACT INITIALIZATION ASSIGNMENT](../key/key-link.md#usb-kws-hcontinitassign) *surface_1*, *surface_2*, *contact_initialization_method_name* ``` 此选项必须与 [*CONTACT](../key/key-link.md#usb-kws-hcontact) 选项结合使用。数据行可以根据需要重复多次，以将接触初始化方法分配给不同的区域。如果省略第一个表面名称，则假定一个包含整个通用接触域的默认表面。如果省略第二个表面名称或与第一个表面名称相同，则假定第一个表面与自身之间的接触。请记住，表面可以定义为跨越多个分离体，因此自接触不限于单个物体与自身的接触。如果省略接触初始化方法名称，则假定使用 Abaqus/Standard 中的默认接触初始化方法。如果指定了接触初始化方法名称，它也必须出现在输入文件模型部分中 [*CONTACT INITIALIZATION DATA](../key/key-link.md#usb-kws-hcontactinitdata) 选项的 NAME 参数的值中。 |

| **Abaqus/CAE 用法：** | Interaction 模块： **Create Interaction**: **General contact (Standard)**: **Contact Properties**: **Initialization assignments: Edit**: 选择左列中的表面和初始化，然后点击中间的箭头将它们转移到接触初始化分配列表中 |
| --- | --- |

#### 为壳表面分配接触初始化

接触初始化分配中的表面可以是单面或双面的。单面表面的相邻面必须具有一致的表面法线方向。如果使用双面进行接触初始化分配，则应变自由调整不会将表面节点移动到相对表面的参考表面之外。

在壳或膜的接触初始化分配中使用单面表面，可以在壳或膜参考表面初始交叉或初始位于彼此错误侧的情况下加强对接触初始化的控制。[图 36.2.4-5](pt09ch36s02aus142.md#rnb-chpinter-sfadoublesided) 显示了壳表面附近段的调整示例。对于左侧所示的情况，假定在接触初始化分配中使用具有指向彼此的法线方向的单面表面。在这种情况下，节点在应变自由调整期间穿过相对参考表面移动。

对于 [图 36.2.4-5](pt09ch36s02aus142.md#rnb-chpinter-sfadoublesided) 右侧所示的情况，假定在接触初始化分配中使用具有指向彼此的法线方向的单面表面。在这种情况下，在单面表面之间观察到初始间隙（如果在接触初始化分配中使用双面表面，也是这种情况）。默认情况下，对于这样的开口不会进行应变自由调整；但是，如果指定了非默认接触初始化方法，且初始开口搜索容差设置为超过初始分离距离的值，则应变自由调整将消除间隙（不将节点移动到相对参考表面之外）。
