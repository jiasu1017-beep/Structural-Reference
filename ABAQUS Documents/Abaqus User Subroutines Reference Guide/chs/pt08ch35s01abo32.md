# 35.1.1 运动约束：概述







以下类型的运动约束可以定义：

**方程：** 线性多点约束可以以方程形式给出（参见["线性约束方程，" 第35.2.1节](pt08ch35s02aus129.md)）。

**多点约束：** 多点约束（MPC）在节点之间指定线性或非线性约束。这些节点之间的关系可以是Abaqus提供的默认类型，或者在Abaqus/Standard中，可以以用户子程序的形式编码。["广义多点约束，" 第35.2.2节](pt08ch35s02aus130.md)解释了MPC的用法并列出了可用的默认约束。

**运动学耦合：** 在Abaqus/Standard中，一个节点或一组节点可以被约束到参考节点。与多点约束类似，运动学耦合约束允许对约束自由度进行一般的逐节点规范（参见["运动学耦合约束，" 第35.2.3节](pt08ch35s02aus131.md)）。

**基于曲面的绑定约束：** 两个曲面可以绑定在一起。第一个曲面（从属曲面）上的每个节点将与其最接近的第二个曲面（主曲面）上的点具有相同的自由度值（参见["网格绑定约束，" 第35.3.1节](pt08ch35s03aus132.md)）。在绑定到梁曲面的曲面单元的情况下，曲面单元与梁之间的偏移距离用于约束定义，包括梁的旋转自由度。

**基于曲面的耦合约束：** 位于曲面上的一组节点可以被约束到参考节点。此约束可以是运动学的，在这种情况下耦合节点组可以被约束到由参考节点定义的刚体运动；或者是分布的，在这种情况下耦合节点组可以以平均值方式被约束到由参考节点定义的刚体运动（参见["耦合约束，" 第35.3.2节](pt08ch35s03aus133.md)）。

**基于曲面的壳到实体耦合：** 三维壳单元网格上的基于边缘的曲面可以耦合到三维实体网格上的基于单元或节点的曲面。耦合通过创建一组内部分布耦合约束来强制执行（参见["壳到实体耦合，" 第35.3.3节](pt08ch35s03aus134.md)）。

**独立于网格的点焊：** 两个或多个曲面可以使用紧固件（如点焊）粘合在一起（参见["独立于网格的紧固件，" 第35.3.4节](pt08ch35s03aus135.md)）。在每个连接的曲面上创建分布耦合约束。连接建模与网格无关。

**埋入单元：** 一个单元或一组单元可以埋入一组宿主单元中（参见["埋入单元，" 第35.4.1节](pt08ch35s04aus136.md)）。Abaqus将搜索埋入单元节点与宿主单元之间的几何关系。如果埋入单元上的节点位于宿主单元内，则该节点处的自由度将通过约束到宿主单元自由度的插值来消除。宿主单元本身不能被埋入。

**释放：** 在Abaqus/Standard中，可以在梁单元的一端或两端释放局部旋转自由度或局部旋转自由度的组合（参见["单元端点释放，" 第35.5.1节](pt08ch35s05aus137.md)。

边界条件也是应力分析中的一种运动约束类型，因为它们定义了结构的支撑或在节点处给出固定位移。边界条件的规范在["Abaqus/Standard和Abaqus/Explicit中的边界条件，" 第34.3.1节](pt07ch34s03aus118.md)中讨论。

连接器单元可用于施加基于单元的运动约束，用于机构类型分析。参见["连接器：概述，" 第31.1.1节](pt06ch31s01abo28.md)。

接触相互作用（在第九部分"相互作用"中描述）可用于强制执行接触体之间的约束。接触相互作用可用于机械以及耦合热机械、耦合热电结构和耦合孔隙流体机械分析。

["过约束检查，" 第35.6.1节](pt08ch35s06aus138.md)描述了Abaqus/Standard中执行的过约束检查和一些过约束的自动解析。

### 节点处的多个运动约束

同一个节点可以用于多个多点约束、运动学耦合约束、绑定约束和约束方程。但是，约束依赖关系在Abaqus/Standard和Abaqus/Explicit中的处理方式不同。

#### Abaqus/Standard中的多个约束

在Abaqus/Standard中，运动约束通常通过消除从属节点处的自由度来施加。一旦变量被消除，它就不能在任何边界条件或任何后续的多点约束、运动学耦合约束、绑定约束或约束方程中被引用。如果您打算将一个在某个约束方程中被消除的变量作为另一个约束方程中的保留变量，则必须对输入进行排序，以便消除变量的约束方程排在其他约束方程之后。MPC类型BEAM、CYCLSYM、LINK、PIN、REVOLUTE、TIE和UNIVERSAL，以及运动学耦合和绑定约束，由Abaqus/Standard在内部排序以在可能时获得正确的消除顺序。

不建议过度链接多点约束、运动学耦合约束和约束方程，这可能导致分析预处理期间性能下降。只要有可能，最好通过使用一个多点约束、运动学耦合约束或约束方程将几个节点（分组为节点集）的行为关联到单个节点。

#### Abaqus/Explicit中的多个约束

在Abaqus/Explicit中，可以以任何顺序定义运动约束，而不考虑约束依赖关系。除由运动学接触对引起的约束外，Abaqus/Explicit同时求解所有运动约束。因此，参与多点约束、约束方程、连接器单元运动约束、刚体约束以及边界条件约束组合的节点将同时满足这些约束，只要它们不冲突。冗余和闭环约束是可以接受的。

由于上述约束与接触约束无关，因此对于参与运动约束和接触对定义两者的节点，应使用惩罚接触算法。惩罚接触算法通过使用惩罚弹簧引入数值软化，不会干扰运动约束。如果参与运动约束的节点用于运动学接触对，接触约束很可能覆盖运动约束。除刚体外，Abaqus/Explicit不会阻止您定义这些条件，但结果无法保证。如果在刚体上的节点上定义了运动约束，则必须对涉及刚体的所有接触对使用惩罚接触算法。

为了在受边界条件以及上述一个或多个运动约束约束的节点处从Abaqus/Explicit获得准确的反力和力矩输出，有时可能需要以双精度运行分析。在这种情况下，双精度运行还将更好地估计反力和力矩所做的功，从而提供更准确的Abaqus/External报告的外部工作能量的值。

Abaqus/Explicit在某些情况下使用惩罚方法求解约束。惩罚根据参与约束的节点质量和稳定时间增量进行加权。惩罚公式尝试近似满足约束（即，施加约束后存在非常小的顺应性不足）。使用惩罚方法求解约束的一种情况是绑定约束的从属节点参与其他约束（如多点约束、运动学耦合约束、约束方程、连接器单元、刚体约束或边界条件约束）。在这种情况下，绑定约束的顺应性不足不会跨步骤边界携带；因此，在某些问题中，可能在步骤边界处观察到嘈杂的加速度和能量不平衡。另一种建模方法（如简单地在绑定约束中反转主从曲面）可能切换到不同的求解方法，从而解决上述不准确性。

在Abaqus/Explicit中，当有两个或多个重叠的分布耦合约束或重叠的分布耦合和绑定约束，并且参与曲面的底层单元密度非常低时，顺应性不足可能导致不准确的解。为底层单元指定合理的密度值可以减少顺应性不足并提高解的准确性。

Abaqus/Explicit始终使用几何非线性公式来强制执行运动约束。即使您将特定分析步骤指定为几何线性，也是如此。因此，这些几何线性分析中的结果可能难以解释，特别是当模型中的载荷较高（位移较大）且应该使用几何非线性公式时。

### 约束节点的初始条件

您不应该将初始条件视为分析开始时的边界条件。当您在受运动约束的一组节点上规定初始条件时，Abaqus处理规定的值，以通过"质量"加权平均方法确定初始值，然后以运动学一致的方式重新分配到约束中涉及的节点：规定约束中涉及的每个节点处的初始值与该节点处的相应"质量"加权。因此，您在节点处指定的初始条件值会被重新计算，在许多情况下，分析开始时这些节点处规定量的输出将与您指定的值不同。正确的建模实践包括以与约束本身一致的方式在约束中涉及的所有节点上指定初始条件。

最好通过一个简单的例子来理解此行为。考虑一个由两个节点组成的模型，每个节点的质量为1.0，由全局方向2和3中的边界条件约束，允许沿全局1方向自由移动，同时它们的相对运动也通过刚性连接（如BEAM连接器）约束。假设您仅在第一个节点上指定了沿全局1方向的初始平动速度为10.0个单位，而未在第二个节点上指定初始条件。因此，Abaqus将认为第二个节点处的初始速度为0.0。此初始速度场与BEAM连接器强制执行的运动约束不一致，因为如果即使在无限短的时间内强制执行初始条件，约束也会被违反。结果是，Abaqus将计算一个初始速度场，以与约束一致的方式重新分配第一个节点的动量。在这个特定示例中，净效应是两个节点最终在全局1方向上具有5.0个单位的初始速度。很可能，这不是您的预期。正确的建模实践在这种情况下是在约束涉及的两个节点上都将初始速度指定为10.0个单位。在这种情况下，Abaqus仍将重新计算初始值，但结果是两个节点上的初始速度为10.0个单位，如预期的那样。

同样的原则适用于更复杂的建模情况。例如，如果在运动约束的节点上规定初始平动速度，则通过计算单个节点速度的质量加权平均值来计算约束节点的平均平动速度。根据运动约束的性质，约束节点上的初始平动速度也可能导致关于约束质心的平均旋转速度。然后根据约束质心处的平均平动和旋转速度重新计算约束每个单独节点的速度。权重中使用的"质量"类型数量根据规定量的性质而变化：如果在旋转速度上规定了初始条件，则在权重中使用节点处的旋转惯量；如果规定了温度初始条件，则在权重中使用节点处的热容；等等。

在所有情况下，您应在约束中涉及的所有节点上指定与约束一致的初始条件。这通常通过在约束涉及的所有节点上指定相同的初始条件来完成。
