### 概述

Abaqus/Standard中一般接触算法相关的数值控制：
- 对于大多数问题，不应修改其默认设置；
- 可用于默认设置无法提供经济有效的解决方案的问题；
- 可用于控制主-从角色和滑动公式；和
- 在某些情况下，可以选择性地应用于一般接触域内的特定区域。

### 接触公式

一般接触算法使用有限滑动、表面-表面接触公式，这在["Abaqus/Standard中的接触公式，" 第38.1.1节"](pt09ch38s01aus177.md)中讨论。在Abaqus/Standard中，一般接触不支持其他接触公式。

### 约束施加方法

一般接触算法默认使用惩罚方法来施加活动接触约束。其他约束施加方法可以作为表面相互作用（即接触属性）定义的一部分指定，这在["Abaqus/Standard中的接触约束施加方法，" 第38.1.2节"](pt09ch38s01aus178.md)中讨论。将接触属性分配给一般接触相互作用在["Abaqus/Standard中一般接触的接触属性，" 第36.2.3节"](pt09ch36s02aus141.md)中讨论。

### 摩擦的数值控制

与摩擦相关的数值控制在["摩擦行为，" 第37.1.5节"](pt09ch37s01aus169.md)中讨论。

### 梁-梁接触

梁-梁接触的激活在["在Abaqus/Standard中定义一般接触相互作用，" 第36.2.1节"](pt09ch36s02aus139.md)中讨论。

### 主和从角色

一般接触使用的表面-表面接触公式使用主-从方法生成单个接触约束，这在["Abaqus/Standard中的接触公式，" 第38.1.1节"](pt09ch38s01aus177.md)中讨论。Abaqus/Standard为一般接触域内涉及断开实体的接触分配默认纯主-从角色。内部表面使用命名约定`General_Contact_Faces_*k*`自动生成，其中`*k*`对应于自动分配的组件编号。默认情况下，编号较低的组件表面将作为编号较高组件表面的主表面。您可以通过在Abaqus/CAE的Visualization模块中查看自动生成的内部表面来确定默认纯主-从角色（见["使用显示组显示模型的子集，" Abaqus/CAE用户指南第78章"](../usi/usi-link.md#uss-dgp)）。默认情况下，身体内的自接触使用平衡主-从接触处理，每个表面节点在某些约束中作为主节点，在其他约束中作为从节点。

例如，如果一般接触域跨越三个断开的实体，则会自动创建以下三个一般接触的内部"组件表面"：
- `General_Contact_Faces_1`
- `General_Contact_Faces_2`
- `General_Contact_Faces_3`

默认情况下，列出的第一个表面对其他两个表面充当主表面，`General_Contact_Faces_2`对`General_Contact_Faces_3`充当主表面。默认情况下，这三个表面中每个表面内的自接触使用平衡主-从接触建模。

#### 指定非默认主-从角色

您可以通过指定纯主-从角色或指定应使用平衡主-从接触来覆盖默认主-从角色。默认主-从处理在大多数情况下效果很好。在修改主-从分配时，除了本节讨论的其他因素外，请记住以下几点：
- 分配替代主-从角色时，不要使用内部生成的组件表面（而是使用您定义的表面名称）。
- 主-从角色分配是模型定义的一部分，不能跨步修改。
- 在["在Abaqus/Standard中定义接触对，" 第36.3.1节"](pt09ch36s03aus145.md#usb-cni-acontactpair-twosurf)中讨论的为接触对分配纯主-从角色的指南也适用于为您重新分配一般接触的纯主-从角色的情况。
- 在["使用对称主-从接触对改善接触建模" in "在Abaqus/Standard中定义接触对，" 第36.3.1节"](pt09ch36s03aus145.md#usb-cni-acontactpair-symm)中讨论的对称（平衡）主-从接触对的限制也适用于为您重新分配一般接触的平衡主-从接触的情况。平衡主-从接触可能由于约束数量增加和过约束的可能性而导致稳健性降低。

| --- | --- |
| | ``` [*CONTACT FORMULATION*](../key/key-link.md#usb-kws-hcontformulation), TYPE=MASTER SLAVE ROLES *surf_1*, *surf_2*, SLAVE ``` 使用以下选项指示第一个表面应被视为主表面： ``` [*CONTACT FORMULATION*](../key/key-link.md#usb-kws-hcontformulation), TYPE=MASTER SLAVE ROLES *surf_1*, *surf_2*, MASTER ``` 如果省略第一个表面名称，则假定为包含整个一般接触域的默认表面。必须指定第二个表面名称。使用以下选项指定两个表面之间应使用平衡主-从接触： ``` [*CONTACT FORMULATION*](../key/key-link.md#usb-kws-hcontformulation), TYPE=MASTER SLAVE ROLES *surf_1*, *surf_2*, BALANCED ``` 如果省略第一个表面名称，则假定为包含整个一般接触域的默认表面。如果省略第二个表面名称，则假定为第一个表面与其自身的接触。 |

##### 自动生成的接触排除

Abaqus/Standard自动为与指定纯主-从角色相反的主-从角色生成接触排除；因此，对于两个表面重叠的区域，自接触被排除。例如，指定`surf_A`和`surf_B`之间的一般接触相互作用应使用`surf_A`被视为从表面的纯主-从接触，将导致为主`surf_A`的面接触从`surf_B`的面生成排除；`surf_A`和`surf_B`之间重叠区域的自接触被排除。如果省略第二个表面名称或与第一个表面名称相同，则会发出错误消息，因为此输入将导致表面的自接触被排除。

### 滑动时接触力重分布的平滑性

您可以控制滑动时节点接触力重分布的平滑性。默认设置（通常适当）导致节点力重分布的平滑性与从表面底层单元的阶次相同；即，线性单元的线性重分布平滑性，二阶单元的二次重分布平滑性。二次重分布平滑性通常倾向于改善收敛行为，并改善快速变化接触应力区域内接触应力的分辨率。然而，二次重分布平滑性倾向于增加每个约束涉及的节点数量，这可能增加方程求解器的计算成本。线性重分布平滑性倾向于提供活动接触区域边缘附近接触应力的更好分辨率，因此偶尔会产生更好的收敛行为。

| **输入文件用法：** | 使用以下选项指示滑动时接触力重分布的平滑性应与从表面底层单元的阶次相同： |
| --- | --- |
| | ``` [*CONTACT FORMULATION*](../key/key-link.md#usb-kws-hcontformulation), TYPE=SLIDING TRANSITION *surf_1*, *surf_2*, ELEMENT ORDER SMOOTHING ``` 如果省略第一个表面名称，则假定为包含整个一般接触域的默认表面。如果省略第二个表面名称，则假定为第一个表面与其自身的接触。使用以下选项指示滑动时接触力重分布的线性平滑性： ``` [*CONTACT FORMULATION*](../key/key-link.md#usb-kws-hcontformulation), TYPE=SLIDING TRANSITION *surf_1*, *surf_2*, LINEAR SMOOTHING ``` 如果省略第一个表面名称，则假定为包含整个一般接触域的默认表面。如果省略第二个表面名称，则假定为第一个表面与其自身的接触。使用以下选项指示滑动时接触力重分布的二次平滑性： ``` [*CONTACT FORMULATION*](../key/key-link.md#usb-kws-hcontformulation), TYPE=SLIDING TRANSITION *surf_1*, *surf_2*, QUADRATIC SMOOTHING ``` 如果省略第一个表面名称，则假定为包含整个一般接触域的默认表面。如果省略第二个表面名称，则假定为第一个表面与其自身的接触。 |

### 一般接触的其他全局数值控制

某些额外的数值接触控制可以从步到步全局修改用于一般接触；您不能为一般接触域内的单个表面对指定接触控制。您可以应用接触稳定化来解决模型中接触建立之前发生的刚性体模式，并且可以调整Abaqus/Standard用于确定接触穿透和分离的容差；这两种技术都在["在Abaqus/Standard中调整接触控制，" 第36.3.6节"](pt09ch36s03aus150.md)中讨论。
