### 概述

Abaqus/CAE是用于创建、提交、监控和评估Abaqus仿真结果 的交互环境，通过运行Abaqus执行过程并指定**cae**参数来执行。

### 命令行选项

**database**

此选项指定要打开的模型数据库文件或输出数据库文件的名称。要指定模型数据库文件，请在文件名中包含`.cae`文件扩展名或不包含文件扩展名。要指定输出数据库文件，请在文件名中包含`.odb`文件扩展名。

**replay**

此选项指定要从中重放Abaqus/CAE命令的文件名。*replay-file*中的命令将在Abaqus/CAE启动时立即执行。如果未给出文件扩展名，则默认扩展名是`.rpy`。您不能使用`replay`选项执行带有控制流语句的脚本。

**recover**

此选项指定要从中重建模型数据库的文件名。*journal-file*中的命令将在Abaqus/CAE启动时立即执行。如果未给出文件扩展名，则默认扩展名是`.jnl`。

**startup**

此选项指定包含要在应用程序启动时运行的Python配置命令的文件名。环境文件中设置的任何配置命令之后，将运行此文件中的命令。当执行命令时，Abaqus/CAE不会将命令回显到重放文件。

**script**

此选项指定包含要在应用程序启动时运行的Python配置命令的文件名。环境文件中设置的任何配置命令之后，将运行此文件中的命令。

可以通过在命令行上输入`--`然后跟一个或多个空格分隔的参数来将参数传递到文件中。这些参数将被Abaqus/CAE执行过程忽略，但可以在脚本中访问。

**noGUI**

此选项指定在不显示图形用户界面（GUI）的情况下运行Abaqus/CAE。如果未指定文件名，则会检出Abaqus/CAE许可证并初始化Python解释器，以允许交互式输入Python或Abaqus脚本界面命令。

如果指定了文件名，则Abaqus/CAE运行文件中的命令并在完成后退出。如果未给出文件扩展名，则默认扩展名是`.py`。此选项对于自动执行预分析或后分析处理任务很有用，而无需运行显示的额外开销。由于未提供界面，脚本不能包含任何用户交互。如果您使用`noGUI`选项，Abaqus/CAE会忽略您提供的任何其他命令行选项。

可以通过在命令行上输入`--`然后跟一个或多个空格分隔的参数来将参数传递到文件中。这些参数将被Abaqus/CAE执行过程忽略，但可以在Python脚本中访问。如果您使用`noGUI`选项，可以使用参数传入一个否则由命令行选项提供的变量。例如，您可以传入一个文件名，否则该文件名将由`script`选项指定。

**noenvstartup**

此选项指定环境文件中的所有配置命令都不应在应用程序启动时运行。此选项可与**script**命令一起使用，以禁止除**script**文件中的配置命令外的所有配置命令。

**noSavedOptions**

此选项指定Abaqus/CAE不应应用存储在`abaqus_v6.14.gpr`中的显示选项设置（例如，渲染样式和基准平面的显示）。更多信息，请参见["保存显示选项设置，"Abaqus/CAE用户指南第76.16节"](../usi/usi-link.md#uss-dsp-preferences)。

**noSavedGuiPrefs**

此选项指定Abaqus/CAE不应应用存储在`abaqus_v6.14.gpr`中的GUI设置（例如，Abaqus/CAE主窗口或其对话框的大小和位置）。

**noStartupDialog**

此选项指定不应显示Abaqus/CAE的**启动会话**对话框。

**custom**

此选项指定包含Abaqus GUI工具包命令的文件名。此选项执行自定义版本的Abaqus/CAE应用程序。更多信息，请参见[Abaqus GUI工具包用户指南第1章，"简介"](../cus/cus-link.md#cus-int-int)。

**guiTester**

此选项启动一个包含Abaqus Python开发环境以及Abaqus/CAE的单独用户界面。Abaqus Python开发环境允许您创建、编辑、单步执行和调试Python脚本。更多信息，请参见[Abaqus脚本用户指南第三部分，"Abaqus Python开发环境"](../cmd/cmd-link.md#cmd-pde)。

您可以指定一个脚本作为此选项的参数，这会提示Abaqus/CAE运行GUI脚本。当到达脚本末尾时，Abaqus/CAE关闭。

**guiRecord**

此选项使您能够在名为`abaqus.guiLog`的文件中记录Abaqus/CAE用户界面中的操作。您也可以通过使用环境变量`ABQ_CAE_GUIRECORD`在启动时设置此选项。**guiRecord**选项不能与**guiTester**选项一起使用。

**guiNoRecord**

此选项使您能够在设置环境变量`ABQ_CAE_GUIRECORD`时禁用用户界面记录。

### 示例

以下示例说明了**cae**执行过程 的命令行选项以及如何将参数传递给Abaqus/CAE。

#### 打开模型数据库

以下命令将执行Abaqus/CAE并加载名为"beam"的模型数据库文件：

```
abaqus cae database=beam
```

#### 将参数传递给脚本

以下命令将在应用程序启动时运行名为"try.py"的Python脚本，并将"argument1"传递给脚本：

```
abaqus cae script=try.py -- argument1
```
如果"try.py"定义为以下内容，上述命令将打印`argument1`
```
import sys
print sys.argv[-1]
```

#### 在不显示图形用户界面的情况下运行Abaqus/CAE

以下命令将运行名为"checkPartValidity.py"的Python脚本，并将参数传递给脚本以指定模型数据库、模型和部件。该脚本由Abaqus/CAE执行；但是，图形用户界面不会显示。

```
abaqus cae noGui=checkPartValidity.py -- test.cae Model-1 Part-1
```
如果"checkPartValidity.py"定义为以下内容，上述命令将打印`Part-1 is valid`
```
import sys
import os

myMdb= sys.argv[-3]
myModel = sys.argv[-2]
myPart = sys.argv[-1]

mdb = openMdb(myMdb)
model = mdb.models[myModel]
part = model.parts[myPart]

if part.geometryValidity:
    sys.__stderr__.write('%s is valid\n' % myPart)
else:
    sys.__stderr__.write('%s is invalid\n' % myPart)
```
