### 概述

Abaqus/Standard中的并行执行：
- 减少大型分析的运行时间；
- 可用于共享内存计算机和计算机集群的元素操作、直接稀疏求解器和迭代线性方程求解器；以及
- 可以在共享内存计算机上使用支持计算的GPGPU硬件来加速直接稀疏求解器。

### 使用默认直接稀疏求解器的并行方程求解

直接稀疏求解器（["直接线性方程求解器，" 第6.1.5节](pt03ch06s01aus46.md)）支持共享内存计算机和计算机集群的并行化。在共享内存计算机或计算机集群的单个节点上，直接稀疏求解器使用基于线程的并行化，并且可用于加速求解的高性能图形卡（GPGPU）。在计算机集群的多个计算节点上，使用混合MPI和基于线程的并行化。

直接稀疏求解器在以下情况下不能在计算机集群的多个计算节点上使用：
- 分析还包括特征值提取过程，或
- 分析需要不支持MPI并行执行元素操作的特征。

此外，在包含以下任何内容的分析中，直接稀疏求解器不能在计算机集群的多个节点上使用：
- 具有变化边界条件的多荷载情况（["多荷载情况分析，" 第6.1.4节](pt03ch06s01aus45.md)），和
- 准牛顿非线性求解技术（["非线性问题的收敛准则，" 第7.2.3节](pt03ch07s02aus51.md)）。

要在计算机集群上执行并行直接稀疏求解器，必须将环境变量**mp_host_list**设置为主机列表（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)）。在主机列表中的机器之间使用基于MPI的并行化。如果主机列表中的那台机器有多个处理器可用，并且模型不包含使用并行分解的空腔辐射（请参阅["并行分解大型空腔"中的"空腔辐射，" 第41.1.1节](pt09ch41s01aus187.md#usb-cni-acavityradiation-parallel)），则在主机机器内使用基于线程的并行化。例如，如果环境文件有以下内容：

```
cpus=8
mp_host_list=[['maple',4],['pine',4]]
```

Abaqus/Standard将通过基于线程的并行化在每台主机上使用四个处理器。跨主机机器将运行总共两个MPI进程（等于主机数量），以便所有八个处理器都由并行直接稀疏求解器使用。

包含并行空腔分解的模型仅使用基于MPI的并行化。因此，MPI在共享内存并行计算机和分布式内存计算集群上都有使用。进程数等于提交作业时请求的CPU数量。当启用并行空腔分解时，元素操作使用基于MPI的并行化并行执行。

| **输入文件用法：** | 将以下选项与命令行输入结合使用以执行并行直接稀疏求解器： |
| --- | --- |
|  | ``` [*STEP](../key/key-link.md#usb-kws-hstep) ``` 在命令行上输入以下内容：**abaqus** **job**=*job-name* **cpus**=*n* 例如，以下输入将在两个处理器上运行名为"beam"的作业：``` abaqus job=beam cpus=2 ``` |

| **Abaqus/CAE用法：** | 步骤模块：步骤编辑器：**其他**：**方法：直接** |
| --- | --- |
|  | 作业模块：作业编辑器：**并行化**：切换开启**使用多个处理器**，并指定处理器数量*n* |

#### 并行直接稀疏求解器的内存需求

并行直接稀疏求解器并行处理多个前沿，同时并行化各个前沿的求解。因此，并行直接求解器比串行求解器需要更多内存。内存需求无法准确提前预测，因为无法预先确定哪些前沿将同时处理。

#### 最小求解时间的方程排序

直接稀疏求解器需要对系统方程进行排序以最小化浮点运算计数。当在计算机集群上使用多台主机时，排序过程并行执行。在共享内存配置中，排序过程不并行执行。并行排序过程将在不同数量的主机机器上运行时计算不同的排序，这将影响直接求解器的浮点运算计数。并行排序可以提供性能改进，特别是对于使用许多主机机器的大型模型，通过显著减少计算排序的时间。如果排序导致直接求解器的浮点运算计数更高，则并行排序可能导致性能下降。

在并行排序过程的排序固有变异性不可接受的情况下，可以使用串行排序过程。您可以从命令行停用并行求解器排序，或使用**order_parallel**环境文件参数（请参阅["使用Abaqus环境设置"中的"命令行默认参数，" 第3.3.1节](pt01ch03s03aus30.md#usb-int-denvfile-commparam)）。

| **输入文件用法：** | 在命令行上输入以下内容以停用并行求解器排序： |
| --- | --- |
|  | **abaqus** **job**=*job-name* **order_parallel**=`OFF` |

| **Abaqus/CAE用法：** | 在Abaqus/CAE中不支持停用并行求解器排序。 |
| --- | --- |

### 使用迭代求解器的并行方程求解

迭代求解器（["迭代线性方程求解器，" 第6.1.6节](pt03ch06s01aus47.md)）仅使用基于MPI的并行化。因此，MPI在共享内存并行计算机和分布式内存计算集群上都有使用。要执行并行迭代求解器，请指定作业的CPU数量。进程数等于提交作业时请求的CPU数量。当使用并行迭代求解器时，元素操作使用基于MPI的并行化并行执行。

| **输入文件用法：** | 将以下选项与命令行输入结合使用以执行并行迭代求解器： |
| --- | --- |
|  | ``` [*STEP](../key/key-link.md#usb-kws-hstep), SOLVER=ITERATIVE ``` 在命令行上输入以下内容：**abaqus** **job**=*job-name* **cpus**=*n* 例如，以下输入将使用迭代求解器在四个处理器上运行名为"cube"的作业：``` abaqus job=cube cpus=4 ``` |

| **Abaqus/CAE用法：** | 步骤模块：步骤编辑器：**其他**：**方法：迭代** |
| --- | --- |
|  | 作业模块：作业编辑器：**并行化**：切换开启**使用多个处理器**，并指定处理器数量*n* |

### Abaqus/Standard中元素操作的并行执行

元素操作的并行执行是所有支持平台上的默认值。命令行和环境变量**standard_parallel**可用于控制元素操作的并行执行（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)和["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行，" 第3.2.2节](pt01ch03s02abx02.md)）。如果使用元素操作的并行执行，求解器也会自动并行运行。对于使用直接稀疏求解器且不包含并行空腔分解的分析，在共享内存计算机上使用元素操作的基于线程的并行化，在计算机集群上使用混合MPI和线程并行方案。对于使用迭代求解器的分析或启用了并行空腔分解的分析，仅支持元素操作的基于MPI的并行化。

当使用元素操作的基于MPI的并行化时，会为每个域创建元素集，并可以在Abaqus/CAE中检查。这些集合名为`STD_PARTITION_*n*`，其中*n*是域编号。

以下过程的元素操作不支持并行执行（线程或基于MPI的并行化）：
- 特征值屈曲预测（["特征值屈曲预测，" 第6.2.3节](pt03ch06s02at02.md)），
- 不使用SIM架构的自然频率提取（["自然频率提取，" 第6.3.5节](pt03ch06s03at10.md)），
- 响应谱分析（["响应谱分析，" 第6.3.10节](pt03ch06s03at15.md)），
- 随机响应分析（["随机响应分析，" 第6.3.11节](pt03ch06s03at16.md)），和
- 不使用SIM架构的基于模式的线性动力学（["瞬态模态动态分析，" 第6.3.7节](pt03ch06s03at12.md)；["基于模式的稳态动态分析，" 第6.3.8节](pt03ch06s03at13.md)；["基于子空间的稳态动态分析，" 第6.3.9节](pt03ch06s03at14.md)；和["复特征值提取，" 第6.3.6节](pt03ch06s03at11.md)）。

以下任何分析仅可通过基于MPI的并行化执行元素操作的并行执行：
- 静态线性扰动（["一般和线性扰动过程，" 第6.1.3节](pt03ch06s01aus44.md)），
- 直接解稳态动力学（["直接解稳态动态分析，" 第6.3.4节](pt03ch06s03at09.md)），
- 稳态传输（["稳态传输分析，" 第6.4.1节](pt03ch06s04at17.md)），
- 耦合温度-位移（["完全耦合热应力分析，" 第6.5.3节](pt03ch06s05at19.md)），
- 压力渗透加载（["压力渗透加载，" 第37.1.7节](pt09ch37s01aus171.md)），和
- 对于使用子结构的模型的静态、隐式动态或直接解稳态动态分析，如果未请求在子结构内恢复结果（["静态应力分析，" 第6.2.2节](pt03ch06s02at01.md)；["使用直接积分的隐式动态分析，" 第6.3.2节](pt03ch06s03at07.md)；["直接解稳态动态分析，" 第6.3.4节](pt03ch06s03at09.md)；["子结构，" 第10.1节](pt04ch10s01.md)）。

使用直接稀疏求解器且上述仅支持元素操作MPI并行化的过程的分析可以在计算机集群上运行。但是，由于不支持基于线程的并行化，因此每个计算节点仅使用一个处理器用于元素操作。

以下情况仅可通过基于线程的并行化执行元素操作的并行执行：
- 不允许空腔并行分解且请求写入重启数据的空腔辐射分析（["空腔辐射，" 第41.1.1节](pt09ch41s01aus187.md)），
- 指定平均温度辐射条件的热传递分析（["热载荷，" 第34.4.4节](pt07ch34s04aus123.md)），
- 使用SIM架构的自然频率提取（["自然频率提取，" 第6.3.5节](pt03ch06s03at10.md)），
- 使用SIM架构的基于模式的线性动力学（["瞬态模态动态分析，" 第6.3.7节](pt03ch06s03at12.md)；["基于模式的稳态动态分析，" 第6.3.8节](pt03ch06s03at13.md)；["基于子空间的稳态动态分析，" 第6.3.9节](pt03ch06s03at14.md)；和["复特征值提取，" 第6.3.6节](pt03ch06s03at11.md)），
- 子结构生成（["定义子结构，" 第10.1.2节](pt04ch10s01aus59.md)），和
- 矩阵生成（["热矩阵生成"简介，第10.3.2节](pt04ch10s03at33.md#usb-anl-amtxgenerategeneral)）。

最后，包含以下任何内容的分析不支持元素操作的并行执行：

#### 元素操作并行执行时的内存管理

在Abaqus/Standard中运行元素操作的并行执行时，指定可以使用的内存上限（请参阅["在Abaqus中管理内存和磁盘使用"中的"Abaqus/Standard分析，" 第3.4.1节](pt01ch03s04aus31.md#usb-int-dmemory-standard)）指定*每个进程*可以分配的最大内存量。

#### 层合连续壳的横向剪切应力输出

在使用Abaqus/Standard中元素操作并行执行时，输出变量CTSHR13和CTSHR23当前不可用。请参阅["连续壳单元库，" 第29.6.8节](pt06ch29s06ael18.md)。

### 结果的一致性

一些物理系统（例如经历屈曲、材料失效或分层的系统）可能对小扰动高度敏感。例如，众所周知，一组看似相同的圆柱壳的实验测量的屈曲载荷和最终配置可能显示出由于边界条件、载荷、初始几何形状等的细微差异而产生的显著分散。当模拟这样的系统时，实验中看到的物理敏感性可能表现为对有限精度效应引起的小数值差异的敏感性。有限精度效应可能在不同数量处理器的运行中导致小数值差异。因此，在模拟物理敏感系统时，您可能会看到在不同处理器数量上运行的作业的数值结果差异（反映实验中看到的差异）。为了从运行到运行获得一致的模拟结果，处理器数量应保持不变。
