# 3.4.7 理解修改后的步骤相关对象





当您在创建对象的步骤中编辑对象时，您会更改该对象在所有活动步骤中的定义。在某些情况下，您也可以在状态为`Propagated`或`Modified`的步骤中编辑对象。在这些情况下，对象的定义会根据分析步骤而变化。

编辑步骤相关对象的效果总结如下。

**如果对象在所选步骤中的状态为`Created`：**

- 您在该步骤中对对象所做的修改会在该步骤中生效，并传播到该条件处于活动状态的所有后续步骤，除非您在后续步骤中再次修改对象。
- 对象在所选步骤中的状态保持为`Created`，并在所有后续步骤中保持不变。有关更多信息，请参阅["理解步骤中对象的状态，" 第3.4.4节"](pt01ch03s04s04.md)。

**如果对象在所选步骤中的状态为`Propagated`或`Modified`：**

- 您在该步骤中对对象所做的修改会在该步骤中生效，并传播到该对象处于活动状态的所有后续步骤。
- 对象在该步骤中的状态变为（或保持为）`Modified`，并在所有其他步骤中保持不变。（换句话说，如果修改前对象在后续步骤中的状态为`Propagated`，则修改后其在后续步骤中的状态仍为`Propagated`。）例如，[图3-18](pt01ch03s04s04.md#int-stepgraph)中一系列通用静态分析步骤上施加的载荷已在Step 3中被修改；即使Step 4中的状态为`Propagated`，修改仍会在Step 4中保持生效。有关更多信息，请参阅["理解步骤中对象的状态，" 第3.4.4节"](pt01ch03s04s04.md)。
- 当您在Interaction编辑器以外的任何编辑器中修改数据时，Abaqus/CAE会在编辑器中指示哪些数据已被修改。如果您将编辑器中的数据更改回其原始值，这些指示会消失。

在某些情况下，您不能编辑对象定义的特定方面，因为它必须保持一致以使分析正确进行。例如，虽然您可以在任何分析步骤中修改载荷的大小，但不能修改施加载荷的区域。指定这种受限数据的编辑器区域在创建对象的步骤之外的所有步骤中都不可用。
