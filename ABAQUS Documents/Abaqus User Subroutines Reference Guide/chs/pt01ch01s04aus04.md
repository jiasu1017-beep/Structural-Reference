### 概述

参数化输入功能允许您创建Abaqus输入文件，其中：
- 通过为每个参数赋值来定义任意数量的输入参数。
- 输入文件中定义的参数用于代替输入量。
- 参数根据其定义进行求值，并在运行分析之前替换参数化输入量。

参数化输入在构建和操作模型方面提供了更大的灵活性。本节讨论不同类型的参数和参数化Abaqus输入量的不同方式。

### 简介

您必须通过为参数赋值来定义希望在分析中使用的所有参数。Python语言（[Lutz, 1999](pt01ch01s04aus04.md#usb-ref-lutz)）用于执行参数求值和替换；因此，参数定义需要遵循本节后面讨论的Python语法规则。然后这些参数可以用于代替输入量。

| **输入文件用法：** | 使用以下选项定义参数： |
| --- | --- |
|  | ``` [*PARAMETER*](../key/key-link.md#usb-kws-mparameter) ``` 使用< >分隔参数来代替输入量。例如，以下输入定义了两个参数`width`和`height`，然后用于定义梁截面属性： ``` [*PARAMETER*](../key/key-link.md#usb-kws-mparameter) width = 2.5 height = width*2 [*BEAM SECTION*](../key/key-link.md#usb-kws-mbeamsection), SECTION=RECT, ELSET=*name*, MATERIAL=*name* <width>, <height> ``` 在这个简单示例中，只需更改参数值即可获得具有不同横截面的梁模型。 |

### 参数

参数是您为其分配值的用户命名变量。当使用参数代替值时，代入该参数的值。有两种基本类型的参数：独立参数和依赖参数。

#### 独立参数

独立参数是不依赖于任何其他参数的参数。以下是独立参数的示例：

```
thickness = 10.0
area = 5.0**2
length = 3.0*sin(45*pi/180.0) # 将度转换为弧度
```

可以使用包含数字和数值运算（如加法、乘法和指数运算）的Python表达式来定义独立参数。本节后面讨论Python中的算术支持。

#### 依赖参数

依赖参数是依赖于其他参数（依赖或独立）的参数。依赖参数可以通过两种方式之一定义：使用数学表达式或使用表格依赖。

##### 表达式依赖

使用包含数字和参数之间运算的Python参数表达式来定义表达式依赖参数。在以下示例中，`area`和`mom_inertia`是依赖参数：

```
width = 2.0
height = 5.0
area = width*height
mom_inertia = area*height**2/12.0
```

##### 表格依赖

参数之间的表格依赖通过指定依赖参数和独立参数以及依赖表来定义。定义参数之间依赖的表必须每行包含与依赖参数数量加上将使用的独立参数数量相同的值。表必须仅包含实数值；依赖参数值放在前面，然后是独立参数值。表中不能使用参数名和字符串。

通过对表中值进行插值来求值表格依赖参数将导致这些参数被分配实数值。如果需要表格依赖参数为整数，则必须如Python语言部分后面所述将实数转换为整数。

当表格依赖参数只是一个独立参数的函数时，表格数据必须按独立参数递增值的顺序给出。然后Abaqus对给定值之间的值进行线性插值。假设依赖参数在表中使用的独立参数范围之外为常量。当表格依赖参数依赖于几个独立参数时，必须在其他独立参数固定值下给出依赖参数随第一个独立参数的变化规律，按第二个独立参数的升序，然后是第三个独立参数的升序，依此类推。表行必须始终按独立参数递增值的顺序排序。此过程确保依赖参数的值对于独立参数的任何值都被完整且唯一地定义。

依赖表定义与依赖分配到特定参数是分开的，这意味着同一个表可以用于多组依赖/独立参数。当存在相同类型输入数据的不同实例时，这很有用；例如，使用相同依赖但不同参数集的多个材料定义。

因为参数求值是过程性的（参见下面的["参数求值"](pt01ch01s04aus04.md#usb-int-iparinput-rules-eval)），所以依赖表必须始终在使用之前定义，以便指定表格参数依赖。

在定义此依赖性的目的上，表格依赖定义中的独立参数被视为独立的；但是，这些"独立"参数可以在前面的参数定义中定义为依赖于其他参数。

| **输入文件用法：** | 使用以下选项定义参数依赖表： |
| --- | --- |
|  | ``` [*PARAMETER DEPENDENCE*](../key/key-link.md#usb-kws-mparameterdepen), TABLE=*name*, NUMBER VALUES=*n* *每行n个值的表* ``` 使用以下选项定义依赖表中使用的依赖和独立参数： ``` [*PARAMETER*](../key/key-link.md#usb-kws-mparameter), TABLE=*name*, DEPENDENT=*(parList)*, INDEPENDENT=*(parList)* ``` |

#### 参数规则

适用于Abaqus输入文件中使用的所有参数的一些一般规则。以下小节描述了这些规则。

##### 参数求值

参数通过按参数定义在输入文件中出现的顺序执行来求值。例如，输入

```
[*PARAMETER*](../key/key-link.md#usb-kws-mparameter)
x = 2
y = x + 3
x = 4
```

给出`x`=4和`y`=5，而不是`x`=4和`y`=7。输入
```
[*PARAMETER*](../key/key-link.md#usb-kws-mparameter)
y = x + 3
x = 4
```

被标记为错误，因为`y`无法通过有序执行输入来求值。换句话说，参数定义没有延迟执行。

由于参数定义始终在任何其他输入选项之前处理，所以可以在输入文件中的任何位置定义参数，甚至在参数已用于代替输入量之后。

参数也可以在用于重启分析的输入文件中定义和使用。但是，原始分析（从中继续重启运行）的输入文件中定义的参数在重启分析中不可用。

##### 参数替换

当处理参数化数据时，Abaqus分配在参数求值结束时确定的参数值。如果用于代替输入量的参数未分配值，则报告错误。随后，分析输入文件处理器对参数值相对于使用它们的选项的有效性执行其常规检查。

用于定义参数、参数依赖表或参数形状变化的数据不能参数化。例如，输入

```
[*PARAMETER SHAPE VARIATION*](../key/key-link.md#usb-kws-mparametershape)
<x>
```

无效；但是，分析输入文件处理器不会为此输入报告错误。

##### 数据类型

参数的数据类型从其定义中推断。给参数赋值整数字面值会产生整数参数。类似地，给参数赋值实数字面值会产生实参数。整数在包含实数的运算中自动提升为实数。给参数赋值字符串字面值会产生字符串参数。

参数使用的输入选项上下文决定了参数必须具有的数据类型。实数数据类型参数应用于代替Abaqus实数输入量。整数（或字符串）类型参数应分别用于代替整数（或字符串）类型输入量。在某些情况下，输入上下文与替换参数类型之间的不匹配会导致分析输入文件处理器将这些实例标记为输入错误。例如，输入

```
[*PARAMETER*](../key/key-link.md#usb-kws-mparameter)
int_pts = 5.0
[*SHELL SECTION*](../key/key-link.md#usb-kws-mshellsection)
10.0, <int_pts>
```

将导致分析输入文件处理器报告错误，因为为壳截面指定的积分点数量必须是整数。但是，输入
```
[*PARAMETER*](../key/key-link.md#usb-kws-mparameter)
thick = 5/4
[*SHELL SECTION*](../key/key-link.md#usb-kws-mshellsection)
<thick>,
```

将被分析输入文件处理器接受而不会标记警告；由于执行整数除法，此输入给出的壳厚度为1（不是1.25）。总之，您可以依赖分析输入文件处理器仅捕获某些数据类型错误。

##### 连续和离散参数

从设计活动（灵敏度分析、参数研究等）的角度来看，参数可以是连续值或离散值。连续值参数是可微的，因此可以用于设计灵敏度分析目的。离散值参数是不可微的，因此不能用于设计灵敏度分析目的；但是，它可以用于参数研究。连续值参数的示例可以是壳厚度或材料属性。离散值参数的示例可以是壳厚度方向的积分点数量，或单元类型。连续值参数通常与物理（设计）输入量一致，而离散值参数通常与有限元（数值近似）输入量一致。

##### 辅助输入文件

参数可以在[*INCLUDE*](../key/key-link.md#usb-kws-minclude)输入文件中定义，但不能在任何其他辅助输入文件中定义。辅助输入文件的名称可以参数化，[*INCLUDE*](../key/key-link.md#usb-kws-minclude)选项中使用的除外。

### 输入量的参数化

Abaqus对"大小"和"形状"量的参数化处理略有不同。形状输入量的参数化在单独的部分讨论（参见["参数化形状变化"，第2.1.2节"](pt01ch02s01aus06.md)）。

大小输入量被理解为包括除与形状相关的量之外的所有Abaqus输入量。大小输入量包括截面属性、材料属性、方向属性、规定条件、相互作用定义和属性，以及分析过程数据。

#### 参数化单个输入量

以下示例展示了使用三个不同数据类型的独立参数对壳截面输入的参数化：

```
[*ELSET*](../key/key-link.md#usb-kws-melset), ELSET=<shell_set>, GEN
1, 111, 10
[*PARAMETER*](../key/key-link.md#usb-kws-mparameter)
shell_set = 'lining'
shell_thick = 1.E2
num_int_pts = 5
[*SHELL SECTION*](../key/key-link.md#usb-kws-mshellsection), ELSET=<shell_set>, MATERIAL=*name*
<shell_thick>, <num_int_pts>
```

#### 参数化输入量组（表达式依赖）

以下示例展示了使用表达式依赖参数对三层复合壳截面的参数化。在此示例中，`thickness`参数可用于统一更改复合截面层的厚度。

```
[*PARAMETER*](../key/key-link.md#usb-kws-mparameter)
thickness = 10.
layer1_thick = 0.15*thickness
layer2_thick = 0.6*thickness
layer3_thick = 0.25*thickness
[*SHELL SECTION*](../key/key-link.md#usb-kws-mshellsection), ELSET=, COMPOSITE
<layer1_thick>,*num int pts, material name, orientation* 
<layer2_thick>,*num int pts, material name, orientation*
<layer3_thick>,*num int pts, material name, orientation*
```

此参数化要求为三个输入量（`layer1_thick`、`layer2_thick`、`layer3_thick`）创建依赖参数，每个都依赖于独立参数（`thickness`）。

#### 参数化输入量组（表格依赖）

以下示例展示了箱形梁截面属性的参数化。梁截面高度和壁厚是依赖于截面宽度的表格参数。

```
[*PARAMETER*](../key/key-link.md#usb-kws-mparameter)
a = 60.
[*PARAMETER DEPENDENCE*](../key/key-link.md#usb-kws-mparameterdepen), TABLE=sectprop, NUMBER VALUES=6
25.0, 1.04,  1.04, 1.04, 1.04, 50.0
50.0, 4.17,  3.13, 2.08, 2.50, 100.0
75.0, 9.38,  6.24, 3.13, 4.90, 150.0
[*PARAMETER*](../key/key-link.md#usb-kws-mparameter), TABLE=sectprop, DEPENDENT=(b, t1, t2, t3, t4),
INDEPENDENT=(a)
[*BEAM SECTION*](../key/key-link.md#usb-kws-mbeamsection), SECTION=BOX, ELSET=beams, MATERIAL=steel
<a>, <b>, <t1>, <t2>, <t3>, <t4>
```

上述参数化创建依赖参数（`b`、`t1`、`t2`、`t3`、`t4`），每个都依赖于独立参数（`a`）。使用表格依赖允许将输入量对参数的依赖定义限制在参数定义中；即，与执行输入量参数化的选项分开。此参数化方法的一个优点是同一个参数依赖表可以用于不同输入选项中的不同参数。例如，您可能希望在建模结构的不同部分使用不同截面尺寸的梁。参数依赖表可以与新的依赖（`bb`、`tt1`、`tt2`、`tt3`、`tt4`）和独立（`aa`）参数一起重用。
```
[*PARAMETER*](../key/key-link.md#usb-kws-mparameter)
aa = 65.
[*PARAMETER*](../key/key-link.md#usb-kws-mparameter), TABLE=sectprop, DEPENDENT=(bb, tt1, tt2, tt3, tt4),
INDEPENDENT=(aa)
[*BEAM SECTION*](../key/key-link.md#usb-kws-mbeamsection), SECTION=BOX, ELSET=columns, MATERIAL=steel
<aa>, <bb>, <tt1>, <tt2>, <tt3>, <tt4>
```

在支持预定义场变量依赖的选项中，此参数化方法在预定义场变量依赖和参数依赖之间提供了明确的分离；因此，场变量依赖和参数依赖永远不会混淆。例如，考虑金属完美塑性属性的情况，其中屈服应力依赖于场变量，同时也参数化依赖于金属合金的碳含量。

```
[*PARAMETER*](../key/key-link.md#usb-kws-mparameter) 
carbon = 0.01 
[*PARAMETER DEPENDENCE*](../key/key-link.md#usb-kws-mparameterdepen), TABLE=yield_data, NUMBER=4 
*ys_fv1 val 1, ys_fv2 val 1, ys_fv3 val 1, carbon val 1* 
*ys_fv1 val 2, ys_fv2 val 2, ys_fv3 val 2, carbon val 2* 
*ys_fv1 val 3, ys_fv2 val 3, ys_fv3 val 3, carbon val 3* 
*ys_fv1 val 4, ys_fv2 val 4, ys_fv3 val 4, carbon val 4* 
[*PARAMETER*](../key/key-link.md#usb-kws-mparameter), TABLE=yield_data, DEPENDENT=(ys_fv1, ys_fv2, ys_fv3),
INDEPENDENT=(carbon) 
[*MATERIAL*](../key/key-link.md#usb-kws-mmaterial), NAME=alloy 
[*PLASTIC*](../key/key-link.md#usb-kws-mplastic), DEPENDENCIES=1
<ys_fv1>, , , *fv val 1*
<ys_fv2>, , , *fv val 2*
<ys_fv3>, , , *fv val 3*
```

例如，考虑金属蠕变属性的情况，其中蠕变材料数据是依赖于金属合金碳含量的表格参数。此外，一个蠕变参数`A`也依赖于预定义场变量。
```
[*PARAMETER*](../key/key-link.md#usb-kws-mparameter) 
carbon = 0.01 
[*PARAMETER DEPENDENCE*](../key/key-link.md#usb-kws-mparameterdepen), TABLE=creepdata, NUMBER=6
*A_fv1 val 1, A_fv2 val 1, A_fv3 val 1, n val 1, m val 1, carbon val 1*
*A_fv1 val 2, A_fv2 val 2, A_fv3 val 2, n val 2, m val 2, carbon val 2*
*A_fv1 val 3, A_fv2 val 3, A_fv3 val 3, n val 3, m val 3, carbon val 3*
*A_fv1 val 4, A_fv2 val 4, A_fv3 val 4, n val 4, m val 4, carbon val 4*
[*PARAMETER*](../key/key-link.md#usb-kws-mparameter), TABLE=creepdata, DEPENDENT=(A_fv1, A_fv2, A_fv3,
n, m), INDEPENDENT=(carbon) 
[*MATERIAL*](../key/key-link.md#usb-kws-mmaterial), NAME=alloy 
[*CREEP*](../key/key-link.md#usb-kws-mcreep), DEPENDENCIES=1
<A_fv1>, <n>, <m>, , *fv val 1*
<A_fv2>, <n>, <m>, , *fv val 2*
<A_fv3>, <n>, <m>, , *fv val 3*
```

此示例表明，可以定义对预定义场变量和/或依赖参数的任意组合依赖。

### Python语言

参数定义中的参数语句需要遵循Python语言的语法和语义（请注意，参数依赖表和参数形状变化定义遵循通常的Abaqus输入语法规则）。此处记录了Python语言的认可子集。

#### 语句长度和延续行

参数定义中的Python语句可以通过在每行末尾用反斜杠字符（\）终止来跨多行继续。[*PARAMETER*](../key/key-link.md#usb-kws-mparameter)关键词行可以使用尾随逗号延续到下一行，因为它们像其他Abaqus关键词行一样处理。

#### 注释

参数定义中的注释以数字字符（#）开头，并继续到行尾。但是，参数依赖表或参数形状变化定义中的注释由通常的Abaqus输入语法约定（**）表示。

#### 参数名称

参数名称必须以字母开头，可以包含下划线字符（_）和数字。参数名称区分大小写。

#### 数据类型

数据类型仅限于字符串、整数和实数。

字符串用单引号或双引号分隔（' '或" "）。不允许使用反向单引号（` `）。字符串不应包含反斜杠字符（\）。

整数通过赋值给整数字面值创建（例如，*aInt * = 2）。

实数通过赋值给实数字面值创建（例如，*aReal *= 1.0）。实数可以带指数或不带指数给出。任何指数必须以`E`或`e`开头。以下行显示输入相同实数的五种可接受方式：

```
-12.345, -1234.5E-2, -0.12345E+2, -0.12345E2, -0.12345e2
```

语法
```
-0.12345D+2
```

（Abaqus输入文件的其他地方允许）但在Python中无效。

#### 类型转换

如果整数和实数在表达式中混合，整数自动提升为实数。可以使用以下方式进行显式类型转换：

| int(*aReal*) | *aReal*转换为整数类型 |
| --- | --- |
| float(*anInt*) | *anInt*转换为实数类型（float与real相同） |
| str(*anIntOrReal*) | *anIntOrReal*转换为字符串类型 |
| '*anIntOrReal*' | *anIntOrReal*转换为字符串类型 |

#### 数值运算符

提供标准运算符支持：

| x | x取反 |
| --- | --- |
| + x | x不变 |
| x + y | x和y的和 |
| x y | x和y的差 |
| x * y | x和y的乘积 |
| x / y | x和y的商 |
| x**y | x的y次方 |

#### 函数

支持以下实用函数：

| abs(x) | x的绝对值 |
| --- | --- |
| acos(x) | x的反余弦（结果以弧度为单位） |
| asin(x) | x的反正弦（结果以弧度为单位） |
| atan(x) | x的反正切（结果以弧度为单位） |
| cos(x) | x的余弦（x以弧度为单位） |
| log(x) | x的自然对数 |
| log10(x) | x的底10对数 |
| pow(x,y) | x的y次方（等同于x**y） |
| sin(x) | x的正弦（x以弧度为单位） |
| sqrt(x) | x的平方根 |
| tan(x) | x的正切（x以弧度为单位） |

#### 字符串运算符

| 'abc' + 'def' | 字符串'abc'和'def'的连接 |
| --- | --- |

### 参数化输入的执行

具有参数化输入文件的作业以通常的方式提交给Abaqus；例如，

```
`abaqus` `job`=*job-name* `input`=*input-file*
```

其中假定存在名为`*input-file*.inp`的输入文件。

Abaqus搜索`*input-file*.inp`和任何[*INCLUDE*](../key/key-link.md#usb-kws-minclude)输入文件，查找参数、参数依赖表和参数形状变化（["参数化形状变化"，第2.1.2节"](pt01ch02s01aus06.md)）定义，以及可能用于代替输入量的<*parameter*>实例内的参数名。如果找到上述任何一项，Abaqus将解释参数化输入文件并执行参数求值和替换任务。

结果，生成了一个修改后的输入文件，该文件不包含参数和参数依赖表定义以及<*parameter*>实例。此文件名为`*job-name*.pes`，随后提交执行分析。参数化输入文件的执行过程与分析输入文件处理器中对参数形状变化定义的额外处理不同，与非参数化输入文件的执行过程没有区别。参数化输入作业生成的所有文件将使用适当的扩展名附加在*job-name*之后命名。

#### 参数检查作业

您可以指定仅执行参数处理（求值和替换）的执行模式。参数检查执行模式与其他执行模式互斥，例如完整分析、数据检查、数据检查的继续、结果转换或恢复（参见["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行"，第3.2.2节"](pt01ch03s02abx02.md)）。

参数检查运行在您在输入中定义了复杂参数化的情况下很有用。在这些情况下，您可能希望在进一步操作之前研究参数求值和替换的结果。

参数检查运行不允许在后续运行中继续执行；作业必须从头重新运行。

| **输入文件用法：** | 在命令行上输入以下内容： |
| --- | --- |
|  | **abaqus** **job**=*job-name* **input**=*input-file* **parametercheck** |

#### 数据文件显示

数据（`.dat`）文件包含分析输入文件处理器生成的模型定义信息。您可以控制分析输入文件处理器生成的输出量；详情请参见["输出"中的"控制写入数据文件的分析输入文件处理器信息量"，第4.1.1节"](pt02ch04s01aus38.md#usb-out-ooutput-data-control)。特别是，您可以指定是否将原始输入（`.inp`）文件回显到数据文件（默认情况下，不会）。

对于参数化输入，此文件通常包含许多参数、参数依赖表和参数形状变化定义，以及许多<*parameter*>实例。为了验证参数化输入的定义，您可以创建原始输入文件的修改版本，显示参数及其值（此文件名为`*job-name*.par`）。您还可以创建`*job-name*.pes`文件，这是原始输入文件的修改版本，不包含参数和参数依赖表定义以及<*parameter*>实例。

| **输入文件用法：** | 使用以下选项将`*job-name*.par`文件的内容打印到数据文件： |
| --- | --- |
|  | ``` [*PREPRINT*](../key/key-link.md#usb-kws-mpreprint), PARVALUES=YES ``` 使用以下选项将`*job-name*.pes`文件的内容打印到数据文件： ``` [*PREPRINT*](../key/key-link.md#usb-kws-mpreprint), PARSUBSTITUTION=YES ``` |
