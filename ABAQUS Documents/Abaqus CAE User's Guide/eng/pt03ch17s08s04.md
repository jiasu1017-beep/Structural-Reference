# 17.8.4 Three-dimensional structured meshing







[Figure 17--49](pt03ch17s08s04.md#mgn-3dmappable) illustrates examples of simple three-dimensional regions that can be meshed using the structured meshing technique. 

**Figure 17–49** Regions that can be meshed using the structured meshing technique.

![](../graphics/mgn-3dmappable.png)

Meshing more complex regions with this technique may require manual partitioning. If you do not partition a complex region, your only meshing option may be the free meshing technique with tetrahedral elements. Meshes constructed using the structured meshing technique consist of hexahedral elements, which are preferred over tetrahedral elements.

The characteristics described below are required to mesh a three-dimensional region successfully using the structured meshing technique:
- The region cannot have any holes, isolated faces, isolated edges, or isolated vertices. For example, the regions shown in [Figure 17--50](pt03ch17s08s04.md#mgn-3dmaplimit) cannot be meshed using the structured meshing technique. **Figure 17--50** Regions that cannot be meshed using the structured meshing technique. ![](../graphics/mgn-3dmaplimit-nls.png) You can eliminate holes (whether they pass all the way through the part instance or just part way through) by partitioning their circumferences into halves, quarters, etc. For example, the four partitions in [Figure 17--51](pt03ch17s08s04.md#mgn-maphole) convert the part instance from one region with a hole to four regions without holes. **Figure 17--51** Partitions can make a part structured meshable. ![](../graphics/mgn-maphole-nls.png)
- You should limit arcs to 90 or less to avoid concavities along sides and at edges. For example, the part instance in [Figure 17--52](pt03ch17s08s04.md#mgn-mapconcave) has been partitioned so that the single region with 180 arcs becomes two regions with 90 arcs. **Figure 17--52** Limit arcs to 90 or less. ![](../graphics/mgn-mapconcave-nls.png)
- All the faces of the region must have geometries that could be meshed using the two-dimensional structured meshing technique. For example, without partitioning, the semicircles at either end of the part in [Figure 17--53](pt03ch17s08s04.md#mgn-mapsemicircle) have only two sides each. (A face must have at least three sides to be meshed using the structured meshing technique.) If you partition the part into two halves, each semicircle is divided into two faces with three sides each. **Figure 17--53** Partitioning creates two faces with three sides. ![](../graphics/mgn-mapsemicircle-nls.png)
- Exactly three edges of the region must meet at each vertex. For example, the vertex at the top of an unpartitioned pyramid in [Figure 17--54](pt03ch17s08s04.md#mgn-mapvertex) is connected to four edges. However, if you partition the pyramid into two tetrahedral regions, the vertex is connected to only three edges for each individual region. **Figure 17--54** After partitioning the vertex is connected to only three edges for each individual region. ![](../graphics/mgn-mapvertex-nls.png)
- The region must be bounded by at least four sides (a tetrahedral region). If a region is bounded by fewer than four sides, you can partition the region as necessary to create additional sides.
- If a region contains virtual topology, the region must be bounded by six sides.
- If a region cannot be meshed using the structured meshing technique, you can use virtual topology to combine faces until the region is bounded by six sides. [Figure 17--55](pt03ch17s08s04.md#mgn-mapvirtual) shows how you can use virtual topology to create a six-sided region that can be meshed using the structured meshing technique. **Figure 17--55** Virtual topology can make a part structured meshable. ![](../graphics/mgn-mapvirtual-nls.png)
- The angles between sides should be as close to 90 as possible; you should partition to eliminate angles greater than 150.
- Each side of the region must match one of the following definitions: - If the region is not a cube, a side must correspond to a single face; that is, the side must not contain multiple faces. - If the region is a cube, a side can be a connected set of faces that are on the same geometric surface. However, each face must have four sides. In addition, the pattern of the faces must allow rows and columns of hexahedral elements to be created in a regular grid pattern along that entire side when the cube is meshed. For example, [Figure 17--56](pt03ch17s08s04.md#mgn-goodpatternandmesh) shows two acceptable face patterns and the resulting regular grid pattern of elements created by meshing the cubes using the structured meshing technique. **Figure 17--56** Acceptable face patterns and the resulting meshes. ![](../graphics/mgn-goodpatternandmesh.png) The sides in [Figure 17--57](pt03ch17s08s04.md#mgn-badsides) do not have acceptable face patterns: **Figure 17--57** Unacceptable face patterns. ![](../graphics/mgn-badsides.png) The face pattern shown on the left is unacceptable for structured meshing because each face has only three sides. Each face in the pattern shown on the right has four sides, but the pattern does not allow a regular grid of elements to be created on the partitioned side of the cube, as shown in [Figure 17--58](pt03ch17s08s04.md#mgn-badpattern). **Figure 17--58** A regular grid of elements cannot be created. ![](../graphics/mgn-badpattern.png)
