# 12.2.4 在 Abaqus/Explicit 之间传递结果

**产品：** Abaqus/Explicit

##### **参考资料**

- ["结果传递概述，" 第 12.2.1 节](pt04ch09s02aus54.md)
- ["Abaqus/Explicit 与 Abaqus/Standard 之间的结果传递，" 第 12.2.2 节](pt04ch09s02aus55.md)
- [*STATE](../key/key-link.md#usb-kws-mstate)

### 概述

Abaqus/Explicit 之间的结果传递允许：
- 在不同的 Abaqus/Explicit 分析之间共享状态
- 模拟涉及多个阶段的制造工艺
- 从中断的分析中恢复

### 传递内容

Abaqus/Explicit 结果传递包括：

- 位移和旋转
- 速度场
- 加速度场
- 应力状态
- 应变状态
- 材料状态变量

### 传递方法

Abaqus/Explicit 使用二进制状态文件进行结果传递：

| **文件扩展名** | **内容** |
| --- | --- |
| .stt | 状态文件 |
| .abq | Abaqus 状态文件 |

| **输入文件用法：** | ``` [*STATE](../key/key-link.md#usb-kws-mstate), FILE NAME=*state_file* ``` |

### 状态文件

状态文件包含分析的完整状态信息：

| **输入文件用法：** | ``` [*STATE](../key/key-link.md#usb-kws-mstate), SAVE ``` |

### 要求

1. **相同版本**：两个分析应使用相同版本的 Abaqus
2. **兼容的单元类型**：单元类型必须兼容
3. **相同的材料模型**：材料定义必须匹配
4. **网格匹配**：节点和单元编号应匹配

### 应用场景

#### 连续成形工艺

1. 第一道次成形分析
2. 传递成形后的状态
3. 第二道次成形分析

#### 序列载荷

1. 执行第一阶段载荷
2. 传递完整状态
3. 执行第二阶段载荷

#### 中断恢复

1. 分析在某个时间点中断
2. 保存状态文件
3. 从中断点恢复分析

### 执行

```
# 保存状态
abaqus job=job1 state=save

# 恢复分析
abaqus job=job2 state=restore, file=job1
```

### 限制

- 仅适用于 Abaqus/Explicit
- 必须使用相同版本的 Abaqus
- 网格必须完全匹配
- 某些单元类型不支持

### 输入文件模板

#### 保存状态

```
*-heading
First stage analysis
**
*STATE, FILE NAME=stage1, SAVE
*STEP
*DYNAMIC, EXPLICIT
*BOUNDARY
*LOAD
*END STEP
```

#### 恢复并继续

```
*-heading
Second stage analysis
**
*STATE, FILE NAME=stage1, RESTORE
*STEP
*DYNAMIC, EXPLICIT
*BOUNDARY
*LOAD
*END STEP
```
