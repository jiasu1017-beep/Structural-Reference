### 概述

以下节点输出量的时间历史记录可在 Abaqus/Standard 分析中生成，并可读取到后续的 Abaqus/Standard 分析中作为顺序耦合多物理场工作流的预定义场：
- 温度
- 归一化浓度
- 电势

当模型中一个或多个物理场之间的耦合仅在一个方向上重要时，可以使用顺序耦合多物理场分析（一种特殊情况是顺序热应力分析（["顺序耦合热应力分析，" 第 16.1.2 节"](pt04ch16s01at39.md)）。虽然非耦合热应力分析是最常见的顺序多物理场工作流，但 Abaqus/Standard 中的预定义场功能直接支持涉及归一化浓度（["质量扩散分析，" 第 6.9.1 节"](pt03ch06s09at28.md)）和电势（["耦合热电分析，" 第 6.7.3 节"](pt03ch06s07at22.md)）的类似顺序工作流。与温度一样，归一化浓度和电势可以从输出数据库（`.odb`）文件读取到后续分析中作为预定义场。

当由先前分析的结果定义时，预定义场通常随位置变化且随时间变化——它们是预定义的，因为它们不会被当前分析改变。当从先前分析读取预定义场时，它们在节点处读取。然后根据需要在单元内进行插值（参见 ["预定义场，" 第 34.6.1 节中的"网格间数据插值"](pt07ch34s06aus128.md#usb-prc-pfields-tempinterpolate)）。可以读取任意数量的预定义场，材料属性可以定义为依赖于它们。此外，如果在材料属性定义中包含热膨胀（["热膨胀，" 第 26.1.2 节"](pt05ch26s01abm52.md)）或场膨胀（["场膨胀，" 第 26.1.3 节"](pt05ch26s01abm53.md)），则在应力分析中会产生体积应变。

预定义场可能通过以下方式影响系统响应：
- 本构行为，如定义为温度或场变量函数的屈服应力；或
- 当在应力/位移分析的材料定义中包含热或场膨胀行为时（["热膨胀，" 第 26.1.2 节"](pt05ch26s01abm52.md) 和 ["场膨胀，" 第 26.1.3 节"](pt05ch26s01abm53.md)）产生的体积应变。

### 为后续分析中的预定义场保存温度、归一化浓度和电势

节点温度、归一化浓度和电势可以存储为时间的函数，用于后续分析。温度可以存储在结果（`.fil`）文件或输出数据库（`.odb`）文件中，但归一化浓度和电势只有在存储在输出数据库文件中才能使用。保存的值必须读取到新分析中作为预定义场。参见 ["输出到数据和结果文件，" 第 4.1.2 节中的"节点输出"](pt02ch04s01aus39.md#usb-out-oprintfile-nodaloutput) 和 ["输出到输出数据库，" 第 4.1.3 节中的"节点输出"](pt02ch04s01aus40.md#usb-out-odboutput-nodaloutput)。

#### 为后续分析中的预定义场保存温度

要作为预定义场读取，节点温度必须存储在结果（`.fil`）文件或输出数据库（`.odb`）文件中作为时间的函数。您可以在非耦合热传递分析或耦合热电分析中请求节点温度输出（NT）。

#### 为后续分析中的预定义场保存归一化浓度

要作为预定义场读取，归一化浓度必须作为时间的函数存储在输出数据库（`.odb`）文件中——与节点温度不同，它们不能直接从结果文件读取。您可以在质量扩散分析中请求节点归一化浓度输出（NNC）。

#### 为后续分析中的预定义场保存电势

要作为预定义场读取，电势必须作为时间的函数存储在输出数据库（`.odb`）文件中——与节点温度不同，它们不能直接从结果文件读取。您可以在耦合热电分析或压电分析中请求节点电势输出（EPOT）。

### 作为温度场传递温度

要在当前分析的不同时间定义温度场，您可以读取存储为热传递结果或输出数据库文件中时间函数的节点温度。节点可以从当前问题中移除；例如，在顺序热应力分析中，表示热传递网格的非结构部分（如绝缘层或冷却液）的单元可以在应力分析中省略。当读取热传递结果文件或输出数据库文件时，不存在于当前分析网格中的节点上的温度将被忽略。

您必须指定包含所需节点温度的热传递分析结果文件或输出数据库文件的名称。文件扩展名是可选的。如果热传递模型和当前分析模型共享相同的网格，则默认为结果文件。如果热传递模型和当前分析模型具有不同的网格，则必须使用输出数据库文件。更多信息请参见 ["预定义场，" 第 34.6.1 节中的"从用户指定文件读取场值"](pt07ch34s06aus128.md#usb-prc-pfields-readvalues)。

如果两个模型都包含部件和装配定义，则需要两个分析的部件（`.prt`）文件才能将温度从热传递分析传递到当前分析。如果热传递模型以部件实例的装配形式定义，则当前分析也必须如此。在传递温度的节点处，部件实例名称和局部节点编号在两个分析中必须相同。

### 从输出数据库传递温度、归一化浓度和电势到预定义场

要为当前分析中的不同时间定义预定义场，您可以读取存储为输出数据库文件中时间函数的节点温度、归一化浓度或电势。节点可以从当前问题中移除。当输出数据库文件上的节点输出变量位于不存在于当前分析网格中的节点上时，它们将被忽略。

您必须指定包含所需节点输出变量的输出数据库文件的名称，以及节点输出标签（NT、NNC 或 EPOT）以识别正在读取的场。更多信息请参见 ["预定义场，" 第 34.6.1 节中的"使用用户指定输出数据库文件中的节点标量输出值定义场"](pt07ch34s06aus128.md#usb-prc-pfields-scalar-output)。

如果两个模型都包含部件和装配定义，则需要两个分析的部件（`.prt`）文件才能将节点结果从原始分析传递到当前分析。如果原始模型以部件实例的装配形式定义，则当前分析也必须如此。在传递节点结果的节点处，部件实例名称和局部节点编号在两个分析中必须相同。

### 输入文件模板

湿应力分析是顺序耦合多物理场分析的一个例子。典型的顺序耦合湿应力分析包括两个 Abaqus/Standard 运行：质量扩散分析和后续应力分析。归一化浓度存储在质量扩散分析的输出数据库文件中，并作为预定义场读取到后续应力分析中。

以下模板显示质量扩散分析 `massdiffusion.inp` 的输入：

```
[*HEADING](../key/key-link.md#usb-kws-mheading)
…
[*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=DC2D4
*（选择质量扩散单元类型）*
…
[*STEP](../key/key-link.md#usb-kws-hstep)
[*MASS DIFFUSION](../key/key-link.md#usb-kws-hmassdiff)
…
*施加载荷和边界条件*
…
** 将所有归一化浓度写入输出
** 数据库文件 `massdiffusion.odb`
[*OUTPUT](../key/key-link.md#usb-kws-houtput), FIELD
[*NODE OUTPUT](../key/key-link.md#usb-kws-hnodeoutput), NSET=NALL
 NNC
[*END STEP](../key/key-link.md#usb-kws-hendstep)
```

以下模板显示后续静态结构分析的输入：

```
[*HEADING](../key/key-link.md#usb-kws-mheading)
…
[*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=CPE4R
*（选择与所用质量扩散单元类型兼容的连续体单元类型）*

[*MATERIAL](../key/key-link.md#usb-kws-mmaterial)
[*EXPANSION](../key/key-link.md#usb-kws-mexpansion), FIELD=1
*（定义场膨胀，以便归一化浓度在应力分析中引起体积应变）*
…
[*STEP](../key/key-link.md#usb-kws-hstep)
[*STATIC](../key/key-link.md#usb-kws-hstatic)
…
*施加结构载荷和边界条件*
…
[*FIELD](../key/key-link.md#usb-kws-hfield), FILE=`massdiffusion.odb`, OUTPUT VARIABLE=NNC, FIELD=1
*将所有归一化浓度从输出数据库文件读取到场变量 1*
…
[*END STEP](../key/key-link.md#usb-kws-hendstep)
```
