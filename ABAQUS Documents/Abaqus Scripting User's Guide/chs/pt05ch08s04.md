# 8.4 切换到GUI模块

当用户从上下文栏的"模块"列表中选择一个模块时，GUI基础设施执行以下操作：

- 调用当前GUI模块的`deactivate`方法。
- 调用用户选择的GUI模块的`activate`方法。

如果需要在进入或离开模块时执行任何特殊处理，你可以编写你自己的`activate`或`deactivate`方法。如果在用户更改模块时需要向kernel发出命令，必须使用`AFXModuleGui`对象的`sendCommandString`方法发出命令。如果不使用`sendCommandString`方法，应用程序在尝试处理命令时可能会挂起。你应该将对`sendCommandString`方法的调用语句包含在`try`块中，以捕获kernel命令生成的任何异常。

要使用脚本切换到GUI模块，可以使用`switchModule`方法。例如，如果想在应用程序启动时切换到你的模块，可以将以下行添加到应用程序启动文件：

```
switchModule('My Module')
```

此行应出现在`app.run()`语句之前。

你可以使用`setSwitchModuleHook`(*function*)方法设置一个回调函数，当用户切换到GUI模块时将调用该函数。每次用户切换到GUI模块时，你的函数将被调用，模块的名称将被传递给函数。例如，

```
def onModuleSwitch(moduleName):
    if moduleName == 'Part':
       # 执行部件模块任务
    elif moduleName == 'Mesh':
       # 执行网格模块任务
    等等。

setSwitchModuleHook(onModuleSwitch)
```
