### 概述

Abaqus/Standard 中通用接触的接触稳定化：
- 通常有助于稳定静态分析中未约束的刚体模式；
- 可选择性地应用于通用接触域内的特定区域；以及
- 可以随时间变化。

### 基于表面相对运动的粘性阻尼的稳定化

接触稳定化基于反对附近表面之间增量相对运动的粘性阻尼，与接触阻尼的方式相同（见 ["接触阻尼，" 第 37.1.3 节](pt09ch37s01aus167.md)）。接触稳定化最常见的目的是稳定在接触闭合和摩擦约束这些运动之前未约束的"刚体运动"。人工稳定化（如接触稳定化）的目标是提供足够的稳定化以实现稳健、高效的仿真，而不降低结果的准确性。在大多数情况下，接触稳定化默认不激活（例外情况在 ["Abaqus/Standard 中与接触建模相关的常见困难，" 第 39.1.2 节](pt09ch39s01aus184.md#usb-cni-acontacttrouble-pointstab) 中的"单点接触"中讨论），因此如果分析中可能存在与未约束刚体模式相关的收敛问题，通常需要激活接触稳定化。激活后，接触稳定化是高度自动化的。

### 为相互作用分配稳定化

通用接触中特定相互作用的接触稳定化分配可以全局或局部进行，作为步骤定义的一部分。在大多数情况下，您只需要指定哪些相互作用有资格使用接触稳定化，而不必调整前面讨论的比例因子。

| **输入文件用法：** | 使用以下选项指定哪些相互作用应使用接触稳定化： |
| --- | --- |
|  | ``` [*CONTACT STABILIZATION](../key/key-link.md#usb-kws-hcontactstab) *surf_1*, *surf_2* ``` 如果省略第一个表面名称，则假定一个包含整个通用接触域的默认表面。如果省略第二个表面名称，则假定第一个表面与自身之间的接触。 |

| **Abaqus/CAE 用法：** | 使用以下选项向各个表面配对分配接触稳定化定义： |
| --- | --- |
|  | Interaction 模块： **Create Interaction**: **General contact (Standard)**: **Contact Properties**: **Stabilization assignments: Edit**: 选择左列中的表面和稳定化名称，然后点击中间的箭头将它们转移到接触稳定化分配列表中 |

### 指定稳定化比例因子

在某些情况下，您可能需要调整与接触稳定化关联的一个或多个比例因子。您可以使用此选项的多个实例来为不同的通用接触相互作用实现不同的比例因子设置。

#### 常数比例因子

如上所述，对于稳定化压力和剪切应力的表达式，比例因子 ![](../graphics/usb_eqn06166.gif) 适用于法和切向稳定，而比例因子 ![](../graphics/usb_eqn06171.gif) 仅适用于切向稳定。对于指定相互作用的常数比例因子 ![](../graphics/usb_eqn06166.gif) 的默认设置为一。

![](../graphics/usb_eqn06171.gif) 的默认设置为零，因此对于指定相互作用默认不存在切向稳定化刚度。在许多情况下，法向-only 接触稳定化就足够了。其他分析可以从切向稳定化刚度中获益；但是，如果您指定 ![](../graphics/usb_eqn06171.gif) 为非零值，请记住，如果附近表面之间发生大的相对切向运动，切向接触稳定化通常会吸收大量能量。稳定化吸收的大能量是分析结果可能受到稳定化显著影响的一个指示。法向接触稳定化不太可能吸收大量能量，因此对结果的影响较小。

| **输入文件用法：** | ``` [*CONTACT STABILIZATION](../key/key-link.md#usb-kws-hcontactstab), SCALE FACTOR=![](../graphics/usb_eqn06166.gif), TANGENT FRACTION=![](../graphics/usb_eqn06171.gif) ``` |

#### 迭代相关比例因子

为了减少或消除接触稳定化显著影响报告解决方案的可能性，可以引入在增量迭代中变化的比例因子。在增量的早期迭代中施加更多稳定化有助于避免在建立主动接触区域之前的数值问题。在增量的后期迭代中减少或关闭稳定化有助于提高增量最终收敛迭代的准确性。

您可以指定这些比例因子。例如，指定"1,0"会导致比例因子在初始迭代中（直到各种收敛度量满足或接近满足）为 unity，然后在最终迭代中重置为零（有效关闭稳定化），直到再次满足收敛检查。

| **输入文件用法：** | ``` [*CONTACT STABILIZATION](../key/key-link.md#usb-kws-hcontactstab), SCALE FACTOR=USER ADAPTIVE ``` |
| --- | --- |

#### 时间相关比例因子

比例因子 ![](../graphics/usb_eqn06168.gif) 和 ![](../graphics/usb_eqn06169.gif) 控制接触稳定化的时间依赖性。默认情况下，![](../graphics/usb_eqn06168.gif) 等于剩余步长的分数。另一个因素根据 ![](../graphics/usb_eqn06177.gif) 变化，其中 ![](../graphics/usb_eqn06178.gif) 是每个增量的缩减因子（默认为 0.1），![](../graphics/usb_eqn06179.gif) 是步骤内的增量数。这些默认值意味着稳定化在同一大小增量中减少了超过一个数量级，并且在步骤的最终增量中不施加稳定化。这些默认值适用于大多数情况下，接触稳定化旨在在建立接触的初始增量中提供稳定化。

提供了两个调整时间相关比例因子的选项：您可以引用将管理 ![](../graphics/usb_eqn06168.gif) 的振幅曲线，并且可以指定 ![](../graphics/usb_eqn06178.gif) 的值（回忆前面给出的表达式 ![](../graphics/usb_eqn06177.gif)）。例如，如果不稳定模式在接触建立后仍然存在，您可能希望 ![](../graphics/usb_eqn06168.gif) 和 ![](../graphics/usb_eqn06169.gif) 在整个步骤中保持为一，这可以通过引用具有恒定值一的振幅并将每个增量缩减因子 ![](../graphics/usb_eqn06178.gif) 设置为一来实现。

| --- | --- |

| **Abaqus/CAE 用法：** | Load 或 Interaction 模块： **Create Amplitude**: **Name:** *name* Interaction 模块： ****Interaction**![](../graphics/images/arrow.gif)**Contact Stabilization**![](../graphics/images/arrow.gif)**Create****: **Reduction factor:** ![](../graphics/usb_eqn06178.gif), **Amplitude:** *name* |

##### 在后续步骤中重置时间相关比例因子

除非指定了振幅引用，否则接触稳定化定义不会影响后续步骤。如果指定了基于总时间的振幅，相同的振幅曲线继续管理后续步骤中 ![](../graphics/usb_eqn06168.gif) 的变化，直到为相互作用分配新的接触稳定化定义。如果指定了基于步骤时间的振幅，振幅曲线在单个步骤中管理 ![](../graphics/usb_eqn06168.gif)，并且 ![](../graphics/usb_eqn06168.gif) 在后续步骤中保持不变（处于结束值），直到为相互作用分配新的接触稳定化定义。在这两种情况下，您还可以重置接触稳定化定义以从步骤中移除稳定化。重置确保先前步骤的接触稳定化选项不会影响当前步骤。

| **输入文件用法：** | ``` [*CONTACT STABILIZATION](../key/key-link.md#usb-kws-hcontactstab), RESET ``` |
| --- | --- |

| **Abaqus/CAE 用法：** | Load 或 Interaction 模块： **Create Amplitude**: **Name:** *name* Interaction 模块： ****Interaction**![](../graphics/images/arrow.gif)**Contact Stabilization**![](../graphics/images/arrow.gif)**Create****: **Reset values from previous steps** |
| --- | --- |

#### 间隙相关比例因子

比例因子 ![](../graphics/usb_eqn06170.gif) 控制作为表面之间局部分离距离函数的接触稳定化。默认情况下，此因子对于零间隙距离为一，当间隙距离大于或等于特征表面尺寸时为零。您可以控制 ![](../graphics/usb_eqn06170.gif) 变为零的间隙距离。不建议为此阈值距离指定大值，因为随着阈值距离增加，每次迭代的解决方案成本有增加的趋势（由于连接性增加）。

| **输入文件用法：** | ``` [*CONTACT STABILIZATION](../key/key-link.md#usb-kws-hcontactstab), RANGE=*distance* ``` |
| --- | --- |

| **Abaqus/CAE 用法：** | Interaction 模块： ****Interaction**![](../graphics/images/arrow.gif)**Contact Stabilization**![](../graphics/images/arrow.gif)**Create****: **Zero stabilization distance:** **Specify:** *distance* |
| --- | --- |

### 接触稳定化定义的层次结构

上面讨论的界面是指定通用接触稳定化的推荐方法；但也可以通过两种其他方式为通用接触相互作用引入接触稳定化。重叠情况下的优先级顺序如下：
- 第一优先级分配给本节讨论的接触稳定化分配选项。
- 第二优先级分配给 ["在 Abaqus/Standard 中调整接触控制" 第 36.3.6 节](pt09ch36s03aus150.md#usb-cni-acontacttrouble-stabilize) 中讨论的接触稳定化分配选项。
- 第三优先级分配给 ["单点接触" 中讨论的默认接触稳定化，在 "Abaqus/Standard 中与接触建模相关的常见困难，" 第 39.1.2 节](pt09ch39s01aus184.md#usb-cni-acontacttrouble-pointstab)。
