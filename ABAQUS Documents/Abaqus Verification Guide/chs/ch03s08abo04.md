# 3.8.1 子模型：概述







子模型是通过从原始"全局"网格上的插值解"驱动"该网格边界节点来分析结构部分的技术，使用不同（通常更精细）的网格（见 ["基于节点的子模型，" Abaqus 分析用户指南第 10.2.2 节](../usb/usb-link.md#usb-anl-asubmodeldisp)）。要执行子模型分析，必须在全局分析中在文件输出上保存节点量，如位移、温度、压力、位移相位等（通常使用粗网格完成）。全局模型文件输出附加到子模型运行（通过 Abaqus 执行步骤上的 **globalmodel** 参数）以驱动子模型上的边界节点（通常使用更精细的网格完成）。两个模型必须使用相同的参考框架。全局和子模型网格可以在同一组单元内具有不同的单元类型：平面实体到平面实体、轴对称实体到轴对称实体、三维实体到三维实体、通用壳到通用壳等。对于壳到实体子模型，全局模型由壳单元组成，子模型由三维连续体单元组成。全局分析和子模型分析之间的步骤类型可以不同。例如，全局模型中的线性静态分析可以驱动子模型中的弹塑性静态解（只要塑性不会影响驱动的边界节点），或者全局模型中的动态分析可以驱动子模型中的静态解（这假设在子模型级别可以忽略惯性力）。此外，全局过程可以在 Abaqus/Standard 中执行以驱动 Abaqus/Explicit 中的子模型过程，反之亦然。例如，全局模型中的 Abaqus/Standard 静态分析可以驱动子模型中的准静态 Abaqus/Explicit 分析。

验证测试根据子模型功能支持的单元类型分为多个部分。在每个部分中，对具有有限数量单元的小型模型测试了单元和步骤的组合。验证了从全局分析插值的驱动节点处的值（或振幅）。在大多数情况下，子模型分析中的应力和应变场与全局分析的结果匹配。然而，在某些问题中，网格太粗糙，无法产生良好的应力和应变一致性。

每个测试由两个输入文件组成：全局分析和子模型分析。相同的全局文件可以驱动多个子模型分析运行，每个运行使用可能与全局分析相同或不同的网格。
