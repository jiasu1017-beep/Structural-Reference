# 3.6.1 Abaqus使用的文件扩展名





**产品：**Abaqus/Standard  Abaqus/Explicit  Abaqus/CFD  

##### **参考**

- ["Abaqus执行程序：概述，" 3.1.1节](pt01ch03s01abo02.md)

### 概述

**abaqus**程序生成多个文件。其中一些文件包含分析、后处理和转换结果，并保留供其他分析选项、重启或后处理使用。本节描述由Abaqus创建和保留的文件。

其他文件仅在Abaqus执行时存在，在运行完成时删除。这些临时文件在临时目录中生成。生成的临时文件的数量和类型取决于分析过程、内存管理参数和环境设置。

Abaqus使用的某些文件扩展名也被其他软件应用程序使用。您必须处理与其他应用程序的任何文件扩展名冲突。

### 文件扩展名

**`abq`**

状态文件，仅由Abaqus/Explicit使用。它由**analysis**、**continue**和**recover**选项写入。由**convert**和**recover**选项读取。此文件需要用于重启。

**`axi`**

对称模型数据文件，仅由Abaqus/Standard使用。它在**datacheck**和**analysis**选项期间的对称模型生成期间写入。

**`bsp`**

包含网格截面轮廓的梁截面属性的文本文件。它在网格梁截面生成期间由Abaqus/Standard写入。

**`c`**

用户子程序或其他专用C文件。

**`c++`**

用户子程序或其他专用C++文件。

**`cpp`**

用户子程序或其他专用C++文件。

**`cid`**

自动释放文件，包含许可恢复和暂停所需的信息。

**`com`**

命令文件，由Abaqus执行程序创建。

**`dat`**

打印输出文件。它由**analysis**、**datacheck**、**parametercheck**和**continue**选项写入。Abaqus/Explicit和Abaqus/CFD不向此文件写入分析结果。

**`f`**

用户子程序或其他专用FORTRAN文件。

**`fil`**

结果文件。它由**analysis**和**continue**选项在Abaqus/Standard中写入，由Abaqus/Explicit中的**convert**=`select`和**convert**=`all`选项写入。

**`fin`**

使用**abaqus ascfil**命令更改`.fil`文件格式时创建的结果文件。它可以是ASCII或二进制格式。（请参阅["ASCII转换结果（`.fil`）文件，" 3.2.13节](pt01ch03s02abx13.md)。）ASCII格式便于在不具有兼容二进制数据格式的机器之间传输数据。

**`inp`**

分析输入文件。当选择**analysis**、**datacheck**和**parametercheck**选项时读取。

**`ipm`**

进程间消息文件。当从Abaqus/CAE运行分析时写入，它包含从Abaqus/Standard、Abaqus/Explicit或Abaqus/CFD发送到Abaqus/CAE的所有消息日志。

**`lck`**

输出数据库的锁定文件。每当以写访问权限打开输出数据库文件时都会写入此文件；它阻止您从多个源同时拥有输出数据库的写权限。当输出数据库文件关闭或创建它的分析结束时，锁定文件会自动删除。**ask_delete**环境文件参数设置不会影响锁定文件。

**`log`**

日志文件，包含当前Abaqus执行程序运行的模块的开始和结束时间。

**`mdl`**

模型文件，由Abaqus/Standard和Abaqus/Explicit使用。它由**datacheck**选项写入。它由**analysis**和**continue**选项在Abaqus/Standard中读取和写入。它由**analysis**和**continue**选项在Abaqus/Explicit中读取。如果元素操作在Abaqus/Standard分析中并行执行，则可能存在多个模型文件。在这种情况下，进程标识符会附加到文件名。此文件需要用于重启。

**`msg`**

消息文件。它由**analysis**、**datacheck**和**continue**选项在Abaqus/Standard和Abaqus/Explicit中写入。如果元素操作在Abaqus/Standard分析中并行执行，则可能存在多个消息文件。在这种情况下，进程标识符会附加到文件名。

**`nck`**

由Abaqus/Standard使用的昵称文件。它存储模型中自由度的内部标识符集。

**`odb`**

输出数据库。它由**analysis**和**continue**选项在Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD中写入。它由Abaqus/CAE（Abq/Viewer）的可视化模块读取，由**convert**=`odb`选项读取。此文件需要用于重启。

**`pac`**

包文件，包含模型信息，仅由Abaqus/Explicit使用。它由**analysis**和**datacheck**选项写入。它由**analysis**、**continue**和**recover**选项读取。此文件需要用于重启。

**`par`**

显示输入参数及其值的参数化输入文件的修改版本。

**`pes`**

显示参数化输入文件的修改版本，该文件不含参数信息（在执行输入参数求值和替换之后）。

**`pmg`**

参数求值和替换消息文件。当输入文件被参数化时写入。

**`prt`**

部件文件，由Abaqus/Standard和Abaqus/Explicit使用。此文件用于存储部件和装配信息，即使输入文件不包含装配定义也会创建。部件文件需要用于重启、导入、顺序耦合热应力分析、对称模型生成和水下冲击分析，即使模型不是用部件实例的装配定义的。此文件也可能需要用于子模型分析。

**`psf`**

Python脚本文件。您必须创建此类文件来定义参数化研究。

**`res`**

重启文件，包含继续先前分析所需的信息，由Abaqus/Standard和Abaqus/Explicit使用。重启文件由**analysis**、**datacheck**和**continue**选项写入。它由任何重启分析读取。

**`sel`**

选定结果文件，由Abaqus/Explicit使用。它由**analysis**、**continue**和**recover**选项写入，由**convert**=`select`选项读取。此文件需要用于重启。

**`sim`**

模型和结果文件，由Abaqus/CFD使用。当指定**resultsformat**=`sim`或`both`选项时，也由Abaqus/Standard和Abaqus/Explicit使用。它由**datacheck**选项写入。它由**analysis**和**continue**选项读取和写入。此文件需要用于重启。

线性动力学数据文件，当指定**resultsformat**=`odb`选项时由Abaqus/Standard使用。它在SIM线性动力学分析（请参阅["在线性动力学分析中使用SIM架构" 6.3.1节动态分析过程概述](pt03ch06s03abo07.md#usb-anl-alineardynamics)）的频率提取过程期间写入，用于存储特征向量、子结构矩阵和其他模态系统信息。此文件需要用于重启。

**`sta`**

状态文件。Abaqus在此文件中写入**analysis**、**continue**和**recover**选项的增量摘要。

**`stt`**

状态文件。它由Abaqus/Standard和Abaqus/Explicit中的**datacheck**选项写入。它由**analysis**和**continue**选项在Abaqus/Standard中读取和写入。它由**analysis**和**continue**选项在Abaqus/Explicit中读取。如果元素操作在Abaqus/Standard分析中并行执行，则可能存在多个状态文件。在这种情况下，进程标识符会附加到文件名。此文件需要用于重启。

**`sup`**

子结构文件，由Abaqus/Standard使用。

**`var`**

包含参数化研究生成的输入文件变体信息的文件。

**`023`**

通信文件，由Abaqus/Standard和Abaqus/Explicit使用。它由**analysis**和**datacheck**选项写入，由**analysis**和**continue**选项读取。
