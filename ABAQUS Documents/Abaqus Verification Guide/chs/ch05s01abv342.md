# 5.1.26 定义温度、场变量和压力应力值





**产品：**Abaqus/Standard、Abaqus/Explicit  

### 测试功能

测试温度、场变量和压力应力程序的应用。第一组测试验证温度和场变量数据从热传递分析正确传递到结构分析。第二组测试验证这些命令与复合结构壳的结合使用。第三组测试验证在顺序热应力分析中温度到边中节点的插值，当热传递分析使用一阶单元而应力分析使用二阶单元时。第四组测试验证温度在不同网格之间正确插值。热传递模型和应力分析模型可能具有不同的网格，当前模型的节点温度将从热传递模型的节点温度进行插值。第五组测试验证使用数据行输入为这两个命令的各种组合正确施加温度和压力。第五组测试验证从热传递分析得到的解相关变量作为场变量正确传递到应力分析中。

在几个测试中请求零增量结果文件输出。此输出用于为后续结构分析定义温度、场变量和压力应力的初始值。

### I. 从结果文件读取温度和场变量数据

### 测试单元

DC1D2    T3D2

### 问题描述

这些测试验证温度和场变量值被正确传递到结构。被分析的结构是由10个T3D2单元组成的悬臂桁架。

使用三个不同的瞬态热传递运行来生成包含温度历史的三结果文件。这些文件将作为温度或场变量数据读入后续应力分析。所有运行都从整个桁架处于某个初始温度开始；然后将整个桁架的温度斜坡上升到某个新温度。

三个热传递运行如下：

**xtfvtrt1.inp**

初始温度：100

最终温度：200

**xtfvtrt2.inp**

初始温度：200

最终温度：250

**xtfvtrt3.inp**

初始温度：200

最终温度，步骤1：180

最终温度，步骤2：100

后续应力分析运行如下：

**xtfvtrs1.inp**

此文件测试使用结果文件设置温度和多个场变量。温度和两个场变量通过从热传递运行的结果文件读取数据设置如下：

xtfvtrt1.fil → 温度

xtfvtrt2.fil → 场变量1

xtfvtrt1.fil → 场变量2

**xtfvtrs2.inp**

此文件测试从结果文件设置场变量，而问题中不存在温度。此测试很重要，因为温度和场变量在内部存储的方式。场变量通过从第一次热传递运行的结果文件读取数据设置如下：

xtfvtrt1.fil → 场变量

**xtfvtrs3.inp**

此文件测试当对分析中未使用的变量存在初始条件规范时温度和场变量的存在。为温度和两个场变量给定初始条件，然后仅由结果文件设置温度和第一个场变量。此外，对于同一场变量包含两个过程，以测试仅使用最后一个命令。温度和场变量通过从热传递运行的结果文件读取数据设置如下：

xtfvtrt1.fil → 温度

xtfvtrt2.fil → 场变量1

**xtfvtrs4.inp**

这是一个涉及温度和一个场变量的三步骤问题。在第一步中，使用幅值曲线将温度设置为200，场变量设置为250。在第二步中，温度斜坡下降到150，场变量由xtfvtrt2.fil的结果文件定义。在第三步中，温度和场变量都重置为其初始条件。

必须通过此测试确认以下内容：
- 使用幅值曲线正确设置温度和场变量。
- 如果使用幅值曲线设置温度和场变量，则忽略初始条件。
- 如果应力分析时间周期与热分析时间周期不同，则正确缩放结果文件数据的时间。
- 如果命令既从数据行也从结果文件读取温度/场变量数据，则数据行输入优先。
- 如果OP参数给定为NEW值，则温度/场变量必须斜坡回到初始条件或设置为数据行上定义的新值。

**xtfvtrsr.inp**

此分析从第三步重新启动xtfvtrs4.inp。执行两个附加步骤。在第一步中，温度通过从xtfvtrt1.fil读取结果文件设置，场变量通过从xtfvtrt2.fil读取结果文件设置。在第二步中，温度使用从xtfvtrt3.fil结果文件的第二步数据设置。

**xtfvtrs5.inp**

此运行设置温度和场值中历史数据的开始和结束步骤及增量数。温度和两个场变量通过从热传递运行的结果文件读取数据设置如下：

xtfvtrt1.fil → 温度，开始增量=1，结束增量=5

xtfvtrt2.fil → 场变量1，开始增量=5，结束增量=8

xtfvtrt1.fil → 场变量2，开始增量=6，结束增量=10

所有数据都从步骤1读取，因此在所有情况下开始和结束步骤都是1。

### II. 复合壳温度载荷

### 问题描述

在Abaqus/Standard中，这些测试验证预定义温度和场变量与复合结构壳的结合使用。温度和场变量结果都从单个先前运行的热传递壳分析生成。同样的分析可用于生成场变量结果，因为在场变量在Abaqus结果文件中与温度存储方式相同。

进行稳态热传递分析以获得复合层厚度方向的温度分布。热传递问题涉及三层复合壳，其顶部和底部表面承受规定的热边界条件。

在后续结构壳模型中，每层使用五个截面点。温度和场变量通过从先前热传递分析每层可用的三个值进行线性插值分配给这五个点。这些分析的结果验证温度和场变量被正确分配。

在Abaqus/Explicit中，代替顺序分析，对三层复合壳进行瞬态耦合动态温度-位移分析，其顶部和底部表面承受规定的热边界条件。规定足够大的步骤时间，使得分析可以达到稳态。每层使用三个温度点。获得的温度分布与精确解进行比较。

### 结果与讨论

热传递运行与复合壳层温度分布的精确解匹配。此外，这些值在Abaqus/Standard中作为温度或场变量正确传递到结构复合壳。在Abaqus/Explicit中，热传递和应力分析同时求解，结果与分析解和Abaqus/Standard解匹配。
- 第1层底部和第2层顶部的温度/场变量为425。
- 第1层顶部和第2层底部的温度/场变量为373.2。
- 第2层顶部和第3层底部的温度/场变量为336.8。
- 第3层顶部的温度/场变量为287.5。

每层顶部和底部之间温度或场变量呈线性变化。

### III. 到边中节点的温度插值

### 问题描述

这些测试验证在顺序热应力分析中温度到高阶单元边中节点的插值，当热传递分析使用一阶单元而应力分析使用二阶单元时。

热传递分析的结果被读入应力分析，应用于热传递分析的初始条件被读入应力分析。在两种分析中，边中节点处的温度从单元的角节点进行插值。温度插值对每个单元按边缘进行。因此，单元边中节点处的温度从相应角节点处的温度进行线性插值。

对一维、二维和三维单元测试边中节点温度插值。

在热传递分析和应力分析的有限单元模型中仅使用一个单元。假设任意材料属性。

### 结果与讨论

使用高阶单元的应力分析结果与使用线性单元获得的结果比较良好。

### V. 使用用户指定区域在不同网格之间传递温度

### 问题描述

本节中的验证问题测试使用用户指定区域在不同网格之间温度的插值。模型由两个零件实例组成，如[图5.1.26-6](ch05s01abv342.md#ver-prc-drivingelsets-model)所示。两个零件之间存在一个微小的间隙。沿间隙施加低间隙热传递，导致两个相邻表面之间产生温度跳跃。在这种情况下，使用常规插值的温度映射可能由于界面表面附近目标节点与该表面附近驱动单元之间的模糊关联而对相邻表面上的节点产生错误的温度分配。不同网格插值功能通过明确指定热传递分析中读取温度的源区域和当前分析中映射温度的目标区域来解决这种模糊性。

**图5.1.26-6** 间隙为说明目的放大的模型几何。

![](../graphics/ver-prc-drivingelsets-model.png)

**边界条件：**

组件在左侧边界保持零温度，在右侧边界承受0.003的恒定表面热通量，在顶部承受1的恒定表面热通量。间隙具有系数为0.01的低间隙热传导。

### 结果与讨论

当前分析中内部零件顶部边界上的节点稍微向上移动，使其落在外部零件内部。进行此移动是为了说明在插值过程中会导致错误驱动单元选择的情况。[图5.1.26-7](ch05s01abv342.md#ver-prc-no-drivingelsets)显示了未指定不同网格插值功能时的映射温度结果。测试表明，当未为不同网格之间温度插值指定区域时，内部零件顶部表面上的那些节点的温度映射不正确。错误发生是因为Abaqus搜索包围当前分析中每个节点或最接近每个节点的父元素。对于内部零件顶部表面上的节点，父元素在外部零件内部找到，导致节点上的温度定义错误。[图5.1.26-8](ch05s01abv342.md#ver-prc-drivingelsets)显示了具有不同网格插值功能时的映射温度结果。该功能通过明确指定插值的源和目标区域来纠正错误。在此测试案例中，从先前热传递分析中的驱动单元集被选择以覆盖与当前分析中驱动节点集所覆盖的相同实例区域；因此，实现了实例到实例的映射。

### VI. 从结果文件读取温度和压力数据

### 测试单元

### 问题描述

这些测试验证当在质量扩散分析中使用温度和压力应力值的各种组合时，温度和压力被正确施加到结构。温度和压力应力初始条件从Abaqus/Standard分析的结果文件读取，使用数据行输入按以下顺序将一系列压力和温度载荷施加到单元的节点：
- 步骤1：浓度从单元角落的0斜坡上升到100。
- 步骤2：沿单元的一条对角线施加压力梯度。
- 步骤3：所有压力重置为初始条件。
- 步骤4：沿步骤2中压力梯度相同的单元对角线施加温度梯度。
- 步骤5：所有温度重置为初始条件。
- 步骤6：沿单元对角线同时施加压力和温度梯度。
