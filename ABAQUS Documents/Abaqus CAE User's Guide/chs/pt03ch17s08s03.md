# 17.8.3 二维结构化网格





如果二维区域具有以下特征，则可以使用结构化网格技术进行网格划分：
- 该区域没有孔、孤立边或孤立顶点。图17-44展示了一些无法进行结构化网格划分的区域。
**图17-44** 无法进行结构化网格划分的区域。
![](../graphics/mgn-2dmaplimit-nls.png)
- 该区域由三到五个逻辑边限定，每个边是一组连通的边。

一般来说，结构化网格为您提供了对Abaqus/CAE生成的网格的最大控制。如果您使用所有四边形单元对四边形区域进行网格划分，则边界周围的单元边总数必须为偶数。对于三边和五边区域，约束方程更复杂。Abaqus/CAE在生成结构化网格时会尽可能尊重种子分布。（种子分布描述种子的间距，不一定是种子的数量。例如，种子的间距是沿边均匀分布还是更集中在一端？）但是，网格必须在区域之间兼容，Abaqus/CAE可能会调整与使用自由网格技术网格化的区域相邻的网格区域的节点。因此，单元节点可能与种子不完全匹配。

图17-45展示了四边形区域边上的种子以及不同网格控制的效果。

**图17-45** 不同网格控制的效果。

![](../graphics/mgn-singletriangle-nls.png)

网格控制效果如下：
- 在四边形区域中，四边形主导结构化网格与种子完全匹配。边界周围有奇数个单元，Abaqus/CAE在网格中插入单个三角形。（相比之下，当您使用结构化四边形主导单元对三边或五边区域进行网格划分时，Abaqus/CAE不会插入任何三角形。生成的网格使用所有四边形单元；但是，生成的网格可能与网格种子不完全匹配。）
- 两个四边形结构化网格与种子不匹配。当您选择最小化网格过渡时，这更加明显。
- 三角形结构化网格也与种子不匹配。Abaqus/CAE通过分割具有最小化网格过渡的四边形结构化网格的对角线来创建三角形网格。

如果边所夹的角度较浅，Abaqus/CAE会自动将边合并为一个逻辑边。例如，图17-46中每个区域有五条边。

**图17-46** 夹角较浅的边。

![](../graphics/mgn-mapped3.png)

但是，由于每个区域中的顶部两条边所夹的角度较浅，Abaqus/CAE将这两条边视为一条逻辑边。因此，四边形区域的网格图案应用于这些区域。如果区域包含虚拟拓扑，则只能在该区域由四条边限定时使用结构化网格技术进行网格划分。您不能使用结构化网格对包含虚拟拓扑的三边或五边区域进行网格划分。

您可以使用**网格控制**对话框中的**重新定义区域角**按钮自行合并边，无论它们所夹的角度如何。（要显示**网格控制**对话框，请从主菜单栏选择****网格**![](../graphics/images/arrow.gif)**控制****。）此技术允许您控制将哪种结构化网格图案应用于二维区域。（此技术不适用于三维区域。）有关更多信息，请参见["重新定义区域角，"第17.18.4节](pt03ch17s18s04.md)。

您计划使用结构化四边形单元进行网格划分的区域必须形状良好；否则，Abaqus/CAE可能会创建无效单元，如图17-47所示。

**图17-47** 区域必须形状良好。

![](../graphics/mgn-notwellshaped-nls.png)

如果网格包含无效单元，您可以使用几种技术来纠正网格：
- 调整网格种子的位置。
- 重新定义区域角。
- 将面分割为更小、形状更好的区域。

每种技术的效果如图17-48所示。

**图17-48** 纠正包含无效单元的网格。

![](../graphics/mgn-wellshaped-nls.png)
