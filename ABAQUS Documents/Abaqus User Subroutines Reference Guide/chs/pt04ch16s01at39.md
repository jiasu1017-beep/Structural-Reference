### 概述

顺序耦合热传递分析：
- 当结构中的应力/变形场依赖于该结构中的温度场时使用，但温度场可以在不知道应力/变形响应的情况下获得；和
- 通常首先进行非耦合热传递分析，然后进行应力/变形分析。

温度场不依赖于应力场的热应力分析是顺序多物理场工作流的常见例子，是 ["顺序耦合的预定义场，" 第 16.1.1 节"](pt04ch16s01at38.md) 中描述的更一般工作流的一种情况。在这种热应力分析中，温度在非耦合热传递分析（["非耦合热传递分析，" 第 6.5.2 节"](pt03ch06s05at18.md)）或耦合热电分析（["耦合热电分析，" 第 6.7.3 节"](pt03ch06s07at22.md)）中计算。

### 保存节点温度

节点温度通过请求输出变量 NT 作为节点输出到结果或输出数据库文件，存储为时间的函数。参见 ["输出到数据和结果文件，" 第 4.1.2 节中的"节点输出"](pt02ch04s01aus39.md#usb-out-oprintfile-nodaloutput) 和 ["输出到输出数据库，" 第 4.1.3 节中的"节点输出"](pt02ch04s01aus40.md#usb-out-odboutput-nodaloutput)。

### 将热传递结果传递到应力分析

温度作为预定义场读取到应力分析中；温度随位置变化且通常随时间依赖。它是预定义的，因为应力分析求解不会改变它。这种预定义场总是在节点处读取到 Abaqus/Standard。然后根据需要插值到单元内的计算点（参见 ["预定义场，" 第 34.6.1 节中的"网格间数据插值"](pt07ch34s06aus128.md#usb-prc-pfields-tempinterpolate)）。应力单元中的温度插值通常比位移插值低一阶，以获得热应变和机械应变的兼容变化。可以读取任意数量的预定义场，材料属性可以定义为依赖于它们。

更多信息，请参见 ["顺序耦合的预定义场，" 第 16.1.1 节中的"作为温度场传递温度"](pt04ch16s01at38.md#usb-anl-asequentiallycoupledfields-temp)。

### 预定义场

除了从热传递分析读取的温度外，还可以指定用户定义的场变量；这些值仅影响场变量依赖的材料属性（如果有的话）。参见 ["预定义场，" 第 34.6.1 节"](pt07ch34s06aus128.md)。

### 材料选项

热分析中的材料必须定义热属性，如导热系数（参见 ["热属性：概述，" 第 26.2.1 节"](pt05ch26s02abo23.md)）。任何力学属性（如弹性）在热分析中将被忽略，但必须为应力分析程序定义。有关 Abaqus/Standard 中可用材料模型的详细信息，请参见第五部分 ["材料"](pt05.md)。

如果在材料属性定义中包含热膨胀（["热膨胀，" 第 26.1.2 节"](pt05ch26s01abm52.md)），则在应力分析中会产生热应变。

### 单元

Abaqus/Standard 中任何热传递单元都可用于热分析。在应力分析中必须选择相应的连续体或结构单元。例如，如果在热传递分析中热传递壳单元类型 DS4 由节点 100、101、102 和 103 定义，则在应力分析程序中必须由这些节点定义三维壳单元类型 S4R 或 S4R5。对于连续体单元，一阶单元网格的热传递结果可以传递到二阶单元网格的应力分析（参见 ["预定义场，" 第 34.6.1 节中的"将二阶应力单元与一阶热传递单元一起使用（中节点功能）"](pt07ch34s06aus128.md#usb-prc-pfields-diffmesh-midside)）。

### 输出

必须通过请求输出变量 NT 将节点温度写入热传递分析结果或输出数据库文件（参见 ["输出到数据和结果文件，" 第 4.1.2 节"](pt02ch04s01aus39.md)）。这些温度将被读取到应力分析程序中。

热传递和应力分析章节中描述了适当的输出变量。所有输出变量都在 ["Abaqus/Standard 输出变量标识符，" 第 4.2.1 节"](pt02ch04s02abv01.md) 中概述。

### 输入文件模板

典型的顺序耦合热应力分析包括两个 Abaqus/Standard 运行：热传递分析和后续应力分析。

以下模板显示热传递分析 `heat.inp` 的输入：

```
[*HEADING](../key/key-link.md#usb-kws-mheading)
…
[*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=DC2D4
*（选择热传递单元类型）*
…
[*STEP](../key/key-link.md#usb-kws-hstep)
[*HEAT TRANSFER](../key/key-link.md#usb-kws-hheattrans)
…
*施加热载荷和边界条件*
…
** 将所有节点温度写入结果或
** 输出数据库文件 `heat.fil/heat.odb`
[*NODE FILE](../key/key-link.md#usb-kws-hnodefile), NSET=NALL
 NT
[*OUTPUT](../key/key-link.md#usb-kws-houtput), FIELD
[*NODE OUTPUT](../key/key-link.md#usb-kws-hnodeoutput), NSET=NALL
 NT
[*END STEP](../key/key-link.md#usb-kws-hendstep)
```

以下模板显示后续静态结构分析的输入：

```
[*HEADING](../key/key-link.md#usb-kws-mheading)
…
[*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=CPE4R
*（选择与所用热传递单元类型兼容的连续体单元类型）*
…
[*STEP](../key/key-link.md#usb-kws-hstep)
[*STATIC](../key/key-link.md#usb-kws-hstatic)
…
*施加结构载荷和边界条件*
…
[*TEMPERATURE](../key/key-link.md#usb-kws-htemperature), FILE=heat
*从结果或输出数据库文件读取所有节点温度，* `heat.fil/heat.odb`
…
[*END STEP](../key/key-link.md#usb-kws-hendstep)
```
