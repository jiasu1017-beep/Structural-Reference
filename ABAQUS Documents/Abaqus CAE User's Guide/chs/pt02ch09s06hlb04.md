# 9.7.4 创建网络ODB连接器





您可以使用网络ODB连接器来访问远程计算机上的输出数据库。例如，您可以将分析提交到高性能UNIX计算服务器，并在本地Windows工作站上查看结果。您可以从任何平台（Windows、UNIX或Linux）创建网络ODB连接器。但是，网络ODB服务器的服务器必须位于UNIX或Linux平台上。Abaqus/CAE通过生成在服务器和客户端之间来回传递的密钥来维护与网络ODB连接器的安全连接。有关更多信息，请参阅["访问网络ODB连接器的安全性如何？，"第9.3.2节"](pt02ch09s03s02.md)。

从主菜单栏选择****文件（File）**![](../graphics/images/arrow.gif)**网络ODB连接器（Network ODB Connector）**![](../graphics/images/arrow.gif)**创建（Create）****来创建连接器。创建网络ODB连接器后，必须通过从主菜单栏选择****文件（File）**![](../graphics/images/arrow.gif)**网络ODB连接器（Network ODB connector）**![](../graphics/images/arrow.gif)**启动（Start）**![](../graphics/images/arrow.gif)***连接器名称*****来启动它。远程系统上必须安装Abaqus才能使Abaqus/CAE建立网络连接。有关更多信息，请参阅["管理网络ODB连接器，"第9.7.6节"](pt02ch09s06hlb06.md)。

在大多数情况下，您将使用Abaqus/CAE在远程系统上启动网络ODB服务器并分配端口号。Abaqus/CAE仅在远程主机上的用户名与本地系统上的用户名相同时才能启动服务器。如果您在建立通信时遇到问题或用户名不同，您可以通过在远程系统上运行**abaqus networkDBConnector**执行过程来启动服务器。有关更多信息，请参阅["网络输出数据库文件连接器，"Abaqus Analysis User's Guide第3.2.24节"](../usb/usb-link.md#usb-int-dremoteodbproc)。

**要与远程系统建立连接：**

1. 从主菜单栏，选择****文件（File）**![](../graphics/images/arrow.gif)**网络ODB连接器（Network ODB Connector）**![](../graphics/images/arrow.gif)**创建（Create）****。
2. 从出现的**网络ODB连接器（Network ODB Connector）**编辑器中，输入远程连接器的名称。随后打开输出数据库时，**打开数据库（Open Database）**对话框会显示远程连接器的名称。Abaqus/CAE也会在**网络ODB连接器管理器（Network ODB Connector Manager）**中显示此名称。
3. 从**网络ODB连接器**编辑器的**基本（Basic）**选项卡页面中，输入以下内容：**主机名（Host name）** 远程系统的URL或IP地址形式的名称；例如，`computeserver.mycompany.com`。**目录（Directory）** 要在远程系统上打开的目录。您输入的目录必须包含您要访问的远程输出数据库，或者必须包含包含远程输出数据库的子目录。
4. 在大多数情况下，您可以单击**确定（OK）**关闭对话框并使用默认配置选项建立远程连接。但是，如果您在与远程系统建立通信时遇到困难，或者您的站点需要特定配置，则可能需要自定义网络ODB连接器。有关更多信息，请参阅["自定义网络ODB连接器，"第9.7.5节"](pt02ch09s06hlb05.md)。
