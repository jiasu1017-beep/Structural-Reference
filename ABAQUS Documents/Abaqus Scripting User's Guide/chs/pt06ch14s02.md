### 14.2.1 主窗口示例

要为特定应用程序创建主窗口，首先需要从`AFXMainWindow`类派生一个新类。在主窗口的构造函数中，你注册应用程序使用的模块和工具集。

以下脚本构建了Abaqus/CAE主窗口。脚本在以下各节中有详细描述。你如何在[第8章，"创建GUI模块"](pt05ch08.md)和[第9章，"创建GUI工具集"](pt05ch09)中构建模块和工具集的详细信息。

```
from abaqusGui import *  
class CaeMainWindow(AFXMainWindow): 
     def __init__(self, app, windowTitle=''): 
         # Construct the GUI infrastructure.       
         #
         AFXMainWindow.__init__(self, app, windowTitle)   

         # Register the "persistent" toolsets.  
         #  
         self.registerToolset(FileToolsetGui(), 
             GUI_IN_MENUBAR|GUI_IN_TOOLBAR)  
         self.registerToolset(ModelToolsetGui(), 
             GUI_IN_MENUBAR)  
         self.registerToolset(CanvasToolsetGui(), 
             GUI_IN_MENUBAR)  
         self.registerToolset(ViewManipToolsetGui(), 
             GUI_IN_MENUBAR|GUI_IN_TOOLBAR) 
         self.registerToolset(TreeToolsetGui(), 
             GUI_IN_MENUBAR)  
         self.registerToolset(AnnotationToolsetGui(), 
             GUI_IN_MENUBAR|GUI_IN_TOOLBAR) 
         self.registerToolset(CustomizeToolsetGui(), 
             GUI_IN_TOOL_PANE)  
         self.registerToolset(SelectionToolsetGui(), 
             GUI_IN_TOOLBAR)  
         registerPluginToolset()  
         self.registerHelpToolset(HelpToolsetGui(), 
             GUI_IN_MENUBAR|GUI_IN_TOOLBAR)  

         # Register modules.  
         #  
         self.registerModule('Part',          'Part')
         self.registerModule('Property',  'Property')
         self.registerModule('Assembly',      'Assembly')
         self.registerModule('Step',          'Step')
         self.registerModule('Interaction', 'Interaction')
         self.registerModule('Load',          'Load')    
         self.registerModule('Mesh',          'Mesh')
         self.registerModule('Job', 'Job')
         self.registerModule('Visualization', 'Visualization')
         self.registerModule('Sketch',        'Sketch')  
```

### 14.2.2 导入模块

`abaqusGui`模块提供对整个Abaqus GUI Toolkit的访问，以及必须与主窗口注册的那些模块（如`FileToolsetGui`）的访问。

### 14.2.3 构建基类

`CaeMainWindow`构造函数中的第一条语句通过调用基类构造函数来初始化该类。通常，你应该始终调用你派生的类的基类构造函数，除非你知道你将覆盖该类的功能。

### 14.2.4 注册持久工具集

与模块注册的工具集不同，与主窗口注册的工具集在应用程序首次启动时就可用。此外，与主窗口注册的工具集在用户切换模块的整个会话期间保持可用。

要注册工具集，调用`registerToolset`方法并传入工具集类的实例。你可以使用`registerHelpToolset`方法向应用程序注册help工具集。以这种方式注册的工具集始终出现在菜单栏中所有其他菜单的右侧。有关更多信息，请参见["注册工具集，"第8.2.6节"](pt05ch08s02.md#cus-app-module-toolsets)。

**注意：**每个应用程序必须注册`viewManipToolsetGui`。

### 14.2.5 注册模块

注册模块会将模块名称放入上下文栏中的**模块**组合框中。注册模块的顺序就是模块将出现在上下文栏中**模块**组合框中的顺序。

要注册模块，调用`registerModule`方法。`registerModule`方法接受以下参数：

**displayedName**

应用程序将在上下文栏中的**模块**组合框中显示的字符串。

**moduleImportName**

指定要导入的模块名称的字符串。你的责任是确保此名称与GUI模块文件名（不带.py扩展名）相同。有关更多信息，请参见["实例化GUI模块，"第8.2.8节"](pt05ch08s02.md#cus-app-module-instantiating)。

**kernelInitializationCommand**

指定在加载模块时发送到kernel的Python命令名称的字符串。
