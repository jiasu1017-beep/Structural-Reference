# 9.10.3 解决输入文件中的冲突





如果您使用**关键词编辑器**编辑了一个关键词，然后使用Abaqus/CAE对引用相同关键词的模型进行了更改，Abaqus/CAE无法确定应在输入文件中合并哪个版本的关键词，并会将表示问题的文本写入输入文件。因此，当您提交模型进行分析时会生成错误。如果您使用**关键词编辑器**显示输入文件，任何冲突的关键词或数据行都会用*冲突（Conflicts）*语句指示。此外，*Conflicts语句指示文本是由Abaqus/CAE还是由**关键词编辑器**生成的。您应该使用**关键词编辑器**删除任何不需要的关键词或数据行。您还应该删除所有*Conflicts语句。

某些输入文件冲突无法使用**关键词编辑器**解决；例如，由对隐藏数据行紧接后的输入文件行的修改引起的冲突（请参阅["将不支持的关键词添加到您的Abaqus/CAE模型中，"第9.10.1节"](pt02ch09s09hlb01.md))。在这些情况下，您可以使用"作业（Job）"模块将完整的输入文件写入文本文件（请参阅["仅写入输入文件，"第19.7.2节"](pt03ch19s07hlb02.md)获取详细信息）。您可以使用文本编辑器修改此输入文件，然后手动提交修改后的输入文件进行分析（请参阅["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行，"Abaqus Analysis User's Guide第3.2.2节"](../usb/usb-link.md#usb-int-danalysisproc))。要在Abaqus/CAE中执行分析，请在"作业（Job）"模块中使用修改后的输入文件作为作业源创建新作业（请参阅["创建新的分析作业，"第19.7.1节"](pt03ch19s07hlb01.md))。
