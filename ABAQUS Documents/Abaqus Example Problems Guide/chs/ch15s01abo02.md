# 15.1.1 Abaqus结果文件用户后处理：概述







本章说明如何编写和利用后处理程序来操作存储在Abaqus结果文件中的数据。结果文件由文件扩展名`.fil`标识，包含基于用户指定输出请求的结果；有关写入结果文件的数据类型更多信息，请参阅["输出到数据和结果文件," Abaqus分析用户指南第4.1.2节](../usb/usb-link.md#usb-out-oprintfile)。标准文件格式是二进制的，但可以为每次运行应用户请求更改为ASCII格式。或者，可以在站点安装期间设置为默认ASCII格式。Abaqus使用FORTRAN单元`8`与结果文件通信。

本章各节中给出了执行常用任务的后处理程序示例。这些包括合并多个结果文件并将结果文件从二进制格式转换为ASCII，反之亦然；计算应力和应变的主值和方向；以及通过以临界屈曲模态形式结合用户指定几何缺陷来计算崩塌分析的扰动网格。

每个后处理程序必须在使用Abaqus执行程序运行（见["制作用户定义的可执行文件和子程序," Abaqus分析用户指南第3.2.17节](../usb/usb-link.md#usb-int-dmakeproc)）时使用**make`参数链接。为正确链接，后处理程序不能包含FORTRAN PROGRAM语句。相反，程序必须以名为ABQMAIN的FORTRAN SUBROUTINE开始。

下面描述了通用编程概念、Abaqus FORTRAN接口和数据处理概念。更多信息请参阅[Abaqus分析用户指南第5章，"文件输出格式,"](../usb/usb-link.md#usbfchapter)。各节中的程序列表提供了关于程序流、如何与使用不同操作系统和FORTRAN编译器的各种计算平台接口、以及如何与Abaqus子程序接口处理数据和记录等方面的详细信息。

有关访问存储在Abaqus输出数据库中的数据的信息，请参阅[Abaqus脚本用户指南第9章，"使用Abaqus脚本接口访问输出数据库,"](../cmd/cmd-link.md#cmd-odb-api-pyc)或[Abaqus脚本用户指南第10章，"使用C++访问输出数据库,"](../cmd/cmd-link.md#cmd-odb-api-cpp)。

### 初始化

后处理程序中使用的变量详细信息在["访问结果文件信息," Abaqus分析用户指南第5.1.3节](../usb/usb-link.md#usb-out-faccessinfo)中讨论。Abaqus使用名为`ARRAY`的512字缓冲区来读取和写入结果文件上的数据。其维度为`ARRAY(513)`。对于64位计算机，整数等效为`JRRAY(513)`，对于32位计算机为`JRRAY(2,513)`。使用`EQUIVALENCE`语句使`ARRAY`和`JRRAY`等价，以简化存储在缓冲区中的数据记录中实数和整数操作的处理。

关于所读取结果文件的FORTRAN单元号和格式的信息在`LRUNIT(2,NRU)`中定义，其中`NRU`是要处理的文件数。第n个文件的FORTRAN单元号存储在`LRUNIT(1,n)`中。关于文件格式的信息存储在`LRUNIT(2,n)`中，对于ASCII格式初始化为1，对于二进制格式初始化为2。如果要由后处理程序创建新结果文件，则输出文件的文件格式通过变量`LOUTF`类似地定义，对于ASCII格式也初始化为1，对于二进制格式初始化为2。输入和输出结果文件的根文件名通过字符变量`FNAME`定义。根文件名的大小写将与`FNAME`定义的大小写相同；Abaqus定义文件扩展名为小写字母。关于文件扩展名命名约定的讨论请参阅["访问结果文件信息," 第5.1.3节](../usb/usb-link.md#usb-out-faccessinfo)。

最终初始化阶段通过调用Abaqus子程序`INITPF`和`DBRNU`在内部完成。FORTRAN接口为

```
CALL INITPF(FNAME, NRU, LRUNIT, LOUTF)
CALL DBRNU (JUNIT)
```

其中调用`INITPF`的参数如上所述，`JUNIT`是连接文件的FORTRAN单元号。

这些整数变量必须在调用子程序之前定义。

### 数据处理

数据处理需要了解每个数据记录。这些记录的详细信息在["结果文件输出格式," Abaqus分析用户指南第5.1.2节](../usb/usb-link.md#usb-out-fformat)中找到。

结果文件中的数据组织使用顺序格式。因此，每个记录必须通过使用接口

```
CALL DBFILE(0, ARRAY, JRCD)
```

调用`DBFILE`以顺序方式检索。此调用可以放在`DO`循环内，循环计数应超过文件中存储的记录数。或者，只要`JRCD`等于0，就可以调用`DBFILE`。第一个参数0表示要读取一个记录。读取的每个记录存储在缓冲区`ARRAY`中并返回给调用程序进行处理。最后一个参数`JRCD`是返回码，除非处理文件结束条件或不完整记录，否则设置为0，在这种情况下`JRCD`设置为1。

如果需要提取或修改某些记录并将其保存到与Abaqus生成的结果文件具有相同数据组织的新结果文件中，则应使用接口

```
CALL DBFILW(1, ARRAY, JRCD)
```

调用子程序`DBFILW`。新结果文件将写入文件扩展名`.fin`。请参阅["访问结果文件的实用程序," Abaqus分析用户指南第5.1.4节](../usb/usb-link.md#usb-out-uresults)。
