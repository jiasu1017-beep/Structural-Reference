# 9.10.4 在输入文件中不使用零件和装配





当您提交分析作业时，Abaqus/CAE使用您的模型定义生成Abaqus/Standard、Abaqus/Explicit或Abaqus/CFD输入文件。Abaqus/CAE模型包含零件和装配；默认情况下，Abaqus/CAE生成的输入文件包含零件和装配。某些Abaqus功能在包含零件和装配的模型中不受支持。Abaqus/CAE在写入不带零件和装配的输入文件时会尝试保留模型的节点和单元标签。如果任何零件或零件实例标签之间没有冲突，Abaqus/CAE在将节点和单元写入输入文件时会保留模型中的标签。相反，如果任何零件或零件实例标签之间出现冲突（例如，两个零件实例具有相同的节点和单元标签），Abaqus/CAE会在写入带有重新编号的节点和单元标签的输入文件之前显示警告。

如果您希望Abaqus/CAE在输入文件中不使用零件和装配，您可以使用以下方法之一更改Abaqus/CAE生成的输入文件的格式：

**当您启动Abaqus/CAE会话时**

要更改Abaqus/CAE生成的输入文件的格式，您可以按如下方式修改Abaqus环境文件（`abaqus_v6.env`）中的**cae_no_parts_input_file**参数：

```
cae_no_parts_input_file=ON
```

如果您使用此方法，则在Abaqus/CAE会话期间无法更改输入文件格式。有关定义环境文件参数的其他信息，请参阅[Abaqus安装和许可指南](../sgb/sgb-link.md#sgb)。

**在Abaqus/CAE会话期间**

您可以使用两种方法更改Abaqus/CAE生成的输入文件的格式。
- 选择您要更改的模型的****模型（Model）**![](../graphics/images/arrow.gif)**模型属性（Model Attributes）**![](../graphics/images/arrow.gif)***模型名称*****，然后切换**不在输入文件中使用零件和装配（Do not use parts and assemblies in input files）**。
- 在Abaqus/CAE主窗口底部的命令行界面中输入以下Abaqus Scripting Interface命令：``` mdb.models[*modelName*].setValues(noPartsInputFile=ON) ```**注意：**默认情况下命令行界面是隐藏的，但它使用与主窗口底部消息区域相同的空间。要访问命令行界面，请单击主窗口左下角的![](../graphics/afxI_commandLine.png)。

如果您使用任一方法，更改后的输入文件格式是模型定义的一部分。当您退出会话并稍后返回模型数据库时，更改后的输入文件格式会被保留。

如果您发现自己反复更改Abaqus/CAE生成的输入文件的格式，您可以创建并运行包含先前Abaqus Scripting Interface命令的宏。有关宏的更多信息，请参阅["管理宏，"第9.11节"](pt02ch09hla02.md)。

**警告：**当Abaqus/CAE写入不带零件和装配的输入文件时，它会尝试保留在Abaqus/CAE中生成的节点和单元标签。但是，如果您使用**关键词编辑器**编辑使用零件和装配输入文件格式的模型中的关键词，然后更改Abaqus/CAE生成的输入文件的格式，则在写入输入文件和重新生成关键词时可能会发生冲突。有关更多信息，请参阅["解决输入文件中的冲突，"第9.10.3节"](pt02ch09s09hlb03.md)。
