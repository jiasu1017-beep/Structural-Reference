### 概述

两个结构求解器之间的协同仿真（求解器交换位移和/或旋转以及共轭场的力和/或力矩）代表了非常强的物理耦合，需要在协同仿真接口处进行特殊处理。Abaqus 通过提供专门的接口处理支持 Abaqus/Standard 和 Abaqus/Explicit 之间的协同仿真。虽然您可以在两个 Abaqus/Standard 分析或两个 Abaqus/Explicit 分析之间执行结构对结构协同仿真，但由于在接口处缺乏适当的处理，不建议这样做。

本节讨论特定于 Abaqus/Standard 到 Abaqus/Explicit 协同仿真的分析设置、执行和限制详细信息。

有关 Abaqus/Standard 到 Abaqus/Explicit 协同仿真的示例，请参阅 ["滑板车与凸起物的动态冲击," Abaqus 例题指南第 2.4.1 节](../exa/exa-link.md#exa-dyn-scooterbump)。

### 识别协同仿真接口区域

Abaqus/Standard 和 Abaqus/Explicit 模型之间的相互作用通过公共接口区域发生。

当耦合 Abaqus/Standard 到 Abaqus/Explicit 时，您可以使用节点集或表面指定接口区域。但是，您必须在 Abaqus/Standard 和 Abaqus/Explicit 中的区域定义中保持一致；如果在一个分析中用节点集或基于节点的表面定义协同仿真区域，则必须在另一个分析中使用相同类型的协同仿真区域定义。对于基于节点的表面，节点必须重合，因为没有提供拓扑信息在 Abaqus/Standard 和 Abaqus/Explicit 模型之间保守地映射场。同样，如果在一个分析中用基于元素的表面定义协同仿真区域，则必须在另一个分析中用基于元素的表面定义协同仿真区域。

您可能在 Abaqus/Standard 和 Abaqus/Explicit 模型定义共享的区域中具有不同的网格。然而，在某些情况下，通过确保在接口处有匹配的节点，您可以提高解决方案的稳定性和准确性（参见 ["不同网格相关的限制"](pt04ch17s03aus99.md#usb-anl-acosimabqtoabq-dissimilar)）。在这些情况下，您可以使用 ["确保接口区域处的匹配节点," Abaqus/CAE 用户指南第 26.4 节"](pt04ch17s02aus98.md) 中描述的建模实践来确保这些匹配节点。

| **输入文件用法：** | 使用以下选项在 Abaqus 模型中将基于元素的或基于节点的表面定义为协同仿真区域： |

### 识别跨协同仿真接口交换的场

对于 Abaqus/Standard 到 Abaqus/Explicit 协同仿真，您不需要定义交换的场；它们根据使用的程序和协同仿真参数自动确定。

### 定义会合方案

对于结构对结构协同仿真，您必须为每个分析指定协同仿真控制并创建配置文件；参数的一致性在执行期间确认。SIMULIA 协同仿真引擎配置文件用于定义时间积分过程和两个 Abaqus 分析之间交换的频率。Abaqus/CAE 自动创建和使用此配置文件。如果您不使用 Abaqus/CAE 执行协同仿真，则必须手动创建配置文件。

以下协同仿真方案有预定义模板可用：
- 您可以强制 Abaqus/Standard 使用与 Abaqus/Explicit 相同的增量大小，或者
- 您可以允许 Abaqus/Standard 中的增量大小与 Abaqus/Explicit 中的不同（subcycling）。

当您创建配置文件时，您会引用这些预定义模板。

| **输入文件用法：** | 使用以下两个选项指定协同仿真控制： |
| --- | --- |
| | ``` [*CO-SIMULATION](../key/key-link.md#usb-kws-hcosimulation), PROGRAM=ABAQUS, CONTROLS=*name* [*CO-SIMULATION CONTROLS](../key/key-link.md#usb-kws-hcosimulationcontrols), NAME=*name* ``` |

| **Abaqus/CAE 用法：** | 相互作用模块：**Create Interaction**: **Standard-Explicit co-simulation** |
| --- | --- |

#### 时间积分方案

您可以强制 Abaqus/Standard 使用与 Abaqus/Explicit 相同的增量大小，或者您可以允许 Abaqus/Standard 中的增量大小与 Abaqus/Explicit 中的不同（subcycling）。您为耦合选择的时间积分方案会影响解决方案的计算成本和准确性，但不会影响解决方案稳定性。

subcycling 方案通常是最具成本效益的，因为 Abaqus/Standard 时间增量（不受任何强制协同仿真时间增量约束）通常比 Abaqus/Explicit 时间增量长得多。然而，当模型中大部分节点处于协同仿真接口时，subcycling 方案可能不太具成本效益。这是因为 Abaqus/Standard 在 Abaqus/Explicit 分析的每个增量执行一组接口稳定操作（"free solve"）。这些 free-solve 操作需要求解与接口节点数量成比例的稠密方程组。在接口节点数量很多的情况下，此接口求解的计算成本可能超过因 subcycling 看到的任何成本节省。因此，对于在协同仿真接口处有 significant 节点 share 的模型，使用 subcycling 方案性能可能更差。

##### 强制 Abaqus/Standard 使用与 Abaqus/Explicit 相同的增量大小

您可以强制 Abaqus/Standard 匹配 Abaqus/Explicit 的增量大小，并且场将在每个共享增量处交换。

| **输入文件用法：** | 在 Abaqus/Standard 分析和 Abaqus/Explicit 分析中使用以下选项： |
| --- | --- |
| | ``` [*CO-SIMULATION CONTROLS](../key/key-link.md#usb-kws-hcosimulationcontrols), TIME INCREMENTATION=LOCKSTEP ``` |

| **Abaqus/CAE 用法：** | 在 Abaqus/Standard 分析和 Abaqus/Explicit 分析中使用以下输入： |
| --- | --- |
| | 相互作用模块：**Create Interaction**: **Standard-Explicit co-simulation**: **Incrementation control**: **Lock time steps** |

##### 允许 Abaqus/Standard 中的增量大小与 Abaqus/Explicit 中的不同

您可以允许 Abaqus/Standard 增量大小与 Abaqus/Explicit 中的不同（subcycling）。在这种情况下，场将根据需要交换。

| **输入文件用法：** | 在 Abaqus/Standard 分析和 Abaqus/Explicit 分析中使用以下选项： |
| --- | --- |
| | ``` [*CO-SIMULATION CONTROLS](../key/key-link.md#usb-kws-hcosimulationcontrols), TIME INCREMENTATION=SUBCYCLE ``` |

| **Abaqus/CAE 用法：** | 在 Abaqus/Standard 分析和 Abaqus/Explicit 分析中使用以下输入： |
| --- | --- |
| | 相互作用模块：**Create Interaction**: **Standard-Explicit co-simulation**: **Incrementation control**: **Allow subcycling** |

#### 控制接口矩阵分解频率

对于 subcycling 时间积分方案，默认情况下在 Abaqus/Standard 中为每个 Abaqus/Explicit 增量执行接口求解。 这个求解成本很高，原因有两个。首先，用于接口求解的接口矩阵是稠密的，其大小随接口节点数量缩放。其次，接口矩阵在每个 Abaqus/Explicit 增量都会发生变化，需要在 Abaqus/Standard 中为每个 Abaqus/Explicit 增量进行分解。您可以通过近似接口矩阵并通常为 Abaqus/Standard 增量持续时间分解一次（而不是为每个 Abaqus/Explicit 增量分解）来降低此成本的影响。但是，如果 Abaqus/Explicit 稳定时间增量发生显著变化，则出于稳定性原因会重新分解接口矩阵。

##### 允许 Abaqus/Standard 每个 Abaqus/Explicit 增量分解一次接口矩阵

每个 Abaqus/Explicit 增量分解一次接口矩阵是默认方法。

| **输入文件用法：** | 在 Abaqus/Standard 分析中使用以下选项： |
| --- | --- |
| | ``` [*CO-SIMULATION CONTROLS](../key/key-link.md#usb-kws-hcosimulationcontrols), FACTORIZATION FREQUENCY=EXPLICIT INCREMENT ``` |

| **Abaqus/CAE 用法：** | 在 Abaqus/CAE 中默认使用每个 Abaqus/Explicit 增量分解一次接口矩阵。 |
| --- | --- |

##### 强制 Abaqus/Standard 每个 Abaqus/Standard 增量分解一次接口矩阵

当接口节点数量很大时，使用此方法可以显著降低接口分解的成本。仅在每个 Abaqus/Standard 增量执行一次接口矩阵分解；使用此分解接口矩阵在每个 Abaqus/Explicit 增量执行接口求解。由于此方法近似接口矩阵，它可能会略微增加协同仿真接口处位移解决方案的漂移。使用此方法，性能提升取决于接口节点数量、subcycling 比率（Abaqus/Standard 与 Abaqus/Explicit 增量的比率）以及模型的大小。对于接口节点大于 100 且 subcycling 比率大于 50 的模型，此方法通常将分析时间减少 1.2 到 3.0 倍。性能提升随 subcycling 比率增大而增加，随模型增大而减少。

| **输入文件用法：** | 在 Abaqus/Standard 分析中使用以下选项： |
| --- | --- |
| | ``` [*CO-SIMULATION CONTROLS](../key/key-link.md#usb-kws-hcosimulationcontrols), FACTORIZATION FREQUENCY=STANDARD INCREMENT ``` |

| **Abaqus/CAE 用法：** | 在 Abaqus/CAE 中不支持每个 Abaqus/Standard 增量分解一次接口矩阵。 |
| --- | --- |

#### 耦合步骤大小

耦合步骤大小是 Abaqus/Standard 和 Abaqus/Explicit 之间两个连续协同仿真数据交换之间的时间段，始终等于当前 Abaqus/Explicit 增量大小。

当使用 subcycling 方法时，此数据交换不表示对 Abaqus/Standard 增量的约束；Abaqus/Standard 分析使用其正常时间增量逻辑推进时间。

### 限制

除 ["准备协同仿真的 Abaqus 分析," 第 17.2.1 节"](pt04ch17s02aus98.md) 中讨论的限制外，以下限制适用于 Abaqus/Standard 到 Abaqus/Explicit 协同仿真。

#### 一般限制

- 当您允许 Abaqus/Standard 增量大小与 Abaqus/Explicit 中的不同时（即当您指定 subcycling 作为协同仿真时间增量控制时），协同仿真接口处的位移兼容性不维持。在这种情况下，速度兼容性得以维持，但随着模拟随时间推进，您可能会看到 Abaqus/Standard 和 Abaqus/Explicit 之间的小量位移不匹配。如果在协同仿真接口处发生严重非线性（如塑性变形），这种"漂移"会更加明显。您可以通过调整 Abaqus/Standard 解决方案参数来控制此漂移，从而减少 Abaqus/Standard 增量大小（例如，通过限制最大时间增量大小或为隐式动态分析指定更小的半增量残差容差）。
- 不允许在协同仿真区域节点上进行节点转换。
- ALE 技术不能用于连接到协同仿真区域节点的单元。
- 可以使用完全耦合温度-位移单元，但不交换温度量。
- Abaqus/Standard 静态应力分析不能与 Abaqus/Standard 到 Abaqus/Explicit 协同仿真中的 lockstep 时间增量方案一起使用。
- 仅允许点和表面区域；不支持耦合体积区域。
- 只能将单个区域定义为接口区域；不支持多个接口区域。

#### 不同网格相关的限制

当您的 Abaqus/Standard 和 Abaqus/Explicit 协同仿真区域网格不同时，以下限制适用：
- 当协同仿真区域网格在旋转自由度存在或不存在方面不均匀时，解决方案准确性可能受到影响；例如，如果连续体单元网格在协同仿真区域接口处局部加固了梁或壳单元。
- 在协同仿真接口附近应力状态显著（接近或超过 1%）相对于材料刚度的情况下，如果 Abaqus/Explicit 和 Abaqus/Standard 模型之间协同仿真区域相邻的网格密度差异很大，您可能会看到显著的不规则网格扭曲。例如，这种效应在超弹性材料的大变形中很常见。通过在 Abaqus/Standard 协同仿真区域选择相似或更细的网格（使用 subcycling 时间积分方案时）或在 Abaqus/Explicit 协同仿真区域选择相似或更细的网格（使用 lockstep 时间积分方案时），您可以最小化此效应。

#### Abaqus/Standard 分析限制

在 Abaqus/Explicit 中没有等效自由度对应的 Abaqus/Standard 单元不能连接到协同仿真区域节点。这些单元包括
- 具有扭转自由度的轴对称单元（CGAX 单元系列）
- 具有不对称变形的轴对称实体单元（CAXA 单元系列）
- 广义平面应变单元（CPEG 单元系列）
- 耦合孔隙压力-位移单元
- 热传递和热电单元
- 声学单元
- 压电单元

以下特定限制也适用：- 协同仿真区域节点不能是绑定约束、MPC 约束或运动耦合约束中的从节点。

#### Abaqus/Explicit 分析限制

当以下模型特征在协同仿真区域定义或附近时，协同仿真解决方案的稳定性和准确性可能受到不利影响：
- 连接到协同仿真区域节点的连接器单元。
- 参与绑定约束、MPC 约束或运动耦合约束的协同仿真区域节点。

使用这些特征时，您应该比较 Abaqus/Standard 和 Abaqus/Explicit 解决方案（例如，协同仿真接口处位移历史的兼容性）作为解决方案准确性的指标。
