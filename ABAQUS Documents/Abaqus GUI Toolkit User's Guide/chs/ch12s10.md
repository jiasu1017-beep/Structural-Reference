# 12.10 Abaqus/Explicit 实例：电路板跌落测试





在此实例中，您将研究安装在防撞可压碎泡沫包装中的电路板以一定角度跌落到刚性表面上的行为。您的目标是评估泡沫包装是否足以防止电路板从 1 米高度跌落时损坏。您将使用 Abaqus/Explicit 中的通用接触功能来模拟不同组件之间的相互作用。图 12-49 显示了电路板和泡沫包装的尺寸（以毫米为单位）和材料特性。

**图 12-49** 尺寸（毫米）和材料特性。

![](../graphics/gsx-dimen-milmat-nls.png)

### 12.10.1 坐标系

虽然电路板将按一定角度跌落，但最简单的方法是使用 [*SYSTEM](../key/key-link.md#usb-kws-msystem) 选项定义与局部直角坐标系对齐的网格（如图 12-49 所示）。[*SYSTEM](../key/key-link.md#usb-kws-msystem) 选项将节点坐标从局部坐标系转换到全局坐标系。此选项允许您在局部坐标系的 *x-z* 平面中定义电路板，该平面相对于全局坐标系旋转了所需的角度。

[*SYSTEM](../key/key-link.md#usb-kws-msystem) 选项通过指定三个点来定义新的坐标系：局部原点、局部 *x* 轴上的点和局部 *x-y* 平面中的点。在为电路板定义节点之前，使用以下选项来倾斜网格，使其以角落地：

```
*SYSTEM
0., 0., 0., .5, .707, .25
-.5, .707, -.5
```
 所有后续节点定义将在此局部坐标系中。要将坐标系重置为默认值，请使用另一个没有数据行的 [*SYSTEM](../key/key-link.md#usb-kws-msystem) 选项。

### 12.10.2 网格设计

该问题的整体网格如图 12-50 所示。

**图 12-50** 电路板和泡沫包装的网格。

![](../graphics/gsx-circuit-board-nls.png)

定义电路板，使壳法线指向所示的方向。定义泡沫包装底部角落为模型原点将确保电路板和包装的正确位置。由于电路板将跌落其上的地面是有效刚性的，因此为此部分使用单个 R3D4 单元。包装是应使用 C3D8R 单元建模的三维实体结构。电路板本身可视为薄平板，上面连接有各种芯片。因此，使用 S4R 单元对电路板建模，并使用 MASS 单元对芯片建模。

由于您将使用壳单元进行电路板，Abaqus/Explicit 默认会在检查接触时使用原始壳单元厚度。电路板及其在泡沫包装中的槽厚度相同（2 mm），以便两个体之间配合紧密。在此示例中，电路板是 10×10 S4R 单元的网格，泡沫包装是 6×7×15 单元的网格（如图 12-51 所示）。

**图 12-51** 包装网格细节。

![](../graphics/gsx-pack-mesh-detail-nls.png)

MASS 单元按图 12-52 所示定位。包装网格在撞击角落附近太粗，无法提供高度准确的结果。但是，网格足以进行低成本初步研究。

**图 12-52** 电路板上质量单元的位置。括号中的数字是相对于电路板左下角局部的 (*x, y*) 坐标（毫米）。

**图 12-53** 所需的节点和单元集。

将电路板单元包含在名为 `BOARD` 的单元集中，并将相应的电路板节点包含在名为 `BOARD` 的节点集中。同样，对于泡沫包装，将单元包含在名为 `PACK` 的单元集中，并将节点包含在名为 `PACK` 的节点集中。单元集将用于引用材料属性，节点集将用于施加初始条件。创建名为 `FLOOR` 的单元集，其中包含地板的刚性单元，并创建名为 `REF` 的节点集，其中包含用于建模地板的刚性表面的参考节点。将模拟芯片的质量单元包含在名为 `CHIPS` 的单元集中。

### 12.10.4 模拟自由落体

有两种方法可以模拟电路板从 1 米高度跌落。您可以对电路板和泡沫进行建模，使其位于刚性表面上方的 1 米高度处，并允许 Abaqus/Explicit 计算重力作用下的运动；然而，这种方法显然不切实际，因为完成模拟的"自由落体"部分需要大量增量。最有效的方法是将电路板和包装建模在非常接近表面的初始位置，并使用初始速度模拟 1 米跌落（4.43 m/s）。

### 12.10.5 检查输入文件——模型数据

我们现在回顾此模拟所需的模型数据，包括模型描述、节点和单元定义、单元和材料属性、边界和初始条件以及表面定义。您可以通过获取和打开输入文件 `circuit.inp` 来查看这些数据。

**模型描述**

此示例中的 [*HEADING](../key/key-link.md#usb-kws-mheading) 选项为您的模型提供了合适的标题。此示例使用 SI 单位。

```
*HEADING
Circuit board drop test
1.0 meter drop
SI units (kg, m, s, N)
```

**节点坐标和单元连接**

使用预处理器在局部坐标系中生成网格。在节点定义之前加上 [*SYSTEM](../key/key-link.md#usb-kws-msystem) 选项，以将节点转换到倾斜坐标系中，如前所述。在 `circuit.inp` 中，泡沫包装和电路板的节点定义如下：

```
*SYSTEM
0., 0., 0., .5, .707, .25
-.5, .707, -.5
*NODE
1,     0.005, -0.010, 0.012
11,    0.005, -0.010, 0.162
.
.
** Reset coordinate system
**
*SYSTEM
```

当您在旋转的局部坐标系中完成节点定义后，再次使用没有数据行的 [*SYSTEM](../key/key-link.md#usb-kws-msystem) 选项，以便其他节点编号以全局坐标系给出。定义刚性表面的节点，使其足够大，以防止可变形体从其任何边缘掉落。使用从泡沫包装底部角落开始的 0.1 mm 垂直间隙，以确保接触表面没有初始过盈闭合。

**单元属性**

为每个单元集提供适当的截面属性。在每个截面选项上包含适当的 MATERIAL 参数，以便每个单元集链接到材料定义。我们将泡沫包装材料命名为 `FOAM`，并将在下一节中定义它。

```
*SOLID SECTION, ELSET=PACK, MATERIAL=FOAM, CONTROLS=HGLASS
*SECTION CONTROLS, NAME=HGLASS, HOURGLASS=ENHANCED

```

对于电路板，在纵向和横向方向输出应力结果最有意义，这些方向与板的边缘对齐。因此，我们需要为电路板网格指定局部材料方向。我们可以使用之前使用 [*SYSTEM](../key/key-link.md#usb-kws-msystem) 选项定义的相同局部坐标系。可以使用带有 DEFINITION=COORDINATES 参数的 [*ORIENTATION](../key/key-link.md#usb-kws-morientation) 选项来实现所需的材料方向。在第一行数据上指定两个点 *a* 和 *b* 的 *x*、*y* 和 *z* 坐标，以定义局部坐标系。在第二行数据上指定绕局部 2-（或 *y*-）轴的额外旋转 90°。然后在 [*SHELL SECTION](../key/key-link.md#usb-kws-mshellsection) 选项上引用 ORIENTATION 的名称。

```
*SHELL SECTION, ELSET=BOARD, MATERIAL=PCB, ORIENTATION=OR1
0.002,
*ORIENTATION, NAME=OR1, SYSTEM=RECTANGULAR,
DEFINITION=COORDINATES
0.5, 0.707, 0.25, -0.5, 0.707, -0.5
2, 90.0

```

使用 [*MASS](../key/key-link.md#usb-kws-mmass) 选项将电路板上每个芯片的质量定义为 0.005 kg。

```
*MASS, ELSET=CHIPS
0.005,
```

通过在 [*RIGID BODY](../key/key-link.md#usb-kws-mrigidbody) 选项上引用单元集 `FLOOR` 和刚体参考节点来定义刚体。必须指定参考节点的实际节点*编号*，而不是节点集名称。

```
*RIGID BODY, ELSET=FLOOR, REF NODE=*<reference node number>*
```

现在需要为电路板和泡沫包装定义材料属性。对于电路板，使用弹性杨氏模量为 45 GPa、泊松比为 0.3、密度为 500 kg/m³ 的 PCB 弹性材料。

*ELASTIC

```

*ELASTIC
 3.E6, 0.0
*DENSITY
 100.,

**边界条件**

通过向参考节点（之前定义为节点集 `REF`）施加固定边界条件，完全约束表示地板的刚性表面。

```
*BOUNDARY
REF, ENCASTRE
```

**初始条件**

电路板和泡沫包装在全局 3 方向给予 4.43 m/s 的初始速度，对应于 1 米自由落体结束时的速度。

```
*INITIAL CONDITIONS, TYPE=VELOCITY
BOARD, 3, -4.43
PACK, 3, -4.43
```

**定义接触**

此问题可以使用任一种接触算法。但是，由于与通用接触不同，接触对中涉及的表面不能跨越多个体，因此使用接触对算法定义接触会更加麻烦。我们在此示例中使用通用接触算法，以展示对更复杂几何形状进行接触定义的简单性。

首先，使用 [*SURFACE INTERACTION](../key/key-link.md#usb-kws-hsurfaceinteraction) 选项定义命名接触属性，摩擦系数定义为 0.3。

```
*SURFACE INTERACTION, NAME=FRIC
*FRICTION
0.3,
```

使用 [*CONTACT](../key/key-link.md#usb-kws-hcontact) 选项定义通用接触相互作用。使用 [*CONTACT INCLUSIONS](../key/key-link.md#usb-kws-hcontactinclusions) 选项上的 ALL EXTERIOR 参数指定对 Abaqus/Explicit 自动定义的无名全包容表面的自接触。[*CONTACT PROPERTY ASSIGNMENT](../key/key-link.md#usb-kws-hcontpropassign) 选项用于将名为 `FRIC` 的接触属性分配给通用接触相互作用。

```
*CONTACT
*CONTACT INCLUSIONS, ALL EXTERIOR
*CONTACT PROPERTY ASSIGNMENT
 , , FRIC
```

### 12.10.6 检查输入文件——历史数据

[*DYNAMIC](../key/key-link.md#usb-kws-hdynamic), EXPLICIT 选项用于选择使用显式积分的动态应力/位移分析。步骤的时间周期定义为 20 ms。

```
*STEP
*DYNAMIC, EXPLICIT
, 0.02
```

**输出请求**

通过在输入文件中包含以下行，将预选场数据写入输出数据库文件：

```
*OUTPUT, FIELD, VARIABLE=PRESELECT
```
每个连接芯片的垂直节点位移（U3）、速度（V3）和加速度（A3）值将作为历史数据写入输出数据库文件。选择了 0.07 ms 的输出间隔。
```
*OUTPUT, HISTORY, TIME INTERVAL=0.07E-3
*NODE OUTPUT, NSET=CHIPS
U3, V3, A3
```

能量值将按整个模型求和写出。具体来说，写出动能（ALLKE）、内能（ALLIE）、弹性应变能（ALLSE）、人工能（ALLAE）以及塑性变形耗散的能量（ALLPD）。

```
*ENERGY OUTPUT
ALLIE, ALLKE, ALLPD, ALLAE, ALLSE
```

步骤结束使用 [*END STEP](../key/key-link.md#usb-kws-hendstep) 选项表示。

### 12.10.8 后处理

通过在操作系统提示符下键入以下内容来启动 Abaqus/Viewer：

```
abaqus viewer odb=circuit
```

**检查材料方向**

可以使用 Abaqus/Viewer 检查从该方向定义获得的材料方向。

**绘制材料方向：**

1. 首先，将视图更改为更方便的设置。如果不可见，请从主菜单栏中选择 ****视图**![](../graphics/images/arrow.gif)**工具栏**![](../graphics/images/arrow.gif)**视图**** 以显示**视图**工具栏。在**视图**工具栏中，选择 X--Z 设置。
2. 从主菜单栏中，选择 ****绘图**![](../graphics/images/arrow.gif)**材料方向**![](../graphics/images/arrow.gif)**在变形形状上****。显示了模拟结束时电路板材料方向的方向。材料方向以不同颜色绘制。材料 1 方向为蓝色，材料 2 方向为黄色，3 方向（如果存在）为红色。
3. 要查看初始材料方向，请选择 ****结果**![](../graphics/images/arrow.gif)**步骤/帧****。在出现的**步骤/帧**对话框中，选择`增量 0`。单击**应用**。Abaqus 显示初始材料方向。
4. 要将显示恢复到分析结束时的结果，请在**步骤/帧**对话框中选择最后一个可用增量；然后单击**确定**。

**结果动画**

您将创建变形的时间历史动画，以帮助您可视化撞击期间电路板和泡沫包装的运动和变形。

**创建时间历史动画：**

1. 在分析结束时绘制变形模型形状。
2. 从主菜单栏中，选择 ****动画**![](../graphics/images/arrow.gif)**时间历史****。变形模型形状的动画开始。
3. 从主菜单栏中，选择 ****视图**![](../graphics/images/arrow.gif)**平行**** 以关闭透视。
4. 在上下文栏中，单击 ![](../graphics/ico_framePause.png) 在完成一个完整周期后暂停动画。
5. 在上下文栏中，单击 ![](../graphics/ico_viewCameraNode.png)，然后在泡沫包装上选择一个靠近撞击地板的角落的节点。当您重新启动动画时，相机将随所选节点移动。如果放大该节点，它将在整个动画过程中保持在视野中。**注意：**要重置相机以跟随全局坐标系，请在上下文栏中单击 ![](../graphics/ico_viewCameraGlobal.png)。

当您查看跌落测试的变形历史时，请注意泡沫何时与地板接触。您应该观察到分析的前 4 ms 内发生初次撞击。第二次撞击发生在约 8 ms 到 15 ms。撞击后约 4 ms 时泡沫和板的变形状态如图 12-56 所示。

**图 12-56** 4 ms 时的变形网格。

**绘制模型能量历史**

绘制各种能量变量与时间的关系图。能量输出可以帮助您评估 Abaqus/Explicit 模拟是否预测了适当的响应。

**绘制能量历史：**

1. 在结果树中，在名为 `circuit.odb` 的输出数据库的**历史输出**上单击鼠标按钮 3。从出现的菜单中选择**筛选**。
2. 在筛选字段中，输入 `*ALL*` 以将历史输出限制为仅能量输出变量。
3. 选择 ALLAE 输出变量，并将其保存为 `Artificial Energy`。

5. 选择 ALLKE 输出变量，并将其保存为 `Kinetic Energy`。

*为什么通过对加速度数据进行积分计算的速度和位移曲线与分析期间记录的速度和位移不同？*

在此示例中，加速度数据已被一种称为混叠的现象损坏。混叠是一种数据损坏，当信号（如 Abaqus 分析的结果）在离散时间点采样时发生，但保存的数据点不足以正确描述信号。混叠现象可以使用数字信号处理（DSP）方法解决，其基本原理之一是奈奎斯特采样定理（也称为香农采样定理）。采样定理要求信号以大于信号最高频率两倍的速率采样。因此，给定采样率可以描述的最大频率内容是该速率的一半（奈奎斯特频率）。以高于采样率奈奎斯特频率的频率对具有大幅振荡的信号进行采样（存储）可能会因混叠而产生显著失真的结果。在此示例中，芯片加速度每 0.07 ms 采样一次，这是 14.3 kHz 的采样率（采样率是采样大小的倒数）。记录的数据存在混叠，因为芯片加速度响应在 7.2 kHz（采样率的一半）以上具有频率内容。

**正弦波的混叠**

为了更好地理解混叠如何扭曲数据，请考虑以各种采样率采样的 1 kHz 正弦波（如图 12-61 所示）。

**图 12-61** 以 1.1 kHz 和 3 kHz 采样的 1 kHz 正弦波。

![](../graphics/gsa-sinewave-nls.png)

根据采样定理，必须以大于 2 kHz 的速率对该信号进行采样以避免混叠失真。我们将评估当采样率大于或小于此值时会发生什么。

考虑以 1.1 kHz 采样率记录的数据；该速率低于所需的 2 kHz 速率。结果曲线表现出混叠失真，因为它是非常误导性的 1 kHz 正弦波表示。

现在考虑以 3 kHz 采样率记录的数据；该速率高于所需的 2 kHz 速率。原始信号的频率内容被捕获而没有混叠。然而，此采样率不够高，无法非常准确地保证采样信号峰值被捕获。要保证记录的局部峰值有 95% 的准确度，采样率必须超过信号频率十倍或更多。

**避免混叠**

在前两个混叠示例（混叠芯片加速度和混叠正弦波）中，仅从混叠数据本身来看，混叠已经发生并不明显。此外，无法仅从混叠数据中唯一地重建原始信号。因此，应注意避免分析结果出现混叠，特别是在最可能发生混叠的情况下。

对混叠的敏感性取决于多种因素，包括输出率、输出变量和模型特性。回想一下，在采样率一半（奈奎斯特频率）以上的频率具有大幅振荡的信号可能会因混叠而显著失真。最可能具有大幅高频内容的两个输出变量是加速度和反作用力。因此，这些变量最容易受到混叠的影响。另一方面，位移本质上频率含量较低，因此不易受到混叠影响。其他结果变量（如应力和应变）介于这两个极端之间。任何降低解高频响应的模型特性都会降低分析对混叠的敏感性。例如，弹性主导的撞击问题比包含能量吸收包装的本电路板跌落测试更容易受到混叠影响。

确保结果中不出现混叠的最安全方法是请求每增量输出。当您执行此操作时，输出率由稳定时间增量决定，这基于模型的最高可能频率响应。但是，请求每增量输出通常不切实际，因为它会产生非常大的输出文件。此外，每增量输出通常比您需要的更多数据；捕获高频解噪声是没有必要的，因为您真正感兴趣的是较低频率的结构响应。避免混叠的替代方法是以较低的速率请求输出，并使用 Abaqus/Explicit 实时滤波功能在将数据写入输出数据库之前去除结果中的高频内容。这种技术比请求每增量输出使用更少的磁盘空间；但是，由您来确保您的输出率和滤波器选择是适当的（以避免混叠或与数字信号处理相关的其他失真）。

Abaqus/Explicit 为场数据和历史数据提供滤波功能。这里只讨论历史数据的滤波。

### 12.10.9 使用输出滤波重新运行分析

在本节中，您将为电路板跌落测试分析的历史输出请求添加实时滤波器。虽然 Abaqus/Explicit 确实允许您创建基于您指定标准的用户定义输出滤波器（Butterworth、Chebyshev I 型和 Chebyshev II 型），但在此示例中我们将使用内置抗混叠滤波器。内置抗混叠滤波器旨在为您提供在指定输出率下记录的结果的最佳无混叠表示。为此，Abaqus/Explicit 内部应用低通、二阶 Butterworth 滤波器，截止频率设置为采样率的六分之一。有关更多信息，请参阅 Dassault Systèmes 知识库上的"过滤 Abaqus 历史输出概述"，网址为 [www.3ds.com/support/knowledge-base](http://www.3ds.com/support/knowledge-base)。有关定义自己的实时滤波器的更多信息，请参阅 ["在 Abaqus/Explicit 中过滤输出和对输出进行运算"，"输出到输出数据库"，Abaqus 分析用户指南第 4.1.3 节](../usb/usb-link.md#usb-out-odboutput-filter)。

**修改历史输出请求**

当 Abaqus 将节点历史输出写入输出数据库时，它会给每个数据对象一个名称，指示记录的输出变量、使用的滤波器（如果有）、节点号和节点集。对于此练习，您将为底部芯片（节点 `403`）创建多个输出请求，这些请求仅在输出采样率方面不同，而采样率不是历史输出名称的一部分。为了轻松区分相似的输出请求，为节点 `403` 创建两个新集合。将其中一个新集合命名为 `BotChip-all`，另一个命名为 `BotChip-largeInc`。

```
*NSET, NSET=BotChip-all
403
*NSET, NSET=BotChip-largeInc
403
```

接下来，为芯片的垂直位移、速度和加速度添加新的历史输出请求。此外，请求顶部芯片（节点集 `BOTPART`）顶部面（截面点 5）的对数应变分量（LE11、LE22 和 LE12）和对数主应变（LEP）的单元输出。对于这些输出请求，每 0.07 ms 记录一次数据，并应用内置抗混叠滤波器。

```
*OUTPUT, HISTORY, TIME INTERVAL=0.07E-3, FILTER=ANTIALIASING
*NODE OUTPUT, NSET=CHIPS

*ELEMENT OUTPUT, ELSET=BOTPART

```

请求每增量对底部芯片的垂直位移、速度和加速度进行历史输出。将节点集 `BotChip-all` 用于此输出请求。

```
*OUTPUT, HISTORY, FREQUENCY=1
*NODE OUTPUT, NSET=BotChip-all
 U3, V3, A3
```

为底部芯片的垂直位移、速度和加速度添加一个更多的输出请求。这次每 0.7 ms 请求一次输出，并应用内置抗混叠滤波器。使用节点集 `BotChip-largeInc`。

```

完成后，底部芯片将有四个历史输出请求（原始请求和这里添加的三个）。

**评估底部芯片的滤波加速度**

当分析完成时，测试使用内置抗混叠滤波器每 0.07 ms 记录的底部芯片加速度历史输出的合理性。方法是保存然后积分滤波加速度数据（集合 `CHIPS` 中节点 `403` 的 `A3_ANTIALIASING`），并将结果与记录的速度和位移数据进行比较，就像之前对未滤波版本的结果所做的那样。这次您应该会发现，通过对滤波加速度进行积分而计算的速度和位移曲线与分析期间写入输出数据库的速度和位移值非常相似。您可能还注意到，无论是否使用内置抗混叠滤波器，速度和位移结果都是相同的。这是因为节点速度和位移曲线的最高频率内容远小于采样率的一半。因此，当数据在未滤波的情况下记录时，没有发生混叠，而当应用内置抗混叠滤波器时，它没有效果，因为没有高频响应可以去除。

接下来，比较每增量记录的加速度 `A3` 历史输出与每 0.07 ms 记录的加速度 `A3` 历史曲线。首先绘制每增量的数据，这样就不会遮盖其他结果。

**绘制加速度历史**

1. 在结果树中，按 `*A3*Node 403*` 筛选**历史输出**容器，然后双击节点集 `BotChip-all` 的加速度 `A3` 历史输出。
2. 使用 **[Ctrl]****+点击**选择集合 `CHIPS` 中 `Node 403` 的两个加速度 `A3_ANTIALIASING` 历史输出对象（一个使用内置抗混叠滤波器过滤，另一个未过滤）；单击鼠标按钮 3 并从出现的菜单中选择**添加到绘图**。*X--Y* 绘图出现在视口中。放大以仅查看结果的前三分之一，并自定义绘图外观以获得类似于图 12-62 的绘图。**图 12-62** 有和没有过滤的加速度输出比较。![](../graphics/gsx-a3filter-exp-v.png)

首先考虑每增量记录加速度历史。此曲线包含大量数据，包括高频解噪声，其幅度非常大，遮盖了加速度的结构显著性较低频率成分。当请求每增量输出时，输出时间增量与稳定时间增量相同，这（为确保稳定性）基于模型最高可能频率响应的保守估计。结构显著频率通常比模型最高频率低两到四个数量级。在此示例中，稳定时间增量范围为 8.4×10⁻⁴ ms 到 8.8×10⁻⁴ ms（见状态文件 `circuit.sta`），对应于约 1 MHz 的采样率；此采样率已为此讨论向下舍入，即使这意味着该值不是保守的。回想采样定理，给定采样率可以描述的最高频率是其一半；因此，此模型的最高频率约为 500 kHz，典型结构频率可能高达 2–3 kHz（比模型最高频率低两个数量级以上）。虽然每增量记录一次的输出包含在 3 到 500 kHz 范围内的许多不良解噪声，但它是保证良好的（无混叠）数据，必要时可以在后处理操作中过滤。

接下来考虑每 0.07 ms 记录一次且未过滤的数据。回想一下，这是我们知道被混叠损坏的曲线。曲线直接从点跳到点，包括在每个 0.07 ms 间隔后原始加速度值。高频噪声的可变性质使得此混叠结果对解中的微小变化非常敏感（例如，由于计算机平台之间的差异），因此您每 0.07 ms 记录的结果可能与图 12-62 所示的结果显著不同。同样，我们通过对混叠加速度（分别为图 12-59 和图 12-60）数据进行积分而生成的速度和位移曲线，对解噪声的微小差异非常敏感。

当对每 0.07 ms 请求的输出应用内置抗混叠滤波器时，在将结果写入输出数据库之前，会过滤掉无法被 14.3 kHz 采样率捕获的过高频率内容。为此，Abaqus 内部定义低通、二阶 Butterworth 滤波器。低通滤波器衰减高于指定截止频率的信号频率含量。理想低通滤波器将完全消除高于截止频率的所有频率，同时对低于截止频率的频率内容没有影响。实际上，在截止频率周围存在一个频率带过渡带，部分衰减。为了补偿这一点，内置抗混叠滤波器的截止频率为采样率的六分之一，低于二分之一采样率的奈奎斯特频率。在大多数情况下（包括此示例），此截止频率足以确保在将数据写入输出数据库之前，已去除高于奈奎斯特频率的所有频率内容。

Abaqus/Explicit 不检查以确保指定的输出时间间隔为内部抗混叠滤波器提供适当的截止频率；例如，Abaqus 不检查是否仅消除了信号噪声。当加速度数据每 0.07 ms 记录时，施加的内部抗混叠滤波器的截止频率为 2.4 kHz。此截止频率几乎与我们之前确定为模型最大物理有意义频率相同的值（比稳定时间增量可以捕获的最大频率低两个数量级以上）。0.07 ms 的输出间隔是专门为此示例选择的，以避免过滤可能是物理有意义的频率内容。接下来，我们将研究当以过大的输出时间间隔应用抗混叠滤波器时的结果。

**绘制过滤加速度历史**

1. 在结果树中，双击节点集 `BotChip-all` 的加速度 `A3` 历史输出。
2. 选择 `Node 403` 的两个过滤加速度 `A3_ANTIALIASING` 历史输出对象；单击鼠标按钮 3 并从出现的菜单中选择**添加到绘图**。*X--Y* 绘图出现在视口中。缩小并自定义绘图外观以获得类似于图 12-63 的绘图。**图 12-63** 以不同输出采样率过滤的加速度。![](../graphics/gsx-a3filter-exp-largeinc-v.png)

图 12-63 清楚说明了当以过大的输出时间增量使用内置抗混叠滤波器时可能出现的一些问题。首先，请注意，当以大时间增量记录加速度时，加速度输出中的许多振荡被过滤掉了。在这个动态撞击问题中，被过滤掉的频率内容的大部分可能是物理有意义的。之前，我们估计结构响应的频率可能高达 2–3 kHz；但是，当采样间隔为 0.7 ms 时，以 0.24 kHz 的低截止频率进行过滤（0.7 ms 的采样间隔对应于 1.43 kHz 的采样频率，其六分之一是 0.24 kHz 的截止频率）。即使每 0.7 ms 记录的结果可能无法捕获所有物理有意义的频率内容，它确实捕获了加速度数据的低频内容而没有混叠失真。请记住，过滤会降低峰值估计值，这是可取的，如果仅过滤解噪声的话，但当物理有意义的解变化被去除时，可能会产生误导。

另一个需要注意的问题是，每 0.7 ms 记录的结果中存在时间延迟。这种时间延迟（或相位偏移）影响所有实时滤波器。滤波器必须有一些输入才能产生输出；因此，过滤结果将包含一些时间延迟。虽然所有实时过滤都会引入一些时间延迟，但随着滤波器截止频率的降低，时间延迟变得更加明显；滤波器必须具有更长时间跨度的输入，以便去除较低频率内容。如果增加滤波器阶数（如果您创建了用户定义的滤波器而不是使用二阶内置抗混叠滤波器，则这是一个选项），也会导致输出时间延迟增加。有关更多信息，请参阅 ["在 Abaqus/Explicit 中过滤输出和对输出进行运算" in "输出到输出数据库，" Abaqus 分析用户指南第 4.1.3 节](../usb/usb-link.md#usb-out-odboutput-filter)。

谨慎使用实时过滤功能。在此示例中，如果我们没有适当的比较数据，我们就无法识别严重过滤数据的问题。一般来说，在 Abaqus/Explicit 中最好使用最少量的过滤，这样输出数据库包含在合理数量的时间点（而不是每增量）记录的解的丰富、无混叠表示。如果需要额外过滤，可以在 Abaqus/Viewer 中作为后处理操作进行。

**在 Abaqus/Viewer 中过滤加速度历史**

在本节中，我们将使用 Abaqus/Viewer 过滤写入输出数据库的加速度历史数据。在 Abaqus/Viewer 中进行后处理过滤比 Abaqus/Explicit 中可用的实时过滤有几个优点。在 Abaqus/Viewer 中，您可以快速过滤 *X–Y* 数据并绘制结果。您可以轻松地将过滤结果与未过滤结果进行比较，以验证过滤器产生了预期效果。使用此技术，您可以快速迭代以找到合适的滤波器参数。此外，Abaqus/Viewer 滤波器不会出现分析期间应用过滤时不可避免的时间延迟。但是，请记住，后处理过滤器无法弥补不良分析历史输出；如果数据被混叠或者物理有意义的频率被去除，没有任何后处理操作可以恢复丢失的内容。

为了演示在 Abaqus/Viewer 中过滤与在 Abaqus/Explicit 中过滤之间的差异，我们将在 Abaqus/Viewer 中过滤底部芯片的加速度，并将结果与 Abaqus/Explicit 写入输出数据库的过滤数据进行比较。

**过滤加速度历史：**

1. 在结果树中，选择节点集 `BotChip-all` 的加速度 `A3` 历史输出，并将其保存为 `A3-all`。
2. 在结果树中，双击 **XYData**；然后在**创建 XY 数据**对话框中选择**对 XY 数据进行运算**。单击**继续**。
3. 在**对 XY 数据进行运算**对话框中，使用等效于 Abaqus/Explicit 内置抗混叠滤波器（当输出增量为 0.7 ms 时）应用的滤波器选项来过滤 `A3-all`。回想一下，内置抗混叠滤波器是二阶 Butterworth 滤波器，截止频率为输出采样率的六分之一；因此，对话框顶部的表达式应显示为：`butterworthFilter ( xyData="A3-all", cutoffFrequency=1/(6*0.0007) )`
4. 单击**绘制表达式**以绘制过滤加速度曲线。
5. 在结果树中，在节点集 `BotChip-largeInc` 的过滤加速度 `A3_ANTIALIASING` 历史输出上单击鼠标按钮 3；并从出现的菜单中选择**添加到绘图**。如果愿意，也可以添加集合 `CHIPS` 中节点 `403` 的过滤加速度历史。*X--Y* 绘图出现在视口中。和之前一样，自定义绘图外观以获得类似于图 12-64 的绘图。**图 12-64** 在 Abaqus/Explicit 和 Abaqus/Viewer 中过滤的加速度比较。![](../graphics/gsx-a3filter-vis-exp-v.png)

在图 12-64 中，可以清楚地看到 Abaqus/Viewer 中的后处理过滤器不会出现分析运行时应用过滤时发生的时间延迟。这是因为 Abaqus/Viewer 滤波器是双向的，这意味着过滤首先在正向传递中应用（引入一些时间延迟），然后在反向传递中应用（去除时间延迟）。由于 Abaqus/Viewer 中的双向过滤，过滤实际上应用了两次，与单程滤波器相比，这导致过滤信号的额外衰减。这就是为什么在 Abaqus/Viewer 中过滤的加速度曲线中的局部峰值比在 Abaqus/Explicit 中过滤的曲线略低的原因。

为了更好地了解 Abaqus/Viewer 的过滤功能，返回**对 XY 数据进行运算**对话框，并使用其他滤波器选项过滤加速度数据。例如，尝试不同的截止频率。

*您能否确认与时间增量大小为 0.07 的内置抗混叠滤波器相关的 2.4 kHz 截止频率是合适的？将截止频率提高到 6 kHz、7 kHz 甚至 10 kHz 会产生显著不同的结果吗？*

您应该发现适度增加截止频率对结果没有显著影响，这意味着当我们以 2.4 kHz 的截止频率进行过滤时，我们可能没有遗漏物理有意义的频率内容。

比较使用 Butterworth 和 Chebyshev I 型滤波器过滤加速度数据的结果。Chebyshev 滤波器需要一个纹波因子参数（*rippleFactor*），它表示您将允许多少振荡以换取改进的滤波器响应；请参阅 ["在 Abaqus/Explicit 中过滤输出和对输出进行运算" in "输出到输出数据库，" Abaqus 分析用户指南第 4.1.3 节](../usb/usb-link.md#usb-out-odboutput-filter) 了解更多信息。对于 Chebyshev I 型滤波器，纹波因子为 0.071 将导致非常平坦的通带，纹波仅为 0.5%。

*当截止频率为 5 kHz 时，您可能不会注意到滤波器之间的太大差异，但截止频率为 2 kHz 时呢？当您增加 *Chebyshev I 型*滤波器的阶数时会发生什么？*

将您的结果与图 12-65 中显示的结果进行比较。

**图 12-65** 使用 Butterworth 和 Chebyshev I 型滤波器过滤的加速度比较。

![](../graphics/gsx-a3filter-vis-v.png)

**注意：**Abaqus/Viewer 后处理过滤器默认是二阶的。要定义更高阶的滤波器，您可以将 *filterOrder* 参数与 **butterworthFilter** 和 **chebyshev1Filter** 运算符一起使用。例如，在**对 XY 数据进行运算**对话框中使用以下表达式，用 2 kHz 的截止频率和 0.017 的纹波因子对 `A3-all` 进行六阶 Chebyshev I 型滤波：

```
chebyshev1Filter ( xyData="A3-all" , cutoffFrequency=2000,
 rippleFactor= 0.017, filterOrder=6)
```

纹波因子为 0.071 的二阶 Chebyshev I 型滤波器是相对较弱的滤波器，因此 2 kHz 截止频率以上的某些频率内容没有被过滤掉。当滤波器阶数增加时，滤波器响应得到改善，因此结果更接近等效 Butterworth 滤波器。有关 Abaqus/Viewer 中可用的 *X–Y* 数据滤波器的更多信息，请参阅 ["对保存的 X--Y 数据对象进行运算，" Abaqus/CAE 用户指南第 47.4 节](../usi/usi-link.md#usv-xyp-operating)。

**在 Abaqus/Viewer 中过滤应变历史**

电路板芯片附近区域的应变是另一个可以帮助我们确定泡沫包装有效性的结果。如果芯片下方的应变超过限制值，将芯片固定在板上的焊料将失效。我们希望识别任何方向上的峰值应变。因此，最大和最小主对数应变是我们感兴趣的。主应变是 Abaqus 结果中从非线性运算符导出的众多结果之一；在此例中，使用非线性函数从各个应变分量计算主应变。其他常见的从非线性运算符导出的结果有主应力、Mises 应力和等效塑性应变。在过滤从非线性运算符导出的结果时需要小心，因为非线性运算符（与线性运算符不同）可以修改原始结果的频率。过滤这样的结果可能会产生不良后果；例如，如果您去除了由应用非线性运算符引入的频率内容的一部分，则过滤结果将是对导出量的扭曲表示。一般来说，您应该避免直接过滤从非线性运算符导出的量，或者在计算导出量之前对底层量进行过滤。

此分析的历史应变输出每 0.07 ms 使用内置抗混叠滤波器记录。为了验证抗混叠滤波器没有扭曲主应变结果，我们将使用过滤的应变分量计算主对数应变，并将其与过滤的主对数应变进行比较。

**计算主对数应变：**

1. 在结果树中，按 `*LE*` 筛选**历史输出**，选择集合 `BOTPART` 中元素 SPOS 表面上的对数应变分量 `LE11`，并将其保存为 `LE11`。
2. 同样，保存同一元素的 `LE12` 和 `LE22` 应变分量，分别为 `LE12` 和 `LE22`。
3. 在结果树中，双击 **XYData**；然后在**创建 XY 数据**对话框中选择**对 XY 数据进行运算**。单击**继续**。
4. 在**对 XY 数据进行运算**对话框中，使用保存的对数应变分量计算最大主对数应变。对话框顶部的表达式应显示为：`(("LE11"+"LE22")/2) + sqrt( power(("LE11"-"LE22")/2,2) + power("LE12"/2,2) )`
5. 单击**另存为**将计算的最大主对数应变保存为 `LEP-Max`。
6. 编辑**对 XY 数据进行运算**对话框中的表达式，以计算最小主对数应变。修改后的表达式应显示为：`(("LE11"+"LE22")/2) - sqrt( power(("LE11"-"LE22")/2,2) + power("LE12"/2,2) )`
7. 单击**另存为**将计算的最小主对数应变保存为 `LEP-Min`。为了使用与分析期间记录的应变相同的 *Y* 轴绘制计算的主对数应变，请将 *Y* 值类型更改为应变。
8. 在结果树的 **XYData** 容器中，在 `LEP-Max` 上单击鼠标按钮 3；并从出现的菜单中选择**编辑**。
9. 在**编辑 XY 数据**对话框中，选择**应变**作为 *Y* 值类型。
10. 同样，编辑 `LEP-Min` 并选择**应变**作为 *Y* 值类型。
11. 使用结果树，绘制 `LEP-Max` 和 `LEP-Min` 以及分析期间记录的主应变（集合 `BOTPART` 中元素的 `LEP1` 和 `LEP2`）。
12. 和之前一样，自定义绘图外观以获得类似于图 12-66 的绘图。**图 12-66** 主对数应变值与时间的关系。![](../graphics/gsx-prin-strain-v.png)

在图 12-66 中，我们看到分析期间记录并使用过滤应变分量计算的主对数应变曲线与过滤主对数应变曲线无法区分。因此，抗混叠滤波器（截止频率 2.4 kHz）没有去除从原始应变数据计算主应变时应用非线性运算引入的任何频率内容。接下来，以 500 Hz 的较低截止频率对应变数据进行过滤。

**以 500 Hz 截止频率过滤主对数应变：**

1. 在结果树中，双击 **XYData**；然后在**创建 XY 数据**对话框中选择**对 XY 数据进行运算**。单击**继续**。
2. 在**对 XY 数据进行运算**对话框中，使用截止频率为 500 Hz 的二阶 Butterworth 滤波器对最大主对数应变 `LEP-Max` 进行过滤。对话框顶部的表达式应显示为：`butterworthFilter(xyData="LEP-Max", cutoffFrequency=500)`
3. 单击**另存为**将计算的最大主对数应变保存为 `LEP-Max-FilterAfterCalc-bw500`。
4. 类似地，使用相同的截止频率为 500 Hz 的二阶 Butterworth 滤波器对 `LE11`、`LE12` 和 `LE22` 对数应变分量进行过滤。将结果曲线分别保存为 `LE11--bw500`、`LE12--bw500` 和 `LE22--bw500`。
5. 现在使用过滤的对数应变分量计算最大主对数应变。**对 XY 数据进行运算**对话框顶部的表达式应显示为：`(("LE11-bw500"+"LE22-bw500")/2) + sqrt( power(("LE11-bw500"-"LE22-bw500")/2,2) + power("LE12-bw500"/2,2) )`
6. 单击**另存为**将计算的最大主对数应变保存为 `LEP-Max-CalcAfterFilter-bw500`。
7. 在结果树的 **XYData** 容器中，在 `LEP-Max-CalcAfterFilter-bw500` 上单击鼠标按钮 3；并从出现的菜单中选择**编辑**。
8. 在**编辑 XY 数据**对话框中，选择**应变**作为 *Y* 值类型。
9. 绘制 `LEP-Max-CalcAfterFilter-bw500` 和 `LEP-Max-FilterAfterCalc-bw500`，如图 12-67 所示。**图 12-67** 计算前后过滤的主对数应变（截止频率 500 Hz）。![](../graphics/gxi-prin-strain-filter-v.png)

在图 12-67 中，您可以看到在主应变计算之前和之后过滤应变数据之间存在显著差异。计算主应变后过滤的曲线失真了，因为应用非线性主应力运算符引入的某些频率内容高于 500 Hz 滤波器截止频率。一般来说，您应该避免直接过滤从非线性运算符导出的量；尽可能在计算所需导出量之前对底层分量进行过滤，然后应用非线性运算符。

**记录和过滤 Abaqus/Explicit 历史输出的策略**

在 Abaqus/Explicit 中每增量记录输出通常会产生比您需要的更多的数据。实时过滤功能允许您以较低的频率请求历史输出，而不会因混叠而产生扭曲的结果。但是，您应该确保您的输出率和过滤选择没有去除物理有意义的频率内容，也没有扭曲结果（例如，引入大的时间延迟或去除由非线性运算符引入的频率内容）。请记住，没有后处理过滤可以恢复分析期间过滤掉的频率内容，也不能从混叠数据中恢复原始信号。此外，如果没有可用于比较的其他数据，可能无法明显发现结果被过度过滤或混叠。一个好的策略是选择相对较高的输出率，并使用 Abaqus/Explicit 过滤器来防止历史输出混叠，以便将有效和丰富的结果写入输出数据库。您甚至可能希望为一些关键位置请求每增量输出。分析完成后，在 Abaqus/Viewer 中使用后处理工具根据需要快速迭代地应用额外过滤。
