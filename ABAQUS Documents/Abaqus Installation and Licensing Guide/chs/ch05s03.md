# 5.3 调整图形卡



本节包含配置 Abaqus/CAE 和 Abaqus/Viewer 以使用尚未合格的图形适配器所需的信息。

### 5.3.1 为什么需要调整？

SIMULIA 在每个版本之前调整和限定有限数量的图形适配器。这些图形适配器的调整参数包含在 Abaqus 中。但是，在版本之间会推出新的图形适配器和现有图形适配器的新驱动程序。调整可能使您能够利用这些新的适配器和驱动程序，而无需等待 Abaqus/CAE 或 Abaqus/Viewer 的新版本。

Abaqus/CAE 和 Abaqus/Viewer 使用 OpenGL 进行高速图形渲染。虽然 OpenGL 标准有严格的合规性测试，但某些功能是实现相关的，需要调整才能正确运行。调整新的图形适配器或驱动程序可确保正确渲染 Abaqus/CAE 和 Abaqus/Viewer 图形，并确保每个系统获得最大渲染性能。

您可以在 **Support** 页面上的 [www.3ds.com/simulia](http://www.3ds.com/simulia) 找到有关合格图形适配器的最新信息。如果您阅读此页面上的信息并遵循本节描述的调整过程，您应该能够正确渲染 Abaqus/CAE 和 Abaqus/Viewer 图形，并以最大性能渲染。如果继续遇到问题，您应该联系您当地的技术支持办事处寻求帮助。

### 5.3.2 如何调整参数？

Abaqus/CAE 和 Abaqus/Viewer 提供以下两种调整图形参数的方法：
- 从主菜单栏选择****视图**![](../graphics/images/arrow.gif)**图形选项****。Abaqus 显示**图形选项**对话框（[图 5-1](ch05s03.md#graphics-options-db)），您可以从其中选择所需的设置。此方法允许您仅从最常用的调整参数中进行选择。**图 5-1****图形选项**对话框。 ![](../graphics/sgb-graphics-options-db-nls.png) **图形选项**对话框中的设置在[Abaqus/CAE User's Guide 第 7 章，"配置图形显示选项"](../usi/usi-link.md#uss-pfm)中有描述。
- 使用 Abaqus Scripting Interface 命令选择所需的设置。您可以在命令行界面 (CLI) 中输入命令并修改调整参数的值。此方法提供对所有调整参数的完整控制，并在["使用 Abaqus Scripting Interface 调整图形参数，"第 5.3.3 节](ch05s03.md#sgb-chp-view-tune-acl)中描述。本节还描述了如何使用 Abaqus Scripting Interface 命令获取有关您正在使用的图形卡的信息。

### 5.3.3 使用 Abaqus Scripting Interface 调整图形参数

您可以在命令行界面中输入 Abaqus Scripting Interface 命令来调整图形参数并查找有关系统上安装的图形适配器的信息。本节说明如何使用 Abaqus Scripting Interface 修改图形选项；Abaqus Scripting Interface 在[Abaqus Scripting User's Guide](../cmd/cmd-link.md#cmd)中有详细描述。

通常，您应该对大多数参数使用默认值。但是，Abaqus 提供了修改参数以修复以下特定问题的功能：
- *hardwareAcceleration* 参数控制许多不同的图形调整参数，通常不应修改。硬件加速选项在["硬件加速（所有平台），"第 5.1.1 节](ch05s01.md#sgb-chp-view-cust-hardware)中讨论。
- *hardwareOverlay* 参数由 *hardwareOverlayAvailable* 参数控制。如果您的系统支持硬件覆盖平面，Abaqus/CAE 和 Abaqus/Viewer 将默认使用它们。如果您的系统支持硬件覆盖平面但视口显示纯色且不显示模型，您可能需要手动设置 *hardwareOverlay*=OFF。
- *contourRangeTexturePrecision* 参数设置计算将结果（轮廓）值转换为纹理值的适当比例时使用的容差。当设置过低时，范围限制附近的值可能错误显示"超出范围"颜色。
- 某些图形适配器不完全支持使用纹理来正确生成轮廓图。如果您遇到显示轮廓图的问题（例如，所有轮廓显示为灰色或系统中止），您需要设置 *textureMapping*=OFF 以在软件中模拟纹理映射。同样，如果您遇到打印轮廓图的问题，您需要设置 *printTextureMapping*=OFF。
- 某些图形适配器不完全支持使用顶点数组处理有关顶点的信息。一些特定问题表明顶点数组未得到完全支持：当您在 Sketcher 中拖动圆的半径时，圆不可见；当您显示 X-Y 图时，轴标签不可见；以及网格的阴影显示中缺少某些面片。如果您遇到这些问题，请设置 *vertexArraysInDisplayLists*=OFF。如果这不能解决问题，请完全抑制顶点数组的使用，方法是将 *vertexArrays*=OFF。
- *backfaceCulling* 参数控制确定为远离查看者的面的面的显示。如果元素的前侧在显示中缺失或显示不完整，请设置 *backfaceCulling*=OFF。
- 对于在直接访问图形硬件时行为不正确的 Linux 系统，您可以禁用直接渲染（设置 *directRendering*=OFF）。
- 当您希望打印的图像不使用 OpenGL 硬件加速呈现时，或者如果您在 Abaqus/CAE (Abaqus/Viewer) 可视化模块中遇到 Probe 功能问题，您可以禁用硬件加速的屏幕外渲染（设置 *accelerateOffScreen*=OFF）。
- 当您希望节省内存时，可以禁用备份存储（设置 *backingStore*=OFF）。当 *accelerateOffScreen*=ON 时，备份存储的内存从图形卡上的内存分配。当 OFF 时，备份存储的内存从系统内存分配。备份存储通过渲染到屏幕外区域来生成。如果视口可以足够快速地重绘，则不会创建备份存储。即使 *backingStore*=ON，如果视口可以足够快速地重绘，也不会创建备份存储。
- *translucencyMode* 参数决定 Abaqus/CAE 是否针对性能、准确性或中间级别优化半透明对象的渲染。较低的值提供更好的性能，而较高的值提供更高的准确性。
- *polygonOffsetConstant* 和 *polygonOffsetSlope* 参数影响屏幕显示，需要为每个图形适配器手动调整。在 Linux 系统上，*printPolygonOffsetConstant* 和 *printPolygonOffsetSlope* 参数通常可以设置为与相应屏幕显示参数相同的值。在 Windows 系统上，*printPolygonOffsetConstant* 和 *printPolygonOffsetSlope* 参数通常不需要调整。
- 如果在启动可视化模块中的视图操作时存在不可接受的延迟，则可以降低 *viewManipDisplayListThreshold* 参数。增加此值可能会增加大型模型的延迟，但应该在视图操作期间产生改进的图形性能。如果设置得较高且模型较大，延迟可能是几秒钟，在过度情况下可能超出系统图形内存并导致显示为空（没有可见模型）。您可以使用以下 Abaqus Scripting Interface 对象调整图形参数：
- GraphicsOptions：GraphicsOptions 对象的成员决定当前图形设置。这些设置可以在会话期间使用 `setValues` 方法进行修改。`setValues` 方法的参数在["setValues，"Abaqus Scripting Reference Guide 第 17.9.1 节](../ker/ker-link.md#ker-graphicsoptions-setvalues-pyc)中有描述。您可以通过在命令行界面中输入以下命令来查看图形参数的当前设置： ``` print session.graphicsOptions ``` 以下输出是典型的： ``` ({'accelerateOffScreen': OFF, 'antiAlias': ON, 'autoFitAfterRotate': OFF, 'backfaceCulling': ON, 'backgroundBottomColor': '#acacc1', 'backgroundColor': '#333366', 'backgroundOverride': ON, 'backgroundStyle': GRADIENT, 'backingStore': ON, 'contourRangeTexturePrecision': 5.0e-06 'directRendering': ON, 'displayLists': ON, 'doubleBuffering': ON, 'dragMode': AS_IS, 'graphicsDriver': OPEN_GL, 'hardwareAcceleration': ON, 'hardwareOverlay': OFF, 'hardwareOverlayAvailable': False, 'highlightMethod': SOFTWARE_OVERLAY, 'highlightMethodHint': (HARDWARE_OVERLAY, SOFTWARE_OVERLAY, XOR, BLEND), 'polygonOffsetConstant': 2.0, 'polygonOffsetSlope': 0.75, 'printPolygonOffsetConstant': 1.0, 'printPolygonOffsetSlope': 0.75, 'printTextureMapping': ON, 'shadersAvailable': True, 'stencil': False, 'textureMapping': ON, 'translucencyMode': 3, 'vertexArrays': ON, 'vertexArraysInDisplayLists': ON, 'viewManipDisplayListThreshold': 40}) ``` **注意：**上述某些参数已弃用。有关访问已弃用参数的信息，请参阅["BackwardCompatibility 对象，"Abaqus Scripting Reference Guide 第 53.4 节](../ker/ker-link.md#ker-utl-deprecated-pyc)。以下命令使用 `setValues` 方法修改 GraphicsOptions 对象的某些成员： ``` session.graphicsOptions.setValues(autoFitAfterRotate=ON, dragMode=AS_IS) ``` 您可以在命令行界面中输入此命令。
- GraphicsInfo：GraphicsInfo 对象的成员提供有关系统上安装的图形适配器的信息。此信息可能有助于确定如何调整图形适配器。成员在["GraphicsInfo 对象，"Abaqus Scripting Reference Guide 第 17.8 节](../ker/ker-link.md#ker-graphicsinfo-pyc)中有描述。成员仅供参考，无法修改。您可以通过在命令行界面中输入以下命令来查看图形信息： ``` print session.graphicsInfo ``` 以下是 Windows 平台上的典型输出： ``` ({'glRenderer': 'Quadro FX 560/PCI/SSE2', 'glVendor': 'NVIDIA Corporation', 'glVersion': (2, 0, '.3'), 'glxClientVendor': None, 'glxClientVersion': (None, None, None), 'glxServerVendor': None, 'glxServerVersion': (5, 1, None)}) ```

**调整 polygonOffsetConstant 和 polygonOffsetSlope 参数**

如果启用了显示列表，您将看不到调整这些参数的效果；因此，在尝试调整图形适配器之前，必须关闭**使用显示列表**。或者，您可以在命令行界面中输入以下命令：

```
session.graphicsOptions.setValues(displayLists=OFF)
```

将拖动模式设置为 AS_IS 有助于微调参数。交互旋转视图将显示是否需要小幅调整。

```
session.graphicsOptions.setValues(dragMode=AS_IS)
```

建议首先调整 *polygonOffsetConstant*，然后调整 *polygonOffsetSlope*。要调整这些参数，您应该首先显示由[Abaqus Scripting User's Guide 第 3.1 节](../cmd/cmd-link.md#cmd-int-overview-examplepart)"创建部件"中的示例脚本生成的部件。要检索脚本，请在命令提示符窗口（操作系统 shell）中使用以下命令：

```
abaqus fetch job=modelAExample
```
从主菜单栏选择****文件**![](../graphics/images/arrow.gif)**运行脚本****，从出现的**运行脚本**对话框中选择示例脚本，然后单击**确定**。示例脚本会创建一个新视口；但是，当您尝试调整图形参数时，Abaqus 应该只显示一个视口。选择任何旧视口并通过从主菜单栏选择****视口**![](../graphics/images/arrow.gif)**删除当前****来删除它们。

**调整 polygonOffsetConstant 参数：**

1. 从**视图**工具栏，应用底视图 ![](../graphics/ico_viewBottom.png)。
2. 在命令行界面中，输入以下命令： ``` session.graphicsOptions.setValues(polygonOffsetSlope=0.0) session.graphicsOptions.setValues(polygonOffsetConstant=0.0) ```
3. 再次显示底视图以刷新显示。
4. 检查模型的可见线条。如果所有线条都不可见，重复第 2 步，小幅增加多边形偏移常量的值；例如，`0.1`。此参数的正常范围在 `0.5` 到 `1.5` 之间，通常两位小数足以提供足够的精度。您应该尝试找到尽可能小的值以产生正确的显示。太大的值可能导致线条似乎漂浮在部件上方。以下图形说明了模型中应该可见的线条。[图 5-2](ch05s03.md#bad-polygon-constant) 说明 *polygonOffsetConstant* 值不正确的情况；某些线条在阴影区域之间缺失。**图 5-2***polygonOffsetConstant* 值不正确。 ![](../graphics/sgb-bad-polygon-constant.png) [图 5-3](ch05s03.md#good-polygon-constant) 说明 *polygonOffsetConstant* 值正确的情况；所有阴影区域都被线条分隔。**图 5-3***polygonOffsetConstant* 值正确。 ![](../graphics/sgb-good-polygon-constant.png)

在得出 *polygonOffsetConstant* 的值后，您可以调整 *polygonOffsetSlope*。

**调整 polygonOffsetSlope 参数：**

1. 从**视图**工具栏，应用等轴测视图 ![](../graphics/ico_viewIso.png)。此视图在至少一个轴上显示 45 度角的边缘。
2. 在命令行界面中，输入以下命令： ``` session.graphicsOptions.setValues(polygonOffsetSlope=1.0) ```
3. 检查模型的可见线条。如果所有线条都不可见，重复第 2 步，小幅增加多边形偏移斜率的值；例如，`0.1` 或 `0.05`。此参数的正常范围在 `1.0` 到 `2.0` 之间，通常两位小数足以提供足够的精度。如果 *polygonOffsetConstant* 值太低，可能会迫使 *polygonOffsetSlope* 很高。*polygonOffsetSlope* 的高值可能导致隐藏多边形的边缘在遇到可见多边形时显示出来。在这种情况下，提高 *polygonOffsetConstant* 值以获得可接受的 *polygonOffsetSlope* 值。[图 5-4](ch05s03.md#bad-polygon-slope) 说明 *polygonOffsetSlope* 值不正确的情况；某些线段在阴影区域之间缺失，孔内缺少某些线条，某些线条应为实线却显示为虚线。**图 5-4***polygonOffsetSlope* 值不正确。 ![](../graphics/sgb-bad-polygon-slope.png) [图 5-5](ch05s03.md#good-polygon-slope) 说明 *polygonOffsetSlope* 值正确的情况；所有阴影区域都被实线分隔。**图 5-5***polygonOffsetSlope* 值正确。 ![](../graphics/sgb-good-polygon-slope.png)

在几个模型上测试调整后的 *polygonOffsetConstant* 和 *polygonOffsetSlope* 值以确保值令人满意。完成图形参数调整后，您应该将显示列表和拖动模式的设置返回到原始值。

当您对参数设置满意时，应按照["使您的图形配置永久化，"第 5.3.4 节](ch05s03.md#sgb-chp-view-tune-permanent)中所述修改环境文件。

### 5.3.4 使您的图形配置永久化

当您对调整参数指定的值满意时，可以通过在环境文件 (`abaqus_v6.env`) 中包含 `onCaeGraphicsStartup` 函数来使更改永久化。为避免与其他图形设置冲突，您应该仅将自定义的 `onCaeGraphicsStartup` 函数添加到主目录中的环境文件（请参阅["使用 Abaqus 环境文件，"第 4.1 节](ch04s01.md)，了解环境文件位置和执行的详细信息）。

DefaultGraphicsOptions 对象的成员决定启动会话时以及在**图形选项**对话框中单击**默认**时启用的默认图形设置。您可以通过在命令行界面中输入以下命令来查看默认图形设置：

```
print session.defaultGraphicsOptions
```

您可以在环境文件 (`abaqus_v6.env`) 中使用 `setValues` 方法来修改 DefaultGraphicsOptions 对象的成员。要在环境文件中设置默认图形选项，必须使用 `session.defaultGraphicsOptions` 对象，而不是从命令行修改的 `session.graphicsOptions` 对象。以下示例环境文件配置您的 Abaqus/CAE 和 Abaqus/Viewer 图形设置：

```
def onCaeGraphicsStartup():
    session.defaultGraphicsOptions.setValues(
        polygonOffsetConstant=1.0,
        polygonOffsetSlope=1.2)
```
