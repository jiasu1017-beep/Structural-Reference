### 概述

多点约束（MPC）：
- 允许在模型的不同自由度之间施加约束；以及
- 可以非常通用（非线性和非齐次）。

最常用的约束可以直接通过选择MPC类型并给出相关数据来获得。可用的MPC类型如下所述；仅在Abaqus/Standard中可用的MPC用(S)标记。

在Abaqus/Standard中，约束也可以通过用户子程序[`MPC`](../sub/sub-link.md#sub-xsl-mpc)给出。

线性约束可以直接通过定义线性约束方程来给出（参见["线性约束方程，" 第35.2.1节](pt08ch35s02aus129.md)）。

在Abaqus/Explicit中，某些多点约束可以使用刚体更有效地建模（参见["刚体定义，" 第2.4.1节](pt01ch02s04aus22.md)）。

几种MPC类型也可用于连接器元素（["连接器元素，" 第31.1.2节](pt06ch31s01alm25.md)）。虽然连接器元素施加相同的运动约束，但连接器不会消除自由度。

MPC约束力不可用作输出量。因此，要输出MPC中施加约束所需的力，应使用等效的连接器元素。连接器元素的力、弯矩和运动输出可直接获得，定义在["连接器元素库，" 第31.1.4节](pt06ch31s01ael25.md)。

### 识别MPC中涉及的节点

对于任何MPC类型，节点集或单个节点都可以作为输入给出。如果第一个条目是节点，则后续条目必须是节点。如果第一个条目是节点集，则后续条目可以是节点集或单个节点。当一组节点上的某个自由度取决于单个节点上的自由度时，后一种选项很有用，例如在某些对称条件或刚体模拟中可能发生的情况。

如果使用节点集，则相应的集合条目将相互约束。如果给出排序的节点集作为输入，则必须确保节点编号使得它们在排序后能正确匹配。未排序节点集中的节点（参见["节点定义，" 第2.1.1节](pt01ch02s01aus05.md)）将按照定义集合时给出的顺序使用。

在Abaqus/Standard中，多点约束不能用于在参考节点以外的节点处连接两个刚体，因为多点约束使用自由度消除，而刚体上的其他节点没有独立的自由度。在Abaqus/Explicit中，可以在多点约束定义中使用刚体参考节点或刚体上的任何其他节点。

Abaqus/CAE使用连接器定义两点之间的多点约束，使用约束定义点与区域中从属节点之间的多点约束。不支持Abaqus/CAE中的集对集多点约束和未排序节点集。

### 与转换坐标系一起使用

可以为连接到MPC的任何节点定义局部坐标系（参见["转换坐标系，" 第2.1.5节](pt01ch02s01aus09.md)）。对于用户定义的MPC有一些特殊考虑，如["MPC，" Abaqus用户子程序参考指南第1.1.14节](../sub/sub-link.md#sub-rtn-umpc)中所述。

### 在一点定义多个多点约束

关于Abaqus/Standard和Abaqus/Explicit中如何处理一点的多个运动约束的详细信息，参见["运动约束：概述，" 第35.1.1节](pt08ch35s01abo32.md)。

在Abaqus/Standard中，MPC通常通过消除给出的第一个节点上的自由度（从属自由度）来施加。MPC类型BEAM、CYCLSYM、LINK、PIN、REVOLUTE、TIE和UNIVERSAL由Abaqus/Standard在内部排序，以便节点作为从属节点的MPC是使用该节点的最后一个MPC。因此，这些MPC组可以以任何顺序给出。然而，即使对于这些MPC，节点也只能作为从属节点使用一次。在其他情况下，从属自由度不应随后用于施加运动约束；这通常排除将MPC定义中的第一个节点用作任何后续多点约束、方程约束、运动耦合约束或绑定约束定义中的独立节点。

### 在隐式动态分析中使用MPC

在隐式动态分析中，Abaqus/Standard对位移严格强制执行MPC。速度和加速度由动态积分算子定义的位移关系推导得出（参见["隐式动态分析，" Abaqus理论指南第2.4.1节](../stm/stm-link.md#stm-anl-dynamics)）。对于线性MPC（如PIN、TIE和网格无关紧固件MPC）和几何线性分析，以这种方式获得的速度精确满足约束。然而，加速度仅近似满足约束。如果在几何非线性分析中使用非线性MPC（如BEAM、LINK和SLIDER），则速度和加速度都仅近似满足约束。在大多数情况下，这种近似是相当准确的，但在某些情况下，涉及MPC的节点加速度可能出现高频振荡。

### 在几何线性Abaqus/Standard分析中使用非线性MPC

如果在几何线性Abaqus/Standard分析中使用非线性MPC（参见["一般和线性扰动过程，" 第6.1.3节](pt03ch06s01aus44.md)），则MPC将被线性化。例如，如果在几何非线性Abaqus/Standard分析中使用MPC LINK，则链接两个节点之间的距离保持不变。如果在几何线性Abaqus/Standard分析中使用，则两个节点之间的距离在投影到节点原始位置之间的连线方向后保持不变。只有当旋转和位移的幅度不小的时候，这种差异才是明显的。

### 在用户子程序中定义MPC

在Abaqus/Standard中，您可以在用户子程序[`MPC`](../sub/sub-link.md#sub-xsl-mpc)中定义多点约束。

用户子程序中定义的约束只能使用同样存在于模型中某个元素上的自由度。例如，如果模型不包含具有旋转自由度的元素，则用户子程序[`MPC`](../sub/sub-link.md#sub-xsl-mpc)不能使用自由度4、5或6。可以通过在模型中的某个位置添加合适的元素来克服此限制，以引入所需自由度。可以添加此元素，使其不影响模型的响应。

用户子程序中定义的约束应用于转换后的自由度。当在用户子程序中激活/停用MPC时，Abaqus/Standard会出现边界非线性。

### 用于网格细化的MPC

| LINEAR | 这是对一阶元素进行网格细化的标准方法。它适用于所涉及节点上的所有活动自由度，包括温度、压力和电势。在Abaqus/Explicit中，可能更倾向于使用基于表面的绑定约束（参见["网格绑定约束，" 第35.3.1节](pt08ch35s03aus132.md)）进行网格细化，特别是当要被约束的网格之一或多个涉及带厚度的壳元素时。 |
| --- | --- |
| QUADRATIC(S) | 这是对二阶元素进行网格细化的标准方法。它适用于所涉及节点上的所有活动自由度，但耦合温度-位移分析和耦合热-电-结构分析中的温度自由度以及耦合孔隙压力分析中的压力自由度除外。对于使用二阶孔隙压力或耦合温度位移元素进行细化，必须将此MPC与P LINEAR或T LINEAR MPC结合使用。 |
| BILINEAR(S) | 这是对三维一阶实体元素进行网格细化的标准方法。它适用于所涉及节点上的所有活动自由度，包括温度、压力和电势。 |
| C BIQUAD(S) | 这是对三维二阶实体元素进行网格细化的标准方法。它适用于所涉及节点上的所有活动自由度，但耦合温度-位移分析和耦合热-电-结构分析中的温度自由度以及耦合孔隙压力分析中的压力自由度除外。对于在三维中使用孔隙压力或耦合温度位移元素进行细化，必须将此MPC与P BILINEAR或T BILINEAR MPC结合使用。 |
| P LINEAR(S) | 此MPC可与QUADRATIC MPC结合使用，用于二阶全耦合孔隙流体流-位移元素的网格细化。它仅适用于压力自由度。对于声学分析，它施加与LINEAR MPC相同的约束。 |
| T LINEAR(S) | 此MPC可与QUADRATIC MPC结合使用，用于二阶全耦合温度-位移和全耦合热-电-结构元素的网格细化。它仅适用于温度自由度。对于热传递分析，它施加与LINEAR MPC相同的约束。 |
| P BILINEAR(S) | 此MPC可与C BIQUAD MPC结合使用，用于三维中孔隙流体流-位移元素的网格细化。它仅适用于压力自由度。对于声学分析，它施加与BILINEAR MPC相同的约束。 |
| T BILINEAR(S) | 此MPC可与C BIQUAD MPC结合使用，用于三维中全耦合温度-位移和全耦合热-电-结构元素的网格细化。它仅适用于温度自由度。对于热传递分析，它施加与BILINEAR MPC相同的约束。 |

#### 将网格细化MPC与壳或梁元素一起使用

Abaqus/Standard壳元素S4R5、S8R5、S9R5和STRI65使用惩罚方法在元素边缘施加横向剪切约束。因此，使用网格细化MPC LINEAR和QUADRATIC可能导致弯曲行为的过度约束或"剪切锁定"。对于这些元素进行网格细化，建议使用三角形元素创建过渡区的分级网格。

Abaqus/Standard中的剪切柔性梁元素（如B31或B32）如果用作网格线上的加强件，而该网格线上使用了网格细化MPC，也会"锁定"。

对于Abaqus/Explicit中的壳元素，旋转自由度不受LINEAR MPC约束；因此，沿约束节点定义的线会形成铰链。

### 用于连接和关节的MPC

| BEAM | 在两个节点之间提供刚性梁，以约束第一个节点的位移和旋转与第二个节点的位移和旋转相对应，相当于两个节点之间存在刚性梁。 |
| --- | --- |
| CYCLSYM(S) | 约束节点以在模型中施加循环对称。 |
| ELBOW(S) | 将ELBOW31或ELBOW32元素的两个节点约束在一起，其中横截面方向![](../graphics/usb_eqn06126.gif)发生变化（参见["具有变形横截面的管道和弯管：弯管元素，" 第29.5.1节](pt06ch29s05alm15.md)）。 |
| LINK | 在两个节点之间提供销钉刚性链接，以保持两个节点之间的距离不变。修改第一个节点的位移以强制执行此约束。如果存在，节点上的旋转不参与此约束。 |
| PIN | 在两个节点之间提供销钉关节。这使得位移相等，但如果存在旋转，则使它们相互独立。 |

#### 使用MPC类型REVOLUTE

此MPC类型仅在Abaqus/Standard中可用。

旋转关节是一种允许两个节点之间围绕在运动过程中旋转的轴进行相对旋转的关节（参见[图35.2.2-13](pt08ch35s02aus130.md#pmpc-revolute)）。关节轴在初始配置中定义为从节点*b*到节点*c*的线。如果这些节点重合，则假定轴为全局*z*轴。关节轴的旋转是节点*b*的旋转。

#### 建模壳到实体元素过渡

SLIDER、SS LINEAR、SS BILINEAR和SSF BILINEAR MPC允许从壳元素建模到实体元素建模在壳表面上进行过渡。此建模技术可用于获得壳-实体交点或其他不连续处的解决方案，其中局部建模应使用完整的三维理论，但结构的其他部分可以建模为壳。壳到实体子建模能力（["子建模：概述，" 第10.2.1节](pt04ch10s02aus60.md)）和基于表面的壳到实体耦合约束（["壳到实体耦合，" 第35.3.3节](pt08ch35s03aus134.md)）也可用于在这种情况下获得更准确的解决方案，且建模工作量大大减少。

在Abaqus/Standard中，MPC用法假定壳和实体元素之间的界面是包含沿网格交点线的壳法线的表面，使得界面法向方向上实体网格侧的节点线是直线。（[图35.2.2-14](pt08ch35s02aus130.md#pmpc-slider-shell-solid)中的线*a*、![](../graphics/usb_eqn06127.gif)、![](../graphics/usb_eqn06128.gif)、…、*b*以及[图35.2.2-19](pt08ch35s02aus130.md#pmpc-sslinear)到[图35.2.2-20](pt08ch35s02aus130.md#pmpc-ssbilinear)中的线![](../graphics/usb_eqn06127.gif)、![](../graphics/usb_eqn06128.gif)、…、![](../graphics/usb_eqn06129.gif)应为直线。）它还假定实体元素的节点在界面表面上均匀分布，如[图35.2.2-14](pt08ch35s02aus130.md#pmpc-slider-shell-solid)和[图35.2.2-19](pt08ch35s02aus130.md#pmpc-sslinear)到[图35.2.2-20](pt08ch35s02aus130.md#pmpc-ssbilinear)所示。对于边缘上的每个壳节点，使用适当类型的MPC类型SS LINEAR、SS BILINEAR或SSF BILINEAR将壳节点约束到通过厚度对应的实体元素节点线或面。然后，使用SLIDER MPC将厚度方向上每个内部节点约束到该线底部和顶部节点定义的直线上。有关示例，参见["多点约束，" Abaqus验证指南第5.1.17节](../ver/ver-link.md#ver-msc-mpc)。

SS BILINEAR和SSF BILINEAR MPC不适用于可变节点实体元素（C3D27、C3D27H、C3D27R和C3D27RH）。

在Abaqus/Standard中，MPC SS LINEAR、SS BILINEAR和SSF BILINEAR消除壳节点上的所有位移分量和两个旋转分量，SLIDER MPC消除界面上每个内部实体元素节点的两个位移分量。因此，界面上所需的任何边界条件（如壳/实体界面与对称平面相交时所需的边界条件）应仅应用于界面实体元素侧的顶部和底部节点。
