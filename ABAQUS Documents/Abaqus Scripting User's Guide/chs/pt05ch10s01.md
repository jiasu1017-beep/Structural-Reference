# 10.1 修改和访问Abaqus/CAE GUI模块和工具集

派生新类以创建修改后的Abaqus/CAE模块和工具集，允许你自定义现有函数而无需更改原始函数。你也可以在用Abaqus GUI工具包创建的新对话框中访问现有的Abaqus/CAE函数。本节涵盖以下主题：

- ["Abaqus/CAE GUI模块和工具集，" 第10.1.1节](pt05ch10s01.md#cus-app-mod-caemodule)
- ["访问Abaqus/CAE函数，" 第10.1.2节](pt05ch10s01.md#cus-app-mod-caeaccess)

### 10.1.1 Abaqus/CAE GUI模块和工具集

Abaqus GUI工具包旨在允许你添加你自己的模块和工具集。通常不建议修改Abaqus/CAE模块和工具集，因为Abaqus/CAE的未来更改可能会"破坏"你的应用程序。但是，如果你确实需要修改某些Abaqus/CAE模块或工具集，可以通过从其中一个派生新类然后添加或删除组件来进行更改。

要派生新类，你必须知道适当的类名，并且必须在你的构造函数中调用该类的构造函数。下表列出了Abaqus GUI工具包中所有可用的Abaqus/CAE模块的类名和注册名称。你可以从`abaqusGui`导入这些类名。

当你注册从Abaqus/CAE模块之一派生的模块时，必须使用表中显示的名称作为主窗口`registerModule`方法中*displayedName*参数的名称。如果不使用显示的名称，某些GUI基础设施组件可能无法正常工作。

| 类名 | 名称 |
| --- | --- |
| PartGui | "Part" |
| PropertyGui | "Property" |
| AssemblyGui | "Assembly" |
| StepGui | "Step" |
| InteractionGui | "Interaction" |
| LoadGui | "Load" |
| MeshGui | "Mesh" |
| OptimizationGui | "Optimization" |
| JobGui | "Job" |
| VisualizationGui | "Visualization" |
| SketchGui | "Sketch" |

当你注册工具集时，必须在`registerToolset`方法中指定工具集在哪些位置（菜单栏、工具栏或工具箱）创建窗口部件。如果你省略了工具集位置标志，该工具集的GUI将不会出现在该位置。下表显示了每个Abaqus/CAE工具集的类名以及指示工具集创建窗口部件位置的标志。你可以从`abaqusGui`导入这些类名。

要注册插件工具集，调用`registerPluginToolset()`方法；不要使用`registerToolset`方法。

有关如何注册Abaqus/CAE使用的工具集和模块的示例，请参阅["主窗口示例，" 第14.2.1节](pt06ch14s02.md#cus-app-custom-main-example)。以下语句显示如何将你自己的工具集添加到可视化模块：

```
# 文件 myVisModuleGui.py：

    from abaqusGui import *
    from myToolsetGui import MyToolsetGui

    class MyVisModuleGui(VisualizationGui):

        def __init__(self):

            # 构造基类。
            #
            VisualizationGui.__init__(self)

            # 注册我的工具集。
            #
            self.registerToolset(MyToolsetGui(),
                GUI_IN_MENUBAR|GUI_IN_TOOLBOX)

    MyVisModuleGui()

# 文件 myMainWindow.py：

    from abaqusGui import *

    class MyMainWindow(AFXMainWindow):

        def __init__(self, app, windowTitle=''):

            ...
            self.registerModule('Visualization',
                'myVisModuleGui')
            ...
```

如果你从Abaqus/CAE工具集派生工具集，必须使用`AFXMainWindow`的`makeCustomToolsets`方法构造该工具集。你必须使用`makeCustomToolsets`方法以确保工具集在应用程序启动期间的适当时间创建。这将避免与也使用该模块的Abaqus/CAE模块发生任何冲突。例如，如果你从基准工具集派生新工具集，必须在`makeCustomToolsets`中创建新工具集。以下示例说明了这种方法。新工具集也将出现在Part模块中，取代标准基准工具集。

```
# 在你的主窗口文件中：

    class MyMainWindow(AFXMainWindow):

        def __init__(self, app, windowTitle=''):

              ...
        def makeCustomToolsets(self):

            from myDtmToolsetGui import MyDtmGui
            # 如果你也想在其中一个模块中注册它，
            # 将工具集存储为主窗口的成员。
            #
            self.myDtmGui = MyDtmGui()

# 在你的模块GUI文件中：

      class MyModuleGui(AFXModuleGui):

          def __init__(self):

              ...
              mw = getAFXApp().getAFXMainWindow()
              self.registerToolset(mw.myDtmGui,
                 GUI_IN_TOOL_PANE|GUI_IN_TOOLBOX)
```

### 10.1.2 访问Abaqus/CAE函数

如果你想从你自己的对话框启动Abaqus/CAE函数，可以通过将适当的目标和选择器连接到你的按钮来实现。你可以使用主窗口的`getTargetFromFunction`和`getSelectorFromFunction`方法获取特定函数的目标和选择器。例如：

```
    mainWindow = getAFXApp().getAFXMainWindow()
    target = mainWindow.getTargetFromFunction('Part->Create')
    selector = mainWindow.getSelectorFromFunction('Part->Create')
    FXButton(self, 'Create Part...', tgt=target, sel=selector )
```

有效函数名的列表可以在"工具"![](../graphics/images/arrow.gif)**自定义**对话框的"函数"标签页中找到。
