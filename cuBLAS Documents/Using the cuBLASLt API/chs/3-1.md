## 3.1. 概述

cuBLASLt 库是一个新的轻量级库，专用于通用矩阵乘法（GEMM）操作，并提供新的灵活 API。该库在矩阵数据布局、输入类型、计算类型以及通过参数可编程性选择算法实现和启发式方法方面增加了灵活性。

一旦用户确定了目标 GEMM 操作的选项集，这些选项可以重复用于不同的输入。这类似于 cuFFT 和 FFTW 首先创建计划，然后对不同输入数据重用相同大小和类型的 FFT。

> **注意**
>
> cuBLASLt 库不保证支持所有可能的尺寸和配置，但是自 CUDA 12.2 update 2 以来，m、n 和批量大小的问题尺寸限制已得到大幅改善。该库的主要目标是提供性能最优的内核，这可能存在一些隐含的限制。某些非标准配置可能需要用户手动处理，通常是将问题分解为更小的部分（请参阅问题尺寸限制）。

### 3.1.1. 问题尺寸限制

问题尺寸限制是 CUDA 网格维度限制的结果。例如，由于网格 *z* 维度的限制，许多内核不支持大于 65535 的批量大小。对于给定的问题，m 和 n 值也存在类似的限制。

当单个内核无法运行某个问题时，cuBLASLt 将尝试将问题分解为多个子问题，并通过在每个子问题上运行内核来求解。

**cuBLASLt 内部问题分解存在一些限制，总结如下：**
不支持 Amax 计算。这意味着 CUBLASLT_MATMUL_DESC_AMAX_D_POINTER 和 CUBLASLT_MATMUL_DESC_EPILOGUE_AUX_AMAX_POINTER 必须保持未设置状态（请参阅 cublasLtMatmulDescAttributes_t）
所有矩阵布局必须将 CUBLASLT_MATRIX_LAYOUT_ORDER 设置为 CUBLASLT_ORDER_COL（请参阅 cublasLtOrder_t）
当 CUBLASLT_MATMUL_DESC_EPILOGUE 设置为 CUBLASLT_EPILOGUE_DRELU_BGRAD 或 CUBLASLT_EPILOGUE_DGELU_BGRAD 时，cuBLASLt 不会沿 n 维度进行分区（请参阅 cublasLtEpilogue_t）

为了克服这些限制，用户可能需要自己分区问题，为每个子问题启动内核，并计算必要的归约以合并结果。

### 3.1.2. 启发式缓存

cuBLASLt 使用启发式方法根据问题尺寸、GPU 配置和其他参数选择最适合执行的矩阵乘法内核。这需要在主机 CPU 上执行一些计算，可能需要数十微秒。为了克服这一开销，建议使用 cublasLtMatmulAlgoGetHeuristic() 查询启发式一次，然后对后续计算重用结果。

对于无法一次性查询启发式然后重用的情况，cuBLASLt 实现了启发式缓存，将矩阵乘法问题映射到之前由启发式选择的内核。启发式缓存使用类似 LRU 的淘汰策略，并且是线程安全的。

用户可以通过 `CUBLASLT_HEURISTICS_CACHE_CAPACITY` 环境变量或 cublasLtHeuristicsCacheSetCapacity() 函数（具有更高优先级）控制启发式缓存的容量。容量以条目数计量，可能会为性能原因向上取整到某个因子的最接近倍数。每个条目占用约 360 字节，但可能会有变化。默认容量为 8192 个条目。

> **注意**
>
> 将容量设置为零会完全禁用缓存。对于没有稳定状态的工作负载，缓存操作的开销可能高于常规启发式计算，禁用缓存可能很有用。

> **注意**
>
> 由于性能原因，缓存并不理想，因此有时需要将容量增加到预期唯一矩阵乘法问题数量的 1.5 倍到 2 倍以上，以实现接近完美的命中率。

另请参阅：cublasLtHeuristicsCacheGetCapacity()、cublasLtHeuristicsCacheSetCapacity()。

### 3.1.3. cuBLASLt 日志记录

cuBLASLt 日志记录机制可以通过在启动目标应用程序前设置以下环境变量来启用：

- CUBLASLT_LOG_LEVEL=<level>，其中 <level> 是以下级别之一：

  - 0 - Off - 日志记录被禁用（默认）
  - 1 - Error - 仅记录错误
  - 2 - Trace - 启动 CUDA 内核的 API 调用将记录其参数和重要信息
  - 3 - Hints - 可能潜在改善应用程序性能的提示
  - 4 - Info - 提供有关库执行的一般信息，可能包含启发式状态的详细信息
  - 5 - API Trace - API 调用将记录其参数和重要信息

- CUBLASLT_LOG_MASK=<mask>，其中 <mask> 是以下标志的组合：

  - 0 - Off
  - 1 - Error
  - 2 - Trace
  - 4 - Hints
  - 8 - Info
  - 16 - API Trace

  例如，使用 CUBLASLT_LOG_MASK=5 启用 Error 和 Hints 消息。

- CUBLASLT_LOG_FILE=<file_name>，其中 <file_name> 是日志文件的路径。文件名可能包含 %i，将被替换为进程 ID。例如 file_name_%i.log。

如果未设置 `CUBLASLT_LOG_FILE`，日志消息将打印到标准输出。

另一种选择是使用实验性的 cuBLASLt 日志 API。请参阅：

cublasLtLoggerSetCallback()、cublasLtLoggerSetFile()、cublasLtLoggerOpenFile()、cublasLtLoggerSetLevel()、cublasLtLoggerSetMask()、cublasLtLoggerForceDisable()

### 3.1.4. 窄精度数据类型的使用

我们在这里称为*窄精度数据类型*首先在 Ada 和 Hopper GPU（计算能力 8.9 及以上）中作为 8 位浮点数据类型（FP8）引入，旨在进一步加速矩阵乘法。可用的 FP8 类型有两种：

- CUDA_R_8F_E4M3 设计为在半精度更小的动态范围内保持准确。E4 和 M3 分别表示 4 位指数和 3 位尾数。更多详细信息，请参阅 __nv__fp8_e4m3。
- CUDA_R_8F_E5M2 设计为在半精度相似的动态范围内保持准确。E5 和 M2 分别表示 5 位指数和 2 位尾数。更多信息请参阅 __nv__fp8_e5m2。

> **注意**
>
> 除非另有说明，FP8 是指 CUDA_R_8F_E4M3 和 CUDA_R_8F_E5M2 两种类型。

在 Blackwell GPU（计算能力 10.0 及以上）中，cuBLAS 添加了对 4 位浮点数据类型（FP4）`CUDA_R_4F_E2M1` 的支持。E2 和 M1 分别表示 2 位指数和 1 位尾数。更多详细信息，请参阅 __nv_fp4_e2m1。

为了保持准确性，窄精度的数据需要在计算前进行缩放或反量化，并在计算后可能进行量化。cuBLAS 提供了多种缩放因子应用模式，定义在 cublasLtMatmulMatrixScale_t 中，并通过 `CUBLASLT_MATMUL_DESC_{X}_SCALE_MODE` 属性进行配置（这里 X 代表 A、B、C、D、D_OUT 或 EPILOGUE_AUX；请参阅 cublasLtMatmulDescAttributes_t）。下表给出了缩放模式概述，后续章节有更多详细信息。

| 模式 | 支持的计算能力 | 张量值数据类型 | 缩放因子数据类型 | 缩放因子布局 |
| --- | --- | --- | --- | --- |
| 张量级缩放 | 8.9+ | CUDA_R_8F_E4M3 / CUDA_R_8F_E5M2 | CUDA_R_32F 1 | 标量 |
| 外向量缩放 | 9.0 | CUDA_R_8F_E4M3 / CUDA_R_8F_E5M2 | CUDA_R_32F | 向量 |
| 128 元素 1D 块缩放 | 9.0 | CUDA_R_8F_E4M3 / CUDA_R_8F_E5M2 | CUDA_R_32F | 张量 |
| 128x128 元素 2D 块缩放 | 9.0 | CUDA_R_8F_E4M3 / CUDA_R_8F_E5M2 | CUDA_R_32F | 张量 |
| 32 元素 1D 块缩放 | 10.0+ | CUDA_R_8F_E4M3 / CUDA_R_8F_E5M2 | CUDA_R_8F_UE8M0 2 | 平铺张量 4 |
| 16 元素 1D 块缩放 | 10.0+ | CUDA_R_4F_E2M1 | CUDA_R_8F_UE4M3 3 | 平铺张量 4 |
| 实验性：每批次张量级缩放 | 10.0+ | CUDA_R_8F_E4M3 / CUDA_R_8F_E5M2 | CUDA_R_32F 1 | 指针数组 |

**脚注：**

**1(1,2)**
数据类型为 CUDA_R_32F 的缩放因子可以为负值，并按原样应用，不会先取绝对值。

**2**
CUDA_R_8F_UE8M0 是一种 8 位无符号纯指数浮点数据类型。更多信息请参阅 __nv_fp8_e8m0。

**3**
CUDA_R_8F_UE4M3 是 CUDA_R_E4M3 的无符号版本。符号位被忽略，因此提供此枚举是为了方便。

**4(1,2)**
有关更多详细信息，请参阅 1D 块缩放因子布局。

> **注意**
>
> 缩放仅适用于窄精度矩阵乘法。如果为非窄精度矩阵乘法设置了任何缩放，cuBLAS 将返回错误。此外，缩放通常仅支持窄精度张量。如果为非窄精度张量设置了相应的缩放，cuBLAS 将返回错误。例外情况是允许 C 张量在张量级缩放模式下对非窄数据类型设置缩放。

> **注意**
>
> 仅当张量级缩放模式支持时，所有矩阵的 cublasLtBatchMode_t 设置为 CUBLASLT_BATCH_MODE_POINTER_ARRAY 时才支持张量级缩放。

### 3.1.4.1. FP8 数据类型的张量级缩放

当所有 FP8 精度张量的 `CUBLASLT_MATMUL_DESC_{X}_SCALE_MODE` 属性（这里 X 代表 A、B、C、D 或 EPILOGUE_AUX；请参阅 cublasLtMatmulDescAttributes_t）设置为 `CUBLASLT_MATMUL_MATRIX_SCALE_SCALAR_32F`（这是 FP8 张量的默认值）时，启用张量级缩放。在这种情况下，cuBLAS 中的矩阵乘法操作定义如下（为说明起见，假设所有张量都使用 FP8 精度）：

$$D = scale_D \cdot (\alpha \cdot scale_A \cdot scale_B \cdot \text{op}(A) \text{op}(B) + \beta \cdot scale_C \cdot C)。$$

这里 \(A\)、\(B\) 和 \(C\) 是输入张量，\(scale_A\)、\(scale_B\)、\(scale_C\)、\(scale_D\)、\(\alpha\) 和 \(\beta\) 是输入标量。这与其他矩阵乘法例程不同，因为每个矩阵都添加了缩放因子。\(scale_A\)、\(scale_B\) 和 \(scale_C\) 用于反量化，\(scale_D\) 用于量化。请注意，所有缩放因子都是乘法应用的。这意味着有时需要根据应用上下文使用缩放因子或其倒数。有关 FP8 的更多信息，请参阅 cublasLtMatmul() 和 cublasLtMatmulDescAttributes_t。

对于此类矩阵乘法，尾声和中间值的绝对最大值计算如下：

$$\begin{split}Aux_{temp} & = \alpha \cdot scale_A \cdot scale_B \cdot \text{op}(A) \text{op}(B) + \beta \cdot scale_C \cdot C + bias, \\
D_{temp} & = \mathop{Epilogue}(Aux_{temp}), \\
amax_{D} & = \mathop{absmax}(D_{temp}), \\
amax_{Aux} & = \mathop{absmax}(Aux_{temp}), \\
D & = scale_D * D_{temp}, \\
Aux & = scale_{Aux} * Aux_{temp}. \\\end{split}$$

这里 \(Aux\) 是矩阵乘法的辅助输出，由传递给 GELU 等尾声函数的值组成，\(scale_{Aux}\) 是可应用于 \(Aux\) 的可选缩放因子，\(amax_{Aux}\) 是缩放前 \(Aux\) 中的最大绝对值。更多信息，请参阅 cublasLtMatmulDescAttributes_t 中的 `CUBLASLT_MATMUL_DESC_AMAX_D_POINTER` 和 `CUBLASLT_MATMUL_DESC_EPILOGUE_AUX_AMAX_POINTER` 属性。

> **注意**
>
> 如上方程所示，bias 在计算 \(Aux_{temp}\) 之前应用。

### 3.1.4.2. 实验性：FP8 数据类型的每批次张量级缩放

当所有 FP8 精度张量的 `CUBLASLT_MATMUL_DESC_{X}_SCALE_MODE` 属性（这里 X 代表 A、B、C、D 或 EPILOGUE_AUX；请参阅 cublasLtMatmulDescAttributes_t）设置为 `CUBLASLT_MATMUL_MATRIX_SCALE_PER_BATCH_SCALAR_32F` 时，启用每批次张量级缩放。

每批次张量级缩放是张量级缩放的一种变体，只是批次中的每个矩阵都有自己的缩放因子。

使用每批次张量级缩放时，\(scale_A\)、\(scale_B\)、\(scale_C\)、\(scale_D\) 和 \(scale_{Aux}\) 必须是长度为 `CUBLASLT_MATRIX_LAYOUT_BATCH_COUNT` 的设备指针数组。

> **注意**
>
> 仅当所有矩阵的 cublasLtBatchMode_t 设置为 CUBLASLT_BATCH_MODE_GROUPED 时才支持每批次张量级缩放。

### 3.1.4.3. FP8 数据类型的外向量缩放

此缩放模式（也称为通道级或行级缩放）是张量级缩放的改进。不是用单个标量乘以矩阵，而是为 \(A\) 的每一行和 \(B\) 的每一列关联一个缩放因子：

$$D_{ij} = \alpha \cdot scale_A^i \cdot scale_B^j \sum_{l=1}^k a_{il}\cdot b_{lj} + \beta \cdot scale_C \cdot C_{ij}。$$

值得注意的是，不支持 \(scale_D\)，因为 \(D\) 仅支持的精度是 `CUDA_R_16F`、`CUDA_R_16BF` 和 `CUDA_R_32F`。

要启用外向量缩放，必须将 `CUBLASLT_MATMUL_DESC_A_SCALE_MODE` 和 `CUBLASLT_MATMUL_DESC_B_SCALE_MODE` 属性设置为 `CUBLASLT_MATMUL_MATRIX_SCALE_OUTER_VEC_32F`，而所有其他缩放模式不得修改。

使用此缩放模式时，\(scale_A\) 和 \(scale_B\) 必须是长度分别为 \(M\) 和 \(N\) 的向量。

### 3.1.4.4. FP8 和 FP4 数据类型的 16/32 元素 1D 块缩放

1D 块缩放旨在克服用单个标量缩放整个张量的限制。在 OCP MXFP 规范中有更详细的描述。块缩放意味着同一 16 或 32 元素相邻值块内的元素被分配一个共享的缩放因子。

目前，块缩放支持 FP8 精度和 FP4 精度张量，不支持混合精度。要启用块缩放，必须将 `CUBLASLT_MATMUL_DESC_{X}_SCALE_MODE` 属性（这里 X 代表 A、B、C、DOUT 或 EPILOGUE_AUX；请参阅 cublasLtMatmulDescAttributes_t）设置为 `CUBLASLT_MATMUL_MATRIX_SCALE_VEC32_UE8M0`（对于所有 FP8 精度张量）或 `CUBLASLT_MATMUL_MATRIX_SCALE_VEC16_UE4M3`（对于所有 FP4 精度张量）。

使用块缩放，cuBLAS 中的矩阵乘法操作定义如下（为说明起见，假设所有张量都使用窄精度）。我们大致遵循 OCP MXFP 规范表示法。

首先，缩放块（或 OCP MXFP 规范中的 MX 兼容格式向量）是一个元组 \(x = \left(S^x, \left[x^i\right]_{i=1}^k\right)\)，其中 \(S^x\) 是共享缩放因子，每个 \(x^i\) 使用 FP8 或 FP4 数据类型存储。

两个缩放块 \(x = \left(S^x, \left[x^i\right]_{i=1}^{k}\right)\) 和 \(y = \left(S^y, \left[y^i\right]_{i=1}^{k}\right)\) 的点积定义如下：

$$Dot(x, y) = S^x S^y \cdot \sum_{i=1}^{k} x^i y^i。$$

对于 \(n\) 个块的序列 \(X = \{x_j\}_{j=1}^n\) 和 \(Y = \{y_j\}_{j=1}^n\)，广义点积定义为：

$$DotGeneral(X, Y) = \sum_{j=1}^n Dot(x_j, y_j)。$$

广义点积可用于通过在 \(K\) 维中每 \(k\) 个 \(A\) 和 \(B\) 元素组合一个缩放因子来定义矩阵乘法（为简单起见，假设 \(K\) 可被 \(k\) 整除）：

$$\begin{split}L & = \frac{K}{k}, \\
A_i & = \left\{{scale_A}_{i,b}, \left[A_{i,(b-1)k+l}\right]_{l=1}^{k}\right\}_{b=1}^L, \\
B_j & = \left\{{scale_B}_{i,b}, \left[B_{(b-1)k+l,j}\right]_{l=1}^{k}\right\}_{b=1}^L, \\
(\left\{scale_A, A\right\} \times \left\{scale_B, B\right\})_{i,j} & = DotGeneral(A_i, B_j).\end{split}$$

现在，完整的矩阵乘法可以写成：

$$\left\{scale_D^{out}, D\right\} = Quantize\left(scale_D^{in}\left(\alpha \cdot \left\{scale_A, \text{op}(A)\right\} \times \left\{scale_B, \text{op}(B)\right\} + \beta \cdot Dequantize(\left\{scale_C, C\right\})\right)\right)。$$

\(Quantize\) 在 1D 块量化一节中解释，\(Dequantize\) 定义为：

$$Dequantize\left(\left\{scale_C, C\right\})\right)_{i,j} = {scale_C}_{i/k,j} \cdot C_{i,j}。$$

> **注意**
>
> 除了量化期间计算的 \(scale_D^{out}\) 之外，还有 \(D\) 的输入标量张量级缩放因子 \(scale_D^{in}\)，仅当缩放因子使用 CUDA_R_8F_UE4M3 数据类型时可用。它用于在量化之前"压缩"计算值。

### 3.1.4.4.1. 1D 块量化

考虑 \(M\) 维度中 \(D\) 的单个 \(k\) 元素块：\(D^b_{fp32} = \left[d^i_{fp32}\right]_{i=1}^k\)。部分块的量化假定缺失值为零。设 \(Amax(DType)\) 为目标精度中可表示的最大值。

以下计算步骤适用于输出和缩放因子数据类型的所有组合。

1. 计算块绝对最大值 \(Amax(D^b_{fp32}) = max(\{|d_i|\}_{i=1}^k)\)。
2. 计算单精度块缩放因子 \(S^b_{fp32} = \frac{Amax(D^b_{fp32})}{Amax(DType)}\)。

**使用 UE8M0 缩放因子计算 FP8 的缩放和转换因子**

> **注意**
>
> 除非另有说明，假定使用 RNE 舍入。

计算包括以下步骤：

1. 从 \(S^b_{fp32}\) 中提取块缩放因子指数（无偏置调整）作为整数 \(E^b_{int}\) 和尾数作为定点数 \(M^b_{fixp}\)（实际实现直接操作位表示）。
2. 向上舍入块指数，保持其在 UE8M0 中可表示的值范围内：\(E^b_{int} = \left\{\begin{array}{ll} E^b_{int} + 1, & \text{if } S^b_{fp32} \text{ is a normal number and }  E^b_{int} < 254 \text{ and } M^b_{fixp} > 0 \\ E^b_{int} + 1, & \text{if } S^b_{fp32} \text{is a denormal number and } M^b_{fixp} > 0.5, \\ E^b_{int}, & \text{otherwise.} \end{array}\right.\)
3. 计算块缩放因子 \(S^b_{ue8m0} = 2^{E^b_{int}}\)。请注意，UE8M0 数据类型的指数偏置为 127。
4. 计算块转换因子 \(R^b_{fp32} = \frac{1}{fp32(S^b_{ue8m0})}\)。

> **注意**
>
> 上述算法与 OCP MXFP 建议的舍入方案不同。

**使用 UE4M3 缩放因子计算 FP4 的缩放和转换因子**

这里我们假设算法提供了一个预计算的张量级缩放因子 \(scale_D^{in}\)，在一般情况下计算如下：

$$scale_D^{in} = \frac{Amax(e2m1) \cdot Amax(e4m3)}{Amax(D_{temp})},$$

其中 \(Amax(D_{temp})\) 是量化前矩阵乘法结果的全局绝对最大值。由于计算此值需要知道整个计算的结果，实际中会使用例如前一次迭代的近似值。

计算包括以下步骤：

1. 计算块缩放因子的窄精度值 \(S^b_{e4m3} = e4m3(S^b_{fp32})\)。
2. 计算块转换因子 \(R^b_{fp32} = \frac{1}{fp32(S^b_{e4m3})}\)。

**应用转换因子**

对于每个 \(i = 1 \ldots k\)，计算 \(d^i = DType(d^i_{fp32} \cdot R^b_{fp32})\)。结果量化块为 \(\left(S^b, \left[d^i\right]_{i=1}^k\right)\)，其中 \(S^b\) 对于使用 UE8M0 缩放因子的 FP8 为 \(S^b_{ue8m0}\)，对于使用 UE4M3 缩放因子的 FP4 为 \(S^b_{ue4m3}\)。

### 3.1.4.4.2. 1D 块缩放因子布局

缩放因子使用平铺布局存储。下图显示了每个 128x4 块在内存中的布局。内存中的偏移量从左到右增加，然后从上到下增加。

以下伪代码可用于将 `inner`（A 和 B 的 K，C 或 D 的 M）和 `outer`（A 的 M，B、C 和 D 的 N）索引转换为块内的线性 `offset`，反之亦然：

```c++
// Indices -> offset
offset = (outer % 32) * 16 + (outer / 32) * 4 + inner

// Offset -> Indices
outer = ((offset % 16) / 4) * 32 + (offset / 16)
inner = (offset % 4)
```

当缩放模式为 `CUBLASLT_MATMUL_MATRIX_SCALE_VEC16_UE4M3` 时，单个缩放因子块应用于 128x64 块；当为 `CUBLASLT_MATMUL_MATRIX_SCALE_VEC32_UE8M0` 时应用于 128x128 块。

多个块以行主序排列。下一张图片显示了一个示例。内存中的偏移量从左到右增加，然后从上到下增加。

一般来说，对于每行有 `sf_inner_dim` 个缩放因子的缩放因子张量，左上坐标为 `(sf_outer, sf_inner)` 的块的偏移量（使用与上述相同的矩阵坐标对应关系）可以使用以下伪代码计算：

```c++
// Indices -> offset
//   注意，由于平铺布局，sf_inner 是 4 的倍数
offset = (sf_inner + sf_outer * sf_inner_dim) * 128
```

> **注意**
>
> 缩放因子的起始地址必须 16 字节对齐。

> **注意**
>
> 请注意，上述布局不允许转置。这意味着即使输入张量可以转置，缩放因子的布局也不会改变。

> **注意**
>
> 请注意，当张量维度不是上述块大小的倍数时，仍需要为存储分配完整的块，并用零填充超出边界的值。此外，在写入输出缩放因子时，内核可能会写入额外的零，因此最好不要对超出边界值的持久性做出任何假设。
