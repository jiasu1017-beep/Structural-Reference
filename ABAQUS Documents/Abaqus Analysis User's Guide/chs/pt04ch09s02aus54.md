# 12.2.1 结果传递概述

**产品：** Abaqus/Standard 和 Abaqus/Explicit

##### **参考资料**

- ["结果传递，" 第 3.2 节](../stm/stm-link.md#stm-anl-results-transfer)
- [*MODEL CHANGE](../key/key-link.md#usb-kws-hmodelchange)
- [*PREDEFINED FIELD](../key/key-link.md#usb-kws-mpredefinedfield)

### 概述

结果传递允许将一个分析的结果作为另一个分析的初始条件。这种技术常用于：

- 多步骤制造工艺模拟
- 不同物理场之间的耦合
- 模型简化（如用子模型替代整体模型）

### 结果传递类型

Abaqus 提供两种主要的结果传递方式：

| **类型** | **描述** | **应用** |
| --- | --- | --- |
| 模型替换 | 用新的模型几何替换部分模型 | 子模型技术 |
| 状态传递 | 传递应力、应变、温度等状态 | 后续分析步骤 |

### 结果传递方法

#### 方法 1：使用 *MODEL CHANGE

`MODEL CHANGE` 选项允许移除或重新激活单元，从而模拟制造工艺。

| **输入文件用法：** | ``` [*MODEL CHANGE](../key/key-link.md#usb-kws-hmodelchange), REMOVE [*MODEL CHANGE](../key/key-link.md#usb-kws-hmodelchange), ADD ``` |

#### 方法 2：使用 *PREDEFINED FIELD

`PREDEFINED FIELD` 选项允许将先前分析的结果作为初始条件。

| **输入文件用法：** | ``` [*PREDEFINED FIELD](../key/key-link.md#usb-kws-mpredefinedfield), TYPE=INITIAL STRESS [*PREDEFINED FIELD](../key/key-link.md#usb-kws-mpredefinedfield), TYPE=INITIAL STATE ``` |

### 结果文件

结果传递需要以下文件：

| **文件** | **内容** |
| --- | --- |
| 结果数据库 (.odb) | 包含完整的结果数据 |
| 重启动文件 (.res) | 包含中间增量数据 |
| 状态文件 (.stt) | 包含模型状态数据 |

### 执行结果传递

使用 `oldjob` 参数指定源分析：

```
abaqus job=new_analysis oldjob=source_analysis
```

### 注意事项

1. **兼容性**：源模型和新模型必须具有兼容性
2. **网格匹配**：节点和单元编号应匹配
3. **单位一致**：两个模型应使用相同的单位制
4. **结果可用性**：所需的增量步必须存在

### 应用示例

#### 焊接分析

1. 首先分析焊接前的零件
2. 将焊接后的残余应力作为初始条件
3. 进行后续服役载荷分析

#### 成形分析

1. 执行冲压成形分析
2. 将成形后的应变状态传递
3. 执行退火步骤
4. 继续后续成形

### 局限性

- 单元类型必须兼容
- 材料模型必须匹配
- 接触状态不能直接传递
- 某些非线性属性可能需要特殊处理

### 输入文件模板

```
[*HEADING](../key/key-link.md#usb-kws-mheading)
…
*从原始分析读取结果*
[*RESTART](../key/key-link.md#usb-kws-mrestart), READ, STEP=1, INC=20
[*STEP](../key/key-link.md#usb-kws-hstep)
[*STATIC](../key/key-link.md#usb-kws-hstatic)
*使用传递的初始条件继续分析*
[*END STEP](../key/key-link.md#usb-kws-hendstep)
```
