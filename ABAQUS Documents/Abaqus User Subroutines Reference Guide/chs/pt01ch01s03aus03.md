# 1.3.1 在Abaqus中定义模型






**产品：** Abaqus/Standard  Abaqus/Explicit  Abaqus/CFD  

##### **参考**

- ["输入语法规则"，第1.2.1节"](pt01ch01s02aus01.md)
- [Abaqus关键词参考指南](../key/key-link.md#key)
- [Abaqus/CAE用户指南](../usi/usi-link.md#usi)

### 概述

Abaqus中的分析由*输入文件*定义，该文件：
- 包含*关键词行*和*数据行*；以及
- 分为*模型*数据和*历史*数据。

### 输入文件

Abaqus输入文件是ASCII数据文件。它可以通过文本编辑器或通过图形预处理程序（如Abaqus/CAE）创建。输入文件由包含Abaqus选项（关键词行）和数据（数据行）的系列行组成。关键词行和数据行的输入语法在["输入语法规则"，第1.2.1节"](pt01ch01s02aus01.md)中描述。

大多数输入文件具有相同的基本结构。输入文件的以下部分被指定用于定义有限元模型：

1. 输入文件通常以[*HEADING](../key/key-link.md#usb-kws-mheading)选项开头，该选项用于定义分析的标题。可以使用任意数量的数据行来给出标题；它们将出现在输出文件的开头（["输出"，第4.1.1节"](pt02ch04s01aus38.md)）。第一标题行将显示在输出每页顶部的标题中。虽然包含标题对检查输入文件的用户有帮助，但[*HEADING](../key/key-link.md#usb-kws-mheading)选项不是必需的。
2. 标题之后，输入文件通常包含*模型*数据部分，用于定义节点、单元、材料、初始条件等。模型数据部分在下面说明。
3. 如果模型被组织成部件实例的装配体，则模型数据进一步分类，必须落在适当的级别：部件、装配体、实例或模型。以部件实例装配体形式定义的模型在["定义装配体"，第2.10.1节"](pt01ch02s10aus28.md)中讨论。
4. 最后，输入文件包含*历史*数据，用于定义分析类型、载荷、输出请求等。步骤定义将输入文件中的模型数据与历史数据分开：第一步定义之前出现的所有内容都是模型数据，第一步定义之内及之后出现的所有内容都是历史数据。历史数据部分在下面说明。

输入文件在执行适当的分析产品Abaqus/Standard、Abaqus/Explicit或Abaqus/CFD之前由"分析输入文件处理器"处理。分析输入文件处理器的功能是解释Abaqus选项、执行必要的完整性检查，并为分析产品准备数据。

大多数计算力学建模选项（单元类型、载荷类型等）在Abaqus/Standard和Abaqus/Explicit中都可用，尽管某些选项仅在一个分析产品中可用。输入文件中使用的所有步骤过程类型必须来自同一分析产品；但是，可以将解决方案从Abaqus/Standard导入到Abaqus/Explicit，反之亦然（参见["导入和传输结果"，第9.2节"](pt04ch09s02.md)），这允许在分析的各个阶段使用最适合每个分析产品（例如，Abaqus/Standard中的静态预载荷，然后是Abaqus/Explicit中的动态分析）。

### 模型数据

模型数据定义节点、单元、材料、初始条件等。

#### 必需模型数据

以下模型数据必须包含在输入文件中以定义有限元模型：

**几何**：模型的几何形状由单元及其节点描述。定义节点和单元的规则和方法在["节点定义"，第2.1.1节"](pt01ch02s01aus05.md)；["单元定义"，第2.2.1节"](pt01ch02s02aus11.md)；和["定义装配体"，第2.10.1节"](pt01ch02s10aus28.md)中描述。结构单元（如梁）的横截面必须定义。特殊特征可以用特殊单元定义，如弹簧、阻尼器、点质量等。可用于建模的单元类型在[第六部分，"单元"](pt06.md)中描述，连同如何定义单元的解释。您可以在数据检查运行后，在Abaqus/CAE的可视化模块中查看初始网格或调整初始过盈后的配置（参见["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行"，第3.2.2节"](pt01ch03s02abx02.md)）。

**材料定义**：材料类型必须与几何的大部分区域相关联。材料库在[第五部分，"材料"](pt05.md)中描述。特殊单元（如弹簧或阻尼器）没有相关联的材料，但必须定义它们的属性。

#### 可选模型数据

以下模型数据可以根据需要包含：

**部件和装配体**：模型的几何形状可以通过将其组织成部件来定义，部件在装配体中相对定位（["定义装配体"，第2.10.1节"](pt01ch02s10aus28.md)）。

**初始条件**：可以指定非零初始条件，如初始应力、温度或速度（["初始条件"，第34.2节"](pt07ch34s02.md)）。

**边界条件**：可以在单个求解变量（如位移或旋转）上施加零值边界条件（包括对称条件）（["Abaqus/Standard和Abaqus/Explicit中的边界条件"，第34.3.1节"](pt07ch34s03aus118.md)）。

**运动约束**：可以定义涉及模型中几个基本求解变量的方程（["线性约束方程"，第35.2.1节"](pt08ch35s02aus129.md)）或多点约束（["一般多点约束"，第35.2.2节"](pt08ch35s02aus130.md)）。

**相互作用**：可以定义部件之间的接触和其他相互作用（["接触相互作用分析：概述"，第36.1.1节"](pt09ch36s01abo33.md)）。

**幅值定义**：可以为以后指定时间相关载荷或边界条件定义幅值曲线（["幅值曲线"，第34.1.2节"](pt07ch34s01aus115.md)）。

**输出控制**：您可以控制模型定义输出到数据文件（["输出"，第4.1.1节"](pt02ch04s01aus38.md)）。

**环境属性**：可能需要定义环境属性，如围绕模型流体的属性。

**分析继续**：可以写入重启数据或使用先前分析的结果，并用新的模型或历史数据（["重新启动分析"，第9.1.1节"](pt04ch09s01aus53.md)）、新网格（["子模型：概述"，第10.2.1节"](pt04ch10s02aus60.md)；["网格到网格解映射"，第12.4.1节"](pt04ch12s04aus86.md)；和["对称模型生成"，第10.4.1节"](pt04ch10s04aus63.md)）或相同或不同的Abaqus程序（["在Abaqus分析之间传输结果：概述"，第9.2.1节"](pt04ch09s02aus54.md)）继续分析。

### 历史数据

分析的目的是预测模型对某种外部载荷或某种非平衡初始条件的响应。Abaqus分析基于*步骤*的概念，步骤在输入文件的历史数据部分描述。（有关步骤的更多信息，请参见["定义分析"，第6.1.2节"](pt03ch06s01abo05.md)。）历史输入数据在步骤内根据需要组合，以定义分析的历史。

可以在分析中定义多个步骤。引入步骤只是为了更改输出请求或更改载荷、边界条件、分析过程等。分析中的步骤数量没有限制。

Abaqus中有两种步骤：一般响应分析步骤，可以是线性或非线性的；以及Abaqus/Standard中的线性扰动步骤（参见["一般和线性扰动过程"，第6.1.3节"](pt03ch06s01aus44.md)）。一般分析步骤对系统的响应历史有贡献；线性扰动步骤允许研究响应历史任何阶段系统的线性化响应。

一般步骤结束时的状态为下一步提供初始条件，使得模拟模型的连续加载变得容易，例如静态预载荷后的动态响应，或制造过程模拟后产品使用期间的加载。

除非被修改，否则以下可选历史数据规定的加载；边界条件；输出控制；辅助控制；以及Abaqus/Explicit中的接触条件从一个一般分析步骤继续到下一个一般分析步骤。例如，在Abaqus/Standard的一般分析步骤中规定求解控制（参见["收敛和时间积分准则：概述"，第7.2.1节"](pt03ch07s02abo11.md)）将保持有效用于所有后续一般分析步骤，直到它们被修改或重置。对于线性扰动步骤，仅当中间没有一般分析步骤且输出控制未重新定义时，输出控制才从一个线性扰动步骤继续到下一个（参见["输出"，第4.1.1节"](pt02ch04s01aus38.md)）。类似地，在Abaqus/CFD分析中规定的条件，除非被修改，否则从一个步骤继续到下一个步骤。

| **输入文件用法：** | 使用以下选项开始步骤定义： |
| --- | --- |
|  | ``` [*STEP](../key/key-link.md#usb-kws-hstep) ``` 使用以下选项结束步骤定义： ``` [*END STEP](../key/key-link.md#usb-kws-hendstep) ``` |

#### 必需历史数据

以下历史数据必须包含在输入文件中以定义分析过程：

**响应类型**：定义分析过程类型的选项必须出现在步骤定义的开头之后。Abaqus可以执行多种类型的分析——线性或非线性、静态或动态等（参见["定义分析"，第6.1.2节"](pt03ch06s01abo05.md)）。分析类型可以随步骤改变。例如，在Abaqus/Standard中，首先可以分析静态预载荷，然后将响应类型更改为瞬态动态。这样，可以基于静态解结束时的条件执行线性或非线性动态分析。

#### 可选历史数据

以下历史数据可以根据需要包含：

**载荷**：通常定义某种形式的外部载荷。例如，可以施加集中载荷或分布载荷（["施加载荷：概述"，第34.4.1节"](pt07ch34s04aus120.md)），可以规定导致热膨胀的温度变化（["热膨胀"，第26.1.2节"](pt05ch26s01abm52.md)），或者可以使用接触条件来施加载荷（["接触相互作用分析：概述"，第36.1.1节"](pt09ch36s01abo33.md)）。载荷可以规定为时间的函数（["幅值曲线"，第34.1.2节"](pt07ch34s01aus115.md)）。此功能可用于规定载荷，如地震事件期间的地面运动、已知加速度，或发动机瞬态期间的温度和压力历史。如果未定义幅值曲线，Abaqus假定载荷在步骤上线性变化，或者根据所选响应类型在步骤开始时瞬时施加载荷（参见["定义分析"，第6.1.2节"](pt03ch06s01abo05.md)）。

**边界条件**：可以在分析期间添加、修改或移除边界条件（["边界条件"，第34.3节"](pt07ch34s03.md)）。

**输出控制**：应力、应变、反作用力、温度和能量等量可以作为输出使用。输出选项在["输出到数据和结果文件"，第4.1.2节"](pt02ch04s01aus39.md)和["输出到输出数据库"，第4.1.3节"](pt02ch04s01aus40.md)中描述；所有输出变量在["输出变量"，第4.2节"](pt02ch04s02.md)中列出。可用的输出文件在["输出"，第4.1.1节"](pt02ch04s01aus38.md)中描述。

**接触**：接触表面和接触相互作用可以作为Abaqus/Explicit分析中的步骤相关历史数据添加、修改或移除（参见["接触相互作用分析：概述"，第36.1.1节"](pt09ch36s01abo33.md)）。

**辅助控制**：您可以覆盖内置在Abaqus中的求解控制。在某些过程中，这些值在过程定义中给出。更一般地说，在Abaqus/Standard中，它们通过定义求解控制来给出（["常用控制参数"，第7.2.2节"](pt03ch07s02aus50.md)）。也可以定义接触问题的求解控制（["在Abaqus/Standard中调整接触控制"，第36.3.6节"](pt09ch36s03aus150.md)；["在Abaqus/Explicit中使用接触对进行接触建模的常见困难"，第39.2.2节"](pt09ch39s02aus186.md)；或["Abaqus/Explicit中一般接触的接触控制"，第36.4.5节"](pt09ch36s04aus159.md)）。

**单元和表面移除/重新激活**：在Abaqus/Standard中，可以随步骤移除或重新激活模型的某些部分。参见["单元和接触对移除与重新激活"，第11.2.1节"](pt04ch11s02aus66.md)。

**协同仿真**：Abaqus模型中的步骤必须定义，使得协同仿真完全包含在单个Abaqus步骤中。此外，Abaqus作业中只能有一个协同仿真。

### 从外部文件包含模型或历史数据

您可以指定包含Abaqus输入文件部分的外部文件。此文件可以包含模型和历史定义数据、注释行以及对其他外部文件的引用。当遇到对外部文件的引用时，Abaqus将立即处理指定文件中的数据。到达文件末尾时，Abaqus将返回处理原始文件。

最多可以使用五级嵌套外部文件引用。UNIX环境变量可用于指定文件名。

| **输入文件用法：** | ``` [*INCLUDE](../key/key-link.md#usb-kws-minclude), INPUT=*file_name* ``` |
| --- | --- |

#### 包含加密数据文件

您可以通过引用在Abaqus输入文件或其他数据文件中包含加密文件。当您引用加密文件时，还必须提供文件的密码。如果密码正确，Abaqus将处理指定文件中的数据，就像处理未加密的外部文件一样。加密输入文件中的材料和连接器行为定义不会写入输出数据库。此外，如果加密文件用作模型任何部分的输入，则抑制输出到数据文件的所有材料和连接器行为定义。有关加密实用程序的详细信息，请参见["加密和解密Abaqus输入数据"，第3.2.38节"](pt01ch03s02abx38.md)。

某些加密文件仅符合条件供拥有特定Abaqus功能许可证（如Abaqus/Explicit）的用户或特定站点的用户包含。如果您尝试包含您没有适当权限的加密文件，Abaqus会发出错误消息。

您不能包含包含参数化输入的加密输入文件。

| **输入文件用法：** | ``` [*INCLUDE](../key/key-link.md#usb-kws-minclude), INPUT=*file_name*, PASSWORD=*password* ``` |
| --- | --- |
