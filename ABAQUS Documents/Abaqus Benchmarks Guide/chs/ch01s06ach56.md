# 1.6.4 无限板的淬火






**产品：** Abaqus/Standard  Abaqus/Explicit

本示例是不耦合热传递和后续热应力分析的说明。有一个案例的半解析解可用（参见 Landau 等人，1960），因此该问题为 Abaqus 中此类分析提供了验证。分析的目的是预测大均匀板在远离板边缘的区域中淬火引起的残余应力，因此可以将其视为除厚度方向外所有方向无限延伸的板。该板由弹性理想塑性材料制成，屈服应力在 121°C (250°F) 以上随温度线性降低。该问题是一维的，因为假定板是无限延伸的：仅在厚度方向存在梯度。板最初处于均匀温度，接近其熔点（此时其屈服应力很小）。假定在此条件下无应力。然后将表面在室温介质中淬火。允许冷却继续，直到板的整体达到室温。

在 Abaqus/Standard 中进行的分析包括顺序热应力分析和完全耦合求解过程。在顺序分析中，瞬态热传递分析之后是热应力分析。在热传递分析期间，温度分布记录在 Abaqus 结果文件中。然后，该温度-时间历史被用作热应力分析的输入。瞬态应力足够大，会引起显著的塑性流动，因此在板达到室温后会留下残余应力。在完全耦合过程中，通过将转化为热量的非弹性耗散分数设置为零来模拟顺序耦合问题。在这个问题中，这会将热响应与机械响应解耦。

在 Abaqus/Explicit 中使用完全耦合求解过程；通过将转化为热量的非弹性耗散分数设置为零，再次模拟上述顺序耦合问题。为了完整起见，在 Abaqus/Explicit 中进行另一个分析，这次使用 [`VUMAT`](../sub/sub-link.md#sub-xsl-vumat) 用户子程序定义材料响应，并假定 0.2 的非弹性耗散分数转化为热量。最后这个分析说明了 [`VUMAT`](../sub/sub-link.md#sub-xsl-vumat) 用户子程序与非弹性热分数、比热容和热导率的结合使用；非弹性能量耗散引起的热通量由 Abaqus/Explicit 自动计算。

### 问题描述

板如图 1.6.4-1](ch01s06ach56.md#sxmquenching-geom)所示。它厚 914.4 mm (36 in)，具有以下属性：

| 弹性模量 | 206.8 GPa (30 × 10^6 lb/in²) |
| --- | --- |
| 泊松比 | 0.3 |
| 屈服应力 | 对于 ![](../graphics/bmk_eqn00729.gif) 121°C 为 248.2 MPa (36000 lb/in²，![](../graphics/bmk_eqn00729.gif) 250°F) |
|  | 248.2(1 – (![](../graphics/bmk_eqn00191.gif) – 121)/111.1) MPa，![](../graphics/bmk_eqn00730.gif) 121°C |
|  | (36000(1 – (![](../graphics/bmk_eqn00191.gif) – 250)/2000) lb/in²，![](../graphics/bmk_eqn00731.gif) 250°F) |
| 密度 | 7832 kg/m³ (0.283 lb/in³) |
| 比热容 | 0.6 kJ/kg°C (0.1431 BTU/lb°F) |
| 热导率 | 58.8 W/m°C (7.872 × 10^-4 BTU/in·sec·°F) |

板表面的膜系数为 193.1 W/m²°C (6.559 × 10^-5 BTU/in²·sec·°F)。

Abaqus/Standard 模拟中使用的有限元网格如图 1.6.4-1](ch01s06ach56.md#sxmquenching-geom)所示。板半厚度使用十个单元。由于问题是一维的，只需要一行单元。未进行网格收敛研究：假定此网格应给出相当准确的结果。这个假设通过与 Landau 等人（1960）描述的结果的一致性得到了确认。对于顺序热应力分析，热传递网格使用 DC2D8（8 节点四边形）和 DC2D4（4 节点四边形）单元类型。对于应力分析，边界条件对应于板表面法线方向所有方向的广义平面应变；也就是说，任何最初垂直于板表面的直线保持直线并垂直于表面，但此类直线之间的距离随板冷却而变化。在顺序热应力分析中，此条件使用四种单元类型建模：轴对称单元 CAX8R 和 CAX4I 以及广义平面应变单元 CPEG4I 和 CPEG8R。为了验证不同网格之间的插值技术，使用 CAX8R 单元的应力分析由使用 DC2D4 单元的分析的温度场驱动。在完全耦合的 Abaqus/Standard 模拟中，使用 CPEG4HT、CPEG8RHT、CAX4T 和 CAX4RT 单元。广义平面应变单元在定义模型厚度方向极限的平面之间预设了零相对旋转。（通过在这些单元的参考节点上引入边界条件来施加此条件。）模型平面中的广义平面应变条件在两个模型中通过在网格左边缘使用对称条件以及使用方程约束来施加网格右边缘所有节点上的相等位移来施加。

在所有二阶模型中使用减缩积分单元。减缩积分是有吸引力的，因为它降低了分析成本，同时提供了更准确的应力预测。当选择二阶单元时，通常建议使用减缩积分。

在 Abaqus/Explicit 模拟中，轴对称板使用 CAX3T 或 CAX4RT 单元建模；对于板假定处于广义平面应变状态的情况，使用 C3D8RT 单元以及适当的约束以确保平面截面保持平面。在每种情况下，板半厚度使用 20 个单元。质量缩放用于降低分析的计算成本。

### 分析序列

Abaqus/Standard 顺序热应力模拟包括瞬态热传递分析，然后是使用热传递分析预测的温度作为问题载荷的热应力分析。Abaqus 使得以这种方式传递温度数据非常简单。在热传递分析中，节点处的温度被写入文件。然后，在应力分析中，这些温度被读回应力模型。这种传递温度的模式基于节点编号：来自热传递分析中节点 N 的温度被应用于应力网格中节点 N。因此，从热传递模型到应力模型，节点编号必须保持相同。Abaqus 不检查节点是否位于相同位置。在某些情况下，非结构组件（如绝缘）只在热传递分析中建模，而不在应力分析中建模。这种情况不会带来问题；如果热传递分析的输出包含不存在于应力分析模型中的节点的温度，这些温度在应力分析中被忽略。

在 Abaqus/Standard 和 Abaqus/Explicit 完全耦合分析中，板的热和机械响应同时确定。

### 控制

以下讨论仅与 Abaqus/Standard 模拟相关。

您可以限制增量中可能发生的最大温度变化，从而确定瞬态温度解在时间上积分的精度。设置此值意味着使用自动时间增量，这在您希望将分析进行到稳态条件的情况下是可取的，因此在解的后期使用大时间增量。在这个例子中，最大温度变化设置为 5.56°C (10°F)。这个选择应该为热传递解提供足够的精度，以正确确定残余应力。

建议初始时间增量为 20 秒，建议时间周期为 4 × 10^6 秒。由于解要达到稳态，时间周期规格相当任意：它必须足够长以达到稳态。设置热传递分析的控制，使得分析应在达到稳态条件时终止。出于此参数的目的，稳态条件定义为所有节点温度的时间变化率低于给定值。在本分析中，此值设置为每分钟 0.556 × 10^-6°C (10^-6°F/秒)。当热传递分析指定在达到稳态时结束分析时，为热传递分析设置控制，使得步在达到稳态条件或完成步指定的时间周期时终止，以先到者为准。因此，在此类情况下通常使用非常大的时间周期。

通常需要指定最小时间增量，以覆盖数据错误或解中意外事件导致自动时间增量方案选择非常小的增量的可能性。在这种情况下，使用 0.5 秒的值。《Abaqus 分析用户指南》第 6.5.2 节 ["不耦合热传递分析"](../usb/usb-link.md#usb-anl-aheattransfer) 建议在温度快速变化时瞬态热传递分析的最小时间增量为

![](../graphics/bmk_eqn00732.gif)

在这个例子中，![](../graphics/bmk_eqn00733.gif) 为 0.9 in，因此此公式建议最小时间增量至少为 6.9 秒。在表面温度突然变化的情况下，小于此值的时间增量可能导致解的初始振荡。然而，本问题的物理学不会产生足够大的温度梯度，从而导致与满足指定最大温度变化的时间增量一起引起此类振荡。

### 结果和讨论

考虑两种情况：初始温度为 1038°C (1900°F)，以及初始温度为 816°C (1500°F)。残余应力如图 1.6.4-2](ch01s06ach56.md#sxmquenching-residuals)所示，并与 Landau 等人（1960）给出的值进行了比较。该图中显示的数值结果基于使用 Abaqus/Standard 获得的解。Abaqus 结果与该参考结果之间的良好一致性验证了此类热应力分析。

表面附近积分点和板中心附近积分点的应力历史如图 1.6.4-3](ch01s06ach56.md#sxmquenching-stresshist)所示。分析早期发生的应力反转在此图中很容易观察到。从该图中也可以清楚地看到 Abaqus/Explicit 和 Abaqus/Standard 获得结果之间的极好一致性。

最后一个 Abaqus/Explicit 分析表明，非弹性热分数可以与 [`VUMAT`](../sub/sub-link.md#sub-xsl-vumat) 用户子程序一起使用，使得在 [`VUMAT`](../sub/sub-link.md#sub-xsl-vumat) 子程序中计算的非弹性耗散在动态完全耦合热应力分析中转化为热量产生。
