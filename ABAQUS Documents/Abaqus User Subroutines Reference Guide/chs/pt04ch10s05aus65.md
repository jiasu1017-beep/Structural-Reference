# 10.5.1 周期介质分析

**产品：** Abaqus/Explicit  

##### **参考文献**

- [*PERIODIC MEDIA](../key/key-link.md#usb-kws-mperiodicmedia)
- [*MEDIA TRANSPORT](../key/key-link.md#usb-kws-mmediatransport)

### 概述

Abaqus/Explicit 中的周期介质分析技术：
- 是一种提供类似欧拉视图到移动结构的拉格朗日技术；
- 可用于有效建模本质上重复的系统，如涉及传送带或连续成型操作的制造过程；
- 与可能需要过大网格的传统建模技术相比，显著加快了分析时间；以及
- 需要拓扑相同的网格化部件来创建模型，这可以通过部件和实例建模范式完成。

### 简介

通常，需要分析工业过程涉及以简单模式重复并通过处理区的部分。一个突出的例子是具有规则间隔包裹的传送带，如图 10.5.1-1 示意性所示，有限元网格实例如图 10.5.1-2 所示。连续成型操作（如金属轧制）也是很好的例子，因为变形材料可以分解为任意数量的相同部分。

为清晰起见，我们将在整个讨论中使用传送带示例来说明与周期介质分析技术相关的许多概念。图 10.5.1-1 显示了传送带的概念分解；实际上，传送带是一个连续实体。

从概念上讲，整体模型可以分解为连接在一起并跨越处理区的块（拓扑相同的网格化结构）。您创建一个定义"构建块"的部件（重复用于建模整个周期介质的网格化结构），然后通过适当定位的实例链构建整个模型。周期介质分析技术提供了一种简单的方法来自动连接这些实例在相邻块的前端和后端。该技术还提供了一种方便的方式来定义载荷和边界条件，这些载荷和边界条件表示链中第一个和最后一个块的未连接端上的物理系统。链的第一个块称为入口，最后一个块称为出口。最后，当周期介质通过处理区移动时，出口的块自动重新排序到入口。使用此技术定义的块（网格化结构）可以通过接触与本质上非周期性的其他建模特征进行相互作用，例如图 10.5.1-1 中描述的滚筒。

周期介质分析技术的核心在于将块从出口重新排序到入口的概念。使用专用算法检测入口何时移动到处理区过远处，并将块从出口直接重新排序到入口。图 10.5.1-1 中的虚线箭头说明了重新排序过程。为确保平滑过渡，在当前步骤开始时存储入口块的必要节点和单元状态数据。当重新排序发生时，存储的节点和单元状态数据被映射到新入口块，任何入口/出口载荷或边界条件被转移到新暴露的块端。

因此，周期介质分析技术为移动重复结构提供了一种方便的类欧拉视图。例如，您可能有兴趣在稳态和瞬态条件下评估滚筒之间某处传送带上包裹的动力学。您在该位置周围定义几个块，必要时定义与滚筒的接触，并提供适当的入口和出口载荷条件。周期介质分析技术提供了一种方便且经济的方式来创建和分析此系统。通过重新使用通过此重新排序过程离开处理区的单元，您可以避免纯拉格朗日模拟所需的入口端的大网格。

### 构建周期介质模型

构建周期介质模型的第一步是识别构成重复结构构建块的模型部分。在图 10.5.1-2 中，一个方形传送带贴片及其顶部的一个不对称形状包裹构成了这样的构建块。如果将几个块串在一起，则可以建模整个传送带和包裹。

#### 定义构建块

定义每个构建块时必须遵守以下要求：
- 必须定义一个未排序单元集以包含构建块中的所有单元，以及
- 必须定义一个未排序节点集以包含构建块中的所有节点。

为确保周期介质推进时信息正确传递，这些未排序集在所有块之间必须拓扑相同。满足此要求的最简单方法是使用部件和实例建模范式。您定义一个对应于构建块的部件，并如上所述定义未排序的单元和节点集。然后，根据需要使用适当的平移和旋转实例化该部件多次，以生成周期介质网格。允许在构建块内使用绑定、耦合和刚体等约束。您必须确保这些约束在所有块中以拓扑相同的方式定义。

周期介质分析技术将这些否则未连接的块连接在一起来创建连续模型。如果在块的连接区域中使用结构单元（如壳），则这些区域边缘上的节点连接到相邻区域。如果使用连续体单元，则这些区域表面上的节点被连接。为了可靠地构建这些约束，必须遵守以下附加要求：
- 块的前后连接端的节点排列必须拓扑相同，
- 相邻块的前后端节点必须重合，
- 初始入口块前后端的节点排列必须仅通过刚体平移而有所不同，以及
- 必须在每个块的前后端使用未排序节点集创建两个基于节点的表面。

基于节点的表面用于在相邻块之间自动生成节点到节点绑定约束，使整个装配体表现为连续实体。

| **输入文件用法：** | 使用以下选项使用上述未排序集和表面定义块序列： |
| --- | --- |
|  | ``` [*PERIODIC MEDIA](../key/key-link.md#usb-kws-mperiodicmedia), NAME=*name* *elset1*, *nodeset1*, *frontsurf1*, *backsurf1* *elset2*, *nodeset2*, *frontsurf2*, *backsurf2* *...* *elsetn*, *nodesetn*, *frontsurfn*, *backsurfn* ``` 每个数据行提供与给定块关联的集和表面名称。 |

#### 在介质端施加载荷和边界条件

在图 10.5.1-1 中所示的示意传送带中，您通常需要在装配体的两端施加载荷或边界条件。在入口点 I 处，施加保持传送带张紧的预张力通常很有用，而在出口点 O 处，传送带速度通常是规定的。随着传送带推进并从出口重新排序到入口的退出块，需要边界条件的节点将发生变化。因此，这些边界条件和载荷不能直接施加在属于块的节点上。

周期介质分析技术允许通过与当前入口和出口基于节点的表面关联的两个控制节点来施加此类加载特征。控制节点类似于其他特征（如运动耦合）中使用的参考节点，并对装配体极端端的节点施加自动定义的类似刚体约束。您可以在这些控制节点上施加载荷和边界条件。类似刚体约束也施加在入口块的前端节点上，但不能施加载荷或边界条件。当退出块被重新排序回入口时，控制点将在新的极端端表面上强制执行这些类似刚体约束，并从先前位置移除类似刚体约束。该过程是自动的，由周期介质分析技术完全管理。

| **输入文件用法：** | 使用以下选项定义入口和出口条件的控制节点： |
| --- | --- |
|  | ``` [*PERIODIC MEDIA](../key/key-link.md#usb-kws-mperiodicmedia), INLET CONTROL NODE=*node*, OUTLET CONTROL NODE=*node* ``` |

#### 定义处理区

当入口块完全进入处理区时，出口块被重新排序回入口，如图 10.5.1-1 中的虚线箭头所示。触发平面控制重新排序发生的确切时间。当位于当前入口点 I 处的节点穿过触发平面时，启动重新排序过程。触发平面使用（通常）静止节点的坐标和用户定义方向的 *z* 轴来定义。局部 *z* 轴方向从入口指向处理区。

| **输入文件用法：** | 使用以下选项通过触发节点和方向定义触发平面： |
| --- | --- |
|  | ``` [*PERIODIC MEDIA](../key/key-link.md#usb-kws-mperiodicmedia), TRIGGER NODE=*node*, ORIENTATION=*orientation* ``` |

### 激活周期介质

重新排序过程可以逐步激活。默认情况下，重新排序过程处于非活动状态。在许多情况下，周期介质在操作条件下的配置只能通过模拟确定。这允许在激活重新排序过程之前执行任意数量的分析步骤。

图 10.5.1-2 和["介质传输，" Abaqus 验证指南第 3.25.1 节](../ver/ver-link.md#ver-prc-mediatransport) 中所示的示例展示了传送带运输初始放置在规则间隔的不对称包裹。在其操作条件下，传送带将被张紧。您可以在 Abaqus/Standard 或 Abaqus/Explicit 中预拉伸传送带装配体。如果预拉伸分析在 Abaqus/Standard 中进行，则需要明确定义相邻块之间的所有绑定以及入口和出口端节点处的边界条件，因为周期介质分析技术仅在 Abaqus/Explicit 中可用。如果预拉伸步骤在 Abaqus/Explicit 中进行，则重新排序过程应在预拉伸步骤期间保持非活动状态。

| **输入文件用法：** | 使用以下选项激活或停用周期介质重新排序过程： |
| --- | --- |
|  | ``` [*MEDIA TRANSPORT](../key/key-link.md#usb-kws-mmediatransport) *periodic_media_name1*, ACTIVE *periodic_media_name2*, INACTIVE ... ``` |

### 建模提示

周期介质分析技术是一个强大的功能；但是，您必须在使用时运用良好的工程判断。以下评论和建议将帮助您避免使用此技术时的常见陷阱：
- 块重新排序过程固有地嘈杂，因为单元块在一端分离并在另一端重新连接。尽管该过程使用适当的材料和运动状态，但小冲击是该过程固有的。建议使用少量与质量成比例的阻尼来抑制此激励。
- 入口控制节点处边界条件和在处理区中施加的任何载荷的组合应确保入口块穿过触发平面而不改变方向。在传送带示例中，良好的建模实践是将固定导向滚筒放置在距离触发平面至少两个块远的地方。
- 对于更复杂的几何形状（如在滚筒之间改变方向的传送带或当传送带本身就是缠绕材料时的包裹缠绕分析），可能需要从直列块开始，然后将传送带滚筒（不是周期介质定义的一部分）移动到所需位置。传送带和滚筒之间的接触相互作用会使传送带变形为所需配置。此附加分析步骤可以极大地简化初始网格的定义。
- 有时可能需要模拟引导段缠绕通过滚筒的过程，就像制造过程开始时的物理现实一样。如果此引导段后面是包含实际包裹的周期块，则可以将周期介质网格附加到常规网格以执行穿线。然后可以将网格的周期介质部分导入到没有引导网格的单独模型中，并且可以执行仅由包装和包裹组成的周期介质的分析。

### 初始条件

可以在周期介质网格的所有节点上指定初始条件。速度初始边界条件可用于最小化达到稳态操作条件所需的求解时间。如果需要预拉伸，则导入自先前分析而不是执行多步骤分析允许将初始条件应用于拉伸配置。由于周期介质定义不会被导入，因此必须在每个需要的分析中重新指定它们。

### 边界条件

入口和出口控制节点是与之关联的周期介质定义中唯一可以指定边界条件的两个节点。此外，只允许速度边界条件。您不得在与周期介质网格关联的任何其他节点上指定边界条件。当周期介质处于活动状态时，如果寻求稳态解，这些边界条件在方向和大小上应保持恒定，以减轻求解噪声。

### 载荷

只能将集中载荷施加到入口和出口控制节点上，以驱动或拉伸周期介质。当周期介质处于活动状态时，这些载荷在方向和大小上应保持恒定。可以根据需要施加重力载荷。也可以指定其他分布载荷；但是，您必须记住，载荷将随块一起移动。

### 材料选项

支持所有可用的材料模型。

### 限制

周期介质分析受以下限制：
- 块内只允许膜、壳、桁架、连续体单元和刚性单元。如果适用，也可以使用钢筋层。
- 不允许在不同块所属的节点之间定义显式约束。
- 质量缩放必须以相同的方式为所有块定义。

周期介质不应涉及
- 定义热接触特性或耦合欧拉-拉格朗日接触的一般接触，或
- 通过接触对算法定义的接触。
