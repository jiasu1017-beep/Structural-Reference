### 概述

Abaqus/Standard中一般接触的接触初始化控制：
- 可用于指定初始过闭合是否应在不产生应力和应变的情况下解决，或作为在多个增量中逐渐解决的干涉配合处理；和
- 可用于指定在无应变调整或干涉配合情况下确定受影响节点的替代搜索区域。

Abaqus/Standard基于初始几何中观察到的间隙或穿透状态初始化接触状态。默认情况下，小的初始接触过闭合通过无应变调整表面节点位置来消除。您可以定义替代接触初始化方法，然后将它们分配给接触相互作用。例如，您可以选择将某些相互作用的初始过闭合作为干涉配合处理。

### 默认接触初始化方法

默认情况下，一般接触算法在预处理期间调整表面节点的初始位置，以消除小的初始表面过闭合，而不在模型中产生应变或应力，如图36.2.4-1所示。这些调整旨在仅校正与网格生成相关的小的不匹配。

**图36.2.4-1** 为解决过闭合而进行无应变调整后的接触表面配置。

![](../graphics/aadjustsurf-zero.png)

一般接触自动为接触相互作用分配主和从角色，如["Abaqus/Standard中一般接触的数值控制，" 第36.2.6节"](pt09ch36s02aus144.md)中所讨论。Abaqus/Standard根据从表面底层单元片的大小计算过闭合容差。如果两个表面在初始时过闭合距离小于计算容差，Abaqus/Standard会将从表面上的这些部分重新定位到相关主表面上（使用无应变调整）。表面之间的初始间隙默认调整保持不变。如果从表面的部分初始过闭合距离大于计算容差，Abaqus/Standard自动为该表面部分及其相关主表面生成接触排除。因此，一般接触不会在模型初始配置中严重过闭合的表面（或表面的部分）之间创建相互作用，这些表面可以在整个分析中自由穿透。

一般接触使用有限滑动、表面-表面接触公式，因此穿透/间隙计算作为有限区域的平均值计算；因此，在调整后，个别表面节点上可能存在穿透和间隙。默认调整不会解决与壳或膜的两个参考表面之间的初始交叉，尽管讨论了解决此类情况的技术（见["为壳表面分配接触初始化"](pt09ch36s02aus142.md#usb-cni-agenlcontinitializationstd-shells)")。

### 定义替代接触初始化方法

如果不需要默认行为，您可以定义替代接触初始化方法。例如，您可能希望增加深度穿透的容差，或指定某些开口应调整为"刚好接触"状态。此外，某些分析要求将初始过闭合作为干涉配合处理，而不是用无应变调整解决。要修改接触初始化行为，您必须定义一个或多个替代接触初始化方法，然后识别哪些表面对将使用哪些方法。

您为每个接触初始化方法分配一个名称。此名称用于将接触初始化方法分配给特定表面对（见下面的["分配接触初始化方法"](pt09ch36s02aus142.md#usb-cni-agenlcontinitializationstd-assign)")。

#### 增加无应变调整的搜索区域

如上["默认接触初始化方法"](pt09ch36s02aus142.md#usb-cni-agenlcontinitializationstd-default)"中所讨论，表面之间的初始间隙和大的初始过闭合不被默认接触初始化方法调整。您可以选择指定相互作用表面上方和下方的非默认搜索距离；位于这些搜索距离内的从表面使用无应变节点调整直接重新定位到其相关主表面上。Abaqus/Standard在计算这些搜索距离时考虑壳厚度。

指定表面上方的搜索距离用于封闭表面之间的小初始间隙。指定表面下方的搜索距离用于增加Abaqus/Standard在执行无应变调整时使用的默认过闭合容差；如果您指定的搜索距离小于默认过闭合容差，Abaqus/Standard使用默认容差代替。与默认初始化行为一样，为大于指定搜索区域的初始过闭合创建接触排除。

增加无应变调整的搜索区域可能会增加分析的计算成本。通常不建议您指定大搜索区域，因为这可能在节点被重新定位大距离时导致网格变形。

| **输入文件用法：** | ``` [*CONTACT INITIALIZATION DATA*](../key/key-link.md#usb-kws-hcontactinitdata), SEARCH ABOVE=*a*, SEARCH BELOW=*b* ``` |
| --- | --- |

| **Abaqus/CAE用法：** | 相互作用模块：****相互作用****![](../graphics/images/arrow.gif)****接触初始化****![](../graphics/images/arrow.gif)****创建****：**用无应变调整解决**：**忽略大于**的过闭合**：*b*，**忽略大于**的初始开口**：*a* |
| --- | --- |

#### 指定初始间隙距离

默认情况下，上述无应变调整将调整初始节点位置，使表面"刚好接触"（零穿透/分离）。或者，Abaqus/Standard可以进行调整以达到您指定的初始间隙距离。调整仅对满足上述搜索区域容差的区域发生。如果需要大的无应变调整以达到指定的初始间隙距离，可能会发生网格变形。

| **输入文件用法：** | ``` [*CONTACT INITIALIZATION DATA*](../key/key-link.md#usb-kws-hcontactinitdata), INITIAL CLEARANCE=*h* ``` |

#### 建模干涉配合

可选地，Abaqus/Standard中的一般接触算法可以将初始过闭合作为干涉配合处理。一般接触算法使用 shrink-fit 方法在分析的第一步（如果第一步使用多个载荷增量）中逐渐解决干涉距离（如图36.2.4-2所示），使得截至并包括特定增量已解决的干涉比例大约对应于完成的步比例。当干涉被解决时会产生应力和应变。与总是在第一个增量中解决完整干涉（这是接触对的默认行为）相比，在"干涉配合载荷"发生非线性响应的情况下，逐渐解决干涉会提高稳健性。通常建议您在干涉配合解决期间不施加其他载荷。

**图36.2.4-2** 接触干涉配合的逐渐解决。

![](../graphics/acontact-interfer-fit-nls.png)

因为在Abaqus/Standard中一般接触使用的表面-表面接触公式在每个约束位置周围的区域中以平均值施加接触条件，当表面-表面约束处于零穿透状态时，可能在从节点处观察到穿透或间隙。

| **输入文件用法：** | ``` [*CONTACT INITIALIZATION DATA*](../key/key-link.md#usb-kws-hcontactinitdata), INTERFERENCE FIT ``` |

##### 增加干涉配合的容差

Abaqus/Standard根据从表面底层单元片的大小计算过闭合容差（见上面的["默认接触初始化方法"](pt09ch36s02aus142.md#usb-cni-agenlcontinitializationstd-default)")。两个表面之间的干涉配合仅影响那些过闭合距离小于计算容差的从表面；对于过闭合距离大于计算容差的表面，接触完全被忽略。

可选地，您可以重新定义过闭合容差，以在干涉配合中包含更大的过闭合。如果您指定的容差小于默认计算的容差，Abaqus/Standard使用默认计算的容差代替。

##### 指定干涉距离

默认情况下，干涉距离由网格的初始过闭合暗示；或者，您可以指定干涉距离。在这种情况下，Abaqus/Standard首先进行节点位置的无应变调整，使得调整配置中的初始过闭合对应于指定的干涉距离，然后调用上面讨论的 shrink fit 方法（如图36.2.4-3所示）。如果需要大的无应变调整以达到指定的干涉距离，可能会发生网格变形。

**图36.2.4-3** 处理与原始网格暗示的干涉不同的指定干涉距离。

![](../graphics/acontact-interfer-fit2-nls.png)

##### 在解决干涉配合时停用摩擦

摩擦模型的存在会降低解决干涉配合的稳健性。通常建议您在Abaqus/Standard解决干涉配合时临时停用摩擦模型。您可以在第一步中停用摩擦模型，而干涉配合正在被解决（使用["在Abaqus/Standard分析期间更改摩擦属性" in "摩擦行为，" 第37.1.5节"](pt09ch37s01aus169.md#usb-cni-afriction-change-std)讨论的"更改摩擦"方法）。

##### 首选使用接触对解决干涉配合的情况

大的干涉可能难以用有限滑动、表面-表面公式解决。使用此公式，过闭合倾向于沿从表面法向方向解决；使用节点-表面公式（仅在接触对算法中可用），过闭合倾向于沿主表面法向方向解决。图36.2.4-4说明了一个示例，其中不同的法向方向在干涉配合期间导致不期望的切向运动。在某些情况下，可能更倾向于使用接触对算法的节点-表面离散化来解决大的初始过闭合（见["在Abaqus/Standard中建模接触干涉配合，" 第36.3.4节"](pt09ch36s03aus148.md))。

### 分配接触初始化方法

您可以将接触初始化方法分配给选定的表面对。

用于接触初始化方法分配的表面名称不必与用于指定一般接触域的表面名称对应。在许多情况下，非默认接触初始化方法将被分配给整体一般接触域的子集。落在一般接触域之外的区域的任何接触初始化分配将被忽略。如果指定相互作用重叠，最后分配优先。

| **输入文件用法：** | 使用以下选项分配接触初始化方法： |
| --- | --- |
| | ``` [*CONTACT INITIALIZATION ASSIGNMENT*](../key/key-link.md#usb-kws-hcontinitassign) *表面_1*, *表面_2*, *接触初始化方法名称* ``` 此选项必须与[*CONTACT*](../key/key-link.md#usb-kws-hcontact)选项结合使用。数据行可以根据需要重复，以将接触初始化方法分配给不同区域。如果省略第一个表面名称，则假定为包含整个一般接触域的默认表面。如果省略第二个表面名称或与第一个表面名称相同，则假定为第一个表面与其自身的接触。请记住，表面可以定义为跨越多个未连接的物体，因此自接触不限于单个物体与自身的接触。如果省略接触初始化方法名称，则假定为Abaqus/Standard中的默认接触初始化方法。如果指定了接触初始化方法名称，它也必须作为输入文件模型部分中[*CONTACT INITIALIZATION DATA*](../key/key-link.md#usb-kws-hcontactinitdata)选项上NAME参数的值出现。 |

#### 为壳表面分配接触初始化

接触初始化分配中的表面可以是单侧或双侧的。单侧表面必须对相邻面具有一致的表面法向方向。如果使用双侧表面进行接触初始化分配，无应变调整不会将表面节点移动到相对表面的参考表面之外。

在壳或膜的参考表面初始交叉或初始处于彼此错误侧的情况下，在接触初始化分配中使用单侧表面可以更好地控制接触初始化。[图36.2.4-5](pt09ch36s02aus142.md#rnb-chpinter-sfadoublesided)显示壳表面附近段的调整示例。对于左侧所示的情况，假定在接触初始化分配中使用具有彼此指向的法向方向的单侧表面。在这种情况下，节点在无应变调整期间穿过相对的参考表面移动。

对于[图36.2.4-5](pt09ch36s02aus142.md#rnb-chpinter-sfadoublesided)右侧所示的情况，假定在接触初始化分配中使用具有彼此指向的法向方向的单侧表面。在这种情况下，在单侧表面之间观察到初始间隙（如果接触初始化分配中使用双侧表面也是如此）。默认情况下，不会对此类开口进行无应变调整；但是，如果指定了非默认接触初始化方法，且初始开口搜索容差设置为超过初始分离距离的值，则无应变调整将封闭间隙（如图所示，而不将节点移动到相对参考表面之外）。

#### 示例

以下接触初始化分配在一般接触分析中作为模型数据指定：
- 将`shrink_fit`全局分配给整个一般接触域；
- 将`shrink_fit_local`局部分配给`表面_A`和`表面_B`之间的接触——明确指定搜索区域以增加默认过闭合容差；
- 将默认Abaqus接触初始化方法局部分配给`表面_C`和`表面_D`之间的接触；和
- 通过在数据行中指定每个表面的一侧`表面_1_TOP`和`表面_2_BOTTOM`（见[图36.2.4-5](pt09ch36s02aus142.md#rnb-chpinter-sfadoublesided)左下角），将`sfa_pickside`局部分配给双侧表面`表面_1`和`表面_2`之间的接触。**图36.2.4-5** 单侧壳表面接触初始化期间的无应变调整。![](../graphics/double-sided-SFA-nls.png)
