# 9.3.1 什么是网络ODB连接器？





网络ODB连接器创建与远程计算机的连接，并允许您访问远程输出数据库。例如，您可以将分析提交到高性能UNIX系统，并在分析仍在运行时在本地Windows工作站上查看结果。

您可以从任何平台（Windows、UNIX或Linux）创建网络ODB连接器。但是，网络ODB服务器必须位于UNIX或Linux平台上；您无法访问位于远程Windows系统上的输出数据库。您只能访问远程输出数据库；无法访问远程模型数据库。

从主菜单栏选择****文件（File）**![](../graphics/images/arrow.gif)**网络ODB连接器（Network ODB Connector）**![](../graphics/images/arrow.gif)**创建（Create）****来创建与远程主机上目录的连接。当您创建网络ODB连接器时，您可以使用Abaqus/CAE自动启动网络ODB服务器并在主机和远程系统上建立通信端口号。或者，您可以使用**abaqus networkDBConnector**执行过程从命令行手动启动网络ODB服务器。如果您从命令行启动服务器，则在随后创建网络ODB连接器时输入执行过程返回的通信端口号。有关更多信息，请参阅["网络输出数据库文件连接器，"Abaqus Analysis User's Guide第3.2.24节"](../usb/usb-link.md#usb-int-dremoteodbproc)。

创建网络ODB连接器后，必须通过从主菜单栏选择****文件（File）**![](../graphics/images/arrow.gif)**网络ODB连接器（Network ODB Connector）**![](../graphics/images/arrow.gif)**启动（Start）**![](../graphics/images/arrow.gif)***连接器名称*****来启动它。远程系统上必须安装Abaqus才能使Abaqus/CAE建立网络连接。有关更多信息，请参阅["创建网络ODB连接器，"第9.7.4节"](pt02ch09s06hlb04.md)和["管理网络ODB连接器，"第9.7.6节"](pt02ch09s06hlb06.md)。

创建并启动网络连接器后，您可以使用它来导航远程主机上的目录结构。当您从主菜单栏选择****文件（File）**![](../graphics/images/arrow.gif)**打开（Open）**从Abaqus/CAE打开数据库时，**目录（Directory）**下的文件选择对话框中会出现**网络连接器（Network connectors）**条目。无论您是尝试打开输出数据库还是模型数据库，该条目都会出现；但是，在打开模型数据库时不能使用网络连接器。有关更多信息，请参阅["使用文件选择对话框，"第3.2.10节"](pt01ch03s02s10.md)。

网络连接器允许您执行以下操作：
- 以只读模式打开远程输出数据库，并使用可视化模块查看输出数据库的内容。有关更多信息，请参阅["打开模型数据库或输出数据库，"第9.7.2节"](pt02ch09s06hlb02.md)。当输出数据库是远程的时，可视化模块的行为不会改变；例如，您可以在远程机器上运行分析时查看输出数据库，并且多个用户可以查看输出数据库。但是，您不能单击**作业管理器（Job Manager）**中的**结果（Results）**来打开与远程分析关联的远程输出数据库。
- 从远程输出数据库导入零件。有关更多信息，请参阅["导入零件，"第10.7.2节"](pt02ch10s07hlb02.md)。
- 从远程输出数据库导入模型。有关更多信息，请参阅["从输出数据库导入模型，"第10.5.3节"](pt02ch10s05s03.md)。
- 升级远程输出数据库。

在大多数可视化模块操作和动画期间，Abaqus/CAE会监控输出数据库以获取更新结果并相应地更新当前视口。如果您正在显示来自远程输出数据库的数据，那么如果通过网络监控数据库所花费的时间很长，Abaqus/CAE的性能可能会下降。要提高性能，您可以减少Abaqus/CAE监控输出数据库更新的频率，或者禁用监控。有关更多信息，请参阅["控制结果缓存，"第42.6.10节"](pt05ch42s04hlb06.md)。
