### 概述

Abaqus/Explicit中通用接触算法使用的接触公式包括：
- 接触曲面权重、曲面极性和滑动公式；和
- 可以有选择性地应用于通用接触域内的特定区域。

通用接触公式使用惩罚方法在曲面之间强制执行接触约束；约束强制执行方法在["Abaqus/Explicit中的接触约束强制执行方法，" 第38.2.3节"](pt09ch38s02aus182.md)中讨论。

### 指定接触公式

目前，您可以为通用接触算法仅指定接触曲面权重和极性。接触公式在通用接触相互作用处于活动状态的所有分析步骤中传播。

用于指定应分配非默认接触公式的区域的曲面名称不必与用于指定通用接触域的曲面名称相对应。在许多情况下，接触相互作用将为一个大域定义，而非默认接触公式将分配给该域的子集。对于落在通用接触域之外的区域的任何接触公式分配都将被忽略。如果指定区域重叠，则以最后一次分配为准。

### 接触曲面权重

一般来说，有限元模型中的接触约束以离散方式应用，这意味着对于硬接触，一个曲面上的节点被约束不得穿透另一个曲面。在纯主从接触中，具有约束的节点是从曲面的一部分，与之相互作用的曲面称为主曲面。对于平衡主从接触，Abaqus/Explicit以惩罚力的形式对每组接触曲面计算两次接触约束：一次以第一个曲面作为主曲面，一次以第二个曲面作为主曲面。两次校正（或力）的加权平均值应用于接触相互作用。

平衡主从接触最小化了接触体的穿透，因此在大多数情况下提供了更好的接触约束强制执行和更准确的结果。在纯主从接触中，主曲面上的节点原则上可以不受阻碍地穿透从曲面（参见[图38.2.1-1](pt09ch38s02aus180.md#agencontact-mast-surf-pen)）。

**图38.2.1-1** 由于粗离散化，纯主从接触中主曲面对从曲面的穿透。

![](../graphics/acontact-mast-surf-pen-nls.png)

Abaqus/Explicit中的通用接触算法在可能的情况下始终使用平衡主从权重；对于涉及基于节点的曲面的接触相互作用（只能作为纯从曲面）和涉及分析刚性曲面的接触相互作用（只能作为纯主曲面），使用纯主从权重。基于曲面的内聚行为也始终使用纯主从算法。但是，您也可以选择为其他相互作用指定纯主从权重。

边缘到边缘接触没有主从关系；两个接触边缘被给予相等的权重。

#### 为节点到面接触指定纯主从权重

您可以指定通用接触相互作用应使用纯主从权重进行节点到面接触。此指定对边缘到边缘接触没有影响，不能用于使基于节点的曲面作为主曲面。当两个原本平的曲面相互接触时，如果较精细的曲面作为从曲面，与平衡主从权重相比，纯主从权重可能产生更均匀的穿透距离分布（从而产生更均匀的压力分布）。如果接触曲面的网格密度差异显著，这一点会特别明显——使用平衡权重时，穿透在粗网格曲面节点附近会较小。

Abaqus/Explicit将自动为与指定相反的主从方向生成接触排除；因此，对于两个曲面的重叠区域，将排除节点到面的自接触。例如，指定`surf_A`和`surf_B`之间的通用接触相互作用应使用纯主从权重，`surf_A`被视为从曲面，将内部生成`surf_A`的面接触`surf_B`节点的排除；`surf_A`和`surf_B`重叠区域的节点到面自接触将被排除。如果省略第二个曲面名称或第二个曲面名称与第一个曲面名称相同，则会发出警告消息，因为此输入将导致曲面节点到面自接触的排除。

| **输入文件用法：** | 使用以下选项指示第一个曲面应被视为从曲面（默认）： |

### 接触曲面极性

默认情况下，通用接触考虑指定包含用于接触目的的所有双侧元素的两侧（双侧元素的侧面标签被忽略）。可以为节点到面和欧拉-拉格朗日接触覆盖此默认值，在某些情况下会产生更准确的接触强制执行。

对于边缘到边缘接触不考虑曲面极性，包括在实体元素面上激活的边缘。

#### 为节点到面和欧拉-拉格朗日接触指定曲面极性

更改双侧元素的极性会迫使接触算法将它们视为实体元素。如果在节点到面接触中从节点有可能"被困"在曲面后面，或者如果双侧曲面的一个侧面包含的材料泄漏到另一侧（在欧拉-拉格朗日接触中），将双侧元素转换为单侧可以获得更高的准确性。如果双侧拉格朗日曲面转换为单侧以与所有欧拉材料曲面接触，则欧拉-拉格朗日接触的性能和内存使用也可能得到改善。

| **输入文件用法：** | 使用以下选项指示应在第二个曲面的定义中指定的（双侧）元素的侧面应考虑与第一个曲面的接触： |
| --- | --- |
| | ``` [*CONTACT FORMULATION*](../key/key-link.md#usb-kws-hcontformulation), TYPE=POLARITY *surf_1*, *surf_2* ``` 使用以下选项指示应考虑第二个曲面中（双侧）元素的SPOS侧与第一个曲面的接触： ``` [*CONTACT FORMULATION*](../key/key-link.md#usb-kws-hcontformulation), TYPE=POLARITY *surf_1*, *surf_2*, SPOS ``` 使用以下选项指示应考虑第二个曲面中（双侧）元素的SNEG侧与第一个曲面的接触： ``` [*CONTACT FORMULATION*](../key/key-link.md#usb-kws-hcontformulation), TYPE=POLARITY *surf_1*, *surf_2*, SNEG ``` 使用以下选项指示应考虑第二个曲面中（双侧）元素的两个侧面与第一个曲面的接触： ``` [*CONTACT FORMULATION*](../key/key-link.md#usb-kws-hcontformulation), TYPE=POLARITY *surf_1*, *surf_2*, TWO SIDED ``` 如果省略第一个曲面名称，则假定为涵盖整个通用接触域的默认曲面。必须指定第二个曲面名称。 |

### 滑动公式

目前，Abaqus/Explicit中的通用接触仅提供有限滑动公式。此公式允许接触曲面任意分离、滑动和旋转。对于首选小滑动或无穷小滑动假设的情况，应使用接触对算法（参见["Abaqus/Explicit中接触对的接触公式，" 第38.2.2节"](pt09ch38s02aus181.md)）。

Abaqus/Explicit旨在模拟高度非线性事件或过程。因为一个曲面上的节点可能接触对面上的任何网格面，所以Abaqus/Explicit必须使用复杂的搜索算法来跟踪曲面的运动。有限滑动接触搜索算法设计为稳健的，但在计算上是高效的。此算法假设曲面之间的增量相对切向运动不会显著超过主曲面网格面的尺寸，但对曲面之间的整体相对运动没有限制。由于显式动态分析中使用的小时间增量，增量运动超过网格面尺寸的情况很少见。在涉及超过材料波速的相对曲面速度的情况下，可能需要减小时间增量。

接触搜索算法在首次引入接触相互作用时使用全局搜索，此后使用分层全局/局部搜索算法。不需要用户控制搜索算法。
