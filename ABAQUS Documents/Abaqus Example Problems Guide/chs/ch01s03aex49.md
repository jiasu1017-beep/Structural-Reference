# 1.3.18 使用 Abaqus/Standard 和 Abaqus/CAE 进行惯性焊接仿真





**产品：**Abaqus/Standard  Abaqus/CAE  

### 目标

本示例演示以下 Abaqus 功能：
- 惯性焊接仿真的热力耦合，
- 使用 Python 脚本和输出数据库脚本方法提取变形构型的半自动重网格划分，
- 在用户子程序中定义复杂的摩擦定律，
- 通过用户子程序定义进行飞轮加载，以及
- 组合和呈现一系列输出数据库（`.odb`）文件的结果。

### 应用描述

本示例研究了如图 [1.3.18--1](ch01s03aex49.md#inertiaweld-configuration) 所示的管道惯性摩擦焊接过程。考虑的具体布置是如图 [1.3.18--2](ch01s03aex49.md#inertiaweld-3d) 所示的焊后配置。在该焊接过程中，动能通过摩擦界面迅速转化为热能。界面温度的快速升高被利用来产生高质量的焊接接头。本示例模拟了焊接过程，观察了初始温升和材料塑性流动。工艺设计中的一个重要因素是控制飞轮的初始速度，使得当飞轮停止时，温度升高到刚好低于熔点，从而导致焊接接头区域材料的显著流动。理解摩擦、材料属性和传热环境是有效惯性焊接工艺的重要设计方面；因此，仿真是工艺设计中的有用工具。

### 几何结构

本示例中的焊接过程如图 [1.3.18--1](ch01s03aex49.md#inertiaweld-configuration) 所示，其中两根管道定位进行环缝焊接。两根管道相同，每根长度为 21.0 mm，内半径为 42.0 mm，外半径为 48.0 mm。管道相邻，在预定焊接界面处初始接触。

### 材料

管道由 Astroloy 制成，这是一种用于燃气轮机部件的高强度合金。图 [1.3.18--3](ch01s03aex49.md#stress-curves) 显示了流动应力曲线随温度和塑性应变率的变化。在与焊接工艺相关的温度下，材料对塑性应变率和温度高度敏感。比热是温度的函数，如图 [1.3.18--4](ch01s03aex49.md#specific-heat) 所示。

其他材料属性定义如下：

| 弹性模量： | 180,000 MPa |
| --- | --- |
| 泊松比： | 0.3 |
| 密度： | 7.8 × 10⁹ Mg/mm³ |
| 导热系数： | 20°C 时 14.7 W/m/°C |
|  | 1200°C 时 28 W/m/°C |

### 初始条件

管道初始设置为 20°C，代表室温。

### 边界条件和加载

对上管道顶面施加 360 MPa 的压力。飞轮的初始旋转速度设置为 48.17 rad/s，即 7.7 转/秒。飞轮的转动惯量为 102,000 Mg·mm²。

### 相互作用

主要相互作用发生在管道之间的焊接界面；然而，次要考虑是焊接飞边与管道侧面接触的可能性。焊接界面摩擦行为假定遵循 Moal 和 Massoni（1995）所描述的规律，其中剪应力与规定压力的比值被观察到是界面滑移率的复杂函数。摩擦滑动产生的热量与塑性变形一起，有助于管道中的温度升高。

### Abaqus 建模方法与仿真技术

Abaqus/CAE 和 Abaqus/Standard 一起使用，以影响焊接仿真，使其能够允许管道在焊接区域发生极端变形。此过程通过 Python 脚本的使用实现了自动化。本示例研究了三种案例。

### 分析案例摘要

| 案例 1 | 飞轮初始速度 = 48.17 rad/s。此案例产生成功的焊接。 |
| --- | --- |
| 案例 2 | 飞轮初始速度 = 20.0 rad/s。此案例说明不成功的焊接场景；飞轮能量不足，无法开始焊接过程。 |
| 案例 3 | 飞轮初始速度 = 70.0 rad/s。此案例说明不成功的焊接场景；飞轮能量过大，导致温度升高到管道材料的液相区。 |

以下各节讨论适用于所有案例的分析考虑。提供了生成模型数据库和 Abaqus/Standard 输入文件的 Python 脚本用于案例 1，脚本中包含执行案例 2 和案例 3 仿真的说明。

### 分析类型

分析是非线性、准静态的热力耦合分析。采用完全耦合的温度-位移过程。

### 分析技术

成功模拟此过程的关键特征是重网格划分。在本示例中，由于焊接区域的大变形，采用多次分析来限制单元畸变。这些分析按顺序执行，在执行之间进行重网格划分，并通过 Python 脚本的使用实现自动化。

在每个重网格点，当前模型构型代表了管道形状和分析网格的显著变化。Abaqus/CAE 用于提取管道的外表面，重新对表面进行种子划分，并重网格化管道区域。此过程使用 Abaqus 脚本接口 [`PartFromOdb`](../ker/ker-link.md#ker-part-partfromodb-pyc) 命令，用于提取表示变形管道的孤儿网格部件。然后将这些部件传递给 [`Part2DGeomFrom2DMesh`](../ker/ker-link.md#ker-part-part2dgeomfrom2dmesh-pyc) 命令。此命令从先前导入的孤儿网格创建几何部件对象。一旦创建了变形部件的轮廓，Mesh 模块中的选项用于重网格化部件。新网格产生新的 Abaqus/Standard 分析，网格到网格的解映射过程从先前的分析映射状态变量（参见["网格到网格的解映射，" Abaqus Analysis User's Guide 第 12.4.1 节](../usb/usb-link.md#usb-anl-amapsolution)）。

### 网格设计

管道建模为轴对称的。使用的单元公式是具有扭转自由度的完全耦合温度-位移轴对称单元（单元类型 CGAX4HT 和 CGAX3HT），其中扭转自由度能够模拟面外方向的旋转和剪切变形。需要混合公式来处理材料在塑性流动过程中的不可压缩性。网格为每根管道分为两个区域。在靠近焊接界面的区域，创建较小的单元（参见图 [1.3.18--5](ch01s03aex49.md#initial-mesh)）。在重网格划分过程中，焊接表面附近的区域被重新计算，使得新的飞边区域也用较小的单元进行网格划分（参见图 [1.3.18--6](ch01s03aex49.md#remesh-design)）。

### 材料模型

为本示例定义的材料模型近似了 Astroloy 的高温行为，Soucail 等人（1992）使用 Norton-Hoff 本构律来描述温度和应变率黏塑性行为。在 Abaqus 中将类似的模型定义为率相关理想塑性材料模型。对于本模型中的加载，这些材料参数导致界面温度超过约 1200°C（接近材料固相线温度 1250°C）后才开始局部塑性流动。高于此温度时，Mises 流动应力对温度和应变率的变化高度敏感。在迭代过程的非线性求解中，需要在高应变率下对流动应力进行特殊调整以避免发散。在材料模型定义中，当应变率为 1.0 × 10⁶ s⁻¹ 时，定义了极端情况下的应力数据。当应变率等于零时，应力数据也被定义为与应变率 1.0 × 10⁻⁵ s⁻¹ 时的应力数据相同。由于大变形，材料模型中不考虑热膨胀。

假定 90% 的非弹性变形能量贡献给内部热生成，这是非弹性热量分数的 Abaqus 默认值。

### 初始条件

整个模型指定初始温度为 20°C。飞轮初始速度指定为用户单元的溶液相关变量初始条件（参见 `[inertiaweld_sub.f](../eif/inertiaweld_sub.f)`）。

### 边界条件

下管道完全固定。上管道没有边界条件。

### 载荷

360 MPa 的压力作为分布载荷施加在上管道的顶面上。

摩擦热产生的结果热通量被认为远大于任何通过对流损失的热量。因此，所有边界都被假定为绝热的。

### 相互作用

定义了五个相互作用：四个接触相互作用和一个执行器-传感器相互作用。

接触相互作用包括定义管道之间焊接界面的相互作用对，如图 [1.3.18--1](ch01s03aex49.md#inertiaweld-configuration) 所示。这对相互作用是对称的：一个相互作用将上管道定义为主面，下管道为从面，第二个相互作用反转主-从配对。这种"平衡主-从"排列对于分析获得更均匀的焊接界面接触压力并避免沙漏效应非常重要，它与软化接触相互作用特性结合，以促进界面两侧节点之间局部接触压力的共享。

为了模拟 Moal 和 Massoni（1995）的摩擦定义，焊接界面摩擦模型在用户子程序 [`FRIC`](../sub/sub-link.md#sub-xsl-fric) 中定义（参见 `[inertiaweld_sub.f](../eif/inertiaweld_sub.f)`）。该模型在某种意义上是非局部的，即所有接触节点的界面压力是施加的 360 MPa 压力，滑动速度基于飞轮的旋转角速度计算。这种摩擦力的处理有助于稳定求解。摩擦热生成基于摩擦牵引力和滑动速度计算。使用非默认接触控制定义来改善收敛性。这些定义包括在接触时延迟摩擦计算以及自动容差控制以避免接触颤振。

两个剩余的接触相互作用处理管道焊接区域附近自接触的可能性。飞边区域的自接触可能在重网格划分过程中造成问题：用于生成新的当前构型几何的 [`Part2DGeomFrom2DMesh`](../ker/ker-link.md#ker-part-part2dgeomfrom2dmesh-pyc) 命令执行曲线拟合操作，可能导致边界自相交，从而导致无效的部件拓扑和网格划分失败。为避免此问题，使用了引入具有小分离距离（0.01 mm）的法向压力的软化接触模型。重要的是将此分离距离保持得尽可能小，以避免任何非物理接触行为。

执行器-传感器相互作用通过用户子程序 [`UEL`](../sub/sub-link.md#sub-xsl-uel) 作用，使模拟连接在上管道上的飞轮成为可能。用户单元在分析中也具有传感器角色，测量焊接接头的镦粗量或轴向缩短量。当超过用户定义的临界镦粗距离时，用户单元子程序调用 `XIT` 来终止分析并发出重网格点信号。这个参数——允许的"镦粗距离"——与焊接区域中的网格变形程度有很好的相关性。

### 分析步骤

仿真序列中每个分析都使用单个完全耦合的温度-位移步骤。由于步骤的持续时间由用户单元控制（如上所述），每个分析的步骤持续时间不是先验已知的。因此，在每个分析中，步骤持续时间设置为仿真的总剩余时间，只有在仿真序列的最后一次分析才能达到该时间。

### 求解控制

对非线性求解使用完全牛顿迭代。自动时间步基于每个增量中的最大温度变化。使用先前增量的结果作为当前增量的初始猜测，采用抛物线外推技术将先前的状态转移到当前状态。

摩擦热生成导致的热通量可能引起接触颤振。为克服此困难，使用具有自动容差参数的接触控制。

### 输出请求

请求温度、应力和等效塑性应变的场输出。请求各种能量测量用于历史输出，允许您记录仿真过程中飞轮的能量变化。

### 收敛

通常，在初始增量中收敛困难，尤其是在重网格划分之后。当变形增加且网格畸变变得严重时，在分析运行序列接近末尾时也会观察到更多迭代。

### 运行步骤

本示例使用提供的 Abaqus/CAE Python 脚本运行。使用以下命令运行示例：

```
abaqus cae –script inertiaweld.py
```

仿真结果产生一系列单独的输出数据库（`.odb`）文件，每个对应一次分析。

为便于后处理，提供了多个脚本用于评估跨文件序列的仿真结果。使用以下命令创建 `inertia_weld_animation.avi` 动画文件：

```
abaqus cae –script inertiaweld_movie.py
```

运行以下 Python 脚本创建 X–Y 绘图：
- `[inertiaweld_xy.py](../eif/inertiaweld_xy.py)`，用于能量结果的历史绘图。
- `[inertiaweld_upset.py](../eif/inertiaweld_upset.py)`，用于组件镦粗量的历史绘图。
- `[inertiaweld_maxtemp.py](../eif/inertiaweld_maxtemp.py)`，用于最高温度的历史绘图。

### 结果讨论与案例比较

本示例主要关注成功的焊接案例——案例 1。提供案例 2 和案例 3 是为了展示惯性焊接工艺的不稳定性，其中正确选择飞轮尺寸（以转动惯量衡量）和飞轮初始速度至关重要。案例 2 和案例 3 说明了为什么仿真是焊接工艺设计中的有价值工具，因为每个案例都清楚地表明所选工艺参数不会产生成功的焊接。

#### 焊接成功：案例 1

示例仿真创建多个输出数据库（`.odb`）文件，需要 22 次重网格划分才能达到 5.0 秒的仿真时间。第一次重网格划分之前第一次分析的结果表明，焊接界面附近的温度升高非常快。在约 1.6 秒时，温度达到 1172°C，材料开始流动，挤压形成飞边（参见图 [1.3.18--7](ch01s03aex49.md#t-178)）。在 2.31 秒和 5 次重网格划分后，飞边延伸得足够多以至于折叠回去并与管道接触（参见图 [1.3.18--8](ch01s03aex49.md#t-231)）。在 3.51 秒时，飞轮速度减缓至 3.51 rad/s，温度开始下降，材料流动减慢。此时可以观察到相当大的飞边积聚（参见图 [1.3.18--9](ch01s03aex49.md#t-351)）。在 5.0 秒时，飞轮停止，温度降至 1000°C 以下，管道被焊接在一起（参见图 [1.3.18--10](ch01s03aex49.md#t-50)）。

图 [1.3.18--11](ch01s03aex49.md#abq-flash-result) 中的飞边预测与 Moal 和 Massoni（1995）所示的结果良好一致。图 [1.3.18--12](ch01s03aex49.md#energy) 绘制了动能历史（使用 `[inertiaweld_xy.py](../eif/inertiaweld_xy.py)` 创建），显示了飞轮如何减速和停止。"RemeshPoints"标记表示重网格划分过程发生的时间。焊接过程中的镦粗距离如图 [1.3.18--13](ch01s03aex49.md#upset) 所示（使用 `[inertiaweld_upset.py](../eif/inertiaweld_upset.py)` 创建）。案例 1 的最高温度曲线如图 [1.3.18--14](ch01s03aex49.md#maxtemp) 所示（使用 `[inertiaweld_maxtemp.py](../eif/inertiaweld_maxtemp.py)` 创建）。

#### 飞轮能量不足：案例 2

如果选择较低的飞轮初始速度，温度可能无法达到材料流动所需的水平。对于此案例，初始速度降低到 20 rad/s。图 [1.3.18--15](ch01s03aex49.md#case2) 显示了在 5.0 秒时的构型，此时变形不大，界面附近的温度约为 250°C。图 [1.3.18--14](ch01s03aex49.md#maxtemp) 中案例 2 的最高温度历史图显示，管道温度仅达到 700°C（大约 1.1 秒后），然后开始冷却。在这种情况下，材料不够热以引发材料流动，焊接不会成功。

#### 飞轮能量过大：案例 3

如果选择较高的飞轮初始速度，材料会变得非常热以至于开始熔化。对于此案例，飞轮的初始速度设置为 70 rad/s。图 [1.3.18--16](ch01s03aex49.md#case3) 显示，温度在约 0.9 秒时迅速达到 1360°C，这远远超过了 1250°C 的熔点，然后收敛失败停止了分析。图 [1.3.18--14](ch01s03aex49.md#maxtemp) 显示了案例 3 中管道最高温度的历史图。在这种情况下，过大的能量导致熔化和焊接失败。
