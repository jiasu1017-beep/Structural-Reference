### 概述

Abaqus/Standard提供了两种用于建模接触和交互问题的算法：一般接触算法和接触对算法。参见["接触交互分析：概述，" 第36.1.1节"](pt09ch36s01abo33.md)对两种算法的比较。本节介绍如何在Abaqus/Standard分析中包含一般接触、如何指定可能参与一般接触交互的模型区域，以及如何从一般接触分析获取输出。

Abaqus/Standard中的一般接触算法：
- 作为模型定义的一部分指定；
- 允许非常简单的接触定义，对所涉及表面类型的限制很少；
- 使用复杂的跟踪算法以确保有效执行适当的接触条件；
- 可与接触对算法同时使用（即，一些交互可以用一般接触算法建模，而其他交互用接触对算法建模）；
- 可用于二维或三维表面；以及
- 使用有限滑动、表面对表面接触公式。

### 定义一般接触交互

一般接触交互的定义包括指定：
- 一般接触算法和定义接触域（即相互作用的表面），如本节所述；
- 接触表面属性（["Abaqus/Standard中一般接触的表面属性，" 第36.2.2节"](pt09ch36s02aus140.md)）；
- 机械接触属性模型（["Abaqus/Standard中一般接触的接触属性，" 第36.2.3节"](pt09ch36s02aus141.md)）；
- 与初始接触状态相关的控制（["控制Abaqus/Standard中的初始接触状态，" 第36.2.4节"](pt09ch36s02aus142.md)）；和
- 算法接触控制（["Abaqus/Standard中一般接触的数值控制，" 第36.2.6节"](pt09ch36s02aus144.md)）。

使用一般接触定义装配体各个组件之间接触的分析示例在["棘轮机构的冲击分析，" Abaqus示例问题指南第2.1.17节](../exa/exa-link.md#exa-dyn-pawlratchet)中描述。

### 用于一般接触的表面

Abaqus/Standard中的一般接触算法允许在其使用的表面中具有相当一般的特征，如["接触交互分析：概述，" 第36.1.1节"](pt09ch36s01abo33.md)中所讨论。有关在Abaqus/Standard中为一般接触算法定义表面的详细信息，请参见["基于单元的表面定义，" 第2.3.2节"](pt01ch02s03aus17.md)。

指定接触域的便捷方法是使用裁剪表面。此类表面可用于通过使用在原始配置中被封闭在指定矩形盒中的接触域来执行"盒中接触"。有关更多信息，请参见["在表面上操作，" 第2.3.6节"](pt01ch02s03aus21.md)。

此外，Abaqus/Standard自动定义一个全包容表面，该表面对规定接触域很方便，如本节后面所述。全包容自动定义表面包含所有基于单元的表面facet。

Abaqus/Standard中的一般接触算法提供建模表面对表面接触、边对表面接触和边对边接触的能力。表面对表面接触公式用作主要公式，可以将边对表面和边对边公式用作补充公式。边对表面接触公式也用于建模梁或桁架单元段与faceted表面之间的接触。同样，边对边接触公式也支持梁或桁架单元段之间的接触。请参见下面的["边对表面接触的考虑"](pt09ch36s02aus139.md#usb-cni-acontactgeneralstd-e-to-s)"和["边对边接触的考虑"](pt09ch36s02aus139.md#usb-cni-acontactgeneralstd-e-to-e)"，了解更多信息。

一般接触算法不考虑涉及解析表面或基于节点的表面的接触，尽管这些表面类型可以在也使用一般接触的分析中包含在接触对中。

### 在分析中包含一般接触

Abaqus/Standard中的一般接触在分析开始时定义。只能指定一个一般接触定义，此定义对分析的每个步骤都生效。

| **输入文件用法：** | 使用以下选项指示一般接触定义的开始： |
| --- | --- |
|  | ``` [*CONTACT](../key/key-link.md#usb-kws-hcontact) ``` 此选项只能在模型定义中出现一次。 |

| **Abaqus/CAE用法：** | Interaction模块：**Create Interaction**：**Step：** **Initial**，**General contact (Standard)** |
| --- | --- |

### 定义一般接触域

您可以通过定义一般接触包含和排除来指定可能相互接触的模型区域。在模型定义中只允许一个接触包含定义和一个接触排除定义。

分析中所有接触包含首先被应用，然后应用所有接触排除，不管它们被指定的顺序如何。接触排除优先于接触包含。一般接触算法将只考虑接触包含定义指定的交互，而不是接触排除定义指定的交互。

一般接触交互通常通过为Abaqus/Standard提供的默认自动生成表面指定自接触来定义。一般接触算法中使用的所有表面可以跨越多个未连接体，因此此算法中的自接触不限于单个身体与自身的接触。例如，跨越两个体的表面的自接触意味着身体之间的接触以及每个身体与自身的接触。

#### 指定接触包含

定义接触包含以指定应考虑接触的模型区域。

##### 为整个模型指定"自动"接触

您可以为Abaqus/Standard自动定义的默认未命名、全包容表面指定自接触。此默认表面包含所有外部单元面，以下述例外情况为准。此是定义接触域的最简单方法。

默认表面不包括仅属于内聚单元的面。实际上，默认表面的生成就好像内聚单元不存在一样。请参见["使用内聚单元建模，" 第32.5.3节"](pt06ch32s05alm42.md)，了解与内聚单元相关的接触建模问题的进一步讨论。

| **输入文件用法：** | 使用以下两个选项为整个模型指定"自动"接触： |
| --- | --- |
|  | ``` [*CONTACT](../key/key-link.md#usb-kws-hcontact) [*CONTACT INCLUSIONS](../key/key-link.md#usb-kws-hcontactinclusions), ALL EXTERIOR ``` 当使用ALL EXTERIOR参数时，[*CONTACT INCLUSIONS](../key/key-link.md#usb-kws-hcontactinclusions)选项不应有数据行。 |

| **Abaqus/CAE用法：** | Interaction模块：**Create Interaction**：**General contact (Standard)**：**Included surface pairs: All* with self** |
| --- | --- |

##### 指定单个接触交互

或者，您可以通过指定单个接触表面对来直接定义一般接触域。自接触只会在对中指定的两个表面重叠（或相同）时建模，并且仅在重叠区域建模。在某些情况下，通过仅在分析中会发生接触的一般接触域的部分表面上包含接触，可以提高计算性能和稳健性。

可以在接触域中包含多个表面对。所有指定的表面必须是基于单元的表面。

| **输入文件用法：** | 使用以下两个选项指定单个接触交互： |
| --- | --- |
|  | ``` [*CONTACT](../key/key-link.md#usb-kws-hcontact) [*CONTACT INCLUSIONS](../key/key-link.md#usb-kws-hcontactinclusions) *surface_1*, *surface_2* ``` 当省略ALL EXTERIOR参数时，必须指定至少一行数据。两个数据条目中的任何一个都可以留空，但每个数据行必须至少包含一个逗号；将为空数据行发出错误消息。如果省略第一个表面名称，则假定为默认未命名、全包容、自动生成的表面。如果省略第二个表面名称或与第一个表面名称相同，则假定第一个表面与自身之间接触。两个数据条目都留空等效于使用ALL EXTERIOR参数。 |

| **Abaqus/CAE用法：** | Interaction模块：**Create Interaction**：**General contact (Standard)**：**Included surface pairs: Selected surface pairs: Edit**，在左侧列中选择表面，然后单击中间的箭头将它们转移到包含对的列表 |
| --- | --- |

```
*surface_1*,
```

```
[*CONTACT INCLUSIONS](../key/key-link.md#usb-kws-hcontactinclusions)
*surface_1*, *surface_1*
```

#### 指定接触排除

您可以通过指定要从接触中排除的模型区域来细化接触域定义。指定接触排除的可能动机包括：
- 避免物理上不合理的接触交互；
- 通过排除可能不会相互作用的模型部分来提高计算性能。

将为所有指定的表面对忽略接触，即使这些交互在接触包含定义中直接或间接指定。

可以从接触域中排除多个表面对。所有指定的表面必须是基于单元的表面。请记住，表面可以定义为跨越多个未连接体，因此自接触排除不限于排除单身体接触。

| **输入文件用法：** | 使用以下两个选项指定接触排除： |
| --- | --- |
|  | ``` [*CONTACT](../key/key-link.md#usb-kws-hcontact) [*CONTACT EXCLUSIONS](../key/key-link.md#usb-kws-hcontactexclusions) *surface_1*, *surface_2* ``` 两个数据条目中的任何一个都可以留空。如果省略第一个表面名称，则假定为默认未命名、全包容、自动生成的表面。如果省略第二个表面名称或与第一个表面名称相同，则假定从接触域中排除第一个表面与自身之间的接触。 |

| **Abaqus/CAE用法：** | Interaction模块：**Create Interaction**：**General contact (Standard)**：**Excluded surface pairs: Edit**，在左侧列中选择表面，然后单击中间的箭头将它们转移到排除对的列表 |
| --- | --- |

##### 自动生成的接触排除

Abaqus/Standard在某些情况下自动为一般接触生成接触排除。
- 接触排除自动为使用接触对算法或基于表面的绑定约束定义的交互生成，以避免这些交互约束的冗余（可能不一致） enforcement。例如，如果为`surface_1`和`surface_2`定义了接触对，并且为整个模型定义了"自动"一般接触，则Abaqus/Standard会为`surface_1`和`surface_2`之间的一般接触生成接触排除，以便这些表面之间的交互仅使用接触对算法建模。这些自动生成的接触排除在整分析中都有效。
- Abaqus/Standard自动为模型中每个刚体的自接触生成接触排除，因为刚体不可能与自身接触。
- 当您为特定一般接触表面对指定纯主-从接触表面加权时，会自动生成与指定相反的主-从方向的接触排除（有关此类接触排除的更多信息，请参见["Abaqus/Standard中一般接触的数值控制，" 第36.2.6节"](pt09ch36s02aus144.md)）。
- Abaqus/Standard为一般接触域中断开体内接触涉及的主-从角色分配默认纯主-从角色，并且默认情况下会为相反的主-从方向生成接触排除。覆盖默认纯主-从分配的选项与替代纯主-从分配或平衡主-从分配在["Abaqus/Standard中一般接触的数值控制，" 第36.2.6节"](pt09ch36s02aus144.md)中讨论。
- 接触排除自动为在模型初始配置中严重过盈的表面部分生成。请参见["控制Abaqus/Standard中的初始接触状态，" 第36.2.4节"](pt09ch36s02aus142.md)，了解更多信息。

以下输入指定接触域基于全包容、自动生成的表面的自接触，但应忽略全包容、自动生成的表面与*surface_2*之间的接触（包括任何重叠区域中的自接触）：

```
*surface_1*,
```

```
*surface_1*, *surface_1*
```

### 边对表面接触的考虑

一般接触算法可以考虑三维边对表面接触。除了建模梁或桁架单元段与faceted表面之间的接触外，它在解决某些交互方面比表面对表面接触公式更有效。当用作表面对表面接触公式的补充时，边对表面接触公式旨在当主动接触区域中相应表面facet的法线方向形成斜角时，避免一个表面的特征边缘局部穿透另一个表面的相对平滑部分。如[图36.2.1-1](pt09ch36s02aus139.md#edge-surface-needed)所示的模型将受益于补充边对表面接触 enforcement，因为在插入加载的某些期间主动接触区域对应于特征边缘。对于[图36.2.1-2](pt09ch36s02aus139.md#edge-surface-notneeded)所示的模型不需要补充边对表面接触，因为表面对表面接触公式能够充分抵抗穿透。

**图36.2.1-1** 涉及特征边缘对表面接触的snap-fit示例，接触区域中表面法线之间成斜角。

![](../graphics/acontact-snap-featureedge.png)

**图36.2.1-2** 主动接触区域周长处具有特征边缘的示例，具有相反的表面法线。

![](../graphics/acontact-parallel-featureedge.png)

表示梁和桁架单元的接触边缘具有圆形横截面，不管梁或桁架单元的实际横截面如何。表示桁架单元的接触边缘的半径源自桁架截面定义上指定的横截面积（它等于具有等效横截面积的实心圆截面的半径）。对于具有圆形横截面的梁，接触边缘的半径等效于截面半径。对于具有非圆形横截面的梁，接触边缘的半径等于围绕截面的外接圆的半径。边对表面接触通过将相关表面包含在一般接触域中来激活。对于梁或桁架元素，默认情况下全包容表面包含基于梁或桁架元素的表面。

默认情况下，当表面用于一般接触交互时，所有适用的facet都与具有至少45度特征角的实体和壳单元边缘一起包含在接触定义中。请参见["Abaqus/Standard中一般接触的表面属性"中的"特征边缘"](pt09ch36s02aus140.md#usb-cni-asurfacepropassignstd-featedge)，了解有关边对表面接触考虑哪些特征边缘的控制讨论。边对表面接触约束从不参与热、电或孔隙压力接触属性。例如，在耦合温度-位移分析中，表面对表面约束可以影响机械和热交互；但是，如果包含边对表面约束，它们将只帮助抵抗穿透。

与特征边缘相关的接触区域取决于网格尺寸；因此，与边对表面接触相关的接触压力（以力/面积的单位）取决于网格。

表面对表面和边对表面接触约束可能同时在同一节点处激活。为了帮助避免数值过度约束问题，边对表面接触约束始终使用惩罚方法强制执行。

### 边对边接触的考虑

一般接触算法可以选择性地考虑边对边接触。实体和壳样表面上的特征边缘、壳周长边缘以及表示梁（和桁架）的边缘可以被包含。

有两种边对边接触公式可用。第一种公式最初为具有厚度的梁之间的接触而开发，使用其中一个梁的径向方向作为接触方向（类似于管对管接触单元所做的，请参见["管对管接触单元，" 第40.3.1节"](pt09ch40s03alm65.md)）。此公式不仅适用于梁边缘，也适用于具有相关壳厚度的壳周长边缘。另一种公式将接触法线方向基于所考虑接触的两个相应边缘的叉积；此公式主要关注非平行边缘之间的接触。

除了选择接触公式外，还必须指定特征角准则以激活特征和周长边缘参与边对边接触。请参见["Abaqus/Standard中一般接触的表面属性"中的"特征边缘"](pt09ch36s02aus140.md#usb-cni-asurfacepropassignstd-featedge)，了解有关边对边接触考虑哪些特征边缘的控制讨论。如果只存在梁边缘，仅指定接触公式就足够了。

梁对梁接触不能用于建模与底层实体或壳单元共享节点的类梁单元之间的接触（例如，用于建模纵向加强筋的梁单元）。

| **输入文件用法：** | 使用以下选项激活边对边接触的两种公式： |
| --- | --- |
|  | ``` [*CONTACT FORMULATION](../key/key-link.md#usb-kws-hcontformulation), TYPE=EDGE TO EDGE, FORMULATION=BOTH ``` 使用以下选项停用边对边接触： ``` [*CONTACT FORMULATION](../key/key-link.md#usb-kws-hcontformulation), TYPE=EDGE TO EDGE, FORMULATION=NO (default) ``` 使用以下选项激活径向边对边接触公式： ``` [*CONTACT FORMULATION](../key/key-link.md#usb-kws-hcontformulation), TYPE=EDGE TO EDGE, FORMULATION=RADIAL ``` 使用以下选项激活基于边缘方向叉积的边对边接触公式： ``` [*CONTACT FORMULATION](../key/key-link.md#usb-kws-hcontformulation), TYPE=EDGE TO EDGE, FORMULATION=CROSS ``` |

| **Abaqus/CAE用法：** | 在Abaqus/CAE中不支持建模边对边接触。 |
| --- | --- |

### 输出

与接触相关的输出变量分为两类：节点变量（有时称为约束变量）和整个表面变量。此外，Abaqus输出与接触交互相关的诊断信息数组，如["Abaqus/Standard分析中的接触诊断，" 第39.1.1节"](pt09ch39s01aus183.md)以及为一般接触生成的内部表面中所讨论的。

有关与热、电和孔隙流体分析相关的变量的更详细讨论，请参见[第37章"接触属性模型"](pt09ch37.md)中相关接触属性的章节。

#### 一般接触域和组件表面

Abaqus/Standard生成与一般接触相关的以下内部表面：
- `General_Contact_Faces`,
- `General_Contact_Edges`,
- `General_Contact_Faces_*k*`, 和
- `General_Contact_Edges_*k*`,

其中`*k*`对应于自动分配的"组件编号"。没有组件编号的两个一般接触内部表面分别包含一般接触域中包含的所有表面面和所有特征边缘。

每个特征边缘组件表面`General_Contact_Edges_*k*`是对应面组件表面`General_Contact_Faces_*k*`的面边缘子集（满足特征边缘准则）。面组件表面彼此没有公共节点。默认情况下，编号较低的面组件表面将对编号较高的面组件表面充当主表面用于表面对表面公式。组件编号不影响边对表面公式考虑的内容。组件表面在两种公式类型的诊断消息中被引用。

内部表面可以使用Abaqus/CAE可视化模块中的显示组查看。Abaqus/Standard生成的内部表面名称不应在模型定义中使用。

#### 节点接触变量

节点接触变量可以在Abaqus/CAE可视化模块中的接触表面上绘制等高线。节点接触变量包括接触压力和力、摩擦剪切应力和力、接触期间表面的相对切向运动（滑移）、表面之间的间隙、热或流体通量每单位面积以及流体压力。写入输出数据库（`.odb`）文件的许多节点接触变量通常对所有接触节点可用，不管它们是充当从节点还是主节点。其他节点接触变量仅在充当从节点的节点处可用。大多数对数据（`.dat`）文件、结果（`.fil`）文件和实用程序子程序`GETVRMAVGATNODE`的接触输出与单个约束相关。对于对输出数据库（`.odb`）文件的接触输出，应用了一些过滤以减少接触输出噪声。

##### 接触压力

接触压力分布在许多Abaqus分析中是关键关注点。您可以查看除解析刚体表面和基于刚体型单元的离散刚体表面（后者限制不适用于一般接触）之外的所有接触表面上的接触压力。您可以查看接触压力误差指示器的等高线图以及接触压力的等高线图，以在接触压力解决方案令人关注的区域中获得接触压力解决方案局部精度的视角（请参见["影响自适应重划分的误差指示器选择，" 第12.3.2节"](pt04ch12s03aus84.md)，了解误差指示器输出的进一步讨论）。

在某些情况下，您可能会观察到接触压力超出实际接触区域，这是由于以下因素：
- 等高线图通过插值节点值构建，这可能导致在接触区域外的facet部分出现非零值。例如，这种效果通常在角落处明显，例如当两个相同大小、对齐的块接触时——如果接触表面缠绕在角落周围，接触压力等高线将略微延伸到角落周围。
- 为了最小化主动接触区域内的接触应力噪声，Abaqus/Standard将节点接触应力计算为节点参与的主动接触约束相关值的加权平均值。应用了一些过滤来减少为主动接触区域边缘（仅弱参与接触约束）的节点报告的接触应力值，但这种过滤不是"完美的"，可能导致接触区域大小看起来有些夸大。类似地，接触状态输出也会在位于主动接触区域边缘的节点处受到影响。在这种情况下，即使接触实际上是开放的，接触状态可能在夸张区域中的节点处被报告为闭合。

由于这些因素，尝试从接触应力分布推断接触力分布可能有些误导。相反，您可以请求节点接触力输出，它准确地表示分析中存在的接触力分布。

##### 由于边对表面和边对边交互产生的接触应力

对于边对表面接触和边对边接触与径向公式（其中主动接触沿一条线），可以在Abaqus/Standard中请求输出变量CLINELOAD到输出数据库（`.odb`）。此接触载荷具有力/长度的单位，与网格无关。只能输出由于边对表面接触（CSTRESSETOS）的接触应力（以力/面积的单位）以可视化边对表面约束主动的区域。边对表面公式通过将每边缘长度的接触力除以代表性表面facet长度来计算以力/面积单位的接触应力。由于接触区域取决于网格尺寸，边对表面接触应力与网格相关。对于使用叉积公式的边对边接触，其中主动接触区域被理想化为一个点，可以请求与网格无关的输出变量CPOINTLOAD（具有力的单位）。

接触应力（CSTRESS）包含来自表面对表面、边对表面和边对边约束的贡献（如果活跃）。在累积来自边对表面和边对边接触约束的贡献时，约束值除以代表性表面facet长度或其平方值，以适当地将它们缩放为具有力/面积的单位。

边缘表示表面平滑度的不连续性，边缘附近的真实接触应力解决方案通常以强梯度为特征。为接触应力（CSTRESSERI）输出的误差指示器通常在涉及边缘的约束显著的区域非常高。

#### 整个表面变量

整个表面变量在Abaqus/Standard中的一般接触中仅略微支持，因为这些变量默认与整个一般接触域相关，而不是与一般接触相关的单个表面。将整个表面变量限制为受一般接触域的一部分影响的唯一方法是指定输出请求中的节点集。整个表面变量计算为一般接触中所有节点（或可选地限于特定节点集）充当从节点时的总和。例如，CFN是由于接触压力作用于从节点的总力。CFN和其他一般接触的整个表面变量通常用途不大，因为一般接触内不同交互对变量的贡献通常会相互抵消，结果通常取决于主和从角色的内部分配。

#### 请求输出

某些接触变量必须作为一组请求。例如，要输出表面之间的间隙（COPEN），您必须请求变量CDISP（接触位移）。CDISP输出COPEN和CSLIP（接触期间表面的切向运动）。可用接触变量和标识符的完整列表在["Abaqus/Standard输出变量标识符，" 第4.2.1节"](pt02ch04s02abv01.md)中给出。

可以通过指定包含作为某些一般接触交互从节点的节点子集的节点集来限制输出请求。形成这些输出请求的说明可在以下章节中找到：
- 要请求对数据（`.dat`）文件的输出，请参见["Abaqus/Standard中的表面输出"在"输出到数据和结果文件，" 第4.1.2节"](pt02ch04s01aus39.md#usb-out-oprintfile-surface)。
- 要请求对输出数据库（`.odb`）文件的输出，请参见["Abaqus/Standard和Abaqus/Explicit中的表面输出"在"输出到输出数据库，" 第4.1.3节"](pt02ch04s01aus40.md#usb-out-odboutput-surface)。

#### 切向结果的输出

Abaqus报告切向变量（摩擦剪切应力、粘性剪切应力和相对切向运动）的值，相对于在表面上定义的局部切向方向。局部切向方向的定义在["Abaqus/Standard中的接触公式"中的"表面上的局部切向方向，" 第38.1.1节"](pt09ch38s01aus177.md#usb-cni-acontactpairform-slipdir)中解释。这些方向并不总是对应于全局坐标系，并且在几何非线性分析中随接触对旋转。

Abaqus/Standard通过获取变量向量与约束点关联的局部切向方向的标量积来计算每个约束点的切向结果，![](../graphics/usb_eqn00238.gif)或![](../graphics/usb_eqn00239.gif)。变量名称末尾的数字指示变量是否对应于第一或第二局部切向方向。例如，CSHEAR1是第一局部切向方向中的摩擦剪切应力分量，而CSHEAR2是第二局部切向方向中的摩擦剪切应力分量。

##### 累积增量相对运动（滑移）的定义

Abaqus/Standard将增量相对运动（也称为滑移）定义为增量相对节点位移向量与局部切向方向的标量积。增量相对节点位移向量测量从节点相对于主表面运动的运动。增量滑移仅在从节点接触主表面时累积。在分析期间所有此类增量滑移的总和报告为CSLIP1和CSLIP2。关于此量计算的详细信息可在["身体之间的小滑动交互，" Abaqus理论指南第5.1.1节](../stm/stm-link.md#stm-ifc-smslidcontact)；["变形体之间的有限滑动交互，" Abaqus理论指南第5.1.2节](../stm/stm-link.md#stm-ifc-slidecontactelem)；和["变形体与刚体之间的有限滑动交互，" Abaqus理论指南第5.1.3节](../stm/stm-link.md#stm-ifc-defbodyrigidsurf)中找到。

#### 扩展为间隙提供的接触开口输出的范围

为了降低计算成本，默认情况下会避免在表面之间的距离大于可能传递接触力（或热通量等）的最小间隙距离的区域进行监视潜在交互点的详细计算。因此，接触开口（COPEN）输出通常不会在表面相对于表面facet尺寸开口超过一小量的情况下提供。您可以扩展Abaqus/Standard提供接触开口输出的范围；将提供高达等于指定"跟踪厚度"的间隙距离的COPEN。使用此控制可能会因额外的接触跟踪计算而增加计算成本，特别是如果您指定较大的跟踪厚度值。
