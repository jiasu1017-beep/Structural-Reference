# 2.1.4 侵蚀弹丸撞击侵蚀板









**产品：** Abaqus/Explicit  

本示例模拟了圆柱形弹丸以2000 m/sec的速度撞击平板装甲板的斜向冲击。弹丸和板使用相同的材料模型，包括渐进损伤的失效模型。本示例展示了通用接触算法在高速冲击过程中建模多个接触体上表面侵蚀的能力。

### 问题描述

未变形网格如图2.1.4-1所示。装甲板厚度为3 mm。为简单起见，对板的相对较小的矩形区域进行建模，在三个切割面上规定完全固定边界条件，在一个切割面上规定*y*轴对称。弹丸长度为10 mm，半径为1 mm，初始速度为2000 m/sec。弹丸的圆柱轴线距板法向20°，弹丸的初始速度与其圆柱轴线对齐。对弹丸的一半进行建模，在切割面上规定*y*轴对称。板和弹丸材料属性相同，杨氏模量为210 GPa，泊松比为0.3，密度为7800 kg/m³。材料的屈服应力被规定为不同等效塑性应变率下等效塑性应变的函数。材料定义还包括渐进损伤的失效模型，这导致Abaqus/Explicit在单元失效时将其从网格中移除。同时使用了延性和剪切起始准则：延性准则以损伤起始时塑性应变的形式规定，作为应力三轴度的表格函数；剪切准则以损伤起始时塑性应变的形式规定，作为剪切应力比的表格函数。假设损伤演化能量为500 N/m。

在分析过程中，两个物体的单元都失效，这需要使用能够适应当前未失效单元暴露表面的基于单元的表面。通用接触算法支持以此方式演化的基于单元的表面（而接触对算法不支持）。为了建模侵蚀接触，用户必须在接触域中包含分析过程中可能暴露的所有表面 faces，包括原本位于物体内部的faces。在本分析中，只有预期参与接触的内部faces被包含在接触域中，以最小化内存使用（包括模型中所有单元的内部faces会使内存使用增加一倍以上）。

默认情况下，通用接触算法不包含节点侵蚀，因此接触节点即使在周围所有单元失效后仍将参与接触计算。这些节点充当可以与活动接触面接触的漂浮点质量。为了比较目的，还进行了包含节点侵蚀的分析，这导致节点在所有周围单元失效后从接触计算中移除（并可导致计算节省）。与此示例中与自由飞行节点相关的动量转移预计是显著的，因此不建议使用节点侵蚀。

### 结果和讨论

分析不同时刻的变形形状如图2.1.4-2到图2.1.4-5所示（在这些图中仅显示活动单元）。如图2.1.4-5所示，弹丸最终穿透板，约为弹丸前半部分的单元在分析过程中失效。在图2.1.4-5中可以看到一些带有活动单元的断裂碎片。这种碎片的节点和暴露面可以参与接触。不再附着于任何活动单元的节点只能在前述节点侵蚀的分析中参与接触（对应于主要输入文件）。图2.1.4-6分别比较了有和没有节点侵蚀的分析的总动能历史。对于没有节点侵蚀的模型，初始动能约32%被冲击吸收；而对于有节点侵蚀的模型，初始动能约26%被吸收。
