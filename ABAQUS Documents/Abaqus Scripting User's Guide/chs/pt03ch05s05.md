# 5.5 自定义对话框







`AFXDialog` 是工具包中其他对话框类的基类。如果其他对话框类都不适合您的需求，您必须从 `AFXDialog` 派生对话框并自行提供大部分对话框处理。本节介绍如何使用 `AFXDialog` 创建自定义对话框。以下主题将被涵盖：

### 5.5.1 自定义对话框概述

`AFXDialog` 是工具包中其他对话框类的基类。如果其他对话框类都不适合您的需求，您必须从 `AFXDialog` 派生对话框并自行提供大部分对话框处理。

`AFXDialog` 类通过提供以下功能扩展了 `FXDialog` 类：
- 允许自动构建操作区域按钮的按钮标志。
- 控制操作区域放置的选项标志。选项标志还决定是否在操作区域和对话框其余部分之间包含分隔符。
- 各种操作区域提交语义的消息 ID。
- 手动添加操作区域按钮的方法。
- **否**、**取消**和**关闭**按钮的自动处理。标题栏右侧的**关闭**（X）按钮也提供自动处理。
- 对话框取消后自动销毁对话框。

更多详细信息请参阅"操作区域"，第 5.5.4 节。

### 5.5.2 构造函数

`AFXDialog` 构造函数有三个原型。三种原型之间的区别是对话框的遮挡行为，如下例所示：
- 以下语句创建一个在与主窗口重叠时始终遮挡主窗口的对话框：``` AFXDialog(title, actionButtonIds=0, opts=DIALOG_NORMAL, x = 0, y = 0, w = 0, h = 0) ```
- 以下语句创建一个在与组件重叠时始终遮挡其所有者组件（通常是对话框）的对话框：``` AFXDialog(owner, title, actionButtonIds=0, opts=DIALOG_NORMAL, x = 0, y = 0, w = 0, h = 0) ```
- 以下语句创建一个可以被应用程序中任何其他窗口遮挡的对话框：``` AFXDialog(app, title, actionButtonIds=0, opts = DIALOG_NORMAL, x = 0, y = 0, w = 0, h = 0) ```

构造对话框时，您将从 `AFXDialog` 类派生。构造函数体中您应该做的第一件事是调用基类构造函数以正确初始化对话框。然后，您将通过添加组件来构建对话框的内容。例如：

```
class MyDB(AFXDialog):

    # My constructor
    def __init__(self):

        # Call base class constructor
        AFXDialog.__init__(self, 'My Dialog', self.DISMISS)

    # Add widgets next...
```

### 5.5.3 尺寸和位置

默认情况下，用户不能调整对话框的大小。但是，如果对话框包含可以拉伸以显示更多条目的文本字段或列表，则应该允许用户调整对话框大小。可以通过在对话框构造函数中指定 DECOR_RESIZE 标志来允许调整大小。

**注意：** 由 `AFXDialog` 创建的对话框不支持最小化和最大化；如果这些标志包含在对话框构造函数中，它们将被忽略。

您永远不应在对话框的构造函数中指定对话框的尺寸和位置。Abaqus GUI Toolkit 会将对话框放置在屏幕上并确定其正确的尺寸。

### 5.5.4 操作区域

对话框的操作区域包含按钮，例如**确定**和**取消**。这些按钮允许用户提交对话框中的值、关闭对话框或执行其他操作。

`AFXDialog` 支持通过对话框构造函数中的位标志自动创建操作区域及其按钮。您可以使用[表 5-1](pt03ch05s05.md#cus-dlg-dialogs-table) 中描述的标志来包含标准操作区域按钮。

**表 5–1** 操作区域标志。
| 按钮标志 | 消息 ID | 标签 | 语义 |
| --- | --- | --- | --- |
| **AFXDialog.OK** | AFXDialog.ID_CLICKED_OK | 确定 | 提交对话框中的值，处理它们，然后隐藏对话框。 |
| **AFXDialog.CONTINUE** | AFXDialog.ID_CLICKED_CONTINUE | 继续… | 提交对话框中的值，隐藏它，然后继续从另一个对话框或提示收集用户输入。 |
| **AFXDialog.APPLY ** | AFXDialog.ID_CLICKED_APPLY | 应用 | 与确定相同，只是对话框不会被隐藏。 |
| **AFXDialog.DEFAULTS** | AFXDialog.ID_CLICKED_DEFAULTS | 默认 | 将对话框中的值重置为其默认值。 |
| **AFXDialog.YES ** | AFXDialog.ID_CLICKED_YES | 是 | 对对话框提出的问题执行肯定操作。 |
| **AFXDialog.NO** | AFXDialog.ID_CLICKED_NO | 否 | 对对话框提出的问题执行否定操作。 |
| **AFXDialog.CANCEL** | AFXDialog.ID_CLICKED_CANCEL | 取消 | 不提交对话框中的值；只是隐藏对话框。可选地，对于 AFXDataDialog，如果用户自上次提交以来更改了任何值，则可能会发布 bailout。 |
| **AFXDialog.DISMISS** | AFXDialog.ID_CLICKED_DISMISS | 关闭 | 隐藏对话框而不采取任何其他操作。 |

`AFXDialog` 还支持以下决定操作区域位置的选项：

**DIALOG_ACTIONS_BOTTOM**

此选项将操作区域放置在对话框底部，这是默认选项。

**DIALOG_ACTIONS_RIGHT**

此选项将操作区域放置在对话框的右侧。

**DIALOG_ACTIONS_NONE**

此选项不创建操作区域；例如，在工具箱对话框中。

您还可以通过在选项中包含以下标志来指定是否应在操作区域和对话框其余部分之间放置分隔符：

**DIALOG_ACTIONS_SEPARATOR**

Abaqus/CAE 的样式是，如果操作区域和对话框其余部分之间已经存在分界，则省略分隔符；例如，一个横跨对话框整个宽度的框架沿对话框底部延伸。以下语句说明如何在带有分隔符的对话框中定义操作区域：

```
class ActionAreaDB(AFXDialog):

    def __init__(self):

        AFXDialog.__init__(self, 'Action Area Example1',
            self.OK|self.APPLY|self.CANCEL,
            DIALOG_ACTIONS_SEPARATOR)

        FXLabel(self, 'Standard action area example dialog.')
```

### 5.5.5 自定义操作区域按钮名称

[表 5-1](pt03ch05s05.md#cus-dlg-dialogs-table) 中的标志涵盖了对话框中您可能需要的所有语义。因此，不需要任何其他自定义标志；但是，在某些情况下，您可能需要为其中一个标准操作使用不同的标签。要为其中一个标准操作使用不同的标签，您不能在构造函数参数中指定任何按钮标志；但是，您可以使用 `appendActionButton` 方法添加您自己的操作区域按钮。`appendActionButton` 方法有两个原型：

```
appendActionButton(buttonId) appendActionButton(text, tgt, sel)
```

原型的第一个版本创建[表 5-1](pt03ch05s05.md#cus-dlg-dialogs-table) 中定义的标准操作区域按钮。原型的第二个版本创建一个标签作为文本参数的按钮。此外，第二个版本允许您设置目标和选择器，以便您可以捕获此按钮的消息并采取相应操作。以下语句说明如何创建自定义操作区域按钮：

```
class ActionAreaDB(AFXDialog):
    def __init__(self):

        AFXDialog.__init__(self, 'Action Area Example 2',
            0, DIALOG_ACTIONS_SEPARATOR)
        FXLabel(self, 'Custom action area example dialog.')
        self.appendActionButton('Highlight', self, 
            self.ID_CLICKED_APPLY)
        self.appendActionButton(self.CANCEL)
```

### 5.5.6 操作按钮处理

`AFXDialog` 和 `AFXDataDialog` 提供对操作区域按钮被点击时发送的消息的自动处理。如果您想执行对话框提供的以外的其他操作，则必须捕获操作区域按钮发送的消息，并在对话框中编写您自己的消息处理程序。

例如，如果您想在用户点击对话框中的**应用**按钮时执行某个操作，则必须捕获 (ID_CLICKED_APPLY | SEL_COMMAND) 消息并将其映射到对话框中的消息处理程序。如需更多信息，请参阅"目标和消息"，第 6.5.4 节。
