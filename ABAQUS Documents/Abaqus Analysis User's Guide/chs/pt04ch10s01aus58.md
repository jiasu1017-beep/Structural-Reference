### 概述

子结构技术是一种模型简化技术，它通过将一组单元替换为单个超单元来减少计算成本。子结构：

- 减少了模型的自由度数
- 保留了组内单元的详细行为
- 允许在后续分析中重复使用
- 主要用于线性静力和模态分析

### 子结构用途

子结构常用于：

1. **大型线性结构**：当大部分结构行为是线性的时候
2. **重复模式**：当结构包含重复的组件时
3. **局部细化**：当只需要某些区域的详细结果时
4. **参数研究**：当需要在不同载荷配置下重复分析时

### 子结构分析流程

1. **定义子结构区域**：选择将作为子结构的单元集
2. **生成子结构**：创建超单元矩阵
3. **使用子结构**：在主分析中引用子结构

### 子结构生成

子结构生成步骤创建超单元矩阵：

| **输入文件用法：** | ``` [*SUBSTRUCTURE GENERATE](../key/key-link.md#usb-kws-hsubstructuregenerate), ELSET=*element_set* ``` |

### 子结构使用

在后续分析中使用子结构：

| **输入文件用法：** | ``` [*SUBSTRUCTURE](../key/key-link.md#usb-kws-hsubstructure), ELSET=*element_set* ``` |

### 子结构界面

子结构界面是子结构与模型其余部分之间的连接点：

| **输入文件用法：** | ``` [*SUBSTRUCTURE GENERATE](../key/key-link.md#usb-kws-hsubstructuregenerate), ELSET=*element_set*, INTERFACE=*node_set* ``` |

### 输出

子结构分析的输出包括：

- 界面节点位移
- 界面节点反力
- 子结构内部单元的详细结果（可选）

| **输入文件用法：** | ``` [*OUTPUT](../key/key-link.md#usb-kws-houtput) [*ELEMENT OUTPUT](../key/key-link.md#usb-kws-helementoutput) ``` |

### 优势

| **优势** | **描述** |
| --- | --- |
| 自由度减少 | 显著减少分析规模 |
| 计算效率 | 加速分析和参数研究 |
| 保留细节 | 子结构内部保持完整行为 |
| 可重用性 | 同一子结构可用于多个分析 |

### 限制

| **限制** | **描述** |
| --- | --- |
| 线性假设 | 仅适用于线性问题 |
| 界面条件 | 界面必须明确定义 |
| 结果精度 | 内部结果可能需要恢复步骤 |
| 内存需求 | 矩阵生成可能需要大量内存 |

### 输入文件模板

#### 生成子结构

```
*-heading
Substructure generation
**
*SUBSTRUCTURE GENERATE, ELSET=PART1, INTERFACE=PART1_INT
*OUTPUT
*ELEMENT OUTPUT
*END STEP
```

#### 使用子结构

```
*-heading
Main analysis with substructure
**
*SUBSTRUCTURE, ELSET=PART1
*STEP
*STATIC
*BOUNDARY
LOADS on interface nodes
*END STEP
```
