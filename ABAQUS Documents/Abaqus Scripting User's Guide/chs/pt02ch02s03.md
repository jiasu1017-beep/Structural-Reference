# 2.3 plugin和定制应用程序





您可以通过两种方式使用Abaqus GUI Toolkit——通过使用plugin架构或创建自定义应用程序。Plugin工具集分层在Abaqus/CAE之上。首先构建Abaqus/CAE应用程序，然后Plugin工具集搜索特定目录，查找将项目添加到顶级**Plug-ins**菜单的文件。如果您的意图仅是向标准Abaqus/CAE应用程序添加功能，并且通过主菜单栏中的**Plug-ins**菜单提供对此功能的访问就足够了，那么Plugin工具集可能会满足您的需求。Plugin在[Abaqus/CAE用户指南](../usi/usi-link.md#usi-customgui)的"使用plugin"第八部分中有详细描述。

相比之下，要创建定制应用程序，您需要从头开始构建应用程序。如果您想除了向Abaqus/CAE添加功能外还想修改Abaqus/CAE的一些标准功能，则应该编写定制应用程序。虽然创建自定义应用程序提供了最大的灵活性，但它比使用Plugin工具集需要更多的工作。但是，自定义应用程序允许您修改Plugin工具集无法控制的应用程序方面。具体来说，自定义应用程序允许您执行以下操作：
- 移除Abaqus/CAE模块或工具集。创建自定义应用程序时，您可以确定哪些模块和工具集加载到应用程序中以及它们出现的顺序。
- 修改Abaqus/CAE模块或工具集。如果要向Abaqus/CAE模块添加或删除功能，必须从Abaqus/CAE模块派生您的模块，然后注册您的模块而不是Abaqus/CAE模块。如果您想向Abaqus/CAE工具集添加或删除功能，请遵循类似的过程。
- 更改应用程序名称和版本号。创建自定义应用程序时，您创建一个启动脚本来初始化应用程序对象的应用程序名称及其版本号。
- 控制启动命令和使用的许可证令牌。创建自定义应用程序时，您可以修改定义用于启动应用程序的命令的站点配置文件。您还可以修改同一站点配置文件以指定应用程序启动时签出的许可证令牌。
