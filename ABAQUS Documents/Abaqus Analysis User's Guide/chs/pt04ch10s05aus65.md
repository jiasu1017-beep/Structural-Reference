# 10.5.1 周期介质分析





**产品：** Abaqus/Explicit

**参考资料**

- [*PERIODIC MEDIA](../key/key-link.md#usb-kws-mperiodicmedia)
- [*MEDIA TRANSPORT](../key/key-link.md#usb-kws-mmediatransport)

### 概述

Abaqus/Explicit中的周期介质分析技术：
- 是一种拉格朗日技术，提供对移动结构的类欧拉视图；
- 可用于有效模拟重复性系统，如涉及传送带或连续成型操作的生产过程；
- 与可能需要过大网格的传统建模技术相比，显著加快分析时间；并且
- 需要拓扑相同的网格化部件来创建模型，这可以通过部件和实例建模范式实现。

### 引言

通常需要分析的行业流程涉及以简单模式重复并通过加工区的部分。一个突出的例子是传送带，上面有规律间隔的包装，如[图10.5.1-1](pt04ch10s05aus65.md#aperiodicmedia-def)示意性地所示，有限元网格示例见[图10.5.1-2](pt04ch10s05aus65.md#belt_transport)。金属轧制等连续成型操作也是很好的例子，因为变形材料可以分解为任意数量的相同部分。

为清楚起见，我们将在整个讨论中使用传送带示例来说明与周期介质分析技术相关的许多概念。[图10.5.1-1](pt04ch10s05aus65.md#aperiodicmedia-def)显示了传送带的概念分解；实际上，传送带是一个连续实体。

概念上，整体模型可以分解为多个块（拓扑相同的网格化结构），这些块连接在一起并跨越加工区。您创建一个定义"构建块"（重复网格化结构，用于对整个周期介质进行建模）的部件，然后通过适当定位和旋转的实例链构建整个模型。周期介质分析技术提供了一种简单的方法来自动连接这些实例在相邻块的前端和后端。该技术还提供了一种方便的方式来定义表示物理系统的载荷和边界条件，这些载荷和边界条件作用在链中第一个和最后一个块未连接的末端。链的第一个块称为入口，最后一个块称为出口。最后，当周期介质通过加工区时，块从出口自动重新排序到入口。使用此技术定义的块（网格化结构）可以通过接触与其他非周期性建模特征交互，如[图10.5.1-1](pt04ch10s05aus65.md#aperiodicmedia-def)中所述的辊。

周期介质分析技术的核心是块从出口重新排序到入口的概念。使用专用算法检测入口何时移动得太远进入加工区，并将块从出口直接重新排序到入口。[图10.5.1-1](pt04ch10s05aus65.md#aperiodicmedia-def)中的虚线箭头说明了重新排序过程。为确保平滑过渡，在当前步开始时存储入口块的必要节点和单元状态数据。重新排序发生时，存储的节点和单元状态数据映射到新入口块，任何入口/出口载荷或边界条件转移到新暴露的块末端。

因此，周期介质分析技术为移动重复结构提供了方便的类欧拉视图。例如，您可能希望评估传送带上某个位置（位于辊之间）处的包装动态。您在该位置周围定义几个块，根据需要定义与辊的接触，并提供适当的入口和出口加载条件。周期介质分析技术提供了一种方便且经济的方式来创建和分析此系统。通过使用重新排序过程离开加工区的元素，您可以避免纯拉格朗日模拟在入口端所需的大网格。

### 构建周期介质模型

构建周期介质模型的第一步是识别构成重复结构构建块的模型部分。在[图10.5.1-2](pt04ch10s05aus65.md#belt_transport)中，一个方形传送带贴片和上面的一个不对称形状包装构成了这样一个构建块。如果您串起多个块，则可以如所示对整个带和包装进行建模。

#### 定义构建块

定义每个构建块时必须遵守以下要求：
- 必须定义一个未排序的单元集以包含构建块中的所有单元，并且
- 必须定义一个未排序的节点集以包含构建块中的所有节点。

为确保周期介质前进时信息正确传递，这些未排序集合在所有块之间必须拓扑相同。实现此要求最简单的方法是使用部件和实例建模范式。您定义一个对应于构建块的部件，并如上所述定义未排序的单元和节点集。然后，根据需要实例化该部件，并进行适当的平移和旋转，以生成周期介质网格。约束（如绑定、耦合和刚体）允许在构建块内。您必须确保这些约束以所有块中拓扑相同的方式定义。

周期介质分析技术将这些否则未连接的块连接在一起以创建连续模型。如果在块的连接区域中使用结构元素（如壳），则这些区域的边缘上的节点连接到相邻区域。如果使用连续体元素，则这些区域面上的节点被连接。为了可靠地构建这些约束，必须遵守以下附加要求：
- 块的前端和后端连接端的节点排列必须拓扑相同，
- 相邻块的正面和背面端节点必须重合，
- 初始入口块正面和背面的节点排列只能通过刚体平移而有所不同，并且
- 必须使用未排序的节点集在每个块的前端和后端定义两个基于节点的表面。

基于节点的表面用于在相邻块之间自动生成节点对节点绑定约束，使整个 assembly 表现为连续实体。

| **输入文件用法：** | 使用以下选项使用上述未排序集合和表面定义块序列： |
| --- | --- |
|  | ``` [*PERIODIC MEDIA](../key/key-link.md#usb-kws-mperiodicmedia), NAME=*name* *elset1*, *nodeset1*, *frontsurf1*, *backsurf1* *elset2*, *nodeset2*, *frontsurf2*, *backsurf2* *...* *elsetn*, *nodesetn*, *frontsurfn*, *backsurfn* ``` 每个数据行提供与给定块关联的集合和表面名称。 |

#### 在介质末端施加载荷和边界条件

在[图10.5.1-1](pt04ch10s05aus65.md#aperiodicmedia-def)所示的示意性传送带中，通常需要在组件两端施加载荷或边界条件。在入口点I处，施加预张力以保持传送带张紧通常很有用，而在出口点O处，通常指定传送带速度。当传送带前进且出口块正从出口重新排序到入口时，需要边界条件的节点将发生变化。因此，这些边界条件和载荷不能直接施加在属于块的节点上。

周期介质分析技术允许通过与当前入口和出口基于节点的表面关联的两个控制节点来施加此类加载特征。控制节点类似于其他特征（如运动耦合）中使用的参考节点，并对组件末端节点自动施加类似刚体的约束。您可以在这些控制节点上施加载荷和边界条件。类似刚体的约束也施加在入口块的正面端节点上，但不能施加载荷或边界条件。当出口块重新排序到入口时，控制点将对新的极端端面强制执行这些类似刚体的约束，并从先前位置移除类似刚体的约束。该过程是自动的，由周期介质分析技术完全管理。

| **输入文件用法：** | 使用以下选项定义入口和出口条件的控制节点： |
| --- | --- |
|  | ``` [*PERIODIC MEDIA](../key/key-link.md#usb-kws-mperiodicmedia), INLET CONTROL NODE=*node*, OUTLET CONTROL NODE=*node* ``` |

#### 定义加工区

当入口块完全进入加工区时，出口块被重新排序到入口，如[图10.5.1-1](pt04ch10s05aus65.md#aperiodicmedia-def)中的虚线箭头所示。触发平面控制重新排序发生的确切时间。当位于当前入口点I的节点穿过触发平面时，启动重新排序过程。触发平面使用通常静止节点的坐标和用户定义方向的*z*轴来定义。局部*z*轴方向从入口指向加工区。

| **输入文件用法：** | 使用以下选项通过触发节点和方向定义触发平面： |
| --- | --- |
|  | ``` [*PERIODIC MEDIA](../key/key-link.md#usb-kws-mperiodicmedia), TRIGGER NODE=*node*, ORIENTATION=*orientation* ``` |

### 激活周期介质

重新排序过程可以按步激活。默认情况下，重新排序过程处于非活跃状态。在许多情况下，周期介质在运行条件下的配置只能通过模拟确定。这允许在激活重新排序过程之前执行任意数量的分析步。

[图10.5.1-2](pt04ch10s05aus65.md#belt_transport)和["介质传输，" Abaqus验证指南第3.25.1节](../ver/ver-link.md#ver-prc-mediatransport)中所示的示例显示了传送带运输最初以规律间隔放置的不对称包装。在运行条件下，传送带将被张紧。您可以在Abaqus/Standard或Abaqus/Explicit中对传送带组件进行预拉伸。如果预拉伸分析在Abaqus/Standard中进行，则需要明确定义相邻块之间的所有绑定以及入口和出口端节点处的边界条件，因为周期介质分析技术仅在Abaqus/Explicit中可用。如果预拉伸步在Abaqus/Explicit中进行，则在预拉伸步期间重新排序过程应保持非活跃状态。

| **输入文件用法：** | 使用以下选项激活或停用周期介质重新排序过程： |
| --- | --- |
|  | ``` [*MEDIA TRANSPORT](../key/key-link.md#usb-kws-mmediatransport) *periodic_media_name1*, ACTIVE *periodic_media_name2*, INACTIVE ... ``` |

### 建模提示

周期介质分析技术是一个强大的功能；但是，在使用它时必须运用良好的工程判断。以下评论和建议将帮助您避免使用此技术时的常见陷阱：
- 块重新排序过程固有地嘈杂，因为块元素在一端分离并在另一端重新连接。虽然该过程使用适当的材料和运动状态，但小的冲击是固有。建议使用少量质量比例阻尼来抑制这种激励。
- 入口控制节点处边界条件与加工区中任何载荷的组合应确保入口块穿过触发平面而不会改变方向。在传送带示例中，一个好的建模实践是将固定导向辊放置在距离触发平面至少两个块的位置。
- 对于更复杂的几何形状（如在辊之间改变方向的传送带或当传送带是包装材料本身的包装分析），可能需要从直序列块开始，然后将传送带辊（不属于周期介质定义）移动到所需位置。传送带和辊之间的接触相互作用会使传送带变形为所需配置。此附加分析步可以大大简化初始网格的定义。
- 有时可能需要模拟引导段穿过辊的穿带过程，就像物理现实中生产过程开始时一样。如果此引导段后面跟着包含实际包装的周期块，则可以将周期介质网格附加到常规网格以执行穿带。然后可以将周期介质部分导入到没有引导网格的单独模型中，并且可以执行仅包含包装和包装的周期介质的分析。

### 初始条件

可以在周期介质网格的所有节点上指定初始条件。速度初始边界条件可用于最小化达到稳态运行条件所需的求解时间。在需要预拉伸的情况下，从先前的分析导入而不是执行多步分析允许将初始条件应用到拉伸构型。由于周期介质定义不会被导入，因此必须在每个需要它们的分析中重新指定。

### 边界条件

入口和出口控制节点是与周期介质定义关联的仅有的两个可以指定边界条件的节点。此外，仅允许速度边界条件。您不得在与周期介质网格关联的任何其他节点上指定边界条件。当周期介质处于活跃状态时，如果寻求稳态解，这些边界条件应在方向和幅度上保持恒定，以减轻求解噪声。

### 载荷

仅集中载荷可以施加到入口和出口控制节点，以驱动或拉伸周期介质。当周期介质处于活跃状态时，这些载荷应在方向和幅度上保持恒定。可以根据需要施加重力载荷。也可以指定其他分布载荷；但是，您必须记住，载荷将随着块一起被重新排序。

### 材料选项

所有可用的材料模型都受支持。

### 限制条件

周期介质分析受以下限制条件约束：
- 块内仅允许膜、壳、桁架、连续体单元和刚性单元。如果适用，也可以使用钢筋层。
- 不允许在属于不同块的节点之间定义显式约束。
- 所有块必须以相同的方式定义质量缩放。

周期介质不应涉及：
- 定义热接触属性或耦合欧拉-拉格朗日接触的一般接触，或
- 通过接触对算法定义的接触。
