# 3.2.4 Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD协同仿真执行





**产品：**Abaqus/Standard  Abaqus/Explicit  Abaqus/CFD

##### **参考**

- [""Abaqus执行过程概述，""第3.1.1节"](pt01ch03s01abo02.md)

### 概述

Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD之间的协同仿真可以通过运行Abaqus协同仿真过程执行。几个参数可以在命令行上设置，也可以在环境文件中设置（参见["使用Abaqus环境设置，"第3.3.1节"](pt01ch03s03aus30.md)）。有关支持的协同仿真交互的详细信息，请参见["协同仿真概述，"第17.1.1节"](pt04ch17s01abo17.md)。

协同仿真分析执行指定的"子"分析，并根据协同仿真配置文件规范指导进程之间的通信。协同仿真执行过程允许您输入单个命令来运行协同仿真，应尽可能使用此方法（请参阅下面的["限制"](pt01ch03s02abx04.md#usb-int-dcosimproc-limit)）。如果您无法使用Abaqus协同仿真执行过程，则需要执行以下操作：
- 使用Abaqus执行过程（["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行，"第3.2.2节"](pt01ch03s02abx02.md)）单独提交协同仿真分析，和
- 调用SIMULIA协同仿真引擎（CSE）director（["SIMULIA协同仿真引擎director执行，"第3.2.3节"](pt01ch03s02abx03.md)）。

协同仿真执行过程支持Abaqus执行过程可用选项的子集；这些选项包含在下面的命令摘要中。

### 命令行参数的组成

对于参与的子分析，您可以使用逗号分隔的选项列表作为命令行参数。命令行参数（请参阅下面的命令行选项）分为以下几类：
- 全局选项，接受单个参数；或
- 适用于Abaqus子分析，接受逗号分隔的列表，其长度是参与的Abaqus作业数量。

### 为Abaqus作业的并行处理分配CPU

您的CPU分配规范适用于Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD子作业。三种方法可用于为并行处理分配子分析作业的CPU：为每个作业指定CPU数量、在分析作业之间分配CPU、以及在分析产品之间分配CPU。

#### 为每个作业指定CPU数量

分配CPU最直接的方法是指定每个子分析使用的CPU数量。您可以使用**cpus**参数提供逗号分隔的值列表。

#### 在分析作业之间分配CPU

您可以指定用于协同仿真分析的CPU总数，以及确定CPU在子分析之间分布的权重因子。此方法使您能够指定直接与资源限制相关的CPU数量，并描述子分析的相对计算需求。您使用**cpus**参数提供一个值用于分配给协同仿真的CPU数量，并使用**cpuratio**参数定义权重因子。

权重因子是浮点数，以归一化方式考虑。例如，如果您希望指定第一个子作业的CPU分配是第二个作业的四倍，则可以提供以下任意配对：

```
cpuratio=4.0,1.0
cpuratio=16,4
cpuratio=0.8,0.2
```

#### 在分析产品之间分配CPU

您可以指定用于协同仿真分析的CPU总数，以及权重因子来确定CPU在协同仿真涉及的分析产品之间的分布。此方法使您能够指定直接与资源限制相关的CPU数量，并根据所使用的分析产品（Abaqus/Standard、Abaqus/Explicit或Abaqus/CFD）描述子分析的相对计算需求。您使用**cpus**参数提供一个值用于分配给协同仿真的CPU数量，并使用环境文件中的**cpus_weight_std**、**cpus_weight_xpl**和**cpus_weight_cfd**环境变量参数定义权重因子（请参见["使用Abaqus环境设置"中的"协同仿真参数"，第3.3.1节"](pt01ch03s03aus30.md#usb-int-denvfile-cosim)。

权重因子以归一化方式解释。例如，如果您希望指定Abaqus/CFD分析的CPU分配是Abaqus/Explicit分析的两倍，则按如下方式在环境文件中定义参数：

```
cpus_weight_xpl=1
cpus_weight_cfd=2
```

#### 分配CPU的舍入考虑

在分析作业或分析产品之间的CPU分配不能导致整数的某些情况下，Abaqus会向下舍入**job**参数中列出的第一个作业的CPU分配，并向上舍入最后列出的作业的分配。例如，如果分配了8个CPU，并且Abaqus/CFD分析的CPU分配是Abaqus/Explicit分析的两倍，则Abaqus/Explicit和Abaqus/CFD之间的分布为2/6（如果Abaqus/Explicit作业首先列出），如果Abaqus/CFD作业首先列出，则为3/5。

### 为Abaqus子分析指定选项

与子分析相关的命令行选项要求您输入逗号分隔的值列表。对于所有子分析选项，列表中的条目数量和顺序必须一致，以获得所需的协同仿真执行行为。例如，在Abaqus/Standard到Abaqus/Explicit协同仿真中，如果将Abaqus/Standard分析的作业名称指定为作业参数的第一个条目，则其余子分析选项的第一个条目将应用于Abaqus/Standard分析。

如果某个选项仅与一个子分析相关，您可以在该选项不适用的分析中输入值`NONE`。如果您希望对所有子分析使用默认设置，或希望使用环境设置来控制行为，则无需在命令行中提供该选项。

### 使用指定协同仿真控制的输入文件执行协同仿真

在Abaqus 6.13之前的版本中，所有协同仿真参数和协同仿真控制都在输入文件中指定，不需要配置文件。为了支持在Abaqus 6.13及更高版本中重用这些输入文件，提供了自动配置文件创建功能。在有限的情况下，您可以自动生成配置文件；即，如果满足以下条件：
- 您的协同仿真由两个子Abaqus分析组成。
- 两个输入文件都包含对Abaqus 6.12协同仿真有效的[*CO-SIMULATION CONTROLS*](../key/key-link.md#usb-kws-hcosimulationcontrols)选项。
- 这些控制符合Abaqus 6.12分析用户手册中描述的协同仿真参数的推荐用法。

如果您省略协同仿真执行过程的**configure**选项，Abaqus会尝试自动生成配置文件，如果成功，生成的配置文件将直接用于分析。自动生成的配置文件的内容由[*CO-SIMULATION CONTROLS*](../key/key-link.md#usb-kws-hcosimulationcontrols)选项、分析过程类型和协同仿真步骤的持续时间决定。

### 限制

协同仿真执行过程有以下限制：
- 分析只能在一台机器上运行，也可以在头节点可由两个子分析作业共享的计算集群上运行。
- 此执行过程不支持与第三方应用程序的协同仿真；有关与第三方应用程序协同仿真的Abaqus作业执行的信息，请参阅第三方程序文档。

### 命令行选项

#### 必需的全局选项

**cosimjob**

此选项指定运行期间生成的协同仿真摘要日志文件的名称。如果命令行上省略了此选项，系统将提示您输入其值。

**configure**

此选项指定协同仿真配置文件的名称。如果命令行上省略了此选项，将尝试根据子分析输入文件中指定的协同仿真控制自动生成配置文件。如果成功，生成的配置文件将用于协同仿真；如果失败，系统将提示您输入值。更多信息，请参见["使用指定协同仿真控制的输入文件执行协同仿真"](pt01ch03s02abx04.md#usb-int-dcosimproc-config)。

#### Abaqus子分析必需的选项

**job**

此选项的逗号分隔值指定运行期间生成的所有Abaqus子分析文件的名称。如果命令行上省略了此选项，系统将提示您输入其值。

#### 并行处理选项

**cpus**

此选项用于指定协同仿真在并行处理期间如何分配CPU。此参数的默认值是Abaqus子分析的数量，可以更改环境文件中的值（参见["使用Abaqus环境设置，"第3.3.1节"](pt01ch03s03aus30.md)。

如果此选项设置为单个值，则该值指定分配给协同仿真的处理器总数，可以在子分析之间或分析产品之间分配。默认情况下，CPU在子分析之间的分配是均匀的，可以进一步使用**cpuratio**参数控制，或者通过设置**cpus_weight_std**、**cpus_weight_xpl**和**cpus_weight_cfd**环境文件参数定义分析产品之间的CPU分布来控制（请参见["使用Abaqus环境设置"中的"协同仿真参数"，第3.3.1节"](pt01ch03s03aus30.md#usb-int-denvfile-cosim)。

如果此选项设置为逗号分隔的值列表，则这些值指定每个子分析使用的处理器数量。

**cpuratio**

此选项的逗号分隔值指定分配给每个Abaqus子分析的处理器分布的相对权重。此选项仅在**cpus**选项设置为单个值时有效。

#### 其他可用的全局选项

**interactive**

此选项导致协同仿真作业交互运行。摘要日志文件将输出到屏幕，子分析摘要输出将写入其单独的日志文件。

**background**

此选项将协同仿真作业提交到后台运行，这是默认设置。协同仿真作业的日志文件输出保存在文件`*cosim-job-name*.log`中，子分析文件`*job-name*.log`保存在当前目录中。

**queue**

此选项将协同仿真作业提交到批处理队列。如果选项出现时没有值，作业将提交到系统默认队列。允许使用带引号的字符串。可用队列是特定于站点的。请联系您的站点管理员以了解有关本地排队功能的更多信息。

**after**

此选项与**queue**选项一起使用，以指定作业将在所选批处理队列中开始的时间。此功能通过Abaqus环境文件为每个站点提供支持。（详细信息请参阅[Abaqus安装和许可指南](../sgb/sgb-link.md#sgb)。）

**timeout**

此选项用于指定协同仿真连接的超时值（以秒为单位）。如果在指定时间内未收到子分析进程之间的任何通信，Abaqus将终止。默认值是3600秒。Abaqus使用的默认超时值可以使用环境文件中的**cosimulation_timeout**参数设置（参见["使用Abaqus环境设置，"第3.3.1节"](pt01ch03s03aus30.md)）。

**listenerport**

此选项用于指定协同仿真director进程使用的TCP/UDP端口号。此选项可用于启用其他协同仿真分析，例如第三方分析作业，加入使用此过程启动的协同仿真。如果未指定此选项，则listenerport值基于可用的开放socket端口以及您使用**portpool**选项设置的任何约束来确定。

**portpool**

此选项用于指定冒号分隔的一对TCP/UDP端口号，表示建立子进程之间连接时使用的端口号范围的起始值和结束值。默认范围是`51000:52000`。Abaqus使用的默认范围可以使用环境文件中的**portpool**参数设置（参见["使用Abaqus环境设置，"第3.3.1节"](pt01ch03s03aus30.md)）。

#### Abaqus子分析的其他选项

**input**

此选项的逗号分隔值指定Abaqus子分析输入文件名，可以带也可以不带`.inp`扩展名（如果未提供扩展名，Abaqus会自动附加）。对于每个子分析，如果未提供此选项，过程将在当前目录中查找名为`*job-name*.inp`的输入文件。如果找不到`*job-name*.inp`，过程将提示输入输入文件名。

**user**

此选项的逗号分隔值指定包含分析中使用的用户子程序的FORTRAN源或目标文件名称。用户例程的名称可以包含路径名，可以带也可以不带文件扩展名。此选项不适用于Abaqus/CFD。

**globalmodel**

此选项的逗号分隔值指定全局模型的结果（`.fil`）文件或输出数据库（`.odb`）文件的名称，从中内插结果以驱动子模型分析。每当子模型分析或子模型边界条件从全局模型的结果读取数据时，都需要此选项。文件扩展名是可选的。如果全局模型同时存在结果文件和输出数据库文件且未给出扩展名，则将使用结果文件。此选项不适用于Abaqus/CFD。

**memory**

此选项的逗号分隔值指定输入文件预处理期间和Abaqus/Standard分析阶段可以分配的最大内存量或物理内存的最大百分比（请参见["在Abaqus中管理内存和磁盘使用，"第3.4.1节"](pt01ch03s04aus31.md)）。此选项不适用于Abaqus/CFD。

**oldjob**

此选项的逗号分隔值指定先前运行的文件名，从中将启动重新启动运行或从中导入结果。不允许使用路径或文件扩展名。当重新启动或导入分析从重新启动文件读取数据时，需要此选项。*oldjob-names*必须与当前的*job-names*不同。

**double**

此选项仅适用于Abaqus/Explicit分析。

此选项的逗号分隔值指定要使用的双精度可执行文件设置；Abaqus/Standard或Abaqus/CFD分析的值始终是`NONE`。Abaqus/Explicit分析的可能值是`both`、`constraint`、`explicit`和`off`。此功能也通过环境变量**double_precision**在Abaqus环境文件中支持（参见["使用Abaqus环境设置，"第3.3.1节"](pt01ch03s03aus30.md)。

如果Abaqus/Standard到Abaqus/Explicit协同仿真省略了**double**选项，则Abaqus/Explicit打包程序和分析将以双精度运行。如果Abaqus/CFD到Abaqus/Explicit协同仿真省略了**double**选项，则Abaqus/Explicit打包程序和分析将以单精度运行。

如果**double**=`both`，Abaqus/Explicit打包程序和分析都将以双精度运行。

如果**double**=`constraint`，Abaqus/Explicit中的约束打包和约束求解器将以双精度运行，而Abaqus/Explicit打包程序和Abaqus/Explicit分析继续以单精度运行。

如果**double**=`explicit`或**double**选项指定时没有值，Abaqus/Explicit分析将以双精度运行，而打包程序仍将以单精度运行。

如果**double**=`off`，则在必要时覆盖环境文件设置，以单精度调用Abaqus/Explicit打包程序和Abaqus/Explicit分析。有关何时使用双精度可执行文件的讨论，请参见["定义分析，"第6.1.2节"](pt03ch06s01abo05.md)。

**scratch**

此选项的逗号分隔值指定用于临时文件的目录名称。在UNIX平台上，默认值是`$TMPDIR`环境变量的值；如果未定义`$TMPDIR`，则为`/tmp`。在Windows平台上，默认值是`%TEMP%`环境变量的值；如果未定义此变量，则为`\TEMP`。在分析期间，将在此目录下创建一个子目录来存放分析临时文件。

**output_precision**

此选项的逗号分隔值指定写入输出数据库文件（`*job-name*.odb`）的节点场输出的精度。使用**output_precision**=`full`可为Abaqus/Standard分析生成双精度场输出。要为Abaqus/Explicit分析获得双精度场输出，除了使用**output_precision**=`full`外，还要使用**double**选项。节点历史输出仅以单精度提供。此选项不适用于Abaqus/CFD。

**field**

此选项仅适用于Abaqus/CFD分析。

此选项的逗号分隔值指定场输出的格式；Abaqus/Standard或Abaqus/Explicit分析的值始终是`NONE`。Abaqus/CFD分析的可能值是`odb`、`exodus`和`nemesis`。

如果**field**=`odb`，场输出将写入输出数据库文件。如果**field**=`exodus`，场输出将写入EXODUS-II格式的文件，每个处理器一个文件。要获取并行执行的单个文件，请使用**field**=`nemesis`；文件使用NEMESIS库以EXODUS-II格式写入。默认值是`odb`。更多信息，请参见["输出"中的"Abaqus/CFD中的替代输出格式"，第4.1.1节"](pt02ch04s01aus38.md#usb-out-ooutput-alternate)。

**history**

此选项仅适用于Abaqus/CFD分析。

此选项的逗号分隔值指定历史输出的格式；Abaqus/Standard或Abaqus/Explicit分析的值始终是`NONE`。Abaqus/CFD分析的可能值是`odb`和`csv`。

如果`history`=`odb`，历史输出将写入输出数据库文件。如果`history`=`csv`，历史输出将写入逗号分隔值格式的文件。

默认值取决于**field**选项的设置。当**field**=`odb`时，默认值是`history`=`odb`。当**field**=`exodus`或`nemesis`时，默认值是`history`=`csv`。更多信息，请参见["输出"中的"Abaqus/CFD中的替代输出格式"，第4.1.1节"](pt02ch04s01aus38.md#usb-out-ooutput-alternate)。

### 示例

以下示例说明了**abaqus cosimulation**执行过程的不同功能和能力。

#### 交互运行Abaqus/Standard到Abaqus/CFD协同仿真

使用以下命令在名为"solid_heat"的热传递分析和名为"fluid"的流体分析之间运行协同仿真，交互进行：

```
abaqus cosimulation cosimjob=cosim_cht 
   job=solid_heat,fluid configure=cosim_cht_config interactive
```

#### 在Abaqus/Explicit到Abaqus/CFD协同仿真中分配CPU

使用以下命令在名为"beam"的Abaqus/Explicit分析和名为"fluid"的Abaqus/CFD分析之间运行协同仿真，并为Abaqus/Explicit作业分配8个核心，为Abaqus/CFD作业分配16个核心：

```
abaqus cosimulation cosimjob=beam_fluid job=beam,fluid cpus=8,16
   configure=beam_fluid_config
```
使用以下命令将获得等效结果：
```
abaqus cosimulation cosimjob=beam_fluid job=beam,fluid 
   cpus=24 cpuratio=1,2 configure=beam_fluid_config
```
或者，您可以在环境文件中为协同仿真环境变量参数指定设置，并运行协同仿真执行过程。使用以下环境文件设置组合：
```
ask_delete=OFF
# The following parameters set the CPU
# allocation by analysis product
cpus_weight_xpl=1
cpus_weight_std=1
cpus_weight_cfd=2
```
使用以下命令：
```
abaqus cosimulation cosimjob=beam_fluid 
   configure=beam_fluid_config job=beam,fluid cpus=24 
```

#### 将Abaqus/Standard到Abaqus/Explicit协同仿真提交到批处理队列

使用以下命令将名为"beam"的Abaqus/Explicit分析和名为"beam2"的Abaqus/Standard分析的协同仿真提交到名为"long"的批处理队列，并为Abaqus/Explicit分析分配8个核心，为Abaqus/Standard分析分配4个核心：

```
abaqus cosimulation cosimjob=beam job=beam,beam2 
   configure=beam_config cpus=8,4 queue=long
```
