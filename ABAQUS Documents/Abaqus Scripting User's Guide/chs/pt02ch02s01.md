# 2.1 kernel和GUI





Abaqus/CAE在两个独立的进程中执行：kernel和GUI。kernel的角色是提供对Abaqus数据库和创建修改这些数据库的命令的访问。GUI的角色是收集用户输入，然后将其打包为命令字符串并发送到kernel执行。GUI对于Abaqus的执行不是必需的——整个模型可以通过kernel脚本构建、分析和后处理，而无需调用GUI。

通常，当您开发一些自定义功能时，首先创建实现该功能的kernel命令。这些命令可以通过从Abaqus/CAE中的命令行界面（CLI）执行来调试。一旦您确定kernel命令从CLI正确工作，您就可以设计GUI来收集命令所需的用户输入。
