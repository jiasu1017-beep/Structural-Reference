# 1.5 支持

Abaqus的技术工程支持（用于创建模型或执行分析的问题）和系统支持（用于安装、许可和硬件相关问题）通过支持办公室网络提供。区域联系信息可从www.3ds.com/simulia上的"位置"页面获取。也可以从www.3ds.com/simulia上的"支持"页面获得支持。联系您的支持办公室时，请指定您需要技术支持（遇到执行Abaqus分析的问题）还是系统支持（Abaqus无法正确安装、许可无法正常工作或其他硬件相关问题）。

我们欢迎对Abaqus软件、支持程序或文档的任何改进建议。我们将确保考虑您提出的任何增强请求以供将来发布。如果您想对SIMULIA提供的服务或产品提出建议，请参阅www.3ds.com/simulia。投诉应通过联系您的支持办公室或访问www.3ds.com/simulia上的"质量保证"页面提出。

## 1.5.1 技术支持

SIMULIA技术支持工程师可以协助澄清Abaqus功能并通过提供使用Abaqus的一般信息及其在特定分析中的应用信息来检查错误。如果您对分析有疑虑，建议您尽早联系我们，因为在项目开始时解决问题通常比在分析结束时尝试纠正分析更容易。

在致电技术支持热线之前，请准备好以下信息，并将其包含在任何书面联系中：
- 您正在使用的Abaqus版本。- Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD的版本号显示在数据（`.dat`）文件的顶部。
- 您正在运行Abaqus的计算机类型。
- 任何问题的症状，包括确切的错误消息（如果有的话）。
- 您已经尝试的解决方法或测试。

对于特定问题的支持，任何可用的Abaqus输出文件都可能有助于回答支持工程师可能提出的问题。

支持工程师将尝试根据模型描述和您遇到的困难描述来诊断您的问题。通常，支持工程师需要模型草图，可以通过电子邮件、传真或邮寄发送。最终结果的图或分析终止附近的结果图也可能需要了解可能导致问题的原因。

如果支持工程师无法根据此信息诊断您的问题，可能会要求您提供输入数据。数据可以附加到在线系统的支持请求中。也可以通过电子邮件、ftp、CD或DVD发送。

所有支持请求都会被跟踪。此跟踪使您（以及支持工程师）能够监控特定问题的进度并检查我们是否有效解决支持问题。您必须注册系统才能检查支持问题。如果您通过系统外部的方式联系我们以讨论现有支持问题，并且您知道支持请求编号，请提及它，以便我们可以查询数据库以查看最新操作。

## 1.5.2 系统支持

Abaqus系统支持工程师可以帮助您解决与Abaqus的安装和运行相关的问题，包括许可困难，这些不在技术支持范围内。

您应该仔细按照《Abaqus安装和许可指南》中的说明安装Abaqus。如果您遇到安装或许可问题，首先查阅《Abaqus安装和许可指南》中的说明以确保正确遵循了这些说明。如果此方法无法解决问题，请在Dassault Systmes知识库（www.3ds.com/support/knowledge-base）中搜索已知的安装问题。如果此方法无法解决您的情况，请在在线系统中创建支持请求。发送任何可用于定义问题的信息：中止分析的错误消息或所遇问题的详细说明。在可能的情况下，请发送**abaqus info**=**support**命令的输出。

## 1.5.3 对学术机构的支持

根据学术许可协议的条款，我们不为学术机构用户提供支持，除非该机构也购买了技术支持。请联系我们获取更多信息。
