# 40.5 理解可视化模块性能





通常，可视化模块中的后处理和图形结果显示速度对于大多数模型来说是相当令人满意的。但是，通常有必要在高性能水平和详细结果绘图之间取得平衡。根据您的后处理需求，您可能希望以性能为代价修改默认显示选项。有许多可用的选项会影响图形显示速度。

为了在可视化模块中获得最佳性能，我们建议以下操作：
- 在后处理期间将结果缓存到内存中，以加快屏幕上图像的生成。请参见"理解结果缓存"，第42.6.4节。
- 尽可能使用纹理映射云图方法。同样，避免使用纹理映射方法不支持的显示选项：线型云图、云图边缘、CAXA或SAXA单元，或围绕其质心收缩的单元。如果使用了此类显示选项，Abaqus/CAE将覆盖用户对纹理映射云图的设置，转而使用较慢的镶嵌方法。请参见"理解云图的渲染方式"，第44.1.2节。
- 对您选择显示的模型标签和符号要有选择性。绘制的模型实体越多，屏幕刷新所需的时间就越长。这同样适用于单元边缘。请参见"控制模型实体的显示"，第55.10节。
- 使用较高的结果平均阈值。结果越连续，云图显示就越快。请参见"理解结果值平均"，第42.6.2节。
- 使用状态字段输出变量而不是基于结果值创建显示组，以实现高性能时间历史动画。Abaqus/CAE在显示每个结果帧时重新计算基于结果的显示组，因此使用此类显示组会降低动画性能。状态字段输出变量允许您从模型绘图中移除满足基于结果的失效标准的单元。请参见"选择状态字段输出变量"，第42.5.6节。
- 不要使用远程显示。不支持此Abaqus/CAE配置；其使用由用户自行决定。使用远程显示永远无法实现性能优化。

在某些情况下，由于机器上的物理内存限制，非常大模型的基于单元的结果绘图可能无法达到最佳性能。增加物理内存或退出其他消耗内存的应用程序可以帮助恢复最佳性能。
