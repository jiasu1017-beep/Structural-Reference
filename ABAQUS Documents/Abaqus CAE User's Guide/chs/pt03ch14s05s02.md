# 14.5.2 重启动输出请求





你可以使用Abaqus创建的重启动文件从前一个分析的指定步骤继续分析。本节介绍如何控制重启动数据的输出。关于如何在后续作业中使用重启动数据的讨论，请参见["重新启动分析，"第19.6节](pt03ch19s06.md)和["什么是模型属性？"第9.2.2节](pt02ch09s02hlb01.md)。

默认情况下，不会为Abaqus/Standard或Abaqus/CFD分析写入重启动信息，而仅为Abaqus/Explicit分析在每个步骤的开始和结束时写入重启动信息。但是，默认重启动请求会为分析中的每个步骤自动创建。**编辑重启动请求**对话框通过在步骤模块中从主菜单栏选择****Output**![](../graphics/images/arrow.gif)**Restart Requests****来调用，允许你指定希望写入重启动信息的频率。

你可以指定Abaqus写入重启动文件的数据的频率；但是，重启动的行为因分析产品而异。

**Abaqus/Standard和Abaqus/CFD**

你可以请求以增量或时间间隔指定频率。对于Abaqus/Standard步骤，你可以选择是按精确时间间隔还是按最接近的近似值写入输出。

**Abaqus/Explicit**

对于Abaqus/Explicit分析，你可以指定Abaqus写入重启动文件的数据的等间距时间间隔的数量。此外，对于Abaqus/Explicit步骤，你可以选择是按精确时间间隔还是按最接近的近似值写入输出。但是，你不能避免为Abaqus/Explicit步骤写入重启动文件的信息；时间间隔数量必须设置为1或更大。

对于Abaqus/Standard或Abaqus/Explicit分析，你可以请求将写入重启动文件的数据覆盖先前增量的数据。如果你选择此选项，Abaqus仅在重启动文件中保留每个步骤的一个增量的信息，从而最小化文件大小。默认情况下，Abaqus不会覆盖数据。

更多信息，请参见["重新启动分析，"第19.6节](pt03ch19s06.md)和["重新启动分析，"Abaqus分析用户指南第9.1.1节](../usb/usb-link.md#usb-anl-arestart)。有关请求重启动输出的详细说明，请参见["配置重启动输出请求，"第14.13.2节](pt03ch14s13hlb02.md)。

你可以使用**abaqus restartjoin**执行程序从重启动分析创建的输出数据库中提取数据，并将数据追加到第二个输出数据库。更多信息，请参见["连接重启动分析的输出数据库(`.odb`)文件，"Abaqus分析用户指南第3.2.21节](../usb/usb-link.md#usb-int-drestartjoinproc)。
