### 概述

Abaqus/Standard中的接触约束施加方法：
- 作为表面相互作用定义的一部分进行指定；
- 决定如何在分析中数值求解物理压力-过盈关系（参见["接触压力-过盈关系，" 第37.1.2节](pt09ch37s01aus166.md)）施加的接触约束；
- 可以严格施加或近似物理压力-过盈关系；
- 可以修改以解决因过度约束导致的收敛困难；并且
- 有时使用拉格朗日乘子自由度。

本节详细讨论Abaqus/Standard中法向接触的可用约束施加方法。Abaqus/Standard中的摩擦约束施加方法独立于法向接触约束进行分配，并在["摩擦行为，" 第37.1.5节](pt09ch37s01aus169.md)中讨论。接触计算中拉格朗日乘子的使用也在本节中介绍。

### Abaqus/Standard中可用的约束施加方法

Abaqus/Standard中有三种接触约束施加方法可用：
- 直接方法尝试严格施加给定的压力-过盈行为，每个约束不进行近似或使用增强迭代。
- 罚方法是对硬接触的刚性近似。
- 增强拉格朗日方法使用与罚方法相同类型的刚性近似，但也使用增强迭代来提高近似的精度。

默认约束施加方法取决于相互作用特性，如下：
- 如果使用"硬"压力-过盈关系，则默认使用有限滑动、表面对表面接触（包括通用接触）的罚方法。
- 如果使用"硬"压力-过盈关系，则默认对使用节点-表面离散的二维自接触使用增强拉格朗日方法。
- 在所有其他情况下，直接方法是默认方法。

选择接触施加方法时应考虑以下因素：
- 直接方法必须用于具有"软化"压力-过盈关系的接触对（参见["接触压力-过盈关系，" 第37.1.2节](pt09ch37s01aus166.md)）。
- 直接方法严格施加与约束公式一致的规定压力-过盈行为
- 罚或增强拉格朗日约束施加方法有时可以提供更有效的解决方案（通常由于每次迭代的计算成本降低和每个分析的整体迭代次数减少），但代价是牺牲一定的解精度。参见下面关于罚方法和增强拉格朗日方法的讨论。
- 应避免因重叠接触定义或接触与其他约束类型的组合而导致的过度约束（参见["过度约束检查，" 第35.6.1节](pt08ch35s06aus138.md)），对于直接施加的硬接触。

### 直接方法

直接方法严格施加给定的压力-过盈行为每个约束，不进行近似或使用增强迭代。

#### 直接方法用于硬压力-过盈行为

直接方法可用于严格施加"硬"压力-过盈关系。在这种情况下始终使用拉格朗日乘子。

#### 直接方法用于软化压力-过盈关系

直接方法是唯一可用于施加"软化"压力-过盈关系的方法。直接方法可用于建模软化接触行为，无论接触公式的类型如何；但是，使用容易过度约束的接触公式建模刚性界面行为可能很困难。如果压力-过盈曲线的斜率超过基础元素刚度的1000倍（由Abaqus/Standard计算），则使用拉格朗日乘子；否则，在没有拉格朗日乘子的情况下施加约束。因此，拉格朗日乘子的使用取决于接触压力。关于软化压力-过盈关系的更多详细信息，请参见["接触压力-过盈关系，" 第37.1.2节](pt09ch37s01aus166.md)。

#### 直接方法的局限性

由于对接触约束的严格解释，使用直接施加方法的硬接触模拟容易出现过度约束问题。因此，对于使用节点-表面离散的二维自接触定义的接触对，直接施加的硬接触不可用。在这种情况下，您可以使用替代施加方法或使用软化压力-过盈关系的直接方法。

您可能会遇到类似的对称主-从接触对的过度约束问题（参见["使用对称主-从接触对改善接触建模" in "在Abaqus/Standard中定义接触对，" 第36.3.1节](pt09ch36s03aus145.md#usb-cni-acontactpair-symm)）。尽管直接施加的硬接触是这些接触对的默认方法，但建议您使用替代施加方法或软化接触关系。

某些二阶元素面在直接施加的硬接触关系中表现不佳。有关此问题的详细信息，请参见["具有二阶面的三维表面和节点-表面公式" in "Abaqus/Standard中接触建模的常见困难，" 第39.1.2节](pt09ch39s01aus184.md#usb-cni-acontacttrouble-3dsurf)。

### 罚方法

罚方法近似硬压力-过盈行为。使用此方法时，接触力与穿透距离成正比，因此会发生一定程度的穿透。罚方法的优点包括：
- 与罚方法相关的数值软化可以减轻过度约束问题并减少分析所需的迭代次数。
- 可以实现罚方法而不使用拉格朗日乘子，从而提高求解器效率。

#### 选择罚方法

Abaqus/Standard提供线性罚方法和非线性罚方法。对于线性罚方法，所谓的罚刚度是常数，因此压力-过盈关系是线性的。对于非线性罚方法，罚刚度在恒定低初始刚度和恒定高最终刚度区域之间线性增加，导致非线性压力-过盈关系。默认罚方法是线性的。

当使用线性罚方法时，Abaqus/Standard默认将罚刚度设置为基础元素刚度的10倍。您可以按比例调整或重新指定罚刚度，如下面的["修改线性罚刚度"](pt09ch38s01aus178.md#usb-cni-acontactconstraints-mod-lin-stiff)"中所述。默认罚刚度导致的接触穿透在大多数情况下不会显著影响结果；但是，这些穿透有时可能导致一定程度的应力不准确（例如，在位移控制的加载和粗网格情况下）。线性罚方法默认用于有限滑动、表面对表面接触公式。

##### 在第一次增量的迭代中调整罚刚度

如果在初始加载时接触状态在大部分接触区域发生变化，则分析第一次增量中通常会出现收敛困难。一种在不牺牲精度的情况下改善收敛行为的方法是：在第一次增量的早期迭代中使用减小的罚刚度，并在第一次增量的最终迭代和后续增量的所有迭代中返回默认罚刚度。在早期迭代中使用减小的罚刚度有助于稳健地找到近似的接触状态分布，而后期迭代的目标是找到精确解，作为第一次增量的收敛解报告。

| **输入文件用法：** | 在第一次增量内缩放罚刚度： |
| --- | --- |
|  | ``` [*CONTACT CONTROLS](../key/key-link.md#usb-kws-hcontactcontrols), STIFFNESS SCALE FACTOR=USER ADAPTIVE ``` |

#### 罚方法的局限性

罚方法不能用于脱粘表面。

如果指定了罚方法，则在使用以下过程的分析步骤中始终使用拉格朗日乘子：
- 设计灵敏度分析（参见["设计灵敏度分析，" 第19.1.1节](pt04ch19s01aus107.md)）
- 直接稳态动力学分析（参见["直接解稳态动力学分析，" 第6.3.4节](pt03ch06s03at09.md)）
- 准牛顿方法（参见["非线性问题的收敛准则，" 第7.2.3节](pt03ch07s02aus51.md)）

如果使用表面元素在外子结构外部定义接触表面（参见["如果存在子结构时的接触建模，" 第36.3.9节](pt09ch36s03aus153.md)），Abaqus/Standard将基础元素刚度解释为零。这可能导致确定默认罚刚度困难，并可能在分析期间导致数值问题。

### 增强拉格朗日方法

线性罚方法可以在增强迭代方案中使用，以减少穿透距离。所谓的增强拉格朗日方法仅适用于硬压力-过盈关系。以下描述了使用此方法时每个增量中发生的序列：

1. Abaqus/Standard使用罚方法找到收敛解。
2. 如果从属节点穿透主表面的距离超过指定穿透容差，则接触压力被"增强"，并执行另一系列迭代直到再次达到收敛。
3. Abaqus/Standard继续增强接触压力并找到相应的收敛解，直到实际穿透小于穿透容差。

增强拉格朗日方法在某些情况下可能需要额外迭代；但是，此方法可以更容易地解决接触条件并避免过度约束问题，同时保持穿透较小。增强拉格朗日方法默认用于使用节点-表面离散的二维自接触。

默认穿透容差是特征界面长度的千分之一，但以下情况除外：
- 如果指定小于1.0的罚刚度比例因子![](../graphics/usb_eqn06358.gif)（使用下面讨论的界面），Abaqus/Standard将自动按![](../graphics/usb_eqn06359.gif)的因子缩放默认穿透容差（将大于或等于1.0）；
- 有限滑动、表面对表面接触的默认穿透容差是特征界面长度的百分之五，受上一bullet点中讨论的缩放约束。

增强拉格朗日方法的默认罚刚度是基础元素刚度的1000倍。如果罚刚度超过Abaqus/Standard计算的基础元素刚度的1000倍，则对增强拉格朗日方法使用拉格朗日乘子；否则，不使用拉格朗日乘子。因此，对于具有默认罚刚度的增强拉格朗日方法，不使用拉格朗日乘子。

#### 增强拉格朗日方法的局限性

增强拉格朗日方法不能用于脱粘表面。

如果指定了增强拉格朗日方法，则在使用以下过程的分析步骤中始终使用拉格朗日乘子：
- 设计灵敏度分析（参见["设计灵敏度分析，" 第19.1.1节](pt04ch19s01aus107.md)）
- 直接稳态动力学分析（参见["直接解稳态动力学分析，" 第6.3.4节](pt03ch06s03at09.md)）
- 准牛顿方法（参见["非线性问题的收敛准则，" 第7.2.3节](pt03ch07s02aus51.md)）

如果使用表面元素在外子结构外部定义接触表面（参见["如果存在子结构时的接触建模，" 第36.3.9节](pt09ch36s03aus153.md)），Abaqus/Standard将基础元素刚度解释为零。这可能导致确定默认罚刚度困难，并可能在分析期间导致数值问题。
