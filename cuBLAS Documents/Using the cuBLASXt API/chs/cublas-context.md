### 2.1.2. cuBLAS 上下文

应用程序必须通过调用 `cublasCreate()` 函数来初始化 cuBLAS 库上下文的句柄。然后，该句柄会被显式传递给每个后续的库函数调用。一旦应用程序完成使用该库，它必须调用 `cublasDestroy()` 函数来释放与 cuBLAS 库上下文相关的资源。

这种方法允许用户在多个主机线程和多个 GPU 环境下显式控制库设置。例如，应用程序可以使用 `cudaSetDevice()` 将不同的设备与不同的主机线程关联起来，在每个主机线程中初始化一个唯一的 cuBLAS 库上下文句柄，该上下文将使用与该主机线程关联的特定设备。随后，使用不同句柄调用的 cuBLAS 库函数会自动将计算分发到不同的设备。

与特定 cuBLAS 上下文关联的设备应在相应的 `cublasCreate()` 和 `cublasDestroy()` 调用之间保持不变。为了让 cuBLAS 库在同一主机线程中使用不同的设备，应用程序必须通过调用 `cudaSetDevice()` 设置要使用的新设备，然后通过调用 `cublasCreate()` 创建另一个与新设备关联的 cuBLAS 上下文。当多个设备可用时，应用程序必须确保在调用与此上下文相关的 cuBLAS 函数之前，与给定 cuBLAS 上下文关联的设备是当前设备（例如通过调用 `cudaSetDevice()`）。

cuBLAS 库上下文与 `cublasCreate()` 调用时的当前 CUDA 上下文紧密耦合。使用多个 CUDA 上下文的应用程序需要为每个 CUDA 上下文创建一个 cuBLAS 上下文，并确保前者不会比后者存在更久。从版本 12.8 开始，cuBLAS 会检测底层 CUDA 上下文是否与图形上下文绑定，并在这种情况下遵循相应的共享内存大小限制。
