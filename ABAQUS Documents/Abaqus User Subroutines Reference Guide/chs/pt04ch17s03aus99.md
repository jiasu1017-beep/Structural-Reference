### 概述

两个结构求解器之间的协同仿真（求解器交换位移和/或旋转以及共轭场的力和/或力矩）代表了非常强的物理耦合，需要在协同仿真接口处进行特殊处理。Abaqus 通过提供专用接口处理支持 Abaqus/Standard 与 Abaqus/Explicit 之间的协同仿真。虽然您可以在两个 Abaqus/Standard 分析或两个 Abaqus/Explicit 分析之间执行结构-结构协同仿真，但由于接口处缺乏适当的处理，不推荐这样做。

本节讨论特定于 Abaqus/Standard 到 Abaqus/Explicit 协同仿真的分析设置、执行和限制详细信息。

有关 Abaqus/Standard 到 Abaqus/Explicit 协同仿真示例，请参阅 ["滑板车与凸起物的动态撞击，" Abaqus 例题指南第 2.4.1 节"](../exa/exa-link.md#exa-dyn-scooterbump)。

### 识别协同仿真接口区域

Abaqus/Standard 和 Abaqus/Explicit 模型之间的相互作用通过公共接口区域进行。

当耦合 Abaqus/Standard 到 Abaqus/Explicit 时，您可以使用节点集或表面指定接口区域。但是，您必须在 Abaqus/Standard 和 Abaqus/Explicit 中保持区域定义一致；如果在一个分析中用节点集或基于节点的表面定义协同仿真区域，则必须在另一个分析中使用相同类型的协同仿真区域定义。对于基于节点的表面，节点必须重合，因为没有拓扑信息来保守地映射 Abaqus/Standard 和 Abaqus/Explicit 模型之间的场。同样，如果在一个分析中用基于单元的表面定义协同仿真区域，则必须在另一个分析中用基于单元的表面定义协同仿真区域。

在某些情况下，Abaqus/Standard 和 Abaqus/Explicit 模型定义中共享的区域可能具有不同的网格。然而，在某些情况下，通过确保在接口处有匹配的节点（参见 ["不同网格相关的限制"](pt04ch17s03aus99.md#usb-anl-acosimabqtoabq-dissimilar)），可以提高求解的稳定性和准确性。在这些情况下，您可以使用 ["确保接口区域处的匹配节点，" Abaqus/CAE 用户指南第 26.4 节"](pt04ch17s03aus99.md#usi-adv-cosim-mesh) 中描述的建模实践来确保这些匹配的节点。

| **输入文件用法：** | 使用以下选项在 Abaqus 模型中将基于单元的表面或基于节点的表面定义为协同仿真区域： |
| --- | --- |
|  | ``` [*CO-SIMULATION REGION](../key/key-link.md#usb-kws-hcosimulationregion), TYPE=SURFACE (default) *surface_A* ``` 使用以下选项在 Abaqus 模型中将节点集定义为协同仿真区域：``` [*CO-SIMULATION REGION](../key/key-link.md#usb-kws-hcosimulationregion), TYPE=NODE *nodeset_A* ``` 每个 Abaqus 分析只能定义一个 [*CO-SIMULATION REGION](../key/key-link.md#usb-kws-hcosimulationregion) 选项。此外，只能定义一个节点集或表面。 |

| **Abaqus/CAE 用法：** | 相互作用模块：**创建相互作用**：**Standard-Explicit 协同仿真**：**表面**或**节点区域**：选择区域 |

### 识别跨协同仿真接口交换的场

对于 Abaqus/Standard 到 Abaqus/Explicit 协同仿真，您不定义交换的场；而是根据所使用的程序和协同仿真参数自动确定。

### 定义会合方案

对于结构-结构协同仿真，您必须为每个分析指定协同仿真控制并创建配置文件；参数的一致性在执行期间确认。SIMULIA Co-Simulation Engine 配置文件用于定义时间增量过程和两个 Abaqus 分析之间交换的频率。Abaqus/CAE 自动创建和使用此配置文件。如果您不使用 Abaqus/CAE 执行协同仿真，则必须手动创建配置文件。

预定义模板可用于以下协同仿真方案：
- 您可以强制 Abaqus/Standard 使用与 Abaqus/Explicit 相同的增量大小，或者
- 您可以允许 Abaqus/Standard 中的增量大小与 Abaqus/Explicit 中的不同（子循环）。

您可以在创建配置文件时引用这些预定义模板。

| **输入文件用法：** | 使用以下两个选项指定协同仿真控制： |
| --- | --- |
|  | ``` [*CO-SIMULATION](../key/key-link.md#usb-kws-hcosimulation), PROGRAM=ABAQUS, CONTROLS=*name* [*CO-SIMULATION CONTROLS](../key/key-link.md#usb-kws-hcosimulationcontrols), NAME=*name* ``` |

| **Abaqus/CAE 用法：** | 相互作用模块：**创建相互作用**：**Standard-Explicit 协同仿真** |

#### 时间增量方案

您可以强制 Abaqus/Standard 使用与 Abaqus/Explicit 相同的增量大小，或者可以允许 Abaqus/Standard 中的增量大小与 Abaqus/Explicit 中的不同（子循环）。您为耦合选择的时间增量方案影响求解的计算成本和准确性，但不影响求解稳定性。

子循环方案通常更具成本效益，因为不受任何强制协同仿真时间增量约束的 Abaqus/Standard 时间增量通常比 Abaqus/Explicit 时间增量大得多。但是，当模型中很大一部分节点位于协同仿真接口时，子循环方案可能不太具成本效益。这是因为 Abaqus/Standard 在 Abaqus/Explicit 的每个增量执行一组在接口处的稳定化操作（"自由求解"）。这些自由求解操作需要对一个稠密方程组进行隐式求解，其规模与接口节点数成正比。在接口节点数量很大的情况下，此接口求解的计算成本可能超过因子循环而节省的任何成本。因此，对于节点中有很大一部分位于协同仿真接口的模型，使用子循环方案时性能可能较差。

##### 强制 Abaqus/Standard 使用与 Abaqus/Explicit 相同的增量大小

您可以强制 Abaqus/Standard 匹配 Abaqus/Explicit 的增量大小，并且场将在每个共享增量处交换。

| **输入文件用法：** | 在 Abaqus/Standard 分析和 Abaqus/Explicit 分析中使用以下选项： |
| --- | --- |
|  | ``` [*CO-SIMULATION CONTROLS](../key/key-link.md#usb-kws-hcosimulationcontrols), TIME INCREMENTATION=LOCKSTEP ``` |

| **Abaqus/CAE 用法：** | 在 Abaqus/Standard 分析和 Abaqus/Explicit 分析中使用以下输入： |
| --- | --- |
|  | 相互作用模块：**创建相互作用**：**Standard-Explicit 协同仿真**：**增量控制**：**锁定时间步** |

##### 允许 Abaqus/Standard 中的增量大小与 Abaqus/Explicit 中的不同

您可以允许 Abaqus/Standard 增量大小与 Abaqus/Explicit 中的不同（子循环）。在这种情况下，场将根据需要交换。

| **输入文件用法：** | 在 Abaqus/Standard 分析和 Abaqus/Explicit 分析中使用以下选项： |
| --- | --- |
|  | ``` [*CO-SIMULATION CONTROLS](../key/key-link.md#usb-kws-hcosimulationcontrols), TIME INCREMENTATION=SUBCYCLE ``` |

| **Abaqus/CAE 用法：** | 在 Abaqus/Standard 分析和 Abaqus/Explicit 分析中使用以下输入： |
| --- | --- |
|  | 相互作用模块：**创建相互作用**：**Standard-Explicit 协同仿真**：**增量控制**：**允许子循环** |

#### 控制接口矩阵分解频率

对于子循环时间增量方案，默认情况下，在 Abaqus/Explicit 的每个增量在 Abaqus/Standard 中执行接口求解。由于两个原因，此求解可能非常昂贵。首先，接口矩阵用于接口求解且其大小随接口节点数缩放。其次，接口矩阵在每个 Abaqus/Explicit 增量变化，需要在 Abaqus/Standard 中为每个 Abaqus/Explicit 增量进行分解。您可以通过近似接口矩阵并在 Abaqus/Standard 增量持续时间内通常分解一次来减少此成本的影响，而不是为每个 Abaqus/Explicit 增量分解。但是，如果 Abaqus/Explicit 稳定时间增量发生显著变化，出于稳定性原因，接口矩阵将被重新分解。

##### 允许 Abaqus/Standard 在每个 Abaqus/Explicit 增量分解接口矩阵

在每个 Abaqus/Explicit 增量分解接口矩阵是默认方法。

| **输入文件用法：** | 在 Abaqus/Standard 分析中使用以下选项： |
| --- | --- |
|  | ``` [*CO-SIMULATION CONTROLS](../key/key-link.md#usb-kws-hcosimulationcontrols), FACTORIZATION FREQUENCY=EXPLICIT INCREMENT ``` |

| **Abaqus/CAE 用法：** | 在每个 Abaqus/Explicit 增量分解接口矩阵是 Abaqus/CAE 中的默认方法。 |

##### 强制 Abaqus/Standard 在每个 Abaqus/Standard 增量分解接口矩阵一次

当接口节点数量很大时，使用此方法可以显著降低接口分解的成本。仅在每个 Abaqus/Standard 增量执行一次接口矩阵分解；在每个 Abaqus/Explicit 增量使用此分解的接口矩阵执行接口求解。由于此方法近似接口矩阵，它可能会略微增加协同仿真接口处位移解的漂移。此方法的性能提升取决于接口节点数、子循环比（Abaqus/Standard 与 Abaqus/Explicit 增量之比）以及模型的大小。对于接口节点数大于 100 且子循环比大于 50 的模型，此方法通常将分析时间减少 1.2 到 3.0 倍。性能提升随子循环比增加而增加，随模型增大而减少。

| **输入文件用法：** | 在 Abaqus/Standard 分析中使用以下选项： |
| --- | --- |
|  | ``` [*CO-SIMULATION CONTROLS](../key/key-link.md#usb-kws-hcosimulationcontrols), FACTORIZATION FREQUENCY=STANDARD INCREMENT ``` |

| **Abaqus/CAE 用法：** | 在每个 Abaqus/Standard 增量分解接口矩阵一次在 Abaqus/CAE 中不受支持。 |

#### 耦合步长

耦合步长是 Abaqus/Standard 和 Abaqus/Explicit 之间连续两次协同仿真数据交换之间的时间段，始终等于当前 Abaqus/Explicit 增量大小。

当使用子循环方法时，此数据交换不表示对 Abaqus/Standard 增量的约束；Abaqus/Standard 分析使用其正常时间增量逻辑推进时间。

### 限制

除 ["准备 Abaqus 分析进行协同仿真，" 第 17.2.1 节"](pt04ch17s02aus98.md) 中讨论的限制外，以下限制适用于 Abaqus/Standard 到 Abaqus/Explicit 协同仿真。

#### 一般限制

- 当您允许 Abaqus/Standard 增量大小与 Abaqus/Explicit 中的不同时（即当您指定子循环作为协同仿真时间增量控制时），协同仿真接口处的位移兼容性不会维持。在这种情况下，速度兼容性得以维持，但随着模拟随时间推进，您可能会看到 Abaqus/Standard 和 Abaqus/Explicit 之间的小量位移不匹配。如果在协同仿真接口处发生严重的非线性（如塑性变形），这种"漂移"会更加明显。您可以通过调整 Abaqus/Standard 求解参数来控制此漂移，从而减少 Abaqus/Standard 增量大小（例如，通过限制最大时间增量大小或为隐式动态分析指定更小的半增量残差容差）。
- 不允许在协同仿真区域节点上进行节点变换。
- ALE 技术不能用于连接到协同仿真区域节点的单元。
- 可以使用完全耦合温度-位移单元，但不交换温度量。
- Abaqus/Standard 静态应力分析不能与 Abaqus/Standard 到 Abaqus/Explicit 协同仿真中的锁定步时间增量方案一起使用。
- 仅允许点和表面区域；不支持耦合体积区域。
- 只能将单个区域定义为接口区域；不支持多个接口区域。

#### 不同网格相关的限制

当您的 Abaqus/Standard 和 Abaqus/Explicit 协同仿真区域网格不同时，以下限制适用：
- 当协同仿真区域网格在旋转自由度的存在或不存在方面不均匀时，求解精度可能受到影响；例如，如果连续体单元网格在协同仿真区域接口处局部用梁或壳单元加固。
- 在协同仿真接口附近的应力状态显著（接近 1% 或更多）相对于材料刚度的情况下，如果 Abaqus/Explicit 和 Abaqus/Standard 模型之间协同仿真区域相邻的网格密度差异很大，您可能会看到显著的网格畸变。例如，这种效应在超弹性材料的大变形中很常见。当使用子循环时间积分方案时，您可以通过在 Abaqus/Standard 协同仿真区域选择相似或更细的网格来最小化此效应，或者当使用锁定步时间积分方案时，通过在 Abaqus/Explicit 协同仿真区域选择相似或更细的网格。

#### Abaqus/Standard 分析限制

以下单元没有与 Abaqus/Explicit 中对应的自由度，不能连接到协同仿真区域节点：
- 具有扭转自由度的轴对称单元（CGAX 单元族）
- 具有不对称变形的轴对称实体单元（CAXA 单元族）
- 广义平面应变单元（CPEG 单元族）
- 耦合孔隙压力-位移单元
- 热传递和热电单元
- 声学单元
- 压电单元

以下特定限制也适用：- 协同仿真区域节点不能是绑定约束、MPC 约束或运动耦合约束中的从节点。

#### Abaqus/Explicit 分析限制

当以下模型特征在协同仿真区域定义或附近时，协同仿真求解的稳定性和准确性可能受到不利影响：
- 连接到协同仿真区域节点的连接器单元。
- 参与绑定约束、MPC 约束或运动耦合约束的协同仿真区域节点。

使用这些特征时，您应该比较 Abaqus/Standard 和 Abaqus/Explicit 在协同仿真接口处的解（如位移历史的兼容性）作为求解准确性的指标。
