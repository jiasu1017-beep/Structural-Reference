# 20.2.7 aStudy.output()





### 20.2.7 **_aStudy_.output()**: 指定参数化研究结果的来源。

**产品:** Abaqus/Standard  Abaqus/Explicit

此命令应在任何结果收集命令之前使用。它用于指定从何处收集参数化研究的结果。

##### **参考**

- ["参数化研究脚本," 第20.1.1节](pt04ch20s01aus108.md)

### **命令:**

**_aStudy_.output** (**file**=, **instance**=, **overlay**=, **request**=, **step**= , **frameValue**= | **inc**= | **mode**= )

### **可选数据:**

**file**

设置 **file** 等于FIL或ODB，以指定结果必须从结果（`.fil`）文件或输出数据库（`.odb`）文件读取。

如果从此命令中省略 **file**，将从结果（`.fil`）文件读取结果。

**instance**

此选项仅当从包含多个实例的输出（`.odb`）数据库收集结果时适用。

设置 **instance** 等于要收集其结果的部件实例的名称。此名称必须用匹配引号括起来。

如果为在装配上定义的集合收集结果，则不得在此命令和 **gather** 命令中都指定 **instance**。

如果从此命令中省略 **instance** 且需要实例名称来收集结果，则必须在 **gather** 命令中指定。

**overlay**

此选项仅当从输出（`.odb`）数据库收集结果时适用。

此选项用于控制内存使用和执行速度之间的权衡。设置 **overlay** 等于OFF（默认）或ON，以指定是否应覆盖用于输出数据库的内存。如果内存使用有问题，设置 **overlay** 等于ON，以指定在执行特定输出数据库的 **gather** 后必须覆盖用于输出数据库的内存。如果执行速度比内存使用更重要，设置 **overlay** 等于OFF，以指定必须为每个输出数据库分配内存。

**overlay** 选项会影响您单独操作输出数据库（`.odb`）文件的能力。当设置 **overlay** 等于OFF时，访问 **gather** 命令的结果后文件将保持打开状态，这可能影响您单独访问或操作文件的能力（例如删除文件）。设置 **overlay** 等于ON，以便在访问基于 **gather** 的结果后单独访问 `.odb` 文件。

**request**

此选项仅当要从输出（`.odb`）数据库收集结果时适用。

设置 **request** 等于FIELD或HISTORY，以指定结果是从输出数据库中的场数据还是从历史数据中收集。

如果从此命令中省略 **request**，将从场数据中收集结果。

**step**

设置 **step** 等于要从中收集结果的的分析步号。

如果从此命令中省略 **step**，则必须在 **gather** 命令中指定。

##### **可选且互斥的数据:**

**frameValue**

此选项仅当要从输出（`.odb`）数据库收集结果时适用。

设置 **frameValue** 等于分析步中要从中收集结果的增量分析步时间或频率值。**frameValue** 也可以设置为符号常量LAST，以指定从步的最后增量收集结果。如果在指定的 **frameValue** 处没有可用结果，将发出警告并从最近的增量收集结果。

如果从此命令中省略 **frameValue**，则从步的最后增量收集结果，或从 **gather** 命令中指定的增量收集。

**inc**

设置 **inc** 等于非频率分析步的增量号，从该增量收集结果。**inc** 也可以设置为符号常量LAST，以指定从指定步的最后增量收集结果。

如果从此命令中省略 **inc**，则从步的最后增量收集结果，或从 **gather** 命令中指定的增量收集。

此选项对于从输出（`.odb`）数据库收集历史结果无效。

**mode**

设置 **mode** 等于频率分析步的模态号，从该模态跨参数化研究变体收集结果。

如果省略 **mode**，则从 **gather** 命令中指定的模态收集结果，或从步的第一模态收集。
