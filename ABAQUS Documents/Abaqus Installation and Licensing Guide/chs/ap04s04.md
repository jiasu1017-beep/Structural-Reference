# D.4 使用网络 ODB 连接器





用户可以创建网络 ODB 连接器来访问远程计算机上的输出数据库（请参阅["访问远程计算机上的输出数据库，"Abaqus/CAE User's Guide 第 9.3 节](../usi/usi-link.md#usi-dbs-networkodb)）。如果满足以下条件，Abaqus/CAE 或 Abaqus/Viewer 可以在远程系统上启动服务器并分配端口号：
- 远程主机上的用户名与本地系统上的用户名相同。
- 远程 shell 命令（`rsh`）或安全 shell 命令（`ssh`）配置为不提示用户输入密码。

Abaqus 通过在服务器和客户端之间来回传递密钥来检查连接的安全性。有关密钥如何生成的信息，请参阅 Dassault Systmes Knowledge Base，网址为 [www.3ds.com/support/knowledge-base](http://www.3ds.com/support/knowledge-base)。如果在远程系统上用户的主目录中存在名为 `.abaqus_net_passwd` 的文件，则 Abaqus 使用该文件中的密码进行身份验证，而不是使用 Abaqus 生成的密钥。Abaqus 检查该用户是唯一有权读取和写入密码文件的账户。此外，用户必须在 30 天后更新文件，密码必须至少八个字符。这些文件在["网络输出数据库文件连接器，"Abaqus Analysis User's Guide 第 3.2.24 节](../usb/usb-link.md#usb-int-dremoteodbproc)中有描述。

如果用户在建立通信时遇到问题，或者用户名不同，他们可以使用 **abaqus networkDBConnector** 执行过程从命令行在远程计算机上手动启动网络 ODB 服务器。如果用户手动启动网络 ODB 服务器，Abaqus 使用 `.abaqus_net_passwd` 中的密码对客户端和服务器之间的连接进行身份验证。如果远程系统上的用户目录中不存在 `.abaqus_net_passwd`，则用户不能从命令行启动网络 ODB 服务器。

您可以通过从 `*abaqus_dir*/*release*/code/bin/SMAExternal/dmbwtr/` 目录中删除 `dmbwtr` 和 `dmbwtrd` 来禁用网络 odb 连接器，其中 *abaqus_dir* 是 Abaqus 父目录，*release* 是您正在使用的 Abaqus 版本。
