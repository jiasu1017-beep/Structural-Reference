# 3.2.17 制作用户定义的执行程序和子程序





**产品：** Abaqus/Standard  Abaqus/Explicit  

##### **参考**

- ["Abaqus执行程序：概述，" 第3.1.1节](pt01ch03s01abo02.md)

### 概述

**abaqus make**实用工具用于创建用户后处理执行程序和Abaqus用户子程序的用户定义库。用于编译和链接用户提供的程序或用户子程序源文件的命令可以使用适当 的Abaqus环境文件参数进行更改；即**compile_cpp**、**compile_fortran**、**link_exe**和**link_sl**。您可以通过提供预编译的对象作为后处理程序的输入来跳过编译步骤。

使用此过程创建的后处理执行程序必须使用Abaqus执行程序运行。这是设置操作系统环境变量以查找Abaqus实用工具库所必需的。要运行用户后处理程序，请使用以下命令：

`**abaqus** `*job-name*``

使用此过程创建的用户子程序共享库通过在Abaqus环境文件中指定**usub_lib_dir**变量来使用。这样做的优点是使用用户子程序的分析可以执行而无需编译或链接用户子程序。

### 命令行选项

**job**

此选项用于创建用户提供的后处理程序。该选项的值指定此过程创建的执行文件的名称。它也用作默认源文件名。

如果命令行上未给出选项，系统将提示您输入此值。

**library**

此选项用于创建用户子程序对象文件和共享库。该选项的值指定要编译和链接的用户子程序源文件的名称。生成的对象和共享库文件放置在命令行**directory**选项指定的目录中。如果未使用**directory**选项，文件将放置在当前工作目录中。

创建的对象文件或文件具有指示用户子程序用于Abaqus/Standard还是Abaqus/Explicit的后缀。Abaqus/Standard对象文件后缀是`—std`。Abaqus/Explicit有单精度和双精度对象文件；对象文件后缀是`—xpl`和`—xplD`。创建的Abaqus/Standard用户子程序共享库称为`standardU`，Abaqus/Explicit共享库称为`explicitU`和`explicitU-D`。如果使用了**directory**选项且它包含具有与正在创建的共享库相适当后缀的对象文件，则这些文件将被链接到共享库。

**user**

此选项仅在与**job**选项结合使用时才有效。它用于指定包含您的程序的源文件或对象文件的名称（如果与*job-name*不同）。如果未提供文件扩展名，系统将查找带有FORTRAN源文件扩展名的选项值。如果找不到此名称的文件，系统将查找带有对象文件扩展名的选项值。

**directory**

此选项仅在与**library**选项结合使用时才有效。它用于指定用户子程序对象和共享库文件（由此过程创建）的目标位置。它也用于指定要链接到正在创建的共享库或库的其他对象文件的位置。如果省略此选项，此过程创建的文件将放置在当前工作目录中。

**object_type**

此选项仅在与**job**选项结合使用时才有效。它用于指定由**job**或**user**选项给定的对象文件的类型，可以是FORTRAN、C或C++。

### 示例

要创建名为"pprocess"的可执行文件（给定同名的FORTRAN源文件），请使用以下命令：

```
abaqus make job=pprocess
```

然后可以使用以下命令运行此程序：
```
abaqus pprocess
```
