# 7.3 表单模式

表单模式使用一个或多个对话框从用户收集输入。本节介绍表单用于显示对话框的两种方法。涵盖以下主题：

- ["表单示例，" 第7.3.1节](pt04ch07s03.md#cus-mod-modes-form-example)
- ["表单构造函数，" 第7.3.2节](pt04ch07s03.md#cus-mod-modes-form-constructor)
- ["getFirstDialog，" 第7.3.3节](pt04ch07s03.md#cus-mod-modes-form-getfirstdialog)
- ["getNextDialog，" 第7.3.4节](pt04ch07s03.md#cus-mod-modes-form-getnextdialog)
- ["从GUI收集输入，" 第7.3.5节](pt04ch07s03.md#cus-mod-modes-form-connecting)

### 7.3.1 表单示例

以下示例说明如何编写表单模式。第一个示例只包含一个对话框；后续示例将扩展此表单以包含多个对话框。

```
from abaqusGui import *
from plateDB import PlateDB

class PlateForm(AFXForm):

    #~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
    def __init__(self, owner):

        AFXForm.__init__(self, owner)

        self.cmd = AFXGuiCommand(self, 'Plate', 'examples')
        self.nameKw = AFXStringKeyword(self.cmd, 'name', True)
        self.widthKw = AFXFloatKeyword(self.cmd, 'width', True)
        self.heightKw = AFXFloatKeyword(self.cmd, 'height', True)

    #~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
    def getFirstDialog(self):

        self.cmd.setKeywordValuesToDefaults()
        return PlateDB(self)
```

### 7.3.2 表单构造函数

您通过从`AFXForm`派生新类开始编写表单模式。在`AFXForm`构造函数的主体中，您必须调用基类构造函数并传入owner，这是此表单所属的模块或工具集GUI。

然后定义模式将使用的命令和关键字。关键字存储为模式的成员，以便对话框可以访问它们。如果您在`AFXGuiCommand`构造函数中设置*registerQuery*=True，模式将在激活时查询命令指定的kernel对象，并自动设置命令关键字的值。有关更多信息，请参阅["保持GUI和命令最新，" 第6.5.3节](pt04ch06s05.md#cus-app-commands-gui-uptodate)。如果没有与您的命令关联的kernel对象（例如，在创建新对象时），您可以通过在构造函数中指定默认值来设置关键字值。

如果您有一个想要用于重新建立对话框默认值的默认对象，可以使用模式的`registerDefaultsObject`方法注册一个对象，当用户按下对话框中的"默认值"按钮时将查询该对象的值。有关更多信息，请参阅["默认值对象，" 第6.5.16节](pt04ch06s05.md#cus-app-commands-gui-defaults)。

默认情况下，对话框显示为非模态的。您可以通过调用`setModal(True)`将对话框更改为模态。在大多数情况下，您只需设置一次行为；但是，您可以通过在`getFirstDialog`或`getNextDialog`方法中调用`setModal`方法根据需要更改行为。有关更多信息，请参阅["模态与非模态，" 第5.2节](pt03ch05s02.md)。

### 7.3.3 getFirstDialog

您必须为模式编写`getFirstDialog`方法。`getFirstDialog`方法应返回模式的第一个对话框。在["表单示例，" 第7.3.1节](pt04ch07s03.md#cus-mod-modes-form-example)中，指向表单的指针被传递到对话框构造函数中。对话框将使用此指针访问模式的关键字。

如果您希望每次显示对话框时出现相同的默认值，必须在返回对话框之前调用`setKeywordValuesToDefaults()`方法，如["表单示例，" 第7.3.1节](pt04ch07s03.md#cus-mod-modes-form-example)中所示。

### 7.3.4 getNextDialog

如果您的模式包含多个对话框，除了`getFirstDialog`方法外，还必须编写`getNextDialog`方法。之前的对话框被传递到`getNextDialog`方法中，这样您就可以确定用户在对话框序列中的位置并相应地采取行动。`getNextDialog`方法应返回序列中的下一个对话框，或者返回`None`以表示已完成从用户收集输入。以下示例是["getFirstDialog，" 第7.3.3节](pt04ch07s03.md#cus-mod-modes-form-getfirstdialog)中示例的修改版本，说明了如何通过三个对话框的序列而不是仅仅一个来收集用户的输入：

```
def getFirstDialog(self):

    self.dialog1 = PlateDB1(self)
    return self.dialog1

def getNextDialog(self, previousDb):

    if previousDb == self.dialog1:
        self.dialog2 = PlateDB2(self)
        return self.dialog2
    elif previousDb == self.dialog2:
        self.dialog3 = PlateDB3(self)
        return self.dialog3
    else:
        return None
```

### 7.3.5 从GUI收集输入

要通过GUI从用户收集输入，模式中定义的关键字必须连接到对话框中的窗口部件。`AFXDataDialog`类在其构造函数中接受一个mode参数。因为表单存储关键字，对话框可以访问这些关键字并将其分配为对话框中窗口部件的目标。因此，GUI可以更新关键字；或者，如果在显示对话框时kernel被更新，关键字可以更新GUI。有关更多信息，请参阅[第5章，"对话框"](pt03ch05.md)。以下示例显示表单的关键字如何连接到对话框中的窗口部件：

```
class PlateDB(AFXDataDialog):

    def __init__(self, mode):

        AFXDataDialog.__init__(self, mode, 'Create Plate',
            self.OK|self.CANCEL, DIALOG_ACTIONS_SEPARATOR)

        va = AFXVerticalAligner(self)
        AFXTextField(va, 15, 'Name:', mode.nameKw, 0)
        AFXTextField(va, 15, 'Width:', mode.widthKw, 0)
        AFXTextField(va, 15, 'Height:', mode.heightKw, 0)
```
