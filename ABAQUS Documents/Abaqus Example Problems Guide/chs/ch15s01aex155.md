# 15.1.2 合并多个结果文件数据并转换文件格式：FJOIN





**产品：** Abaqus/Standard  Abaqus/Explicit

本示例说明了如何使用FORTRAN程序从不同的Abaqus结果文件中提取特定数据，并将数据合并到单个结果文件中。该程序还可用于转换结果文件的格式。

### 后处理

有时需要将多个结果文件合并为单个文件，或者通过从不同结果文件中检索选定的数据来创建新的结果文件。**abaqus append**工具通过从第二个结果文件中剥离头信息，并将步信息追加到第一个结果文件的末尾来连接两个结果文件。有关此工具的更多信息，请参见["连接结果（`.fil`）文件，" Abaqus Analysis User's Guide的第3.2.14节](../usb/usb-link.md#usb-int-dappendfil)。

此示例后处理程序演示了如何使用FORTRAN程序从同一模型的不同分析创建的结果文件中提取特定信息。本示例中，三个分析中的应力和应变记录将被合并以创建新的结果文件。

### 编程细节

在运行或修改此程序之前，应审查["Abaqus结果文件用户后处理概述，"第15.1.1节](ch15s01abo02.md)中关于编程概念和Abaqus FORTRAN接口的一般讨论。还建议审查Abaqus Analysis User's Guide第5章"文件输出格式"中的结果文件格式。

程序`FJOIN`（在Abaqus发布媒体上命名为`fjoin.f`）提示输入`NRU`、`LRUNIT(1,NRU)`、`LRUNIT(2,NRU)`和`FNAME`的值。然后调用子程序`INITPF`和`DBNRU`来完成必要的初始化和文件连接。数据处理从双重`DO`循环开始，通过调用`DBFILE`逐一读取所有要处理的记录。可以跳过记录，也可以将记录写入新结果文件（可进行或不进行修改）。每条记录由其记录键标识，记录键存储在记录的第二条目中（请参见["结果文件输出格式，" Abaqus Analysis User's Guide的第5.1.2节](../usb/usb-link.md#usb-out-fformat)。

每个文件包含多个头记录，`1900`系列。这些记录包含有关模型的一般信息。使用同一模型的不同分析将这些记录中的基本信息放置在这些记录中。因此，当合并来自同一模型不同分析的结果文件时，应保留第一个被处理文件中的所有`1900`系列记录。后续文件中的类似记录应跳过，以避免重复和混淆。但是，应保留`1910`、`1911`、`1922`和`1980`记录。它们对子结构内的结果输出请求和自然频率提取结果的处理很有用。

每个分析增量的数据从增量起始记录开始，由记录键`2000`标识。记录`2000`之后是对应于Abaqus输入文件中指定的文件输出选项所请求的数据的记录。记录`1`（元素头记录）在请求元素变量时会自动写入结果文件。它在后处理中很有意义，因为它包含有关元素数据的重要信息，包括数据在元素内的位置（即，数据是在元素积分点、质心、节点等处写入的）。对于本示例，记录`11`和`21`（分别是应力和应变记录）被写入结果文件，因为请求了应力和应变。增量结束记录由记录键`2001`标识。当遇到文件结束条件且先前处理的记录是`2001`记录时，将对当前FORTRAN单元号执行FORTRAN `CLOSE`，以便开始处理下一个文件。

### 程序编译和链接

**abaqus make**工具旨在编译和链接此类后处理程序。它还会在编译期间提供`aba_param.inc`文件。编译和链接`FJOIN`程序的命令如下：

```
abaqus make job=fjoin
```

如果在编译或链接过程中发现FORTRAN错误，必须重复此命令。必要时可以更改**abaqus make**工具使用的命令。[Abaqus Installation and Licensing Guide](../sgb/sgb-link.md#sgb)列出了每种计算机类型的典型编译和链接命令。

### 程序执行

在程序执行之前，必须运行分析作业以生成程序要读取的结果文件。本示例中运行了三个作业。这些分析的输入文件是`fjoin002.inp`、`fjoin003.inp`和`fjoin004.inp`。这些分析的结果文件以二进制格式输出，称为`fjoin002.fil`、`fjoin003.fil`和`fjoin004.fil`。

`FJOIN`程序将通过FORTRAN单元`2`、`3`和`4`读取这些文件。新文件的名称将为`fjoinxxx`。在运行程序之前，必须将结果文件重命名为`fjoinxxx.002`、`fjoinxxx.003`和`fjoinxxx.004`。请注意，根文件名是相同的（使用`FNAME`定义），扩展名设置为用于打开文件的FORTRAN单元号。

当使用命令`abaqus fjoin`执行程序时，第一个提示将是

```
Enter the number of files to be joined:
```

输入`3`以设置`NRU=3`。第二个提示将是
```
Enter the unit number of input file # 1:
```

输入`2`以定义`LRUNIT(1,1)=2`。在第三个提示处，
```
Enter the format of input file # 1 (1-ASCII, 2-binary):
```

输入`2`。这设置`LRUNIT(2,1)=2`，表示正在读取的文件是二进制的。第二个和第三个提示会对每个要处理的附加文件重复。然后程序会询问新结果文件应该以ASCII还是二进制格式写入，
```
Enter the format of the output file (1-ASCII, 2-binary):
```

输入`2`以设置`LOUTF=2`，指定为新结果文件选择二进制格式。输出文件的格式可能与输入文件的格式不同，因此该程序也可用于转换结果文件的格式。最后，当程序发出提示
```
Enter the name of the input files (w/o extension):
```

输入`fjoinxxx`以定义`FNAME`（输入文件必须已给定根文件名`fjoinxxx`；输出文件将创建为`fjoinxxx.fin`）。

一旦处理完第*n*个文件，消息`END OF FILE #` *n*就会被写入终端。所有文件处理完成后，程序停止，并创建新的结果文件。该程序创建的新结果文件包含每个元素所有积分点处以及所有节点处的应力和应变记录。

### 分析描述

该结构是一个10×10的单位厚度方形板。板位于*X–Y*平面内，其底边与*x*轴重合，左边与*y*轴重合。有限元模型采用2×2的CPS8R单元网格。材料为线弹性，弹性模量=30×10⁶，泊松比=0.3。执行三个位移控制的加载步分析。

在第一个分析（[fjoin002.inp](../eif/fjoin002.inp)）中，通过沿顶边施加0.25的垂直位移、沿右边界施加0.25的水平位移，以及沿左边和底边的对称边界条件，对板施加双轴张力。

在第二个分析（[fjoin003.inp](../eif/fjoin003.inp)）中，通过沿顶边施加0.25的水平位移，同时固定底边，并允许沿左边和右边的水平位移随*y*线性变化，来迫使结构以简单剪切方式变形。垂直位移处处为零。

在第三个分析（[fjoin004.inp](../eif/fjoin004.inp)）中，通过沿顶边节点在*y*方向施加0.25的位移，以及沿*x*轴和*y*轴的节点施加对称边界条件，对板施加单轴张力。
