## 2.8. 类BLAS扩展

本节描述执行矩阵-矩阵运算的BLAS扩展函数。

### 2.8.1. cublas<t>geam()

此函数支持64位整数接口。

此函数执行矩阵-矩阵加法/转置运算。

$C = \alpha\text{op}(A) + \beta\text{op}(B)$

其中，$\alpha$ 和 $\beta$ 为标量，$A$、$B$ 和 $C$ 为以列优先格式存储的矩阵，维度分别为 $\text{op}(A)$ $m \times n$、$\text{op}(B)$ $m \times n$ 和 $C$ $m \times n$。对于矩阵 $A$：

$\text{op}(A) = \left\{ \begin{matrix}
A & {\text{if }\textsf{transa == CUBLAS\_OP\_N}} \\
A^{T} & {\text{if }\textsf{transa == CUBLAS\_OP\_T}} \\
A^{H} & {\text{if }\textsf{transa == CUBLAS\_OP\_C}} \\
\end{matrix} \right.$

$\text{op}(B)$ 对矩阵 $B$ 的定义类似。

如果 C 不与 A 或 B 重叠，则该操作是原位（out-of-place）操作。

原位模式支持以下两种操作：

$C = \alpha\text{*}C + \beta\text{op}(B)$

$C = \alpha\text{op}(A) + \beta\text{*}C$

对于原位模式，如果 `C == A`，则 `ldc == lda` 且 `transa == CUBLAS_OP_N`。如果 `C === B`，则 `ldc == ldb` 且 `transb == CUBLAS_OP_N`。如果用户不满足上述要求，则返回 `CUBLAS_STATUS_INVALID_VALUE`。

该操作包括以下特殊情况：

用户可以通过设置 `*alpha = beta = 0` 将矩阵 C 重置为零。

用户可以通过设置 `*alpha = 1 and *beta = 0` 对矩阵 A 进行转置。

| 参数 | 内存 | 输入/输出 | 含义 |
| --- | --- | --- | --- |
| handle |  | 输入 | cuBLAS 库上下文的句柄。 |
| transa |  | 输入 | 运算 op(A)，为非转置或（共轭）转置。 |
| transb |  | 输入 | 运算 op(B)，为非转置或（共轭）转置。 |
| m |  | 输入 | 矩阵 op(A) 和 C 的行数。 |
| n |  | 输入 | 矩阵 op(B) 和 C 的列数。 |
| alpha | 主机或设备 | 输入 | 用于乘法的 <type> 标量。如果 *alpha == 0，则 A 不必是有效输入。 |
| A | 设备 | 输入 | <type> 数组，维度为 lda x n，其中如果 transa == CUBLAS_OP_N，则 lda >= max(1, m)，否则为 lda x m 且 lda >= max(1, n)。 |
| lda |  | 输入 | 用于存储矩阵 A 的二维数组的leading dimension。 |
| B | 设备 | 输入 | <type> 数组，维度为 ldb x n，其中如果 transb == CUBLAS_OP_N，则 ldb >= max(1, m)，否则为 ldb x m 且 ldb >= max(1,n)。 |
| ldb |  | 输入 | 用于存储矩阵 B 的二维数组的leading dimension。 |
| beta | 主机或设备 | 输入 | 用于乘法的 <type> 标量。如果 *beta == 0，则 B 不必是有效输入。 |
| C | 设备 | 输出 | <type> 数组，维度为 ldc x n，其中 ldc >= max(1, m)。 |
| ldc |  | 输入 | 用于存储矩阵 C 的二维数组的leading dimension。 |

此函数可能返回的错误值及其含义如下表所示。

| 错误值 | 含义 |
| --- | --- |
| CUBLAS_STATUS_SUCCESS | 操作成功完成 |
| CUBLAS_STATUS_NOT_INITIALIZED | 库未初始化 |
| CUBLAS_STATUS_INVALID_VALUE | 如果 m < 0 或 n < 0，或<br>如果 transa 不是 CUBLAS_OP_N、CUBLAS_OP_T 和 CUBLAS_OP_C 之一，或<br>如果 transb 不是 CUBLAS_OP_N、CUBLAS_OP_T 和 CUBLAS_OP_C 之一，或<br>如果 transa == CUBLAS_OP_N 时 lda < max(1, m)，否则 lda < max(1, n)，或<br>如果 transb == CUBLAS_OP_N 时 ldb < max(1, m)，否则 ldb < max(1, n)，或<br>如果 ldc < max(1, m)，或<br>如果 A == C 且 (transa != CUBLAS_OP_N) || (lda != ldc)，或<br>如果 B == C 且 (transb != CUBLAS_OP_N) || (ldb != ldc)，或<br>如果 alpha 或 beta 为 NULL |
| CUBLAS_STATUS_EXECUTION_FAILED | 函数在 GPU 上启动失败 |

### 2.8.2. cublas<t>dgmm()

此函数支持64位整数接口。

此函数执行矩阵-矩阵乘法：

$C = \left\{ \begin{matrix}
{A \times diag(X)} & {\text{if }\textsf{mode == CUBLAS\_SIDE\_RIGHT}} \\
{diag(X) \times A} & {\text{if }\textsf{mode == CUBLAS\_SIDE\_LEFT}} \\
\end{matrix} \right.$

其中，$A$ 和 $C$ 为以列优先格式存储的矩阵，维度为 $m \times n$。如果 `mode == CUBLAS_SIDE_RIGHT`，则 $X$ 为大小为 $n$ 的向量；如果 `mode == CUBLAS_SIDE_LEFT`，则 $X$ 为大小为 $m$ 的向量。$X$ 从一维数组 x 中以步长 `incx` 收集。`incx` 的绝对值是步长，`incx` 的符号是步长的方向。如果 `incx` 为正，则从第一个元素开始正向遍历 x。否则，从最后一个元素开始反向遍历 x。X 的公式为：

$X\lbrack j\rbrack = \left\{ \begin{matrix}
{x\lbrack j \times incx\rbrack} & {\text{if }incx \geq 0} \\
{x\lbrack(\chi - 1) \times |incx| - j \times |incx|\rbrack} & {\text{if }incx < 0} \\
\end{matrix} \right.$

其中，如果 `mode == CUBLAS_SIDE_LEFT`，则 $\chi = m$；如果 `mode == CUBLAS_SIDE_RIGHT`，则 $\chi = n$。

示例1：如果用户想执行 $diag(diag(B)) \times A$，则 $incx = ldb + 1$，其中 $ldb$ 是矩阵 `B` 的leading dimension，无论是行优先还是列优先格式。

示例2：如果用户想执行 $\alpha \times A$，有两种选择：要么使用 `*beta == 0` 和 `transa == CUBLAS_OP_N` 的 cublas<t>geam()，要么使用 `incx == 0` 和 `x[0] == alpha` 的 cublas<t>dgmm()。

该操作是原位的。原位模式仅在 `lda == ldc` 时有效。

| 参数 | 内存 | 输入/输出 | 含义 |
| --- | --- | --- | --- |
| handle |  | 输入 | cuBLAS 库上下文的句柄。 |
| mode |  | 输入 | 如果 mode == CUBLAS_SIDE_LEFT 则左乘，如果 mode == CUBLAS_SIDE_RIGHT 则右乘 |
| m |  | 输入 | 矩阵 A 和 C 的行数。 |
| n |  | 输入 | 矩阵 A 和 C 的列数。 |
| A | 设备 | 输入 | <type> 数组，维度为 lda x n，其中 lda >= max(1, m) |
| lda |  | 输入 | 用于存储矩阵 A 的二维数组的leading dimension。 |
| x | 设备 | 输入 | 一维 <type> 数组，如果 mode == CUBLAS_SIDE_LEFT，则大小为 abs(incx) x m；如果 mode == CUBLAS_SIDE_RIGHT，则大小为 abs(incx) x n |
| incx |  | 输入 | 一维数组 x 的步长。 |
| C | 设备 | 输入/输出 | <type> 数组，维度为 ldc x n，其中 ldc >= max(1, m)。 |
| ldc |  | 输入 | 用于存储矩阵 C 的二维数组的leading dimension。 |

此函数可能返回的错误值及其含义如下表所示。

| 错误值 | 含义 |
| --- | --- |
| CUBLAS_STATUS_SUCCESS | 操作成功完成 |
| CUBLAS_STATUS_NOT_INITIALIZED | 库未初始化 |
| CUBLAS_STATUS_INVALID_VALUE | 如果 m < 0 或 n < 0，或<br>如果 mode 不是 CUBLAS_SIDE_LEFT 和 CUBLAS_SIDE_RIGHT 之一，或<br>如果 lda < max(1, m)，或<br>如果 ldc < max(1, m) |
| CUBLAS_STATUS_EXECUTION_FAILED | 函数在 GPU 上启动失败 |

### 2.8.3. cublas<t>getrfBatched()

`Aarray` 是指向以列优先格式存储的矩阵的指针数组，维度为 `n x n`，leading dimension 为 `lda`。

此函数对每个 `Aarray[i]`（i = 0, …, `batchSize-1`）执行 LU 分解，分解方程如下：

$\text{P}\text{*}{Aarray}\lbrack i\rbrack = L\text{*}U$

其中，`P` 是表示带行交换的部分主元选择的置换矩阵。`L` 是单位对角线的下三角矩阵，`U` 是上三角矩阵。

形式上，`P` 可以写为置换矩阵 `Pj` 的乘积，对于 `j = 1,2,...,n`，即 `P = P1 * P2 * P3 * .... * Pn`。`Pj` 是一个置换矩阵，在执行 `Pj*x` 时交换向量 x 的两行。`Pj` 可以通过以下 Matlab 代码由 `PivotArray[i]` 的第 j 个元素构造：

```c++
// In Matlab PivotArray[i] is an array of base-1.
// In C, PivotArray[i] is base-0.
Pj = eye(n);
swap Pj(j,:) and Pj(PivotArray[i][j]  ,:)
```

`L` 和 `U` 被写回原始矩阵 `A`，`L` 的对角线元素被丢弃。`L` 和 `U` 可以通过以下 Matlab 代码构造：

```c++
// A is a matrix of nxn after getrf.
L = eye(n);
for j = 1:n
    L(j+1:n,j) = A(j+1:n,j)
end
U = zeros(n);
for i = 1:n
    U(i,i:n) = A(i,i:n)
end
```

如果矩阵 `A(=Aarray[i])` 是奇异的，getrf 仍然会工作，`info(=infoArray[i])` 的值报告 LU 分解无法继续的第一个行索引。如果 info 为 `k`，则 `U(k,k)` 为零。方程 `P*A == L*U` 仍然成立，但 `L` 和 `U` 的重构需要不同的 Matlab 代码，如下所示：

```c++
// A is a matrix of nxn after getrf.
// info is k, which means U(k,k) is zero.
L = eye(n);
for j = 1:k-1
    L(j+1:n,j) = A(j+1:n,j)
end
U = zeros(n);
for i = 1:k-1
    U(i,i:n) = A(i,i:n)
end
for i = k:n
    U(i,k:n) = A(i,k:n)
end
```

此函数旨在用于矩阵尺寸较小、启动开销是重要因素的情况。

如果 `PivotArray` 为 NULL，则 cublas<t>getrfBatched 支持非主元 LU 分解。

cublas<t>getrfBatched 支持任意维度。

cublas<t>getrfBatched 仅支持计算能力2.0或更高版本。

| 参数 | 内存 | 输入/输出 | 含义 |
| --- | --- | --- | --- |
| handle |  | 输入 | cuBLAS 库上下文的句柄。 |
| n |  | 输入 | Aarray[i] 的行数和列数。 |
| Aarray | 设备 | 输入/输出 | 指向 <type> 数组的指针数组，每个数组维度为 n x n，lda >= max(1, n)。矩阵 Aarray[i] 不应重叠；否则，将产生未定义行为。 |
| lda |  | 输入 | 用于存储每个矩阵 Aarray[i] 的二维数组的leading dimension。 |
| PivotArray | 设备 | 输出 | 大小为 n x batchSize 的数组，包含每个 Aarray[i] 分解的主元序列，以线性方式存储。如果 PivotArray 为 NULL，则禁用主元选择。 |
| infoArray | 设备 | 输出 | 大小为 batchSize 的数组，info(=infoArray[i]) 包含 Aarray[i] 分解的信息。<br>如果 info == 0，则执行成功。<br>如果 info = -j，则第 j 个参数具有非法值。<br>如果 info = k，则 U(k, k) == 0。分解已完成，但 U 正好是奇异的。 |
| batchSize |  | 输入 | A 中包含的指针数量 |

此函数可能返回的错误值及其含义如下表所示。

| 错误值 | 含义 |
| --- | --- |
| CUBLAS_STATUS_SUCCESS | 操作成功完成 |
| CUBLAS_STATUS_NOT_INITIALIZED | 库未初始化 |
| CUBLAS_STATUS_INVALID_VALUE | 参数 n < 0 或 batchSize < 0 或 lda <0 |
| CUBLAS_STATUS_EXECUTION_FAILED | 函数在 GPU 上启动失败 |

### 2.8.4. cublas<t>getrsBatched()

此函数求解形式如下的线性方程组数组：

$\text{op}(A\lbrack i \rbrack) X\lbrack i\rbrack = B\lbrack i\rbrack$

其中，$A\lbrack i\rbrack$ 是已经过带主元 LU 分解的矩阵，$X\lbrack i\rbrack$ 和 $B\lbrack i\rbrack$ 是 $n \times {nrhs}$ 矩阵。此外，对于矩阵 $A$：

$\text{op}(A\lbrack i\rbrack) = \left\{ \begin{matrix}
{A\lbrack i\rbrack} & {\text{if }\textsf{trans == CUBLAS\_OP\_N}} \\
{A^{T}\lbrack i\rbrack} & {\text{if }\textsf{trans == CUBLAS\_OP\_T}} \\
{A^{H}\lbrack i\rbrack} & {\text{if }\textsf{trans == CUBLAS\_OP\_C}} \\
\end{matrix} \right.$

此函数旨在用于矩阵尺寸较小、启动开销是重要因素的情况。

如果 `devIpiv` 为 NULL，则 cublas<t>getrsBatched() 支持非主元 LU 分解。

cublas<t>getrsBatched() 支持任意维度。

cublas<t>getrsBatched() 仅支持计算能力2.0或更高版本。

| 参数 | 内存 | 输入/输出 | 含义 |
| --- | --- | --- | --- |
| handle |  | 输入 | cuBLAS 库上下文的句柄。 |
| trans |  | 输入 | 运算 op(A)，为非转置或（共轭）转置。 |
| n |  | 输入 | Aarray[i] 的行数和列数。 |
| nrhs |  | 输入 | Barray[i] 的列数。 |
| Aarray | 设备 | 输入 | 指向 <type> 数组的指针数组，每个数组维度为 n x n，lda >= max(1, n)。 |
| lda |  | 输入 | 用于存储每个矩阵 Aarray[i] 的二维数组的leading dimension。 |
| devIpiv | 设备 | 输入 | 大小为 n x batchSize 的数组，包含每个 Aarray[i] 分解的主元序列，以线性方式存储。如果 devIpiv 为 NULL，则忽略所有 Aarray[i] 的主元选择。 |
| Barray | 设备 | 输入/输出 | 指向 <type> 数组的指针数组，每个数组维度为 n x nrhs，ldb >= max(1, n)。矩阵 Barray[i] 不应重叠；否则，将产生未定义行为。 |
| ldb |  | 输入 | 用于存储每个解矩阵 Barray[i] 的二维数组的leading dimension。 |
| info | 主机 | 输出 | 如果 info == 0，则执行成功。<br>如果 info = -j，则第 j 个参数具有非法值。 |
| batchSize |  | 输入 | A 中包含的指针数量 |

此函数可能返回的错误值及其含义如下表所示。

| 错误值 | 含义 |
| --- | --- |
| CUBLAS_STATUS_SUCCESS | 操作成功完成 |
| CUBLAS_STATUS_NOT_INITIALIZED | 库未初始化 |
| CUBLAS_STATUS_INVALID_VALUE | 如果 n < 0 或 nrhs < 0，或<br>如果 trans 不是 CUBLAS_OP_N、CUBLAS_OP_T 和 CUBLAS_OP_C 之一，或<br>如果 lda < max(1, n)，或<br>如果 ldb < max(1, n) |
| CUBLAS_STATUS_EXECUTION_FAILED | 函数在 GPU 上启动失败 |

### 2.8.5. cublas<t>getriBatched()

`Aarray` 和 `Carray` 是指向以列优先格式存储的矩阵的指针数组，维度为 `n x n`，leading dimension 分别为 `lda` 和 `ldc`。

此函数对矩阵 `A[i]`（i = 0, …, `batchSize-1`）执行求逆运算。

在调用 cublas<t>getriBatched 之前，矩阵 `A[i]` 必须首先使用 cublas<t>getrfBatched 例程进行分解。在调用 cublas<t>getrfBatched 之后，由 `Aarray[i]` 指向的矩阵将包含矩阵 `A[i]` 的 LU 因子，由 `(PivotArray+i)` 指向的向量将包含主元序列。

在 LU 分解之后，cublas<t>getriBatched 使用前向和后向三角求解器来完成矩阵 `A[i]`（i = 0, …, `batchSize-1`）的求逆。求逆是原位的，因此 Carray[i] 的内存空间不能与 Array[i] 的内存空间重叠。

通常，cublas<t>getrfBatched 的所有参数都会传递给 cublas<t>getriBatched。例如：

```c++
// step 1: perform in-place LU decomposition, P*A = L*U.
//      Aarray[i] is n*n matrix A[i]
    cublasDgetrfBatched(handle, n, Aarray, lda, PivotArray, infoArray, batchSize);
//      check infoArray[i] to see if factorization of A[i] is successful or not.
//      Array[i] contains LU factorization of A[i]

// step 2: perform out-of-place inversion, Carray[i] = inv(A[i])
    cublasDgetriBatched(handle, n, Aarray, lda, PivotArray, Carray, ldc, infoArray, batchSize);
//      check infoArray[i] to see if inversion of A[i] is successful or not.
```

用户可以从 cublas<t>getrfBatched 或 cublas<t>getriBatched 检查奇异性。

此函数旨在用于矩阵尺寸较小、启动开销是重要因素的情况。

如果 cublas<t>getrfBatched 是通过非主元方式执行的，则 cublas<t>getriBatched 的 `PivotArray` 应为 NULL。

cublas<t>getriBatched 支持任意维度。

cublas<t>getriBatched 仅支持计算能力2.0或更高版本。

| handle |  | 输入 | cuBLAS 库上下文的句柄。 |
| n |  | 输入 | Aarray[i] 的行数和列数。 |
| Aarray | 设备 | 输入 | 指向 <type> 数组的指针数组，每个数组维度为 n x n，lda >= max(1, n)。 |
| lda |  | 输入 | 用于存储每个矩阵 Aarray[i] 的二维数组的leading dimension。 |
| PivotArray | 设备 | 输出 | 大小为 n*batchSize 的数组，包含每个 Aarray[i] 分解的主元序列，以线性方式存储。如果 PivotArray 为 NULL，则禁用主元选择。 |
| Carray | 设备 | 输出 | 指向 <type> 数组的指针数组，每个数组维度为 n x n，ldc >= max(1, n)。矩阵 Carray[i] 不应重叠；否则，将产生未定义行为。 |

### 2.8.6. cublas<t>matinvBatched()

`A` 和 `Ainv` 是指向以列优先格式存储的矩阵的指针数组，维度为 `n x n`，leading dimension 分别为 `lda` 和 `lda_inv`。

此函数对矩阵 `A[i]`（i = 0, …, `batchSize-1`）执行求逆运算。

此函数是 cublas<t>getrfBatched() 加 cublas<t>getriBatched() 的快捷方式。但是，如果 `n` 大于32，则此函数不起作用。如果不大于32，用户必须通过 cublas<t>getrfBatched() 和 cublas<t>getriBatched()。

如果矩阵 `A[i]` 是奇异的，则 `info[i]` 报告奇异性，与 cublas<t>getrfBatched() 相同。

| 参数 | 内存 | 输入/输出 | 含义 |
| --- | --- | --- | --- |
| handle |  | 输入 | cuBLAS 库上下文的句柄。 |
| n |  | 输入 | A[i] 的行数和列数。 |
| A | 设备 | 输入 | 指向 <type> 数组的指针数组，每个数组维度为 n x n，lda >= max(1, n)。 |

### 2.8.7. cublas<t>geqrfBatched()

`Aarray` 是指向以列优先格式存储的矩阵的指针数组，维度为 `m x n`，leading dimension 为 `lda`。`TauArray` 是指向向量的指针数组，每个向量的维度至少为 `max (1, min(m, n))`。

此函数使用 Householder 反射对每个 `Aarray[i]`（`i = 0, ...,batchSize-1`）执行 QR 分解。每个矩阵 `Q[i]` 表示为基本反射子的乘积，存储在每个 `Aarray[i]` 的下半部分，如下所示：

```text
Q[j] = H[j][1] H[j][2] . . . H[j](k), where k = min(m,n).
```

每个 H[j][i] 的形式为：

```text
H[j][i] = I - tau[j] * v * v'
```

其中，`tau[j]` 是实标量，`v` 是实向量，满足 `v(1:i-1) = 0` 和 `v(i) = 1`；`v(i+1:m)` 在退出时存储在 `Aarray[j][i+1:m,i]` 中，`tau` 存储在 `TauArray[j][i]` 中。

此函数旨在用于矩阵尺寸较小、启动开销是重要因素的情况。

cublas<t>geqrfBatched 支持任意维度。

cublas<t>geqrfBatched 仅支持计算能力2.0或更高版本。

| 参数 | 内存 | 输入/输出 | 含义 |
| --- | --- | --- | --- |
| handle |  | 输入 | cuBLAS 库上下文的句柄。 |
| m |  | 输入 | Aarray[i] 的行数。 |
| n |  | 输入 | Aarray[i] 的列数。 |
| Aarray | 设备 | 输入 | 指向 <type> 数组的指针数组，每个数组维度为 m x n，lda >= max(1, m)。 |
| lda |  | 输入 | 用于存储每个矩阵 Aarray[i] 的二维数组的leading dimension。 |
| TauArray | 设备 | 输出 | 指向 <type> 向量的指针数组，每个向量维度为 max(1 ,min(m, n))。 |
| info | 主机 | 输出 | 如果 info == 0，则传递给函数的参数有效<br>如果 info < 0，则位置 -info 的参数无效 |
| batchSize |  | 输入 | Aarray 中包含的指针数量 |

### 2.8.8. cublas<t>gelsBatched()

`Aarray` 是指向以列优先格式存储的矩阵的指针数组。`Carray` 是指向以列优先格式存储的矩阵的指针数组。

此函数求取超定方程组批次的最小二乘解：它求解如下所述的最小二乘问题：

```c++
minimize  || Carray[i] - Aarray[i]*Xarray[i] || , with i = 0, ...,batchSize-1
```

退出时，每个 `Aarray[i]` 被其 QR 分解覆盖，每个 `Carray[i]` 被最小二乘解覆盖。

cublas<t>gelsBatched 仅支持非转置运算，并且仅求解超定方程组（m >= n）。
