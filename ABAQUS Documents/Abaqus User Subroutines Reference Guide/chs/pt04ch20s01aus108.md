**产品：** Abaqus/Standard  Abaqus/Explicit  

##### **参考文献**

- ["参数化输入，" 第1.4.1节](pt01ch01s04aus04.md)
- ["参数化形状变化，" 第2.1.2节](pt01ch02s01aus06.md)

### 概述

参数化研究允许您生成、执行和收集多个分析的結果，这些分析仅在用于替代输入量的某些参数值上有所不同。

参数化研究可以通过以下方式执行：
- 从参数化输入"模板"文件生成不同的参数化变体。
- 准备一个脚本（扩展名为`.psf`的文件），其中包含Python（Lutz, 1996）指令，用于生成、执行和收集参数化输入文件的参数化变体的输出。

用于参数化研究脚本的Python命令将在本节讨论。

### 引言

参数化研究需要执行多次分析，以提供有关结构或组件在设计空间中不同设计点的行为信息。这些分析的输入仅在分配给参数化关键词输入文件（以`.inp`扩展名标识）参数的值上有所不同。

Abaqus中的参数化研究需要一个用户开发的Python脚本文件（以`.psf`扩展名标识），其中包含定义参数化研究的Python命令。例如，假设您希望执行一个参数化研究，其中改变壳的厚度。您需要创建一个参数化输入文件（在此示例中，名为`shell.inp`的文件），其中包含参数定义

```
[*PARAMETER](../key/key-link.md#usb-kws-mparameter)
thick1 = 5.
```

和参数用法：
```
[*SHELL SECTION](../key/key-link.md#usb-kws-mshellsection),ELSET=*name*, MATERIAL=*name*
<thick1>
```

您可以通过开发一个`.psf`文件来创建参数化研究，该文件包含指定要分析的不同设计的Python指令脚本，如下所示：
```
**thick = ParStudy**(**par**='thick1', **name**='shell')
**thick.define**(CONTINUOUS, **par**='thick1', **domain**=(10., 20.))
**thick.sample**(NUMBER, **par**='thick1', **number**=5)
**thick.combine**(MESH)
```

这些脚本命令创建五个设计，对应的截面厚度分别为10.、12.5、15.、17.5和20.0。这些厚度中的每一个都将依次替换`shell.inp`中参数定义中指定的5.值。然后您可以在`.psf`文件中提供额外的Python脚本命令，指示Abaqus执行以下操作：
- 使用`shell.inp`文件作为模板，生成多个`shell_*id*.inp`文件和相应的Abaqus作业。（附加到输入文件名的标识符*id*在参数化研究中是每个设计唯一的。）此操作的Python命令示例为
``` **thick.generate**(**template**='shell') ```
在此示例中，`shell_*id*.inp`文件仅在要使用的壳厚度值上有所不同。
- 执行表示参数化研究不同变体的所有Abaqus作业。此操作的Python命令为
``` **thick.execute**(ALL) ```

通常，您希望从参数化研究生成的大量数据中查看某些关键结果。Abaqus提供以下功能：
- 指定参数化研究结果来源的命令。例如：
``` **thick.output**(**file**=ODB, **step**=1, **inc**=LAST) ```
上述命令将输出位置设置为输出数据库（`.odb`）文件第一步的最后一帧。默认行为是从结果（`.fil`）文件中给定步骤的最后一帧收集结果。
- 从参数化研究生成的多个分析中收集所需结果并在文件或表中报告的命令。例如，用于收集和报告每个设计中关键节点处位移值的Python脚本命令序列为：
``` **thick.gather**(**results**='n33_u', **variable**='U', **node**=33, **step**=1)
**thick.report**(PRINT, **par**='thick1', **results**=('n33_u.2')) ```
上述命令收集每个设计的结果记录`'n33_u'`（节点33在分析第一步结束时的位移向量），然后打印所有设计的U2分量（结果记录的第二分量）位移表。
- 使用Abaqus/CAE的可视化模块可视化跨多个分析收集的X-Y曲线数据的能力。一个典型示例是获取关键节点处位移值与壳厚度值的X-Y曲线图。这通过在ASCII文件中收集适当的参数化研究结果来实现，该文件可以被可视化模块读取以显示图形。

### 参数化研究的组织

Abaqus中的参数化研究与定义设计空间的特定参数集相关联。在参数化研究中只能更改参数的值。如果您希望考虑一组不同的参数，则必须创建新的参数化研究。选择参数化研究中要考虑的参数后，必须指定每个参数的定义方式。参数在本质上被区分为连续或离散，并且可以具有域和参考值。

要分析的设计空间中的设计点通过为每个参数指定样本值（采样）并组合参数样本以创建设计点集来创建。提供了少量简单的命令用于参数值采样和组合采样的参数值；这些命令将在后面详细描述。

在任何组合参数样本指定之前，必须给出参数化研究中参数的初始定义和采样。首次组合后，可以在指定下一个组合之前更改任何单个参数的初始定义和/或采样，从而在一个参数化研究中提供很大的灵活性。

参数定义中给出的参数的可能值域和参考值可以通过在采样期间不同地指定它们来临时重新定义。只要在采样期间指定了参数域和参考值，您就不必在参数定义中指定它们。

可以对所有设计施加设计约束。违反任何约束的设计将被消除。

最后，在所有参数化研究变体分析完成后，您可以跨参数化研究的所有或某些设计收集和报告结果。

总之，Abaqus中的参数化研究组织如下：
- 创建参数化研究。
- 定义参数：定义参数类型（连续或离散值）和可能的参数域和参考值。
- 采样参数：指定采样选项和数据，并可能临时重新定义参数域和参考值。
- 组合参数样本以创建设计点集。
- 约束设计（可选）。
- 生成设计和分析作业数据。
- 执行所研究设计的选定设计的分析作业。
- 收集所研究设计的选定设计的关键结果。
- 报告收集的结果。

**注意：**定义、采样和组合的步骤序列可以根据需要重复多次，以创建所有所需的设计集。可以在一个输入文件包含的模型上执行多个参数化研究。通常，会在输入文件中定义和使用更多参数来替代输入量，而不是涉及任何特定参数化研究的参数。在这些情况下，未涉及特定参数化研究的参数将在该参数化研究期间保留其在输入文件中定义的值。因此，我们可以将输入文件中定义的参数值视为代表一个标称设计；参数化研究通过覆盖某些（或全部）参数的值来创建修改的设计。

### 定义设计空间

设计空间由选择要研究的参数以及指定参数类型和可能值来定义。

#### 参数化研究创建

使用`***aStudy*=ParStudy**`脚本命令（参见["创建参数化研究，" 第20.2.8节](pt04ch20s02asr08.md)）创建参数化研究并选择要考虑变化的独立参数。`***aStudy***`是分配给您由此命令创建的参数化研究对象的Python变量名。参数化研究对象的方法用于执行参数化研究的所有操作。

| **输入文件用法：** | ``` ***aStudy*=ParStudy** (**par**=, **name**=, **verbose**=, **directory**=) ``` |
| --- | --- |

#### 参数定义

使用`***aStudy*.define**`命令（参见["为参数化研究定义参数，" 第20.2.3节](pt04ch20s02asr03.md)）指定参数类型（选择CONTINUOUS或DISCRETE符号；符号是在特定命令中选择选项的符号常量），并可选择指定参数的可能值域和参数的参考值。如果在此命令中未指定域和/或参考值，则可以在参数采样中指定。

参数的重新定义被视为完全重新定义；即，不保留该参数先前定义中的任何信息。

| **输入文件用法：** | ``` ***aStudy*.define** (*token*, **par**=, **domain**=, **reference**=) ``` |
| --- | --- |

##### CONTINUOUS参数类型

在这种情况下，参数可以取连续域中最小值和最大值之间的任何值；例如，**domain**=(3., 10.)。

##### DISCRETE参数类型

在这种情况下，参数只能取定义离散域的列表中指定的值；例如，**domain**=(1, 4, 9, 16)。

### 采样和组合参数值以创建设计点集

在使用组合操作创建第一组设计点之前，必须对参数化研究中的每个参数进行采样。在执行后续组合操作之前，可以重新定义或重新采样参数化研究中的任何参数。

#### 参数采样

使用`***aStudy*.sample**`命令（参见["为参数化研究采样参数，" 第20.2.10节](pt04ch20s02asr10.md)）并选择可用符号之一（INTERVAL、NUMBER、REFERENCE或VALUES）来选择如何进行采样。必须给出的采样数据取决于如何进行采样，如下所述。

##### 按INTERVAL采样

此采样命令假定您指定了参数值的可能域，并希望在该域中以固定间隔采样参数值。始终对参数的极值进行采样。采样的参数值数量取决于间隔和域。因为极值被采样了，所以最后一个采样间隔通常小于您指定的间隔。

此采样命令中的域规范是可选的：
- 如果在此命令中指定了域，它将临时重新定义**define**命令中指定的域。
- 如果在此命令中未指定域，则使用**define**命令中的域规范进行采样。
- 如果在此命令或**define**命令中都未指定域，则会标记错误。

对于连续参数和离散参数，采样间隔的解释不同：
- 对于连续值参数，样本间隔的间隔基于数值。例如，对于具有**domain**=(10., 35.)的连续参数指定**interval**=10.，将为此参数采样值10.、20.、30.和35.
- 对于离散值参数，样本间隔的间隔基于值列表的索引。索引意味着条目在列表中的位置，从位置0开始，依次为位置1、2、3等。在这种情况下，**interval**必须是整数。例如，对于具有**domain**=(1., 2., 3., 5., 7., 10.)的离散参数指定**interval**=2，将为此参数创建样本值10.、5.、2.和1.

间隔可以有正值或负值（不允许零）。正间隔表示从连续参数的最小值或离散参数值列表中的第一个值开始采样（前向采样）。负间隔表示从连续参数的最大值或离散参数值列表中的最后一个值开始采样（反向采样）。当使用TUPLE组合操作时，反向采样很有用（参见参数样本组合的讨论）。

INTERVAL选项有两个特殊情况值得注意：
- 大于连续参数值范围或离散参数值数量的正间隔值将采样连续参数的最小值和最大值或离散参数列表中的第一个和最后一个值。
- 大于连续参数值范围或离散参数值数量的负间隔值（绝对值）将采样连续参数的最大值和最小值或离散参数列表中的最后一个和第一个值。

| **输入文件用法：** | ``` ***aStudy*.sample** (INTERVAL, **par**=, **interval**=, **domain**=) ``` |
| --- | --- |

##### 按NUMBER采样

此采样选项假定您指定了参数值的可能域，并希望在该域中采样固定数量的参数值。除下面记录的特殊情况外，始终对参数的极值进行采样。参数以相等间隔采样（离散参数有一些例外，如下所述），间隔大小取决于采样的数量以及域。

此采样命令中的域规范是可选的：
- 如果在此命令中指定了域，它将临时重新定义**define**命令中指定的域。
- 如果在此命令中未指定域，则使用**define**命令中的域规范进行采样。
- 如果在此命令或**define**命令中都未指定域，则会标记错误。

对于连续参数和离散参数，采样间隔的计算和解释不同：
- 对于连续值参数，样本间隔的间隔基于数值。例如，对于具有**domain**=(10., 25.)的连续参数指定**number**=4，将为此参数采样值10.、15.、20.和25.
- 对于离散值参数，样本间隔的间隔基于值列表的索引（索引从零开始）。例如，对于具有**domain**=(1., 2., 3., 5., 7., 10., 12.)的离散参数指定**number**=3，将为此参数创建样本值1.、5.和12.您指定的离散参数样本数量可能不允许等间隔采样；例如，对于上述离散参数指定**number**=5或**number**=6不允许等间隔采样。这通过采样最接近等间隔的参数值来解决，方法是将采样索引四舍五入到列表中最接近的索引。例如，对于上述离散参数指定**number**=5将创建样本值1.、3.、5.、10.和12.采样值1.和12.是因为它们是极值。第二个采样值是列表中第三个值（值3.）的解释如下：采样间隔=（最高索引-最低索引）/（**number**-1）=（6-0）/(5-1)=1.5；然后第二个采样值应该是索引=0+1.5=1.5的值；由于索引必须是整数，我们四舍五入到索引=2，因此采样列表中的第三个值。其他采样值可以类似地解释。相同的规则用于字符串类型离散参数。例如，对于具有**domain**=('C3D8', 'C3D8R', 'C3D8I', 'C3D8H')的离散参数指定**number**=3，将创建样本值'C3D8'、'C3D8I'和'C3D8H'。

NUMBER选项有三种特殊情况值得注意：
- 指定**number**=1将采样参数的中心值，当感兴趣的设计空间中心时很有用。这是NUMBER选项不使用参数极值采样的唯一情况。
- 指定**number**=2将采样参数的极值，当感兴趣的设计空间边界时很有用。
- 指定**number**=3将采样参数的中心值和极值，当感兴趣的设计空间中心和边界时很有用。

不允许指定**number**=0。**number**可以使用负值；这表示采样按反向顺序进行。对于连续参数，反向顺序意味着第一个采样值最大，最后一个采样值最小。对于离散参数，反向顺序意味着第一个采样值是列表中的最后一个，最后一个采样值是列表中的第一个。反向采样在TUPLE组合操作使用时很有用（参见参数样本组合的讨论）。

| **输入文件用法：** | ``` ***aStudy*.sample** (NUMBER, **par**=, **number**=, **domain**=) ``` |
| --- | --- |

##### 按REFERENCE采样

此采样选项允许您为参数指定参考值，并相对于此参考值采样参数值。这对于研究与现有（参考）设计相关的替代设计很有用。

此采样命令创建关于参考值对称的样本值，间隔为给定间隔的倍数；此外，参考值也被采样。样本中的参数值数量取决于您指定的对称对的数量。

此采样选项中的参考值规范是可选的：
- 如果在此命令中指定了参考值，它将临时重新定义**define**命令中指定的参考。
- 如果在此命令中未指定参考值，则使用**define**命令中的参考规范进行采样。
- 如果在此命令或**define**命令中都未指定参考值，则会标记错误。

对于连续参数和离散参数，参考值的解释不同：
- 对于连续值参数，**reference**是关于它创建对称样本的参数数值。
- 对于离散值参数，**reference**是关于它创建对称样本的值列表的索引。

超出参数域定义的参考值会被标记为错误。

对于连续参数和离散参数，采样间隔的解释不同：
- 对于连续值参数，采样间隔的间隔基于数值。例如，对于连续参数指定**reference**=50.、**interval**=10.和**numSymPairs**=2，将为此参数创建样本值30.、40.、50.、60.和70.
- 对于离散值参数，采样间隔的间隔基于值列表的索引（索引从零开始）；在这种情况下，**interval**必须是整数值。例如，对于具有**domain**=[1, 2, 3, 5, 7, 10, 12, 15, 20, 25]的离散参数指定**reference**=5、**interval**=2和**numSymPairs**=2，将为此参数创建样本值25、15、10、5和2。

指定的**interval**可以具有正值或负值，但不允许零值。正间隔表示采样值列表从连续参数的最小采样值或离散参数值列表最接近开头的采样值开始（前向采样）。负间隔表示采样值列表从连续参数的最大采样值或离散参数值列表最接近末尾的采样值开始（反向采样）。反向采样在TUPLE组合操作使用时很有用（参见参数样本组合的讨论）。

您指定的对称对数量必须为零或正整数；将对称对数量设置为零表示仅采样参考值。

此命令中的域规范是可选的：
- 如果在此命令中指定了域，它将临时重新定义**define**命令中指定的域。
- 如果在此命令中未指定域，则使用**define**命令中的域规范进行采样。
- 对于离散值参数，当在此命令或**define**命令中都未指定域时，会标记错误。

对于离散值参数，需要域规范（在此命令或**define**命令中），因为必须知道可以采样的可能离散值。虽然对于连续值参数不需要域规范，但可以给出。无论对于离散参数还是连续参数，域规范都可用于限制使用REFERENCE选项采样的值数量，因为域被视为可能采样值的边界。例如，对于具有**domain**=(35., 100.)的连续参数指定**reference**=50.、**interval**=10.和**numSymPairs**=3，将为此参数采样值40.、50.、60.、70.和80.域的最小值在此采样中作为边界。

| **输入文件用法：** | ``` ***aStudy*.sample** (REFERENCE, **par**=, **reference**=, **interval**=, **numSymPairs**=, **domain**=) ``` |
| --- | --- |

##### 按VALUES采样

此采样选项假定您希望直接创建参数样本值。您必须指定实际的参数值，无论参数是连续还是离散。

在**define**命令中指定的参数域不影响使用此选项为参数采样的值。

| **输入文件用法：** | ``` ***aStudy*.sample** (VALUES, **par**=, **values**=) ``` |
| --- | --- |

#### 参数样本的组合

使用`***aStudy*.combine**`命令（参见["为参数化研究组合参数样本，" 第20.2.1节](pt04ch20s02asr01.md)）从参数样本创建设计点集。选择使用以下符号之一进行组合方式：MESH、TUPLE或CROSS。每个组合命令的使用都会创建一组设计点，这些点被分组到设计集中。如果组合操作创建的设计复制了现有设计集中的设计，则重复的设计会被立即删除。参数化研究中的设计总数（在应用任何设计约束之前）是每个设计集中设计数量的总和。

您可以为设计集命名；如果不命名，则使用默认值。默认命名约定是：参数化研究中第一个非用户命名的设计集为*p1*，第二个为*p2*，依此类推。设计集名称用于帮助识别各个设计。使用与先前指定的设计集名称相同的名称命名的设计集表示这是设计集的重新指定，因此会覆盖先前存在的那个。

| **输入文件用法：** | ``` ***aStudy*.combine** (*token*, **name**=) ``` |
| --- | --- |

##### MESH组合

此组合选项表示要将参数的每个采样值与参数化研究中每个其他参数的每个采样值组合。

以下示例说明了MESH组合选项的用法。在具有如下定义和采样的两个参数的研究中

```
**study=ParStudy**(**par**=('par1', 'par2'))
**study.define**(DISCRETE, **par**='par1',

##### TUPLE组合

此组合选项创建由采样参数值的*n*元组组成的设计集，其中*n*是参数化研究中的参数数量。每个*n*元组由每个参数的一个采样值组成。例如，在三参数研究中，三个参数的第一个采样值组成第一个3元组，三个参数的第二个采样值组成第二个3元组，依此类推。当任何参数样本用完采样值时，元组创建停止。

以下示例说明了TUPLE组合操作的用法。对于具有如下定义和采样的两参数研究

```
**study=ParStudy**(**par**=('par1', 'par2'))
**study.define**(DISCRETE, **par**='par1',

**study.define**(CONTINUOUS, **par**='par2', **domain**=(10., 60.))
**study.sample**(INTERVAL, **par**='par1', **interval**=1)
**study.sample**(INTERVAL, **par**='par2', **interval**=10.)

##### CROSS组合

此组合选项创建"十字形"模式的设计，如下所示：单个参数采样的每个值与参数化研究中所有其他参数的**define**命令中指定的参考值组合。要使用CROSS组合选项，必须为参数化研究中的所有参数在**define**命令中指定参考值。

### 参数化研究设计的生成和执行

一旦指定了所需的设计点，就需要生成相应的分析作业数据并执行分析。

#### 生成分析作业数据

使用`***aStudy*.generate**`脚本命令（参见["生成参数化研究的分析作业数据，" 第20.2.6节](pt04ch20s02asr06.md)）为每个设计生成输入文件。

必须指定从中生成每个设计输入文件的参数化模板输入文件的名称。

参数化研究生成的输入文件的命名约定如下：
- 每个分析作业的名称将以模板输入文件名开头；例如，`shell`。
- 参数化研究的名称（您在**ParStudy**命令中使用**name=**选项指定）将被附加，前面带有下划线(_)；例如，对于使用`study = ParStudy(name='thickness')`命令定义的参数化研究，为`shell_thickness`。（如果未给出参数化研究名称，则参数化研究名称默认为定义该研究的Python脚本文件的名称。）如果模板输入文件名和参数化研究名称相同，则不重复名称。
- 设计集名称（您在**combine**命令中指定或默认创建的）将被附加，前面带有下划线(_)；例如，对于上述参数化研究的第一个设计集（默认命名），为`shell_thickness_p1`。
- 设计名称（在**combine**命令中自动创建的）将被附加，前面带有下划线(_)；例如，对于上述参数化研究第一个设计集中的第一个设计，为`shell_thickness_p1_c1`。

通常，每个输入文件都有`.inp`扩展名。您可以在执行之前检查和/或编辑这些输入文件。

**generate**命令创建一个具有`.var`扩展名的文件，其中包含参数化研究的描述。该文件被赋予参数化研究名称——例如，`*studyName*.var`——并包含所有生成的设计列表以及与每个设计关联的参数值。您可以检查和/或编辑此文件；但是，编辑此文件将影响参数化研究设计的結果收集（参见"收集结果"）。

每次使用**generate**命令时，都会创建新版本的`*studyName*.var`文件，反映所有先前**combine**命令指定的设计。

define、sample和combine步骤在执行generate命令之前完成。因此，您可以引用模板输入文件中不存在或不是独立参数的参数。这些错误由**generate**命令检测和标记。

| **输入文件用法：** | ``` ***aStudy*.generate** (**template**) ``` |
| --- | --- |

#### 执行参数化研究设计的分析

使用`***aStudy*.execute**`命令（参见["执行参数化研究设计的分析，" 第20.2.4节](pt04ch20s02asr04.md)）执行研究设计的分析。

该命令将在Python进程的控制下提交多个分析作业以执行。通过指定此命令的ALL或DISTRIBUTED选项，可以评估所有设计而无需进一步用户交互，或者通过指定INTERACTIVE选项来交互控制分析的執行。在交互情况下，系统将提示您输入进一步执行指令。提示允许您：
- 指定在进程暂停并再次提示您之前要执行的分析数量。
- 执行剩余的分析而无需进一步用户交互。
- 指定在进程暂停并再次提示您之前要跳过的执行数量。
- 停止执行。

交互选项很有用，因为它提供了以下机会：
- 研究已执行分析的结果。
- 删除分析生成的不必要文件，以在分析许多设计时节省磁盘空间。
- 仅分析参数化研究的某些设计。

ALL和INTERACTIVE符号用于在您的机器上顺序执行Abaqus分析。DISTRIBUTED符号可用于在多台机器或一台机器的多个CPU上调度分析作业。

DISTRIBUTED选项仅在UNIX操作系统的变体上可用。特别是，实现依赖于操作系统对`rsh`、`rcp`和`xhost` UNIX命令的支持。由于使用这些命令，对于参数化研究的分布式执行，参数化研究本身必须在您的本地计算机上执行。如果在分析期间输出二进制结果，则本地和远程计算机必须二进制兼容。在使用分布式执行功能之前，需要在Abaqus环境文件中配置适当的队列接口。

当您发出**execute**命令时，默认情况下，参数化研究设计的每次分析分析都由Abaqus在后台模式下执行，与使用的命令选项无关。Abaqus对每个设计分析创建的文件将覆盖任何同名现有文件，而不会提示您。

您可以使用**execOptions**选项为每个分析的执行命令添加任何必要的Abaqus执行选项（参阅["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行，" 第3.2.2节](pt01ch03s02abx02.md)）。

| **输入文件用法：** | ``` ***aStudy*.execute** (*token*, **files**= , **queues**= , **execOptions**= ) ``` |
| --- | --- |

### 参数化研究结果

一旦执行了与参数化研究关联的分析，就可以检查关键结果在不同设计中的变化。首先，必须从每个分析的结果文件或输出数据库中收集结果；然后，必须报告这些结果。

`***aStudy*.output**`命令（参见["指定参数化研究结果的来源，" 第20.2.7节](pt04ch20s02asr07.md)）可用于指定要收集的结果来源。`***aStudy*.output**`命令的所有参数都是可选的：要收集结果的文件的规范、分析步骤和增量（非频率步骤）或模态（频率步骤）。如果未指定文件，则将使用结果（`.fil`）文件。如果未指定步骤，则必须在**gather**命令中指定（非频率步骤的增量或频率步骤的模态的默认值是步骤的最后增量或步骤中计算的第一模态，除非在**gather**命令中指定）。某些参数仅适用于输出（`.odb`）数据库：实例名称、请求类型（场或历史）、要收集结果的帧值，以及在访问不同输出数据库时是否应覆盖用于访问输出数据库的内存。

收集结果的来源规范将对所有后续**gather**命令保持有效，直到重新指定来源。重新指定收集来源被视为完全重新指定；即，不保留先前收集来源规范的任何信息。

| **输入文件用法：** | ``` ***aStudy*.output** (**file**=, **instance**=, **overlay**=, **request**=, **step**=, **frameValue**= | **inc**= | **mode**=) ``` |
| --- | --- | --- | --- |

#### 收集结果

使用`***aStudy*.gather**`命令（参见["收集参数化研究的结果，" 第20.2.5节](pt04ch20s02asr05.md)）从每个分析的结果文件或输出数据库中收集结果。

在每次使用**gather**命令时，必须指定与收集的结果记录关联的名称。此标签用于在**report**命令中引用结果记录。

从结果（`.fil`）文件收集结果时，必须通过指定`.fil`列标题下出现的可用输出变量标识符键之一来选择要收集的每个结果记录（参见["Abaqus/Standard输出变量标识符，" 第4.2.1节](pt02ch04s02abv01.md)或["Abaqus/Explicit输出变量标识符，" 第4.2.2节](pt02ch04s02xbv01.md)）。例如，可以指定U或S变量标识符键，但不能指定U1或S11变量标识符键。此外，可以指定MODAL变量标识符键以收集特征值结果记录（那些用记录键1980写入结果文件的记录）；在这种情况下，不需要变量位置数据。

从输出（`.odb`）数据库收集结果时，必须通过指定`.odb`列标题下出现的可用输出变量标识符键之一来选择要收集的每个结果记录（参见["Abaqus/Standard输出变量标识符，" 第4.2.1节](pt02ch04s02abv01.md)或["Abaqus/Explicit输出变量标识符，" 第4.2.2节](pt02ch04s02xbv01.md)）。对于场输出，不得指定分量，而对于历史输出，需要分量编号；例如，可以为场输出指定U或S变量标识符键，而可以为历史输出指定U1或S11变量标识符键。除非输出在装配级别，否则必须在**gather**命令的参数中提供实例名称。实例名称要求的一个例外情况是输出（`.odb`）数据库是从未定义为部件实例装配的模型生成的，这从输出数据库中存在名为Assembly-1的单个装配和名为PART-1-1的单个部件实例推断出来的。在这种情况下，您不必明确引用实例PART-1-1。

结果记录的分量名称是自动创建的。例如，命令

```
**myStudy.gather**(**results**='e52_stress', **variable**='S', **element**=52)
```

创建结果记录`e52_stress`，其六个分量（在三维实体单元的情况下）命名为：`e52_stress.1`（S11应力分量）、`e52_stress.2`（S22应力分量）、`e52_stress.3`（S33应力分量）、`e52_stress.4`（S12应力分量）、`e52_stress.5`（S13应力分量）和`e52_stress.6`（S23应力分量）。

必须给出的变量位置数据取决于指定的输出变量标识符键。（参阅["收集参数化研究的结果，" 第20.2.5节](pt04ch20s02asr05.md)，了解位置数据的描述。）必须给出足够的变量位置数据以定义唯一的结果记录。

| **输入文件用法：** | ``` ***aStudy*.gather** (**request**=, **results**=, **step**=, **frameValue**= | **inc**= | **mode**=, **variable**=, *additional location data*) ``` |
| --- | --- | --- | --- |

#### 报告结果

使用`***aStudy*.report**`脚本命令（参见["报告参数化研究结果，" 第20.2.9节](pt04ch20s02asr09.md)）报告从参数化研究结果文件中收集的结果。使用PRINT、FILE或XYPLOT选项指定要生成的报告类型：
- PRINT表示要将结果表（带标题）打印到默认输出设备（屏幕）。您可能希望限制表中的列数，以便使表可读。
- FILE表示要将结果表（带标题）写入ASCII文件。
- XYPLOT表示要将结果表（不带标题）写入ASCII文件，该文件可稍后读入Abaqus/CAE的可视化模块以显示X-Y曲线。

表中的每一行代表参数化研究中的一个设计。表中的列可以代表参数值、收集结果值或设计名称。

可以在**report**命令中指定一个或多个参数。如果未指定参数，默认为参数化研究中的所有参数都包含在表中。对应每个参数的列显示每个包含在表中的设计中该参数的值。

可以指定设计集名称以将表中的行限制为属于该集的设计（参阅前面描述的**combine**命令）。如果未指定设计集，默认为所有设计都包含在表中。

使用**variations**=ON指定表的第一列必须显示设计名称。如果未指定**variations**=ON或包含**variations**=OFF，则表中不包含设计名称列。

必须作为序列指定要报告的结果名称；例如，单元33的Mises应力、单元52的S22应力和节点10的U3位移在以下三个单独命令中收集：

```
**myStudy.gather**(**results**='e33_sinv', **variable**='SINV', **element**=33)
**myStudy.gather**(**results**='e52_s', **variable**='S', **element**=52)
**myStudy.gather**(**results**='n10_u', **variable**='U', **node**=10)
```

可以使用以下**report**命令在单个表中打印这些结果：
```
**myStudy.report**(PRINT,
    **results**=('e33_sinv.1', 'e52_s.2', 'n10_u.3'))
```

此示例不仅显示如何将不同类型（单元、节点等）的收集结果收集在单个表中，还显示如何引用结果记录的分量（Mises应力是SINV的第一个分量，S22是S的第二个分量，U3是U的第三个分量——参阅["结果文件输出格式，" 第5.1.2节](pt02ch05s01afi01.md)或[Abaqus脚本用户指南](../cmd/cmd-link.md#cmd)，了解结果如何分别存储在结果文件和输出数据库中的描述）。

当使用FILE符号或XYPLOT符号时，必须给出文件名以指定要写入结果的文件。在同一会话中使用相同文件名发出的后续**report**命令会将新结果追加到文件。但是，在不同会话中使用相同文件名发出的后续**report**命令会覆盖现有文件。

| **输入文件用法：** | ``` ***aStudy*.report** (*token*, **results**=, **par**=, **designSet**=, **variations**=, **file**=) ``` |

### 参数化研究的执行

要执行参数化研究，必须准备参数化输入文件（`*inputFile*.inp`）。此输入文件是用于生成研究参数化变体的模板，必须包含使用参数替代输入量所需的参数定义。参数必须在模板文件中定义；它们不能在模板文件引用的任何include文件中定义。

此外，必须准备一个Python脚本文件`*scriptFile*.psf`，其中包含用于参数化研究操作的脚本指令。

通常，您使用编辑器准备Python脚本文件，然后使用Abaqus执行命令**abaqus** **script**=*scriptFile*执行此文件。此命令启动Python解释器并执行脚本文件中的指令。或者，您可以简单地使用Abaqus执行命令**abaqus** **script**启动Python解释器，而不给出脚本文件。在这种情况下，Python解释器保持活动状态，您可以交互地执行其他命令或使用Python命令**execfile('*fileName*')**执行包含在文件中的其他命令。在UNIX机器上可以使用**[Ctrl]**+**d**或在Windows机器上使用**[Ctrl]**+**z**终止Python解释器。

由于您可能希望迭代开发脚本，因此通常最好执行先前准备的脚本；在这种情况下，您只需返回并编辑脚本文件并重新执行，直到对结果满意为止。

您可以使用常规操作系统命令监控参数化变体分析的进度。

#### 在多个会话中执行

您可能希望在研究参数化变体执行后多次收集和报告结果。可以在一个会话中定义、生成和执行参数化研究，并在单独的会话中收集和报告结果。当您开始新会话时，只需重新发出用于创建参数化研究的命令。

### 参数化研究结果的可视化

特定参数化研究变体分析的结果可以像任何其他单分析结果一样进行可视化。

跨参数化研究设计收集的结果的可视化需要结果收集。对于可视化，必须将结果报告到ASCII文件（使用**gather**命令中的XYPLOT选项），这些文件可被Abaqus/CAE的可视化模块读取，以生成结果与参数值或设计名称的X-Y曲线图。

### 脚本命令

参数化研究使用Python语言（Lutz, 1996）在扩展名为`.psf`的文件中进行脚本编写。提供了从**ParStudy**类构造的参数化研究对象，其方法使参数化研究的脚本编写变得简单；这些方法在本章中描述。

### 脚本命令的语法

脚本命令通常具有以下形式：

```
***aStudy*.*method*** (*token*, *data*)
```

`***aStudy***`是方法适用的参数化研究对象；此对象是使用参数化研究构造函数命令构造的。`***method***`是要使用的方法；例如，**define**、**sample**或**execute**。

大多数（但不是全部）命令有一个*token*用于选择命令的选项；例如，`***aStudy*.define (CONTINUOUS, par= )`**表示在参数化研究中定义一个或多个连续参数。符号始终以大写给出，并且是互斥的。

对于大多数（但不是全部）命令，必须指定额外的*data*。

### Python语言规则

脚本文件中的参数化研究脚本必须遵循Python语言的语法和语义。这里描述了此语言的一些重要方面（更一般的Python语言规则在["参数化输入，" 第1.4.1节](pt01ch01s04aus04.md)中讨论）。

#### 注释

注释必须以#符号开头。注释理解为延续到行尾。例如，

```
#
# This parametric study ...
#
studyTempEffects.generate(template='shell') #use shell input file
```

#### 大小写敏感性

所有变量和方法名称、符号以及字符串字面量在所有操作系统上都区分大小写。例如，

```
study.execute( ) # is valid
study.Execute( ) # is not valid because of the capital E
```

```
study.sample(NUMBER, ...) # uses the valid token NUMBER
study.sample(number, ...) # lower case token is not valid
```

```
study.generate(template='aFile') # 'aFile' is different
study.generate(template='afile') # from 'afile'
```

#### 字符串

字符串使用成对的单引号（' '）或双引号（" "）表示。不能使用反向单引号（`）来表示此目的。例如，

```
"double quoted string"
'single quoted string'
```

#### 打印

Python **print**命令可用于获取任何Python对象的打印表示。例如，

```
print 'MY TEXT'
```

将在标准输出设备上打印`MY TEXT`。

#### 列表和元组

参数化研究的脚本方法接受整数、实数和字符串类型。这些原语类型在许多情况下可以可选地包含在元组或列表结构中。尽管列表和元组在Python中有一些差异，但它们可以在参数化研究脚本命令中互换使用；它们只是代表有序的项序列。列表或元组中的项必须用逗号分隔并括在括号或方括号中。例如，

```
aStudy.define(CONTINUOUS, par=('xCoord',))# tuple contains a
                                          # single string item
aStudy.define(CONTINUOUS, par=['xCoord']) # list contains a
                                          # single string item
aStudy.define(CONTINUOUS, par=('xCoord', 'yCoord')) # tuple
aStudy.define(CONTINUOUS, par=['xCoord', 'yCoord']) # list
```

#### 缩进

Python使用缩进来分组语句块。因此，Python语句应该与前一条语句从同一列开始，除非Python需要语句分组。

### 访问参数化研究的数据

在某些情况下，直接编程访问参数化研究的数据是可取的。因此，研究的所有重要数据都存储在可以访问为参数化研究对象的数据成员的存储库中。存储库具有与Python字典类似的接口和行为。存储库数据存储为键值对；并提供了用于访问存储库键和值的方法。语法`*aValue = aRepository[aKey]*`用于检索与存储库键关联的值。使用存储库的**keys()**方法可以获得存储库的键列表；例如，`*allKeys = aRepository*.keys()`。类似地，使用存储库的**values()**方法可以获得存储库的值列表；例如，`*allValues = aRepository*.values()`。以下参数化研究脚本显示了如何访问参数化研究的参数存储库以及如何获取参数`t1`的参数名称列表和样本值列表的示例：

```
studyTempEffect = ParStudy(par=('t1', 't2'))
studyTempEffect.define(CONTINUOUS, par=('t1', 't2'))
studyTempEffect.sample(VALUES, par='t1', values=(200.,300.,400.))
studyTempEffect.sample(VALUES, par='t2', values=(250.,350.,450.))
parRepository = studyTempEffect.parameter
listOfParameters = parRepository.keys()
t1Sample = parRepository['t1'].sample
```

脚本导致以下赋值：`listOfParameters = ['t1', 't2']`和`t1Sample = [200.0, 300.0, 400.0]`。Python **print**命令可用于获取存储库内容的信息。

#### 参数化研究存储库

参数化研究具有以下作为数据成员的存储库和对象：
- `***aStudy*.parameter**：按参数名称键控的参数对象存储库。每个参数对象具有**name**、**type**、**domain**、**reference**和**sample**数据成员。
- `***aStudy*.designSet**：按设计集名称键控的设计集存储库。每个设计集表示为设计点列表。
- `***aStudy*.job**：按相应分析输入文件名（不带`.inp`扩展名）键控的分析作业对象存储库。每个作业对象具有**design**、**status**、**root**、**designSet**和**designName**数据成员。
- `***aStudy*.resultData**：结果记录存储库，按通过将结果名称、下划线字符(_)，以及设计名称依次附加而构造的名称键控。对于从结果（`.fil`）文件检索的结果，每个结果记录的格式与相应输出变量的结果（`.fil`）文件记录相同。对于从输出（`.odb`）数据库检索的场结果，每个结果记录将是包含结果分量的元组。对于从输出（`.odb`）数据库检索的历史结果（只能检索单个分量），结果记录将是包含单个值的元组。
- `***aStudy*.table**：表对象，包含由上次使用的**report**命令格式化的表的表示。表对象具有**title**、**variation**、**designs**和**results**数据成员。
