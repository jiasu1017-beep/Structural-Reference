### 概述

准备协同仿真的 Abaqus 分析涉及以下内容：
- 识别协同仿真分析的 Abaqus 分析步骤；
- 识别 Abaqus 模型中的协同仿真接口区域；以及
- 识别协同仿真期间交换的场。

本节概述了准备协同仿真 Abaqus 分析。本节的讨论是通用性的，可能不适用于每种产品配对。["Abaqus 求解器之间的协同仿真," 第 17.3 节](pt04ch17s03.md) 提供了 Abaqus 求解器之间协同仿真的设置、执行和限制详细信息。对于 Abaqus 与第三方分析程序之间的协同仿真，请参阅相应的用户指南。

### 识别用于协同仿真分析的 Abaqus 步骤

协同仿真事件不需要从 Abaqus 分析的第一步开始。但是，它确实需要从分析步骤开始时开始，并在该分析步骤内结束。因此，您需要在 Abaqus 中定义步骤持续时间，使得协同仿真事件的开始落在 Abaqus 分析步骤的开始，并定义该特定步骤使得协同仿真事件在该步骤结束前结束。Abaqus 模型的常规载荷和边界条件照常指定。

与耦合分析的通信在协同仿真事件开始时启动，并在达到协同仿真事件时间时终止。当在协同仿真事件时间之前达到分析步骤结束时，或者当分析无法继续时（例如由于收敛问题），Abaqus 可能终止协同仿真事件。在这种情况下，会向所有客户端发出警告消息，并终止协同仿真。

### 识别协同仿真接口区域

两个 Abaqus 模型之间或 Abaqus 模型与第三方分析模型之间的相互作用通过称为协同仿真接口区域的公共接口区域发生。协同仿真接口区域可以是离散点集、表面区域或体积区域。您必须在接口区域定义中保持一致；如果在一个分析中定义了表面协同仿真区域，则必须在另一个分析中定义表面协同仿真区域。此外，这些协同仿真区域需要共定位并具有相同的区域边界。

#### 通过离散点相互作用

相互作用可以通过一组离散点发生，其中仅节点位置信息（无单元拓扑信息（如受荷面积））定义协同仿真接口区域。在这种情况下，空间映射仅限于点对点映射，您必须确保模型之间有匹配的节点。

在 Abaqus 中，您可以使用节点集或基于节点的表面来定义由离散点组成的协同仿真接口区域。

| **输入文件用法：** | 使用以下选项在 Abaqus 模型中将节点集定义为协同仿真区域： |
| --- | --- |
| | ``` [*CO-SIMULATION REGION](../key/key-link.md#usb-kws-hcosimulationregion), TYPE=NODE *nodeset_A* ``` 使用以下选项在 Abaqus 模型中将基于节点的表面定义为协同仿真区域： ``` [*SURFACE](../key/key-link.md#usb-kws-msurface), TYPE=NODE *nodeset_A* [*CO-SIMULATION REGION](../key/key-link.md#usb-kws-hcosimulationregion), TYPE=SURFACE *node-based surface name* ``` |

#### 通过表面相互作用

不同域之间的相互作用通过公共接口表面发生。例如，当流体与固体相互作用而不穿透它时，流体-固体界面通过表面定义。在这种情况下，节点位置和单元拓扑信息都定义了协同仿真接口，并执行适当的空间映射以在不同表面网格之间保守地映射场。

| **输入文件用法：** | 使用以下选项在 Abaqus 模型中将基于元素的表面定义为协同仿真区域： |
| --- | --- |
| | ``` [*CO-SIMULATION REGION](../key/key-link.md#usb-kws-hcosimulationregion), TYPE=SURFACE (default) *element-based surface name* ``` |

#### 通过体积相互作用

重叠域之间的相互作用通过体积发生。在这种情况下，节点位置和单元拓扑信息都定义了协同仿真区域，并执行适当的空间映射以在不同体积网格之间保守地映射场。

接口区域由元素集定义。

| **输入文件用法：** | 使用以下选项在 Abaqus 模型中将体积定义为协同仿真区域： |
| --- | --- |
| | ``` [*CO-SIMULATION REGION](../key/key-link.md#usb-kws-hcosimulationregion), TYPE=VOLUME *elset_A* ``` |

### 识别跨协同仿真接口交换的场

域模型的耦合可以通过在协同仿真接口处规定的载荷和/或边界条件来进行。此外，质量、转动惯量和热容项也可以交换。基于物理和相互作用类型及其 enforcement，您必须指定在协同仿真期间在 Abaqus 分析中导入和/或导出的场。

协同仿真接口可以由一组离散点（节点）、表面区域或体积区域组成。并非所有场都可以跨所有区域类型交换。

本节概述了 Abaqus 中所有可用的场。有关在两个 Abaqus 求解器之间交换的场的详细信息，请参阅 ["结构对结构协同仿真," 第 17.3.1 节](pt04ch17s03aus99.md) 和 ["流体对结构和共轭热传递协同仿真," 第 17.3.2 节](pt04ch17s03aus100.md)。有关 Abaqus 与第三方分析程序之间交换的场的详细信息，请参阅相应的用户指南。

| **输入文件用法：** | 使用以下选项将场数据导入 Abaqus： |
| --- | --- |
| | ``` [*CO-SIMULATION REGION](../key/key-link.md#usb-kws-hcosimulationregion), IMPORT *region_A*, *import_field_1* *region_A*, *import_field_2* ``` 使用以下选项从 Abaqus 导出数据： ``` [*CO-SIMULATION REGION](../key/key-link.md#usb-kws-hcosimulationregion), EXPORT *region_A*, *export_field_1* *region_A*, *export_field_2* ``` |

##### 位移

平移自由度的位移（场 ID UT 或 U）可以由 Abaqus/Standard 和 Abaqus/Explicit 导出。位移可以由 Abaqus/Standard、Abaqus/Explicit 和 Abaqus/CFD 导入。导入时，位移从上一个交换时间点的值斜坡上升到下一个目标时间点的值。在隐式动态分析中，导入位移时必须导入速度 and加速度。位移在全局坐标系中。

位移可用于 Abaqus/Standard 和 Abaqus/Explicit 中的点、表面区域和体积区域，以及 Abaqus/CFD 中的表面区域。

位移可以在 Abaqus/CAE 的可视化模块中查看。

##### 速度和加速度

平移自由度的速度（场 ID VT 或 V）和加速度（场 ID AT 或 A）可以由 Abaqus/Standard（用于瞬态程序）和 Abaqus/Explicit 导入和导出。在隐式动态分析中，导入速度或加速度时，必须导入所有三个场——位移、速度和加速度。速度可以由 Abaqus/CFD 导入。速度和加速度在全局坐标系中。

速度和加速度可用于 Abaqus/Standard 和 Abaqus/Explicit 中的点、表面区域和体积区域，以及 Abaqus/CFD 中的表面区域。

##### 旋转

旋转（场 ID UR）可以由 Abaqus/Standard 和 Abaqus/Explicit 导入和导出。在隐式动态分析中，导入旋转时必须导入旋转速度和旋转加速度。旋转在全局坐标系中。

旋转可用于点和表面区域。

旋转可以在 Abaqus/CAE 的可视化模块中查看。

##### 旋转速度和旋转加速度

旋转速度（场 ID VR）和旋转加速度（场 ID AR）可以由 Abaqus/Standard（用于瞬态程序）和 Abaqus/Explicit 导入和导出。在隐式动态分析中，导入旋转速度或旋转加速度时，必须导入所有三个场——旋转、旋转速度和旋转加速度。旋转速度和旋转加速度在全局坐标系中。

旋转速度和旋转加速度可用于点和表面区域。

##### 当前坐标

当前节点坐标（场 ID COORD）可以由 Abaqus/Standard 和 Abaqus/Explicit 导出。坐标是变形结构的当前坐标，无论执行的是小位移还是大位移分析。通常，当结果在不同接口区域之间映射时，首选导出位移（场 ID UT 或 U）而不是当前坐标。在耦合分析客户端不保留原始坐标的情况下，可能需要发送当前坐标值而不是位移。

当前坐标可用于 Abaqus/Standard 和 Abaqus/Explicit 中的点、表面区域和体积区域。

##### 集中力

集中力（场 ID CF），如果导入，在 Abaqus/Standard 中从上一个交换时间点的值斜坡上升到下一个目标时间点的值，在 Abaqus/Explicit 中在交换间隔内保持恒定。集中力在全局坐标系中。

导出集中力时，Abaqus/Standard 传输在具有指定位移的界面节点处的反作用力。反作用力在全局坐标系中导出。

集中力可用于 Abaqus/Standard 和 Abaqus/Explicit 中的点、表面区域和体积区域。

对于 Abaqus/Standard 模拟，可以通过请求输出变量 CF 在 Abaqus/CAE 的可视化模块中查看集中法向力。

##### 集中力矩

集中力矩（场 ID CM），如果导入，在 Abaqus/Standard 中从上一个交换时间点的值斜坡上升到下一个目标时间点的值，在 Abaqus/Explicit 中在交换间隔内保持恒定。集中力矩在全局坐标系中。

集中力矩可用于 Abaqus/Standard 和 Abaqus/Explicit 中的点和表面区域。

对于 Abaqus/Standard 模拟，可以通过请求输出变量 CM 在 Abaqus/CAE 的可视化模块中查看集中力矩。

##### 牵引向量

牵引向量（场 ID TRSHR），由 Abaqus/CFD 支持，导出界面表面上的流体总牵引力（法向和剪切分量）。通常，导出的牵引向量在流体-结构模拟中导入 Abaqus/Standard 或 Abaqus/Explicit 时集成为集中力（场 ID CF）。

牵引向量是全局笛卡尔坐标系中的力向量。

牵引向量可用于 Abaqus/CFD 中的表面区域。

##### 法向压力

法向压力（场 ID PRESS），支持 Abaqus/Standard 导入，是表面的法向牵引分量。导入 Abaqus/Standard 时，压力值从上一个交换时间点的值斜坡上升到下一个目标时间点的值。在大多数情况下，优选导入集中力（场 ID CF），因为这些力包含法向和剪切牵引分量。对于膜状结构，可能最好导入压力。

对于 Abaqus/Standard 模拟，可以通过请求输出变量 P 在 Abaqus/CAE 的可视化模块中查看法向压力。

##### 节点温度

节点温度（场 ID NT）可以由 Abaqus/Standard 和 Abaqus/Explicit 导入和导出，可以由 Abaqus/CFD 导入（作为场 ID TEMP）。导入 Abaqus/Standard 和 Abaqus/CFD 时，温度值从上一个交换时间点的值斜坡上升到下一个目标时间点的值。

温度值可以在结构单元的顶面（SPOS）或底面（SNEG）上交换。不能在双侧表面上交换温度。当在顶面和底面上交换温度时，定义两个不同的区域；一个在顶面交换温度，另一个在底面交换温度。对于体积区域，仅使用自由度 NT11，不应将其用于在由非热单元类型离散的体积上交换温度值。

对于 Abaqus/Standard 模拟，可以通过请求输出变量 NT 在 Abaqus/CAE 的可视化模块中查看节点温度值。

##### 热通量

在 Abaqus/Standard 和 Abaqus/Explicit 中，在节点处进入的热使用集中热通量（场 ID CFL）。集中热通量可用于点、表面区域和体积区域。

热通量值可以在结构单元的顶面（SPOS）或底面（SNEG）上交换。不能在双侧表面上交换热通量。当在顶面和底面上交换热通量时，定义两个不同的区域；一个在顶面交换热通量，另一个在底面交换热通量。

对于 Abaqus/Standard 模拟，可以通过请求输出变量 CFL 在 Abaqus/CAE 的可视化模块中查看集中热通量值。

使用表面热通量（场 ID HFL）在 Abaqus/Standard 中用于进入表面的分布热通量，或在 Abaqus/CFD 中用于离开表面的分布热通量。分布热通量仅适用于表面区域。

### 定义耦合和会合方案

不同类型的分析具有不同的时间积分要求，这将影响或决定协同仿真中分析之间交互的频率，以获得准确和稳健的解决方案。例如，考虑隐式和显式动态分析之间时间积分的差异。此外，Abaqus/Standard 可以自动调整增量大小，以为瞬态问题获取经济且准确的解决方案（参见 ["定义分析中的增量," 第 6.1.2 节](pt03ch06s01abo05.md#usb-anl-aover-incrementation)）。例如，考虑模拟扩散过程的瞬态热传递分析；这里分析可能在解中高梯度的开始处使用小时间增量，在达到稳定状态时在分析结束时使用大时间增量。

用于控制这些协同仿真交换的参数取决于您使用的协同仿真接口。
- 您在协同仿真配置文件中定义协同仿真算法和相关交换参数。
- 对于使用 Abaqus/Standard 和 Abaqus/Explicit 的结构对结构协同仿真，您还必须在输入文件中提供协同仿真控制参数。

### 详细配置文件的耦合和会合方案

您在详细配置文件中定义协同仿真耦合和会合方案。

#### 耦合方案

耦合方案定义分析程序之间的交换顺序，以及耦合模拟是否可以串行、并行或隐式迭代方式运行。在决定耦合方案时，您应考虑解决方案稳定性问题以及对计算资源利用的影响

##### 并行显式耦合方案（Jacobi）

在并行显式耦合方案中，两个模拟同时执行，交换场以更新下一目标时间的相应解决方案。并行耦合方案可以更高效地利用计算资源；然而，与顺序方案相比，它被认为不太稳定，应仅用于使用小耦合步骤的弱耦合物理模拟。协同仿真 partner 分析还必须指定 Jacobi 耦合算法。

##### 顺序显式耦合方案（Gauss-Seidel）

在顺序显式耦合方案中，模拟按顺序执行。一个分析领先，而另一个分析落后于协同仿真。协同仿真 partner 分析还必须指定 Gauss-Seidel 耦合算法。

顺序显式耦合方案应仅用于使用小耦合步骤的弱耦合物理模拟。

##### 迭代耦合方案

在迭代耦合方案中，模拟按顺序执行。一个分析领先，而另一个分析落后于协同仿真。执行每个耦合步骤的多次交换，直到满足终止准则。

终止准则取决于协同仿真中的分析；有关 Abaqus 与第三方分析产品之间的协同仿真，请参阅相应的用户指南。

#### 时间积分方案

Abaqus 每个耦合步骤可能需要多个增量，或者您可以强制 Abaqus 每个耦合步骤使用单个增量。

通常，Abaqus 可以在耦合步骤期间执行多个增量（称为"subcycling"）。在 subcycling 期间，Abaqus/Standard 将载荷和边界条件（膜属性除外）从上一个耦合步骤结束时的值斜坡上升到目标时间的值，而在 Abaqus/Explicit 中，载荷在耦合步骤开始时施加并在耦合步骤期间保持恒定。

Subcycling 允许 Abaqus 使用自己的时间积分来达到目标耦合时间；具体来说，如果存在需要减少增量大小的非线性事件，它允许 Abaqus 减少增量大小。

在某些情况下，您可以强制 Abaqus 使用由耦合步骤大小决定的时间增量大小（即 no subcycling）。这允许两个求解器使用相同的时间积分，避免在耦合步骤期间对量进行插值。当以这种"lockstep"方式进行时，Abaqus 将无法减少时间增量来解析非线性事件，因此，在非线性事件要求减少增量大小的情况下，将终止模拟。

Abaqus 不要求以特定单位系统运行分析。通常，创建 Abaqus 模型时使用的单位系统可能与第三方程序模型使用的单位系统不同。当两个单位系统不同时，两个程序之间交换的场必须经过单位转换。有关进一步建模细节，请参阅相应的第三方程序文档。

### 重启协同仿真

导入 Abaqus/Standard、Abaqus/Explicit 或 Abaqus/CFD 的场不会保存到 Abaqus 重启数据库。因此，要重启协同仿真，耦合分析必须在重启分析开始时发送场。这些场必须与 Abaqus 分析计算的共轭场平衡，以维持平衡。您必须同步分析之间写入的重启信息，以确保模拟在同一解（步骤）时间重启。有关更多信息，请参阅 ["重启分析中的同步协同仿真中写入的重启信息" 第 9.1.1 节"](pt04ch09s01aus53.md#usb-anl-arestart-synch)。具体而言，Abaqus 重启的特定步骤/增量号对应的求解时间必须与耦合分析求解时间对应。

### 限制

以下限制适用：
- Abaqus 模型中的步骤必须定义为使得协同仿真完全适合单个 Abaqus 步骤。此外，Abaqus 作业中只能有一个协同仿真。您可以使用重启功能为分析执行多个协同仿真（参见 ["重启分析," 第 9.1.1 节](pt04ch09s01aus53.md)）。
- 在梁、管道和桁架单元上或三维单元边缘上定义的协同仿真表面或体积不能用作接口区域。您应使用离散点来传递载荷和边界条件。
- 在修改的三角单元或修改的四面体单元上定义的协同仿真表面或体积不能用作接口区域。
- 二次耦合温度-位移单元不能用在使用耦合温度-位移程序的协同仿真中作为接口区域。
- 执行协同仿真时，指定时间点的输出可能无法在请求的时间满足，这取决于同步参数。

根据所使用的第三方分析程序，可能有进一步的限制。有关更多信息，请参阅相应的第三方程序文档。
