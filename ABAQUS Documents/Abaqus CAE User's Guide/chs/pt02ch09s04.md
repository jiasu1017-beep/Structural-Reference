# 9.4 理解创建和分析模型时生成的文件





当您启动会话并开始定义模型时，Abaqus/CAE会生成以下文件：

**重放文件（`*abaqus*.rpy`）**

重放文件包含Abaqus/CAE命令，记录您在会话期间执行的几乎所有建模操作。有关更多信息，请参阅["重放Abaqus/CAE会话，"第9.5.1节"](pt02ch09s05s01.md)。

当您从主菜单栏选择****文件（File）**![](../graphics/images/arrow.gif)**保存（Save）****并保存模型数据库时，Abaqus/CAE会保存以下文件：

**模型数据库文件（`*model_database_ name*.cae`）**

模型数据库文件包含模型和分析作业。有关更多信息，请参阅["什么是Abaqus/CAE模型数据库？，"第9.1节"](pt02ch09s01.md)。

**日志文件（`*model_database_ name*.jnl`）**

日志文件包含Abaqus/CAE命令，这些命令将复制已保存到磁盘的模型数据库。有关更多信息，请参阅["重新创建已保存的模型数据库，"第9.5.2节"](pt02ch09s05s02.md)。

当您继续处理模型时，Abaqus/CAE会继续在重放文件中记录您的操作。此外，Abaqus/CAE会保存以下文件：

**恢复文件（`*model_database_ name*.rec`）**

恢复文件包含Abaqus/CAE命令，这些命令将复制内存中模型数据库的版本。模型数据库恢复文件仅包含自您上次保存以来更改模型数据库的命令。有关更多信息，请参阅["重新创建未保存的模型数据库，"第9.5.3节"](pt02ch09s05s03.md)。

当您提交作业进行分析时，Abaqus/Standard和Abaqus/Explicit会创建一组文件；有关这些文件的完整列表，请参阅["Abaqus使用的文件扩展名，"Abaqus Analysis User's Guide第3.6.1节"](../usb/usb-link.md#usb-int-dfileextensions)。以下列表描述了Abaqus/Standard和Abaqus/Explicit创建的一些文件及其与Abaqus/CAE的关系：

**输入文件（`*job_name*.inp`）**

当您提交作业进行分析时，Abaqus/CAE会生成一个输入文件，供Abaqus/Standard或Abaqus/Explicit读取。有关更多信息，请参阅["分析模型的基本步骤，"第19.2.1节"](pt03ch19s02s01.md)。

**输出数据库文件（`*job_name*.odb`）**

输出数据库文件包含您的分析结果。您使用"步骤（Step）"模块的[输出请求管理器](pt03ch14s04s04.md)来选择在分析期间以及以什么速率将哪些变量写入输出数据库。输出数据库与您从"作业（Job）"模块提交的作业相关联；例如，如果您将作业命名为`FrictionLoad`，则分析会创建一个名为`FrictionLoad.odb`的输出数据库。

当您打开输出数据库时，Abaqus/CAE会加载可视化模块并允许您查看内容的图形表示。您还可以从输出数据库中将零件作为网格导入。如果以写权限打开文件，您可以向输出数据库文件保存X-Y数据对象；否则，一旦创建了输出数据库，您就无法修改其内容。

**输出数据库锁文件（`*job_name*.lck`）**

每当以写访问权限打开输出数据库文件时，都会写入锁文件（`*job_name*.lck`），包括在分析运行并向输出数据库文件写入输出时。锁文件防止您从多个来源同时拥有输出数据库的写权限。当输出数据库文件关闭或创建它的分析结束时，锁文件会自动删除。

**重启文件（`*job_name*.res`）**

重启文件用于继续在完成之前停止的分析。您使用"步骤（Step）"模块来指定哪些分析步骤应写入重启信息以及写入频率。如果您使用的是Abaqus/Explicit，则在"步骤"模块中提供的重启信息控制写入状态文件（`*job_name*.abq`）的数据。有关更多信息，请参阅["配置重启输出请求，"第14.13.2节"](pt03ch14s13hlb02.md)。

**数据文件（`*job_name*.dat`）**

数据文件包含分析输入文件处理器的打印输出，以及在分析期间写入的选定结果的打印输出。Abaqus/CAE自动请求在每个步骤结束时生成当前分析过程的默认打印输出；您无法使用Abaqus/CAE对数据文件的内容进行任何其他控制。

**消息文件（`*job_name*.msg`）**

消息文件包含关于求解进度的诊断或信息性消息。您可以使用"步骤（Step）"模块来控制输出到消息文件的诊断信息。有关更多信息，请参阅["诊断打印，"第14.5.3节"](pt03ch14s05s03.md)。

**状态文件（`*job_name*.sta`）**

状态文件（`*job_name*.sta`）包含关于分析进度的信息。此外，您使用"步骤（Step）"模块来请求将单个节点上单个自由度的值输出到状态文件。有关更多信息，请参阅["自由度监视器请求，"第14.5.4节"](pt03ch14s05s04.md)。

**结果文件（`*job_name*.fil`）**

结果文件以可以被其他应用程序（如后处理程序）读取的格式包含分析中的选定结果。子模型分析可以从输出数据库或结果文件读取全局模型结果。默认情况下，从Abaqus/CAE进行的分析不会创建结果文件。有关更多信息，请参阅["子模型，"Abaqus Analysis User's Guide第10.2节"](../usb/usb-link.md#usbasubmod)和[第38章，"子模型"](pt04ch38.md)。

**注意：**Abaqus/Standard和Abaqus/Exclusive在分析作业时写入数据、消息和状态文件的错误和警告可以由"作业（Job）"模块监控；有关更多信息，请参阅["监控分析作业的进度，"第19.2.6节"](pt03ch19s02hlb01.md)。

当您在可视化模块中打开输出数据库文件并创建新的场输出变量时（有关更多信息，请参阅["创建新的场输出，"第42.7节"](pt05ch42s05.md)），Abaqus/CAE会生成以下文件：

**临时输出数据库文件（`*job_name*.ods`）**

临时输出数据库文件（`*job_name*.ods`）包含一个"会话步骤"，其中您创建的场输出变量（通过对场或帧进行操作）会被保存。当原始输出数据库文件（场输出源自该文件）关闭或Abaqus/CAE会话结束时，此文件会自动删除。

在大多数情况下，Abaqus/CAE生成的文件会被写入工作目录。工作目录是您启动Abaqus/CAE会话的目录，除非您通过从主菜单栏选择****文件（File）**![](../graphics/images/arrow.gif)**设置工作目录（Set Work Directory）****来更改目录。有关更多信息，请参阅["设置工作目录，"第9.7.8节"](pt02ch09s06hlb08.md)。
