### 概述

Abaqus/Explicit中通用接触算法使用的接触公式包括：
- 接触表面权重、表面极性和滑动公式；并且
- 可以有选择性地应用于通用接触域内的特定区域。

通用接触公式使用罚方法在表面之间施加接触约束；约束施加方法在["Abaqus/Explicit中的接触约束施加方法，" 第38.2.3节](pt09ch38s02aus182.md)中讨论。

### 指定接触公式

目前您只能为通用接触算法指定接触表面权重和极性。接触公式在通用接触相互作用活跃的所有分析步骤中传播。

用于指定应分配非默认接触公式的区域的表面名称不必与用于指定通用接触域的表面名称对应。在许多情况下，接触相互作用将为一个大域定义，而非默认接触公式将被分配给该域的子集。任何针对落在通用接触域之外的区域的接触公式分配将被忽略。如果指定区域重叠，则最后一个分配优先。

|  | 此选项必须与[*CONTACT](../key/key-link.md#usb-kws-hcontact)选项结合使用。它每个步骤中每个TYPE参数值最多出现一次；数据行可以根据需要重复多次，以将接触公式分配给不同区域。 |

### 接触表面权重

通常，有限元模型中的接触约束以离散方式施加，这意味着对于硬接触，一个表面上的节点被约束为不穿透另一个表面。在纯主-从接触中，具有约束的节点是从属表面的一部分，与其相互作用的表面称为主表面。对于平衡主-从接触，Abaqus/Explicit以惩罚力的形式对每组接触表面计算两次接触约束：一次将第一个表面作为主表面，一次将第二个表面作为主表面。两个修正（或力）的加权平均值应用于接触相互作用。

平衡主-从接触最小化接触体的穿透，因此，在大多数情况下提供更好的接触约束施加和更准确的结果。在纯主-从接触中，主表面上的节点原则上可以不受阻碍地穿透从属表面（参见[图38.2.1-1](pt09ch38s02aus180.md#agencontact-mast-surf-pen)）。

**图38.2.1-1** 由于粗离散，纯主-从接触中主表面穿透从属表面。

![](../graphics/acontact-mast-surf-pen-nls.png)

Abaqus/Explicit中的通用接触算法尽可能使用平衡主-从权重；当涉及基于节点的表面（只能作为纯从属表面）和涉及解析刚性表面的接触相互作用（只能作为纯主表面）时，使用纯主-从权重。基于表面的内聚行为也始终使用纯主-从算法。但是，您也可以选择为其他相互作用指定纯主-从权重。

边缘对边缘接触没有主-从关系；两个接触边缘被赋予相等的权重。

#### 为节点-面接触指定纯主-从权重

您可以指定通用接触相互作用应使用纯主-从权重进行节点-面接触。此规格对边缘-边缘接触没有影响，也不能用于使基于节点的表面作为主表面。当两个最初平的表面相互接触时，如果较 refined 的表面作为从属表面，与平衡主-从权重相比，纯主-从权重可能会产生更均匀的穿透距离分布（从而产生更均匀的压力分布）。如果接触表面的网格密度差异显著，这一点可能特别明显——使用平衡权重，粗网格化表面节点附近的接触穿透会较小。

Abaqus/Explicit将自动为与指定主-从方向相反的方向生成接触排除；因此，任何两个表面重叠区域的面-节点自接触将被排除。例如，指定通用接触相互作用在*surf_A*和*surf_B*之间应使用纯主-从权重，*surf_A*被视为从属表面，将导致为*surf_A*的面接触*surf_B*的节点生成内部排除；*surf_A*和*surf_B*之间的重叠区域的面-节点自接触将被排除。如果省略第二个表面名称或第二个表面名称与第一个相同，则会发出警告消息，因为此输入将导致表面的面-节点自接触被排除。

| **输入文件用法：** | 使用以下选项指示第一个表面应被视为从属表面（默认）： |
| --- | --- |
|  | ``` [*CONTACT FORMULATION](../key/key-link.md#usb-kws-hcontformulation), TYPE=PURE MASTER-SLAVE *surf_1*, *surf_2*, SLAVE ``` 使用以下选项指示第一个表面应被视为主表面： ``` [*CONTACT FORMULATION](../key/key-link.md#usb-kws-hcontformulation), TYPE=PURE MASTER-SLAVE *surf_1*, *surf_2*, MASTER ``` 如果省略第一个表面名称，则假定为涵盖整个通用接触域的默认表面。必须指定第二个表面名称。 |

### 接触表面极性

默认情况下，通用接触考虑指定用于接触目的的所有双面元素的两个面（双面元素的side标签被忽略）。对于节点-面接触和欧拉-拉格朗日接触，可以覆盖此默认设置，在某些情况下可以获得更准确的接触约束施加。

对于边缘-边缘接触不考虑表面极性，包括在实体元素面上激活的边缘。

#### 为节点-面接触和欧拉-拉格朗日接触指定表面极性

更改双面元素的极性会强制接触算法将其视为实体元素。如果从属节点有可能在节点-面接触中"卡在"表面后面，或者如果双面表面一侧的材料泄漏到另一侧（在欧拉-拉格朗日接触中），则通过将双面元素转换为单面可以获得更高的精度。如果双面拉格朗日表面与所有欧拉材料表面接触时将双面拉格朗日表面转换为单面，也可以观察到性能和内存使用的改善。

| **输入文件用法：** | 使用以下选项指示应在第一个表面的接触中考虑第二个表面定义中指定的双面元素的两个面： |
| --- | --- |
|  | ``` [*CONTACT FORMULATION](../key/key-link.md#usb-kws-hcontformulation), TYPE=POLARITY *surf_1*, *surf_2* ``` 使用以下选项指示应在第一个表面的接触中考虑第二个表面中双面元素的SPOS面： ``` [*CONTACT FORMULATION](../key/key-link.md#usb-kws-hcontformulation), TYPE=POLARITY *surf_1*, *surf_2*, SPOS ``` 使用以下选项指示应在第一个表面的接触中考虑第二个表面中双面元素的SNEG面： ``` [*CONTACT FORMULATION](../key/key-link.md#usb-kws-hcontformulation), TYPE=POLARITY *surf_1*, *surf_2*, SNEG ``` 使用以下选项指示应在第一个表面的接触中考虑第二个表面中双面元素的两个面： ``` [*CONTACT FORMULATION](../key/key-link.md#usb-kws-hcontformulation), TYPE=POLARITY *surf_1*, *surf_2*, TWO SIDED ``` 如果省略第一个表面名称，则假定为涵盖整个通用接触域的默认表面。必须指定第二个表面名称。 |

### 滑动公式

目前，Abaqus/Explicit中通用接触仅提供有限滑动公式。此公式允许接触表面任意分离、滑动和旋转。对于首选小滑动或无限小滑动假设的情况，应使用接触对算法（参见["Abaqus/Explicit中接触对的接触公式，" 第38.2.2节](pt09ch38s02aus181.md)）。

Abaqus/Explicit旨在模拟高度非线性事件或过程。因为一个表面上的节点可能接触另一表面上任何面元，Abaqus/Explicit必须使用复杂的搜索算法来跟踪表面运动。有限滑动接触搜索算法设计为稳健且计算效率高。此算法假设表面之间的增量相对切向运动不会显著超过主表面面元的尺寸，但表面之间没有总体相对运动的限制。由于显式动态分析中使用的较小时间增量，增量运动超过面元大小的情况很少见。在涉及超过材料波速的相对表面速度的情况下，可能需要减小时间增量。

接触搜索算法在首次引入接触相互作用时使用全局搜索，此后使用分层全局/局部搜索算法。不需要用户控制搜索算法。
