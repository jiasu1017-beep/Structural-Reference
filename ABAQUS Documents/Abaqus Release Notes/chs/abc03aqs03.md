# 3.3 通过合并输出数据库文件查看优化进程





**产品：** Abaqus/CAE

**优势：** 优化过程的每个设计周期现在都会创建一个单独的输出数据库文件。要在 Visualization 模块中查看优化结果，必须将单独的输出数据库文件与优化结果合并为一个输出数据库文件。

**描述：** 以前版本的 Abaqus/CAE 将每个设计周期的分析结果存储到单个输出数据库文件中，导致输出数据库文件非常大，消耗大量磁盘空间且处理缓慢。默认情况下，Abaqus/CAE 继续在每个设计周期后保存优化数据。但是，每个设计周期创建的单独输出数据库文件现在仅在指定间隔保存。或者，您可以按如下方式控制分析数据保存到输出数据库文件的速率：
- 每个设计周期后，
- 从初始、第一个或最后一个设计周期开始，或
- 每 *n* 个设计周期后。

要在 Visualization 模块中查看优化结果，必须首先将单独的输出数据库文件和优化文件合并为单个输出数据库文件。合并输出数据库文件时，您可以选择要合并的步骤和载荷工况；您还可以选择要包含在合并输出数据库文件中的场变量。此外，如果您的优化过程包含多个模型，您可以选择要包含的模型；Abaqus/CAE 为每个模型创建一个合并的输出数据库文件。

**Abaqus/CAE 使用：**
```
Job 模块：
    ****Optimization**![](../graphics/images/arrow.gif)**Combine****，**Name：** *设计响应名称*
```
