# 10.5.2 从Abaqus输入文件导入模型





您可以通过从主菜单栏选择****文件（File）**![](../graphics/images/arrow.gif)**导入（Import）**![](../graphics/images/arrow.gif)**模型（Model）****使用Abaqus输入文件将模型导入Abaqus/CAE。从输入文件导入的Abaqus关键词被合并到新模型中；例如，如果杨氏模量是从[*ELASTIC*](../key/key-link.md#usb-kws-melastic)关键词导入的，它将在"属性（Property）"模块中可用。不支持的关键词在导入期间会被忽略。仅在大小写不同的模型组件标签不会一致地导入Abaqus/CAE；因此，您不应该使用大小写来区分相同类型的模型组件。输入文件不一定是完整的；例如，它可能不包含任何历史数据。因为输入文件无法存储Abaqus/CAE模型数据库中的所有数据，所以您不应使用输入文件来归档模型数据。

以下功能可以从Abaqus输入文件导入到模型中：
- 节点和单元
- 曲面、节点和单元集合以及接触节点集合
- 自适应网格控制
- 材料、截面和方向定义
- 模型和材料描述
- 相互作用和相互作用属性
- 离散紧固件（先前在Abaqus/CAE中定义）
- 载荷和边界条件（在全球坐标系中）
- 幅值
- 过程、输出请求和监视变量
**注意：**导入具有大量属性的模型（如步骤或预定义场）（大约8000或更多）可能需要大量时间。

请参阅["输入文件读取器支持的关键词，"第A.2节"](ap01s02.md)，获取输入文件读取器支持的关键词完整列表。并非所有关键词数据行都支持导入；当遇到不支持的条目时，Abaqus/CAE会发出警告消息。您可以使用关键词编辑器来包含输入文件读取器不支持的选项；有关使用关键词编辑器的详细说明，请参阅["将不支持的关键词添加到您的Abaqus/CAE模型中，"第9.10.1节"](pt02ch09s09hlb01.md)。

### 导入零件

零件以网格形式从输入文件导入；网格由节点和单元定义以及分配的单元类型组成。输入文件读取器可以导入包含大多数常用单元类型的网格。但是，输入文件读取器无法导入包含以下单元类型的网格：
- 管道支撑单元（ITS*）
- 用户定义的单元（U*）

以下单元类型可以导入，但它们的截面属性尚不支持：- 分布耦合单元（DCOUP*）
- 拖链单元（DRAG*）
- 框架单元（FRAME*）
- 间隙接触应力/位移单元（GAPCYL、GAPSPHER和GAPUNI）
- 界面单元（INTER*、ISL*、IRS*、ISP*、ITT*和DINTER*）
- 关节单元（JOINT*）
- 线弹簧单元（LS*）

可以导入孔隙压力粘性单元，但您无法在Abaqus/CAE模型中查看它们。但是，分析完成后，您可以在可视化模块中查看孔隙压力粘性单元。

您可以使用"网格"模块更改从输入文件导入的单元分配的单元类型。

### 导入集合和曲面

导入功能根据任何[*ELSET*](../key/key-link.md#usb-kws-melset)或[*NSET*](../key/key-link.md#usb-kws-mnset)关键词以及其他支持的关键词上的ELSET或NSET参数创建集合。如果输入文件是按照零件实例的装配编写的，Abaqus/CAE会在创建零件和装配集合时保留您的意图。如果集合是在零件（可变形或刚体）内部定义的，Abaqus/CAE会创建一个零件集合。当您在"装配"模块中实例化零件时，您可以引用零件集合；但是，与装配相关的模块仅提供对零件集合的只读访问。如果集合是在装配内定义的，Abaqus/CAE会创建一个装配集合。更多信息，请参阅["零件集合和装配集合有什么不同？，"第73.2.2节"](pt06ch73s02s02.md)。

相比之下，如果输入文件不是按照零件实例的装配编写的，Abaqus/CAE会尝试最小化创建的集合数量。在大多数情况下，输入文件中的集合仅作为装配集合导入。但是，如果截面分配引用了某个集合，Abaqus/CAE会将该集合仅作为零件集合导入。

可以导入基于单元的曲面；但是，Abaqus/CAE会将基于节点的曲面导入为节点集合，而不是曲面。因此，导入的基于节点的曲面会出现在集合管理器中，而不是曲面管理器中。

### 导入描述

作为输入文件中注释行出现的模型描述和材料描述会被导入Abaqus/CAE。紧跟在输入文件标题之前的注释行存储模型描述，在导入时可在**编辑模型属性（Edit Model Attributes）**对话框中使用。紧跟在材料和材料行为定义之前的注释行存储材料描述，在导入时可在**编辑材料（Edit Material）**对话框中使用。

来自**编辑作业（Edit Job）**对话框的作业描述显示为输入文件标题的数据行。如果需要，您可以修改此数据行以包含有关作业的更多详细信息；但是，此数据行不会导入Abaqus/CAE。

### 导入相互作用、约束和紧固件

仅当接触对在第一个分析步骤中被激活时，Abaqus/CAE才会导入重新激活的接触对。

根据输入文件识别所涉及节点的方式，Abaqus/CAE会将多点约束导入为MPC连接器截面和线要素或MPC约束。

完全定义的离散紧固件包含连接线 和连接器截面分配。为了在导入时将这些离散紧固件重新创建为Abaqus/CAE中原始建模的形式，当您提交作业进行分析时，Abaqus/CAE会将特殊注释行写入输入文件。这些特殊注释行紧跟在用于定义离散紧固件的Abaqus关键词之前，以`**@ABQCAE`开头，被Abaqus求解器忽略。

无法从输入文件导入装配的紧固件和基于点的（网格无关的）紧固件。

### 分析子模型

当您使用Abaqus/Standard或Abaqus/Explicit分析子模型时，您需要在Abaqus执行过程中提供包含全局解决方案的输出数据库或结果文件的名称；文件名不会出现在输入文件中。因此，当您导入分析子模型的输入文件时，必须指定包含将驱动子模型的全局解决方案的输出数据库或结果文件的名称。选择****模型（Model）**![](../graphics/images/arrow.gif)**编辑属性（Edit Attributes）**![](../graphics/images/arrow.gif)***模型名称*****，然后在**子模型（Submodel）**选项卡页面上输入包含全局解决方案的文件名称。
