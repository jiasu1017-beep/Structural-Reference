# 47.2 Specifying and saving X–Y data objects







### 47.2 Specifying and saving X--Y data objects

This section explains the various methods of specifying and saving *X–Y* data objects. *X–Y* data objects can originate from analysis results, an external file, keyboard entry, or operations on previously saved *X–Y* data objects. The following topics are covered:
- ["Reading X--Y data from output database history output," Section 47.2.1](pt05ch47s02hlb01.md)
- ["Reading X--Y data from output database field output," Section 47.2.2](pt05ch47s02hlb02.md)
- ["Reading X--Y data through the thickness of a shell," Section 47.2.3](pt05ch47s02hlb03.md)
- ["Reading X--Y data from all active free body cuts," Section 47.2.4](pt05ch47s02hlb04.md)
- ["Reading X--Y data from an ASCII file," Section 47.2.5](pt05ch47s02hlb05.md)
- ["Entering X--Y data from the keyboard," Section 47.2.6](pt05ch47s02hlb06.md)
- ["Saving an X--Y data object," Section 47.2.7](pt05ch47s02hlb07.md)
- ["Copying a session X--Y data object to an output database file," Section 47.2.8](pt05ch47s02hlb08.md)
- ["Loading an X--Y data object to the current session," Section 47.2.9](pt05ch47s02hlb09.md)
- ["Editing individual data points in an X--Y data object," Section 47.2.10](pt05ch47s02hlb10.md)

For information on specifying *X–Y* data objects by obtaining field output results along a path through your model, see [Chapter 48, "Viewing results along a path](pt05ch48.md)”; for information on specifying *X–Y* data objects by operating on existing *X–Y* data, see ["Operating on saved X--Y data objects," Section 47.4](pt05ch47s03.md).
