### 概述

Abaqus/Standard中的并行执行：
- 减少大型分析的计算时间；
- 可用于共享内存计算机和计算机集群的元素操作、直接稀疏求解器和迭代线性方程求解器；并且
- 可以在共享内存计算机上为直接稀疏求解器使用支持计算的GPGPU硬件。

### 使用默认直接稀疏求解器的并行方程求解

直接稀疏求解器（["直接线性方程求解器，" 6.1.5节](pt03ch06s01aus46.md)）支持共享内存计算机和计算机集群的并行化。在共享内存计算机或计算机集群的单个节点上，线程级并行化用于直接稀疏求解器，并且可以使用高端图形卡（支持通用处理（GPGPU））来加速求解。在计算机集群的多个计算节点上，使用混合MPI和线程级并行化。

在以下情况下，直接稀疏求解器不能用于计算机集群的多个计算节点：
- 分析还包括特征值提取过程，或者
- 分析需要不支持元素操作的基于MPI的并行执行的功能。

此外，对于包含以下任一情况的分析，直接稀疏求解器不能用于计算机集群的多个节点：
- 具有变化边界条件的多载荷工况（["多载荷工况分析，" 6.1.4节](pt03ch06s01aus45.md)），和
- 准牛顿非线性求解技术（["非线性问题的收敛准则，" 7.2.3节](pt03ch07s02aus51.md)）。

要在计算机集群上执行并行直接稀疏求解器，必须将环境变量**mp_host_list**设置为主机列表（请参阅["使用Abaqus环境设置，" 3.3.1节](pt01ch03s03aus30.md)）。机器之间使用基于MPI的并行化。如果主机列表中的机器有多个处理器且模型不包含使用并行分解的腔体辐射（请参阅["并行分解大型腔体" 41.1.1节"腔体辐射"](pt09ch41s01aus187.md#usb-cni-acavityradiation-parallel)），则主机列表内的机器使用线程级并行化。例如，如果环境文件有以下内容：

```
cpus=8
mp_host_list=[['maple',4],['pine',4]]
```
Abaqus/Standard将通过线程级并行化在每台主机上使用四个处理器。将在主机机器之间运行两个MPI进程（等于主机数），以便所有八个处理器都由并行直接稀疏求解器使用。

包含并行腔体分解的模型仅使用基于MPI的并行化。因此，MPI用于共享内存并行计算机和分布式内存计算集群。进程数等于作业提交期间请求的CPU数。当启用并行腔体分解时，元素操作使用基于MPI的并行化并行执行。

#### 并行直接稀疏求解器的内存要求

并行直接稀疏求解器并行处理多个矩阵，还要并行化单个矩阵的求解。因此，直接并行求解器比串行求解器需要更多内存。内存要求无法准确预知，因为无法先验地确定哪些矩阵将同时处理。

#### 最小求解时间的方程排序

直接稀疏求解器需要对方程组进行排序以最小化浮点运算计数。排序过程在计算机集群上使用多台主机时并行执行。在共享内存配置中，排序过程不并行执行。并行排序过程在不同的主机数上运行时将计算不同的排序，这将影响直接求解器的浮点运算计数。并行排序可以提供性能改进，特别是对于使用许多主机的大型模型，通过显著减少计算排序的时间。如果排序确定导致直接求解器的浮点运算计数更高，并行排序可能导致性能下降。

如果并行排序固有的排序变化不可接受，则可以使用串行排序过程。您可以从命令行停用并行求解器排序，也可以使用**order_parallel**环境文件参数（请参阅["使用Abaqus环境设置，" 3.3.1节](pt01ch03s03aus30.md#usb-int-denvfile-commparam)中的"命令行默认参数"）。

| **输入文件用法：** | 在命令行上输入以下内容以停用并行求解器排序： |
| --- | --- |
|  | **abaqus** **job**=*job-name* **order_parallel**=`OFF` |

| **Abaqus/CAE用法：** | 在Abaqus/CAE中不支持停用并行求解器排序。 |

### 使用迭代求解器的并行方程求解

迭代求解器（["迭代线性方程求解器，" 6.1.6节](pt03ch06s01aus47.md)）仅使用基于MPI的并行化。因此，MPI用于共享内存并行计算机和分布式内存计算集群。要执行并行迭代求解器，请为作业指定CPU数。进程数等于作业提交期间请求的CPU数。当使用并行迭代求解器时，元素操作使用基于MPI的并行化并行执行。

| **输入文件用法：** | 使用以下选项与命令行输入结合来执行并行迭代求解器： |
| --- | --- |
|  | ``` [*STEP](../key/key-link.md#usb-kws-hstep), SOLVER=ITERATIVE ``` 在命令行上输入以下内容：**abaqus** **job**=*job-name* **cpus**=*n* 例如，以下输入将使用迭代求解器在四个处理器上运行名为"cube"的作业：``` abaqus job=cube cpus=4 ``` |

| **Abaqus/CAE用法：** | 步骤模块：步骤编辑器：**其他**：**方法：迭代** |
| --- | --- |
|  | 作业模块：作业编辑器：**并行化**：切换到**使用多个处理器**，并指定处理器数*n* |

### 在Abaqus/Standard中元素操作的并行执行

元素操作的并行执行是所有支持平台上的默认设置。命令行和环境变量**standard_parallel**可用于控制元素操作的并行执行（请参阅["使用Abaqus环境设置，" 3.3.1节](pt01ch03s03aus30.md)和["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行，" 3.2.2节](pt01ch03s02abx02.md)）。如果使用元素操作的并行执行，求解器也会自动并行运行。对于使用直接稀疏求解器且不包含并行腔体分解的分析，在共享内存计算机上使用线程级并行化，在计算机集群上使用混合MPI和线程并行方案。对于使用迭代求解器的分析或如果启用了并行腔体分解，则仅支持元素操作的基于MPI的并行化。

当使用元素操作的基于MPI的并行化时，会为每个域创建元素集，可以在Abaqus/CAE中检查。集合名为`STD_PARTITION_*n*`，其中*n*是域号。

对于以下过程，不支持元素操作的并行执行（线程或基于MPI的并行化）：
- 特征值 buckling预测（["特征值 buckling预测，" 6.2.3节](pt03ch06s02at02.md)），
- 不使用SIM架构的自然频率提取（["自然频率提取，" 6.3.5节](pt03ch06s03at10.md)），
- 响应谱分析（["响应谱分析，" 6.3.10节](pt03ch06s03at15.md)），
- 随机响应分析（["随机响应分析，" 6.3.11节](pt03ch06s03at16.md)），和
- 不使用SIM架构的基于模态的线性动力学（["瞬态模态动态分析，" 6.3.7节](pt03ch06s03at12.md)；["基于模态的稳态动态分析，" 6.3.8节](pt03ch06s03at13.md)；["基于子空间的稳态动态分析，" 6.3.9节](pt03ch06s03at14.md)；和["复特征值提取，" 6.3.6节](pt03ch06s03at11.md)）。

对于包含以下任一情况的分析，元素操作的并行执行仅通过基于MPI的并行化可用：
- 静态线性扰动（["一般和线性扰动过程，" 6.1.3节](pt03ch06s01aus44.md)），
- 直接解稳态动态（["直接解稳态动态分析，" 6.3.4节](pt03ch06s03at09.md)），
- 稳态输运（["稳态输运分析，" 6.4.1节](pt03ch06s04at17.md)），
- 耦合温度-位移（["完全耦合热应力分析，" 6.5.3节](pt03ch06s05at19.md)），
- 压力渗透载荷（["压力渗透载荷，" 37.1.7节](pt09ch37s01aus171.md)），和
- 使用子结构的静态、隐式动态或直接解稳态动态分析（如果未请求在子结构内恢复结果）（["静态应力分析，" 6.2.2节](pt03ch06s02at01.md)；["使用直接积分的隐式动态分析，" 6.3.2节](pt03ch06s03at07.md)；["直接解稳态动态分析，" 6.3.4节](pt03ch06s03at09.md)；["子结构，" 10.1节](pt04ch10s01.md)）。

使用直接稀疏求解器且上述仅支持元素操作基于MPI并行化的任何过程的分析可以在计算机集群上运行。但是，由于不支持线程级并行化，因此每个计算节点仅使用一个处理器用于元素操作。

对于以下情况，元素操作的并行执行仅通过线程级并行化可用：
- 不允许腔体并行分解且请求写入重启数据的腔体辐射分析（["腔体辐射，" 41.1.1节](pt09ch41s01aus187.md)），
- 指定平均温度辐射条件的热传递分析（["热载荷，" 34.4.4节](pt07ch34s04aus123.md)），
- 使用SIM架构的自然频率提取（["自然频率提取，" 6.3.5节](pt03ch06s03at10.md)），
- 使用SIM架构的基于模态的线性动力学（["瞬态模态动态分析，" 6.3.7节](pt03ch06s03at12.md)；["基于模态的稳态动态分析，" 6.3.8节](pt03ch06s03at13.md)；["基于子空间的稳态动态分析，" 6.3.9节](pt03ch06s03at14.md)；和["复特征值提取，" 6.3.6节](pt03ch06s03at11.md)），
- 子结构生成（["定义子结构，" 10.1.2节](pt04ch10s01aus59.md)），和
- 矩阵生成（["生成热矩阵"简介，10.3.2节](pt04ch10s03at33.md#usb-anl-amtxgenerategeneral)）。

最后，对于包含以下任一情况的分析，不支持元素操作的并行执行：
- 元素矩阵输出请求（["Abaqus/Standard中的元素矩阵输出" 4.1.1节"输出"](pt02ch04s01aus38.md#usb-out-ooutput-elemmatrix)），
- 替代求解技术（["近似实现" 6.5.3节"完全耦合热应力分析"](pt03ch06s05at19.md#usb-anl-acouptempdisp-approximp)；["近似实现" 6.7.3节"耦合热电分析"](pt03ch06s07at22.md#usb-anl-ajouleheating-approximp)；和["收敛准则" 7.2.3节"非线性问题"](pt03ch07s02aus51.md#usb-anl-aconvergcriteria-separated)）中的分离方法除外），
- 重启时输出延续（["重启分析中的输出延续" 9.1.1节](pt04ch09s01aus53.md#usb-anl-arestart-continue)），
- 从Abaqus/Explicit导入（["在Abaqus分析之间传输结果：概述，" 9.2.1节](pt04ch09s02aus54.md)），
- 如果请求在子结构内恢复结果，则使用子结构（["子结构，" 10.1节](pt04ch10s01.md)），和
- 自适应网格划分（["在Abaqus/Standard中定义ALE自适应网格域，" 12.2.6节](pt04ch12s02aus82.md)）。

#### 元素操作并行执行时的内存管理

在Abaqus/Standard中运行元素操作的并行执行时，指定可以使用的内存上限（请参阅["Abaqus/Standard分析" 3.4.1节"管理内存和磁盘使用"](pt01ch03s04aus31.md#usb-int-dmemory-standard)）指定每个进程可以分配的最大内存量。

#### 堆叠连续壳的横向剪切应力输出

在使用Abaqus/Standard中元素操作的并行执行时，输出变量CTSHR13和CTSHR23当前不可用。请参阅["连续壳元素库，" 29.6.8节](pt06ch29s06ael18.md)。

### 结果的一致性

某些物理系统（例如经历 buckling、材料失效或分层的系统）可能对小扰动高度敏感。例如，众所周知，一组看似相同的圆柱壳的实验测量的 buckling载荷和最终配置可能由于边界条件、载荷、初始几何等微小差异而显示显著分散。当模拟这样的系统时，实验中看到的物理敏感性可能表现为对有限精度效应引起的小数值差异的敏感性。有限精度效应可能在不同处理器数上运行作业时导致小数值差异。因此，在模拟物理敏感系统时，您可能会看到不同处理器数作业之间的数值结果差异（反映实验中看到的差异）。要获得每次运行一致的模拟结果，处理器数应保持不变。
