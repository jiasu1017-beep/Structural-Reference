# 3.2.6 Abaqus/CAE执行





**产品：** Abaqus/CAE  

##### **参考**

- ["Abaqus执行程序：概述，" 第3.1.1节](pt01ch03s01abo02.md)

### 概述

Abaqus/CAE是一个用于创建、提交、监控和评估Abaqus仿真结果的交互环境，通过运行Abaqus执行程序并指定**cae**参数来执行。

### 命令行选项

**database**

此选项指定要打开的模型数据库文件或输出数据库文件的名称。若要指定模型数据库文件，请在文件名中包含`.cae`文件扩展名或不包含文件扩展名。若要指定输出数据库文件，请在文件名中包含`.odb`文件扩展名。

**replay**

此选项指定要重放Abaqus/CAE命令的文件名。*replay-file*中的命令将在Abaqus/CAE启动时立即执行。如果未给出文件扩展名，则默认扩展名为`.rpy`。您不能使用`replay`选项来执行包含控制流语句的脚本。

**recover**

此选项指定要重建模型数据库的文件名。*journal-file*中的命令将在Abaqus/CAE启动时立即执行。如果未给出文件扩展名，则默认扩展名为`.jnl`。

**startup**

此选项指定包含在应用程序启动时运行的Python配置命令的文件的名称。此文件中的命令在任何已在环境文件中设置的配置命令之后运行。Abaqus/CAE在执行时不将这些命令回放到重放文件中。

**script**

此选项指定包含在应用程序启动时运行的Python配置命令的文件的名称。此文件中的命令在任何已在环境文件中设置的配置命令之后运行。

可以通过在命令行上输入`--`，然后用 一个或多个空格分隔参数来将参数传递到文件中。这些参数将被Abaqus/CAE执行程序忽略，但可以在脚本中访问。

**noGUI**

此选项指定在不显示图形用户界面(GUI)的情况下运行Abaqus/CAE。如果未指定文件名，则会检出Abaqus/CAE许可证，并初始化Python解释器以允许交互式输入Python或Abaqus脚本界面命令。

如果指定了文件名，Abaqus/CAE将运行该文件中的命令，并在完成后退出。如果未给出文件扩展名，则默认扩展名为`.py`。此选项对于自动执行预处理或后处理分析任务非常有用，而无需运行显示界面。由于不提供界面，脚本不能包含任何用户交互。如果您使用`noGUI`选项，Abaqus/CAE将忽略您提供的任何其他命令行选项。

可以通过在命令行上输入`--`，然后用 一个或多个空格分隔参数来将参数传递到文件中。这些参数将被Abaqus/CAE执行程序忽略，但可以在Python脚本中访问。如果您使用`noGUI`选项，您可以使用参数来传递一个否则由命令行选项提供的变量。例如，您可以传递一个否则由`script`选项指定的文件名。

**noenvstartup**

此选项指定在应用程序启动时不应运行环境文件中的所有配置命令。此选项可与**script**命令结合使用，以禁止**script**文件中除那些之外的 所有配置命令。

**noSavedOptions**

此选项指定Abaqus/CAE不应应用存储在`abaqus_v6.14.gpr`中的显示选项设置（例如，渲染样式和基准平面的显示）。有关更多信息，请参阅["保存您的显示选项设置，" Abaqus/CAE用户指南第76.16节](../usi/usi-link.md#uss-dsp-preferences)。

**noSavedGuiPrefs**

此选项指定Abaqus/CAE不应应用存储在`abaqus_v6.14.gpr`中的GUI设置（例如，Abaqus/CAE主窗口或其对话框的大小和位置）。

**noStartupDialog**

此选项指定不应显示Abaqus/CAE的**Start Session**对话框。

**custom**

此选项指定包含Abaqus GUI工具包命令的文件的名称。此选项执行Abaqus/CAE的定制版本应用程序。有关更多信息，请参阅[Abaqus GUI工具包用户指南第1章"简介"](../cus/cus-link.md#cus-int-int)。

**guiTester**

此选项启动一个包含Abaqus Python开发环境以及Abaqus/CAE的单独用户界面。Abaqus Python开发环境允许您创建、编辑、单步执行和调试Python脚本。有关更多信息，请参阅[Abaqus脚本用户指南第三部分"Abaqus Python开发环境"](../cmd/cmd-link.md#cmd-pde)。

您可以将脚本指定为此选项的参数，这会提示Abaqus/CAE运行GUI脚本。当到达脚本末尾时，Abaqus/CAE关闭。

**guiRecord**

此选项使您能够在名为`abaqus.guiLog`的文件中记录Abaqus/CAE用户界面中的操作。您也可以通过使用环境变量`ABQ_CAE_GUIRECORD`在启动时设置此选项。**guiRecord**选项不能与**guiTester**选项一起使用。

**guiNoRecord**

此选项使您能够在设置环境变量`ABQ_CAE_GUIRECORD`时禁用用户界面记录。

### 示例

以下示例说明了**cae**执行程序的命令行选项以及如何将参数传递给Abaqus/CAE。

#### 打开模型数据库

以下命令将执行Abaqus/CAE并加载名为"beam"的模型数据库文件：

```
abaqus cae database=beam
```

#### 向脚本传递参数

以下命令将在应用程序启动时运行名为"try.py"的Python脚本，并将"argument1"传递给脚本：

```
abaqus cae script=try.py -- argument1
```

如果"try.py"定义为
```
import sys
print sys.argv[-1]
```
上述命令将打印`argument1`。

#### 在不显示图形用户界面的情况下运行Abaqus/CAE

以下命令将运行名为"checkPartValidity.py"的Python脚本，并将参数传递给脚本，指定模型数据库、模型和部件。该脚本由Abaqus/CAE执行；但是，图形用户界面永远不会显示。

```
abaqus cae noGui=checkPartValidity.py -- test.cae Model-1 Part-1
```

如果"checkPartValidity.py"定义为
```
import sys
import os

myMdb= sys.argv[-3]
myModel = sys.argv[-2]
myPart = sys.argv[-1]

mdb = openMdb(myMdb)
model = mdb.models[myModel]
part = model.parts[myPart]

if part.geometryValidity:
    sys.__stderr__.write('%s is valid\n' % myPart)
else:
    sys.__stderr__.write('%s is invalid\n' % myPart)
```
上述命令将打印`Part-1 is valid`。
