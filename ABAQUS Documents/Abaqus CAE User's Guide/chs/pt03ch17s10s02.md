# 17.10.2 使用四边形和四边形主导单元的自由网格





使用四边形单元的自由网格是二维区域的默认网格技术。四边形单元的自由网格技术可应用于任何平面或曲面。使用此技术生成的网格示例如图17-87所示。自由网格通常不是对称的，即使零件或零件实例本身是对称的。

**图17-87** 使用四边形单元生成的自由网格。

![](../graphics/mgn-fasten.png)

当您创建四边形或四边形主导网格时，Abaqus/CAE允许您在两种网格算法之间进行选择。更多信息，请参见["中轴算法和前进波算法有什么区别？"第17.7.6节](pt03ch17s07s06.md)。

**中轴**

当您使用中轴算法用四边形单元对复杂区域进行自由网格划分时，Abaqus/CAE创建将区域分割成简单结构化网格区域的内部分割，然后对较小的区域进行布种。

如果您使用中轴算法对区域进行网格划分，然后重新对该区域进行网格划分（例如，在修改种子之后），Abaqus/CAE会存储内部分割，新网格的生成速度会更快。此外，内部分割允许Abaqus/CAE以与生成粗网格大致相同的时间生成细网格。您不能使用中轴算法对包含虚拟拓扑的区域进行网格划分，并且它与不精确的零件配合效果不佳。

一般来说，由于以下原因，使用四边形单元的基于中轴的自由网格不会完全匹配网格种子：
- Abaqus/CAE尝试在相邻区域和内部分割创建的较小区域之间平衡种子。
- Abaqus/CAE尝试最小化单元畸变。

但是，当您应用固定种子约束时，Abaqus/CAE会精确匹配种子数量，并尝试精确匹配种子位置。图17-88展示了一个具有固定种子和可移动种子的二维板，以及生成的全四边形网格。

**图17-88** 固定种子和可移动种子以及中轴网格算法。

![](../graphics/mgn-fixedseeds-medialaxis.png)

您只应在少数边上指定固定种子，否则Abaqus/CAE可能无法生成网格。例如，如果在图17-88所示的板上的孔之一周围指定固定种子，则全局布种变得过度约束，Abaqus/CAE无法生成网格。

使用中轴算法，使用四边形主导单元生成的自由网格类似于使用全四边形单元生成且没有过渡最小化的自由网格；但是，Abaqus/CAE会插入一些孤立的三角形以尝试更紧密地匹配网格种子。Abaqus/CAE可以使用四边形主导单元比使用全四边形单元更快地生成网格。

**前进波**

当您使用前进波算法用四边形单元对复杂区域进行自由网格划分时，Abaqus/CAE在区域边界生成四边形单元，并随着它系统地移动到区域内部继续生成四边形单元。

当您选择前进波算法时，Abaqus/CAE会精确匹配种子（除非您正在创建三维旋转网格，且被旋转的轮廓接触旋转轴）。图17-89展示了精确匹配种子的四边形网格。

**图17-89** 固定种子和可移动种子以及前进波网格算法。

![](../graphics/mgn-fixedseeds-advancingfront.png)

一般来说，使用前进波算法生成的网格过渡比使用中轴算法生成的过渡更容易接受；然而，在窄区域中精确匹配种子可能会损害网格质量。与中轴算法不同，您可以将前进波算法与不精确零件和包含虚拟拓扑的区域结合使用。

如果您选择前进波算法，Abaqus/CAE也会在适当的地方使用映射网格。（映射网格与结构化网格相同，但仅适用于四边形区域。）有关更多信息，请参见["什么是映射网格？"第17.8.2节](pt03ch17s08s02.md)和["Abaqus/CAE何时可以应用映射网格？"第17.8.6节](pt03ch17s08s06.md)。当使用映射网格时，Abaqus/CAE会对网格种子进行微调。如果不希望种子更改，可以使用网格控制来防止使用映射网格。更多信息，请参见["分配网格控制，"第17.18.1节](pt03ch17s18s01.md)。

默认情况下，当Abaqus/CAE使用中轴算法生成自由四边形网格时，它会最小化网格过渡。最小化网格过渡会生成更好的网格，生成速度也更快；但是，生成的节点会进一步偏离网格种子。图17-90展示了使用中轴算法（有无网格过渡最小化）和使用前进波算法进行网格划分的相同平面零件实例。

**图17-90** 网格过渡和网格算法的效果。
