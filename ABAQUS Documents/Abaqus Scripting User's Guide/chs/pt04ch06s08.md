### 6.8.1 自动在kernel对象上注册查询

查询提供了一种机制，允许GUI进程在kernel中的数据发生更改时收到通知。["保持GUI和命令最新，" 第6.5.3节](pt04ch06s05.md#cus-app-commands-gui-uptodate)描述了如何使用`AFXGuiCommand`构造函数的*registerQuery*参数。*registerQuery*参数是一个布尔标志，指定是否在指定kernel命令正在编辑的对象上自动注册查询。如果`AFXGuiCommand`构造函数中指定的kernel对象发生更改，基础设施将使用最新值更新GUI中的关键字。因此，您不需要显式注册查询。默认情况下，*registerQuery*=False，查询不会自动注册。

例如，

```
cmd = AFXGuiCommand(mode,'setValues', mdb.models[%s].parts[%s],
    True)
```

在此示例中，如果用户更改当前部件，指向`setValues`方法的路径将更新以反映新的当前部件。因此，当用户点击"确定"提交自定义对话框时，模式会发出一个修改当前部件的`setValues`命令。

### 6.8.2 手动在kernel对象上注册查询

对于与命令没有直接关系的对象（如repository），您可能希望自己注册查询。您可以使用`registerQuery`方法在Abaqus/CAE对象上注册查询。此方法接受一个回调函数作为参数。当注册查询的对象发生更改时，基础设施会自动调用`registerQuery`方法中提供的函数。例如，

```
from abaqusGui import *

def onPartsChanged():
    print 'The parts repository changed.'
    keys = mdb.models['Model-1'].parts.keys()
    print 'The new keys are:', keys

mdb.models['Model-1'].parts.registerQuery(onPartsChanged)
```

在前面的示例中，如果创建、删除、重命名或编辑了部件，将调用`onPartsChanged`方法。

`registerQuery`方法接受一个可选的第二个参数，该参数决定在首次注册查询时是否调用回调。默认情况下，此参数为True，回调将在首次注册查询时被调用。如果您将False指定为第二个参数，则在首次注册查询时不会调用查询回调。延迟查询回调可以防止某些情况下的错误；有关更多信息，请参阅["在`kernelAccess`代理对象上使用`registerQuery`，" 第6.8.3节](pt04ch06s08.md#cus-com-commands-notifyofkernel-registerissues)。

由于注册的查询会在kernel和GUI进程之间产生"流量"，因此您不需要时应取消注册查询。要取消注册查询，请使用`unregisterQuery`方法，并传入您在`registerQuery`方法中使用的相同参数。在大多数情况下，您在需要查询的对话框的`show`方法中注册查询。类似地，您在对话框的`hide`方法中取消注册查询。如果您不取消注册查询，而查询在对话框未显示时触发，则如果回调尝试修改对话框中的窗口部件，应用程序可能会中止。

如果用户在以下示例中创建、删除、重命名或编辑部件，应用程序将调用`onPartsChanged`方法并更新对话框：

```
class MyDialog(AFXDataDialog):

    ...

    def onPartsChanged(self):

        # 更新对话框中部件列表的代码

    def show(self):

        from kernelAccess import mdb
        mdb.models['Model-1'].parts.registerQuery(
            self.onPartsChanged)
        AFXDataDialog.show(self)

    def hide(self):

        from kernelAccess import mdb
        mdb.models['Model-1'].parts.unregisterQuery(
            self.onPartsChanged)
        AFXDataDialog.hide(self)
```

### 6.8.3 在`kernelAccess`代理对象上使用`registerQuery`

可以在`kernelAccess`模块代理对象上调用`registerQuery`方法，而不是在`abaqusGui`代理对象上。但是，在内部，查询始终在`abaqusGui`代理对象上注册。两种代理对象并非始终完全同步。在大多数情况下，这无关紧要。但是，如果在kernel上进行更改时注册了查询，可能会导致问题。例如，

```
from kernelAccess import mdb

def onPartsChanged():
    print 'The parts repository changed.'
    keys = mdb.models['Model-1'].parts.keys() # OK
    print 'The new keys are:', keys
    if keys:
       mdb.models['Model-1'].parts[keys[0]].registerQuery(onPartsChanged) # Not OK
       # 在内部，registerQuery方法将在abaqusGui.mdb上调用...

mdb.models['Model-1'].parts.registerQuery(onPartsChanged)
```

如果随后发出了影响parts repository中名称的`changeKey`命令，上述示例将失败。键（部件名称）使用`kernelAccess mdb`代理对象获取，包含更改后的名称。但是，`registerQuery`方法（基于`abaqusGui`代理对象）在`changeKey`命令完成之前看不到新名称。在回调中调用新命名的部件对象上的`registerQuery`（使用`keys()[0]`）是在`changeKey`命令完成之前、回调使用新名称时进行的。由于parts repository的`abaqusGui`代理尚未更新，因此会引发`KeyError`。

在GUI中运行上述示例，创建一个部件，然后在命令行界面（CLI）中输入以下内容：

```
>>> mdb.models['Model-1'].parts.changeKey('Part-1', 'ROD')
```

您将收到以下错误：

```
Traceback (most recent call last):
 File "*path and filename of the example script*",
line 9, in onPartsChanged
    mdb.models['Model-1].parts[partNames[0]].registerQuery
(onPartsChanged) # Not OK
KeyError: 'ROD'
```

该错误可能导致Abaqus/CAE停止响应。将`registerQuery`的第二个参数设置为False可防止回调立即被调用，并防止此潜在错误。

### 6.8.4 识别自定义kernel数据何时更改

要在GUI中接收对自定义kernel对象所做更改的通知，这些kernel对象必须使用`customKernel`模块提供的特殊类。`customKernel`模块提供以下特殊类，它们都能够通知GUI类内容何时发生更改：

- `CommandRegister`允许您创建通用类。有关更多信息，请参阅["CommandRegister类，" Abaqus Scripting User's Guide的第5.6.3节](../cmd/cmd-link.md#cmd-use-extendingacl-commandregister)。
- `RepositorySupport`允许您在其他repository下创建repository。有关更多信息，请参阅["RepositorySupport，" Abaqus Scripting User's Guide的第5.6.6节](../cmd/cmd-link.md#cmd-use-extendingacl-abaqus-repository)。
- `RegisteredDictionary`允许您创建自定义字典。有关更多信息，请参阅["注册的字典，" Abaqus Scripting User's Guide的第5.6.7节](../cmd/cmd-link.md#cmd-use-extendingacl-dictionaries)。
- `RegisteredList`允许您创建自定义列表。有关更多信息，请参阅["注册的列表，" Abaqus Scripting User's Guide的第5.6.8节](../cmd/cmd-link.md#cmd-use-extendingacl-lists)。
- `RegisteredTuple`允许您创建自定义元组。有关更多信息，请参阅["注册的元组，" Abaqus Scripting User's Guide的第5.6.9节](../cmd/cmd-link.md#cmd-use-extendingacl-tuples)。

有关`customKernel`模块的更多信息，请参阅["扩展Abaqus Scripting Interface，" Abaqus Scripting User's Guide的第5.6节](../cmd/cmd-link.md#cmd-use-extendingacl)。
