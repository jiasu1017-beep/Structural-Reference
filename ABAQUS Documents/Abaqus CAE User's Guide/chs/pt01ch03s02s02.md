# 3.2.2 输入表达式





如果对话框或提示区域中的字段期望浮点数或复数，您可以输入算术表达式，如图3-1所示。

**图3-1** 文本字段中的表达式。

![](../graphics/int-expression-nls.png)

表达式由内置于Abaqus/CAE中的Python解释器计算。算术表达式会被其值替换；如果您重新打开包含表达式的对话框，则只能看到值。变量如`pi`和函数如`sin()`是可用的，因为Abaqus/CAE在您启动会话时导入了Python数学模块。因此，您可以输入任何可以由Python内置函数或Python数学模块计算的表达式。有关更多信息，请参阅Python官方主页上的[内置函数](http://www.python.org/doc/current/lib/built-in-funcs.md)和[数学模块](http://www.python.org/doc/current/lib/module-math.md)文档。

为确保您的表达式按预期计算，您应注意以下几点：
- 如果将数字作为整数输入，Python将执行整数除法并向下舍入任何余数。例如，Python将`3/2`解释为`1`，将`1/2`解释为`0`。相反，Python将`3./2`解释为`1.5`，将`1/2.`解释为`0.5`。
- Python将带前导零的数字解释为八进制数（例如，`0123`被解释为`83.0`）。但是，Abaqus/CAE会在Python解释之前忽略文本字段中数字的前导零；这些数字被计算为小数。
- Python将`e`解释为自然对数的底；即`e`等于`2.71828182846`，`e+2`等于`4.71828182846`。
- 如果"e"字符前面有数字，Python会将其解释为指数，而不是自然对数。例如，Python将`2e+2`解释为2×10²，等于`200`。
- Python将`2e+`解释为2×10⁰，等于`2`。类似地，Python将`2e++11`解释为2×10⁰+11，等于`13`。

如果您不确定Python如何解释您的表达式，可以在命令行输入表达式；Abaqus/CAE会在消息区域打印结果解释值。要访问命令行界面，请单击主窗口左下角的![](../graphics/afxI_commandLine.png)。有关更多信息，请参阅["主窗口的组成，" 第2.2.1节"](pt01ch02s02hlb01.md)。

您还可以通过在操作系统提示符下输入`abaqus python`并在出现的Python提示符下输入表达式来测试Abaqus/CAE如何解释表达式。提示行和一些对话框不允许您输入表达式。作为替代，您可以在命令行或Python提示符下输入表达式，然后将结果值粘贴到提示行或对话框中。
