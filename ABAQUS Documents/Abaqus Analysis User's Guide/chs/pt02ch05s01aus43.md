# 5.1.3 访问结果文件信息





**产品：**Abaqus/Standard  Abaqus/Explicit

##### **参考**

- [『访问结果文件：概述』，第 5.1.1 节](pt02ch05s01abo03.md)
- [『结果文件输出格式』，第 5.1.2 节](pt02ch05s01afi01.md)
- [『访问结果文件的实用程序』，第 5.1.4 节](pt02ch05s01abu01.md)

### 概述

Abaqus 结果（`.fil`）文件使用内部数据管理例程写入，以尽量减少 I/O 成本。后处理程序必须使用这些相同的 Abaqus 数据管理例程来读取结果文件。必须调用以下实用程序来从 Abaqus 结果文件获取数据：
- `INITPF`
- `DBRNU`
- `DBFILE`
- `POSFIL`

您还可以通过使用以下实用子程序以 Abaqus 结果文件的格式写入文件：
- `INITPF`
- `DBFILW`

这些实用子程序的语法在『访问结果文件的实用程序』，第 5.1.4 节中描述。

### 读取浮点和整数变量

要读取记录中的浮点和整数变量，后处理程序中可以使用以下编码：

```
INCLUDE 'aba_param.inc'
      DIMENSION ARRAY(513), JRRAY(NPRECD,513)
      EQUIVALENCE (ARRAY(1),JRRAY(1,1))
```
使用此技术，例如，在每次使用 `LOP`=0 调用 `DBFILE` 之后，记录键可用作
```
KEY =  JRRAY (1,2)
```
使用 `aba_param.inc` 消除了需要不同版本的代码来处理单精度和双精度的麻烦。文件 `aba_param.inc` 定义了适当的 `IMPLICIT REAL` 语句，并根据机器使用单精度还是双精度将 `NPRECD` 的值设置为 1 或 2。当使用 **abaqus make** 实用程序编译和链接后处理程序时，会从 Abaqus 安装的 `site` 子目录中引用文件 `aba_param.inc`（见下文）。

### 链接后处理程序

后处理程序必须在运行 Abaqus 执行程序时使用 **make** 参数进行链接（请参阅『制作用户定义的可执行文件和子程序』，第 3.2.17 节）。为了正确链接，后处理程序不能包含 FORTRAN PROGRAM 语句。相反，程序必须以名为 ABQMAIN 的 FORTRAN SUBROUTINE 开头。

编译、链接和运行后处理程序包括两个步骤。例如，如果后处理程序的名称是 `postproc.f`，请使用以下命令来编译和链接 `postproc.f`：

```
`abaqus make job=postproc`
```
然后必须使用以下命令运行程序：
```
`abaqus postproc`
```

### 调用实用子程序来读取结果文件

在访问任何结果文件之前，必须调用子程序 `INITPF`。此子程序包含通过调用 `INITPF` 分配给结果文件的所有 FORTRAN OPEN 语句；因此，您的代码不能包含这些单元的任何 OPEN 语句。Abaqus 根据提供给 `LRUNIT(1,K1)` 和 `FNAME` 的信息为给定单元构造文件名，如『访问结果文件的实用程序』，第 5.1.4 节中所讨论的。

在读取第一个结果文件之前还必须调用子程序 `DBRNU`，然后在需要切换到读取另一个结果文件时再次调用。此子程序只是建立正在读取的结果文件的 FORTRAN 单元号；不返回信息。`DBRNU` 可以在 `INITPF` 之前或之后调用，但必须在 `DBFILE` 之前调用。

子程序 `DBFILE` 用于从结果文件读取每条记录。此子程序将一次返回一条记录，格式如『结果文件输出格式』，第 5.1.2 节中所述。

#### 示例

以下程序读取结果文件中的所有 von Mises 应力，并获得最大值。然后，它打印此值以及发生该值的单元、截面点和积分点编号。

在此示例中，FORTRAN 单元 8 用于读取结果文件，結果文件的名称假定为 `TEST.fil`。结果文件假定为二进制文件，并且只读取一个结果文件。因此，`LRUNIT` 被维化为 `LRUNIT(2,1)`；在调用 `INITPF` 例程时，`NRU` 设置为 1，`LRUNIT(1,1)` 设置为 8，`LRUNIT(2,1)` 设置为 2。将不会写入新的结果文件，因此 `LOUTF` 设置为零。

```
      SUBROUTINE ABQMAIN
C     计算最大 von Mises 应力及其位置
C
      INCLUDE 'aba_param.inc'
      CHARACTER*80 FNAME
      DIMENSION ARRAY(513),JRRAY(NPRECD,513),LRUNIT(2,1)
      EQUIVALENCE (ARRAY(1),JRRAY(1,1))
C
C     文件初始化
C
      FNAME='TEST'
      NRU=1
      LRUNIT(1,1)=8
      LRUNIT(2,1)=2
      LOUTF=0
      CALL INITPF(FNAME,NRU,LRUNIT,LOUTF)
      JUNIT=8
      CALL DBRNU(JUNIT)
C
C     循环读取结果文件中的所有记录
C
      STRESS=0.
      DO 100 K1=1,99999
C
        CALL DBFILE(0,ARRAY,JRCD)
        IF(JRCD.NE.0)GO TO 110
        KEY=JRRAY(1,2)
C
        IF(KEY.EQ.1) THEN
C
C          单元标题记录：
C          提取单元、截面点、积分点编号
C
           JEL=JRRAY(1,3)
           JPNT=JRRAY(1,4)
           JSPNT=JRRAY(1,5)
C
C     Abaqus/Standard 的应力不变量记录
        ELSE IF(KEY.EQ.12)THEN
C     Abaqus/Explicit 的应力不变量记录
        ELSE IF(KEY.EQ.75)THEN
C
C          提取 von Mises 应力
C
           IF(ARRAY(3).GT.STRESS)THEN
              STRESS=ARRAY(3)
              KEL=JEL
              KPNT=JPNT
              KSPNT=JSPNT
           END IF
        END IF
C
 100  CONTINUE
 110  CONTINUE
C
      WRITE(6,120) KEL,KPNT,KSPNT,STRESS
 120  FORMAT(5X,'ELEMENT',I5,5X,'POINT',I4,5X,'SECTION POINT',
     1 I4,5X,'STRESS',1PG12.3)
      STOP
      END
```
有关其他示例，请参阅 Abaqus 例题指南第 15 章『Abaqus 结果的后处理』。

### 以结果文件格式写入文件

子程序 `DBFILW` 可以用于以 Abaqus 结果文件的格式写入文件，以修改文件信息或在后处理之前添加其他信息。必须在 `DBFILW` 之前调用子程序 `INITPF`。

文件将写入 FORTRAN 单元 9，扩展名为 `.fin`。当首次调用 `DBFILW` 时，Abaqus 会打开单元 9；您的编码不能打开或重新定义单元 9，但必须确保在作业之后保存 FORTRAN 单元 9。

『合并来自多个结果文件的数据并转换文件格式：FJOIN』，Abaqus 例题指南第 15.1.2 节，包含了一个使用子程序 `DBFILW` 合并不连续结果文件的特定记录的示例。后处理需要连续的结果文件；如果您在分析过程中编写了结果文件，而在重新开始分析时编写了新的结果文件，而没有使文件保持连续，则必须在后处理之前使文件保持连续。『地震运动下的悬臂梁分析』，Abaqus 基准指南第 1.4.13 节，也展示了使用 `DBFILW` 合并结果文件。或者，可以使用 **abaqus append** 实用程序合并结果文件，如『连接结果（`.fil`）文件』，第 3.2.14 节中所述。

`DBFILW` 子程序也可用于将 Abaqus 结果文件从二进制格式转换为 ASCII 格式，以将其从一台计算机系统传输到另一台计算机系统。或者，可以使用 **abaqus ascfil** 执行程序自动进行此转换，如『结果（`.fil`）文件的 ASCII 翻译』，第 3.2.13 节中所述。
