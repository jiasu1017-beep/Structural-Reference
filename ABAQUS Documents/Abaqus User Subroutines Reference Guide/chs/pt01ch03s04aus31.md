# 3.4.1 在Abaqus中管理内存和磁盘使用





**产品：** Abaqus/Standard  Abaqus/Explicit  

##### **参考**

- ["Abaqus执行程序：概述，" 第3.1.1节](pt01ch03s01abo02.md)
- ["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)

### 概述

对于小型分析，计算机资源的管理通常是次要问题，但对于大型模型，智能使用磁盘和内存资源是分析过程的关键部分。对于中大型分析，您会发现有必要修改资源管理设置。

### 理解资源使用

对于Abaqus，磁盘和内存实际上是两种相似的存储数据方式。分析完成后需要的数据必须最终写入磁盘；但在分析期间，磁盘和内存提供功能等效的存储机制。通常磁盘是更丰富的资源，而内存提供更快的存储数据访问。Abaqus资源管理的关键在于这个简单的权衡。

#### Abaqus数据

Abaqus分析生成的数据实际上有两种类型。第一种是"输出"数据，这些数据必须在分析完成后保留。输出数据通常是您需要用于后处理的结果，或者是重新启动分析所需的数据。如上所述，这些数据必须在分析完成之前存储在磁盘上。

此外，分析会产生大量"临时"或临时数据。这些数据仅在分析运行期间需要。临时数据可分为两组：性能关键数据和通用数据。性能关键数据始终存储在内存中，而通用数据可以存储在内存或磁盘上。

#### 需求和注意事项

要运行分析，必须满足以下要求：
- 必须有足够的磁盘空间来保存请求的输出数据。
- 必须有足够的内存来保存所有性能关键数据。
- 必须有足够的磁盘空间或内存资源来保存所有通用临时数据。

如果满足上述要求，则可以完成分析；但是，对于Abaqus/Standard，您会发现让Abaqus使用额外的内存通常会提高性能。随着计算机集群、专用共享内存计算机的普及，以及最重要的为分析分配处理器和内存的作业排队系统的出现，使用所有内存资源来提高性能是最合理的。

通常Abaqus/Standard在分析阶段会分配机器上可用系统内存的很大一部分，但您可以使用**memory**参数手动指定内存使用限制（请参阅下面的["资源管理参数](pt01ch03s04aus31.md#usb-int-dmemory-params)"）。在Abaqus/Explicit分析阶段，不会将临时数据写入磁盘，因为大多数临时数据都是性能关键的。

### 资源管理参数

Abaqus资源管理参数分为两类：内存管理和磁盘管理。每个都可以通过一个环境文件参数进行调整。以下各节说明如何最好地利用此参数。有关环境文件的更多信息，请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)。

#### 内存管理参数

**memory**参数用于限制在Abaqus/Standard分析阶段以及在Abaqus/Standard和Abaqus/Explicit分析之前执行的输入文件处理阶段可以使用的内存量。

如果您未定义**memory**参数，Abaqus会自动检测机器上的物理内存，并分配此可用内存的百分比。默认百分比是特定于平台的，但它们通常代表可用物理内存的很大一部分。有关默认内存分配设置的详细信息，请参阅Dassault Systmes知识库：[www.3ds.com/support/knowledge-base](http://www.3ds.com/support/knowledge-base)。

您可以通过指定物理内存的百分比或以兆字节或千兆字节为单位的绝对限制来覆盖默认内存分配。百分比通过在指定限制后跟"%"符号来表示。兆字节和千兆字节的单位通过在指定限制后跟"mb"或"gb"来表示。如果未指定单位，则假定为兆字节。例如，使用以下任何设置：

```
`memory`=`"2048 mb"`
`memory`=`"2 gb"`
`memory`=`"25 %"`
```

在具有8千兆字节物理内存的机器上，Abaqus使用最多2千兆字节的内存。内存设置值必须用引号括起来。**memory**指定的值必须对于所使用的机器是合理的。Abaqus/Standard不检查数值有效性。为了与操作系统内存测量工具保持一致，Abaqus将兆字节定义为1,048,576字节，而不是1,000,000字节。千兆字节的单位适用类似规则。

Abaqus/Explicit分析阶段没有内存管理参数，因为在此阶段不会将临时数据写入磁盘。

可以为主机、用户或特定作业设置环境文件参数（请参阅["使用Abaqus环境设置，" 第3.3.1节](pt01ch03s03aus30.md)进行进一步讨论）。因为一台具有大量内存的机器上运行良好的默认内存设置对于另一台内存较少的机器来说可能不是理想的，因此按机器改变默认内存设置可能很有用。

#### 磁盘管理参数

输出数据的管理在["输出，" 第4.1.1节](pt02ch04s01aus38.md)中详细讨论。输出数据写入启动作业的目录中的文件。

Abaqus/Standard临时文件写入单独的临时目录。您可以使用**scratch**环境文件参数控制用于保存临时文件的目录。由于在整个分析阶段对临时数据的频繁访问，确保临时磁盘的高I/O速度对分析性能至关重要。

如上所述，Abaqus/Explicit不会将临时数据写入磁盘，因此您只需关注输出数据的适当管理。

### 输入文件处理和**数据检查**

通常，输入文件处理期间所需的内存量不大。作业分析阶段所需的内存和磁盘空间更值得关注。Abaqus无法估计完成输入文件处理所需的内存量。可以通过使用运行Abaqus的命令中的**datacheck**参数（请参阅["Abaqus/Standard、Abaqus/Explicit和Abaqus/CFD执行，" 第3.2.2节](pt01ch03s02abx02.md)）执行数据检查运行，以获得完成分析阶段所需内存的估计值。以下给出了执行数据检查（其中包括输入文件处理阶段）的**memory**参数设置的一些常规指南。

#### 内存设置指南

通常，您不必更改默认内存设置。如果作业由于默认设置下内存不足而失败，则需要使用具有更多内存的机器来运行作业。如果您需要通过为**memory**环境文件参数指定一个值来覆盖默认行为，[表3.4.1-1](pt01ch03s04aus31.md#usb-int-dmemorytable)列出了一些针对各种规模问题的典型数据检查内存设置。**memory**的实际所需值可能因问题的特征而有很大差异。

**表3.4.1-1** 执行数据检查分析时的典型**memory**设置。
| 自由度 | **内存** |
| --- | --- |
| 250,000 | 250兆字节 |
| 100万 | 750兆字节 |
| 250万 | 1200兆字节 |
| 500万 | 2000兆字节 |

### Abaqus/Standard分析

根据执行环境和机器上运行的典型作业大小，**memory**可以按机器或按作业设置。更详细的指南在以下部分中提供。当需要按作业设置**memory**时，建议您运行数据检查分析并根据内存估计值设置**memory**。这些估计值以表的形式写入打印输出（`.dat`）文件中，标题为"MEMORY ESTIMATE"。此表中有两列与内存使用相关。第一相关列标记为"MINIMUM MEMORY REQUIRED"，指定将关键临时数据保存在内存中所需的**memory**设置。如果**memory**设置低于此值，则尝试运行分析将产生警告，并且由于内存不足，作业可能无法运行完成。第二相关列标记为"MEMORY TO MINIMIZE I/O"，指定将所有临时数据（关键数据和通用数据）保存在内存中所需的**memory**。如果**memory**指定的内存大于"MINIMUM MEMORY REQUIRED"，则Abaqus/Standard自动使用额外内存（直到内存限制）来提高否则将写入磁盘的通用临时数据的访问速度。当**memory**不足以将所有通用临时数据保存在内存中时，Abaqus/Standard决定哪些数据应该写入磁盘，哪些应该保存在内存中，这是基于它们对分析性能的相对重要性。因此，临时数据使用的实际磁盘空间可能从接近零到"MEMORY TO MINIMIZE I/O"不等，具体取决于**memory**设置。可以在不重新运行分析输入文件处理器的情况下，在从数据检查继续的分析中更改**memory**设置。

#### 内存设置指南

**memory**参数允许您指定Abaqus在输入文件处理和分析阶段可以使用的内存限制。您可以指定在特定机器上通常应该可供Abaqus使用的设置。设置可以在主机环境文件中的特定机器上修改。设置可以根据需要在作业特定环境文件中的单个作业中进行修改。特定机器的合理设置取决于运行问题的规模和除了机器上可用物理内存之外机器的使用方式。您应该知道物理内存和虚拟内存之间的区别。当使用虚拟内存时，机器的操作系统只是使用磁盘作为额外内存。虽然这可能有用，但内存访问可能需要I/O操作，从而带来相当大的性能惩罚。因此，以下关于管理Abaqus/Standard内存的指南始终相对于机器上的物理内存给出。虚拟内存只应在必要时使用，并且要意识到相关的性能惩罚。

##### 在单用户机器上设置**memory**

对于专用于运行Abaqus/Standard的单用户机器，使用**memory**的默认设置是明智的。如果估计表明作业需要的内存超过此值，则作业太大，无法在此机器上高效运行。此时强烈建议将分析移到具有更多内存资源的另一台机器上。

对于同时用于运行Abaqus/Standard和其他应用程序的单用户机器，设置较低的**memory**限制是有意义的。如果分析需要的内存超过指定值，您可以决定增加**memory**并继续作业。但是，Abaqus/Standard将不得不与其他应用程序争夺内存，这将损害Abaqus/Standard和其他应用程序的效率。如果其他应用程序是交互式的，性能下降可能会出现问题。在这种情况下，您可能决定推迟继续分析，直到机器可以专用于运行Abaqus/Standard。

##### 在多用户机器上设置**memory**

在多用户机器上设置**memory**的指南与单用户机器非常相似，只是必须判断机器上每个用户可以期望为单个分析使用的内存量。一个合理的方法可能是将机器的物理内存除以预期的同时作业数。另一个合理的方法是将机器的物理内存除以CPU总数，然后乘以当前作业使用的CPU数。如果同时作业之间的内存需求不均匀，您可能希望以不均匀的方式分配机器的物理内存。一般来说，为了确保可接受的性能，多用户机器上的用户需要相互协调以正确设置内存限制。

##### 使用队列时设置**memory**

队列通常有相关的内存限制，选择作业的适当队列需要一些判断。建议您运行数据检查分析，并根据打印输出文件中提供的估计选择队列。然而，对于大型分析，即使是数据检查分析也可能需要大量内存。为数据检查分析选择适当队列需要一些特定类别问题的经验。您可能希望最初将数据检查运行提交到具有非常大内存限制的队列，以获得必要的估计值。然后可以选择适当的队列来实际运行作业。如果作业要提交到共享内存机器，将**memory**设置为队列内存限制的约90%是合理的。如果作业要提交到计算机集群，使用默认内存设置是合理的。
