# 35.5.1 Element end release







**Product: **Abaqus/Standard  

##### **References**

- ["Kinematic constraints: overview," Section 35.1.1](pt08ch35s01abo32.md)
- [*RELEASE](../key/key-link.md#usb-kws-mrelease)

### Overview

Element end release:
- allows a rotational degree of freedom or a combination of rotational degrees of freedom to be released at one or both ends of an element or element set;
- can be used in geometrically linear or nonlinear analysis; and
- is available only for beam and pipe elements in Abaqus/Standard.

### Introduction

Element end release is used to model hinged connections (hinged in one, two, or three orthogonal directions) at one or both ends of the element. By releasing rotational degrees of freedom, an element end is allowed to rotate freely relative to the node about the chosen degrees of freedom. Any rotational degrees of freedom that are not released are shared with the node. You must be careful not to release a given degree of freedom at a node for all elements that share that node; otherwise, the node has no stiffness for that degree of freedom and Abaqus/Standard issues zero pivot warning messages.

Element end release operates on the element local degrees of freedom. See ["Beam element cross-section orientation," Section 29.3.4](pt06ch29s03alm09.md), for a definition of the local axes (![](../graphics/usb_eqn06151.gif), ![](../graphics/usb_eqn06152.gif), *t*) for beam-type elements. The rotational degrees of freedom affected by the release are the rotation about the local ![](../graphics/usb_eqn06151.gif)-axis, the rotation about the local ![](../graphics/usb_eqn06152.gif)-axis, and the rotation about the local *t*-axis for beams in space. For beams in a plane, only the rotation about the local ![](../graphics/usb_eqn06151.gif)-axis is active (which coincides with rotations about the negative global *z*-axis).

#### Equivalent MPCs

If only one rotational degree of freedom is released, the kinematic constraint is equivalent to MPC type REVOLUTE plus MPC type PIN between two nodes. If two rotational degrees of freedom are released, the kinematic constraint is equivalent to MPC type UNIVERSAL plus MPC type PIN. If all rotational degrees of freedom are released, the kinematic constraint is equivalent to MPC type PIN. See ["General multi-point constraints," Section 35.2.2](pt08ch35s02aus130.md), for details.

### Identifying the element end involved in the release

Either element sets or individual elements can be specified for a release definition. Degrees of freedom can be released at the first, second, or first and second ends of an element. The first end of the element, S1, is node 1 on the element as defined by the element connectivity; the second end, S2, is the last node (node 2 or 3, as appropriate) on the element. See ["Beam element library," Section 29.3.8](pt06ch29s03ael14.md), for a definition of the node ordering for beam elements.

### Identifying the local rotational degrees of freedom involved in the release

Rotation combination codes rather than degrees of freedom are specified to identify the rotational degrees of freedom involved in the release. 

| M1 | refers to the rotation about the ![](../graphics/usb_eqn06151.gif)-axis, |
| --- | --- |
| M2 | refers to the rotation about the ![](../graphics/usb_eqn06152.gif)-axis, |
| M1-M2 | refers to a combination of rotational degrees of freedom about the ![](../graphics/usb_eqn06151.gif)-axis and the ![](../graphics/usb_eqn06152.gif)-axis, |
| T | refers to the rotation about the *t*-axis, |
| M1-T | refers to a combination of rotational degrees of freedom about the ![](../graphics/usb_eqn06151.gif)-axis and the *t*-axis, |
| M2-T | refers to a combination of rotational degrees of freedom about the ![](../graphics/usb_eqn06152.gif)-axis and the *t*-axis, and |
| ALLM | represents a combination of all the rotational degrees of freedom (i.e., M1, M2, and T). |

| **Input File Usage: ** | ``` [*RELEASE](../key/key-link.md#usb-kws-mrelease) *element number or element set, element end ID, release combination code* ``` |
| --- | --- |
|  | For example, to release the rotational degree of freedom about the ![](../graphics/usb_eqn06151.gif)-axis at the first end of element 10 and all the rotational degrees of freedom at the second end of the element, use the following input: ``` [*RELEASE](../key/key-link.md#usb-kws-mrelease) 10, S1, M1 10, S2, ALLM ``` |

### Use with transformed coordinate systems

Transformations applied to released nodes (["Transformed coordinate systems," Section 2.1.5](pt01ch02s01aus09.md)) have no influence on the release. The release operates on the local degrees of freedom for the element.

### Reading the data from an alternate input file

The data for a release definition can be contained in a separate input file.

| **Input File Usage: ** | ``` [*RELEASE](../key/key-link.md#usb-kws-mrelease), INPUT=*file_name* ``` |
| --- | --- |
|  | If the INPUT parameter is omitted, it is assumed that the data lines follow the keyword line. |
