# *RESTART







### *RESTART保存和重用数据及分析结果。

**警告：**此选项可能创建大量数据。

此选项用于控制重新启动数据的写入和读取。

**产品：**Abaqus/Standard  Abaqus/Explicit  Abaqus/CFD  Abaqus/CAE  

**类型：**模型或历史数据  

**级别：**模型，步骤  

**Abaqus/CAE：**用于保存重新启动数据的步骤模块；用于执行重新启动分析的作业模块

##### **参考：**

- ["重新启动分析，" Abaqus Analysis User's Guide第9.1.1节](../usb/usb-link.md#usb-anl-arestart)

### 在Abaqus/Standard分析中控制重新启动数据的写入和读取

### **至少需要以下参数之一：**

READ

包含此参数以指定此分析是对先前分析的重新启动。基本模型定义数据（单元、材料、节点）在此类重新启动中无法更改。但是，可以添加单元集、节点集和[*AMPLITUDE](ch01abk09.md)表，并且可以更改已分析历史部分之后的历史数据。

WRITE

包含此参数以指定在分析期间写入重新启动数据。

### **如果使用READ参数，则为可选参数：**

CYCLE

如果新分析从先前的低循环疲劳分析重新启动，则将此参数设置为指定步骤中低循环疲劳分析的循环号，分析将在该循环号之后恢复。由于新分析只能从先前低循环疲劳分析中加载周期的结束重新启动，因此INC参数不相关，如果指定了CYCLE参数则会被忽略。

如果省略此参数，重新启动将从指定步骤的最后一个周期开始。

END STEP

此参数指定用户希望终止正在从中进行重新启动的分析的当前步骤。

当用户希望重新定义加载历史、输出选项或容差控制等时，此参数很有用。如果包含此参数，则数据必须包含进一步的步骤定义，以定义分析将如何继续。

如果省略此参数，Abaqus/Standard将继续分析以完成当前步骤，如同在正在从中进行重新启动的运行中所定义的那样。

INC

将此参数设置为步骤中增量号，分析将在该增量号之后恢复。

如果省略此参数，重新启动将从STEP参数指定的步骤结束开始。

ITERATION

如果新分析从先前的直接循环分析重新启动，则将此参数设置为直接循环分析中指定步骤的迭代号，分析将在该迭代号之后恢复。由于新分析只能从先前直接循环分析中加载周期的结束重新启动，因此INC参数不相关，如果指定了ITERATION参数则会被忽略。

如果省略此参数，重新启动将从指定步骤的最后一次迭代结束开始。

STEP

将此参数设置为要进行重新启动的步骤号。

如果省略此参数，分析将从找到的最后一个可用步骤重新启动。

### **如果使用WRITE参数，则为可选参数：**

FREQUENCY

此参数指定写入重新启动信息的增量。例如，FREQUENCY=2将在增量2、4、6等处写入重新启动信息。

对于直接循环分析，此参数指定写入重新启动信息的迭代号；重新启动信息仅在迭代结束时写入。

对于低循环疲劳分析，此参数指定写入重新启动信息的循环号；重新启动信息仅在循环结束时写入。

当在分析中请求重新启动时，重新启动信息总是在步骤结束时写入。设置FREQUENCY=0以停止写入重新启动数据。

FREQUENCY和NUMBER INTERVAL参数是互斥的。默认为FREQUENCY=1。

NUMBER INTERVAL

将此参数设置为步骤中间隔的数量，在这些间隔处写入[*RESTART](ch17abk16.md)数据。此参数的值必须为正整数。

在执行协同仿真时，指定NUMBER INTERVAL参数是同步重新启动信息的推荐方法。在这种情况下，Abaqus/Standard将在由NUMBER INTERVAL参数决定的时间之后立即在协同仿真目标时间写入重新启动信息。

FREQUENCY和NUMBER INTERVAL参数是互斥的。默认为FREQUENCY=1。

TIME MARKS

设置TIME MARKS=YES（默认）以在NUMBER INTERVAL参数决定的确切时间写入结果。

设置TIME MARKS=NO以在NUMBER INTERVAL参数决定的时间之后立即结束的增量处写入重新启动。

当在协同仿真中与NUMBER INTERVAL参数一起使用时，TIME MARKS参数始终设置为NO，以在NUMBER INTERVAL参数决定的时间之后在协同仿真目标时间写入重新启动信息。

OVERLAY

此参数指定每个步骤仅保留一个增量，从而最小化所需的存储空间。

当包含OVERLAY参数时，每个写入的增量都会覆盖同一步骤的先前增量（如果有）。如果省略此参数，则为每个增量保留数据。在任何情况下，每个步骤的最后一个增量都会被保留。

**此选项没有关联的数据行。**

### 在Abaqus/Explicit分析中控制重新启动数据的写入和读取

### **至少需要以下参数之一：**

READ

包含此参数以指定此分析是对先前分析的重新启动。基本模型定义数据（单元、材料、节点）在此类重新启动中无法更改。但是，可以添加单元集、节点集和[*AMPLITUDE](ch01abk09.md)表，并且可以更改已分析历史部分之后的历史数据。

WRITE

包含此参数以指定在分析期间写入重新启动数据。

### **如果使用READ参数，则为必需参数：**

STEP

将此参数设置为要进行重新启动的步骤号。如果省略END STEP和INTERVAL参数，分析将在此步骤中最后一个可用间隔之后恢复。

### **如果使用READ参数，则为可选参数：**

END STEP

包含此参数以指定应在此时终止当前步骤。当使用此参数时，需要INTERVAL参数。

当用户希望重新定义加载历史、接触表面定义等时，此参数很有用。

如果省略此参数，Abaqus/Explicit将从最后一个可用的重新启动间隔继续分析，以完成当前步骤，如同在正在从中进行重新启动的运行中所定义的那样。

INTERVAL

将此参数设置为步骤中间隔号，分析将在该间隔号之后恢复。当使用此参数时需要END STEP参数。

### **如果使用WRITE参数，则为可选参数：**

NUMBER INTERVAL

将此参数设置为步骤中间隔的数量，在这些间隔处写入[*RESTART](ch17abk16.md)数据。此参数的值必须为正整数。默认为NUMBER INTERVAL=1。

Abaqus/Explicit总是在步骤的开始和结束时写入重新启动数据。例如，如果NUMBER INTERVAL=10，Abaqus/Explicit将写入11个重新启动状态，包括步骤开始处的值和步骤中10个均匀分布间隔处的值。

对于协同仿真，Abaqus/Explicit将在NUMBER INTERVAL参数决定的时间之后立即在协同仿真目标时间写入重新启动信息。

TIME MARKS

设置TIME MARKS=NO（默认）以在NUMBER INTERVAL参数决定的时间之后立即结束的增量处写入重新启动。

设置TIME MARKS=YES以在NUMBER INTERVAL参数决定的确切时间写入结果。当在[*DYNAMIC](ch04abk43.md)选项（["显式动态分析，" Abaqus Analysis User's Guide第6.3.3节](../usb/usb-link.md#usb-anl-aexpdynamic)）上包含FIXED TIME INCREMENTATION或DIRECT USER CONTROL参数时，不能使用TIME MARKS=YES。

对于协同仿真，TIME MARKS参数始终设置为NO，以在NUMBER INTERVAL参数决定的时间之后在协同仿真目标时间写入重新启动信息。

OVERLAY

此参数指定每个步骤仅保留一个增量，从而最小化所需的存储空间。

当包含OVERLAY参数时，每个写入的增量都会覆盖同一步骤的先前增量（如果有）。如果省略此参数，则为每个增量保留数据。在任何情况下，每个步骤的最后一个增量都会被保留。

SINGLE

此参数指定在运行双精度可执行文件时仅写入单精度重新启动数据。

当运行单精度可执行文件时，此参数被忽略，并且将仅写入单精度重新启动数据。如果省略此参数，则在运行双精度可执行文件时将写入双精度重新启动数据。

**此选项没有关联的数据行。**
