# 2.2.4 将钢筋定义为单元属性





**产品：**Abaqus/Standard  Abaqus/Explicit

##### **参考**

- [*PRESTRESS HOLD*](../key/key-link.md#usb-kws-hprestresshold)
- [*REBAR*](../key/key-link.md#usb-kws-mrebar)

### 概述

在壳和膜单元中定义钢筋的首选方法是将增强层定义为单元截面定义的一部分（文档见["定义钢筋，"第2.2.3节"](pt01ch02s02aus13.md)）。在实体中定义钢筋的首选方法是将增强曲面或膜单元嵌入"宿主"实体单元中，如["嵌入单元，"第35.4.1节"](pt08ch35s04aus136.md)中所述。本节描述了在壳、膜和连续体单元中将钢筋定义为单元属性的替代方法。此方法比["定义钢筋，"第2.2.3节"](pt01ch02s02aus13.md)中描述的方法更麻烦，并且不允许在Abaqus/CAE中可视化钢筋和钢筋结果。

基于单元的钢筋：
- 用于在实体、膜和壳单元中定义单向增强；
- 可以在实体单元中定义为单独的增强筋；
- 可以定义为壳、膜和实体单元中均匀分布的钢筋层（此类层被视为平滑层，其恒定厚度等于每个钢筋横截面积除以钢筋间距）；
- 可用于耦合温度-位移单元，但不影响热传导率和比热；
- 可用于耦合热-电-结构单元，但不影响电导率、热传导率和比热；
- 在Abaqus/Standard中不影响模型质量；
- 不能用于热传递或质量扩散分析的单元；
- 不能与三角形壳和膜单元或三角形、三棱柱和四面体实体单元一起使用；和
- 具有与底层单元不同的材料属性。

### 为钢筋集分配名称

您必须为钢筋集分配一个名称。此名称可用于定义钢筋预应力和输出请求。特定单元或单元集中的每一层钢筋必须分配一个单独的名称。

### 在三维壳和膜单元中定义钢筋

可以在三维壳和膜单元中定义等参钢筋和斜向钢筋。钢筋不能与三角形壳或膜一起使用。

如果需要三角形壳或膜，可以使用 collapse 的四边形壳或膜。结果的钢筋方向将取决于钢筋的类型（等参或斜向）。必须仔细定义钢筋，因为单元会变形。此技术应仅在结果不关键且应力梯度不高的区域中使用。

钢筋的刚度计算使用与底层壳或膜单元计算相同的积分点。有关壳和膜单元的更多信息，请参见["壳单元：概述，"第29.6.1节"](pt06ch29s06abo27.md)和["膜单元，"第29.1.1节"](pt06ch29s01alm05.md)。

#### 在三维壳和膜单元中定义等参钢筋

等参钢筋沿单元中等参线的映射对齐（参见[图2.2.4-1](pt01ch02s02aus14.md#krebar-iso-shellmemb)）。

**图2.2.4-1** 未变形三维壳或膜单元中的"等参"钢筋。

![](../graphics/krebar-iso-shellmemb-nls.png)

如果包含钢筋的单元的相对边缘不平行，则钢筋方向在单元内的每个积分点处都会不同（参见[图2.2.4-2](pt01ch02s02aus14.md#erebar-dir-distort)）。

**图2.2.4-2** 变形三维壳或膜单元中的"等参"钢筋方向（虚线表示钢筋方向）。

![](../graphics/erebar-dir-distort.png)

钢筋间距在物理空间中是固定的。间距*s*和钢筋面积*A*用于确定等效平滑层的厚度，。如果包含钢筋的单元边缘不平行，则通过一个边缘的实际钢筋数量与通过相对边缘的钢筋数量不同（在等参空间中相对）。

您可以指定包含钢筋的单元；每个钢筋的横截面积*A*；壳平面中的钢筋间距*s*；以及在等参空间中绘制时钢筋所平行的边缘编号（参见[图2.2.4-1](pt01ch02s02aus14.md#krebar-iso-shellmemb)）。此外，对于壳单元，您可以指定钢筋在壳厚度方向上相对于壳中面的位置（正方向为壳法线正方向）。如果壳厚度由节点厚度定义（["节点厚度，"第2.1.3节"](pt01ch02s01aus07.md)），则此距离按节点厚度定义的厚度与截面定义的厚度的比率进行缩放。如果壳厚度用分布定义（["分布定义，"第2.8.1节"](pt01ch02s08aus26.md)），则此距离按分布定义的单元厚度与默认厚度的比率进行缩放。如果壳具有复合截面，其层厚度用分布定义（["分布定义，"第2.8.1节"](pt01ch02s08aus26.md)），则此距离按分布定义的单元层厚度之和与默认层厚度之和的比率进行缩放。

斜向钢筋不必与单元边缘相似；它们可以位于相对于局部1轴的任何规定角度。钢筋方向必须以两种方式之一定义，如图2.2.4-3所示：

**图2.2.4-3** 三维壳或膜中的"斜向"钢筋。

![](../graphics/erebar-skew-3d-shell-1storder-nls.png)

1. 钢筋可以相对于默认投影局部坐标系定义（参见["约定，"第1.2.2节"](pt01ch01s02aus02.md)）。
2. 钢筋可以相对于用户定义的局部坐标系定义（参见["方向，"第2.2.5节"](pt01ch02s02aus15.md)）。

可选地与壳或膜截面定义关联的方向定义对钢筋角方向定义没有影响。如果壳或膜在空间中弯曲，局部1方向将在整个单元中变化，斜向钢筋也将相应变化。

对于壳单元，使用分布（["分布定义，"第2.8.1节"](pt01ch02s08aus26.md)）定义局部坐标系对钢筋角方向定义没有影响。

如果钢筋横截面积为*A*，钢筋间距*s*，则等效"平滑"增强层的厚度应为。

### 在轴对称壳和膜单元中定义钢筋

轴对称膜中的钢筋必须位于膜参考曲面，而轴对称壳中的钢筋可以位于壳参考曲面或可以偏离中面。轴对称壳和膜中的钢筋可以相对于*r*–*z*平面具有任何方向。轴对称壳中的周向钢筋示例参见[图2.2.4-6](pt01ch02s02aus14.md#erebar-circ-axi-shell)，径向钢筋示例参见[图2.2.4-7](pt01ch02s02aus14.md#erebar-radial-axi-shell)。

**图2.2.4-6** 轴对称壳单元中周向钢筋示例。

![](../graphics/erebar-circ-axi-shell-nls.png)

**图2.2.4-7** 轴对称壳单元中径向钢筋示例。

![](../graphics/erebar-radial-axi-shell-nls.png)

您可以指定每个钢筋的横截面积*A*；钢筋间距*s*；对于壳单元，钢筋在壳厚度方向上相对于壳中面的位置（正方向为壳法线正方向）；相对于*r*–*z*平面的角度方向（以度为单位）；以及测量钢筋间距的径向位置。如果壳的厚度由节点厚度定义（["节点厚度，"第2.1.3节"](pt01ch02s01aus07.md)），则距中面的距离将按节点厚度定义的厚度与截面定义的厚度的比率进行缩放。如果壳的厚度用分布定义（["分布定义，"第2.8.1节"](pt01ch02s08aus26.md)），则距中面的距离将按分布定义的单元厚度与默认厚度的比率进行缩放。

如果为无扭曲轴对称壳或膜中的钢筋指定了0或90以外的方向角度，Abaqus假定钢筋是平衡的（即，一半钢筋位于指定角度，另一半位于角度），内部计算相应处理。详见["Abaqus理论指南"第3.7.1节"二维钢筋建模"](../stm/stm-link.md#stm-elm-rebars2d)。如果使用对称模型生成功能（["对称模型生成，"第10.4.1节"](pt04ch10s04aus63.md)）从轴对称壳或膜模型创建三维模型，只有平衡钢筋会被正确转换。轴对称模型中平衡钢筋的定义将在三维模型中产生平衡钢筋；不平衡钢筋的这种转换不可用。广义轴对称膜中带扭曲的不平衡钢筋将被正确转换。

如果给出了钢筋间距的径向位置，则当径向位置变化时，钢筋的总横截面积将保持恒定；此行为对应于周向方向上钢筋数量保持恒定，并意味着钢筋平滑层的厚度减小，钢筋间距随*r*增加而增加（参见[图2.2.4-7](pt01ch02s02aus14.md#erebar-radial-axi-shell)）。如果省略钢筋间距的径向位置（或设置为零），Abaqus假定钢筋间距保持恒定；相应平滑层的厚度保持恒定，使得。

### 在连续体单元中定义钢筋

二维或三维连续体（实体）单元可以包含钢筋；不能在三棱柱、棱柱、四面体或无限单元中定义钢筋。如果需要三角形或楔形单元，可以使用collapse的四边形或砖块单元。在包含钢筋的单元时，请小心。检查钢筋的位置和方向是否正确很重要。

钢筋定义为单根或层。在后一种情况下，该层是每个单元中的一个曲面；您需要在曲面中提供钢筋方向。

#### 在平面和轴对称连续体单元中定义钢筋层

默认情况下，钢筋形成一个层，该层位于与模型平面成直角的曲面上。您可以定义此钢筋曲面与模型平面相交的线，如下所述。

钢筋层内钢筋的方向通过给出角度（以度为单位）来定义，该角度是模型平面中的相交线与钢筋之间的角度。此角度在物理三维空间中测量，而不是在等参空间中测量。详见["Abaqus理论指南"第3.7.1节"二维钢筋建模"](../stm/stm-link.md#stm-elm-rebars2d)。相交线上沿正方向是从被相交边缘编号较低边缘到较高边缘， 正角度表示钢筋向下 orient 进入模型平面（其中平面平行于平面应变分析中的*z*轴或轴对称分析的![](../graphics/usb_eqn00123.gif)轴），如图2.2.4-8所示。

##### 定义斜向钢筋

对于斜向钢筋，钢筋层与模型平面的交线可以与单元的任何两个边缘相交。钢筋相交于两个边缘。交点位于通过沿每个相交边缘的分数距离定义的位置。在图中，钢筋层*A*由沿边缘1的比率和沿边缘2的比率定义。钢筋层*B*由沿边缘3的比率和沿边缘4的比率定义。

在连续体单元中定义斜向钢筋可能会显著增加Abaqus/Explicit分析的运行时间。在大多数情况下，单元的稳定时间增量将由钢筋的稳定时间增量决定，这取决于钢筋长度。钢筋长度由包括钢筋在单元中的位置、钢筋间距、钢筋面积和钢筋在钢筋曲面内的方向等因素决定。如果连续体单元中的斜向钢筋被定义为与两个相邻单元边缘相交，则 resulting 钢筋长度可能远小于平均单元边缘长度，从而导致非常小的单元稳定时间增量。

#### 在二维轴对称和广义平面应变连续体单元中定义单根钢筋

#### 在三维连续体单元中定义钢筋层

默认情况下，三维连续体单元中的钢筋定义为位于曲面中的层。曲面最容易相对于单元的等参映射立方体来定义。因此，在生成网格之前必须考虑如何定义钢筋；如果在设计网格时没有考虑到钢筋曲面，钢筋定义可能非常低效。

在等参映射立方体中，钢筋曲面始终有两个边缘（彼此相对）平行于等参方向。等参方向在[图2.2.4-12](pt01ch02s02aus14.md#krebar-3d-solid)中定义。您可以指定此等参方向（1、2或3）。

**图2.2.4-12** 三维单元的等参方向和边缘定义。

![](../graphics/krebar-3d-solid-nls.png)

单元的特定面（在此等参方向上）用于定义曲面其他两个边缘的位置；面在[图2.2.4-12](pt01ch02s02aus14.md#krebar-3d-solid)中定义，其中也定义了面的边缘。

如果定义了等参钢筋，钢筋曲面的两个不平行于用户指定等参方向的边缘将平行于其他两个等参方向之一；在等参映射立方中，等参坐标在钢筋曲面上为常数。[图2.2.4-13](pt01ch02s02aus14.md#erebar-3d-2iso-layers)用包含两层等参钢筋的单元说明了这个概念。

**图2.2.4-13** 具有两层等参钢筋的单元。

![](../graphics/erebar-3d-2iso-layers-nls.png)

每个曲面的位置由所选等参方向的面边缘（[图2.2.4-12](pt01ch02s02aus14.md#krebar-3d-solid)）的距离分数*f*给出；您必须指定从中测量分数距离的边缘。

如果定义了斜向钢筋，钢筋曲面的两个不平行于用户指定等参方向的边缘通常不平行于其他等参方向之一。钢筋曲面的这两个边缘的位置由钢筋曲面与相交面的边缘交点指定（对于所选等参方向）；交点由沿每个面边缘的分数距离*f*给出。（请注意，对于斜向钢筋，分数距离是沿边缘的；对于等参钢筋，分数距离是从边缘测量的。）沿边缘的分数距离从边缘的第一个节点测量。必须给出所有四个分数距离，但只有两个可以非零。

钢筋层内钢筋的方向角是在等参映射立方体中定义的；它以度为单位测量，是钢筋曲面与所选等参方向的相交线与钢筋之间的角度。相交线的正方向是从编号较低的边缘到编号较高的边缘；钢筋的正方向指向单元内部。如[图2.2.4-14](pt01ch02s02aus14.md#erebar-3d-geom-skew-exa)所示。方向角在等参映射立方体中的钢筋层中定义；因此，等参和斜向钢筋的定义相同。

**图2.2.4-14** 三维斜向钢筋建模的方向示例，等参方向2。显示在映射等参单元中。

![](../graphics/erebar-3d-geom-skew-exa-nls.png)

如果钢筋层在物理空间中不是平的，则每个积分点处的方向角可能不同。由于只能为每个单元定义一个方向角，因此必须使用单元的平均值方向角；对于合理的网格，此近似不应显著影响结果。

### 初始条件

初始条件（["Abaqus/Standard和Abaqus/Explicit中的初始条件，"第34.2.1节"](pt07ch34s02aus116.md)）可用于定义钢筋的预应力或解相关值。

#### 在钢筋中定义预应力

对于定义了增强的结构（如钢筋混凝土结构），您可以使用初始条件来定义钢筋中的预应力。

在Abaqus/Standard这种情况下，必须在通过初始静态分析步骤（["静态应力分析，"第6.2.2节"](pt03ch06s02at01.md)）主动加载之前（或者可能仅带有"恒载"）使结构达到平衡状态——参见["Abaqus/Standard和Abaqus/Explicit中的初始条件"中的"定义初始应力"，第34.2.1节"](pt07ch34s02aus116.md#usb-prc-pinitialcond-stress)。

| **输入文件用法：** | ``` [*INITIAL CONDITIONS*](../key/key-link.md#usb-kws-minitialcond), TYPE=STRESS, REBAR *element number or element set name, rebar name, prestress value* ``` |
| --- | --- |

#### 在Abaqus/Standard中保持钢筋中的预应力

如果在钢筋中定义了预应力，除非预应力保持固定，否则允许在平衡静态分析步骤期间发生变化；这是因为自平衡应力状态建立时结构应变的结果。一个例子是混凝土预应力的预张类型，其中增强腱最初被拉伸到所需张力，然后再被混凝土覆盖。混凝土固化并与钢筋粘合后，释放初始钢筋张力会将载荷传递到混凝土，在混凝土中引入压应力。混凝土中的 resulting 变形会减少钢筋中的应力。

或者，您可以在此初始平衡解期间保持一些或所有钢筋中定义的初始应力恒定。一个例子是混凝土后张预应力；允许钢筋穿过混凝土滑动（通常它们在管道中），预应力载荷由某些外部来源（预应力千斤顶）维持。钢筋中预应力的大小通常是设计要求的一部分，不应随着混凝土在预应力载荷下压缩而减小。通常，预应力仅在分析的第一步中保持恒定。这通常是预应力的更常见假设。

如果在保持预应力恒定的步骤之后的分析步骤中预应力未保持恒定，则由于混凝土中的额外变形，钢筋中的应力将发生变化。如果没有额外变形，钢筋中的应力将保持在初始条件设定的水平。如果加载历史导致在保持预应力恒定的步骤之后的步骤中混凝土或钢筋中未引入塑性变形，则在移除这些步骤中施加的载荷后，钢筋中的应力将恢复到初始条件设定的水平。

| **输入文件用法：** | ``` [*PRESTRESS HOLD*](../key/key-link.md#usb-kws-hprestresshold) ``` |
| --- | --- |

### 输出

钢筋力输出在钢筋积分位置可用，输出变量为RBFOR。钢筋力等于钢筋应力乘以当前钢筋横截面积。钢筋的当前横截面积是通过假设钢筋由不可压缩材料制成来计算的，无论实际材料定义如何。对于膜或壳单元中的钢筋，输出变量RBANG和RBROT分别识别单元内等参或斜向钢筋的当前方向以及由于有限变形而导致的钢筋相对旋转。这些量是相对于单元中用户指定的等参方向测量的，而不是默认局部单元系统或方向定义的系统。详见["壳、膜和曲面单元中的钢筋建模，"Abaqus理论指南第3.7.3节"](../stm/stm-link.md#stm-elm-rebarshell)。

有关应力、应变等其他输出量的信息，请参见["Abaqus/Standard输出变量标识符，"第4.2.1节"](pt02ch04s02abv01.md)和["Abaqus/Explicit输出变量标识符，"第4.2.2节"](pt02ch04s02xbv01.md)。对于具有多个积分点的膜或壳单元中的钢筋，输出量可在积分点和单元质心处使用。

#### 指定壳和膜单元中钢筋角度输出的方向

输出量RBANG和RBROT可以从壳或膜平面中的任一等参方向测量。您可以指定所需的等参方向，从该方向测量钢筋角度（1或2）。在轴对称壳和膜中，第一个等参方向与经向方向重合，第二个等参方向与周向方向重合。钢筋角度从等参方向到钢筋测量，正角度定义为绕单元法线方向逆时针旋转。默认方向是第一个等参方向。

| **输入文件用法：** | 使用以下任一选项： |
| --- | --- |
|  | ``` [*REBAR*](../key/key-link.md#usb-kws-mrebar), ELEMENT=SHELL, MATERIAL=*mat*, ISODIRECTION=*n* [*REBAR*](../key/key-link.md#usb-kws-mrebar), ELEMENT=AXISHELL, MATERIAL=*mat*, ISODIRECTION=*n* [*REBAR*](../key/key-link.md#usb-kws-mrebar), ELEMENT=MEMBRANE, MATERIAL=*mat*, ISODIRECTION=*n* [*REBAR*](../key/key-link.md#usb-kws-mrebar), ELEMENT=AXIMEMBRANE, MATERIAL=*mat*, ISODIRECTION=*n* ``` |

例如，用户定义的局部坐标系用于定义壳单元中的斜向钢筋（斜角），RBANG的输出值为75，如图所示。

 ELSET1, 0.01, 0.1, 0.0, 30.
[*ORIENTATION*](../key/key-link.md#usb-kws-morientation), SYSTEM=RECTANGULAR, NAME=ORIENT
