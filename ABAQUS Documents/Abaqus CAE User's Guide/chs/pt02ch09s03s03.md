# 9.3.3 调整缓存大小以提高网络ODB的性能





当您启动Abaqus/CAE会话时，会在临时文件目录中创建一个缓存。当您使用网络ODB连接器从远程输出数据库读取时，Abaqus/CAE将此缓存用于本地数据存储。缓存极大地提高了访问远程输出数据库数据时Abaqus/CAE中可视化模块的性能。

Abaqus/CAE允许缓存增长到足以包含所有打开的远程输出数据库中所有数据的大小。但是，Abaqus/CAE将缓存大小限制为目录中总可用空间的80%。例如，如果临时目录有35GB的可用空间，Abaqus/CAE将允许缓存增长到28GB。或者，您可以使用Abaqus环境文件`abaqus_v6.env`中的**nodb_cache_limit**参数来限制缓存大小。您必须将**nodb_cache_limit**参数设置为缓存大小将被限制到的兆字节数。例如，

```
`nodb_cache_limit`=20000
```

会将最大缓存大小设置为20GB。Abaqus/CAE仅在会话期间根据需要使用此缓存空间，实际缓存大小可能明显小于您指定的限制。**nodb_cache_limit**的最小值为500，表示缓存大小限制为500MB。如果您将最大缓存大小设置为大于可用可用空间，Abaqus/CAE会将其减少到等于可用可用空间的值。

当从远程输出数据库读取数据时，Abaqus/CAE使用缓存来提高其性能。通过网络访问数据的速度明显低于从本地磁盘驱动器访问数据的速度。因此，远程输出数据库的性能将明显慢于本地输出数据库的性能。缓存通过保留已通过网络传输的数据来减少这种性能差异，从而减少网络数据传输的需要。但是，如果缓存不够大，Abaqus/CAE将不得不通过网络传输更多数据，性能会下降。

在大多数情况下，您不必使用**nodb_cache_limit**参数来调整缓存大小。但是，如果缓存消耗过多磁盘空间并降低系统上其他应用程序的速度，您可能需要减小缓存的大小。同样，如果缓存太小而无法支持所有远程输出数据库且Abaqus/CAE性能下降，您可能需要增加缓存的大小。如果无法增加缓存大小，则应关闭一些远程输出数据库。

如果所需的缓存大小大于临时文件目录中的可用空间，您可以使用Abaqus **scratch**环境文件参数将临时文件目录移动到更大的磁盘驱动器。有关更多信息，请参阅["使用Abaqus环境设置，"Abaqus Analysis User's Guide第3.3.1节"](../usb/usb-link.md#usb-int-denvfile)和["管理Abaqus中的内存和磁盘使用，"Abaqus Analysis User's Guide第3.4.1节"](../usb/usb-link.md#usb-int-dmemory)。
