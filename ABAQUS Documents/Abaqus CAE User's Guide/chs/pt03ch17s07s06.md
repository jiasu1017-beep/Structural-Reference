# 17.7.6 中轴算法和前进波算法有什么区别？





中轴算法和前进波算法是Abaqus/CAE在执行以下操作时可用于生成网格的两种网格划分方案：
- 使用自由网格技术对曲面进行四边形或四边形主导单元网格划分。
- 使用扫掠网格技术对实体区域进行六面体或六面体主导单元网格划分。（Abaqus/CAE通过使用两种算法从源侧扫掠到目标侧生成的四边形和四边形主导单元来生成六面体和六面体主导网格。）

两种算法描述如下：

**中轴**

中轴算法首先将待网格化的区域分解为一组更简单的区域。然后，该算法使用结构化网格技术用单元填充每个简单区域。如果待网格化的区域相对简单且包含大量单元，则中轴算法比前进波算法生成网格的速度更快。最小化网格过渡可能会提高网格质量。网格过渡选项仅适用于四边形和六面体网格划分。有关更多信息，请参见["什么是网格过渡？"第17.7.5节](pt03ch17s07s05.md)。

**前进波**

前进波算法在区域边界生成四边形单元，并随着它系统地移动到区域内部继续生成四边形单元。

前进波算法生成的单元将始终精确遵循您的种子（对于四边形主导和六面体主导网格除外，除非您正在创建三维旋转网格，且被旋转的轮廓接触旋转轴）。对于其他网格，前进波算法生成的单元将始终比中轴算法生成的单元更紧密地遵循种子。如果待网格化的区域包含虚拟拓扑，则只能使用前进波算法生成网格。

如果您选择前进波算法，您可以允许Abaqus/CAE在适当的地方使用映射网格。（映射网格与结构化网格相同，但仅适用于四边形区域。）有关更多信息，请参见["什么是映射网格？"第17.8.2节](pt03ch17s08s02.md)和["Abaqus/CAE何时可以应用映射网格？"第17.8.6节](pt03ch17s08s06.md)。

由于前进波算法生成的单元遵循您的种子，生成的网格可能在窄区域中包含一些畸变单元。如图17-35所示。

**图17-35** 在某些情况下，前进波算法生成的单元存在一些畸变。

![](../graphics/mgn-algorithm-goodmedial-nls.png)

相反，前进波算法可能会生成具有更一致长宽比的更均匀尺寸的单元，如图17-36所示。均匀的单元尺寸在分析中起着重要作用；例如，如果您为Abaqus/Explicit分析创建网格，网格中的小单元可能会过度控制时间步长。此外，如果单元遵循您的种子很重要，则前进波算法更可取。

**图17-36** 在某些情况下，前进波算法产生更均匀的网格。

![](../graphics/mgn-algorithm-goodadvancing.png)

在某些情况下，当您对多个区域进行网格划分时，Abaqus/CAE会在区域边界生成带有剪切单元的网格。一个区域中的节点可能与相邻区域中的节点定位不同，当Abaqus/CAE合并相邻网格时，这会在公共边界处导致剪切。图17-37显示多个扫掠区域以及中轴算法生成的相应网格。

**图17-37** 使用中轴算法时，相邻区域之间的网格剪切显著。

![](../graphics/mgn-meshshear-medial.png)

前进波算法将源侧的节点定位在与您的种子相同的位置；因此，网格剪切将减少。图17-38显示使用相同布种的前进波算法网格划分的相同零件。但是，如前所述，您可能需要尝试这两种算法以获得最佳网格。

**图17-38** 使用前进波算法时，相邻区域之间的网格剪切减少。

![](../graphics/mgn-meshshear-advfront.png)
