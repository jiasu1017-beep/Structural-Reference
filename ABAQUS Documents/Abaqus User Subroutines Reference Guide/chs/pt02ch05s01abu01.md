### 概述

Abaqus结果（`.fil`）文件可以使用本节中描述的实用程序进行访问。访问是在分析之后由用户编写的后处理程序进行的，或者在Abaqus/Standard中，可以在分析期间通过用户子程序[`URDFIL`](../sub/sub-link.md#sub-xsl-urdfil)进行。

以下实用子程序可用：
- `DBFILE`（从文件读取）
- `DBFILW`（写入文件）
- `DBRNU`（设置文件的单元编号）
- `INITPF`（初始化文件）
- `POSFIL`（确定文件中的位置；仅在Abaqus/Standard中可用）

这些实用子程序按字母顺序在下面描述。

只有子程序`DBFILE`和`POSFIL`可以从用户子程序[`URDFIL`](../sub/sub-link.md#sub-xsl-urdfil)中调用。

### `DBFILE`（从文件读取）

#### 接口

```
CALL DBFILE(LOP,ARRAY,JRCD)
```

#### 需要提供给实用程序的变量

`LOP`

一个标志，在调用`DBFILE`之前必须设置此标志以指示操作。设置`LOP`=0以读取文件中的下一条记录；设置`LOP`=2以倒回当前正在读取的文件（例如，如果需要多次读取文件，由于它是顺序文件，必须倒回）。如果使用`LOP`=2，必须首先将文件读到最后，并且只能在到达文件末尾时倒回。

#### 从实用程序返回的变量

`ARRAY`

包含文件中一条记录的数组，格式如["结果文件输出格式，" 第5.1.2节](pt02ch05s01afi01.md)中所述。当`LOP`=0时，此数组将由数据管理程序填充，每次执行`DBFILE`调用时将包含文件中下一条记录的内容。`ARRAY`必须在例程中充分维化，以容纳文件中最大的记录。对于几乎所有情况，500个字足够了。例外情况是：如果问题定义包含使用超过此数量的状态变量的用户元素或用户材料，或者使用了大量保留自由度数的子结构（有关子结构的更多详细信息，请参见["使用子结构，" 第10.1.1节](pt04ch10s01aus58.md)）。当结果文件在Abaqus以双精度运行的系统上写入时，`ARRAY`必须在例程中声明为双精度。

`JRCD`

当使用`LOP`=0调用`DBFILE`时，如果读取到文件末尾标记，则返回非零。

### `DBFILW`（写入文件）

#### 接口

```
CALL DBFILW(LOP,ARRAY,JRCD)
```

#### 需要提供给实用程序的变量

`ARRAY`

包含要写入文件的一条记录的数组，格式如["结果文件输出格式，" 第5.1.2节](pt02ch05s01afi01.md)中所述。

`JRCD`

返回代码（0 – 记录成功写入，1 – 记录未写入）。

`LOP`

当前未使用。

### `DBRNU`（设置文件的单元编号）

#### 接口

```
CALL DBRNU(JUNIT)
```

#### 需要提供给实用程序的变量

`JUNIT`

要读取的结果文件的FORTRAN单元编号。有效的单元编号为：8用于读取`.fil`文件，15–18，或大于100的编号。

### `INITPF`（初始化文件）

#### 接口

```
CALL INITPF(FNAME,NRU,LRUNIT,LOUTF)
```

#### 需要提供给实用程序的变量

`FNAME`

定义正在读取或写入的文件根文件名（即不带扩展名的名称）的字符字符串。`FNAME`必须声明为`CHARACTER*80`，可以包括目录规范以及根文件名。每个单独文件的扩展名由下面的`LRUNIT`数组定义。有关文件命名约定的讨论请参见下文。

`NRU`

一个整数，给出后处理程序将读取的结果文件数量。通常只读取一个结果文件，但有时需要读取多个结果文件——例如，将它们合并成一个文件。

`LRUNIT`

一个整数数组，必须在后处理程序中维化为`LRUNIT(2,NRU)`，并且在调用`INITPF`之前必须包含以下数据：

`LRUNIT(1,K1)`是读取第K1个结果文件的FORTRAN单元编号。有效的单元编号为：8用于读取`.fil`文件，15–18，或大于100的编号。所有其他单元编号由Abaqus保留。有关基于单元编号的命名约定，请参见下文。

`LRUNIT(2,K1)`是一个整数，如果第K1个结果文件以二进制文件形式写入，则必须设置为2；如果第K1个结果文件以ASCII格式写入，则必须设置为1。

`LOUTF`

只有当调用`INITPF`的程序也将以Abaqus结果文件格式写入输出文件时，才需要定义（例如，如果结果文件正在合并成单个结果文件，或者结果文件正在从二进制转换为ASCII格式）。在这种情况下，如果输出文件要以二进制文件形式写入，`LOUTF`应设置为2；如果要以ASCII文件形式写入，则设置为1。此结果文件将以`.fin`文件扩展名写入。有关写入结果文件的讨论，请参见["访问结果文件信息，" 第5.1.3节](pt02ch05s01aus43.md)；有关此文件命名的信息，请参见下文。

#### 文件命名约定

文件扩展名源自`LRUNIT(1,K1)`的值。如果`LRUNIT(1,K1)`为8，文件名将使用扩展名`fil`。任何其他单元编号将导致文件扩展名为`0nn`，其中`nn`是分配给`LRUNIT(1,K1)`的编号。例如，如果`LRUNIT(1,K1)`为15，文件扩展名为`.015`。如果通过`LOUTF`的非零值指示了输出文件，其扩展名将为`.fin`。

例如，要读取文件`xxxx.fil`，请将`LRUNIT(1,K1)`设置为8，并将字符变量`FNAME`设置为`xxxx`（使用赋值或数据语句）。如果需要，`FNAME`可以包括目录规范、设备名称或路径。操作系统环境和shell变量不会正确转换，因此不应使用。

Abaqus生成的所有错误消息都写入FORTRAN单元6。在大多数机器上，如果程序以交互方式运行，错误消息默认直接打印到屏幕。您可以在主程序中为单元6包含一个open语句，以将消息重定向到文件。如果希望读取或写入`LRUNIT`中指定以外的单元，可能必须在程序中包含这些单元的OPEN语句（取决于所使用的计算机）。此类辅助文件的单元编号应大于100，以避免与Abaqus内部文件冲突。

### `POSFIL`（确定文件中的位置）

`POSFIL`实用程序仅在Abaqus/Standard中可用。

#### 接口

```
CALL POSFIL(NSTEP,NINC,ARRAY,JRCD)
```

#### 需要提供给实用程序的变量

`NSTEP`

所需的步。如果此变量设置为0，将读取第一个可用的步。

`NINC`

所需的增量。如果此变量设置为0，将读取指定步的第一个可用增量。

#### 从实用程序返回的变量

`ARRAY`

包含请求的步和增量的结果文件中记录2000的值的实数数组。

`JRCD`

返回代码（0 – 找到指定的增量，1 – 未找到指定的增量）。如果在结果文件中找不到请求的步和增量，`POSFIL`将返回错误，并将您定位在结果文件的末尾。

#### 使用`POSFIL`定位

您可能会发现同时将`NSTEP`和`NINC`设置为0来调用`POSFIL`会很方便，这样可以跳过在分析开始时写入结果文件的信息（参见["结果文件输出格式，" 第5.1.2节](pt02ch05s01afi01.md)），从而开始从写入文件的第一个增量开始读取。

`POSFIL`不能用于在结果文件中向后移动：您不能使用`POSFIL`找到文件中的给定增量，然后稍后再次调用`POSFIL`来读取比第一个找到的更早的增量。如果尝试这样做，`POSFIL`将返回错误，指示未找到请求的增量。
