# 19.6.6 在修改现有分析步骤后重启





您可以使用重启功能来分析模型，然后在继续分析之前修改现有步骤。本节描述此用法的一个示例。

假设您已完成以下操作：
- 创建了名为 `Model-A` 的模型，该模型有两个分析步骤，`Step-1` 和 `Step-2`。
- 使用步骤模块中的**编辑重启请求**对话框在每个步骤结束时输出重启信息。
- 创建了使用 `Model-A` 的作业 `Job-A`。
- 分析了模型。

在检查 `Job-A` 的结果后，您意识到需要更改 `Step-2` 中的内容、`Step-2` 中的输出请求，或 `Step-2` 中的规定条件（如载荷或边界条件）。您可能还想添加一些带有附加输出请求和载荷以及边界条件更改的新步骤。您知道您仍然可以使用 `Step-1` 的结果；但是，您意识到 `Step-2` 的结果不再有效。重启功能允许您重新计算 `Step-2` 的结果，而无需重复 `Step-1` 的计算。以下步骤描述了建议遵循的过程：

1. 将 `Model-A` 复制到新模型（例如 `Model-A-restart`），并使 `Model-A-restart` 成为当前模型。
2. 执行以下操作：- 对 `Step-2` 进行所需的更改。- 在 `Step-2` 之后添加新步骤。- 在 `Step-2` 和后续步骤中创建新的规定条件。
3. 从主菜单栏，选择 ****模型**![](../graphics/images/arrow.gif)**编辑属性**![](../graphics/images/arrow.gif)**Model-A-restart****。在出现的**编辑模型属性**对话框中，执行以下操作：- 输入 `Job-A` 作为将从中读取重启数据的作业。- 设置重启位置：- 输入 `Step-1` 以指示将从中读取数据的步骤。- 选择**从步骤结束时重启**。`Step-2` 将在 `Step-1` 结束时继续分析。
4. 将 `Job-A` 复制到使用 `Model-A-restart` 的新作业（例如 `Job-A-restart`）。Abaqus/CAE 将作业类型设置为 `Restart`。
5. 提交 `Job-A-restart` 进行分析。
