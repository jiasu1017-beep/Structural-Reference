# 3.2.35 与ZAERO交换Abaqus数据





**产品：** Abaqus/Standard  

##### **参考**

- ["Abaqus执行程序：概述，" 第3.1.1节](pt01ch03s01abo02.md)

### 概述

**abaqus tozaero**接口使您能够在Abaqus和ZAERO分析产品之间交换气动弹性数据。通过在应用程序之间使用此接口，您可以在Abaqus中对模型进行结构模态分析，将其转移到ZAERO进行气动弹性分析，然后再转移回Abaqus进行应力分析。

### 通用文件

通用文件是Abaqus和ZAERO之间数据交换的方式。它由四个数据集组成：2411（描述节点和坐标数据）、2414（描述质量归一化模态）、2420（描述全局坐标系）以及2453（以文本格式描述质量矩阵）或2453b（以二进制格式描述质量矩阵）。

您可以使用**mode**参数指定通用文件的输出格式。选择文本格式使您能够在文本编辑器中修改通用文件，但会使文件大小增加到二进制格式的两倍以上。文本是默认格式，也是ZAERO支持的唯一格式。[表3.2.35-1](pt01ch03s02abx35.md#table-zaero-unv-text)和[表3.2.35-2](pt01ch03s02abx35.md#table-zaero-unv-binary)分别描述了质量矩阵数据集的文本格式和二进制格式。

### 准备Abaqus分析输入文件

在接口创建通用文件之前，您必须对Abaqus输入(`.inp`)文件进行以下添加，然后运行Abaqus：
- 在特征频率提取分析中对特征向量进行归一化，使其相对于结构的质量矩阵。此归一化是必要的，因为翻译器假定模态是质量归一化的；如果在此步骤之前跳过此步骤，则翻译的模态将不正确，并会产生错误结果而不会发出警告或错误。有关更多信息，请参阅["自然频率提取，" 第6.3.5节](pt03ch06s03at10.md)。
- 在分析步骤中包含以下行：
  ```
  [*ELEMENT MATRIX OUTPUT*](../key/key-link.md#usb-kws-helemmatout), ELSET=allelements, MASS=YES, OUTPUT FILE=USER DEFINED, FILE NAME=*mtx-file-name*
  ```
  其中`allelements`是包含应包含在全局质量矩阵中的所有元素的定义元素集。矩阵输出将被放入文件`*mtx-file-name*.mtx`；您不应指定`.mtx`扩展名，因为Abaqus会自动添加。

### 工作流程

本节介绍Abaqus和ZAERO之间三个主要步骤的输入和输出。

#### 在Abaqus中进行模态分析

Abaqus模态分析使用Abaqus输入文件，并向输出数据库(`.odb`)文件和矩阵(`.mtx`)文件输出以下数据：结构模型节点、坐标系、模态频率、广义质量、模态形状和质量矩阵。

#### 在ZAERO中进行气动弹性分析

气动弹性分析需要ZAERO输入文件和toZAERO创建的通用文件。ZAERO输出由于气动力而在结构节点上的力和力矩数据到通用文件。

#### 在Abaqus中进行应力分析

然后可以将ZAERO输出的力和力矩用于静态（线性或非线性）Abaqus分析，以计算挠度、应力和载荷。

### 命令行选项

**job**

此选项用于指定Abaqus输入文件的名称。它也是通用输出数据库和质量矩阵文件的默认名称。

**unvfile**

此选项用于指定通用文件的名称（如果与`*job-name*`不同）。如果未提供`.unv`扩展名，Abaqus会自动添加。

**odbname**

此选项用于指定Abaqus输出数据库文件的名称（如果与`*job-name*`不同）。如果未提供`.odb`扩展名，Abaqus会自动添加。

**mtxfile**

此选项用于指定包含Abaqus生成的元素质量矩阵的文件的名称。如果未提供`.mtx`扩展名，Abaqus会自动添加。

**step**

此选项指定包含Abaqus特征频率提取结果的步骤号。默认值为1。

**注意：**您必须在特征频率提取分析中对特征向量进行归一化，使其相对于结构的质量矩阵。有关更多信息，请参阅["自然频率提取，" 第6.3.5节](pt03ch06s03at10.md)。

**mode**

此选项指定通用文件的输出格式。如果此选项设置为`binary`，Abaqus以二进制格式写入部分通用文件以节省空间。如果此选项设置为`text`，Abaqus以全文本格式写入整个文件。默认值为`text`，这是ZAERO当前支持的唯一模式。
