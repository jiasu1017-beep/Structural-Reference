# 3.4.5 描述对象状态的术语





Abaqus/CAE使用以下术语来描述对象在特定步骤中的状态：

**Created**

对象已创建并在该步骤中变为活动状态。规定条件在步骤中变为活动状态的点取决于与该步骤关联的振幅变化。有关更多信息，请参阅"Abaqus分析用户指南"中的"规定条件"["定义分析，" 第6.1.2节](../usb/usb-link.md#usb-anl-aover)。

**Computed**

分析产品将计算该步骤中对象的值。

**Modified**

对象的定义已在该步骤中修改。规定条件在步骤过程中的变化取决于与该步骤关联的振幅变化。

**Propagated**

对象已在分析的前一步骤中创建、修改或计算，并继续在该步骤中保持活动状态。

**Inactive**

对象已在该步骤或前一步骤中被停用。它将在所有后续步骤中保持停用状态，直到您重新激活它。您不能在创建对象的步骤中停用该对象。规定条件变为非活动状态的点取决于与该步骤关联的振幅变化。有关更多信息，请参阅"Abaqus分析用户指南"中的"规定条件"["定义分析，" 第6.1.2节](../usb/usb-link.md#usb-anl-aover)。

您不能停用预定义场；预定义场的非活动状态意味着该场已重置为初始步骤中指定的值。对象恢复其初始值的点取决于与该步骤关联的振幅变化。有关更多信息，请参阅"Abaqus分析用户指南"中的"规定条件"["定义分析，" 第6.1.2节](../usb/usb-link.md#usb-anl-aover)。

**N/A**

对象对该步骤的计算没有任何影响。

以下术语仅适用于线性扰动步骤：

**Built into base state**

在任何先前一般分析步骤中创建的活动对象将成为基础状态的一部分，在线性扰动步骤期间无法更改。

**Propagated from base state**

在任何先前一般步骤中创建的对象将成为该过程的基础状态，但可由用户修改或停用。

**Deactivated from base state**

在任何先前一般步骤中创建的对象在该线性扰动步骤中被停用。停用状态仅适用于线性扰动步骤，不会传播到剩余步骤。

有关线性扰动步骤的信息，请参阅["一般和线性扰动过程，" Abaqus分析用户指南第6.1.3节](../usb/usb-link.md#usb-anl-alinearnonlinear)。

以下术语仅适用于模态动力学步骤：

**Built into modes**

在任何先前频率分析步骤中活动的边界条件将用于模态计算，因此将被构建到基于模态的线性扰动过程和子空间动力学过程的模态中。在这些基于模态和子空间动力学过程期间，边界条件无法更改。

有关模态动力学步骤的信息，请参阅["瞬态模态动力学分析，" Abaqus分析用户指南第6.3.7节](../usb/usb-link.md#usb-anl-amodaldynamic)。
