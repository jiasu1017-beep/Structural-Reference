# 36.4.5 Abaqus/Explicit中一般接触的接触控制






**产品：** Abaqus/Explicit

### 概述

一般接触算法的接触控制：
- 可用于选择性地缩放一般接触域内特定区域的默认惩罚刚度；
- 可用于控制节点在所有连接的面和边缘因单元失效侵蚀后是否从一般接触域中移除；
- 可用于在一般接触域内特定区域激活节点-表面接触的非默认跟踪算法；
- 可用于控制是否需要执行检查以防止一般接触表面折叠并反转自身；
- 可用于修改一般接触域中一个或多个表面对的默认初始过闭合解决方法；和
- 可用于修改默认接触厚度缩减检查。

### 缩放默认惩罚刚度

一般接触算法使用惩罚方法来施加接触约束（有关更多信息，请参阅["Abaqus/Explicit中的接触约束施加方法，" 第38.2.3节"](pt09ch38s02aus182.md)）。将接触力与穿透距离关联的"弹簧"刚度由Abaqus/Explicit自动选择，使得对时间增量的影响最小，但在大多数分析中允许的穿透不显著。如果存在以下任何因素，分析中可能会产生显著的穿透：
- 位移控制的加载
- 接触界面上纯粹弹性或随变形硬化的材料
- 具有相对较小自身质量并通过边界条件以外的方法（例如连接器）约束参与接触的变形单元（特别是膜和表面单元）
- 具有相对较小的自身质量或旋转惯性并通过边界条件以外的方法（例如连接器）约束参与接触的刚体

请参阅["Abaqus Benchmarks Guide第1.1.11节"](../bmk/bmk-link.md#bmk-anl-hertzcontact)，了解其中前两个因素结合导致默认惩罚刚度的接触穿透显著的示例。

您可以指定一个缩放因子，用于修改一般接触域内指定相互作用的惩罚刚度。此缩放可能会影响自动时间增量。由于维持数值稳定性所必需的时间增量减少，使用大的缩放因子可能会增加分析所需的计算时间（请参阅["Abaqus/Explicit中的接触约束施加方法，" 第38.2.3节"](pt09ch38s02aus182.md)，了解更多讨论）。

用户指定的可变质量缩放在计算质量增加时不考虑接触的影响。通常，这种影响不显著，因为默认惩罚刚度只会将稳定时间增量减少非常小的量。但是，如果指定了高的惩罚缩放因子，则尽管指定了质量缩放，稳定时间增量也可能显著减少。

用于指定应分配非默认惩罚刚度的区域的表面名称不必与用于指定一般接触域的表面名称对应。在许多情况下，接触相互作用将为大域定义，而非默认惩罚刚度将分配给该域的子集。如果分配了非默认惩罚刚度的表面落在一般接触域之外，则控制分配将被忽略。如果指定区域重叠，最后分配优先。

| **输入文件用法：** | ``` [*CONTACT CONTROLS ASSIGNMENT*](../key/key-link.md#usb-kws-hcontcntrlassign), TYPE=SCALE PENALTY *surface_1*, *surface_2*, *scale_factor* ``` |
| --- | --- |
| | 此选项必须与[*CONTACT*](../key/key-link.md#usb-kws-hcontact)选项结合使用。每个步骤最多应出现一次；数据行可以根据需要重复，以将惩罚刚度缩放因子分配给不同区域。如果省略第一个表面名称，则假定为包含整个一般接触域的默认表面。如果省略第二个表面名称或与第一个表面名称相同，则指定的接触控制被分配给第一个表面与其自身之间的接触相互作用。请记住，表面可以定义为跨越多个未连接的实体，因此自接触不限于单个实体与自身的接触。 |

### 节点侵蚀控制

您可以控制节点在所有周围的面和边缘因单元失效侵蚀后是否保留在接触域中。默认情况下，这些节点保留在接触域中，并充当自由浮点质量，可以与仍属于接触域的面接触。您可以指定基于单元的表面的节点应在所有连接到的接触面和接触边缘侵蚀后侵蚀（即从接触域中移除）。仅通过基于节点的表面包含在接触域中的节点永远不会从接触域中移除。

如果不指定节点侵蚀，计算成本可能会随着自由飞行节点增加，特别是在并行进行的分析中。增加的计算成本与自由飞行节点可能移动到远离保持活动的单元的距离有关，这会拉伸接触域的体积，从而倾向于增加接触搜索成本以及并行分析中处理器之间通信的成本。但是，在某些情况下，涉及自由飞行节点的接触可能贡献显著的动量传递，如果指定了节点侵蚀，则不会被考虑。

| **输入文件用法：** | ``` [*CONTACT CONTROLS ASSIGNMENT*](../key/key-link.md#usb-kws-hcontcntrlassign), NODAL EROSION=NO ``` |
| --- | --- |
| | 此选项必须与[*CONTACT*](../key/key-link.md#usb-kws-hcontact)选项结合使用。此参数设置适用于整个一般接触域。 |

### 激活节点-表面接触的非默认跟踪算法

提供了一种非默认接触跟踪算法，它利用更多的局部拓扑和几何信息来跟踪节点和面之间的接触。此算法在某些建模情况下可能导致更稳健的接触跟踪，例如在折叠气囊充气事件期间。

跟踪算法按表面逐个表面激活。您必须指定需要激活跟踪算法的表面名称。接触域中指定表面的节点接触属于自身表面（自接触）或属于任何其他表面的面（未排除节点-表面接触）的所有接触相互作用将使用非默认节点-表面跟踪方案跟踪。

用于指定应使用非默认跟踪算法的区域的表面名称不必与用于指定一般接触域的表面名称对应。在许多情况下，接触相互作用将为大域定义，而非默认跟踪算法将被分配给该域的子集。如果需要激活非默认跟踪算法的表面落在一般接触域之外，则控制分配被忽略。

| **输入文件用法：** | ``` [*CONTACT CONTROLS ASSIGNMENT*](../key/key-link.md#usb-kws-hcontcntrlassign), TYPE=FOLD TRACKING *surface_1* ``` |
| --- | --- |
| | 此选项必须与[*CONTACT*](../key/key-link.md#usb-kws-hcontact)选项结合使用。每个步骤最多应出现一次；数据行可以根据需要重复，以在接触域的不同区域激活非默认跟踪算法。如果省略表面名称，则假定为包含整个一般接触域的默认表面。 |

### 激活折叠反转检查

如果一般接触表面包含尖锐折叠，重大加载事件（例如，折叠气囊充气期间遇到的那些）可能导致一个或多个折叠反转。折叠在最可能在未在形成折叠的面边缘上激活边缘-边缘接触的折叠处发生。边缘-边缘约束的存在通常防止折叠反转。在没有边缘-边缘接触约束的情况下，折叠的反转可能导致节点-表面接触跟踪算法出错，并可能导致在反转折叠形成的一个面上被跟踪的节点在跟踪面的错误侧"卡住"。为避免这种情况，可能需要为包含尖锐折叠的模型激活折叠反转检查。折叠反转检查检测折叠即将反转的情况，并对面形成折叠施加力场以防止折叠反转。

折叠反转检查按表面逐个表面激活。您必须指定需要激活折叠反转检查的表面名称。如果为特定表面激活，则折叠反转检查适用于该表面内的所有折叠。

用于指定应激活折叠反转检查的区域表面名称不必与用于指定一般接触域的表面名称对应。在许多情况下，接触相互作用将为大域定义，而折叠反转检查将在该域的子集中激活。如果需要激活折叠反转检查的表面落在一般接触域之外，则控制分配被忽略。

| **输入文件用法：** | ``` [*CONTACT CONTROLS ASSIGNMENT*](../key/key-link.md#usb-kws-hcontcntrlassign), TYPE=FOLD INVERSION CHECK *surface_1* ``` |
| --- | --- |
| | 此选项必须与[*CONTACT*](../key/key-link.md#usb-kws-hcontact)选项结合使用。每个步骤最多应出现一次；数据行可以根据需要重复，以在接触域的不同区域激活折叠反转检查。如果省略表面名称，则假定为包含整个一般接触域的默认表面。 |

### 激活边缘-边缘接触的默认跟踪算法

默认接触跟踪算法比替代跟踪算法利用更多的局部信息来跟踪边缘之间的接触，并且通常减少所需的全局跟踪范围。使用此算法可能会减少具有广泛边缘-边缘接触定义的分析（例如折叠气囊充气模拟）的计算时间，在充气事件期间，可能需要激活气囊膜表面上所有特征边缘以在充气事件期间准确施加接触。

可以明确指定默认跟踪算法，尽管如果未为跟踪算法指定接触控制，接触域中的所有边缘-边缘接触将使用默认跟踪算法强制执行。

| **输入文件用法：** | ``` [*CONTACT CONTROLS ASSIGNMENT*](../key/key-link.md#usb-kws-hcontcntrlassign), TYPE=ENHANCED EDGE TRACKING（默认） ``` |
| --- | --- |
| | 此选项必须与[*CONTACT*](../key/key-link.md#usb-kws-hcontact)选项结合使用。此参数设置适用于整个一般接触域。 |

### 边缘-边缘接触的替代跟踪算法

提供了一种替代接触跟踪算法，它在跟踪边缘之间的接触时比默认跟踪算法利用更少的局部信息。此算法通常会增加所需的全局跟踪范围，因此在大多数分析中会增加计算时间。当指定替代边缘跟踪算法时，接触域中的所有边缘-边缘接触将使用此算法强制执行。

| **输入文件用法：** | ``` [*CONTACT CONTROLS ASSIGNMENT*](../key/key-link.md#usb-kws-hcontcntrlassign), TYPE=EDGE TRACKING ``` |
| --- | --- |
| | 如果指定，此选项必须与[*CONTACT*](../key/key-link.md#usb-kws-hcontact)选项结合使用。此参数设置适用于整个一般接触域。 |

### 初始过闭合解决的控制

默认情况下，Abaqus/Explicit自动调整表面位置以去除模拟第一步中存在于一般接触域中的小初始过闭合。来自单独接触定义、边界条件、绑定约束和刚体约束的冲突调整可能导致初始过闭合的不完全解决。无法通过重新定位节点解决的初始过闭合被存储为初始接触偏移，以避免分析开始时产生大的接触力。

或者，在某些情况下，可能需要完全避免表面对之间的节点调整，并将表面之间的所有初始过闭合视为临时接触偏移。然后，您可以指定不应通过节点调整解决的初始过闭合的表面，而应存储为偏移。

| **输入文件用法：** | ``` [*CONTACT CONTROLS ASSIGNMENT*](../key/key-link.md#usb-kws-hcontcntrlassign), AUTOMATIC OVERCLOSURE RESOLUTION *surface_1*, *surface_2*, *STORE OFFSETS* ``` |
| --- | --- |
| | 此选项必须与[*CONTACT*](../key/key-link.md#usb-kws-hcontact)选项结合使用。每个步骤最多应出现一次；数据行可以根据需要重复，以将非默认过闭合解决方法分配给不同区域。如果省略第一个表面名称，则假定为包含整个一般接触域的默认表面。如果省略第二个表面名称或与第一个表面名称相同，则指定的接触控制被分配给第一个表面与其自身之间的接触相互作用。 |

#### 边缘-边缘相互作用的初始过闭合解决控制的效果

接触偏移与单个节点-面元和边缘-边缘组合相关联。在滑动时，Abaqus/Explicit尝试将接触偏移转移到不同的节点-面元或边缘-边缘配对。然而，在涉及单个节点或边缘或带角落表面的多次接触的某些情况下，接触偏移可能无法在滑动时保持（即可能变为零）。导致接触偏移值在增量之间不连续的局限性（对于边缘-边缘接触比节点-表面接触更可能）可能局部降低解的质量，导致解依赖于所使用的处理器数量，或导致分析中止。这些局限性可以通过更仔细地由预处理器定位表面节点来避免，或者在许多情况下，通过允许发生无应变调整来避免。

### 接触厚度缩减检查的控制

默认情况下，一般接触算法要求接触厚度不超过表面面元边缘长度或对角线长度的某个分数。此分数通常根据单元的几何形状以及单元是否靠近壳周缘而在20%到60%之间变化。一般接触算法将在必要时自动缩减接触厚度，而不影响底层单元的单元计算中使用的厚度。

为检查是否需要在模型中任何特定区域缩减厚度，接触算法首先为每个接触节点分配完整厚度，表示为以节点为中心直径等于厚度的球。然后，缩减厚度，使得球不与任何未直接连接到节点的相邻面元重叠，防止形成虚假的自接触。然后，将壳周缘上的节点在面元平面中最大移动50%的面元尺寸，远离周缘，以消除接触对算法（见["为Abaqus/Explicit中的接触对分配表面属性，" 第36.5.2节"](pt09ch36s05aus161.md))中发生的"牛鼻"效应。如果壳周缘节点的厚度大于最大周缘偏移的两倍，则执行最终厚度缩减以消除"牛鼻"的剩余部分。

 如果默认厚度缩减在模型的特定区域不可接受，您可以通过接触排除定义从自接触中排除这些区域（见["在Abaqus/Explicit中定义一般接触相互作用，" 第36.4.1节"](pt09ch36s04aus155.md)）并激活接触厚度缩减检查的控制。

| **输入文件用法：** | 使用以下选项在从自接触中排除的模型区域消除厚度缩减，同时仍在壳周缘处缩减厚度，其中周缘偏移不足以避免"牛鼻"效应： |
| --- | --- |
| | ``` [*CONTACT CONTROLS ASSIGNMENT*](../key/key-link.md#usb-kws-hcontcntrlassign), CONTACT THICKNESS REDUCTION=SELF ``` 使用以下选项在从自接触中排除的模型区域和所有壳周缘处消除厚度缩减（如果厚度大于最大周缘偏移的两倍，则壳周缘节点处将形成"牛鼻"）： ``` [*CONTACT CONTROLS ASSIGNMENT*](../key/key-link.md#usb-kws-hcontcntrlassign), CONTACT THICKNESS REDUCTION=NOPERIMSELF ``` |

### 摩擦接触中壳和梁厚度偏移的增量旋转考虑

默认情况下，摩擦的滑移增量计算不考虑壳和梁厚度偏移的增量旋转，并且摩擦约束不对由于壳或梁厚度而偏移于接触界面的节点施加力矩。在大多数情况下，忽略这些效应对结果的影响很小；然而，在某些应用中可能很显著。

[图36.4.5-1](pt09ch36s04aus159.md#contact-thickness-rotation)显示了一个示例，其中表面厚度显著影响滑移增量计算（因此影响粘附条件的正确施加）。此示例涉及与滚柱导轨摩擦接触的壳表面，在接触区域没有相对滑动。壳的参考表面（包含壳节点）在接触区域偏离滚柱的参考表面半个壳厚度。如图所示，由于厚度偏移的旋转，两个参考表面之间应存在一些切向运动差异。粘附接触区域中的壳节点应比滚柱上的接触点具有稍大的增量位移，因为壳节点离旋转轴更远；然而，默认情况下，粘附在滚柱上的壳节点的增量位移将与滚柱上接触点的增量位移相同。为在这种情况下提高准确性，您可以指定应考虑结构旋转项。

**图36.4.5-1** 壳厚度对滑移增量的影响。

![](../graphics/contact-thickness-rotation-nls.png)

摩擦约束应对由于壳或梁厚度而偏移于接触界面的节点施加力矩，以对抗与摩擦力偶相关的净力矩。[图36.4.5-2](pt09ch36s04aus159.md#contact-thickness-moment)中所示的施加节点力矩抵消了相关摩擦力偶的力矩，使得与摩擦约束相关的净力和净力矩为零。但是，默认情况下，Abaqus/Explicit忽略此力矩，并在节点由于壳和梁厚度而偏移于接触界面时产生净力矩。为在这种情况下提高准确性，您可以指定应考虑结构旋转项。
