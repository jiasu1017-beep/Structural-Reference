# 11.4 重启分析







多步骤模拟不必在单个作业中定义。事实上，通常希望分阶段运行复杂模拟。这允许您检查结果并确认分析按预期执行，然后继续下一阶段。Abaqus 重启分析功能允许重启模拟并计算模型对附加载荷历史的响应。

重启分析功能在 ["重启分析，" Abaqus Analysis User's Guide 第 9.1.1 节](../usb/usb-link.md#usb-anl-arestart) 中详细讨论。

### 11.4.1 重启和状态文件

Abaqus/Standard 重启（`.res`）文件和 Abaqus/Explicit 状态（`.abq`）文件包含继续先前分析所需的信息。在 Abaqus/Explicit 中，包（`.pac`）文件和选定结果（`.sel`）文件也用于重启分析，必须在第一个作业完成时保存。此外，两种产品都需要输出数据库（`.odb`）文件。对于大型模型，重启文件可能变得非常大；当请求重启数据时，默认情况下每个增量或间隔都会写入它们。因此，控制写入重启数据的频率很重要。有时允许在步骤期间覆盖重启文件上的数据是有用的。这意味着在分析结束时，每个步骤只有一组重启数据，对应于每个步骤结束时的模型状态。但是，如果分析因某些原因（如计算机故障）中断，则可以从上次写入重启数据的位置继续分析。

### 11.4.2 写入重启文件

[*RESTART*](../key/key-link.md#usb-kws-mrestart) 选项控制重启文件的写入。虽然该选项可以出现在输入文件中的任何位置，但它通常作为步骤定义的一部分出现。与其他输出请求选项一样，在 [*RESTART*](../key/key-link.md#usb-kws-mrestart) 选项中定义的值适用于当前步骤和任何后续一般步骤，直到在后续步骤中修改该选项。该选项

```
[*RESTART*](../key/key-link.md#usb-kws-mrestart), WRITE, FREQUENCY=*<n>*
```

每 *n* 个增量向重启文件写入数据。无论最后一个增量是否可以被 *n* 整除，都会在每个步骤结束时写入重启数据。如果省略 FREQUENCY 参数，则每个增量都会写入数据。

对于大型模型，重启文件可能变得非常大，因此通常包含 OVERLAY 参数来控制重启文件的大小。此参数允许在步骤期间覆盖重启文件上的数据。

### 11.4.3 读取重启文件

当从先前分析的结束处重启模拟时，在 [*RESTART*](../key/key-link.md#usb-kws-mrestart) 选项上使用 READ 参数。您还可以使用 STEP 和 INC 参数指定要重启的分析的载荷历史中的特定点。执行重启模拟时，[*RESTART*](../key/key-link.md#usb-kws-mrestart) 选项应紧跟在 [*HEADING*](../key/key-link.md#usb-kws-mheading) 选项之后。在此重启输入文件中不需要出现模型数据，因为分析模型数据将从重启文件读取。只能在重启输入文件中修改节点集定义、单元集定义、幅值定义和附加历史数据。

**继续中断的运行**

新分析直接从先前分析的指定步骤和增量继续。如果给定的步骤和增量不对应于先前分析的结束，Abaqus 将模拟所有先前定义的载荷历史数据的剩余部分，然后才尝试模拟输入文件中提供的任何新载荷历史数据。因此，如果分析被计算机故障中断，以下输入文件将按原始定义完成分析：

```
[*HEADING*](../key/key-link.md#usb-kws-mheading)
Restart of interrupted run
[*RESTART*](../key/key-link.md#usb-kws-mrestart), READ, STEP=*<step>*, INC=*<increment>*
```

**继续附加步骤**

如果先前分析成功完成，并且您已查看结果，想要将附加步骤添加到载荷历史，则指定的步骤和增量应是先前分析的最后步骤和最后增量。或者，可以省略它们，默认情况下 Abaqus 将读取重启文件中最后可用的数据。[*RESTART*](../key/key-link.md#usb-kws-mrestart) 选项后面跟着任何新的步骤定义。

```
[*HEADING*](../key/key-link.md#usb-kws-mheading)
Add new step data
[*RESTART*](../key/key-link.md#usb-kws-mrestart), READ, STEP=*<last step>*, INC=*<last increment>*
[*STEP*](../key/key-link.md#usb-kws-hstep)
*... new step definition...*
[*END STEP*](../key/key-link.md#usb-kws-hendstep)
```

**更改分析**

有时，在查看先前分析的结果后，您可能希望从中间点重启分析并以某种方式更改剩余载荷历史数据——例如，添加更多输出请求，更改载荷，或调整分析控制。当步骤超过其最大增量数时，通常需要这样做。如果按上述方式重启分析，Abaqus 认为分析在步骤中途，继续尝试完成该步骤，然后立即再次超过最大增量数。

在这种情况下，应在 [*RESTART*](../key/key-link.md#usb-kws-mrestart) 选项上包含 END STEP 参数，以指示当前步骤应在 [*RESTART*](../key/key-link.md#usb-kws-mrestart) 选项上指定的步骤和增量处终止，所有先前定义的历史数据应被忽略。然后，模拟可以继续使用在 [*RESTART*](../key/key-link.md#usb-kws-mrestart) 选项之后定义的新步骤。例如，如果步骤仅允许最多 20 个增量，少于完成步骤所需的增量数，则以下重启输入文件将允许 Abaqus 重启模拟并完成施加的载荷：

```
[*HEADING*](../key/key-link.md#usb-kws-mheading)
Continue an analysis that exceeded the maximum number of
increments
[*RESTART*](../key/key-link.md#usb-kws-mrestart), READ, STEP=*<step>*, INC=20, END STEP
[*STEP*](../key/key-link.md#usb-kws-hstep), INC=100
*... repeat step definition...*
[*END STEP*](../key/key-link.md#usb-kws-hendstep)
```
在这种情况下，完整步骤定义，包括施加的载荷和边界条件，应与原始运行中指定的完全相同，但以下情况除外：
- 应增加增量数。
- 新步骤的总时间应是原始步骤总时间减去第一步中完成的时间。例如，如果步骤最初指定为 100 秒，分析在步骤时间为 20 秒时耗尽增量，则重启分析中步骤的持续时间应为 80 秒。
- 以步骤时间表示的任何幅值定义需要重新指定以反映新步骤的时间刻度。以总时间表示的幅值定义不需要更改，前提是使用了上述修改。

任何载荷或规定边界条件的幅度保持不变，因为它们在一般分析步骤中始终是总值。
