### 概述

可以使用本节中描述的实用程序访问 Abaqus 结果（`.fil`）文件。访问是在分析之后通过用户编写的后处理程序进行的，或者在 Abaqus/Standard 中，通过用户子程序 [`URDFIL`](../sub/sub-link.md#sub-xsl-urdfil) 在分析过程中进行。

以下实用子程序可用：
- `DBFILE`（从文件读取）
- `DBFILW`（写入文件）
- `DBRNU`（设置文件的单元号）
- `INITPF`（初始化文件）
- `POSFIL`（确定文件中的位置；仅在 Abaqus/Standard 中可用）

这些实用子程序在以下字母顺序中描述。

只能从用户子程序 [`URDFIL`](../sub/sub-link.md#sub-xsl-urdfil) 中调用子程序 `DBFILE` 和 `POSFIL`。

### `DBFILE`（从文件读取）

#### 接口

```
CALL DBFILE(LOP,ARRAY,JRCD)
```

#### 需要提供给实用程序的变量

`LOP`

一个标志，您必须在调用 `DBFILE` 之前设置它，指示操作。设置 `LOP`=0 以读取文件中的下一条记录；设置 `LOP`=2 以倒回当前正在读取的文件（例如，如果需要多次读取文件，则必须倒回，因为它是顺序文件）。如果使用 `LOP`=2，则必须首先将文件读取到末尾，并且仅在到达文件末尾时应倒回。

#### 从实用程序返回的变量

`ARRAY`

包含文件中一条记录的数组，格式如『结果文件输出格式』，第 5.1.2 节中所述。当 `LOP`=0 时，此数组将由数据管理程序填充，每次执行 `DBFILE` 调用时，文件中的下一条记录的内容将进入该数组。必须在您的程序中充分维化 `ARRAY` 以包含文件中的最大记录。对于几乎所有情况，500 个字就足够了。例外情况是如果问题定义包括用户单元或使用超过此数量的状态变量的用户材料，或者使用了大量保留自由度子结构（有关子结构的更多详细信息，请参阅『使用子结构』，第 10.1.1 节）。当结果文件在以双精度运行 Abaqus 的系统上写入时，`ARRAY` 必须在您的程序中声明为双精度。

`JRCD`

当使用 `LOP`=0 调用 `DBFILE` 时，如果读取到文件结束标记，则返回非零值。

### `DBFILW`（写入文件）

#### 接口

```
CALL DBFILW(LOP,ARRAY,JRCD)
```

#### 需要提供给实用程序的变量

`ARRAY`

包含要写入文件的单条记录的数组，格式如『结果文件输出格式』，第 5.1.2 节中所述。

`JRCD`

返回码（0 – 记录成功写入，1 – 记录未写入）。

`LOP`

当前未使用。

### `DBRNU`（设置文件的单元号）

#### 接口

```
CALL DBRNU(JUNIT)
```

#### 需要提供给实用程序的变量

`JUNIT`

要读取的结果文件的 FORTRAN 单元号。有效的单元号是 8（读取 `.fil` 文件）、15–18 或大于 100 的数字。

### `INITPF`（初始化文件）

#### 接口

```
CALL INITPF(FNAME,NRU,LRUNIT,LOUTF)
```

#### 需要提供给实用程序的变量

`FNAME`

定义正在读取或写入的文件根文件名（即不带扩展名的名称）的字符字符串。`FNAME` 必须声明为 `CHARACTER*80`，可以包括目录规范以及根文件名。以下每个单独文件的扩展名由下面的 `LRUNIT` 数组定义。请参阅下面的文件命名约定讨论。

`NRU`

一个整数，给出后处理程序将读取的结果文件的数量。通常只读取一个结果文件，但有时需要读取多个结果文件——例如，将它们合并到单个文件中。

`LRUNIT`

一个整数数组，必须在后处理程序中维化为 `LRUNIT(2,NRU)`，并且必须在调用 `INITPF` 之前包含以下数据：

`LRUNIT(1,K1)` 是 FORTRAN 单元号，将在其上读取第 K1 个结果文件。有效的单元号是 8（读取 `.fil` 文件）、15–18 或大于 100 的数字。所有其他单元号由 Abaqus 保留。请参阅下面的基于单元号的命名约定。

`LRUNIT(2,K1)` 是一个整数，如果第 K1 个结果文件以二进制文件形式写入，则设置为 2；如果第 K1 个结果文件以 ASCII 格式写入，则设置为 1。

`LOUTF`

仅在调用 `INITPF` 的程序也将以 Abaqus 结果文件格式写入输出文件时需要定义（例如，如果结果文件被合并到单个结果文件中，或者结果文件正从二进制转换为 ASCII 格式）。在这种情况下，如果输出文件要写成二进制文件，则应将 `LOUTF` 设置为 2；如果输出文件要写成 ASCII 文件，则应设置为 1。此结果文件将使用文件扩展名 `.fin` 写入。请参阅『访问结果文件信息』，第 5.1.3 节，以获取有关写入结果文件的讨论；请参阅下文以获取有关此文件命名的信息。

#### 文件命名约定

文件扩展名源自 `LRUNIT(1,K1)` 的值。如果 `LRUNIT(1,K1)` 是 8，则文件名将使用扩展名 `fil` 构造。任何其他单元号都将导致文件扩展名为 `0nn`，其中 `nn` 是分配给 `LRUNIT(1,K1)` 的编号。例如，如果 `LRUNIT(1,K1)` 是 15，则文件扩展名为 `.015`。如果通过 `LOUTF` 的非零值指示了输出文件，则其扩展名将为 `.fin`。

例如，要读取文件 `xxxx.fil`，请将 `LRUNIT(1,K1)` 设置为 8，并将字符变量 `FNAME` 设置为 `xxxx`（使用赋值或数据语句）。如果需要，`FNAME` 可以包括目录规范、设备名称或路径。操作系统环境和 shell 变量不会正确转换，因此，不应使用。

Abaqus 生成的所有错误消息都写入 FORTRAN 单元 6。在大多数机器上，如果以交互方式运行程序，错误消息默认直接打印到屏幕。您可以在主程序中为单元 6 包含一个 open 语句，以将消息重定向到文件。如果您希望读取或写入除 `LRUNIT` 中指定的单元以外的单元，则可能必须在程序中包含这些单元的 OPEN 语句（取决于所使用的计算机）。此类辅助文件的单元号应大于 100，以避免与 Abaqus 内部文件冲突。

### `POSFIL`（确定文件中的位置）

`POSFIL` 实用程序仅在 Abaqus/Standard 中可用。

#### 接口

```
CALL POSFIL(NSTEP,NINC,ARRAY,JRCD)
```

#### 需要提供给实用程序的变量

`NSTEP`

所需的步骤。如果此变量设置为 0，则将读取第一个可用步骤。

`NINC`

所需的增量。如果此变量设置为 0，则将读取指定步骤的第一个可用增量。

#### 从实用程序返回的变量

`ARRAY`

包含请求的步骤和增量的结果文件记录 2000 的值的实数数组。

`JRCD`

返回码（0 – 找到指定的增量，1 – 未找到指定的增量）。如果在结果文件中未找到请求的步骤和增量，`POSFIL` 将返回错误，并将您定位在结果文件的末尾。

#### 使用 `POSFIL` 定位

您可能会发现将 `NSTEP` 和 `NINC` 都设置为 0 来调用 `POSFIL` 很方便，这样可以跳过在分析开始时写入结果文件的信息（请参阅『结果文件输出格式』，第 5.1.2 节），从而开始读取写入文件的第一条增量。

`POSFIL` 不能用于在结果文件中向后移动：您不能使用 `POSFIL` 来找到文件中的给定增量，然后稍后再次调用 `POSFIL` 来读取比首先找到的更早的增量。如果尝试这样做，`POSFIL` 将返回错误，指示未找到请求的增量。
