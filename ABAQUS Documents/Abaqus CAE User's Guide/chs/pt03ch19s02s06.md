# 19.2.7 远程提交作业






配置作业时，您可以请求 Abaqus/CAE 将作业路由到指定远程 UNIX 或 Linux 主机上的指定队列。您可以通过在作业编辑器的**提交**选项卡页面中选择关联的队列名称来指定远程队列。

**注意：**作业编辑器显示基于本地计算机当前会话的有效环境文件的内存使用、并行化和精度默认设置。当您将作业提交到远程计算机时，Abaqus 会用远程计算机环境文件中的设置替换默认设置。作业编辑器中的非默认设置会与作业一起保存，无论您在何处运行分析。

作业编辑器中显示的每个队列名称都引用您的 Abaqus 环境文件中的一个条目，您可以在其中指定您希望如何在主机上运行作业。换句话说，当您在作业编辑器中选择队列名称时，您不仅指定了主机上的所需队列，还指定了其他选项，例如您要在其上运行作业的主机上的目录，以及作业完成时要复制回本地目录的文件。

您可以通过将以下内容添加到 Abaqus 环境文件来指定远程运行作业的首选项：

```
def onCaeStartup():
    import os

    def makeQueues(*args):
        session.Queue(*name*,
                      *queueName*,
                      *hostName*,
                      *fileCopy*,
                      *directory*,
                      *driver*,
                      *localPlatform*,
                      *filesToCopy*,
                      *description*)

    addImportCallback('job', makeQueues)
```
此条目使用 Abaqus 命令语言编写。以下列表描述了上述条目中的每个参数。

***name***

您希望出现在作业编辑器中的队列名称。

***queueName***

主机上现有队列的名称。（有关在主机上创建队列的信息，请参阅 [Abaqus Installation and Licensing Guide](../sgb/sgb-link.md#sgb)。）

***hostName***

主机计算机的名称。默认为本地计算机的名称。

***fileCopy***

当分析完成时，此参数的值决定分析文件是否将被复制回提交作业的目录。默认值为 `ON`。

***directory***

主机上您要运行作业的目录名称。您必须对此目录有写权限。默认为本地目录（您提交作业的目录）。

***driver***

在主机上执行 Abaqus/Standard 或 Abaqus/Explicit 的命令名称。默认为 `abaqus`。

***localPlatform***

本地计算机上的平台。您可以指定 `UNIX` 或 `WINDOWS`；默认为 `UNIX`。

***filesToCopy***

作业完成时要复制回本地目录的分析文件的三字母扩展名。默认情况下，复制以下扩展名的文件：`log`、`dat`、`msg`、`sta`、`odb`、`res`、`abq` 和 `pac`。

**注意：**重启（`.res`）文件、Abaqus/Explicit 状态（`.abq`）文件和打包（`.pac`）文件是平台相关的；如果您的本地平台和远程平台设置不同，您将无法复制和使用这些文件而无需某种翻译。上面列出的所有其他文件可以跨平台复制而不会有任何困难。

***description***

队列的简短描述。

`name` 和 `queueName` 参数必须包含在每个队列定义中。但是，如果您在队列定义中不包含任何其他参数，则会自动提供默认值。示例队列定义如下所示：
```
def onCaeStartup():
    import os

    def makeQueues(*args):
        session.Queue(name='long',
                        queueName='aba_long',
                        hostName='jobserver',
                        directory='/scratch/' + os.environ['USER'])
    addImportCallback('job', makeQueues)
```
上述示例中的命令配置了以下内容：

**`name`**

作业编辑器中显示的队列名称为 `long`。

**`queueName`**

主机上的队列名称为 `aba_long`。

**`hostName`**

主机计算机的名称为 `jobserver`。

**`directory`**

Abaqus 将存储输入文件的目录以及与作业关联的所有其他文件的目录为 `/scratch/`*您的用户名*。

由于上述条目中省略了 *fileCopy*、*driver*、*localPlatform* 和 *filesToCopy* 参数，这些参数的默认选项会自动分配给此队列。

如果您想创建两个或更多队列，您可以根据需要多次重复包含 `session.Queue` 命令的行。例如，以下 Abaqus 环境文件条目指定了两个队列，一个名为 `long`，另一个名为 `job`：

```
def onCaeStartup():
    import os

    def makeQueues(*args):
        session.Queue(name='long',
                        queueName='aba_long',
                        hostName='jobserver',
                        directory='/scratch/' + os.environ['USER'])
        session.Queue(name='job',
                        queueName='aba_job',
                        hostName='jobserver',
                        fileCopy=OFF)

    addImportCallback('job', makeQueues)
```

[「监控分析作业进度，」第 19.2.6 节](pt03ch19s02hlb01.md) 中描述的监控功能可用于远程运行的作业，就像本地运行的作业一样。但是，作业的输出数据库与您可能已请求的任何其他分析文件一样，在作业完成之前不会复制到您的本地目录。因此，如果您想使用可视化模块查看在远程系统上运行的分析生成的结果，则必须创建并启动网络输出数据库连接器。有关更多信息，请参阅[「访问远程计算机上的输出数据库，」第 9.3 节](pt02ch09s03.md)。
