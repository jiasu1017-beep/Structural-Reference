# E 验证过程





## 附录 E：验证过程

验证过程检查所有已许可的 Abaqus 产品的安装，并报告每个产品的验证成功或失败。验证过程在 Abaqus 安装程序完成后自动运行，但只验证产品的子集。验证过程也可以作为 Abaqus 命令选项运行，并在 Windows 平台的**开始**菜单中于安装后的任何时间运行。

该过程为每个已许可的产品运行验证问题，并将结果与参考值进行比较。命令行选项不受许可类型的影响；也就是说，验证过程尝试验证命令中指定的所有产品。在运行验证过程之前，会检查所选产品的许可要求。如果检测到教学学术许可证，则在安装期间运行的验证过程仅检查 Abaqus/Standard、Abaqus/Explicit 和 Abaqus/CAE。

如果产品未获得许可，则跳过产品验证。命令行选项 `-NoLicenseCheck` 可用于绕过这些检查。

所有 Abaqus 产品的验证问题在安装期间自动从磁盘提取。运行时，验证过程尝试在当前工作目录中创建一个名为 `verify` 的新目录。在 Abaqus 安装过程中，`verify` 目录在版本的 `installation_info` 目录中创建。

**从命令行运行验证过程：**

通过输入以下命令运行过程：

```
*abaqus* verify [-adams -all -ams -tosca -cae -catiav4 -catiav5
-cfd -contextHelp -cPerf -dcatiav5 -design -docUrl -exp -foundation
-help -install -ioPerf -log -make -moldflow -noGui
-NoLicenseCheck -parallel -param -parasolid -proe
-retainFiles -scripting -std -swi -user_exp -user_std
-verbose -viewer]
```

**常用选项**

| `-all` | 验证所有已许可的产品。所有其他验证选项（`log` 和 `NoLicenseCheck` 除外）都被忽略。 |
| --- | --- |
| `-help` | 打印 `verify` 用法摘要。 |
| `-install` | 仅验证主要 Abaqus 产品。仅当您的系统有可用的 Fortran 编译器时，才会测试带用户子程序的 Abaqus/Standard 和带用户子程序的 Abaqus/Explicit。 |
| `-log` | 将所有输出定向到当前工作目录中名为 `verify.log` 的文件。 |

**产品选项**

| `-ams` | 验证 Abaqus/AMS。 |
| --- | --- |
| `-tosca` | 验证 Tosca for Abaqus。 |
| `-cae` | 验证 Abaqus/CAE。 |
| `-cfd` | 验证 Abaqus/CFD。 |
| `-design` | 验证 Abaqus/Design。 |
| `-exp` | 验证 Abaqus/Explicit。 |
| `-foundation` | 验证 Abaqus/Foundation。 |
| `-param` | 验证参数化研究。 |
| `-std` | 验证 Abaqus/Standard。 |
| `-user_exp` | 验证带用户子程序的 Abaqus/Explicit。 |
| `-user_std` | 验证带用户子程序的 Abaqus/Standard。 |
| `-viewer` | 验证 Abaqus/Viewer。 |

**翻译器选项**

| `-adams` | 验证 Abaqus Interface for MSC.ADAMS。 |
| --- | --- |
| `-catiav4` | 验证 Geometry Translator for CATIA V4。 |
| `-catiav5` | 验证 Geometry Translator for CATIA V5。 |
| `-dcatiav5` | 验证 Direct Geometry Import for CATIA V5（Direct Geometry Import 是 CATIA V5 Associative Interface 的组件）。验证 Abaqus/CAE 中的几何导入功能；不验证 CATIA V5 Associative Interface 插件的安装或功能。 |
| `-moldflow` | 验证 Abaqus Interface for Moldflow。 |
| `-parasolid` | 验证 Geometry Translator for Parasolid。 |
| `-proe` | 验证 Geometry Translator for Pro/ENGINEER（Geometry Translator 是 Pro/ENGINEER Associative Interface 的组件）。验证 Abaqus/CAE 中的几何导入功能；不验证 Pro/ENGINEER Associative Interface 插件的安装或功能。 |
| `-swi` | 验证 Geometry Translator for SolidWorks（Geometry Translator 是 SolidWorks Associative Interface 的组件）。验证 Abaqus/CAE 中的几何导入功能；不验证 SolidWorks Associative Interface 插件的安装或功能。 |

**附加选项**

| `-contextHelp` | 验证 Abaqus/CAE 上下文相关帮助功能。 |
| --- | --- |
| `-cPerf` | 验证 Abaqus/CAE 性能。 |
| `-docUrl` | 验证 Abaqus HTML 文档 URL。 |
| `-ioPerf` | 验证 I/O 性能。 |
| `-make` | 验证 **abaqus make** 实用程序。 |
| `-noGui` | 验证 Abaqus/CAE 和 Abaqus/Viewer 的 `-noGUI` 选项。 |
| `-NoLicenseCheck` | 需要 `-all`、`-install` 或产品选项列表。运行指定产品的验证过程，但绕过所有许可检查。该过程尝试验证所有选定产品，不论许可情况如何。 |
| `-parallel` | 使用并行化验证 Abaqus 分析作业。 |
| `-scripting` | 验证 Abaqus 脚本界面。 |
| `-retainFiles` | 在 `verify` 目录中保留所有验证文件（默认情况下，成功验证后会删除文件）。 |
| `-verbose` | 包含用于调试目的的附加详细信息。 |

如果为验证选择的产品未获得许可，则会显示所选产品的 Abaqus 警告，并继续验证其他选定产品。

**在 Windows 平台上运行验证过程：**

1. 从任务栏，单击**开始**。
2. 从菜单中，选择 ****程序**![](../graphics/images/arrow.gif)**Abaqus *release***![](../graphics/images/arrow.gif)**Abaqus Verification****，其中 *release* 对应于您要验证的版本。此过程尝试验证所有已许可的产品。使用命令行选项验证单个产品。

**审查和解决验证过程失败**

如果验证过程成功完成，所有文件和 `verify` 目录都会被删除（除非您使用 `retainFiles` 选项）。验证失败的所有产品的错误诊断保留在 `verify` 目录中。审查这些错误消息非常重要。

以下建议可能帮助您纠正导致验证过程失败的常见安装错误：
- 确保许可文件已正确安装。如果许可文件有问题，错误消息将写入标准输出。
- 确保您没有尝试执行您未获得许可的 Abaqus 产品。
- 确保操作系统和编译器级别与 Program Directory 中为此版本指定的级别一致。（请参阅 [http://media.3ds.com/support/progdir](http://media.3ds.com/support/progdir)。选择 SIMULIA Abaqus 作为产品**系列**，选择 Abaqus 6.14 作为**级别**，然后在左侧窗格中选择**先决条件**。）

如果错误消息和这些建议不足以验证安装，请在 Dassault Systmes Knowledge Base 中查找有关安装问题、解决方案和验证的信息，网址为 [www.3ds.com/support/knowledge-base](http://www.3ds.com/support/knowledge-base)。

如果您仍然无法解决问题，请联系您当地的办事处或代表寻求帮助。支持选项概述可在 Dassault Systmes Knowledge Base 中找到，网址为 [www.3ds.com/support/knowledge-base](http://www.3ds.com/support/knowledge-base)。
