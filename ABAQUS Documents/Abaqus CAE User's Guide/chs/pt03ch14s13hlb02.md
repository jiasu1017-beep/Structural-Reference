# 14.13.2 配置重启动输出请求

从主菜单栏选择**输出**![](../graphics/images/arrow.gif)**重启动请求**来配置写入重启动文件的信息。更多信息，请参见["重新启动分析，"第19.6节"](pt03ch19s06.md)，和["重新启动分析，"第9.1.1节，Abaqus分析用户指南"](../usb/usb-link.md#usb-anl-arestart)。

**要配置重启动请求：**

1. 从主菜单栏，选择**输出**![](../graphics/images/arrow.gif)**重启动请求**。**编辑重启动请求**对话框出现，其中包含当前模型中步骤的列表。
2. 从对话框中，选择步骤并指定Abaqus写入重启动信息的频率。执行以下操作：**Abaqus/Standard和Abaqus/CFD**对于Abaqus/Standard或Abaqus/CFD分析步骤，按增量或时间间隔输入频率：
   - 要在每个常规增量数之后写入重启动信息，请在**频率**列中输入该数字。
   - 要在步骤中以特定数量的等间距时间间隔写入重启动信息，请在**间隔**列中输入该数字。重启动信息在时间间隔指定的时间之后立即结束的增量时写入。在Abaqus/Standard分析中，您可以开启**时间标记**列以在时间间隔指定的精确时间写入重启动信息。默认情况下，不会为Abaqus/Standard或Abaqus/CFD分析步骤写入重启动信息（即，频率设置为零）。
   **Abaqus/Explicit**对于Abaqus/Explicit分析步骤，输入写入重启动信息的等间距时间**间隔**数。默认情况下，Abaqus/Explicit仅在每个步骤的开始和结束时写入重启动信息（即，**间隔**设置为一）。您无法避免为Abaqus/Explicit分析步骤写入重启动文件数据；**间隔**数必须设置为一或更大。默认情况下，Abaqus/Explicit在时间间隔指定的时间之后立即结束的增量时写入重启动信息。开启**时间标记**列以在时间间隔指定的精确时间写入重启动信息。
3. 对于Abaqus/Standard或Abaqus/Explicit分析，您可以请求仅保留每个步骤中一个增量的信息或每个指定增量的信息。
   - 开启**覆盖**列以请求仅在重启动文件中保留每个步骤中一个增量的信息，从而最小化文件大小。当步骤完成时，保留最后一个增量。
   - 关闭**覆盖**列以请求在重启动文件中保留每个指定增量或时间间隔的信息。您只能当频率大于零时使用**覆盖**列。
4. 对分析的每个步骤重复步骤2和3。
5. 完成重启动输出请求的配置后，点击**确定**关闭**编辑重启动请求**对话框。
