### 概述

以下整体单元输出量的值可在 Abaqus/Standard 时谐涡流分析中生成，并可读取到后续 Abaqus/Standard 分析中作为顺序耦合多物理场工作流的点载荷：
- Joule 热耗散率
- 磁体力强度

当仅在时谐涡流分析中耦合很重要时（加载对结构机械或热响应的影响不足以影响原始时谐涡流分析的有效性），可以使用顺序耦合多物理场分析将电磁产生的载荷（从时谐涡流分析）应用到热传递、耦合温度-位移或应力/位移分析中。

### 保存 Joule 热耗散或磁体力强度用于后续分析

您可以在时谐涡流分析中请求 Joule 热耗散输出（EMJH）或磁体力强度输出（EMBF）。只有存储在输出数据库（`.odb`）文件中的值可用于顺序耦合。

### 转换结果以供后续使用

整体单元量使用 **abaqus emloads** 实用程序转换为节点载荷量。该实用程序将 Joule 热耗散输出转换为集中热通量，将磁体力强度输出转换为点载荷。此实用程序还支持在不同网格之间转换结果。更多信息，请参见 ["热载荷和磁载荷的映射，" 第 3.2.25 节"](pt01ch03s02abx25.md)。

#### 转换限制

当在不同网格之间转换结果值时，只要热传递、耦合温度-位移或应力/位移分析中的模型域与时谐涡流分析中的模型域匹配，就能确保净通量的全局守恒。**abaqus emloads** 实用程序中使用的保守映射算法在时谐涡流分析中的网格比"目标"代表网格更细的情况下，还能提供点通量值（体载荷或集中热通量）的局部平滑分布。在这种情况下，如果"目标"代表网格更细或与时谐涡流分析中的网格大小相似，则可能会在节点位置观察到零转换通量值。在这些情况下，您仍会观察到通量的全局守恒，但您的解可能在局部受到不利影响。您可以通过始终使用更细的网格进行时谐涡流分析来纠正这些情况。

### 从输出数据库传递节点载荷到集中载荷

要为热传递、耦合温度-位移或应力/位移分析定义载荷，您可以从 **abaqus emloads** 实用程序创建的输出数据库（`.odb`）文件读取节点集中热通量和点载荷。

### 输入文件模板

在此示例中，热通量值从时谐涡流分析存储在输出数据库中。这些值在转换为点热通量后，作为集中通量读取到后续分析中。

以下模板显示时谐涡流分析 ``electromagnetic.inp`` 的输入：

```
[*HEADING](../key/key-link.md#usb-kws-mheading)
…
[*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=EMC3D8
*（选择电磁单元类型）*
…
[*STEP](../key/key-link.md#usb-kws-hstep)
[*ELECTROMAGNETIC](../key/key-link.md#usb-kws-helectromagnetic), LOW FREQUENCY, TIME HARMONIC
…
*施加载荷和边界条件*
…
** 将单元 Joule 热耗散结果写入输出
** 数据库文件 `electromagnetic.odb`
[*OUTPUT](../key/key-link.md#usb-kws-houtput), FIELD
[*ELEMENT OUTPUT](../key/key-link.md#usb-kws-helementoutput), ELSET=CONDUCTOR
 EMJH
[*END STEP](../key/key-link.md#usb-kws-hendstep)
```

以下模板显示热传递分析 `heattransfer.inp` 的输入，该分析引用由 **abaqus emloads** 实用程序创建的输出数据库 `pointflux.odb`，其中包含从存储在 `electromagnetic.odb` 中的时谐涡流分析结果映射的量：

```
[*HEADING](../key/key-link.md#usb-kws-mheading)
…
[*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=DC3D8
*（选择热传递连续体单元类型）*
…
[*STEP](../key/key-link.md#usb-kws-hstep)
[*HEAT TRANSFER](../key/key-link.md#usb-kws-hheattrans), STEADY STATE
…
*施加热传递载荷和边界条件*
…
[*CFLUX](../key/key-link.md#usb-kws-hcflux), FILE=`pointflux.odb`
*从输出数据库读取所有节点热通量值并作为集中节点通量施加*
…
[*END STEP](../key/key-link.md#usb-kws-hendstep)
```
