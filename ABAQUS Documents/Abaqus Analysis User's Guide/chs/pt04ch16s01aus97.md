### 概述

在 Abaqus/Standard 时谐涡流分析中生成的以下整个单元输出量的值可以读入后续的 Abaqus/Standard 分析，作为序贯耦合多物理场工作流的点载荷：
- Joule 热耗散率
- 磁体力强度

序贯耦合多物理场分析可用于在热传递、耦合温度-位移或应力/位移分析中施加电磁生成的载荷（来自时谐涡流分析）。在许多情况下，耦合仅在时谐涡流分析中是重要的；载荷对结构机械或热响应的影响不足以影响原始时谐涡流分析的有效性。

### 保存 Joule 热耗散或磁体力强度以供后续分析使用

您可以在时谐涡流分析中请求 Joule 热耗散输出（EMJH）或磁体力强度输出（EMBF）。只有存储在输出数据库（`.odb`）文件中的值可用于序贯耦合。

### 转换结果以供后续使用

整个单元量使用 **abaqus emloads** 实用程序转换为节点载荷量。该实用程序将 Joule 热耗散输出转换为集中热通量，将磁体力强度输出转换为点载荷。此实用程序还支持在不同网格之间转换结果。有关更多信息，请参阅 ["映射热载荷和磁载荷," 第 3.2.25 节](pt01ch03s02abx25.md)。

#### 转换限制

当在不同网格之间转换结果值时，确保热传递、耦合温度-位移或应力/位移分析中的模型域与时谐涡流分析中的模型域匹配时，将确保净通量的全局守恒。**abaqus emloads** 实用程序中使用的保守映射算法还在时谐涡流分析中的网格比"目标"代表网格更细的情况下提供点通量值（体载荷或集中热通量）的局部平滑分布。在这种情况下，当"目标"代表网格更细或与时谐涡流分析中的网格大小相似时，您可能会观察到具有零转换通量值的节点位置。在这些情况下，您仍会观察到通量的全局守恒，但您的解决方案可能在局部受到不利影响。您可以通过始终使用更细的网格进行时谐涡流分析来纠正这些情况。

### 将节点载荷从输出数据库传输到集中载荷

要在热传递、耦合温度-位移或应力/位移分析中定义载荷，您可以从 **abaqus emloads** 实用程序创建的输出数据库（`.odb`）文件读取节点集中热通量和点载荷。

### 输入文件模板

在此示例中，热通量值存储在时谐涡流分析的输出数据库中。这些值在转换为点热通量后，作为集中通量读入后续分析。

以下模板显示时谐涡流分析 ``electromagnetic.inp`` 的输入：

```
[*HEADING](../key/key-link.md#usb-kws-mheading)
…
[*ELEMENT](../key/link.md#usb-kws-melement), TYPE=EMC3D8
*(Choose the electromagnetic element type)*
…
[*STEP](../key/key-link.md#usb-kws-hstep)
[*ELECTROMAGNETIC](../key/key-link.md#usb-kws-helectromagnetic), LOW FREQUENCY, TIME HARMONIC
…
*Apply loads and boundary conditions*
…
** Write element Joule heat dissipation results to the output
** database file, `electromagnetic.odb`
[*OUTPUT](../key/key-link.md#usb-kws-houtput), FIELD
[*ELEMENT OUTPUT](../key/key-link.md#usb-kws-helementoutput), ELSET=CONDUCTOR
 EMJH
[*END STEP](../key/key-link.md#usb-kws-hendstep)
```

以下模板显示热传递分析 `heattransfer.inp` 的输入，该分析引用由 **abaqus emloads** 实用程序创建的输出数据库 `pointflux.odb`，其中包含从时谐涡流分析结果映射的量，存储在 `electromagnetic.odb` 中：

```
[*HEADING](../key/key-link.md#usb-kws-mheading)
…
[*ELEMENT](../key/key-link.md#usb-kws-melement), TYPE=DC3D8
*(Choose the heat transfer continuum element type)*
…
[*STEP](../key/key-link.md#usb-kws-hstep)
[*HEAT TRANSFER](../key/key-link.md#usb-kws-hheattrans), STEADY STATE
…
*Apply heat transfer loads and boundary conditions*
…
[*CFLUX](../key/key-link.md#usb-kws-hcflux), FILE=`pointflux.odb`
*Read in all nodal heat flux values from the output database and apply as concentrated nodal fluxes* 
…
[*END STEP](../key/key-link.md#usb-kws-hendstep)
```
