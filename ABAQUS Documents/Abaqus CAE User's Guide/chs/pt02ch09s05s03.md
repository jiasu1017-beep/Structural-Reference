# 9.5.3 重新创建未保存的模型数据库





当您启动新会话并更改模型时，Abaqus/CAE会将这些更改记录到模型数据库恢复文件（`abaqus*n*.rec`）中。如果您随后保存模型数据库，Abaqus/CAE会将恢复文件中的命令追加到该模型数据库的日志文件（`*model_database_name*.jnl`）中，并删除恢复文件。如果您继续对模型进行更改，Abaqus/CAE会创建一个新的恢复文件（`*model_database_name*.rec`）来记录自上次保存以来的更改。在下次保存时，恢复文件中的命令会被追加到日志文件中，恢复文件会被删除。日志文件包含重建整个模型数据库所需的所有命令。例如，[表9-1](pt02ch09s05s03.md#usi-dbs-files-recover-table)显示了Abaqus/CAE对名为`engine`的模型的模型数据库、恢复和日志文件所做的更改。

**表9-1** 建模更改及其对模型数据库、恢复和日志文件的影响。
| 用户操作 | Abaqus/CAE操作 | 文件 |
| --- | --- | --- |
| 启动Abaqus/CAE会话 | 无 | 无 |
| 进行模型更改 | 在恢复文件中记录命令 | `abaqus1.rec` |
| 保存模型数据库 | 创建模型数据库文件将恢复命令复制到日志文件删除恢复文件 | `engine.cae``engine.jnl` |
| 对模型进行更多更改 | 在恢复文件中记录命令 | `engine.rec``engine.cae`（过时）`engine.jnl`（过时） |
| 保存模型数据库 | 更新模型数据库文件将恢复命令追加到日志文件删除恢复文件 | `engine.cae`（已更新）`engine.jnl`（已更新） |

如果您的Abaqus/CAE会话意外中止——例如，由于计算机断电——恢复文件仍将在下次会话中供Abaqus/CAE使用。Abaqus/CAE首先检查是否存在形式为`abaqus*n*.rec`的恢复文件；如果存在这样的文件，它可能来自之前意外中止的会话，也可能来自您在同一目录中启动的其他Abaqus/CAE会话。因为Abaqus/CAE无法区分这两种情况，也无法自动确定您是否要实现这些更改，所以Abaqus/CAE会提示您三个选项：恢复更改并删除恢复文件、不恢复更改并删除恢复文件，或忽略恢复文件，因为其更改属于另一个Abaqus/CAE会话。当您恢复更改时，如果您认为最后发出的命令导致会话终止，则可以跳过恢复文件中的最后一个命令。

如果恢复文件属于模型数据库（`*model_database_name*.rec`），Abaqus/CAE不会检测到该恢复文件，直到您尝试打开该模型数据库。当您尝试打开模型数据库时，Abaqus/CAE会提示您恢复或忽略更改。如果您恢复更改，Abaqus/CAE会将数据库恢复文件中的更改追加到日志文件中，并删除数据库恢复文件；如果您选择忽略更改，Abaqus/CAE会删除恢复文件，并且不会实现文件中描述的任何模型更改。
