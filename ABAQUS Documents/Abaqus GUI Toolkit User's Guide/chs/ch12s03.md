# 12.3 在 Abaqus/Standard 中定义接触





在 Abaqus/Standard 中定义接触对的第一步是使用 [*SURFACE](../key/key-link.md#usb-kws-msurface) 选项创建表面。下一步是使用 [*CONTACT PAIR](../key/key-link.md#usb-kws-hcontactpair) 选项指定相互作用的表面。每个接触相互作用都引用一个表面相互作用定义，该定义使用 [*SURFACE INTERACTION](../key/key-link.md#usb-kws-hsurfaceinteraction) 选项创建。可以为表面相互作用定义分配接触压力-间隙关系和摩擦属性。

对于通用接触，表面定义是可选的，因为当使用 [*CONTACT](../key/key-link.md#usb-kws-hcontact) 选项时会自动创建一个全包容的基于单元的表面。然而，可以使用特定的表面配对来包括默认表面未包括的区域（[*CONTACT INCLUSIONS](../key/key-link.md#usb-kws-hcontactinclusions)），排除模型不同区域之间的相互作用（[*CONTACT EXCLUSIONS](../key/key-link.md#usb-kws-hcontactexclusions)），或覆盖全局接触属性分配。例如，如果要将某个摩擦系数应用于模型中除少数表面外的所有表面，可以分配全局摩擦系数，然后使用 [*CONTACT PROPERTY ASSIGNMENT](../key/key-link.md#usb-kws-hcontpropassign) 选项覆盖给定用户定义表面对的此属性。

### 12.3.1 定义表面

使用 [*SURFACE](../key-key.md#usb-kws-msurface) 选项通过识别形成表面的所有单元面来创建表面。这与定义分布式压力荷载的方式非常相似。

**连续体单元上的表面**

二维、一阶连续体单元（如 CPE4）有四个面，分别由节点 1-2、2-3、3-4 和 4-1 定义的段组成，如图 12-4 所示。面标识符由字母"S"后跟面号组成。例如，使用以下选项块将如图 12-4 所示的单元的面 2 包含在名为 `FLANGE1` 的表面中：

```
*SURFACE, NAME=FLANGE1
5, S2
```

**图 12-4** 二维、一阶（4 节点）单元上的面号。

![](../graphics/gss-four-node-nls.png)

与 Abaqus 中的许多选项一样，单元编号和单元集都可以使用；使用单元集可以使大表面的定义更容易。在同一 [*SURFACE](../key/key-link.md#usb-kws-msurface) 选项块中同时指定单元集和单个单元是有效的，如图 12-5 和下面的示例所示。表面 `TOPSURF` 由图 12-5 所示的单元面组成，创建如下：

```
*ELSET, ELSET=TOP, GENERATE
5, 8
*SURFACE, NAME=TOPSURF
TOP, S3
5, S4
8, S2
```

**图 12-5** 形成表面 `TOPSURF` 的面号和单元。

![](../graphics/gss-topsurf.png)

Abaqus 可以自动确定二维和三维连续体单元的自由面并使用它们创建表面。要使用此功能，只需将构成表面的所有单元的自由面包含在 [*SURFACE](../key/key-link.md#usb-kws-msurface) 选项的数据行上。可以使用单元集或单个单元编号。如果包含内部单元，Abaqus 将忽略它们。例如，图 12-5 所示的表面也可以使用以下方式定义

```
*SURFACE, NAME=TOPSURF
TOP,
```

**壳单元、膜单元和刚性单元上的表面**

对于壳单元、膜单元和刚性单元，必须指定构成接触表面的单元侧。正单元法线方向的一侧称为 SPOS，而负单元法线方向的一侧称为 SNEG，如图 12-6 所示。如第 5 章"使用壳单元"中所讨论的，单元的连接性定义了正的单元法线。可以在 Abaqus/Viewer 中查看正的单元法线。

**图 12-6** 二维壳单元或刚性单元上的表面。

![](../graphics/gss-surface-nls.png)

以下选项块将表面 `SURF1` 定义为由单元集 `SHELLS` 中所有单元的 SPOS 面组成的表面：

```
*SURFACE, NAME=SURF1
SHELLS, SPOS
```

关于在 Abaqus 中可以创建的表面类型的限制，请参阅 ["表面定义，" Abaqus 分析用户指南第 2.3 节](../usb/usb-link.md#usbdefsurfs)；在开始接触模拟之前请阅读这些内容。

**刚性表面**

刚性表面是刚性体的表面。可以定义为解析形状，也可以基于与刚性体关联的单元的底层表面。

解析刚性表面通过定义一系列连接的线段、圆弧和抛物线来创建。[*RIGID BODY](../key/key-link.md#usb-kws-mrigidbody) 选项上的 ANALYTICAL SURFACE 参数将解析刚性表面（使用 [*SURFACE](../key/key-link.md#usb-kws-msurface) 选项上的 TYPE 参数定义）与刚性体绑定。必须在模型定义中定义 [*RIGID BODY](../key/key-link.md#usb-kws-mrigidbody) 选项。[*SURFACE](../key/key-link.md#usb-kws-msurface) 选项上的 TYPE 参数定义表面的维数，它有三个可能值：
- 使用 TYPE=SEGMENTS 定义二维解析刚性表面。
- 使用 TYPE=CYLINDRICAL 定义在平面外方向无限延伸的三维解析刚性表面。
- 使用 TYPE=REVOLUTION 定义旋转对称的三维解析刚性表面。

以下是图 12-7 所示的名为 `SRIGID` 的二维解析刚性表面的示例输入：

```
*SURFACE, TYPE=SEGMENTS, NAME=SRIGID
START, 5.0, 0.0
LINE, 10.0, 0.0
CIRCL, 15.0, 5.0, 10.0, 5.0
```
 其中刚体定义为
```
*RIGID BODY, ANALYTICAL SURFACE=SRIGID, REF NODE=10000
```

**图 12-7** 解析刚性表面。

![](../graphics/gss-analrigidsurf-nls.png)

离散化刚性表面基于构成刚性体的底层单元；因此，它们可以比解析刚性表面具有更复杂的几何形状。离散化刚性表面使用与可变形体表面完全相同的方式使用 [*SURFACE](../key/key-link.md#usb-kws-msurface) 选项定义。

### 12.3.2 接触相互作用

使用接触对方法，您通过在 [*CONTACT PAIR](../key/key-link.md#usb-kws-hcontactpair) 选项上给出表面名称来定义 Abaqus/Standard 模拟中两个表面之间可能的接触。定义接触对时，必须确定相对滑动的大小是小还是有限。默认是更通用的有限滑动公式。如果两个表面的相对运动小于单元面特征长度的一小部分，则小滑动公式是合适的。通过在 [*CONTACT PAIR](../key/key-link.md#usb-kws-hcontactpair) 选项上包含 SMALL SLIDING 参数来选择小滑动公式。使用小滑动公式将产生更高效的分析。

每个接触对必须引用一个表面相互作用定义，这与每个单元必须引用单元属性定义的方式非常相似。使用 [*CONTACT PAIR](../key/key-link.md#usb-kws-hcontactpair) 选项上的 INTERACTION 参数来引用 [*SURFACE INTERACTION](../key/key-link.md#usb-kws-hsurfaceinteraction) 选项，在该选项中可以定义不同的表面相互作用模型，如 [*FRICTION](../key/key-link.md#usb-kws-hfriction)。

以下示例：

```
*CONTACT PAIR, INTERACTION=FRIC, SMALL SLIDING
FLANGE1, FLANGE2
*SURFACE INTERACTION, NAME=FRIC
*FRICTION
0.1,
```
指定表面 `FLANGE1` 和 `FLANGE2` 可能相互相互作用，并且发生的相对滑动量将被视为很小。表面之间的相互作用包括摩擦，摩擦系数为 0.1。

使用通用接触方法，您不需要显式定义和配对各个表面。通过使用 [*CONTACT](../key/key-link.md#usb-kws-hcontact) 选项及其相关子选项，Abaqus/Standard 自动定义一个"全包容"基于单元的表面，并强制该表面成员之间的接触。因此，接触定义大大简化。以下示例：

```
*CONTACT
*CONTACT INCLUSIONS, ALL EXTERIOR
*CONTACT PROPERTY ASSIGNMENT
 , , FRIC,
*SURFACE INTERACTION, NAME=FRIC
*FRICTION
0.1
```
指定给定模型中所有外部面可能相互相互作用。所有表面之间的相互作用包括摩擦，摩擦系数为 0.1。

### 12.3.3 从属表面和主表面

默认情况下，Abaqus/Standard 中的接触对使用纯主-从接触算法：一个表面上的节点（从属）不能穿透组成另一个表面（主）的段，如图 12-8 所示。该算法对主表面没有限制；它可以在从属节点之间穿透从属表面。如图 12-8 所示，[*CONTACT PAIR](../key/key-link.md#usb-kws-hcontactpair) 选项上给出的两个表面的顺序决定了哪个表面是主表面，哪个是从属表面；第一个表面是从属表面，第二个是主表面。

**图 12-8** 主表面可以穿透从属表面。

![](../graphics/gss-master-surf-nls.png)

由于严格的主-从公式，您必须小心正确选择从属和主表面，以获得最佳可能的接触模拟。以下是一些简单的规则：
- 从属表面应该是网格更细密的表面；并且
- 如果网格密度相似，从属表面应该是底层材料更软的表面。

Abaqus/Standard 中的通用接触算法在相互作用的表面之间以平均方式强制接触；Abaqus/Standard 自动分配主从角色。

### 12.3.4 接触离散化

Abaqus/Standard 提供两种接触离散化方法：传统的节点-表面方法和表面-表面方法。节点-表面离散化方法定义每个从属节点与主表面之间的接触条件。表面-表面离散化方法在定义接触约束时考虑主表面和从属表面的形状。接触对算法可以使用任一种离散化方法；通用接触仅使用表面-表面方法。

### 12.3.5 小滑动和有限滑动

当使用小滑动公式时，Abaqus/Standard 在模拟开始时建立从属节点与主表面之间的关系。Abaqus/Standard 确定主表面上的哪个段将与从属表面上的每个节点相互作用。它在整个分析过程中维护这些关系，从不更改哪个主表面段与哪个从属节点相互作用。如果通过在 [*STEP](../key/key-link.md#usb-kws-hstep) 选项上设置 NLGEOM 参数等于 YES 来包含几何非线性，则小滑动算法会考虑主表面的任何旋转和变形，并更新传递接触力的荷载路径。如果模型中未包含几何非线性，则主表面的任何旋转或变形都会被忽略，荷载路径保持固定。

有限滑动接触公式要求 Abaqus/Standard 持续跟踪主表面的哪部分与每个从属节点接触。这是一种非常复杂的计算，特别是当两个接触体都是可变形体时。这种模拟中的结构可以是二维或三维的。Abaqus/Standard 还可以模拟可变形体的有限滑动自接触。当结构折叠到自身上时，就会发生这种情况。

可变形体与刚性表面之间的有限滑动公式不像两个可变形体之间的有限滑动公式那么复杂。主表面为刚性的有限滑动模拟可以为二维和三维模型执行。

接触对算法可以考虑小滑动或有限滑动效应；通用接触仅考虑有限滑动效应。

### 12.3.6 单元选择

接触单元的选择很大程度上取决于所使用的接触强制。例如，对于传统接触公式（即节点-表面离散化），通常最好对将形成从属表面的模型部分使用一阶单元。二阶单元有时可能会导致问题，因为这些单元计算恒定压力的相容节点荷载方式。恒定压力 *P* 在面积为 *A* 的二维二阶单元上的相容节点荷载如图 12-9 所示。

**图 12-9** 二维二阶单元上恒定压力的等效节点荷载。

![](../graphics/gss-nodloadconstpress-nls.png)

节点-表面接触公式基于作用在从属节点上的力做出重要决策。算法很难判断图 12-9 所示的等效节点荷载分布是表示恒定接触压力还是实际跨单元的变化。三维二阶块单元的等效节点力更加令人困惑，因为对于恒定压力，它们甚至没有相同的符号，使得算法很难正确工作，特别是对于不均匀接触。因此，为避免这些问题，当二阶三维块或楔形单元的面与节点-表面公式一起用于定义从属表面时，Abaqus/Standard 会自动向该面添加中面节点。具有中面节点的二阶单元面的等效节点荷载对于恒定压力具有相同的符号，尽管它们的大小仍然存在相当大的差异。

对于一阶单元，施加压力的等效节点力始终具有一致的符号和大小；因此，给定的节点力分布所代表的接触状态没有歧义。

如果您使用的是节点-表面公式且几何复杂需要使用自动网格生成器，则应使用 Abaqus/Standard 中的改进二阶四面体单元（C3D10M）。这些单元设计用于复杂接触模拟；常规二阶四面体单元（C3D10）在其角节点处的接触力为零，导致接触压力预测较差。改进的二阶四面体单元可以准确地计算接触压力。

常规二阶单元通常可以与表面-表面公式一起使用而不会有困难。

### 12.3.7 接触算法

了解 Abaqus/Standard 用于求解接触问题的算法将帮助您理解消息文件中的诊断输出并成功进行接触模拟。

如图 12-10 所示，Abaqus/Standard 中的接触算法建立在第 8 章"非线性"中讨论的牛顿-拉夫森技术之上。

**图 12-10** Abaqus/Standard 中的接触算法。

![](../graphics/gss-contact-alg-nls.png)

Abaqus/Standard 在每个增量开始时检查所有接触相互作用的状态，以确定从属节点是开放还是闭合。如果节点闭合，Abaqus/Standard 确定它是滑动还是粘着。Abaqus/Standard 为每个闭合节点施加约束，并从接触状态从闭合变为开放的任何节点移除约束。然后 Abaqus/Standard 执行迭代，并使用计算出的修正更新模型的配置。

在更新后的配置中，Abaqus/Standard 检查从属节点处接触条件的变化。任何在迭代后间隙变为负或零的节点的状态已从开放变为闭合。任何接触压力变为负的节点的状态已从闭合变为开放。如果在任何迭代中检测到接触变化，Abaqus/Standard 将其标记为*严重不连续迭代*。

Abaqus/Standard 继续迭代，直到严重不连续足够小（或不发生严重不连续）并且平衡（通量）容差满足。或者，您可以选择不同的方法，即 Abaqus/Standard 将继续迭代，直到不发生严重不连续，然后再检查平衡。

消息和状态文件中每个完成的增量的摘要显示有多少迭代是严重不连续迭代，有多少是平衡迭代（平衡迭代是不发生严重不连续的那些迭代）。增量的迭代总数是这两者的和。对于某些增量，您可能会发现所有迭代都标记为严重不连续迭代（当在每次迭代中检测到小的接触变化且最终满足平衡时会发生这种情况）。

Abaqus/Standard 应用涉及穿透变化、残余力变化以及从前一次迭代到下一次迭代的严重不连续数量的复杂标准来确定是否应继续或终止迭代。因此，原则上不需要限制严重不连续迭代的最大数量。默认的严重不连续迭代最大限制为 50，实际上应该始终大于增量中的实际迭代次数。这使得运行需要大量接触变化的接触问题成为可能，而无需更改控制参数。
