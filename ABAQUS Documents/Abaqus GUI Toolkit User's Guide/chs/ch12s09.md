# 12.9 Abaqus/Explicit 中的建模考虑





我们现在讨论以下建模考虑：正确的表面定义、过约束、网格细化以及初始过盈闭合。

### 12.9.1 正确的表面定义

为每个接触算法定义表面时必须遵循某些规则。通用接触算法对可参与接触的表面类型限制较少；但是，二维和基于节点的表面只能与接触对算法一起使用。

**连续表面**

通用接触算法使用的表面可以跨越多个未连接的体。多个表面面可以共享一条公共边。相反，接触对算法使用的所有表面必须是连续且单连通的。连续性要求对构成接触对算法有效或无效的表面定义有以下含义：
- 在二维中，表面必须是具有两个终端的简单非相交曲线或闭合环。图 12-37 显示了有效和无效二维表面的示例。**图 12-37** 接触对算法的有效和无效二维表面。![](../graphics/gsx-val-inval-2d-nls.png)
- 在三维中，属于有效表面的单元面的边缘可以在表面的周边上或与其他面共享。两个形成接触表面的单元面不能仅在共享节点处连接；它们必须沿公共单元边缘连接。单元边缘不能由两个以上的表面面共享。图 12-38 说明了有效和无效的三维表面。**图 12-38** 接触对算法的有效和无效三维表面。![](../graphics/gsx-val-inval-3d-nls.png)
- 此外，可以定义三维双面表面。在这种情况下，壳单元、膜单元或刚性单元的两侧都包含在同一表面定义中（如图 12-39 所示）。**图 12-39** 有效的双面表面。![](../graphics/gsx-double-side-nls.png)

**扩展表面**

Abaqus/Explicit 不会自动将表面扩展到用户定义的表面周边之外。如果一个表面上的节点与另一表面接触，并且沿表面滑动直到到达边缘，它可能"掉落边缘"。这种行为可能特别麻烦，因为节点可能稍后从表面的后侧重新进入，从而违反运动约束并导致该节点处的加速度大幅跳跃。因此，良好的建模做法是将表面稍微扩展到实际接触的区域之外。一般来说，我们建议完全覆盖每个接触体；额外的计算成本很小。

图 12-40 显示了由块单元构建的两个简单的盒状体。

**图 12-40** 表面周边。

![](../graphics/gsx-surf-perim-nls.png)

上面的盒子只在盒子顶面定义了接触表面。虽然这是 Abaqus/Explicit 中允许的表面定义，但缺乏超出"原始边缘"的扩展可能会出现问题。在下面的盒子中，表面缠绕在侧壁上一些距离，从而延伸到平坦的上表面之外。如果接触仅发生在盒子顶面，这种扩展的表面定义通过使任何接触节点都不会进入接触表面后面来最小化接触问题。

**一致的表面法线**

壳单元、膜单元或刚性单元上的单面表面必须定义得使法线方向在表面遍历时不会"翻转"。图 12-44 显示了法线从一个单元到下一个单元不连续的 SAX1 单元网格。面标识符 SPOS 表示表面在具有正外向法的面上，而面标识符 SNEG 表示相反。如果使用所有单元的 SPOS 面定义表面，Abaqus/Explicit 将发出警告消息，说明表面无效。

**图 12-44** 不一致的表面法线。

![](../graphics/gsx-inconsist-norms-nls.png)

如果使用如图所示的表面定义（使用 SPOS 和 SNEG 面标识符来适应不一致的单元法线），则可以使用此网格定义有效的表面。

对于双面表面，所有底层壳单元、膜单元或刚性单元的法线不需要具有一致的正方向；如果可能，Abaqus/Explicit 将定义其面具有一致法线的表面，即使底层单元没有一致的法线。如果单元法线都是一致的，则面法线将与单元法线相同；否则，为表面选择任意正方向。如果无法使面法线一致（例如，如果表面包含壳的 T 形交点），则该表面可以与通用接触算法一起使用，但不能与接触对算法一起使用。

**高度翘曲的表面**

通用接触算法不需要对翘曲表面进行特殊处理。但是，当与接触对算法一起使用的表面包含高度翘曲的面时，必须使用比表面不包含高度翘曲时更昂贵的跟踪方法。为使解尽可能高效，Abaus 会监控表面的翘曲，并在表面变得过于翘曲时发出警告；如果相邻面的法线方向相差超过 20°，Abaqus 会发出警告消息。一旦表面被认为是高度翘曲的，Abaqus 会从更高效的接触搜索方法切换到更准确的搜索方法，以处理高度翘曲表面带来的困难。

为效率起见，Abaqus 不会在每个增量都检查高度翘曲的表面。刚性表面仅在步骤开始时检查高翘曲，因为刚性表面在分析过程中不会改变形状。可变形表面默认每 20 个增量检查一次高翘曲。一些分析可能具有突然变得严重的表面翘曲，使得默认的每 20 增量检查频率不足。用户可以通过在 [*CONTACT CONTROLS](../key/key-link.md#usb-kws-hcontactcontrols) 选项上设置 WARP CHECK PERIOD 参数来更改翘曲检查的频率。一些分析中表面翘曲可能小于 20°，但也可能需要与高度翘曲表面相关的更准确的接触搜索方法。使用 [*CONTACT CONTROLS](../key/key-link.md#usb-kws-hcontactcontrols) 选项上的 WARP CUT OFF 参数来重新定义定义高翘曲的角度。

**刚性单元离散化**

复杂的刚性表面几何形状可以使用刚性单元建模。Abaqus/Explicit 中的刚性单元不会被平滑处理；它们按照用户的定义保持为面。因为用户定义的表面与 Abaqus/Explicit 使用的表面完全相同，所以面状表面的缺点是需要更高的网格细化来准确描述光滑体。然而，使用大量刚性单元来定义刚性表面不会显著增加 CPU 成本。但是，大量刚性单元确实会增加显著的内存开销。

用户必须确保刚性体上任何曲线几何形状的离散化是足够的。如果刚性体离散化太粗糙，则可变形体上的接触节点可能"挂住"，导致错误结果（如图 12-45 所示）。

**图 12-45** 粗刚性体离散化的潜在影响。

![](../graphics/gsx-potential-effect-nls.png)

挂在尖角上的节点可能在沿刚性表面继续滑动之前被困住一段时间。一旦有足够的能量释放以滑过尖角，节点将动态地捕捉，然后与相邻面接触。这种运动会导致嘈杂的解。刚性表面越细化，接触从属节点的运动就越平滑。通用接触算法包括一些数值特征舍入，可以防止节点挂住成为离散刚性表面的问题。此外，接触约束的罚函数强制减少了挂住发生的趋势。解析刚性表面通常应与刚性体的接触对算法一起使用，前提是形状是挤压轮廓或旋转表面。

### 12.9.2 模型过约束

正如不应在同一节点处定义多个冲突的边界条件一样，在同一节点处强制执行的多点约束和接触条件（使用运动方法强制执行）通常不应定义，因为它们可能产生冲突的运动约束。除非约束完全相互正交，否则模型将过约束；结果解将相当嘈杂，因为 Abaqus/Explicit 试图满足冲突的约束。在相同节点上起作用的罚函数接触约束和多点约束不会产生冲突，因为罚函数约束不像多点约束那样严格地强制执行。

### 12.9.3 网格细化

对于接触以及所有其他类型的分析，解随着网格的细化而改进。对于使用纯主-从方法的接触分析，特别重要的是从属表面要有足够的细化，以便主表面面不会过度穿透从属表面。平衡主-从方法不需要从属表面高度细化即可具有适当的接触一致性。网格细化通常在使用纯主-从接触的可变形体和刚性体之间最为重要；在这种情况下，可变形体始终是纯从属表面，因此必须细化到足以与刚性体上的任何特征相互作用。图 12-46 显示了如果从属表面的离散化与主表面上的特征尺寸相比很差时可能发生的穿透示例。如果可变形表面更细化，刚性表面的穿透将不那么严重。

**图 12-46** 从属表面离散化不足的示例。

![](../graphics/gsx-inadeq-surf-nls.png)

**绑定约束**

使用 [*TIE](../key/key-link.md#usb-kws-mtie) 选项可防止最初接触的表面穿透、分离或相对滑动。因此，绑定约束是网格细化的简单方法。由于两个接触表面之间存在的任何间隙（无论多小）都将导致节点未绑定到相对接触边界，因此必须使用 ADJUST=YES 参数来确保两个表面在分析开始时完全接触。

绑定约束公式约束平动和可选的转动自由度。当使用带结构单元的绑定接触时，必须确保任何无约束旋转不会引起问题。

### 12.9.4 初始接触过盈闭合

Abaqus/Explicit 将自动调整接触表面上节点的未变形坐标，以移除任何初始过盈闭合。当使用平衡主-从方法时，两个表面都会被调整；当使用纯主-从方法时，只有从属表面被调整。与在分析第一步中定义的接触相关的调整表面以移除过盈闭合所产生的位移不会引起任何初始应变或应力。当存在冲突约束时，初始过盈闭合可能无法通过重新定位节点完全解决。在这种情况下，当使用接触对算法时，会在分析开始附近产生严重的网格扭曲。通用接触算法将任何未解决的初始穿透存储为偏移量，以避免大的初始加速度。

在后续步骤中，任何调整节点以移除初始过盈闭合的操作都会产生应变，这些应变通常会在一个非常短暂的增量中引起严重的网格扭曲。当使用运动约束方法时尤其如此。例如，如果节点过盈闭合了 1.0×10⁻³ m，而增量时间为 1.0×10⁻⁷ s，则为校正过盈闭合而施加在节点上的加速度为 2.0×10¹¹ m/s²。如此大的加速度施加到单个节点通常会导致关于变形速度超过材料波速的警告，以及稍后关于严重网格扭曲的警告，因为大加速度已显著变形了相关单元。即使是非常轻微的初始过盈闭合也会对运动接触产生极大的加速度。一般来说，重要的是在第二步及以后，您定义的任何新接触表面都不会过盈闭合。
