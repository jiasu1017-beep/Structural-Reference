# 77.1.1 颜色编码概念






您可以通过对当前视口中可视图几何和网格单元进行颜色编码来区分模型或输出数据库中的组件。Abaqus/CAE 根据指定特定类型对象（如零件、截面分配、边界条件或显示组）的颜色映射（**颜色映射**）来应用颜色编码，这些映射指定了分配给每个项目的颜色。在[图77-1](pt07ch77s01hlb01.md#uss-cco-colorcode-basics)所示的示例中，颜色映射按零件进行，每个行描述了在此模型中三个零件定义之一分配的颜色。因此，**颜色编码** 对话框提供了一个图例，描述了当前视口中当前显示的所有颜色编码。

**图77-1** **颜色编码** 对话框。

![](../graphics/uss-cco-colorcode-basics-nls.png)

颜色编码分两层应用。所有几何和网格单元首先用 **初始颜色** 着色，这是一个可自定义设置，默认为灰色。（有关更多信息，请参见["更改初始颜色，" 第77.2节](pt07ch77s02.md)。）然后，Abaqus/CAE 根据您选择的颜色映射中的颜色和对象在初始颜色之上应用颜色编码。如果您应用诸如边界条件、载荷或集合之类的颜色映射（对于这些颜色映射，Abaqus/CAE 通常仅对模型中的不同点或表面进行颜色编码），区域将以初始颜色保持可见。

Abaqus/CAE 会自动为模型中的所有项目创建颜色映射，方法是将每个项目的名称与 **自动颜色列表** 中的颜色关联。颜色通过将 **自动颜色列表** 与项目名称的字母列表匹配来选择；因此，在[图77-1](pt07ch77s01hlb01.md#uss-cco-colorcode-basics)中的 **零件** 示例中，**Block** 用 **自动颜色列表** 中的第一种颜色进行颜色编码，**Floor** 用第二种颜色进行颜色编码，以此类推。颜色分配仅取决于项目名称；因此，在 **颜色编码** 对话框中重新排序这些项目不会更改颜色分配。

**注意：**如果某个区域由模型中的两个或更多项目共享，例如当皮肤截面分配给表面而实体截面分配给整个块时，公共区域将使用字母列表中名称出现较晚的项目的颜色定义进行着色。Abaqus/CAE 在视口中显示每个颜色分配时会覆盖第一个颜色映射颜色定义。

因为每个颜色映射是项目名称和颜色定义之间的链接集合，所以颜色映射在 Abaqus/CAE 的模块之间以及模型数据库与其输出数据库之间保持持久。但是，因为 Abaqus/CAE 依赖对象名称进行颜色编码，所以当您重命名对象时，您无法保留与该对象关联的颜色编码。相比之下，当您从模型中删除对象定义时，Abaqus/CAE 通常会删除对象的颜色定义；这两个例外是材料和截面定义，即使在您删除定义之后，它们的颜色编码在视口中仍然保留。要在删除材料或截面定义后刷新当前视口中的颜色编码，您必须应用颜色映射或切换到不同的模块。

Abaqus/CAE 还为每个模块提供默认颜色映射。例如，当您在网格模块中显示 **网格默认值** 颜色映射时，Abaqus/CAE 会根据可网格性对视口中的项目进行颜色编码。每个模块的默认颜色映射仅在该模块中可用；您不能使用 **网格默认值** 颜色映射中的分配在属性模块中对对象进行颜色编码。模块默认映射无法编辑，模块默认映射对应于 Abaqus/CAE 在未应用颜色编码时使用的默认颜色。

颜色映射是特定于视口的，在某些情况下在模块之间保持持久。模块之间颜色编码的持久性取决于您是否为当前模块显示了默认颜色映射：
- 如果为模块选择了默认颜色映射，Abaqus/CAE 在切换模块时会自动更改颜色映射。例如，如果您在装配模块中选择了 **装配默认值** 颜色映射，然后切换到网格模块，Abaqus/CAE 会自动应用 **网格默认值** 颜色映射。
- 如果选择了非默认颜色映射，Abaqus/CAE 在切换模块时会保留颜色映射。例如，如果您在装配模块中选择了 **零件实例** 颜色映射，然后切换到网格模块，Abaqus/CAE 继续按零件实例名称而不是可网格性进行颜色编码。

当您更改颜色映射时，Abaqus/CAE 仅刷新当前视口中的颜色编码，同时保留其他非活动可见视口中的任何颜色编码。当您向会话添加新视口时，新视口继承先前当前视口的颜色映射。

创建颜色映射后，您可以通过更改其任何单个对象分配的颜色来自定义任何颜色映射（模块默认值除外）。您还可以切换颜色映射中单个对象的活动状态，这控制是否在当前视口中对单个对象进行颜色编码。活动状态为非活动的对象的颜色编码使用初始颜色呈现。**颜色编码** 对话框还提供排序和过滤工具，您可以使用它们来显示颜色映射中对象的子集。当颜色映射有许多对象时，这些工具可以帮助您集中显示。
