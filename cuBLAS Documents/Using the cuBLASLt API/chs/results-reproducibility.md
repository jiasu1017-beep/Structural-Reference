### 2.1.4. 结果可重现性

按照设计，来自特定工具包版本的所有 cuBLAS API 例程，在具有相同架构和相同 SM 数量的 GPU 上执行时，每次运行都会生成相同的位级结果。但是，不能保证跨工具包版本的位级可重现性，因为实现可能会因某些实现更改而有所不同。

当多个 CUDA 流处于活动状态或使用定点仿真时，此保证不再成立。如果有多个并发流处于活动状态，库可能会通过选择不同的内部实现来优化总体性能。

> **注意**

注意
多流执行的非确定性行为是由于库在选择并行流中运行的例程的内部工作空间时进行了优化。为了避免此影响，用户可以：

1. 使用 `cublasSetWorkspace()` 函数为每个使用的流提供独立的工作空间，或
2. 每个流使用一个 cuBLAS 句柄，或
3. 使用 `cublasLtMatmul()` 代替 GEMM 系列函数，并提供用户自有工作空间，或
4. 设置调试环境变量 `CUBLAS_WORKSPACE_CONFIG` 为 `:16:8`（可能会限制整体性能）或 `:4096:8`（将使库在 GPU 内存中的占用量增加约 24MiB）。

定点仿真的非确定性行为是由于工作空间内存需求较大（详见定点工作空间要求）。这需要使用 `cudaMallocAsync()` 动态分配内存，分配失败会导致回退到非仿真例程。为了避免此影响，用户可以通过 `cublasSetWorkspace()` 提供工作空间以满足定点仿真工作空间要求。

任何这些设置都将允许即使有多个并发流共享单个 cuBLAS 句柄也能实现确定性行为。

此行为预计在未来的版本中会有变化。

对于某些例程（如 `cublas<t>symv()` 和 `cublas<t>hemv()`），可以使用 `cublasSetAtomicsMode()` 选择一种显著更快的替代例程。在这种情况下，由于使用原子操作进行计算，结果不能保证位级可重现性。
