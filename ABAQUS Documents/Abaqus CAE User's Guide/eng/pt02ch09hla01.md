# 9.6 Using the File menu







Use the items under **File** on the main menu bar to do the following:
- Select ****File**![](../graphics/images/arrow.gif)**New Model Database**![](../graphics/images/arrow.gif)**With Standard/Explicit Model**** to create a new model database for an Abaqus/Standard or an Abaqus/Explicit analysis. You can also click ![](../graphics/ico_fileNew.png) in the **File** toolbar. For more information, see ["Creating a new model database," Section 9.7.1](pt02ch09s06hlb01.md).
- Select ****File**![](../graphics/images/arrow.gif)**New Model Database**![](../graphics/images/arrow.gif)**With CFD Model**** to create a new model database for an Abaqus/CFD analysis. For more information, see ["Creating a new model database," Section 9.7.1](pt02ch09s06hlb01.md).
- Select ****File**![](../graphics/images/arrow.gif)**New Model Database**![](../graphics/images/arrow.gif)**With Electromagnetic Model**** to create a new model database for an electromagnetic analysis. For more information, see ["Creating a new model database," Section 9.7.1](pt02ch09s06hlb01.md).
- Select ****File**![](../graphics/images/arrow.gif)**Open**** to open an existing model database or output database. You can also click ![](../graphics/ico_fileOpen.png) in the **File** toolbar. For more information, see ["Opening a model database or an output database," Section 9.7.2](pt02ch09s06hlb02.md).
- Select ****File**![](../graphics/images/arrow.gif)**Network ODB Connector**** to create a connection with a remote host that you can use to read a remote output database. For more information, see ["Creating a network ODB connector," Section 9.7.4](pt02ch09s06hlb04.md).
- Select ****File**![](../graphics/images/arrow.gif)**Close ODB**** to close an output database. For more information, see ["Closing the current output database," Section 9.7.7](pt02ch09s06hlb07.md).
- Select ****File**![](../graphics/images/arrow.gif)**Set Work Directory**** to change the work directory. For more information, see ["Setting the work directory," Section 9.7.8](pt02ch09s06hlb08.md).
- Select ****File**![](../graphics/images/arrow.gif)**Save**** to save the current model database. You can also click ![](../graphics/ico_fileSave.png) in the **File** toolbar. For more information, see ["Saving the current model database," Section 9.7.9](pt02ch09s06hlb09.md).
- Select ****File**![](../graphics/images/arrow.gif)**Save As**** to save the current model database to a new file with a different name. For more information, see ["Saving the current model database to a new file with a different name," Section 9.7.11](pt02ch09s06hlb11.md).
- Select ****File**![](../graphics/images/arrow.gif)**Compress MDB**** to compress the current model database. For more information, see ["Compressing the file size of the current model database," Section 9.7.12](pt02ch09s06hlb12.md).
- Select ****File**![](../graphics/images/arrow.gif)**Save Display Options**** to save your customized part, assembly, and Visualization module display settings. For more information, see ["Understanding Abaqus/CAE GUI settings," Section 3.6](pt01ch03s06.md), and ["Saving your display options settings," Section 76.16](pt07ch76hla15.md).
- Select ****File**![](../graphics/images/arrow.gif)**Save Session Objects**** to save session-specific object definitions such as view cuts, display groups, or paths to a file, model database, or output database. For more information, see ["Managing session objects and session options," Section 9.9](pt02ch09s08.md).
- Select ****File**![](../graphics/images/arrow.gif)**Load Session Objects**** to load previously saved session-specific object definitions into the current session. For more information, see ["Managing session objects and session options," Section 9.9](pt02ch09s08.md).
- Select ****File**![](../graphics/images/arrow.gif)**Import**![](../graphics/images/arrow.gif)**Sketch**** to import a planar sketch. For more information, see ["Importing sketches," Section 10.7.1](pt02ch10s07hlb01.md).
- Select ****File**![](../graphics/images/arrow.gif)**Import**![](../graphics/images/arrow.gif)**Part**** to import a part. For more information, see ["Importing parts," Section 10.7.2](pt02ch10s07hlb02.md).
- Select ****File**![](../graphics/images/arrow.gif)**Import**![](../graphics/images/arrow.gif)**Model**** to import a model. For more information, see ["Importing a model," Section 10.8](pt02ch10hla01.md).
- Select ****File**![](../graphics/images/arrow.gif)**Export**![](../graphics/images/arrow.gif)**Sketch**** to export the current sketch. For more information, see ["Exporting a sketch to an ACIS-, IGES-, or STEP-format file," Section 10.9.1](pt02ch10s08hlb01.md).
- Select ****File**![](../graphics/images/arrow.gif)**Export**![](../graphics/images/arrow.gif)**Part**** to export the current part. For more information, see ["Exporting a part to an ACIS-, IGES-, STEP-, or VDA-format file," Section 10.9.2](pt02ch10s08hlb02.md).
- Select ****File**![](../graphics/images/arrow.gif)**Export**![](../graphics/images/arrow.gif)**Assembly**** to export the part instances in the assembly. For more information, see ["Exporting the assembly to an ACIS-format file," Section 10.9.3](pt02ch10s08hlb03.md).
- Select ****File**![](../graphics/images/arrow.gif)**Export**![](../graphics/images/arrow.gif)**VRML**** to export the current viewport to a VRML-format file. For more information, see ["Exporting viewport data to a VRML-format file," Section 10.9.4](pt02ch10s08hlb04.md).
- Select ****File**![](../graphics/images/arrow.gif)**Export**![](../graphics/images/arrow.gif)**3DXML**** to export the current viewport to a 3D XML-format file. For more information, see ["Exporting viewport data to a 3D XML-format file," Section 10.9.5](pt02ch10s08hlb05.md).
- Select ****File**![](../graphics/images/arrow.gif)**Export**![](../graphics/images/arrow.gif)**OBJ**** to export the current viewport to an OBJ-format file. For more information, see ["Exporting viewport data to an OBJ-format file," Section 10.9.6](pt02ch10s08hlb06.md).
- Select ****File**![](../graphics/images/arrow.gif)**Run Script**** to execute a file containing Abaqus Scripting Interface commands. For more information, see ["Replaying an Abaqus/CAE session," Section 9.5.1](pt02ch09s05s01.md), and ["Creating and running your own scripts," Section 9.5.4](pt02ch09s05s04.md).
- Select ****File**![](../graphics/images/arrow.gif)**Macro Manager**** to store your actions in a macro file as a sequence of Abaqus Scripting Interface commands. You can also run a macro and rename an existing macro. For more information, see ["Creating and running a macro," Section 9.5.5](pt02ch09s05s05.md).
- Select ****File**![](../graphics/images/arrow.gif)**Print**** to print all or selected viewports and annotations. You can also click ![](../graphics/ico_filePrint.png) in the **File** toolbar. For more information, see [Chapter 8, "Printing viewports](pt01ch08.md)."
- Select ****File**![](../graphics/images/arrow.gif)**Abaqus PDE**** to open the Abaqus Python development environment. The Abaqus PDE is a separate application used to create, edit, test, and debug scripts. For more information, see ["An overview of the Abaqus Python development environment," Section 7.1 of the Abaqus Scripting User's Guide](../cmd/cmd-link.md#cmd-pde-over).
- Select ****File**![](../graphics/images/arrow.gif)**Exit**** to exit the Abaqus/CAE session. For more information, see ["Exiting an Abaqus/CAE session," Section 2.1.2](pt01ch02s01s02.md).
